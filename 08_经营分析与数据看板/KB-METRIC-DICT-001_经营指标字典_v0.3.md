## 结论先行（这是什么 / 怎么用 / 边界）

- **这是什么**：一张把散落在 4 个方法论页 + 1 张花厅坊重判卡里的经营指标**收口成一处的速查字典**（索引层）。全库现有约 33 个常用经营指标分散在 RM/IM 工具体系、时间效率体系、7D 框架、客流人效方法论里，查一个指标要翻 5 处——本字典解决"在哪查"。v0.3 新增**五项候补**（§候补），来自 2026-07-05 业务模块（经营看板模板 v0.1 / 营销策划模板 v0.1）填肉实战暴露的缺口，**候补项签字前不入正表**。
- **怎么用**：按 5 组（①业绩规模 ②毛利效益 ③库存周转 ④时间/空间效率 ⑤客流客单连带）定位指标 → 看定义/公式/口径速记 → **落地方法一律回跳"来源 SSOT 页"**（本字典不重定义方法，只指路）。
- **边界（3 条铁律）**：① **本字典是索引层，不是权威定义层**——任何指标的口径以来源 SSOT 页为准，字典与源页冲突时以源页为准；② **锚点/基准数值只搬运源页已有值**，逐项标 `observed/inferred/pending` + 来源页，未造任何新数；③ 源页之间**口径打架的**，单列 §7「⚠️口径冲突待裁」如实列出，不粉饰、不替六哥裁决。本件为 draft 速查件，**非 stable、非铁律、非标准**。

## ① 业绩规模（销售额分解树·RM-01）

> 根公式：**销售额 = 客数 × 客单价 = (既存客×来频 + 新客) × (件数 × 件均价)**。全字典所有维度最终收敛到这条公式的某个变量。

| 指标 | 定义 | 公式 | 口径说明 | 锚点/基准 | 预警线 | 来源SSOT |
|---|---|---|---|---|---|---|
| 销售额 | 门店总营业额 | 客数 × 客单价 | 注意含烟/不含烟总盘差异（见⑦冲突①） | 花厅坊客单锚点 906.0万（含烟总销）／19分表合计 833万（不含烟）〔observed·重判卡〕 | — | [[KB-DX-MATH-001_零售数学诊断工具体系_v0.1]] RM-01 |
| 客数 | 成交顾客数 | 既存客×来频 + 新客 | 多数店无客流设备，用 POS 笔数代理，须标"非真实人次" | — | 客流降+客单价涨=危险信号（消耗老客） | 同上 RM-01 |
| 客单价 | 单笔平均消费额 | 销售额 ÷ 客数 | — | 花厅坊约 42元〔案例·MATH〕 | 调改后30天内降<15%不触发整改（保护期） | 同上 RM-01 |
| 1客点数（件数） | 每单购买件数 | 件数 ÷ 客数 | 反映关联购买强度 | 花厅坊约 4.2〔案例·MATH〕 | ↓=关联购买弱化 | 同上 RM-01 |
| 1点单价（件均价） | 每件平均单价 | 销售额 ÷ 件数 | ↓=高价品(M域)没卖动 | — | ↓→查M域SKU占比 | 同上 RM-01 |
| 来店频率（来频） | 既存客单位时间到店次数 | 到店次数 ÷ 会员数 | 需会员数据；无则用复购代理 | — | 生鲜日清不稳→来频崩塌 | [[KB-DX-TRAFFIC-001_客流人效诊断方法论_v0.1]] §四 |
| 新增客数 | 新客/新会员数 | 期内首单新客计数 | 领先指标=进店转化率/首单核销 | — | 有R3/R4硬伤时禁做拉新投放 | 同上 §四 P1 |

## ⑦ ⚠️ 口径冲突待裁（批判性陪练·不粉饰）

> 以下为收口时发现的**源页之间口径打架/未定项**，字典无权裁决，如实登记待六哥定。引用相关指标的 Agent 须先看本节。**本次 v0.2 归并未解决这 5 项，仍待裁。**（v0.3 承接原样保留，未动一字；是否随 v0.3 一并裁，见候补节末"待裁问题②"。）

**冲突① 动销率分母口径未定（双层未裁）**
- 层A·含烟 vs 不含烟：含烟≈57.5%，剔除烟601个SKU后≈60.0-60.3%，恰踩框架60%绿线〔重判卡§②，inferred·由已发布md推算，精确值需对落盘csv重跑〕。
- 层B·在售分母 vs 全库分母：任务指出此维度未定，**5 张源页均未裁定**——是否把停售/已下架SKU计入分母直接改变动销率数值。
- 连带影响：生鲜占比 f 同受含烟/不含烟分母影响（66.9% vs 73%），进而牵动 7D 的 D2 权重校准（f=70.74%→D2≈42.1）。**动销率、f、7D权重三者口径须同一次裁定，否则连环错**。

**冲突② "人效"多定义未统一**
- TRAFFIC「人时效率」= 销售额 ÷ 总工时（元/工时，全时段）。
- FULLSTORE「D4-4 时段人效」= 高峰单量 ÷ 在岗人时（笔数口径，仅高峰）。
- FULLSTORE「D6-3 人员成本效率」= 销售额 ÷ 人工总成本（成本产出比）。
- 三者分子/分母/时段窗口均不同，同挂"人效"标签易混用；7D 已刻意拆分以避 MECE 违规，但与 TRAFFIC 页命名未对齐。

**冲突③ 证据分级 A-E vs A-D 未统一**
- FULLSTORE 用 A-E 五级（按交叉验证深度），全店诊断卡用 A-D 四级（按证据来源类型），同一字母语义不同（POS单源在卡记A、在7D仅C级）。已列 FULLSTORE §9 待裁决项④，字典沿用其过渡语义映射，不自造。

**冲突④ 毛利率金额口径不可信（花厅坊 ERP 进价虚高）**
- D2-5/D5-1 明示生鲜 ERP 进价常虚高，毛利率仅参考，打分须用销售/动销口径；D5-2 周转天数因进价不准降 C 级，现金流 P0 须先取实际进货单据。凡引用"综合毛利率/生鲜毛利率"须带此口径警示。

**冲突⑤ 任务所称"坪效锚点 D2=42.11/D1=10.09"实为 7D 校准权重（非坪效）**

## 五项候补（2026-07-05·业务模块填肉实战暴露）

> **来源与状态**：经营看板模板 v0.1 与营销策划模板 v0.1 填肉实战中暴露的字典缺口。**候补项编号接续正表（34–38），签字前不入正表、不具 SSOT 效力**。锚点/阈值遵守正表铁律②：不编行业基准，无源即 pending。

### 34 · 日频报损金额（元/日）

| 维度 | 内容 |
|---|---|
| 定义 | 当日登记的报损商品金额合计，作为**损耗率（月频）的日频代理指标** |
| 公式 | Σ 当日报损单金额（元/日） |
| 数据源 | POS 报损单；无报损模块的店用手工登记表 |
| 频率 | 日 |
| 层级 | L0 老板日看板 |
| 阈值 | pending（待花厅坊实测，跑出日常波动带后再定预警线） |
| 失效边界 | ① 无报损纪律的店，日报损金额=登记意愿而非真实损耗，数值系统性偏低；② 日频波动大（进货日/雨天/临期集中处理日），单日值不可直接判好坏，须看 7 日滚动；③ 生鲜"隐性损耗"（降价甩卖折价部分）不进报损单，本指标只抓显性报损 |
| 备注 | 兼有**倒逼建损耗台账**的作用——多数店损耗率只能估算（见正表②损耗率"多数店无台账"），日频登记是台账从 0 到 1 的最小抓手（来源：经营看板模板 v0.1） |

### 35 · 固定成本（月）

| 维度 | 内容 |
|---|---|
| 定义 | 与销量无关的月度固定支出合计，**保本销售额（正表② D5-5）的分子构成项** |
| 公式 | 房租 + 人工 + 水电 + 杂费（月度合计，元/月） |
| 数据源 | 老板另供（POS/ERP 无此数据）；附最小填表：房租___ / 人工___ / 水电___ / 杂费___ |
| 频率 | 月（数额稳定，季度复核一次即可） |
| 层级 | L0（保本点计算前置件） |
| 阈值 | 不适用（成本构成项，非绩效指标，无预警线） |
| 失效边界 | ① 老板自家物业不付租/家人帮工不计薪时，账面固定成本虚低→保本点虚低，须按机会成本补记口径提示；② 人工含老板自身工资与否须先定口径，前后不一致则保本点月度不可比；③ 有大额季节性支出（空调电费夏冬翻倍）时"月固定"假设失真 |

### 36 · 同比基准锚（"去年同期"取数规范）

| 维度 | 内容 |
|---|---|
| 定义 | 同比比较时"去年同期"的取数口径规范：**同一自然周/自然月对齐**，非随意取 365 天前的某段 |
| 公式 | 同比增速 = (本期值 − 去年同期值) ÷ 去年同期值；同期 = 同一自然周（ISO 周号）或同一自然月 |
| 数据源 | 历史 POS 销售数据（须至少 12 个月连续留存） |
| 频率 | 随被比指标（周/月） |
| 层级 | L1 及以上分析件 |
| 阈值 | 不适用（取数规范，非指标本身） |
| 失效边界 | ① **无历史数据强制 pending，禁用环比冒充同比**——环比含节令噪声（春节/中秋月环比必失真），冒充即误判；② 春节漂移月（1/2 月）自然月对齐失效，须按农历对齐或合并 1-2 月比较；③ 门店中途大调改（面积/品类结构变动）后，同比失去可比性，须加"结构变动"标注 |
| 备注 | **全库当前零同比锚**（花厅坊数据窗口不足一年）——本条先立规范占位，数据跑满一年自然长出，届时逐指标回填 observed 锚 |

### 37 · Z类清货进度（%）

| 维度 | 内容 |
|---|---|
| 定义 | Z 类（3 月 0 销，见正表③ ABC-Z）清货执行进度，衡量"诊断→动作"落地率 |
| 公式 | 已清 Z 类 SKU 数 ÷ 计划清货 SKU 数 × 100% |
| 数据源 | 停购打标清单（分子基准）+ 周盘点结果（已清确认） |
| 频率 | 周 |
| 层级 | L1 执行看板（调改战役期） |
| 阈值 | pending（待花厅坊首轮清货实跑出基准节奏后再定周进度预警线） |
| 失效边界 | ① "已清"口径须先定：库存清零 vs 撤出陈列 vs 系统停购打标，三者进度可差数周，混用则进度虚高；② 供应商可退货的 SKU 清得快、不可退的压尾慢，进度 % 掩盖结构差异，须附"不可退残余清单"；③ 计划清货数中途增删（新一轮 ABC-Z 跑出新 Z 类）时分母漂移，须锁定批次编号分批计算 |

### 38 · 品类渗透率（%）

| 维度 | 内容 |
|---|---|
| 定义 | 购买过该品类的小票占全部小票的比例，衡量品类触达顾客的广度 |
| 公式 | 购买该品类的小票数 ÷ 总小票数 × 100% |
| 数据源 | POS 小票明细（**须含单据号**，按单据号去重统计） |
| 频率 | 月（战役期可周） |
| 层级 | L1 品类分析 / 营销策划北极星 |
| 阈值 | pending（无源页基准；花厅坊小票明细可导出后实测再定） |
| 失效边界 | ① **花厅坊当前导出受限（无小票明细/无单据号）→ 退化口径：不可算，强制 pending；禁用"品类销售额占比"冒充渗透率**——销售额占比是金额口径，渗透率是客流触达口径，一个大单可撑高占比但渗透率纹丝不动，冒充即把"卖给少数人卖得多"误读为"触达了很多人"；② 品类归级口径（L2 vs L3）不同渗透率差数倍，引用必标归级层；③ 促销期渗透率虚高（凑单/囤货），不可与常态期直接比较 |
| 备注 | 已被 [[KB-INSTORE-MKT-001]]（营销策划模板 v0.1）立为**北极星指标**，但 v0.2 字典缺条——本条补口——补条不等于可算，取数前提（单据号明细）未满足时一切引用标 pending（来源：营销策划模板 v0.1） |
