## §1. 业务定位

```
商品库 = 晟果新零售咨询专属跨客户复用底座
       = vault Year 2-3 数据飞轮核心壁垒
       = V4.0 品类表的实例化（V4.0 是结构 / 商品库是血肉）
```

---

## §2. 与 V4.0 品类表的关系

```
V4.0 品类表（308 L4 节点）→ 结构 / 骨架 / 唯一最终版
商品库（A+B+C 三层）        → 实例 / 血肉 / 多 SKU 挂在 L4 节点下

约束：商品库每个 SKU 必须有 category_l4 FK 指向 V4.0
关系：1 对多（L4 → 多 SKU）

例：
  V4.0 L4 [10103] = 调味品/酱油/生抽
  商品库 SKU：
    A-12345 / 海天生抽 500ml 玻璃瓶 / category_l4=10103
    A-12346 / 千禾头道酱油 500ml / category_l4=10103
    ...
```

---

## §3. 何时使用商品库

| 场景 | 用途 |
|---|---|
| 客户数据进 vault | 客户 SKU 文本 → 商品库匹配（4 层算法 / W27+）|
| 客户诊断分析 | 标准品名 + 标准品类 / 跨客户对比 |
| 跨客户复用 | 客户 A 标注 → 商品库累积 → 客户 B 受益（数据飞轮）|
| V4.0 升级 | V4.0 → V4.1 时商品库重新分类（G11 月度审计）|
