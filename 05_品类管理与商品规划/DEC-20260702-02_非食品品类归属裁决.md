## 现象 / 背景

V4.0/V6.0 品类系统覆盖食品三大类（休闲食品 / 水饮料 / 基础食品），但花厅坊实际销售中 **23.4%** 销额来自 V4.0 未覆盖品类：

| 品类 | 销额占比 | 当前状态 |
|---|---|---|
| 冷冻类（雪糕/饺子/火锅食材） | 11.2% | 无 L3 归属 |
| 酒精饮料（白酒/啤酒/红酒） | 9.5% | 无 L3 归属 |
| 火腿肠（双汇/金锣） | 2.7% | 归属争议（速冻 vs 卤制） |
| 茶叶（袋泡茶/散茶） | <0.1% | 无 L3 归属 |

上述品类在商品基础表中无系统归属，导致品类角色判定、动销分析、陈列决策均有盲区。

2026-05-09 治理决议留有"选项A/B/C"悬而未决；KB-CAT-ARCH-001（2026-06-27 签字）已隐含"扩全品类"路径，但无正式归属规则。

## 决策选项

| 方案 | 内容 | 风险 |
|---|---|---|
| **A（推荐）** | 采用 KB-CAT-ARCH-001 F/D/M 系已有框架，明确各品类归属规则（见下） | 极低：框架已在，只需签归属映射规则 |
| B | 建立独立"花厅坊扩展品类表"，与 V6.0 并列管理 | 高：新增第三套命名体系，维护负担翻倍 |

## 推理

KB-CAT-ARCH-001 已完成全品类框架设计（六哥 2026-06-27 签字 stable），方案A是在已有框架上挂数据，无需另建系统。旧治理决议（2026-05-09）已标注 deprecated，以本裁决为准。

## 预期结果

- 花厅坊 23.4% 无归属销额全部纳入系统
- 动销分析、品类角色判定覆盖率从 76.6% → 99%+
- 商品基础表剩余 20.5% 无 L3 标签 SKU 可批量标注

## ✍️ 六哥签字区

> 裁决生效日：2026-07-02
> 签字：✅ 六哥（2026-07-02）

**签字后生效的执行文字**：

1. **冷冻类**（雪糕/饺子/火锅食材/速冻汤圆等）→ F系（KB-CAT-ARCH-001 F冷冻便食，F01 速冻汤圆水饺 / F02 速冻丸类 / F03 速冻预制 / F10 方便面 / F11 自热食品 / F12 即食粥罐头）。
2. **火腿肠**（双汇/金锣等加工肉）→ F02 速冻丸类（归入 F系 即食加工肉）。
3. **酒精饮料**（白酒/啤酒/红酒/洋酒）→ 表A 顾客视角归 D2 酒水；表B 运营视角归 M4 烟酒专营（证照/资金管理逻辑）。
4. **茶叶**（袋泡茶/散茶等）→ E2 冲调饮品 D13 茶叶；V6.0 新增 L3 袋泡茶适用。
5. 以上归属即日写入商品基础表，完成 F/D/M/E 系无 L3 标签 SKU 的批量标注。
6. **非食日化**（洗护/纸品/母婴用品等）→ 暂挂 H系（KB-CAT-ARCH-001 §2 H01–H12），待百货扩展表就绪后一并落地，当前不强制标注（不阻塞 DEC-20260702-01 执行）。
