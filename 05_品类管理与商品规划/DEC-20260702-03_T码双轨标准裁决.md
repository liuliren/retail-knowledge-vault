## 现象 / 背景

全库存在两套 T 码体系并行，编号含义完全不同：

| T码 | 8分法（KB-CAT-ARCH-001，跨系统标准） | 12分法（花厅坊休食表，门店诊断） |
|---|---|---|
| T01 | 膨化食品 | 膨化食品 ✓ |
| T02 | **糖果巧克力** | **膨化** ✗ 冲突 |
| T03 | 饼干 | 饼干 ✓ |
| T04 | 坚果炒货 | **饼干** ✗ 冲突 |
| T06 | 蜜饯果干 | **肉干肉脯** ✗ 冲突 |
| T07 | 方便速食 | **巧克力** ✗ 冲突 |
| T08 | 冷冻便食 | **糖果** ✗ 冲突 |
| T09–T12 | 不存在 | 果冻布丁/辣条豆制品/儿童零食/其他 |

任何引用 T 码的脚本（category_mapper.py）、报告或陈列手册，在两套体系切换时均会产生系统性错误。

## 决策选项

| 方案 | 内容 | 风险 |
|---|---|---|
| **A（推荐）** | 两套码共存，作用域明确划分（见下）。V6.0 精读卡中的 mapper 对照表为权威映射文件 | 低：保留 12分法的门店细分价值，只需明确作用域边界 |
| B | 全面废弃 12分法，统一迁移至 8分法；果冻布丁/辣条等新增 L3 节点 | 中：信息损失（儿童零食/辣条独立大类的诊断价值），迁移工程量较大 |

## 推理

12分法包含门店实战细分依据（果冻布丁对应儿童客群权重 0.9、辣条独立大类），废弃会损失已积累的标注语料。两套体系只要作用域清晰、映射表明确，可以共存。V6.0 精读卡已提供 mapper 对照表，技术上开箱即用。

## 预期结果

- category_mapper.py 引用明确：跨系统查询用 8分法，门店内部分析用 12分法
- `/lint` T码冲突报警消除
- 花厅坊休食表保留细分价值，对外报告自动换算至 8分法

## ✍️ 六哥签字区

> 裁决生效日：2026-07-02
> 签字：✅ 六哥（2026-07-02）

**签字后生效的执行文字**：

1. **跨系统标准 T码**：KB-CAT-ARCH-001 §2 T01–T08（8分法）为全库、脚本、报告、陈列手册的**唯一引用标准**。
2. **门店诊断内部码**：花厅坊休食商品品类表 T01–T12（12分法），作用域限定为花厅坊门店内部分析，**不得**用于跨系统/跨店引用。
3. **强制映射规则**：凡引用 T01–T12 时，须附对应的 8分法 T码；V6.0 精读卡 mapper 对照表为权威映射文件（唯一 SSOT）。
4. 旧休食表"T02=膨化"定义即日废弃；T02 统一指 KB-CAT-ARCH-001 定义的"糖果巧克力"。
5. 花厅坊休食表加注声明："本表 T码为门店内部细分码，跨系统查询须映射至 KB-CAT-ARCH-001"。
