# 购物任务图谱 v1.1

## 一、文件定位

### 1.1 本文件是什么

本文件是**社区超市选品的"购物任务"方法论 v0.1 草案**——把零售选品的最小决策单元从「商品」切换为「购物任务（Jobs-to-be-Done）」，覆盖休食 9 任务 + 水饮 7 任务 = 16 个购物任务，每任务定义结构化 SKU 特征族 + 门店画像激活权重 + 大湾区本地化标记。

### 1.2 本文件不是什么

- ❌ **不是软件产品功能定义**——本图谱当前是**咨询判断工具**，不是系统功能模块
- ❌ **不是 SaaS 订阅产品的内核**——任何"AI 打标 / 选品系统软件"开发计划已封存（参 `90_素材暂存与待整理/选品系统知识资产卷宗/`）
- ❌ **不是 7D 诊断系统的接口规范**——7D PRD 已封存

### 1.3 服务的两个目标

1. **花厅坊样板打造**：休食 / 水饮区调改的诊断框架与 SKU 打标依据
2. **新客户开拓销售表达**：向潜在客户解释"我们怎么做选品"的核心方法论展示

### 1.4 与卷宗源的关系

- **卷宗源文件**：`90_素材暂存与待整理/选品系统知识资产卷宗/01_方法论核心资产/购物任务图谱_v1.1_FINAL.md`（已封存，不修改）
- **本文件**：是卷宗源在 vault 内的**改造版**，按 vault 治理标准重新表达。所有任务定义、典型 SKU、边界规则保持与卷宗源一致；新增 §一、§六、§九、§十、§十一、§十二、§十三、§十四 vault 适配章节。
- **使用约定**：vault 内后续引用优先使用本文件；卷宗源作为原始素材留痕。

## 二、购物任务图谱总览

### 2.1 设计原则（4 条前置约定）

1. **任务不是互斥分类，是"购买动机标签"**——SKU 多任务多权重，禁止单任务硬归类
2. **任务由"SKU 特征族"反推，不是凭空切分**——每任务必须能映射到结构化 SKU 特征（包装/品牌/价格/品类/口味）
3. **任务是"门店画像激活权重"的最小决策单元**——同任务在不同门店画像下激活强度不同
4. **大湾区本地化必须显性嵌入**——凉果陈皮 / 拜山祭祀 / 煲汤备货 / 儿童早餐奶 / 配辣凉茶等本地特征显式标记

### 2.2 16 任务速览

| 类目 | 任务 |
|---|---|
| 休食 9 | 解馋独享 / 办公室零食 / 聚会(HOSTING) / 朋友分享(CO-DISCOVERING) / 家庭时光(CO-LIVING) / 郊游 / 年货送礼 / **儿童零食（花厅坊专项）** / **中老年常备（凉果陈皮·大湾区差异化）** |
| 水饮 7 | 日常解渴 / 佐餐 / 功能性 / 聚会饮品 / 户外偶发 / 送礼 / **儿童饮品（花厅坊专项）** |

### 2.3 鉴别口诀（最易混淆的 3 个边界）

聚会 / 朋友分享 / 家庭时光不是按"人数"切分，按**消费动机**切分：

| 任务 | 本质 | 关键词 |
|---|---|---|
| 聚会 | HOSTING（招待）| 面子货、大牌、大包装 |
| 朋友分享 | CO-DISCOVERING（共同尝鲜）| 新奇、话题、网红 |
| 家庭时光 | CO-LIVING（家人共享）| 信任、混合装、各取所需 |

**判断口诀：先看是不是家人 → 再看几个人 → 最后看看重什么（面子 / 新奇 / 信任）**

## 三、休食 9 个购物任务

### 任务 1：解馋独享

| 字段 | 内容 |
|---|---|
| 核心场景 | 1 人即时消费，情绪驱动（嘴巴寂寞、压力大、追剧、加班、单身夜宵）|
| 人数 / 时机 / 动机 | 1 人 / 即时消费、计划性低 / 情绪 driven |
| 包装规格 | 单包小规格（30-80g），独立小包 |
| 品牌敏感度 / 价格带 | 低 / 3-10 元 |
| 品类倾向 | 辣条、魔芋爽、小面筋、薯片小包、素肉 |
| 口味倾向 | **强口味**（辣、酸辣、麻辣、咸香）|

**典型 SKU**：卫龙大面筋 26g、麻辣魔芋爽 15g、乐事小袋装、卫龙辣条
**关键边界**：vs 办公室零食 = 即时冲动 vs 计划囤货
**大湾区本地化**：辣味偏好不如川渝，但年轻群体网红爆款（辣条、魔芋）依然成立

### 任务 2：办公室零食

| 字段 | 内容 |
|---|---|
| 核心场景 | 白领工位囤货、抽屉常备、下午茶补能 |
| 人数 / 时机 / 动机 | 1 人为主 / 计划购买 / 抗饿+解闷+补能，要求"不打扰" |
| 包装规格 | 中等规格 50-150g，独立小包数量分装 |
| 品牌敏感度 / 价格带 | 中 / 8-25 元 |
| 品类倾向 | 坚果、能量棒、蛋黄派、蔬果干、轻食饼干 |
| 关键属性 | **无气味、不掉渣、耐放、可分次吃** |

**典型 SKU**:三只松鼠每日坚果(独立装)、好丽友派盒装、沃隆混合坚果、谷物棒系列
**关键边界**:vs 解馋独享 = 抗饿/补能(理性) vs 情绪释放(感性)

### 任务 3:聚会(HOSTING)

| 字段 | 内容 |
|---|---|
| 核心场景 | 5+ 人非家庭聚会(同学/同事聚餐、朋友过夜、世界杯、生日趴),**你是组织者** |
| 人数 / 关系 / 动机 | 5+ / 非家人 / **招待场面 + 面子货** |
| 包装规格 | **大桶 / 大盒 / 大包**(200g+) |
| 品牌敏感度 / 价格带 | **高** / 15-50 元 / 单品 |
| 品类倾向 | 罐装薯片、大盒夹心饼、品牌坚果罐、糖果大袋 |
| 关键属性 | **上桌好看**,分给客人不掉价 |

**典型 SKU**:乐事/可比克罐装薯片、好丽友派大盒装、奥利奥分享装、三只松鼠坚果罐 500g
**关键边界**:vs 朋友分享 = HOSTING(招待别人) vs CO-DISCOVERING(共同尝鲜)

### 任务 4:朋友分享(CO-DISCOVERING)

| 字段 | 内容 |
|---|---|
| 核心场景 | 2-4 人小范围(闺蜜下午茶、周末小聚、新人尝鲜),**话题驱动** |
| 人数 / 关系 / 动机 | 2-4 / 非家人平等关系 / **新奇尝鲜 + 话题谈资** |
| 包装规格 / 价格带 | 中等 80-200g / 10-30 元 |
| 品类倾向 | **网红爆款、地方特产、新口味、小红书/抖音爆款** |
| 关键属性 | **有话题、可讨论、可拍照** |

**典型 SKU**:网红爆款(每月迭代)、进口小众零食、新口味产品(咸蛋黄/芝士麻辣)、地方特色(柳州螺蛳粉味薯片)
**大湾区本地化**:奶茶店周边、港式怀旧零食、本地网红品牌

### 任务 5:家庭时光(CO-LIVING)

| 字段 | 内容 |
|---|---|
| 核心场景 | 家人围坐场景(追剧、电影夜、节日团聚、周末家庭日) |
| 人数 / 关系 / 动机 | 3-5 家庭单位 / 家人 / **家人共享、各取所需** |
| 包装规格 / 价格带 | **家庭装、混合大礼包 300g+** / 20-80 元 |
| 品类倾向 | 综合零食礼包、混合干果、家庭装薯片、爆米花桶、瓜子坚果罐 |
| 关键属性 | **多样化、覆盖不同口味偏好** |

**典型 SKU**:旺旺大礼包、沃隆每日坚果(家庭装)、三只松鼠综合礼包、大桶爆米花
**关键边界**:vs 聚会 = 家人信任 vs 非家人面子;vs 年货送礼 = 自家消费 vs 送给他人

### 任务 6:郊游 / 户外

| 字段 | 内容 |
|---|---|
| 核心场景 | 远足、野餐、车载、出游携带、登山——**便携消费** |
| 时机 / 动机 | 户外、移动中 / **便携、抗折损、补能量** |
| 包装规格 | **便携、不易碎、密封、轻量** |
| 品牌敏感度 / 价格带 | 中 / 5-25 元 |
| 品类倾向 | 能量棒、压缩饼干、肉脯、独立坚果、巧克力派、糖果 |
| 关键属性 | **水分少、不融化、不怕颠、儿童友好** |

**典型 SKU**:牛肉粒/肉脯独立装、士力架(耐高温款)、独立包装坚果、能量棒
**大湾区本地化**:周末徒步、自驾游频率高,全年都有需求

### 任务 7:年货送礼

| 字段 | 内容 |
|---|---|
| 核心场景 | 春节、中秋、端午、回老家伴手、商务送礼 |
| 时机 / 动机 | 节日强季节性 / **面子 + 心意 + 地方特色** |
| 包装规格 | **精品礼盒装、地方特色礼盒** |
| 品牌敏感度 / 价格带 | **极高** / 50-300 元+ |
| 品类倾向 | 礼盒装坚果、传统糖点礼盒、地方特产礼盒 |
| 关键属性 | **包装精美、价格带明显高、地方文化感** |

**典型 SKU**:三只松鼠坚果礼盒、香港大班冰皮月饼、老婆饼/凤梨酥礼盒、老字号年货礼盒
**大湾区本地化**:春节年货市场、清明饼礼盒、中秋月饼礼盒——节日峰极强

### 任务 8:儿童零食(花厅坊专项)

| 字段 | 内容 |
|---|---|
| 核心场景 | 小学生放学路过自购、家长备货、节日给孩子 |
| 人数 / 关系 | 儿童为主 / 儿童自购 + 家长代购 |
| 动机 | **口感偏甜 + IP/卡通驱动 + 家长认证(健康)** |
| 时机 | 放学时段、周末家庭采购 |
| 包装规格 / 价格带 | **小规格、卡通包装、亮色** / 1-15 元(学生自购倾向 1-5 元) |
| 品类倾向 | 含奶饼干、儿童糖果、卡通棒棒糖、果冻、巧克力豆、儿童派 |
| 关键属性 | **含奶量、低盐、营养标识、IP 形象** |

**典型 SKU**:旺仔小馒头、喜之郎果冻、旺旺仙贝、卡通棒棒糖、奥利奥小包装、蓝胖子
**大湾区本地化**:花厅坊有小学 → 该任务在花厅坊门店权重显著高于纯白领商圈门店

### 任务 9:中老年常备 / 佐茶(大湾区差异化)

> **大湾区差异化任务**:大湾区社区超市的中老年客群有非常明确的、与年轻群体完全不同的购买行为。这些 SKU 在前 8 任务里都放不进去——独立成任务后,凉果陈皮在打标系统里有正确的任务级承载。

| 字段 | 内容 |
|---|---|
| 核心场景 | 老年人日常嗑食 / 佐茶习惯 / 在家慢消费 |
| 时机 / 动机 | 日常、高频、计划购买 / **习惯性嗑食 / 佐茶 / 怀旧** |
| 包装规格 | 中等 100-300g、自封袋 |
| 品牌敏感度 / 价格带 | 低(认产品类型不认品牌) / 5-25 元 |
| 品类倾向 | **凉果陈皮、话梅、嘉应子、瓜子(日嗑)、山楂条、九制陈皮** |
| 关键属性 | **慢消费、耐放、咸甜复合口感** |

**典型 SKU**:化州橘红凉果、九制陈皮、话梅、嘉应子、袋装瓜子(200g+)、山楂条、果丹皮
**与花厅坊既有方法论的关联**:花厅坊调改时把"凉果陈皮"作为差异化锚点之一——本任务就是这个锚点的**任务级承载**(待花厅坊实测校准)

## 四、水饮 7 个购物任务

### 任务 1:日常解渴
即时饮用、单瓶、最高频;包装 350-550ml 单瓶;价格带 2-6 元
**典型**:怡宝矿泉水、康师傅冰红茶、农夫山泉、维他柠檬茶

### 任务 2:佐餐场景
配早餐 / 午餐 / 外卖、配辣;单瓶 / 盒 / 利乐包;3-8 元
**典型**:早餐奶(蒙牛/伊利)、维他奶、王老吉/加多宝(配辣)、果汁
**大湾区本地化**:配辣 → 凉茶(王老吉 = 大湾区配辣首选);早茶 → 普洱/铁观音瓶装

### 任务 3:功能性
运动后、熬夜、解辣、护胃;单瓶;4-10 元
**典型**:脉动、宝矿力、红牛、东鹏特饮、苏打水

### 任务 4:聚会饮品
多人聚餐、围坐、餐厅外带;**1.25L+ 大瓶 / 多瓶组合**;8-25 元
**典型**:大瓶可乐雪碧、芬达、大瓶果汁、苏打水组合

### 任务 5:户外偶发
登山、拜山、郊游、自驾;**整箱、便携多瓶**;15-40 元/箱
**典型**:整箱矿泉水、整箱运动饮料、整箱椰子水
**大湾区本地化**:清明拜山祭祀(瓶装水、利乐包果汁/牛奶作为祭品+消费品)

### 任务 6:送礼 / 节日
礼盒装伴手;礼盒、盒装多瓶;50-200 元
**典型**:蒙牛特仑苏礼盒、椰汁礼盒、维他奶礼盒、燕窝水礼盒

### 任务 7:儿童饮品(花厅坊专项)
学生自购 / 家长备货;小瓶 / 利乐包 / 多支装;1-8 元
**典型**:旺仔牛奶、优酸乳、爽歪歪、AD 钙奶、阳光美味果汁、小瓶豆奶
**关键边界**:vs 佐餐 = 专给儿童(卡通/儿童口味) vs 配餐成人

## 六、花厅坊画像的初步应用

### 6.1 花厅坊门店画像定性

**画像类型:综合体 + 小学 + 老社区**(v0.1 待二访实地核验)

3 个差异化锚点(v0.1 经验判断):
1. **小学放学时段** → 儿童零食 0.9 / 儿童饮品 0.9(最强差异化)
2. **老社区中老年常住客群** → 中老年常备 0.8(大湾区凉果陈皮承接)
3. **综合体家庭采购** → 家庭时光 0.8 / 年货送礼 0.8

### 6.2 花厅坊休食区调改的任务驱动思路(v0.1)

| 调改方向 | 任务依据 | 关键 SKU 验证 |
|---|---|---|
| 强化儿童零食区(黄金陈列位) | 任务 8 / 权重 0.9 | 旺仔小馒头、喜之郎果冻、卡通棒棒糖等是否齐全 |
| 强化中老年常备区(凉果陈皮区) | 任务 9 / 权重 0.8 | 化州橘红凉果、九制陈皮、话梅是否齐全;本地老字号是否引入 |

## 八、SKU 多任务多权重打标规则

### 8.1 打标结构

每个 SKU 至少 1 个主任务(权重 ≥ 0.5),可附 1-2 个次任务(权重 0.1-0.4)。**权重之和 = 1.0**。

### 8.2 打标示例

| SKU | 主任务 | 次任务 1 | 次任务 2 |
|---|---|---|---|
| 卫龙大面筋 26g | 解馋独享 (0.7) | 朋友分享 (0.2) | 儿童零食 (0.1) |
| 三只松鼠坚果礼盒 | 年货送礼 (0.8) | 家庭时光 (0.2) | — |
| 旺仔牛奶 125ml × 12 | 儿童饮品 (0.7) | 家庭时光 (0.2) | 日常解渴 (0.1) |
| 化州橘红凉果 200g | 中老年常备 (0.8) | 解馋独享 (0.2) | — |
| 1.25L 可口可乐 | 聚会饮品 (0.7) | 家庭时光 (0.3) | — |
| 整箱怡宝 555ml × 24 | 户外偶发 (0.5) | 日常解渴 (0.4) | 聚会饮品 (0.1) |
| 大桶爆米花 500g | 家庭时光 (0.6) | 聚会 (0.3) | 郊游 (0.1) |
| 沃隆每日坚果 25 包 | 办公室零食 (0.6) | 家庭时光 (0.3) | 郊游 (0.1) |

### 8.3 咨询师人工打标工作流(不是 AI 软件)

> ⚠️ **本工作流当前是咨询师人工判断流程,不是 AI 软件功能**。任何 AI 打标小工具开发已封存(参卷宗 §02 PRD 草案,已封存)。

1. 咨询师查看 SKU 基础属性(包装/品牌/价格/品类)
2. 对照 §三 §四 任务定义判断主任务(动机优先 > 包装规格 > 价格带)
3. 判断次任务(1-2 个,权重之和与主任务 = 1.0)
4. **必须填写"判断理由"**——这是后续校准与方法论沉淀的核心
5. 累积 100+ SKU 后回看判断理由,沉淀打标判断手册(v0.2 升级时整理)

## 九、与 IPC / CDT / HS / DisplayMap 的关系

任务图谱不替代既有方法论,是 vault 现有方法论的**新维度补充**:

| 既有方法论 | 任务图谱的补充关系 |
|---|---|
| **IPC**([[商品配置理论]] §8 / [[大湾区IPC与TF函数本地化_MC-GBA-03]]) | IPC 看价格带分布,任务图谱看动机分布。**两者交叉**:每任务 × 每价格带 → SKU 数热力图,发现"任务被错误地集中在某个价格带"问题 |
| **CDT 顾客决策树**([[品类管理]] / [[商品配置理论]] §三) | CDT 是分类树,任务图谱是动机标签。**CDT 解决"商品按什么逻辑分类",任务图谱解决"为什么买"** |
| **HS 矩阵**([[商品配置理论]] §9) | HS 矩阵是 S 轴顾客视角 × H 轴商品视角;任务图谱可作为 **S 轴(顾客视角)的具体化**——把抽象的"顾客视角"转为可操作的 16 任务 |
| **DisplayMap**(`06_陈列规划与DisplayMap` 待建立) | 任务图谱 + 门店画像权重 → 陈列资源分配的依据。例:花厅坊儿童零食 0.9 → 黄金货架优先给儿童零食区 |
| **52周MD**([[52周MD]]) | 任务图谱的"年货送礼""节庆"任务对应 52 周节庆节奏;任务图谱给出 SKU 级承载,52周MD 给出时间级承载 |
| **品类角色**([[品类管理]] / 待独立成文) | 任务图谱可作为品类角色判断的依据:高激活权重任务下的核心 SKU = Destination;低权重任务的边缘 SKU = Convenience |

## 十一、可用于客户销售表达的简化话术

> ⚠️ 以下话术 v0.1 草案,待 1-2 个真实客户验证后调整。完整销售包结构待 `10_咨询交付模板/` 销售包 v0.1 立项。

### 11.1 三步 demo 话术(向潜在客户讲解"我们怎么做选品")

**第 1 步(拿出图谱)**:
> "我们做选品,不是先想'卖什么商品',是先想'顾客为什么买'。社区超市的休食区,背后有 9 个购物任务;水饮区有 7 个。每个任务都有完全不同的商品逻辑。"

**第 2 步(指向客户门店类型)**:
> "比如您这家店,我初判的画像类型(参 §五),这种画像下不同任务的激活权重应该有明显差异——把高权重任务做对了,您的店就能跟同业态其他类型的店拉开差距。"

**第 3 步(呼应客户痛点,保留诊断空间)**:
> "您说营业额跌了 X%——可能存在某些高权重购物任务没有被充分承接,例如儿童零食、中老年常备或家庭时光等,需要结合您门店的画像和 SKU 数据来验证。我可以给您先做一份 2-3 页的快速判断报告,确认到底是哪几个任务的结构问题。"

### 11.2 配套销售素材

- **快速判断报告(2-3 页)**:基于图谱 + 客户门店画像初判,给出 3-5 条最优先 SKU 引进 / 淘汰建议(待 `10_咨询交付模板/` 销售包 v0.1 立项时正式制作)
- **凉果陈皮差异化案例**:以花厅坊为样板(待花厅坊样板成熟后正式形成)

### 11.3 不该用的话术

- ❌ 不要说"AI 选品系统"——卷宗已封存软件方向
- ❌ 不要说"SaaS 订阅"——客户痛点 TOP7 排序中"选品工具"是不紧迫痛点
- ❌ 不要展示数据库 schema / 字段表——这是产品研发语言,不是客户语言
- ❌ 不要把花厅坊门店画像直接套用到其他客户——必须现场判断每家客户的门店画像
