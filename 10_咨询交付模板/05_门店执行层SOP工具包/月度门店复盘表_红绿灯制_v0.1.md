# 月度门店复盘表·红绿灯制

## 用途

与vault既有的诊断分析工具(如[[KB-DX-FULLSTORE-001_全店7D诊断框架2.0_v0.1]])不同，本表是**轻量、高频、店长自己能填**的月度例行复盘工具，用于日常运营节奏管理，不需要顾问介入。

## 表格结构

### 第一部分：三大核心指标（红绿灯判定）

| 门店：___________ | 月份：_____年____月 | 复盘日期：____月___日 |
|---|---|---|

| 核心指标 | 实际完成（情况说明） | 结果判定 |
|---|---|---|
| 1. 总销售额(元) | | □达标 □未达标(目标差额:________) |
| 2. 综合毛利率(%) | | □达标 □未达标 |
| 3. 综合损耗率(%) | | □正常 □超标(异常品类:________) |

### 第二部分：结构化复盘

| 做得好的(本月亮点) | 搞砸了的(本月问题) | 其它 |
|---|---|---|
| (例：阳光玫瑰陈列好，卖爆了) | (只写核心痛点，例：猪肉到货太晚，顾客流失) | |
| 店长填1. | 店长填1. | |
| 2. | 2. | |
| 3. | 3. | |

## 使用说明

1. **红绿灯制的核心价值**：不追求复杂的多维度分析，用最少的3个指标+"达标/未达标"的二元判定，让店长每月能5分钟填完，不因为复杂而不填。
2. **与诊断方法论的关系**：本表是"日常例行体检"，vault既有的7D诊断/M-DEC决策链是"深度会诊"——两者不冲突，本表触发异常时才需要引入深度诊断工具。
3. **建议频率**：月度，或按门店实际运营节奏调整为四周一次。
4. **与巡店表的关系**：[[分时段巡店检查表_通用模板_v0.1]]管"每日执行"，本表管"月度指标"——巡店发现的问题若持续影响指标，应在本表"本月问题"栏体现。
