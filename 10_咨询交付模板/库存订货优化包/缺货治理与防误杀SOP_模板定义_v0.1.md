# 缺货治理与防误杀SOP 模板定义 v0.1

> 本文件是 [[社区超市库存订货优化包_主定义_v0.1]] 的 **4 号交付物**:**店长照做**的缺货治理操作手册。
> **只引用、不重定义**:数量为先、追加式订货防误杀全部锚 [[数量管理]] §0/§11 误区7。
> 上游:[[有货率与安全库存设定表_模板定义_v0.1]](再订货点)、[[ABC分级订货系数表_模板定义_v0.1]](订货量);下游:[[周转看板_模板定义_v0.1]] 周复盘缺货率。
> ⚠ 客户版走 [[报告导出系统_report-export_v1.0]] 出 0 术语版(店长素养不高:一句话规则 + 看哪个数 + 谁负责)。

## 1. 模板定位

给店长一套**照着做就能不缺核心货**的操作 SOP:**看哪个数、低于多少补、谁负责**,并内置"**追加式订货防误杀**"——区分真畅销 vs 因常缺货而显得滞销的假滞销,防止把好商品误砍。**数量为先**(§0):先把缺货治住,再谈金额毛利。

## 2. 核心 SOP(店长三步·照做)

| 步 | 看哪个数 | 规则 | 谁负责 |
|---|---|---|---|
| ① 看 | **当前库存 vs 再订货点 ROP** | 库存 **< 再订货点 → 立即补** | 店长/补货员每日巡 |
| ② 补 | **ABC 级 + 订货系数** | A 类高频小批量、按系数下单;C 类低频 | 订货员 |
| ③ 核 | **缺货 SKU 清单** | 核心 A 类**不得断货**;断了当天追因 | 店长 |

> **三业务线措辞(按紧迫度)**:库存 < 安全库存 →「触发」紧急处理(最急);库存 < 再订货点 →「提示」下采购单(正常);库存 > 最大库存 →「建议」清理(最缓)(§6.6C⑤)。

## 3. ⭐追加式订货防误杀(本 SOP 的灵魂)

> **追加式订货("卖多少补多少")是库存管理最可怕之处**——慢性毒药,长期卖场一成不变、客流失、难逆转(§11 误区7)。

### A · 卖多 ≠ 畅销,卖少 ≠ 滞销
- **7-11 X/Y/Z 例**:Z 一小时卖完=真畅销却销量最少(早早卖断货);X/Y 销量大,可能是 Z 缺货后顾客**无奈替代购买**=实为滞销。
- **机理**:销量 ≠ 需求。销量只记"卖出去的",记不了"想买没买到的"。库存 200、顾客要 300,系统只记 200,那 100 真实需求被永久隐藏 → **越缺货越显得卖得少 → 越不敢多订 → 越缺货**(§6.7G + §6.6C 越畅销越缺货)。

### B · 防误杀判定流程(下任何"减订/淘汰"结论前必过)
```
某 SKU 看似卖得少 / 想砍
  → 问1:它最近常断货吗?(查缺货天数)
        是 → 假滞销!先补回、设安全库存,观察真实需求,不砍
        否 → 问2:是季节/促销结束/新品观察期吗?
              是 → 暂不动,按节奏判
              否 → 问3:小类内是不是目的品/必需款?(小类内算ABC)
                    是 → 保护,不砍
                    否 → 才可进减订/淘汰流程
```

### C · 误判防线(对照表·砍货前逐条排查)
① 新品保护期 ② 季节/节日 ③ **缺货导致低销(头号杀手)** ④ 陈列位置差 ⑤ 价格异常 ⑥ 规格不适配 ⑦ 数据缺失/POS分类错 ⑧ 非显性需求(老人/本地/儿童) ⑨ 供应商临时问题 ⑩ 促销期异常高销 ⑪ 负毛利/特价清仓。

## 4. 缺货追因(每次断货填一行)

| 字段 | 填法 |
|---|---|
| 缺货 SKU/代号 | — |
| ABC 级 | A 类优先 |
| 缺货原因 | 报货少/配送慢/采购质量/需求突增 |
| 是否核心A断货 | 是=红线事件 |
| 责任部门 | 报损单注明原因,统计最多者主动担责改进(§6.6H) |
| 补救动作 | 紧急补 / 调安全库存 |

> **轻量动作(小店今天能做)**:报损/缺货单注明原因,一段时间后原因统计最多的部门主动担责(§6.6H)。

## 5. 怎么用(顾问交付 → 店长执行)

1. 顾问把 2 号表的"再订货点"写成店长能看的补货卡;
2. 店长每日巡:库存低于再订货点就补,核心 A 类绝不断;
3. **看似要砍的货,先过 §3 防误杀流程**;
4. 每次断货填 §4 追因,周复盘看缺货率(接 [[周转看板_模板定义_v0.1]])。

## 6. 使用边界

不做:替代安全库存计算(用 2 号表的结果)、替代 ABC 系数(用 3 号表)、追加式"卖多少补多少"(本 SOP 明确反对)。

## 7. 引用的数量管理判据

§0 数量为先(先治缺货再看金额) · §11 误区7 追加式订货=慢性毒药/卖多≠畅销(7-11 X/Y/Z)/订货以结构优化为前提 · §6.7G 销量≠需求(被缺货截断的销量) · §6.6C 越畅销越缺货 + 三业务线触发/提示/建议措辞 · §6.6G ABC 小类内算防误杀 · §6.6H 报损单注明原因倒逼担责。
