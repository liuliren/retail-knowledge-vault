# 有货率与安全库存设定表 模板定义 v0.1

> 本文件是 [[社区超市库存订货优化包_主定义_v0.1]] 的 **2 号交付物**:把"该备几个、低于几个就补"算成逐 SKU 的**安全库存 + 再订货点**。
> **只引用、不重定义**:Z 值法/泊松/有货率公式全部锚 [[数量管理]] §6.6C/§6.6F。
> 上游:[[库存订货诊断报告_模板定义_v0.1]] + POS 日均销/提前期;下游:[[ABC分级订货系数表_模板定义_v0.1]](订货量)、[[缺货治理与防误杀SOP_模板定义_v0.1]](按再订货点补货)。
> ⚠ 客户版走 [[报告导出系统_report-export_v1.0]] 出 0 术语版(只留"该备几个/低于几个补",隐去 Z/σ/泊松术语)。

## 1. 模板定位

逐 SKU(或逐小类)设定**目标有货率 → 安全库存 → 再订货点**的设定表。回答数量管理最核心一问:**这个商品该备几个、低于几个就必须补**。**不是需求预测表**——安全库存对付不确定性,预测对付平均需求,两件事分开(§6.6C 铁律)。

## 2. 列定义(逐列含义 + 怎么填 + 判据)

| 列 | 含义 | 怎么填 | 判据(数量管理§) |
|---|---|---|---|
| **SKU / 商品代号** | 商品标识(客户版用代号) | 从清洗后档案取 | — |
| **小类(=一个顾客需求)** | ABC 与防误杀的口径单位 | 必填,ABC 在小类内算 | §6.6G ABC 口径=小类内算,非全场(防误杀) |
| **ABC 级** | A/B/C 分级 | 小类内按销量/毛利/客流排序定 | §6.6G A:数量10–20%/价值70–80%;B:~30%/15–25%;C:~50%/~5% |
| **日均销 DMS** | 移动平均日均销量 | POS 近 30/60 天移动平均 | §6.6.J/§6.6.P 管道库存=DMS×提前期 |
| **日均销标准差 σ** | 需求波动 | `STDEV`(数据点≥30,实操≥14);趋势/季节品取**预测误差**的σ | §6.6C 三量化①;数据点<14 标"σ不可信" |
| **提前期 VP(天)** | 采购前期+下单到货+检查期 | 按供应商实测;**缺此列则全表失效** | §6.6C VP 含采购前期+检查期;§6.6.O-F④ |
| **服务水平目标(有货率)** | 目标有货率% | A 严 B 适 C 松;差异化:常规95%/促销97%/团购100% | §6.6C 差异化服务水平·分阶梯 |
| **Z 值** | 服务水平系数 | 查表/`NORMSINV(有货率)` | §6.6C 95%→1.64、99%→2.33、1σ→84.1%、2σ→97.7% |
| **安全库存 SS** | 不确定性缓冲量 | 正态:`SS=Z×σ×√VP`;长尾:见 §3 | §6.6C VP 公式;多期=平方和开方非倍数 |
| **管道库存** | 提前期内会卖掉的量 | DMS×VP | §6.6.P 管道库存=DMS×提前期 |
| **再订货点 ROP** | 低于此值就补货 | **ROP=安全库存+管道库存** | §6.6.P 再订购点=安全库存+管道库存 |
| **有货率(实测/复盘)** | 上期实际有货率 | `1−缺货天数÷(总天数−无需求天数)` | §6.6C ⭐扣无需求天数 |
| **备注/数据可信度** | 仅参考标记等 | 进价不准/σ不可信/待采购周期 | §6.6.O-F④ |

## 3. 背后判据(填表必读)

### A · 有货率公式(口径关键)
**有货率 = 1 − 缺货天数 ÷(总天数 − 无需求天数)**。分母**必须扣掉无需求天数**(那几天本就没人买,不该惩罚有货率)。例:10 天里 2 天无库存 → 75%,而非简单 1−2/10(§6.6C)。三词同义:有货率=现货率=服务水平。

### B · Z 值法(正态·连续高频品)
服务水平梯度:不放安全库存=50%(按销定货中位起点)→ 1σ→84.1% → 2σ→97.7% → **95%→Z=1.64**(`NORMSINV(0.95)`)→ 99%→Z=2.33。**投资回报递减**:95%→99% 只升 0.04 但 Z 几乎翻倍 → 别盲目追 100%,分阶梯(90%即时/95%在2–3天/100%在一个订货周期)。
**含提前期**:`SS = Z × σ × √VP`(VP=提前期天数,含采购前期+检查期);多期不确定性=各期标准差**平方和开方**(√VP,不是×VP)。

### C · 长尾用泊松(门店绝大多数 SKU)
低频/不连续/长尾品**不符合正态**,Z 值法硬套=系统性算错。改用泊松:`POISSON.DIST(X, 平均值, True)`,X=备货量,平均值=补货周期内平均需求,返回有货率。例(进口啤酒·周补·周均2箱):备4箱→95%、备5箱→98%。只需一个参数(平均值),比正态少一个 σ,落地更轻(§6.6F)。

### D · 二选一不是替代
连续/高频/近正态走 §B Z 值法;低频/长尾走 §C 泊松。按需求形态分工(§6.6F)。

### E · 动态调整(别设一次锁死)
安全库存随真实 VP 重算(越畅销 VP 越长越易缺货,§6.6C 根问题);三触发:① 一进一出(新品进→同需求老品调低);② 春节/大促提前调高;③ 高进货成本/现金采购品少放(§6.6C 动态调整)。

## 4. 怎么用(顾问 → 店长)

1. 顾问按表算出每个 SKU 的 ROP,导入系统或写成补货卡;
2. 店长**只需看"低于再订货点就补"**(无需懂 Z/σ);
3. A 类盯紧、C 类放松(§6.6G 管控强度 A严/B适/C松);
4. 缺采购周期列 → 该 SKU 标"待数据",不强行给 ROP。

## 5. 使用边界 + 进价坑

不做:需求预测(那是另一件事)、把安全库存分摊进预测(系统会当过高预测删掉→越畅销越缺货,§6.6C 铁律)。
⚠ **进价不准**:本表不依赖进价(只用日均销/σ/提前期),故 ROP 不受进价坑影响;但若客户要看"压了多少钱",金额档标"仅参考"(见 [[花厅坊数据质量坑]])。

## 6. 引用的数量管理判据

§6.6C 有货率公式(扣无需求天数)/服务水平 Z 值表(95%→1.64)/SS=Z×σ×√VP/三量化/动态调整 · §6.6F 长尾泊松 POISSON.DIST · §6.6G ABC 小类内算 + 管控强度 · §6.6.P 管道库存=DMS×提前期、再订货点=安全库存+管道库存 · §6.6.O-F④ 采购周期列缺失则失效。
