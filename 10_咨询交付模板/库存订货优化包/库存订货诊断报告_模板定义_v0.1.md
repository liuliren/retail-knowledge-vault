# 库存订货诊断报告 模板定义 v0.1

> 本文件是 [[社区超市库存订货优化包_主定义_v0.1]] 的 **1 号交付物**:库存订货现状的**老板一页结论**。
> **只引用、不重定义**:所有判据来自 [[数量管理]],本文件只做"产品化封装"。
> 上游:POS 进销存(经 [[POS数据清洗与商品标准化SOP_v1.0]] 清洗)→ **本报告** → 下游:[[有货率与安全库存设定表_模板定义_v0.1]] / [[呆滞与售罄诊断清单_模板定义_v0.1]]。
> ⚠ 客户最终交付版**必走 [[报告导出系统_report-export_v1.0]] 出 0 术语版**(红黄绿表保留、术语全部口语化、去条码/进价/供应商)。

## 1. 模板定位

社区超市库存订货现状的**老板视角一页结论**:用红黄绿三色一眼看清"缺货丢了多少销售、积压压了多少现金、货转得快不快",并明确**数据能信到几分**。**不写长篇报告,产出可裁决的现状盘 + 三步动作**。判据全部来自 [[数量管理]],本报告只负责"翻译成老板话 + 红黄绿"。

## 2. 标准结构(4 段·按金字塔原理结论先行)

```
一句话结论 → 红黄绿现状表 → 三步动作 → 数据基础(边界声明)
```

### 第 1 段 · 一句话结论
一句话点出**本店库存订货的最大失血点**(缺货为主 / 积压为主 / 两头失血 / 周转拖慢)。例句模板:"本店【X 类】好卖的常缺货、【Y 类】不好卖的压着现金,整体货转得【偏慢/正常】——先治缺货、再清呆滞、同步把订货量算出标准。"

### 第 2 段 · 红黄绿现状表(核心)
> 三色判读规则见 §3;**指标口径与阈值全部锚 [[数量管理]],不自创**。

| 维度 | 指标 | 本店值 | 红黄绿 | 判读(口语) |
|---|---|---|---|---|
| 缺货 | 有货率(核心 A 类) | __% | 🔴/🟡/🟢 | 好卖的够不够卖 |
| 缺货 | 缺货 SKU 数 / 占比 | __ | | 多少品在断货 |
| 积压 | 呆滞库存占比 | __% | | 多少钱压成死货 |
| 积压 | 库存可销天数 DOS | __天 | | 现有货还能卖几天 |
| 周转 | 库存周转率 IR | __ | | 货转得快不快 |
| 周转 | 边际周转率信号 | 增/降 | | 库存涨得比销售快否 |

### 第 3 段 · 三步动作
按"先数量后金额"顺序(§0 诊断顺序原则):① **先治缺货**(核心 A 类先补回,见 [[缺货治理与防误杀SOP_模板定义_v0.1]]);② **再清呆滞**(腾现金,见 [[呆滞与售罄诊断清单_模板定义_v0.1]]);③ **同步定标准**(安全库存 + ABC 订货系数,见 [[有货率与安全库存设定表_模板定义_v0.1]]、[[ABC分级订货系数表_模板定义_v0.1]])。每步标:谁负责 / 何时见效方向(**方向非承诺**)。

### 第 4 段 · 数据基础(边界声明·必填)
- **数据来源 / 周期**:POS 近 30/60/90 天进销存;
- **⚠ 进价不准坑(铁律)**:花厅坊式进价/成本不准时,**毛利率与周转率 IR 档一律标"仅参考"**——IR=12×(1−毛利率)÷ITO 直接吃毛利率,毛利失真则周转档失真(见 [[花厅坊数据质量坑]]);
- **缺采购周期列 = 订货建议失效**:无"采购周期/交货期"列时,安全库存与再订货点**算不出**(§6.6.O-F④),须先补列或标"待数据";
- **fact_layer**:缺货/积压现状=observed;周转/毛利档=inferred(进价不准时)。

## 3. 红黄绿判读规则(锚数量管理阈值)

| 维度 | 🟢 绿(健康) | 🟡 黄(关注) | 🔴 红(失血) | 判据(数量管理§) |
|---|---|---|---|---|
| 有货率 | 核心 A 类 ≥95% | 90–95% | <90% | §6.6C 服务水平=有货率·95%为常用档 |
| 呆滞占比 | 极低 | 上升中 | 攀升/偏高 | §6.6.S 呆滞=超一年无需求·占比=库存健康度核心指标 |
| 库存可销天数 DOS | 接近本店标准 | 略超 | 远超标准(如标准30天而库存45天) | §6.6D 可销天数=现有库存÷日均销 |
| 周转率 IR | 0.9–1.5 | 接近边界 | 过高(积压)/过低(易缺货) | §6.6.O-C IR=12×(1−毛利率)÷ITO·0.9–1.5合理 |
| 边际周转 | 库存增速<业务增速 | 持平 | 库存增速>业务增速 | §6.6.I 边际周转率·积压早期信号 |

> 🟢=周转(良性) / 🟡=安全(中性) / 🔴=呆滞(不良),与 [[周转看板_模板定义_v0.1]] 三色同源(§6.6.S)。

## 4. 怎么填(顾问操作)

1. 拉 POS → 清洗(标准化品类/小类/日均销)→ 算六个指标;
2. **先查数量再看金额**(§0):先看有货率/缺货数/呆滞,再看周转/毛利;
3. 对每个红/黄项,**先过误判防线**(缺货致假滞销?季节?促销?见 [[缺货治理与防误杀SOP_模板定义_v0.1]]),再下结论;
4. 进价不准 → 周转/毛利档标"仅参考";缺采购周期 → 相关结论标"待数据";
5. 客户版 → 走 [[报告导出系统_report-export_v1.0]] 出 0 术语红黄绿卡。

## 5. 使用边界

不做:全店逐 SKU 明细报告、长篇咨询报告、业绩承诺、替老板做经营决策。
不输出:条码 / 进价裸值 / 供应商名 / SKU 明细(客户版)。

## 6. 引用的数量管理判据

§0 诊断顺序(先数量后金额) · §6.6C 有货率/服务水平 · §6.6D 库存可销天数 · §6.6.O-C 周转率 IR=12×(1−毛利率)÷ITO·0.9–1.5 · §6.6.I 边际周转率 · §6.6.S 库存结构三类红黄绿 · §6.6.O-F④ 采购周期列缺失则失效。
