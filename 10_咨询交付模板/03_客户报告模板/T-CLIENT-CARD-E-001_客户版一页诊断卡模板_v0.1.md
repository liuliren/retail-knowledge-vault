# T-CLIENT-CARD-E-001 · 客户版一页诊断卡模板 v0.1

> **来源**：反向固化自 2026-07-04 花厅坊首诊三张实跑卡（L0 到店作战卡 / L1 老板卡 / L2 店长卡，见 `09_门店案例与项目复盘/乐易购花厅坊店/00_项目总览/`）。本模板 = 其中**发客户的 E 层一页卡**的可复用骨架。
> **受众**：店老板 / 店长（素养不高）。一页读完，30 秒抓住结论。

## 一、一页结构（六段·顺序固定）

```
┌ ① 店名代号 + 诊断日期 + 卡类型（如「DEMO-01店 · 2026-07-XX · 商品力诊断」）
├ ② 一句话总判（大字·人话·动作+数量，如「撤 X 款长尾，腾位补差异化，主推锁死不断货」）
├ ③ 红绿灯表格（3–5 行：维度/小类 × 现状(人话) × 问题 × 动作，🔴撤 🟡观 🟢补 🔵盯）
├ ④ 调改方向（2–3 条，各一行人话：保留什么 / 压缩什么 / 补什么）
├ ⑤ 落地三步（本周做什么 · 谁做 · 怎么算做完）
└ ⑥ 数据边界一行（页脚：「晟果新零售 · 仅供本店 · 数据期间 XX–XX · 进价不准 · 金额仅参考」）
```

## 二、填充规则（E 层铁律）

1. **0 术语 0 条码 0 进价 0 供应商**——ABCZ/SKU/动销率等后厨词不上卡；指标用「指标名(白话注)」内联括注，如「动销率(在卖占比)」。
2. **红黄绿蓝 = 动作语义**，不是好坏评分：🔴红=撤/下架、🟡黄=观/再看一个月、🟢绿=补/差异化、🔵navy=盯/不断货。色即动作，禁挪作他用。
3. **C 级证据禁精确 %**——只有 B 级实证（POS/真单据）可给区间数；现场观察/口头了解只写「强/中/弱」档位，不编「XX%」假精确（07-04 实跑教训）。
4. **客户真名须过授权链**——本卡发客户本人可署门店名；一旦对外（媒体/样板案例）必查《客户名称对外使用授权链清单》，未授权即代号化（P1-GOV-ClientNameAuth-001）。
5. 无数字不判断：定性词必挂结论级数字（占比/件数），但不出 SKU 明细；>10 行清单配 xlsx 附件，不塞卡。
6. 防误杀提示：撤架前必留一行「引流/形象款先保护，撤前店长确认是真不动、不是缺货」。

## 三、与 report-export / mdcard 引擎对接

**渲染管线**（不改引擎，本模板只是内容契约）：
`card.json（_data/）→ gen_card.py 编译 A/B HTML → render.py 渲染 PNG`，引擎在 `/Users/davidliu/KnowledgeBase/.claude/skills/report-export/cards/`；受众分层与存放规范走 `/mdcard` v1.1（本卡 = mdcard 五层里的 **L1 老板卡**，PNG 与源 md 同目录同名，密级走 frontmatter 不拆目录）。

**card.json 字段 → 模板槽位映射表**（据 `_data/` 4 份黄金样例归纳，字段齐全一致）：

| card.json 字段 | 本模板槽位 | 说明 |
|---|---|---|
| `store` / `category` / `diag_type` | ① 店名代号+日期行 | store 发客户可真名；对外须代号 |
| `action_title` | ② 一句话总判 | 铁律：必须是结论句（动作+数量） |
| `sub` | ② 总判下副句 | 一句人话解释「为什么不伤营收」 |
| `kpis[]`（action/value/unit/desc/light） | ③ 红绿灯表格 | light ∈ remove/observe/add/hold = 🔴🟡🟢🔵 |
| `pareto`（segments/legend/caption） | ③ 表格旁佐证图 | 销售贡献分布，证明「撤尾部不伤营收」 |
| `remove`（groups[].tag/why/skus） | ④ 调改方向·撤 | tag 必取自 IssueTag 体系，不自创 |
| `observe` / `add` / `hold` | ④ 调改方向·观/补/盯 | `add.logic` = 六哥的 10% 判断位，不下放 AI |
| `b`（lead/why/steps[]/closing） | ⑤ 落地三步（B 动作卡） | steps 每条带 light + checklist，店长照做 |
| `footer` | ⑥ 数据边界行 | 固定免责声明，无 logo |

## 四、示例（DEMO 代号 + 虚构区间值·禁用真实客户数字）

> **DEMO-01店 · 2026-07-XX · 商品力诊断**
> **总判：撤 4–6 款长尾，腾位补差异化，主推锁死不断货。**
>
> | 小类 | 现状(人话) | 动作 |
> |---|---|---|
> | 长尾款 | 近三月各只卖 1–2 个，占着排面 | 🔴 撤 4–6 款 |
> | 边缘款 | 偶有动销，看不准 | 🟡 观一个月，不动再撤 |
> | 差异化 | 别家没有的特色款缺位 | 🟢 补 1–2 款（选品另定） |
> | 主推款 | 卖得最好，偶发缺货 | 🔵 盯，每天扫架不断货 |
>
> **落地三步**：① 本周店长核撤架清单（确认真不动，非缺货）；② 采购下单补差异化款；③ 主推款每日巡架。
> _晟果新零售 · 仅供本店 · 数据期间约三个月 · 进价不准 · 金额仅参考_
