# G07 vault 月度审计 SOP v0.1

> **本 SOP**：vault 治理的"月度深扫"操作 / 与 [[G03_Lint]] 周浅扫互补 / IBM 工程化纪律。
>
> **触发**：每月 1 日（自动 cron / 或人工触发）/ 6 哥每月固定 3 小时执行。
>
> **目标**：vault 全量 70+ 核心文件 6 维评分 / 输出 punch list / 客户验证 / 跨业态可复用度评估。

---

## §1. SOP 总流程（6 步）

```
Step 1（30 min）：扫 vault 70+ 核心文件清单 / 自上月变化（新增 / 修改 / 删除）
Step 2（90 min）：6 维审计每文件（M+I+O+A+T+AC = 100 分）
Step 3（30 min）：输出 punch list（按严重度排序 / P0 立即 / P1 当月 / P2 下月）
Step 4（15 min）：与上月对比（升级 / 退步 / 趋势）
Step 5（15 min）：6 哥 sign off（vault 内部治理验收 / 启明/高彬可作为客户业务视角验证对象 / 但不作为内部 sign off 人）
Step 6（15 min）：沉淀 [[数据一致性追溯案例库]] / [[Vault健康仪表]]
```

---

## §2. 6 维审计标准（与 vault 1 月期机制 §2.1 对齐）

| # | 维度 | 标准 | 评分点 |
|---|---|---|---|
| **M** | McKinsey 咨询品质 | Issue tree / Pyramid / MECE / 结论先行 | 20 分 |
| **I** | IBM 工程治理 | Process / Audit Trail / 版本记录 / 反幻觉 | 20 分 |
| **O** | Oracle 数据架构 | Single source / FK 完整 / 字段定义清晰 | 15 分 |
| **A** | Apple 用户中心 | 顾问能复用 / 店长能执行 / 客户能听懂 | 15 分 |
| **T** | Tesla 工业化迭代 | First Principles / 实战 OTA / 版本演进可见 | 10 分 |
| **AC** | Anthropic / Claude | HHH / Constitutional / fact_layer / Karpathy 涌现 | 20 分 |

**判定**：≥ 90 分通过 / 80-89 微调 / < 80 重大升级 / < 60 重写
