# G11 商品库治理审计 SOP v0.1

> **本 SOP**：vault 第 5 个月度审计 / 商品库层（与 G07/G08/G09/G10 协同）。
>
> **触发**：每月 1 日（人工触发）/ 6 哥每月固定 30 分钟执行（W23+ 起步 / 数据少时 15 min）。
>
> **目标**：商品库健康（去重 / 清洗 / 版本一致 / 跨客户匿名化 / FK 完整）。

---

## §1. SOP 总流程（5 步 / 共 30 分钟）

```
Step 1（5 min）：扫商品库 SKU 总数 + 三层分布（vs 上月）
Step 2（10 min）：去重检测（同 sku_id / 同 barcode / 同 sku_name_standard）
Step 3（5 min）：FK 完整性（category_l4 → V4.0 / brand_alias_id / package_unit）
Step 4（5 min）：跨客户匿名化检测（Layer C 是否含客户销售数据 / 必须脱敏）
Step 5（5 min）：沉淀月度报告 + [[Vault健康仪表]]
```

## §3. 与 G07-G10 协同

```
G07 vault 月度审计    内容层（70 文件 6 维评分）
G08 vault 数据治理    数据契约层（每次进库）
G09 工程操作纪律审计  工程操作层（6 大类规则）
G10 Memory 治理审计   记忆层（评分衰减 / 双向打通）
G11 商品库治理审计    商品库层（去重 / FK / 匿名化）⭐ 本 SOP

5 SOP 协同 = vault 完整治理审计闭环：
  内容 (G07) × 数据 (G08) × 操作 (G09) × 记忆 (G10) × 商品库 (G11)
```

---

## §4. 起步阶段（W23-W26）注意点

```
W23：商品库起步 / 数据少 / 审计 ~15 min
W24：Layer B 填实（V4.0 308 L4）/ 重点检测 FK
W25：Layer C 花厅坊起步 / 重点检测客户匿名化
W26：Layer A OFF 食品起步 / 重点检测重复 SKU（OFF vs Layer B 重叠）
```

- ☐ 跨客户匿名化 0 残留
