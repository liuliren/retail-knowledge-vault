## 2. 扫描项目（每月必跑，共 5 项）

### S1 — 软归档未物理迁移
```bash
grep -rl "^status: archived" --include="*.md" . | \
grep -v "05_审计与档案\|99_归档\|99_原始素材\|_RESUME"
```
**判据**：命中 → C档（列清单确认 → git mv → commit）

---

### S2 — Claude 输出区 30 日+ 积压
```bash
find "98_AI协作中枢/01_Claude_Code/Claude输出区" -name "*.md" | \
  awk -F'/' '{print $NF}' | grep -E "^[0-9]{4}-[0-9]{2}-[0-9]{2}" | \
  awk -F'_' '{if ($1 < "CUTOFF") print $0}'
```
> CUTOFF = 当月28日往前30天的日期，格式 `2026-07`

**判据**：超过 30 天的文件 → 归入 `Claude输出区归档/YYYY-MM/`

**豁免**（以下即便超30天也保留在输出区）：
- `ready_to_publish` / `signed` 状态的媒体稿（待六哥手动发布）
- 文件名含 `_draft` 且仍在当前主线任务中的草稿
- `htf_*.csv` 等品类底座数据文件（待专项数据治理处理）

---

### S3 — 治理文件异位（不该在 00_入口与总索引 根目录的文件）
```bash
ls "00_入口与总索引/" | grep -v "^01_\|^02_\|^03_\|^04_\|^05_\|^06_\|^_wiki\|README\|维基主页\|MOC"
```
**判据**：命中孤立文件 → C档迁移到 `05_审计与档案/`

---

### S4 — 设计稿/草案文件（>60天 且 不在归档区）
```bash
find . -name "*.md" | grep -v "99_归档\|99_原始素材\|05_审计\|Claude输出区归档" | \
  grep -E "(草案|设计稿|_draft)" | \
  awk -F'/' '{print $NF}' | grep -E "^202[0-9]-0[1-4]-"
```
**判据**：花厅坊项目内的旧设计草稿 → 评估是否软归档（status改archived + 物理迁移）

---

### S5 — 快速 lint 截面（孤儿/断链/缺 summary）
```bash
# 依赖 G03_Lint；若脚本未实装，改用 grep 抽样：
grep -rL "^summary:" --include="*.md" "01_科学零售方法论/" | head -10
```
**判据**：缺 summary 的 KB 页 → A档补写（直接做）

## 3. 执行四档映射

| 发现疑似客户数据裸值 | D | 立即停止+报告 |
