# 诊断输出质量控制规范

## 一、文件定位

### 1.1 本文件是什么

本文件规范商品力诊断输出中的**判断质量、证据等级、结论边界、风险表达**，防止把主观判断、未验证假设、模拟案例或不完整数据写成正式结论。

**核心任务：** 让"咨询师能写出客户敢信、不会被打脸、可追溯"的诊断结论。

### 1.2 本文件不是什么

- ❌ 不是客户报告模板（PPT / PDF 模板另立项）
- ❌ 不是 POS 数据清洗规范（[[POS数据清洗与商品标准化规范_v0.1]] 的事）
- ❌ 不是首访执行手册（[[社区超市商品力诊断首访执行手册_v0.1]] 的事）
- ❌ 不是销售包话术（[[三层销售打法_SOP_v0.1]] 等的事）

### 1.3 服务目标

服务三个月生存战 P2 新客户开拓的"**诊断报告交付质量**"——这是客户判断"是否再签项目 / 陪跑"的关键。

---

## 二、适用场景与不适用场景

### 2.1 适用场景

- ✅ 商品力快速诊断报告写作（标准版 2-3 页 / 深度版 5-8 页）
- ✅ 看店有感写作（800-1500 字）
- ✅ 阶段项目诊断报告 / 调改方案
- ✅ 长期陪跑顾问的月度诊断 / 复盘报告
- ✅ 内部诊断结论（不发客户但需要内部记录）

### 2.2 不适用场景

- ❌ 客户对话现场口头表达（应使用 [[客户对话开局工具_v0.1]]）
- ❌ 销售包话术（应使用 [[三层销售打法_SOP_v0.1]]）
- ❌ 内部方法论文件（属于 `01_科学零售方法论`）

## 三、质量控制总原则（5 条不可妥协）

| # | 原则 | 解释 |
|---|------|------|
| 1 | **事实与判断分离** | "X SKU 60 天未动"是事实；"应该淘汰"是判断 |
| 2 | **已验证与待确认分离** | "POS 显示 3 月销售 X 元"是已验证；"猜测客户库存压力大"是待确认 |
| 3 | **初步观察与最终结论分离** | 首访 1 小时给的是初步观察；诊断报告（7-15 天）给的是诊断结论 |
| 4 | **内部表达与客户表达分离** | 内部可写"老板执行力差"；客户版必须写"门店执行节奏待优化" |
| 5 | **不承诺未经验证结果** | 禁止说"做完调改销售提升 X%"；必须说"调改后建议观察 30 天数据" |

## 四、三色标记法

> 用于标注资料采集状态 + 诊断结论可信度。

### 4.1 4 色定义

| 颜色 | 含义 | 使用场景 |
|---|---|---|
| 🟢 **绿色** | 已采集，可直接用于诊断 | 已清洗的 POS 数据；已确认的客户访谈纪要 |
| 🟡 **黄色** | 已采集但需核实 | 客户口述的销售数字；未交叉验证的库存数据 |
| 🔴 **红色** | 未采集，必须补采 | 缺失的核心品类 SKU 数据；缺失的关键访谈 |
| ⚪ **灰色** | 暂不需要（或不影响当前诊断范围）| 非诊断品类的数据；远期才需要的资料 |

### 4.2 标记规则

- 资料索引表的 status 字段必须能映射到三色（参 [[客户项目资料入库标准_v0.1]] §九）
- 诊断报告中**每条结论**应注明背后资料的颜色（如：根据 🟢 销售数据 + 🟡 老板访谈得出...）
- **🔴 红色资料缺失**时**禁止**直接出诊断结论；必须先补采

### 4.3 示例

> 🟢 客户主店休食区在售 SKU 共 280 个（数据来源：POS 90 天明细，已清洗）
> 🟡 客户口述毛利率约 22%（数据来源：老板访谈，未与 POS 交叉验证）
> 🔴 商品档案中 30% SKU 无供应商字段（数据来源：商品档案 .xls，必须补采）

### 4.4 不能误用的情况

- ❌ 不能把 🟡 黄色资料**写成 🟢 绿色**（自欺欺人）
- ❌ 不能把 🔴 红色资料**用于结论**（无源结论）
- ❌ 不能把 ⚪ 灰色资料**冒充 🟢 绿色**（虚假证据）

## 五、P0 / P1 / P2 优先级法

### 5.1 优先级定义

| 级别 | 含义 | 资料缺口示例 | 整改动作示例 |
|---|---|---|---|
| **P0** | 没有它无法诊断 / 必做 | 缺失诊断品类 90% SKU 数据 → 无法诊断 | 立即砍掉死库存 SKU |
| **P1** | 没有它诊断会变浅 / 应做 | 缺失店长访谈 → 诊断片面 | 优化儿童零食陈列位 |
| **P2** | 不影响首版诊断 / 可做 | 缺失竞品深度调研 → 后续可补 | 季度性 SKU 调整 |

### 5.2 资料缺口优先级判断（参 [[客户项目资料入库标准_v0.1]] §九 9 种状态）

| 资料状态 | 优先级 | 后续动作 |
|---|---|---|
| 🔴 红色 + 涉及诊断品类核心数据 | **P0** | 立即触发二访补采 |
| 🔴 红色 + 涉及非诊断品类 | **P2** | 可入二访清单 |
| 🟡 黄色 + 涉及核心结论的关键依据 | **P1** | 触发交叉验证 |
| 🟡 黄色 + 仅作为辅助参考 | **P2** | 备注存疑即可 |
| ⚪ 灰色 | 不进入优先级排序 | — |

### 5.3 整改动作优先级判断

| 动作类型 | 优先级标准 |
|---|---|
| 立即砍死库存 / 立即补缺货 A 类 | **P0** |
| 调整黄金陈列位 / 优化端架主题 | **P1** |
| 季度性 SKU 调整 / 长期客群分层调整 | **P2** |

### 5.4 二访任务优先级判断

参 [[社区超市商品力诊断首访执行手册_v0.1]] §十一.1：

- **P0**：必须二访补采的核心数据 / 关键访谈
- **P1**：应当二访补采的次要资料
- **P2**：时间允许才补采的辅助资料

## 六、证据等级法（A/B/C/D 4 级）

> **核心：不同证据等级 → 不同输出语气与边界。**

### 6.1 4 级定义

| 等级 | 含义 | 适用范围 |
|---|---|---|
| **A 级** | 系统数据 + 现场照片 + 访谈三重一致 | 可作为正式诊断结论；可写入客户报告 |
| **B 级** | 系统数据**或**现场照片支持 | 可作为初步判断；客户报告需标"基于 X 数据" |
| **C 级** | 只有访谈或主观观察 | 只能作为待验证假设；客户报告慎用，必须与 A/B 级证据交叉 |
| **D 级** | 未经验证 / 仅凭顾问直觉 | **禁止**写成结论；只能作为内部待办 |

### 6.2 各等级能输出 / 不能输出

| 等级 | 能输出 | 不能输出 |
|---|---|---|
| A 级 | 正式诊断结论 / 量化判断 / 客户可执行建议 | — |
| B 级 | 初步判断 / 方向性建议 + 标注"基于 X 数据" | 量化承诺 / 量化结论 |
| C 级 | 待验证假设 + "需要 X 数据进一步验证" | 任何写入客户报告的"独立结论" |
| D 级 | 内部待办 / 私人判断 | 任何对外表达 |

### 6.3 等级判断示例

| 判断内容 | 证据 | 等级 |
|---|---|---|
| "休食区死库存 SKU 共 35 个，金额 X 元" | POS 数据（A） + 现场照片（A） + 老板访谈（A）三重一致 | **A 级** |
| "猜测客户客群中老年占 60%" | 仅老板访谈说"老人多"（C） | **C 级** |
| "建议立即砍掉这 5 个 SKU" | A 级证据 → A 级结论 | A 级 |
| "感觉客户陈列资源浪费严重" | 仅顾问观察（D） + 无照片支撑 | **D 级**（不可写入报告）|

## 七、结论边界法

### 7.1 首访只能输出（5 类）

| 输出类型 | 示例 |
|---|---|
| 初步观察 | "我注意到您的儿童零食区陈列在角落" |
| 疑似问题 | "可能存在儿童零食陈列资源未被充分利用" |
| 待验证方向 | "需要看小学放学时段的客流和儿童零食销售数据" |
| 下一步资料需求 | "需要 30 天 SKU 销售数据 + 儿童零食区现场照片" |
| 快速诊断建议 | "建议先做 1 周快速诊断，验证假设" |

### 7.2 首访不能输出（5 类）

| 不能输出 | 原因 |
|---|---|
| ❌ 最终整改方案 | 没数据，凭直觉 = 不专业 |
| ❌ 具体增长承诺 | 跨越证据边界 |
| ❌ 完整品类调整方案 | 需要诊断报告（7-15 天）|
| ❌ 人员考核结论 | 不在商品力专业范围 |
| ❌ 大规模调改承诺 | 越级承诺 |

### 7.3 诊断报告（快速诊断）能输出 / 不能输出

| 能输出 | 不能输出 |
|---|---|
| ABCD 分类清单（A 级证据） | 调改后销售提升数字（无证据）|
| TOP 5 主力 SKU + TOP 5 死库存（A 级证据）| 调改后毛利提升数字（无证据）|
| 1 个核心品类的 IPC 价格带分析（A 级证据）| 跨品类整改方案（超出标准版范围）|
| 调改建议方向（B 级证据，标注"建议方向"）| 必须按建议执行的强承诺 |

### 7.4 阶段项目报告 / 长期陪跑报告 边界

参 [[商品力顾问_报价单_v0.1]] §九 不做事项清单。本文件不展开。

## 八、客户版与内部版表达区分

### 8.1 内部版 vs 客户版的差异

| 维度 | 内部版可以写 | 客户版必须写 |
|---|---|---|
| 老板执行力 | "老板对数据敏感度低，决策凭直觉" | "门店决策机制以经验判断为主，建议引入数据辅助" |
| 店长能力 | "店长不懂 ABCD 分类" | "店长在商品力专业方法论上有提升空间" |
| 数据缺陷 | "客户 POS 字段错乱，30% SKU 无毛利字段" | "数据基础设施待完善，建议优先补全字段" |
| 执行短板 | "员工执行力差，方案落不了地" | "门店执行节奏与方案落地速度待匹配" |
| 真实问题（直接版）| "陈列资源浪费严重，30% 端架长期不动" | "陈列资源使用频率有提升空间，部分端架可重新规划" |

### 8.2 表达转换规则

| 内部版 | 客户版 |
|---|---|
| 否定式（"做错了"）| 改善式（"有提升空间"）|
| 直接判断（"应该砍"）| 建议式（"建议评估是否保留"）|
| 量化批评（"30% 是死货"）| 量化中性（"30% 处于低动销状态"）|
| 责任归咎（"老板凭直觉"）| 系统建议（"建议引入数据辅助"）|

### 8.3 客户版必须遵守

- ✅ 专业克制（不夸大）
- ✅ 可沟通（不否定老板经验）
- ✅ 可接受（不刺痛客户面子）
- ✅ 有行动建议（不只指出问题）
- ❌ 不直接否定老板既有决策

## 九、诊断结论写法标准

### 9.1 5 段式写法（每条诊断结论必备）

```
观察事实（事实是什么 + 证据来源 + 等级） →
证据来源（来自哪份资料） →
初步判断（推断了什么） →
影响风险（如果不处理会怎样） →
下一步验证 / 建议动作
```

### 9.2 5 个示例（不同等级 + 不同品类）

#### 示例 1（A 级 / 休食死库存）

> **观察事实**：客户主店休食区在售 280 个 SKU 中，35 个 SKU 60 天未动销（占比 12.5%）。
> **证据来源**：POS 90 天销售明细（🟢 已清洗）+ 现场照片（🟢 已采集 + 已核实）。证据等级 **A 级**。
> **初步判断**：这 35 个 SKU 属于 D 类（零销售），符合 ABCD 分类的"必砍"标准。
> **影响风险**：占用约 X% 货架位置 + 占用约 X 元库存资金（待商品档案校准后量化）。
> **下一步建议**：建议立即砍掉这 35 个 SKU；将货架位让给主力品类。

#### 示例 2（B 级 / 价格带）

> **观察事实**：客户主店休食区 8-25 元价格带 SKU 占比 35%（低于建议参考值 65%）。
> **证据来源**：POS 90 天销售明细（🟢）。无现场对照（🟡）。证据等级 **B 级**。
> **初步判断**：核心客群价格带可能未被充分承接。
> **影响风险**：核心价格带客户可能选不到合适的 SKU，转向其他门店。
> **下一步建议**：进一步对照大湾区社区超市同业态数据 + 现场再核查；如确认偏低，建议增加 8-25 元 SKU 数。

#### 示例 3（C 级 / 客群结构）

> **观察事实**：根据老板访谈，客户主店中老年客群占比"约 60%"。
> **证据来源**：老板访谈（🟡 未与现场客流交叉验证）。证据等级 **C 级**。
> **初步判断**：本店可能符合"中老年高占比社区超市"画像。
> **影响风险**：如判断准确，凉果陈皮等中老年品类应作为差异化锚点；如判断偏差，可能错配陈列资源。
> **下一步验证**：建议做 1 周客流时段观察 + 中老年品类销售占比交叉验证。**本判断不可作为独立诊断结论使用，须与 A/B 级证据交叉。**

#### 示例 4（A 级 / 端架陈列）

> **观察事实**：客户主店 13 个端架中，6 个端架陈列商品在过去 90 天未发生主题更换。
> **证据来源**：现场照片（🟢 全 13 端架照片）+ 老板访谈（🟢 老板自述"端架一直是这些货"）。证据等级 **A 级**。
> **初步判断**：陈列资源使用频率偏低，主题营销节奏未充分发挥。
> **影响风险**：节庆 / 季节性销售机会未被承接（如端午、煲汤季）。
> **下一步建议**：建议建立月度端架主题轮换机制；可参考 [[大湾区全年节庆节奏表_MC-GBA-05]] 的 6 张作战卡。

#### 示例 5（D 级 / 不可作为结论）

> **观察事实**（**D 级，仅顾问内部待办**）：感觉客户老板对儿子接班焦虑较大。
> **证据来源**：仅顾问 1 次访谈中老板提到"我做不动了"（无其他证据）。证据等级 **D 级**。
> **结论边界**：**本判断仅为内部待办**，不写入诊断报告。
> **下一步**：在后续访谈中通过开放式问题验证（如"您未来 3 年最希望这家店变成什么样？"）。

## 十、不同证据等级下的表达模板

### 10.1 A 级表达模板

```
"根据 [证据来源]（A 级），我们判断 [具体结论]，
建议 [可执行行动]，预计 [时间窗口] 内完成。"
```

**适用：** 客户版 + 内部版均可。

### 10.2 B 级表达模板

```
"基于 [证据来源]（B 级），初步判断 [方向性结论]。
此判断尚需 [需要的额外证据] 进一步验证。
建议先 [小范围验证动作]，再决定是否扩大执行。"
```

**适用：** 内部版可独立使用；客户版必须明示"基于 X 数据"。

### 10.3 C 级表达模板

```
"根据 [访谈 / 主观观察]（C 级），我们注意到 [现象描述]。
此判断尚未经过数据 / 现场交叉验证，**仅为待验证假设**。
建议下一步采集 [需要的数据 / 现场观察] 进行核实。"
```

**适用：** **仅内部版**；客户版**禁止**作为独立结论。

### 10.4 D 级表达模板

```
[仅记录于内部待办文件，不写入任何对外文档]

待办：[假设描述]
触发验证条件：[何时验证]
责任人：[顾问代号]
```

**适用：** **仅内部待办**；任何对外文档**禁止出现**。

## 十一、风险红线（8 项）

| # | 红线 | 违反后果 |
|---|------|------|
| 1 | ❌ 不用 C / D 级证据写最终结论 | 诊断报告基础崩塌 |
| 2 | ❌ 不用单张照片下完整判断 | 证据不足 + 误诊 |
| 3 | ❌ 不用老板一句话替代数据 | C 级当 A 级用 |
| 4 | ❌ 不编造增长数字（销售提升 / 毛利提升 / 库存下降）| 跨越证据边界 + 客户信任崩塌 |
| 5 | ❌ 不把模拟客户写成真实案例 | 商业伦理 + 法律风险 |
| 6 | ❌ 不把花厅坊未成熟样板写成已完成案例：5 月底前**只能用"正在做 / 正在打样 / 正在形成样板"**语气 | 违反卷宗 + 销售包红线 |
| 7 | ❌ 不把初步观察包装成正式诊断 | 跨越结论边界 |
| 8 | ❌ 不把客户版写成批评报告 | 客户关系断裂 |

## 十二、诊断输出复查清单

> 每份诊断输出（看店有感 / 快速诊断报告 / 阶段项目报告）发出前必须完整勾选。

### 12.1 内容复查（10 项）

- ☐ 每条结论是否有证据支撑（不能无源）
- ☐ 每条结论的证据等级是否明确标注（A/B/C/D）
- ☐ C 级证据是否仅作为"待验证假设"使用（未冒充独立结论）
- ☐ D 级证据是否未出现在任何对外文档
- ☐ 不确定项是否标注"待确认"
- ☐ 是否区分客户版和内部版（措辞 / 直白度）
- ☐ 客户版是否符合 §八.3 4 个必须
- ☐ 是否有过度承诺（增长 / 毛利 / 库存数字）
- ☐ 是否涉及客户敏感信息（姓名 / 报价 / 商业秘密）
- ☐ 是否能追溯到原始资料或现场观察（资料索引表 file_id）

### 12.2 表达复查（5 项）

- ☐ 是否使用 §九.1 五段式（观察 → 证据 → 判断 → 风险 → 建议）
- ☐ 内部版是否有"老板执行力差"等不适合外发的措辞（必须先转换）
- ☐ 是否避免直接否定老板经验
- ☐ 是否避免方法论术语堆砌（参 [[客户对话开局工具_v0.1]] §四）
- ☐ 花厅坊样板表达是否使用"正在做 / 正在打样 / 正在形成样板"

### 12.3 复查不通过的处理

- 任意 1 项内容复查不通过 → **不允许发出**；返工至通过
- 任意 1 项表达复查不通过 → **不允许发客户版**（内部版可保留）
