# 社区超市商品力诊断首访执行手册

## 一、文件定位

### 1.1 本文件是什么

本文件是「**社区超市商品力快速诊断**」（首单入口·定价按 Q1 口径 988 首诊，合同口径见 [[服务合同模板_988确认单与陪跑协议_v1.0]] 模板一；原为 [[商品力顾问_报价单_v0.1]] §五 第一阶，已由其取代，2026-07-07）的**首访现场执行 SOP**。

**核心任务：** 把销售包对话转化为可执行的首访动作；把首访产出转化为诊断报告输入。

**适用咨询师：** 六哥本人 + 未来培养的助理咨询师。

### 1.2 本文件不是什么

- ❌ 不是完整咨询项目方案（那是阶段项目 / 长期陪跑的事）
- ❌ 不是售前销售包（那是销售包 3 模板的事）
- ❌ 不是客户项目资料入库标准（另文件）
- ❌ 不是 POS 数据清洗规范（另文件）
- ❌ 不是诊断输出质量控制规范（另文件）

### 1.3 服务目标

服务三个月生存战 P2 新客户开拓的"**首访 → 首单**"转化主战场。

## 二、适用场景与不适用场景

### 2.1 适用场景

| 客户类型 | 典型场景 |
|---|---|
| 单店社区超市老板 | 1 家店、500-2000㎡、家族经营 |
| 1-3 家门店小连锁 | 各店差异显著、店长能力参差 |
| 商品结构混乱但有改善意愿 | 老板已经知道"得改"但缺方法 |
| 对咨询谨慎但愿意先试一次 | 不接受 6-12 万半年合同，但愿意花 3-8 千试一次 |

### 2.2 不适用场景（建议拒绝首访）

- ❌ 30+ 家店连锁（超出单兵交付能力）
- ❌ 大卖场 / 精品超市 / 折扣店 / 便利店（业态差异大）
- ❌ 想要"一次性翻倍营业额"的客户（错位预期）
- ❌ 不愿付任何费用、只想"先试效果"的白嫖客户

## 三、首访执行总原则

> 5 条不可妥协的执行原则。

| # | 原则 | 解释 |
|---|------|------|
| 1 | **先观察，后判断** | 进店先 5-10 分钟扫一眼，再做开场对话；判断必须有现场观察支撑 |
| 2 | **先诊断，后方案** | 首访只识别问题，不给完整方案；完整方案是诊断报告的事 |
| 3 | **先小入口，后大项目** | 推进的目标是 3-8 千的快速诊断（第一阶），不是 6-12 万陪跑（第三阶） |
| 4 | **不承诺未经验证结果** | 禁止说"3 个月让您营业额涨 30%"等具体数字承诺 |
| 5 | **不把首访当成完整咨询项目** | 首访是销售流水线中段，不是终点；离场前不签合同 |

## 四、首访前准备清单

### 4.1 客户基础信息（拜访前 24 小时确认）

- ☐ 门店名称 / 地址 / 营业面积（待确认）
- ☐ 营业年限 / 老板姓名（去标识化）/ 店长姓名（去标识化）
- ☐ 主要业态定位（社区超市 / 综合体小超 / 工业区便利店）
- ☐ 周边核心竞品 1-3 家（用做现场对照）
- ☐ ERP 系统名称 / 数据导出可行性（待确认）
- ☐ 拜访时间 / 时长 / 参与人

### 4.2 资料清单（首访当天客户应提供，理想情况）

- ☐ 近 30-90 天 SKU 销售数据（POS .xls 导出）
- ☐ 商品档案（条码 / 品名 / 价格 / 供应商）
- ☐ 当前库存数据（金额 + 周转天数估算依据）
- ☐ 现场已有照片（如有）
- ☐ 老板自述最头疼的 1-3 个问题

### 4.3 顾问携带工具

- ☐ 老板访谈问题库打印版（参 §七，挑 8-12 题）
- ☐ [[5分钟门店诊断卡_T-08]] A5 卡片或手机笔记
- ☐ 名片（不带 PRD / SaaS 介绍 / 报价单）
- ☐ 笔记本 + 笔
- ☐ 拜访前 5 分钟翻阅 [[客户对话开局工具_v0.1]] §二 检查清单
- ☐ 手机充满电（拍照 + 录音备份）

### 4.4 拍照和资料使用边界

- ✅ 客户允许范围内拍照，**不偷拍**
- ✅ 拍照前告知"我拍这些用于您的诊断报告，不外传"
- ❌ 不拍客户的财务报表 / 银行流水 / 员工身份证等敏感信息
- ❌ 不在客户不在场时擅自查看 ERP / 数据库
- ❌ 客户离开诊断关系后，所有原始资料 30 天内移交或销毁

### 4.5 数据不全时的降级方案

| 客户实际情况 | 降级动作 |
|---|---|
| 客户没准备数据 | 以现场观察为主 + 老板访谈深度问题 + 现场拍照 ≥ 30 张 |
| 客户拒绝提供 SKU 数据 | 用 [[5分钟门店诊断卡_T-08]] 现场出 9 分制评分；不强求 |
| 客户没 ERP / POS | 重点访谈 + 凭专业能力做初判；告知"详细诊断需要 POS 数据" |
| 客户希望"现场就出方案" | 礼貌拒绝："首访不出方案，是为了避免拍脑袋。诊断需要 7-15 天" |

## 五、首访任务包（T01-T12）

> 12 个任务包，每个含目标 / 动作 / 标准 / 成果 / 验收 / 常见问题 / 优先级。
> **优先级口径：P0 = 必做；P1 = 应做；P2 = 时间允许做，否则入二访。**

### T01 客户基础信息确认（P0）

| 项 | 内容 |
|---|---|
| **任务目标** | 在拜访前 + 首访开场前 5 分钟，完成客户基础信息口径核对 |
| **执行动作** | 电话/微信确认门店地址、面积、ERP 系统、老板/店长姓名、首访参与人、时长 |
| **采集标准** | 参 [[客户项目资料入库标准_v0.1]] 基础信息字段（待该文件落盘后明确）|
| **输出成果** | 1 份《首访前客户基础信息核对表》（A4 一页）|
| **验收标准** | 基础信息 8 项中 ≥ 6 项有明确答案 |
| **常见问题** | 客户对面积、SKU 数等"凭感觉"答；不必苛求精确，留待二访校准 |

### T02 老板访谈（P0）

| 项 | 内容 |
|---|---|
| **任务目标** | 识别老板真实痛点 + 决策风格 + 长期目标 |
| **执行动作** | 从 §七 问题库挑 8-12 题；不打断老板抱怨；记录关键数据 |
| **采集标准** | 访谈纪要 ≥ 1500 字；含 ≥ 3 个客户主动透露的具体数据点 |
| **输出成果** | 1 份《老板访谈纪要》|
| **验收标准** | 含 TOP1-3 痛点回应 + 至少 1 个 TOP4 家族接班相关线索（如有）|
| **常见问题** | 老板抱怨过多导致跑题；用"您说说看您最近最头疼的事"重新聚焦 |

### T03 店长 / 员工访谈（P1）

| 项 | 内容 |
|---|---|
| **任务目标** | 交叉验证老板说法 + 识别执行落地阻力 |
| **执行动作** | 与店长在卖场或休息区单独访谈 15-30 分钟（不要在老板面前）|
| **采集标准** | 访谈纪要 ≥ 800 字 |
| **输出成果** | 1 份《店长访谈纪要》|
| **验收标准** | 含 ≥ 1 个店长视角的真实问题（与老板视角不同）|
| **常见问题** | 店长在老板面前不敢讲真话；尽量找机会单独谈 |

### T04 经营数据采集（P0）

| 项 | 内容 |
|---|---|
| **任务目标** | 拿到诊断需要的销售/毛利数据 |
| **执行动作** | 要求客户导出 POS 数据（标准 90 天）；现场协助导出更稳妥 |
| **采集标准** | 标准期间近 90 天 SKU 级；最低降级到月度汇总 |
| **输出成果** | 客户原始 POS .xls（按命名规范保存到 `99_原始素材/POS_导出/`，参 [[客户项目资料入库标准_v0.1]]）|
| **验收标准** | 覆盖核心诊断品类（休食 / 水饮 / 调味 等）≥ 70% SKU |
| **常见问题** | 客户数据导出权限不在自己手里；预约下次专项导出，列入 T12 二访 |

### T05 商品档案采集（P1）

| 项 | 内容 |
|---|---|
| **任务目标** | 拿到 SKU 主数据 |
| **执行动作** | 要求客户导出商品档案表（条码/品名/规格/价格/供应商）|
| **采集标准** | 至少含条码 / 品名 / 售价 / 供应商 4 个核心字段 |
| **输出成果** | 客户原始商品档案 |
| **验收标准** | 覆盖核心诊断品类 ≥ 70% SKU；与 T04 POS 数据可关联 |
| **常见问题** | 商品档案与 POS SKU 不对应；记录差异，留待 `/posclean` skill 处理（`13_数据分析与工具脚本/POS清洗库_v0.1/`；原 [[POS数据清洗与商品标准化规范_v0.1]] 已由其取代，2026-07-07）|

### T06 库存数据采集（P1）

| 项 | 内容 |
|---|---|
| **任务目标** | 拿到当前库存（金额 + SKU 级）|
| **执行动作** | 要求客户导出库存表 |
| **采集标准** | 含库存金额 + 库存数量 + 最近一次盘点日期 |
| **输出成果** | 客户原始库存 |
| **验收标准** | 含金额 + 含 SKU 级（不只是品类汇总）|
| **常见问题** | 库存数据未实时；以导出时刻为准，记录导出时间 |

### T07 现场照片采集（P0）

| 项 | 内容 |
|---|---|
| **任务目标** | 覆盖 12 个区域的整体照片 |
| **执行动作** | 每个区域 ≥ 3 张照片（参 §八 拍照规范）|
| **采集标准** | 至少 30 张照片，按命名规范命名 |
| **输出成果** | 30+ 张照片，按 §八 命名规则整理 |
| **验收标准** | 12 区域 ≥ 10 个被覆盖 |
| **常见问题** | 客流高峰拍不到全景；选客流低峰时段补拍 |

### T08 重点品类细拍（P0）

| 项 | 内容 |
|---|---|
| **任务目标** | 诊断对象品类的深度照片 |
| **执行动作** | 诊断品类（休食 / 水饮 / 调味 二选一）每组货架正面平拍 + 价签特写 + 端架 |
| **采集标准** | 每组货架 ≥ 5 张（正面平拍 + 上中下三层 + 价签特写）|
| **输出成果** | ≥ 20 张深度照片 |
| **验收标准** | 诊断品类全部货架被覆盖 |
| **常见问题** | 货架灯光暗；闪光灯 / 手机补光；如仍不清晰，列入 T12 二访 |

### T09 陈列资源采集（P1）

| 项 | 内容 |
|---|---|
| **任务目标** | 清点端架、堆头、冷柜、收银台陈列资源 |
| **执行动作** | 现场清点 + 拍照 + 记录使用状态（节庆主题 / 永久陈列 / 闲置 / 关联陈列）|
| **采集标准** | 端架数量 / 堆头数量 / 冷柜数量 / 收银台陈列资源 全部清单 |
| **输出成果** | 1 份《陈列资源清单表》 |
| **验收标准** | 含资源数量 + 使用状态 + 主题 |
| **常见问题** | 客户对"端架/堆头"定义不清；用拍照辅助 + 现场指认 |

### T10 商圈与竞品采集（P2）

| 项 | 内容 |
|---|---|
| **任务目标** | 周边竞品调研 |
| **执行动作** | 500m 内核心竞品 + 1km 内大型竞品 拍照 + 客流观察 |
| **采集标准** | ≥ 5 家竞品照片（外观 + 入口 + 核心品类货架）|
| **输出成果** | 1 份《商圈与竞品照片包》 |
| **验收标准** | 核心竞品（赵一鸣 / 钱大妈 / 折扣店 / 大型超市）≥ 3 家 |
| **常见问题** | 首访时间不够；首访 P2，可入二访 P1 |

### T11 首访复盘（P0）

| 项 | 内容 |
|---|---|
| **任务目标** | 当天 / 次日做首访复盘 |
| **执行动作** | 填写《首访验收表》（参 §十）+ 写《看店有感》800-1500 字（参 [[三层销售打法_SOP_v0.1]] §四.2）|
| **采集标准** | 参 §十 100 分制 |
| **输出成果** | 1 份《首访验收表》+ 1 份《看店有感》|
| **验收标准** | ≥ 70 分（参 §十）|
| **常见问题** | 复盘流于形式；纪律是 24 小时内必须写完 |

### T12 二访待办识别（P0）

| 项 | 内容 |
|---|---|
| **任务目标** | 基于首访验收结果，列二访清单 |
| **执行动作** | 参 §十一 二访补采机制 |
| **采集标准** | 每项含 P0/P1/P2 优先级 + 触发时机 |
| **输出成果** | 1 份《二访补采清单》|
| **验收标准** | 覆盖首访验收 < 90 分的所有缺失项 |
| **常见问题** | 首访 ≥ 90 分时不需要二访；明示 |

## 六、首访现场流程标准（S0-S7）

| 阶段 | 时长 | 参与人 | 关键动作 | 输出物 | 风险提醒 |
|---|---|---|---|---|---|
| **S0 首访前准备** | 拜访前 24 小时 | 顾问 | 完成 §四 准备清单；T01 基础信息确认 | 《首访前客户基础信息核对表》| 不准备 = 现场被动 |
| **S1 开场与定调** | 0-5 分钟 | 顾问 + 老板 | 寒暄不卖；进店扫 5-10 分钟（[[5分钟门店诊断卡_T-08]] 简化版）；用现场观察反推切入点（[[客户对话开局工具_v0.1]] §五.5）| 现场口头观察 1-2 句 | 不要立刻谈方法论 |
| **S2 老板访谈与数据确认** | 5-25 分钟 | 顾问 + 老板 | 从 §七 问题库挑 8-12 题（T02）；同步确认 T04-T06 数据导出可行性 | 访谈纪要 + 数据导出承诺 | 老板抱怨过多易跑题 |
| **S3 现场走访采集** | 25-55 分钟 | 顾问 + 店长（如可能）| 12 区域观察（§八）；T07 全店照片 + T08 重点品类细拍 + T09 陈列资源 | 30+ 张照片 + 陈列资源清单 | 不要现场对店员讨论方案 |
| **S4 商品结构初步判断** | 55-70 分钟 | 顾问 + 老板 | 回办公区 / 收银台旁 / 店外咖啡馆，与老板坐下来讨论 1-2 个具体观察 | 现场口头反馈 1-2 个观察 | 不给完整方案；不承诺数字 |
| **S5 快速诊断产品说明** | 70-80 分钟 | 顾问 + 老板 | 介绍《社区超市商品力快速诊断》（定价与合同口径见 [[服务合同模板_988确认单与陪跑协议_v1.0]]·Q1 口径 988 首诊）；标准版 / 深度版差异 | 报价区间口头说明 | 不要顺势报半年陪跑 |
| **S6 下一步成交推进** | 80-90 分钟 | 顾问 + 老板 | 用 §十二 话术推进；不强推；约 7-14 天后再聊；离场前留钩子 | 下次见面约定 | 不强推 |
| **S7 首访后复盘** | 拜访后 24 小时 | 顾问独立 | T11 首访复盘 + T12 二访待办 | 《首访验收表》+《看店有感》+《二访补采清单》| 24 小时内必须写完 |

**节奏纪律：**
- ❌ 不允许超过 120 分钟（疲劳期判断质量下降）
- ❌ 不允许在第一次见面就报价
- ❌ 不允许跳过现场走访（"看店有感"必须含具体观察）

## 七、老板访谈问题库（30 题 × 11 维度）

> **使用纪律：** 不要全部问完。**首访 8-12 题为限**，按客户类型挑选。问题是工具，不是流程。

### 7.1 销售增长（3 题）

| # | 问题 | 提问目的 | 可能暴露 | 后续诊断线索 |

## 八、现场观察与照片采集标准

### 8.1 12 区域观察清单

| 区域 | 看什么 | 怎么拍 | 合格照片标准 | 对应商品力问题 |

### 8.2 拍照规范

| 项 | 标准 |
|---|---|
| 分辨率 | 至少 1080p（手机默认即可）|

## 十一、二访补采机制

### 11.1 二访触发条件

- 首访验收 < 90 分（任一类未达满分）
- 客户在首访时承诺补提供数据但未提供
- 现场照片关键区域缺失
- 老板未到场（仅店长接待）等核心访谈缺失

### 11.2 首访资料完整性复盘流程

1. T11 首访当天/次日填《首访验收表》
2. 识别 < 满分项（每项扣分原因）
3. 列入《二访补采清单》
4. 标注 P0/P1/P2 优先级

### 11.3 二访补采清单字段

| 字段 | 说明 |
|---|---|
| # | 编号 |
| 缺失项 | 具体缺什么（如"赵一鸣竞品照片缺失"）|
| 类别 | 数据 / 照片 / 访谈 / 商圈 / 其他 |
| 优先级 | P0 必补 / P1 应补 / P2 可补 |
| 触发时机 | 立即 / 1 周内 / 月底前 |
| 补采方式 | 现场二访 / 客户远程发送 / 第三方采集 |
| 状态 | 未启动 / 已请求 / 已收到 / 已核实 |

### 11.4 二访前沟通话术

> "Z 老板，上次首访我整理了一份材料，发现还差 X / Y / Z 几项。**这些不补齐，诊断报告会有盲区。** 您看下次什么时候方便，我过去 1 小时把这几项补上？"

### 11.5 二访执行清单（参考）

- ☐ 补全缺失数据 / 照片 / 访谈
- ☐ 现场对照首访初步观察，验证或修正
- ☐ 与店长（如首访未访）单独深谈
- ☐ 现场协助客户导出 POS / 库存（如客户技术不熟）

### 11.6 二访完成后的复查

- 重新填《首访验收表》
- 总分 ≥ 90 分 → 启动诊断
- < 90 分 → 评估是否第 3 次访问 / 还是放弃该客户

## 十二、快速诊断成交衔接

> 首访结束后如何自然转入《社区超市商品力快速诊断》。

### 12.1 衔接原则

- 不强推
- 不夸大
- 不承诺结果
- 强调先小范围验证
- 强调诊断费是买"看清问题"，不是买神奇方案
- 不一上来推长期陪跑

### 12.2 衔接话术（4 句）

| 客户类型 | 话术 |
|---|---|
| 单店 / 现金流紧张 | "Z 老板，今天聊得挺有收获。我回去 3-5 天会写一份'看店有感'发您，800-1500 字，免费。如果您看完觉得有价值，我们可以聊一下'商品力快速诊断'——3-5 千，1 周时间，给您挑出您店里 5 个最赚钱、5 个最不赚钱的 SKU。**这个诊断费是买清楚问题，不是买神奇方案。**" |
| 小连锁 / 想看更深 | "如果您愿意先看一个完整的诊断样本，我可以做深度版——6-8 千，2 周时间，把您一个核心品类彻底诊断一遍。" |
| 担心诊断费白花 | "我的诊断费有抵扣机制——**您 30 天内决定签项目或半年陪跑，这个 X 千 100% 抵进去**。等于您没多花。" |
| 留白 / 不强推 | "今天先聊到这。我回去写'看店有感'，您先看，**不着急回复**。" |

详细话术参 [[三层销售打法_SOP_v0.1]] §五.4 三阶报价成交话术 + [[服务合同模板_988确认单与陪跑协议_v1.0]]（Q1 定价口径；原 [[商品力顾问_报价单_v0.1]] §五 已被取代，2026-07-07）。

## 十三、风险红线（10 项）

> **以下行为一旦发生即触发自我警报，必须立即停止。**

| # | 红线 | 违反后果 |
|---|---|---|
| 1 | ❌ **不当场承诺增长数字**（"3 个月让您涨 30%"）| 客户失去信任 |
| 2 | ❌ **没看数据不下最终结论** | 暴露专业不严谨 |
| 3 | ❌ **不直接否定老板经验**（"您这个做错了"）| 关系断裂 |
| 4 | ❌ **不把方法论讲得太玄**（IPC/CDT/HS/TF 术语堆砌）| 触发"凭什么懂我的店"异议 |
| 5 | ❌ **不把未成熟案例说成已完成案例**：5 月底前花厅坊**只能用"正在做 / 正在打样 / 正在形成样板"**语气 | 违反卷宗 + 销售包红线 |
| 6 | ❌ **不一上来推长期陪跑**（6-12 万）| 客户被价格吓退 |
| 7 | ❌ **不现场随意给大规模整改方案** | 不专业；越界 |
| 8 | ❌ **不擅自修改客户原始文件**（POS .xls / 商品档案 等）| 数据完整性破坏 |
| 9 | ❌ **不泄露客户敏感信息**（姓名 / 门店数 / 报价沟通 等）| 商业伦理 + 法律风险 |
| 10 | ❌ **不覆盖原始 POS 数据和照片**（保留原始；清洗后另存）| 不可追溯 |
