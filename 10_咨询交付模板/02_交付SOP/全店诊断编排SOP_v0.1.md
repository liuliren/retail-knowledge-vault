## §0 适用场景

**何时触发本SOP**：客户需要的不是单个品类的汰换建议，而是"整店经营哪里出血、先动哪个品类"的全局判断与一页可拍板的全店卡。

| 维度 | 适用 | 不适用（转其他SOP） |
|---|---|---|
| 门店类型 | 500–2000㎡ 社区生鲜超市（生鲜占比目标≥40%） | 大卖场（>3000㎡，框架权重不成立）/ 纯生鲜菜市场（无标品结构）/ 便利店（<300㎡） |
| 客户诉求 | 整店全景诊断 + 品类优先级排序 + 改造路线 | 只测1个品类 → 转 `/diagnose`（单品类skill）；只要5分钟初判 → 转 T-08 |
| 数据条件 | 有SKU级POS（最好90天）或至少有6大类汇总 | 完全无任何数据 → 走 §9.1 降级路径 |
| 区域 | 大湾区（M5本地场景壁垒可计分） | 非大湾区：M5降级为"区域文化场景"通用项，权重不变 |

**与单品类诊断卡的分工边界**（裁决既有 critical_gap）：
- 单品类卡 = 钻一口井（一个L3/L4小类的SKU级汰换/陈列）。
- 全店卡 = 画整张地图 + 决定钻哪几口井。**全店卡不重复单品类卡的SKU明细**，只承接其结论摘要 + 优先级。
- 本SOP是"地图"，§5 通过调用单品类skill来"钻井"，§7 把井的结论收编进地图。

## §3 Phase 1：POS 全店扫描〔定量轨起点 · 对账节点①〕

> 🔵 **这是"卷数据"轨的入口**。本Phase只产生定量事实，**不打质性分**（质性分在P2）。两轨此刻分开跑，到P4才合并——杜绝"边走边拍脑袋"污染数据。

**【输入】** §2 的数据包（必须项①②③）。
**【执行方】** Claude执行（脚本算 + Opus解读），六哥不介入。
**【步骤】**
1. **1.1 数据清洗**〔脚本〕：去退货、统一品类映射、标活动期。输出干净SKU表（不进上下文，只出指标）。
2. **1.2 六大类销额结构卡**〔脚本+Opus〕：按花厅坊v3.0锁定格式产出——
   `类别 | 实际销售金额 | 占比 | 实际销售数量`，30T与90T双期对比。**这是对账节点①的定量输入落点**。
3. **1.3 三层口径锁定**〔Opus〕：严格区分**客流（进店人次）≠ 客单数（POS订单数）≠ 客单价（销售÷客单数）**。客流无独立计数时，字段标注"当前=客单数代替，非真实进店人次"（承接v3.0 pending 铁律，不得混用）。**此标注须在P2走店访谈和P4对账中同步传递——见§4.4客流传播说明。**
4. **1.4 数学闭合自检**〔脚本〕：日均销售÷日均客单数 = 推算客单价，与报表客单价误差应 <2%（时段错开1天属正常）。超2%→标数据质量警告。
5. **1.5 库存与滞销扫描**〔脚本〕：ABC-Z分类（A=前70%/B=70-90%/C=90-100%/Z=连续3月=0）；滞销SKU数、库存积压SKU数（按品类拆）。
6. **1.6 按销额排序"可下钻品类清单"**〔Opus〕：列出所有有独立SKU级POS的L3/L4小类，按销售额占比**降序**排列，标注每个是否有独立POS（为P3触发规则备料）。

**【输出】** **全店定量诊断底稿**（内部·含6大类卡+三层口径+ABC-Z+可下钻品类排序表）。
**【时间】** 2.0h（全Claude）。
**【质量标准】** ①三层口径无混用；②数学闭合误差<2%或已标警告；③每个数字标 fact_layer（observed/inferred/pending）；④裸条码0残留（脱敏自检过）。

**Claude执行模板（1.2 输出）**：
```
| 类别 | 30T销售金额 | 占比 | 90T销售金额 | 90T占比 | 日均 | fact_layer |
|------|-----------|------|-----------|--------|------|-----------|
| 生鲜 | ...        | ...% | ...        | ...%   | ...  | observed  |
| ...  |            |      |            |        |      |           |
全店认知订正提示：销售第一大类 = ____（如花厅坊=生鲜70.74%，非食品）
```

## §4 Phase 2：框架 M1-M5 走店评分〔质性轨起点 · 对账节点②〕

> 🟡 **这是"卷现场"轨的入口**。六哥带着P1的定量底稿**纸质打印件**进店，但走店时**先凭眼睛打分**，不被数据牵着走（避免确认偏误）；定量与质性的碰撞留到P4。

**【输入】** P1定量底稿（打印件）+ 空白M1-M5评分表 + KB-DX-QUESTION-001（78题题库）。
**【执行方】** 六哥主刀（现场判断=驭马人层，不可下放）；Claude执行（同步录入证据照片与分数，不替六哥打分）。
**【步骤】** 建议走店路线：**入口印象(M1先看生鲜区位置)→主通道→生鲜区(M1)→标品区(M2/M3)→收银区(M4问老板)→大湾区特色区(M5)→死角复扫**。

每模块现场至少覆盖以下核心快判项（题号见 KB-DX-QUESTION-001 对应模块；M4为依赖问老板的题，题号确切）：

| 模块 | 分值 | 必覆盖核心快判项（出自框架v2.0快判 + 78题库） | 证据收集要求 |
|---|---|---|---|
| **M1 生鲜护城河** | 30 | 生鲜区是否在入口✓；猪肉三档完整（白猪≥50%/黑猪独立/高档猪有增长）；日清机制（无隔夜肉+阶梯打折签）；叶菜喷雾加湿+分类清晰；下午3-5点加工烟火气 | 肉/菜/果区各≥3张正面照 |
| **M2 品类结构** | 25 | CDT是否清晰（注：CDT=顾客决策树，指同品类内顾客按价格/品牌/规格的选购路径）；三层商品结构（核心/普遍/搭配）是否齐；五分法角色是否错位（注：五分法=核心款/普遍款/搭配款/防御款/形象款五类角色分工）；**价格带集中度（见M2-04预警项）** | 重点货架全架照 |
| **M3 陈列与空间** | 20 | 视线层(1.4-1.6m)是否放A类；端架是否节庆/A类爆品；销售-陈列比是否错配（销售强空间小→增格） | 端架/堆头/视线层照 |
| **M4 数据健康度** | 15 | **M4-01**生鲜占比(≥40%合格/<30%失守)；**M4-02**近1月客流变化（若P1已标客流=客单数代替，此题改问：近1月POS客单数变化如何）；**M4-04**哪个品类区顾客越来越少去；**M4-05**综合毛利率结构；**M4-10**能否说出前5单品；**M4-12**多久看一次数据 | 老板访谈录音/笔记（12题中≥6题靠问） |
| **M5 大湾区特色** | 10 | 煲汤料/腊味/拜神/凉果等本地场景是否做透；大湾区特化品类宽度是否上调30-50% | 特色区照 |
| M6 顾问综合判断 | 不计分 | 一词心智标签 + 整体氛围（干净/拥挤/冷清/热闹）+ 老板经营意识 | 必填，质性 |

**M2-04价格带预警项（修正：不在走店阶段打正式分）**：

> 走店阶段只记观察预警，P4用P3量化数据填正式分。原因：判断价格带集中度>70%需数SKU数量，现场目测不可靠，在P2打正式分会迫使执行者依赖POS（污染质性轨）或凭感觉（失去量化意义）。

```
M2-04: 主力货架同品类SKU价签是否明显集中在1-2个价位区间（目测超过半数）
→ P2只打预警：⚠️ 或 ✓（"价格带看似集中"或"价格带看似有梯次"）
→ 备注栏填：预计集中度___% 待P3量化
→ P4填正式分（用P3中价CR5数据）
```

**评分填写格式**（每模块一行）：
```
模块 | 走店质性分S（满分内打分）| 关键扣分点（对应78题哪几题）| 证据等级A-E | 现场照编号
M1   | 22/30                  | 无日清机制/黑猪缺            | B(数据+现场) | IMG_01-08
```

> **走店质性分S = 这一轮纯现场+访谈得出的分**，是对账节点②的质性输入，**先记着，不要在现场就拿POS去改它**。

### §4.1 证据等级完整定义（A-E五级）

> **以下定义为本SOP内嵌权威定义，不外链，全文统一使用此表。** 全卡不得混用旧版A-D四级系统。

| 等级 | 定义 | 可否写独立结论 |
|---|---|---|
| **A** | POS数据 + 现场实测 + 老板/员工访谈 三方一致 | ✅ 可写，最高可信度 |
| **B** | 上述三方中任意两方一致 | ✅ 可写，须注明缺少的一方 |
| **C** | 单一来源（仅现场目测 或 仅老板口述），待另一轨验证 | ⚠️ 可写但必须注明"待验证"，不得写成定论 |
| **D** | 顾问估算/间接推算，无直接证据支撑 | ❌ 禁写独立结论，只可作辅助参考 |
| **E** | 多轨数据冲突且六哥尚未裁决，该模块分数暂空 | ❌ 禁写任何结论，须在P4校准前完成裁决 |

### §4.2 走店阶段使用的术语内联简释

> 以下术语在走店快判中出现但KB页不会同步打开，简释内联于此，保证SOP自闭合。

- **CDT（顾客决策树）**：顾客在同品类内选购时，按"哪个维度先筛"的心智路径。如冰激凌类：先看品牌→再看价格→再看口味，这就是一棵CDT。CDT清晰= 货架布局与顾客选购逻辑对齐；CDT模糊= 品牌/规格/价格混排，找不到决策支点。
- **五分法（SKU角色分工）**：一个品类内的SKU按销售贡献和战略意图分五类——核心款（高销额撑场）/ 普遍款（补充选择）/ 搭配款（配套连带）/ 防御款（阻击竞品但利润低）/ 形象款（拉高品类认知·低销可保留）。五分法错位= 形象款占了核心位，或防御款价格倒挂成了主销款。
- **角色闸（汰换防误杀机制）**：汰换SKU前的最后检验——先判断这款低销SKU的"角色"：若它是防御款，不能因销低就删；若它是形象款，维持陈列即可；只有确认它既非防御/形象又持续滞销，才进汰换名单。"角色闸防误杀已复核"= 执行者已对每个候选汰换款逐一确认其角色不属于保留类。

## §8 交付确认清单（交付前逐条过 · 不过不签）

1. ☐ **双轨对账完整**：P4校准表5个模块全有Q锚点或"封顶C/Q缺失"标注，无空轨。
2. ☐ **时间未超预算**：全程≤1.5工作日；若超，已走§9降级并在卡上标注降级口径。
3. ☐ **下钻覆盖率已记录**：N品类覆盖销额__%，剩余结转二轮已注明。
4. ☐ **三层口径无混用**：客流/客单数/客单价分清，代替项已标注；P2/P4口径传播已执行。
5. ☐ **生鲜毛利红线**：未凭失真进价出生鲜汰换结论。
6. ☐ **证据等级齐**：全卡统一使用A-E五级系统（定义见§4.1），不得混用旧版A-D四级；C/D/E级结论已按§4.1规则标注。
7. ☐ **隐私护栏**：裸条码0残留；raw未进上下文；客户真名对外已查授权链或脱敏。
8. ☐ **fact_layer齐**：observed/inferred/pending 标全，pending清单进Section⑨边界。
9. ☐ **AI/六哥分工无空白**：每个汰换取舍有六哥决策单签字（Z红线未代签）；角色闸已复核（定义见§4.2）。
10. ☐ **六哥终审签字**：E层对外卡经六哥签字（CLAUDE.md §3③）。
11. ☐ **非生鲜覆盖透明**：本轮非生鲜≥5%品类中，已下钻的X个/共Y个，未下钻的Y-X个及其合计销额占比已在Section⑤明确标注；不存在因N额度冲突导致的系统性漏检而未告知。

## §9 降级执行方案

> 触发任一即降级，并在全店卡Section⑨边界明确标注降级口径与精度损失，不得隐瞒。

### §9.1 客户无POS数据（或只有6大类汇总）
- **路径**：跳过P3全部下钻（无SKU级数据无法跑diagnose skill）。
- P1只做6大类层结构卡（销额占比+目测库存）；P2走店评分照常（质性轨可独立成立）；P4校准时所有模块Q缺失→证据等级**全封顶C**，全店卡通篇标"基于现场观察的初步判断，需补POS验证"。
- 产出降级为"全店初判卡"（对标T-08升级版），明确告知客户"补齐90天POS后可升级为正式全店诊断"。
- 时间压缩至 ≤0.5工作日。

### §9.2 客户不配合（老板不给数据/不让久留）
- **路径**：以T-08五段落5分钟快诊为骨架，P2走店压缩到30分钟（只打M1+M4两个最高杠杆模块，生鲜占比+数据健康），P1/P3全跳过。
- M4靠现场目测+收银台观察代替问老板（M4-02客流看收银台、M4-01生鲜占比目测货架面积）。
- 产出"5分钟门店速判"，证据等级全D（顾问直觉/待验证），**禁止写成正式诊断结论**（质控红线⑥⑦），只作成交钩子。

### §9.3 时间不够（突发压缩到<1工作日）
- **优先级砍法（按杠杆从低到高砍）**：先砍M5(10分)与M3(20分)的现场深扫 → 再把P3下钻N上限从5降到**TOP2**（只钻销额最大的两个可调改品类）→ P4校准只做M1/M2/M4三个高权重模块。
- 保留不可砍的核心：P1六大类结构卡 + M1生鲜占比 + 三层口径闭合（这是全店卡的认知地基）。
- 全店卡标注"快速版·覆盖__%权重"，约定二次上门补全M3/M5/剩余下钻。
