# 单小类诊断样板（L3→Item→SKU 可复制通用版）v1.0

> ⚠ 文件名仍带 `_v0.1`（内容已升 v1.0 active）；重命名属治理 §3④ 需六哥签字，暂保留旧名以免断链。

> **结论先行**：本样板用「一个单小类」走通"POS 明细 → 脱敏结构诊断 → 客户可读一页简报"的完整链路。
> **铁律**：**模板可复用，结论不可复用**——每个小类的约束值（SKU 数、价格带临界、Item 拆分）必须各自实测，禁止把上一个小类的数字搬过来。
> **来源**：四链判断方法（B/C1/C2a/C2b-1）消化自零售老刘；单小类样板化与实跑验证为六哥 IP。首次跑通见 [[2026-06-19_T03饼干_L3到Item到SKU实跑验证_v0.4]]（花厅坊 T03 饼干，client_confidential）。

---

## 0. 适用与边界

- **适用**：对单个 L3 小类做结构诊断与调改方向判断（不是全店、不是单 SKU 砍单）。
- **先选哪个小类**：优先选**任务维度丰富、风险低**的小类（跨办公/早餐/儿童/分享/解馋/健康/礼盒等多任务的，拆 Item 信号最强；常规性角色试错代价小）。
- **不做**：不出 SKU 级砍货清单（除非六桶全过 + 现场 + 店长确认）；不新增 M-DEC；不把 [推断] 写成定论。
- **权限**：本样板 internal_methodology；跑出的客户结果 client_confidential；对外只出 E 层一页简报（0 术语 0 条码）。

---

## 1. 十步诊断流程

> 每一步标注**输入字段**与**缺数时的降级处理**（多数门店首跑会缺库存/去年同期/陈列，缺就标 pending，不假装）。

| 步 | 动作 | 关键输入 | 缺数降级 |
|---|---|---|---|
| **1 选小类** | 按任务丰富度 + 风险选 | 品类表角色/占比草案 | — |
| **2 盘数据** | 诚实盘点 markdown 有什么、SKU 级缺什么 | 现有表 schema | 缺的全列 pending 清单 |
| **3 取数+字段映射** | POS 明细按条码聚合，定时间窗 | 条码/品名/零售价/进价/销量/销额/类别/销售日期 | 无销售日期→ 无法算滞销天数 |
| **4 基础结构** | SKU 数 / 子类分布 / 价格带分布 | 价格、品名 | 品名无品类词→子类映射弱，标注 |
| **5 动销汇总** | 总销量/销额/毛利率、零销/低销、集中度 | 销量、销额、进价 | 进价缺→毛利标 [推断] |
| **6 ABC 结构** | 临时 ABC（按销额累计）、头部/尾部 | 销额 | 标"临时 ABC，非门店确认" |
| **7 价格带结构** | 低/中/高分布 + 销额占比 → 过密/漏档 | 价格、销额 | 临界值 [推断]，联动 [[M-DEC-010]] |
| **8 库存与滞销** | days_no_sales；join 库存判"有货卖不动" | 销售日期、库存快照 | 无库存→只出滞销天数，不下"有货滞销"判定 |
| **9 Item/需求点映射** | 品名关键词→Item；超配/欠配；低频刚需/必留 | 品名、购物任务图谱、画像 | unmapped 高→**必须人工拆分**，不靠关键词硬判 |
| **10 动作建议** | 保留/替换/瘦身/补齐/观察/待补数 | 上述全部 | 无明细不点具体 SKU |

## 2. 四链判断要点（B / C1 / C2a / C2b）

- **C1 · SKU 数约束**：有没有超配（某段/尾部挤）、欠配（某价格带/需求点空）？**不给固定目标 SKU 数**除非库存+去年同期+订货齐；缺数只给方向。
- **C2a · 宽类窄品**：哪些 Item 宽（可窄品/替换）、哪些窄（低频刚需不砍）；**替换非净减**——好品替差品，省出的位给高端/刚需。
- **B · 六桶防误杀**：真滞销候选必须过六桶才可议砍——①真滞销 ②缺货假滞销 ③陈列差假滞销 ④价格带错位 ⑤生命周期衰退 ⑥系统/替换假象。任一桶 pending → **不出砍单**。
- **C2b · 类为先/可得性**：先保需求点被覆盖（可得性），再谈单品优化。

---

## 3. 数据驱动纠错（必做，样板的灵魂）

> 这一步是 T03 实跑最有价值的产物：**用更全/更新的数据，回头校验前一版的每条发现**。

每条发现打一个标签：**成立 / 增强 / 减弱 / 被推翻 / 数据不足**。

- 凡被更全数据**推翻**的判断（典型：早期子集数据误判某需求点"缺口"，全量数据显示其实不缺），**必须显式记录纠错**，并回改下游简报。
- 纠错记录进诊断正文的对比表 + 简报修订说明——这既是诚信护栏，也是方法论可信度的证据。

---

## 4. 数据安全边界（每跑必守）

1. 原始 Excel/CSV **只读不改**；不脱离原始目录加工。
2. 输出**不含**条码 / 进价 / 供应商 / SKU 明细清单；只出脱敏汇总。
3. 中间明细文件不入 git（`raw_exports/`、`*_明细.*` 已被 .gitignore 覆盖）。
4. 所有 SKU 级临界值/数字未经门店确认的，标 `[推断]`；缺的字段标 `pending`，不假装填满。

---

## 5. 输出物（两层）

| 层 | 产物 | 受众 | 约束 |
|---|---|---|---|
| C（内部） | 完整实跑验证文档（含对比表、六桶、字段缺口） | 顾问内部 | client_confidential，不外发 |
| E（对外） | **一页诊断简报** | 老板/店长 | 0 模型术语、0 条码、结论先行、留数据边界；被纠错的旧判断须 superseded |

---

## 6. 复制到下一个小类时

1. **复制本模板，清空所有数字**（约束值不可继承）。
2. 重跑第 3–10 步，得到该小类自己的实测。
3. 第 3 节纠错对照该小类自己的前版，不对照别的小类。
4. 跑通后回流：若某判据在多个小类稳定复现 → 提报六哥，考虑蒸馏为 M-DEC（candidate→active 需签字）。

> **升级路径**：✅ 已达成——经 T03 饼干 / T01 膨化 / T05 果干蜜饯三小类验证，**2026-06-22 六哥签字升 `active`，并入 [[M2_选品配置簇]] 作为标准交付件**。后续接新客户直接以本样板起跑。

**品类特异（结论不可复用的实证，三例）**：价格带天花板因品类而异——膨化无高端（高=0/max13.9）、饼干微有（高2/max22.8）、果干跨度大（高6/max44.9）。**价格带预警（M-DEC-010）必须按品类分型，不可一刀切。**

**实跑暴露的样板修订点**：
1. **第 1 步「选小类」会被真实数据收敛到品类编码口径**——计划的「薯片」在 POS 里归「膨化 020201」。选品类时以 POS 类别码为准。
2. **价格带预警（第 7 步 / M-DEC-010）须按品类属性分型**：膨化天花板天然低（高价带=0），「补高端」未必符合品类属性。**不可把饼干「补中高端」结论机械套给膨化**——这是「结论不可复用」的活证。
3. **第 5 步毛利**：若 POS 含进价列，毛利为**实算**（非 [推断]）；批次间字段可能不一致，按实到字段标注。

## 8. 交付硬化检查（自解释 / 可组合 / 去黑盒 / 最小回流）
> 源自 [[2026-06-22_Agent-Native蒸馏应用_艾逗笔9条启发_SRC]]，把「好 skill 铁律」落到本样板的交付质量上。

- **自解释（启发1）**：店长拿到本样板的输出（一页简报），不需六哥口头补充就能照做。**验收**：随机给一个没听过讲解的店长试读，能做出 ≥80% 正确动作。凡需口头补的地方=样板缺口，回填。
- **可组合（启发1）**：本样板的输出（单小类结构诊断）须能直接当下游输入——喂给 [[M2_选品配置簇]] 选品 / 陈列 SOP，接口对不上即断点。**验收**：诊断→选品→陈列能无人工翻译串成一条链。
- **去黑盒（启发7）**：每条核心建议六哥都能用一句话讲清推理；**讲不清的不签字、退回**。所有未经门店确认的数字标 `[推断]`，缺字段标 `pending`。这是 10% 护城河的可解释底线。
- **最小回流（启发3）**：现场拍板后 30 秒口述一条「现象 / 决策 / 一句话推理 /（留空）结果」，周末批量回填为 M-DEC。**不追求完美，追求不断流**；决策 2–4 周后回填结果，才从「记录」升级为「长期记忆」。
- **晋升门槛**：同一现象/判据在 **≥3 个独立小类或门店**重复 → 提报候选 M-DEC（如本样板已产出 [[M-DEC-013_单小类长尾结构治理_v0.1_候选]]）；跨**第二客户**再验证后升 active。
