# 诊断后行动计划编制SOP v0.1

> **本文是流程定义（SOP + 一页纸模板），不是某家门店的真实行动计划。**
> **定位**：全店诊断卡的「下游施工总包」。诊断卡回答「店有什么病、先打哪个山头」（§5 TOP3 / §6 改造清单 / §7 D档）；本SOP回答「老板这个月、这季度，到底动手做哪 3 件、谁做、做到什么样算完成」。
> **一句话价值**：诊断卡是顾问视角的「应做清单」，本SOP把它**穿过老板的真实约束（钱/人/时间/意愿）**，压成一张老板愿意签字、且真能在每周 ≤4 小时投入下跑完的执行书。**没穿过约束的改造清单，是 PPT，不是计划。**
> **状态**：draft（v0.1）。含「铁律/标准/红线」字样，按全局宪法《标准审议铁律》须经 ≥3 轮多角色论证方可升 stable；当前为草案，不得被其他 Agent 引用为已生效红线。

## §0 使用说明

### 0.1 何时用（触发条件）
- 全店经营诊断卡已**填写完整**（§3 评分、§5 TOP3、§6 改造清单、§7 D档均有内容），进入「落地交付」环节。
- 触发词：六哥说「出行动计划 / 给老板一张执行表 / 把诊断变成动作 / 30天计划 / 落地方案」。
- **不用本SOP的场景**：诊断卡尚未填完（先做诊断）；只改一个货架（用单品类诊断卡自带的调改清单）；纯战略咨询无执行落地（用别的交付件）。

### 0.2 输入与输出（接口契约）

| | 内容 | 来源 | 说明 |
|---|---|---|---|
| **输入** | 已填完的《全店经营诊断卡》——重点取 §2 一句话结论、§3.3 校准后评分、§5 TOP3、§6 优先改造清单（顾问视角候选动作初稿，Phase 2 Step 0 消费它）、§7 D档清单、§8 数据采集清单（看哪些结论是 C/D 级待验证） | 诊断卡实例 | §6 是顾问初稿，本SOP Phase 2 Step 0 优先消费它、补全要素、过滤约束，产出老板共识后候选清单（SSOT 在本SOP输出） |
| **输出** | 1 份《30/60/90天行动计划书》（一页纸·老板可签字）+ 1 张《老板约束清单》（内部留底，不进计划书正面） | 本SOP §6 模板 | — |

### 0.3 谁来做（执行方约定·全SOP统一）
全SOP每一步都须明确执行方，三类：

| 执行方 | 角色 | 干什么 |
|---|---|---|
| **老板/店长** | 决策者+约束源 | 给约束（钱/人/时间）、确认问题感知、拍板 D 档、签字 |
| **六哥** | 顾问+裁决者 | 主导问题共识、优先级裁决、对赌式签字背书（90/10 的 10%，不可代签） |
| **Claude** | 杠杆+起草 | 从诊断卡提候选动作、按约束过滤、算工时预算、起草计划书、设计检查点（90/10 的 90%）——以§0.5 选定的接入模式运作 |

### 0.4 总时长与节奏（铁律：一次坐下 ≤3.5 小时谈完）
> *【张力】行动计划必须在诊断反馈的**同一场会**里谈完并签字。拖到「回去想想」=80% 流产——老板的执行意愿在看到诊断结果的当天最高，每过一天衰减。故五个 Phase 设计为可连续跑、总时长 ≤3.5h 的单场会。*

| Phase | 名称 | 主执行方 | 时间预算 | 输出物 |
|---|---|---|---|---|
| — | Claude接入模式确认 | 六哥（会前5分钟）| ≤5 min | 选定模式A或模式B |
| 0 | 老板资源摸底 | 老板说·六哥记录 | ≤30 min | 约束清单 + 场景分支 |
| 1 | 问题确认（共识） | 六哥主导 | ≤45 min | 已共识优先问题清单 |
| 2 | 行动项生成 | Claude辅助 | ≤60 min | 候选行动清单（已过滤） |
| 3 | 优先级裁决 | 六哥主导 | ≤30 min | 30/60/90 分层动作 |
| 4 | 计划书输出 | Claude生成·六哥审 | ≤30 min | 一页纸计划书（签字） |
| 5 | 首检设计 | Claude·六哥定 | ≤15 min | 14天检查点 |
| | **合计** | | **≤3.5 h** | |

**时间溢出触发协议**（Phase 1保护机制）：若 Phase 1 处理第二个🔴项时总累计时间已超 40 分钟，触发时间保护——第二🔴项当场降级为「验证动作+首检复议」，不在当场走完四步协议；Phase 2 由会场内完成改为六哥会后 48 小时内用 Claude 补完并微信发给老板确认，老板 24 小时内无异议视为认同。**须在 Phase 1 介绍规则时提前告知老板此备用机制**，不得事后偷改流程。

### 0.5 Claude 接入模式协议（会前5分钟必选一种）

> *【修订说明·原致命缺陷①修复】原SOP多处标注「Claude现场记录/Claude逐项问/Claude生成」，但未定义Claude如何出现在门店现场会。此节补充两种合法接入模式，六哥会前5分钟选定，全场一致执行。*

**两种合法接入模式，选一，不混用**：

| 模式 | 名称 | 适用场景 | 操作步骤 | 老板感知 |
|---|---|---|---|---|
| **模式 A（实时介入）** | 六哥带设备 | 门店有可靠网络；六哥可携带笔记本/iPad；会议节奏允许Phase间隙停顿 | ①会议开始前5分钟：在Claude Code加载本SOP + 诊断卡上下文（约800 tokens）；②每个Phase结束后3-5分钟休息时：把本Phase结构化笔记提交Claude处理，取回下阶段所需输出；③六哥本人在会议桌上做简要手写笔记，Phase间隙补充给Claude | 老板知道顾问在用AI辅助工具（大湾区社区店老板普遍接受，建议如实说「我用AI帮我整理数据」）|
| **模式 B（前置分段）** | 会前预处理 | 门店网络不稳/无设备携带条件/六哥判断老板不适宜知晓AI介入 | ①会前：六哥把诊断卡§6候选动作清单交Claude预处理，生成按问题分类的候选行动初稿（约10-15条带五要素）；②现场：六哥只用纸质版候选清单主导共识与裁决，不在现场操作Claude；③会后48小时：把会议约束笔记交Claude做约束过滤 + 计划书起草，六哥审后发给老板确认签字 | 老板不感知AI参与；计划书署名仅顾问 |

**模式B的签字时序调整**：模式B下，计划书无法当场签字，须注意：Phase 4 产出初稿 → 会后48小时发老板微信确认 → 老板线上/下次见面签字。**「单场会签字」铁律在模式B下放宽为「48小时内完成签字」**。

**选择建议**（大湾区社区店背景）：花厅坊等已建立信任关系的门店优先用模式A（节省来回时间）；首次合作或老板警惕心强的门店用模式B（避免干扰信任建立）。

## §1 核心设计原则（全SOP红线·先读）

> *【填写说明】本节是 5 条不可违反的设计约束，是后面所有 Phase 的裁判规则。任何 Phase 的产出违反本节，作废重做。*

1. **每周老板亲投 ≤4 小时铁律**：老板**本人**亲自动手的改造工时，30天 3 件加总 ≤ 4h/周 × 4 周 = **16 工时/月**。超出的动作，要么换执行人（店长/员工/外包），要么砍掉。这是现实约束，不是建议——老板还要看店、进货、管人，没有更多时间（见 §2.5 工时预算表）。
2. **30天硬上限 3 件**：防失焦（CLAUDE §2 原则7「少而深」）。第 4 件再好也进 parking lot。3 件做完比 8 件半拉子强。
3. **可逆优先、止血先行**：先做可逆、低成本、快见效的「止血」动作（如改陈列、调价签、补叶菜品种），后做不可逆、重投入的「手术」动作（如砍 SKU、改供应商、动装修）。
4. **每件必挂验收标准（可量化）**：无验收标准的动作 = 空话。「改善生鲜陈列」不合格，「叶菜从 6 种补到 ≥10 种、肉档下午 4 点起 7-8 折」合格。
5. **诊断顺序不跳步**（KB-METHOD-CHAIN-001 铁律）：动作排序服从五层链——先确认目标顾客清楚 → 商品配置 → 品类角色 → 单品执行 → 数学验证。禁止「看着百货乱就先理百货」式的表象动作。

## §2 Phase 0 — 老板资源摸底（≤30 min）

> *【目的】在生成任何动作之前，先把老板的「弹药库」摸清楚。后续所有动作只在这个边界内选。先摸约束、后生成动作 = 避免出一堆老板做不了的「正确废话」。*

**执行方**：老板/店长口述 · 六哥记录（模式A下Claude同步整理，模式B下六哥手写笔记会后交Claude）
**时间预算**：≤30 min
**输出物**：《老板约束清单》（内部留底，不进计划书正面）+ 场景分支判断
**质量标准**：四类约束每类都有**具体数字或具体人名**，不能是「资金还行/人手够用」这种模糊话；**Phase 0 结束前必须完成场景分支判断**（见 §2.3）。

### 2.1 四类约束摸底问法

| 约束类型 | 必问问题 | 记录字段（要数字/人名）|
|---|---|---|
| **资金约束** | 「这个月，您愿意为改造动用多少钱？分清『能马上花的现金』和『要等回款的』。」 | 本月可动用现金 ___元；可延期资金 ___元；红线：单项超 ___元须您二次拍板 |
| **人力约束** | 「这些改造，谁来动手？店里谁能调、谁绝对不能动（如收银/采购顶梁柱）？」 | 可执行人：___（姓名+岗位+每周可挪 ___h）；不可调：___（原因）；是否可短期外包：□是 □否 |
| **时间约束** | 「您本人每周能亲自花在改造上的时间，老实说，几个小时？」 | 老板本人 ___h/周（铁律上限按 4h 校验，超报的须当场压回现实值）|
| **优先偏好** | 「不管顾问怎么排，您自己最想先改的是哪一块？为什么？」 | 老板最想先改：___；理由：___；情绪强度：□随口说说 □很坚持 |

### 2.2 约束清单（输出模板·内部留底）

```
门店代号：___    摸底日期：___    记录人：六哥（Claude整理·模式A）/ 六哥手写（模式B）
─────────────────────────────────
[资金]  本月可动现金 ___元 / 可延期 ___元 / 单项拍板线 ___元
[人力]  可执行人：___（___h/周）；不可动：___；外包：□可□否
[时间]  老板本人：___h/周（≤4h 校验：□通过 □已压回 ___h）
[偏好]  最想先改：___（情绪强度：___）
[场景]  □标准场景（有可支配员工）  □夫妻店变体（老板=唯一执行人）→ 激活§2.3变体方案
─────────────────────────────────
```

> *【张力·时间约束的真相】老板嘴上说「我每周能花 10 小时」基本是假的——开会当场的热情值，不是日常稳态值。六哥须把任何 >4h 的自报值**当场压回 ≤4h**，并说明：「我们按每周 4 小时排，少排但排得动，比排满了第二周就垮强。」剩余动作一律找其他执行人或进 60-90 天。*

### 2.3 场景分支检测门（Phase 0 结束前必判·修订说明·原致命缺陷②修复）

> *【说明】原SOP已在§8张力总表承认夫妻店场景静默失效，但没有分支路径。本节补充显式检测门。*

**判断条件**：约束清单填完后，六哥即时判断：

**若同时满足以下两条 → 激活夫妻店变体方案**：
1. 人力约束栏：可执行人只有老板本人（无固定员工/无可委托的岗位）
2. 人力约束栏：外包选项「□否」

**夫妻店变体方案（激活后替代标准配额）**：
- 30天动作件数：从最多3件**降至最多2件**
- 执行人栏：改为「顾问陪同+老板操作」（六哥每月至少陪同1次现场执行，≤4h）
- 老板工时上限：从 ≤4h/周 改为 **≤2h/周**（老板还要开店，陪做动作由顾问承担操作主力）
- 一页纸模板执行人栏：标注「顾问陪同完成」
- 60-90天动作：维持不变，但均须六哥介入才推进

**标准场景（不激活变体）**：按本SOP其余Phase正常推进，工时上限 ≤4h/周，执行人可为店员/外包。

## §3 Phase 1 — 问题确认（老板共识·≤45 min）

> *【目的】诊断卡 §5 TOP3 是顾问算出来的；但老板不认同的问题，给他写进计划也不会做。本 Phase 把「顾问的 TOP3」和「老板脑子里的问题」对齐，产出一份**老板点头**的优先问题清单（可能与诊断卡 TOP3 略有出入，这是允许的）。*

**执行方**：六哥主导（顾问视角讲数据），老板回应
**时间预算**：≤45 min（触发溢出协议见 §0.4）
**输出物**：《已共识优先问题清单》（3 条，标注共识状态）
**质量标准**：每条问题都有「老板认同度」标注；分歧项有明确处理记录，不含糊带过；每条问题已标注证据等级。

### 3.0 Phase 1 前置：证据等级核查（≤5 min·修订说明·原致命缺陷④修复的前置步骤）

> *【目的】区分「数据本身出错」与「数据无误但解读分歧」——这两种情形要走完全不同的处理路径，混同处理会让顾问越说越错。本前置步骤强制先做分类，再走共识流程。*

六哥在正式亮出每个 TOP 问题之前，先核查该问题所用数据的证据等级（来自《诊断输出质量控制规范》）：

| 证据等级 | 定义 | Phase 1 处置 |
|---|---|---|
| **A级** | 直接现场观测或原始 POS 数据，零推断 | 正常走 §3.1 共识流程，可用强表述「数据显示 X」 |
| **B级** | 有较强间接证据支撑的推断 | 正常走共识流程，用中等表述「数据高度指向 X」 |
| **C级** | 估算/类比/经验推断，待现场核实 | 亮出结论时主动降调：「数据指向 X，但这是推断，需要您帮我核实一下」；在共识表🟡栏直接标注「数据待核实」，**不走🔴不认同协议** |
| **D级** | 纯假设，无数据支撑 | **不进 Phase 1 共识**；先补数据再谈 |

**关键判断**：若老板在 Phase 1 说出具体数字证伪了你的C级结论（如「周转不是45天，我们账期是25天，一直转得动」），这是**数据错误**，不是判断分歧——正确动作是「退回修正诊断卡数据，重新评估该问题是否仍在 TOP3」，不是走§3.3说服协议。

### 3.1 逐条确认流程（六哥对每个 TOP 问题走一遍）

对诊断卡 §5 的每个问题，六哥按下面三步走：
1. **念现象+数据**：「您这家店，生鲜占 70%，但后场库存压了 X 万、周转 Y 天，超过了您的结算账期——这意味着钱卡在货上，现金流在变紧。」（用 A/B 级数据；C级数据须先降调，见 §3.0）
2. **问感知**：「您自己有没有感觉到这个？平时进货付款是不是越来越紧？」
3. **记共识度**：老板的反应归入下面三档。

### 3.2 老板认同度三档 + 处理策略（核心机制）

| 认同度 | 老板反应 | 处理策略 |
|---|---|---|
| **✅ 认同** | 「对，我也觉得是这」 | 直接进共识清单，按诊断优先级排 |
| **🟡 半信** | 「有点道理，但我觉得没那么严重」 | 进清单，但在计划书里**配一个低成本验证动作**（先量化它，用事实说话），不一上来就重投入 |
| **🔴 不认同** | 「不是这样的 / 这不是问题」 | **先判断是数据错误还是判断分歧**（见§3.3分流），再走对应协议 |

### 3.3 「老板不认同」处理协议（修订说明·原致命缺陷④修复·增加数据错误vs判断分歧分流）

> *【设计前提】🔴不认同有两种本质不同的情形，必须在第一步分流，走错了路只会越说越错。*

**第一步：分流判断（判断是哪种不认同）**

| 情形 | 识别特征 | 处置路径 |
|---|---|---|
| **A. 数据本身出错** | 老板说出具体数字推翻了诊断数据（如「我们账期只有25天，不是45天」）；或新信息令C/D级推断站不住脚 | **退回修正**：六哥当场承认数据错误，修正诊断卡该条的证据，重新判断是否仍在 TOP3；若降出 TOP3，此条不再走共识流程。这是顾问专业性的体现，不是示弱。 |
| **B. 数据无误但解读分歧** | 老板说「这个数是这样，但我觉得没问题/这行就这样」；数据本身没有被新事实推翻，只是老板对其严重性/重要性判断不同 | 走下面的四步协议 |

> *【红线】情形A的数据错误绝不可走四步说服协议——用更多数据去「证明」一个本身算错的结论，结果是越说越错，越说老板越不信。*

**四步协议（仅适用于情形B·判断分歧）**：
1. **先尊重感知，不辩论**：「您比我懂这家店，我先听您说为什么。」——让老板把本地理由讲完。老板讲出来的理由，常常是顾问漏掉的关键变量（如供应商绑定、街坊关系、季节性）。
2. **亮数据指向，不下结论**：「我理解。我这边数据看到的是 X，它**可能**指向 Y。我不确定，但我们能不能用一个小动作先验证一下？」——把「我认为」改成「数据指向，待验证」。
3. **三种收口**：
   - **老板被说服** → 转 ✅ 认同，进清单。
   - **老板坚持、且讲出了顾问没考虑的本地变量** → 顾问采纳，**修正自己的判断**，把这条降级或移出 TOP3，并在约束清单记一行「因[本地变量]调整」。这是顾问该认的——硬塞会毁掉信任。
   - **老板坚持、但讲不出新变量（纯情绪/惯性）** → 进清单但标 🔴，配「最低成本验证动作」（如「先连续 7 天记录这个数」），用 14 天首检的事实回来谈，**不在第一次会上硬碰**。
4. **全程记录分歧**：所有 🔴 项写进约束清单的「分歧留底」栏，注明老板理由 + 数据指向 + 约定的验证方式。**绝不在老板不点头时把动作硬写进签字版计划书**——签了不做的字，比不签更伤交付信誉。

> *【张力·不认同处理的边界】这套协议会不会让顾问「太软」？会，如果验证动作也省了。所以红线是：**可以不在第一次会上赢，但必须留一个会产生事实的验证动作**。14 天后事实回来，由数据而非由顾问嘴来赢这场争论。*

## §5 Phase 3 — 优先级裁决（六哥主导·≤30 min）

> *【目的】把 §4 的 5-8 条候选，砍成 30天3件 + 60天2件 + 90天1-2件的分层动作。这是 90/10 的 10%——裁决权在六哥，不下放给 Claude。*

**执行方**：六哥裁决（不可下放），Claude 算矩阵坐标 + 记录
**时间预算**：≤30 min
**输出物**：《30/60/90 分层动作表》
**质量标准**：30 天 ≤3 件且工时 ≤ 本店上限；每层动作有承接逻辑；止血动作数量达标。

### 5.1 影响力 × 执行难度矩阵（Claude 给坐标，六哥拍板·修订说明·原致命缺陷③修复·增加评分锚点）

> *【说明】原SOP矩阵无评分锚点，Claude和六哥对同一动作可能打出相反象限。本节补充可操作的三档评分标尺，确保坐标可还原、可复述。*

**评分锚点表（Claude算坐标时须列出评分依据）**：

| 维度 | 低 | 中 | 高 |
|---|---|---|---|
| **影响力** | 月营业额预计影响 <1%；或改造后30天内无可见顾客行为变化 | 月营业额预计影响1%-5%；或30天内有局部可见变化（如单品类复购提升） | 月营业额预计影响 >5%；或顾客日常行为可明显感知（如核心品类充足率改变引客流变化）|
| **执行难度** | 不需要新供应商；不需要>3000元一次性资金；老板或指定员工3天内可独立完成 | 满足以下任一：需要联系新供应商但不换主力供应商；需要3000-10000元；需要1-2周反复调整 | 满足以下任一：需要更换主力供应商或引入新品类；需要>10000元；需要外部专业支持；或不可逆（装修/大规模淘汰SKU）|

**坐标记录要求**：Claude 给每个候选动作标注象限时，须附一句说明，例如：「叶菜补品种：影响力高（核心品类·30天内顾客可感知复购变化）+ 难度低（调品种不换供应商·老板1h决策）→ ①象限」。六哥确认坐标后拍板时域。

```
影响力 高
      │  ② 战略动作        ① 即时止血
      │  （高影响·高难度）  （高影响·低难度）
      │   → 60-90 天        → 30 天先做
      │  ────────────────────────────────
      │  ④ 不做/parking     ③ 顺手做
      │  （低影响·高难度）  （低影响·低难度）
      │   → 砍               → 有空填充
      └──────────────────────────────── 执行难度 高
```

裁决规则：
- **①象限**（高影响·低难度）= 30 天 3 件的首选，**先抢这格**。
- **②象限**（高影响·高难度）= 60-90 天战略动作，多为不可逆（须配 §7 D 档）。
- **③象限** = 30 天 3 件没满时的填充，或顺带做。
- **④象限** = 砍，进 parking lot，不写进计划书。

### 5.3 止血 / 改善原则（修订说明·原致命缺陷③修复·移除40/60不可验证固定比例）

> *【修订说明】原SOP写「止血40/改善60」，但30天最多3件（整数集）无法精确达到40:60。本节替换为两条可验证的操作原则。*

**两条可验证的操作原则（替代原40/60表述）**：
1. **止血件数下限**：30天 N 件中，**≥1件须为止血动作**（可在14天首检时产生可测结果的快速干预）。这确保老板在首检时看到变化、愿意继续。
2. **全改善时的保护机制**：若 30 天全部动作为长周期改善（完成周期>21天），须在 §5.5 备注栏注明「14天首检将核查哪个里程碑替代终态验收」，否则首检失去核查基础。

**两种特殊场景（六哥现场判断）**：
- **现金流告急**（周转天数 > 账期）：30天件数中止血件数增至 ≥2 件，优先保命。
- **基础极好且老板信任度高**：可全部为改善动作，但须满足第 2 条保护机制。

**配比精神（保留但不作量化标准）**：止血快见效稳信心，改善真正解决结构问题——两者都需要，六哥按店况判断侧重，不追求固定比例。

### 5.4 「老板偏好位」机制（修订说明·原致命缺陷④修复·增加冲突解决规则）

> *【设计】老板在 Phase 0 说的「最想先改」，若与矩阵裁决不符，怎么办？*

规则：**允许老板偏好动作占用 30 天件数中的最多 1 件**——前提是：
1. 该动作不违反 §1 红线（可逆、在约束内、有验收标准）；
2. **该偏好项在 Phase 1 中未被评为🔴不认同**（见下方冲突解决规则）；
3. Claude 须在计划书内部备注标注其「机会成本」（占了一个本可给①象限的位）；
4. 六哥判断：老板情绪强度高（Phase 0 标「很坚持」）→ 给这 1 个位换执行配合度，是划算的交换。

**偏好位适用范围**：
- 老板偏好项属于🟡半信（非数据最优但老板半认同）→ 可用偏好位
- 老板偏好项未进入 Phase 1 评估范围（如偏好项不在 TOP3 内）→ 可用偏好位
- 老板偏好项情绪强度「随口说说」→ 不触发偏好位机制，直接按矩阵排

**冲突解决规则（§3.3 协议优先于偏好位机制）**：

> *【最常见困难场景】老板 Phase 0 说「最想先改 X」（情绪强度：很坚持），但 Phase 1 诊断数据指向 X 不是最优动作，老板走完四步协议仍维持🔴不认同。此时两套机制同时激活，给出相反指令。优先级规则如下：*

**若老板偏好项在 Phase 1 已被评为🔴且四步协议走完仍维持🔴 → §3.3 协议优先，偏好位机制不适用**。

处置路径：将偏好项的最低成本验证动作**写入 D 档（§6.2）**，并在14天首检时带回事实——不进入 30 天签字动作。

*【为什么偏好位不能覆盖🔴决议】签了但数据不支持的动作，14天首检一旦验证失败，不仅浪费了一个30天名额，还破坏顾问判断的可解释性（违反宪法§1「讲不清推理的结论不签字」）。用D档验证动作取代偏好位，是用低成本保住长期信任的更优解。*

> *【张力·为什么给老板偏好位】纯按数据排的最优计划，若老板心里不服，执行率会打对折。给 1 个位让老板「做自己想做的」，是用一点理论最优换执行依从性——在社区小店（老板=唯一决策人+主要执行人）几乎总是划算的。但**只给 1 个位且须通过🔴测试**：给到 2 个，计划就从「顾问主导」滑向「老板拍脑袋」，失去诊断价值。*

## §7 Phase 5 — 首次进度检查设计（14 天首检·≤15 min）

> *【目的·为什么是 14 天不是 30 天】30 天才检查，等于给了拖延 4 周的空间——社区小店老板一忙，3 件事能拖到月底一件没动。14 天卡一刀，是「还来得及补救」的最后窗口：滞后了，剩下 16 天还能追；等到 30 天发现没做，这个诊断周期就废了。*

**执行方**：Claude 设计检查清单 + 设提醒，六哥定检查方式
**时间预算**：≤15 min
**输出物**：《14 天首检卡》
**质量标准**：3 件各有可现场核的进度判据；调整规则写死（超预期/滞后各怎么办）；每件均已确认14天中间态（见§7.0）。

### 7.0 14 天中间态推导规则（填表前执行·修订说明·新增补充原缺失）

> *【说明】原SOP「14天中间态」字段全靠六哥现场拍脑袋。本节补充推导规则，确保中间态可事先定义、首检时可客观核查。*

**推导规则（每件动作选一种）**：

| 动作类型 | 中间态推导方式 | 举例 |
|---|---|---|
| **量化目标型**（如SKU数/损耗率/价格）| 中间态 = 当前值到目标值之差的 50% | 目标：叶菜≥10种，当前6种，差4种 → 14天中间态：已补≥8种 |
| **里程碑型**（如换供应商/培训员工/调整陈列布局）| 中间态 = 首个独立可验证的子动作完成 | 换供应商 → 中间态：「已约谈≥2家备选供应商并获报价」 |
| **长周期型**（动作最短完成周期>14天，如设备采购）| 中间态 = 启动动作完成（可验证）| 采购冷柜 → 中间态：「已下询价单/已联系≥2家厂商」，记录在首检卡备注栏 |

### 7.2 调整规则（写死·防加压式管理）

| 状态 | 含义 | 处置规则 |
|---|---|---|
| **✅ 超预期** | 提前做完/效果超出 | **可加项**——从 parking lot 提 1 件①象限动作补进。但仍守「在跑动作」件数上限（标准≤3件，夫妻店变体≤2件）|
| **🟡 滞后** | 做了但没到中间态 | **找原因，不加压**——是资源不够？执行人没空？还是动作本身难？对症调（换人/拆小/延期），**绝不简单催「快点做」** |
| **🔴 未动** | 一点没动 | **回到 Phase 0 查约束是否摸错**——多半是当初工时/资金/执行人估错了，或老板根本没认同（🔴 项）。修约束，重排，不是骂老板 |

> *【张力·14 天首检的核心纪律】首检是「医生复诊」不是「老师查作业」。滞后的根因 80% 不在老板懒，而在**计划没穿透约束**（Phase 0 没摸准）或**问题没真共识**（Phase 1 的 🔴 项硬塞了）。把滞后当成对自己 Phase 0-1 质量的反馈，比当成老板的问题，更接近真相。*

- 14 天首检 = 中途纠偏（本SOP）；30 天复查 = 诊断卡 §9 的正式前后对比（重出诊断卡或核改造件）。
- 首检发现的约束误判 → 回填进诊断卡 §8 数据采集清单（哪些是当初估错的 C 级），让下一轮更准。

## §9 待六哥裁决（10% 不可代签）

> *【模板治理】本SOP v0.1 含「铁律/标准/红线/必须」字样，按《标准审议铁律》尚未完成三轮论证，升 stable 前须六哥逐项签字。*

### 待裁决项①：≤4h/周工时上限是否为硬铁律
**问题**：4h/周来自「社区小店老板时间现实」的推断（inferred），未经多店验证。夫妻店、有完整团队的店、连锁小区店的老板可投入时间差异很大。
**裁决选项**：① 固定 4h 为统一上限；② 改为「按店型分档」（夫妻店 2h / 有团队 6h / 老板甩手掌柜 1h）；③ 不设固定值，每店 Phase 0 现场定。
**六哥签字状态**：⬜ 待签字

### 待裁决项②：「老板偏好位」机制是否纳入标准
**问题**：给老板 1 个非最优动作位，是用理论最优换执行依从。这在社区小店成立，但若推广到有管理团队的客户，可能被视为「顾问不专业、迁就客户」。
**裁决选项**：① 纳入为标准机制（社区店场景，已附🔴测试门）；② 仅作为「可选话术」不写进 SOP 硬步骤；③ 删除，坚持纯数据排序。
**六哥签字状态**：⬜ 待签字

### 待裁决项③：14 天首检 vs 其他间隔
**问题**：14 天来自「30 天周期的中点 + 还来得及补救」的推断。是否对所有改造周期都最优，未验证。
**裁决选项**：① 固定 14 天；② 按动作最短周期定（取最快见效动作周期 ×1.5）；③ 双检查点（7 天轻量电话 + 14 天上门）。
**六哥签字状态**：⬜ 待签字

### 待裁决项④：本SOP与诊断卡 §6 的职责边界
**问题**：本SOP已将诊断卡 §6 定位为「顾问初稿」（Phase 2 Step 0 消费它），本SOP产出为「约束穿透后的老板共识版正式计划」。这是推荐方向，但未经六哥确认。
**裁决选项**：① 采纳上述定位（§6=顾问候选初稿，本SOP §5=老板共识后正式计划，SSOT在本SOP）；② 合并，诊断卡不再出 §6，直接调本SOP；③ 保持并行，注明 §6=顾问初稿、本SOP=落地终稿（当前已实施）。
**六哥签字状态**：⬜ 待签字

### 待裁决项⑤：Claude 接入模式是否须向老板声明
**问题**：模式A下，老板可能目睹六哥在现场使用设备辅助。大湾区社区生鲜超市老板对AI辅助的接受度尚未系统评估。
**裁决选项**：① 模式A下如实说「我用AI帮我整理数据」（当前建议）；② 不主动说明，老板问才解释；③ 所有首次合作客户强制用模式B。
**六哥签字状态**：⬜ 待签字
