# 商品诊断报告 模板定义 v0.1

> 本文件是「社区超市30天商品提效包」**商品诊断报告**的模板定义（顾问诊断结构标准），是诊断环节的字段与结构来源。
> **不是**客户真实报告、不是对外销售页、不是方法论总纲。
> **只引用、不重定义**：方法骨架见商品力总纲，武器见 M1-M8，决策规则见 M-DEC，工具承接见 SKU动作建议表。
> 上游：门店数据+现场 → **本报告** → 下游：[[SKU动作建议表_模板定义_v0.1]] → DisplayMap/店长执行 → 周复盘 → 结项报告。

## 1. 模板定位

社区超市30天商品提效包的**商品诊断报告结构标准**：诊断社区超市的商品结构/SKU表现/品类结构/价格带/毛利/库存/陈列/需求匹配，找出**本轮 30 天内可执行、可复盘、可转入 SKU动作表**的商品机会。**不写长篇咨询报告，产出可裁决的问题清单/机会清单/SKU动作输入**。

## 2. 使用场景

30天提效包项目首轮诊断；单/双重点品类深调改；多店复制时各店诊断；花厅坊样板复盘（验证素材，非主定义）。

## 3. 与 30天商品提效包的关系

属其**S04 商品诊断**环节的标准产物（见主定义 §9）；上承 S02 首访/S03 数据清洗，下接 SKU动作建议表（S05）。

## 4. 与商品力体系的关系

属 L4 落地的诊断封装；引用 L2 商品力（[[商品力总纲]]）+ 经营控制链（金额→数量→品质）；调用 L3 武器 M1-M6；**不重定义任何方法论**。

## 5. 与 RetailOS 的关系

RetailOS = 方法引擎（减法 A-G + 选品 4 闸）；本报告**调用其裁决逻辑产出诊断结论**，不重写 RetailOS、不另起理论层。

## 6. 与 SKU动作建议表的关系

本报告是 SKU动作建议表的**上游输入源**：诊断结论 → §14 SKU动作输入清单 → 映射进 SKU动作建议表（接口见 §17）。报告负责"发现问题/机会"，SKU表负责"逐 SKU 裁决+执行+复盘"。

> 完整主链：**本报告 → [[SKU动作建议表_模板定义_v0.1]] → [[周复盘表_模板定义_v0.1]] → [[结项报告_模板定义_v0.1]]**。诊断结论经 SKU动作表执行、周复盘验证后，在结项报告收口。

## 7. 报告使用对象

顾问主导撰写；老板可读（§15 摘要）；店长可理解执行（§16 转译）。

## 8. 报告边界

**做**：重点品类商品诊断、问题/机会清单、SKU动作输入、老板/店长摘要。
**不做**：全店全品类诊断、长篇咨询报告、对外销售页、业绩承诺、替代客户经营决策。

## 9. 标准报告结构（18 部分）

> 顾问按此结构撰写真实报告（本模板只定义结构，不含真实数据）。

1. **封面信息**：项目名称/门店/诊断周期/诊断品类范围/数据周期/现场日期/版本号/顾问/保密说明。
2. **诊断摘要**：核心结论/最大问题/最大机会/优先动作/不建议动作/需老板决策事项/需店长执行事项。
3. **门店与品类背景**：门店定位/客群特征/商圈要点/经营面积·品类范围/本轮诊断品类/数据完整性说明。
4. **数据输入与可信度**：销售/商品档案/库存/毛利/供应商/现场陈列/缺货记录/数据缺失项/数据可信度评级（见 §15 证据等级）。
5. **商品结构诊断**：SKU数量结构/品类·子品类结构/ABC·ABCZ结构/动销·滞销结构/长尾结构/新品·老品结构/停购·清尾结构/结构性问题判断。
6. **销售与毛利诊断**：销售贡献/毛利贡献/毛利率异常/负毛利·特价清仓识别/高销低毛利/低销高毛利/价格录入异常/促销期异常识别。
7. **库存与缺货诊断**：当前库存状态/可售天数/缺货风险/高销缺货/低销积压/停购未清/补货动作识别/淘汰·清尾动作识别。
8. **价格带诊断**：价格带分布/中价带是否过密/低中高价占比/价格带缺口/价格带重叠/高端补位机会/平价引流机会/与 [[M-DEC-010_休食小类价格带结构预警_v1.0]] 的关系。
9. **顾客需求匹配诊断**：高频便利/本地老人/儿童/健康/节日·季节/便捷早餐·即食/非显性需求/待验证需求。
10. **陈列与货架诊断**：当前货架位置/层板/排面数/黄金层占用/高销低位/低销高位/品类不集中/品牌不集中/是否进入 DisplayMap 调整。
11. **供应商与补货能力诊断**：主供应商表现/稳定性/供货能力/补货周期/引进能力/供应商临时问题识别/供应商能力库引用。
12. **问题清单**（字段见 §13 下表）。
13. **机会清单**（字段见 §14 下表）。
14. **SKU动作输入清单**（字段见 §17 接口）。
15. **老板汇报摘要**（结构见 §20）。
16. **店长执行摘要**（结构见 §21）。
17. **风险与边界**：数据不完整/季节误判/促销异常/缺货误判/陈列误判/特价清仓·负毛利误判/非显性需求误判风险 + 不承诺销售结果 + 不替代客户经营决策。
18. **下一步动作**：进入 SKU动作建议表 / DisplayMap调整 / 店长执行表 / 周复盘 / 结项报告。

## 10. 诊断输入要求

销售明细(近30/60/90天)/商品档案(品类·子品类·条码·货号)/库存(即时·库龄)/毛利·进价·售价/供应商(主供应商·可得性)/现场陈列(6机位)/缺货记录/数据缺失项 + 数据质量分级。口径见主定义 §11/§12；清洗执行走 `/posclean` skill + `13_数据分析与工具脚本/POS清洗库_v0.1/`（原 [[POS数据清洗与商品标准化规范_v0.1]] 已由其取代，2026-07-07）。

## 11. 诊断模块（10 个，必含）

1. 商品结构诊断　2. 销售与毛利诊断　3. 库存与缺货诊断　4. 价格带诊断　5. 顾客需求匹配诊断　6. 陈列与货架诊断　7. 供应商与补货能力诊断　8. 问题清单　9. 机会清单　10. SKU动作输入清单。

> 模块武器锚定：1/8→M2·M6；2→M3；3→M1；4→M3(M-DEC-010)；5→M5；6→M4；7→底座/供应商能力库。

## 12. 诊断结论分级

| 级别 | 含义 |
|---|---|
| **P0** | 必须立即处理 |
| **P1** | 本轮 30 天优先处理 |
| **P2** | 观察 / 后续处理 |
| **待确认** | 证据不足，需人工复核 |

## 16. 误判防线（沿用 SKU动作建议表 11 类，不新增）

① 新品保护期　② 季节/节日因素　③ 缺货导致低销　④ 陈列位置差导致低销　⑤ 价格异常导致低销　⑥ 规格不适配导致低销　⑦ 数据缺失/POS分类错误　⑧ 非显性需求　⑨ 供应商临时问题　⑩ 促销期异常高销　⑪ 负毛利/特价清仓误判。

## 17. 输出到 SKU动作建议表的接口

> §14 SKU动作输入清单 ≠ SKU动作建议表本身，是**诊断报告→SKU动作表的接口清单**。

**SKU动作输入清单字段**：SKU代号/货号 · 品类 · 子品类 · 问题/机会来源 · 初步建议动作 · 商品角色 · 证据来源 · 误判风险 · 是否需要人工确认。

**字段映射（诊断报告 → SKU动作建议表）**：

| 诊断报告 | → | SKU动作建议表 |
|---|---|---|
| 问题/机会来源 | → | 动作原因 |
| 诊断结论分级(P0/P1/P2) | → | 动作优先级 |
| 推荐动作方向 | → | 建议动作（10类） |
| 证据等级(A-E) | → | 证据来源 |
| 误判防线(①-⑪) | → | 误判风险 |
| 需求匹配判断 | → | 顾客需求点 / 是否待验证需求 |
| 价格带诊断 | → | 价格带位置 |
| 负毛利/特价清仓识别 | → | 是否特价清仓/负毛利异常 |
| 陈列诊断 | → | 是否进入DisplayMap调整 |

## 18. 输出到 DisplayMap 的接口

- **进入 DisplayMap**：换位/放大/缩面类、黄金层占用、高销低位/低销高位、品类·品牌不集中；
- **只进 SKU动作表不进 DisplayMap**：纯调价/淘汰/补货（不涉及货位变更）；
- **待陈列数据再进**：表E/M-DEC-011 未装表时标"待陈列数据"。

## 19. 输出到周复盘的接口

- **P0/P1 动作** → 进入 **7 天复盘**；
- **观察类/待确认类** → 进入 **15 天复盘**；
- **淘汰/引进/结构调整类** → 进入 **30 天复盘**。

## 20. 老板汇报摘要结构

三句话结论 / 三个关键问题 / 三个优先机会 / 需老板拍板事项 / 预计 30 天动作收益方向（**方向非承诺**）/ 不建议做的事。

## 21. 店长执行转译结构

本周先做什么 / 哪些 SKU 先确认 / 哪些陈列先拍照 / 哪些缺货先核实 / 哪些清尾先执行 / 哪些引进先找供应商确认。
