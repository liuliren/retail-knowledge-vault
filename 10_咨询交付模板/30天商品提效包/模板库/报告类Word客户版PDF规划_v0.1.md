# 报告类 Word 客户版 / PDF 规划 v0.1

## 1. 文件定位

- 本文件是报告类 Word **内部模板 → 客户版 → PDF** 的转换规划；
- **不是**客户版文件、**不是** PDF 文件、**不含**真实数据；
- 用于指导后续客户版生成、PDF 导出、脱敏检查与对外交付。
- internal_only，本身不发客户。

## 2. 当前内部版资产

| 内部模板 | 当前状态 | 用途 | 可直接对外 | 原因 |
|---|---|---|---|---|
| [[商品诊断报告_Word模板_使用说明_v0.1]]（docx）| 已生成(空模板) | 对内诊断报告 | ✗ | 含内部判断/方法接口/分级/附录脱敏规则 |
| [[结项报告_Word模板_使用说明_v0.1]]（docx）| 已生成(空模板) | 对内结项报告 | ✗ | 含内部判断/裁决细节/内部风险等级 |

**结论：内部版不可直接对外，必须转客户版（专业改写，非另存 PDF）。**

## 3. 内部版 / 客户版 / PDF版三层关系

1. **内部版 Word**：给顾问；保留诊断过程/证据来源/内部判断/方法接口/注意事项；internal_only。
2. **客户版 Word**：给老板/店长；保留结论/行动/边界/下一步；删内部方法痕迹与敏感字段；client_ready。
3. **客户版 PDF**：最终交付/汇报；锁版、不可编辑；单独 QC。

| 版本 | 使用对象 | 内容深度 | 可编辑 | 对外 | 安全要求 |
|---|---|---|---|---|---|
| 内部版 Word | 顾问 | 全（含判断/证据/方法接口）| 是 | ✗ | internal_only，不发客户 |
| 客户版 Word | 老板/店长 | 结论+行动+边界+下一步 | 是(客户填执行) | ✓(脱敏后) | 去内部痕迹/无敏感/无承诺 |
| 客户版 PDF | 老板/汇报 | 同客户版Word(锁版) | 否 | ✓ | 锁版+单独QC，仅从客户版导出 |

## 4. 客户版必须删除的内容

Claude / GPT / AI协作 / Obsidian / Git / commit / frontmatter / internal_only / RetailOS / M-DEC / 三力 / M1-M8 / 方法武器 / 误判引擎 / Cluster / 候选池 / P0.5·P1·P2 内部任务标记 / 内部价格策略 / 真实条码 / 原始 POS 明细 / 未脱敏 SKU 清单 / 内部审计备注 / 对客户不必要的技术字段。
> 这些可在内部版存在，客户版必须删除或改写。

## 5. 客户版必须保留的内容

项目背景 / 诊断范围 / 资料完整度 / 数据可信度 / 核心问题 / 关键发现 / SKU动作摘要 / 陈列·DisplayMap 动作摘要 / 客户需执行事项 / 晟果已完成事项 / 风险与边界 / 下一步建议 / 复盘节点 / 附录中必要字段说明。

## 9. 客户版禁止承诺清单

禁止：保证提升 / 一定增长 / 必然改善 / 确保业绩 / 业绩翻番 / 保证毛利提升 / 保证销售增长 / 保证库存下降 / 承诺 ROI / 承诺固定效果。

| 禁用表达 | 替代表达 |
|---|---|
| 保证销售提升 | 用于提升商品调整的可执行性，实际结果取决于数据质量、执行力度和顾客反应 |
| 确保库存下降 | 帮助识别库存结构问题，并形成分层处理建议 |
| 一定改善毛利 | 提供价格带与毛利结构的调整方向 |
| 承诺 ROI / 固定效果 | 提供可执行的商品动作与复盘机制，效果需结合门店实际执行评估 |

- 客户版不是内部版删几行，必须重写为客户能理解的语言；

- 不得把 AI/Git/Obsidian 工作流痕迹带给客户；
