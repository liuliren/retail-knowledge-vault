# 周复盘表 模板定义 v0.1

> 本文件是「社区超市30天商品提效包」**周复盘表**的模板定义（顾问与门店执行过程中的周度复盘结构标准）。
> **不是**客户真实复盘报告、不是结项报告、不是对外销售页。
> **只引用、不重定义**：动作枚举/误判防线沿用 [[SKU动作建议表_模板定义_v0.1]]，不新增。
> 主链位置：商品诊断报告 → SKU动作建议表 → **周复盘表** → 结项报告。

## 1. 模板定位

30天提效包的**动作复盘结构标准**：复盘 SKU动作/陈列/补货/清尾/引进/店长执行是否被执行、是否有效、是否需调整，产出"继续/调整/放大/停止/转入结项"判断。

## 2. 使用场景

30天提效包执行期周度复盘（7/15/30 天节点）；店长×顾问联合复盘；多店复制时各店复盘汇总；花厅坊样板复盘（验证素材，非主定义）。

## 3. 与 30天商品提效包的关系

属 **S07 经营复盘** 环节标准产物（见主定义 §9/§15）；7天观察→15天修正→30天结项，对接结项报告。

## 4. 与商品诊断报告的关系

承接 [[商品诊断报告_模板定义_v0.1]] §19 复盘接口：P0/P1 动作→7天；观察/待确认→15天；淘汰/引进/结构调整→30天。

## 5. 与 SKU动作建议表的关系

逐行复盘 SKU动作表的 G 裁决区动作 + H 复盘区字段；本表是 SKU动作表复盘区的**展开与跟踪载体**；动作枚举/误判防线**完全沿用，不新增**。

## 6. 与 DisplayMap 调整表的关系

陈列类动作（换位/放大/缩面）复盘需核对 DisplayMap 是否落地；缺陈列数据时标"待陈列数据"（表E/M-DEC-011 待装表）。

## 7. 复盘对象

SKU动作、陈列动作、补货动作、清尾动作、引进行动、店长执行动作。

## 8. 复盘周期

7天 / 15天 / 30天 / 临时复盘。

## 9. 复盘边界

**做**：动作执行核对、数据/现场变化核对、误判复核、有效性判断、下一步动作、老板/店长摘要、结项沉淀。
**不做**：真实复盘报告、结项报告本身、对外销售页、业绩承诺、替代客户经营决策。

## 10. 标准复盘流程（11 步）

1. 回看本周应执行动作 → 2. 核对执行状态 → 3. 核对数据变化 → 4. 核对现场变化 → 5. 复核误判风险 → 6. 判断动作有效性 → 7. 确认下一步动作 → 8. 同步责任人与期限 → 9. 输出老板摘要 → 10. 输出店长执行清单 → 11. 结论沉淀到结项报告。

## 11. 周复盘表结构（7 字段组）

### A. 基础信息
复盘日期 / 项目名称 / 门店 / 复盘周期 / 复盘周次 / 复盘品类 / 复盘人 / 参与人。

### B. 动作来源
来源表 / 来源编号 / SKU代号·货号 / 品类 / 子品类 / 原问题·机会 / 原建议动作 / 原动作优先级 / 原商品角色 / 原误判风险 / 原证据等级。

### C. 执行核对
责任人 / 计划完成日期 / 实际完成日期 / 当前执行状态 / 未完成原因 / 是否按建议执行 / 执行偏差说明 / 店长反馈。

### D. 数据变化
复盘前·后销售表现 / 销售变化 / 复盘前·后毛利表现 / 毛利变化 / 复盘前·后库存状态 / 库存变化 / 缺货变化 / 可售天数变化。

### E. 现场变化
陈列位置变化 / 排面变化 / 价格牌·促销牌变化 / 缺货状态变化 / 顾客反馈 / 店员反馈 / 现场照片是否补齐 / 是否需要 DisplayMap 复核。

### F. 误判复核
是否触发误判复核 / 触发的误判类型(§14.7) / 复核结论 / 是否需要人工确认 / 是否需要延长观察。

### G. 复盘结论
动作有效性 / 复盘结论分级 / 下一步动作 / 是否进入下周继续跟踪 / 是否进入结项报告 / 老板是否需拍板 / 备注。

## 12. 复盘字段说明

| 字段组 | 关键字段 | 填写责任 | 说明 |
|---|---|---|---|
| A 基础信息 | 复盘周期/周次/品类 | 顾问 | 定位本次复盘 |
| B 动作来源 | 来源表/编号/原动作/原误判风险 | 顾问预填 | 从 SKU动作表/诊断报告带入，**保持可回溯** |
| C 执行核对 | 执行状态/是否按建议执行/偏差 | 店长填 | 先看"做没做" |
| D 数据变化 | 销售/毛利/库存/缺货变化 | 系统派生+顾问 | 分档表达，不填真实金额 |
| E 现场变化 | 陈列/排面/价签/反馈/照片 | 店长+顾问 | 现场验证 |
| F 误判复核 | 是否触发/误判类型/复核结论 | 顾问 | 防误杀回路 |
| G 复盘结论 | 有效性/分级/下一步动作 | 顾问主导 | 产出处置 |

## 13. 复盘结论分级

| 级别 | 含义 |
|---|---|
| **P0** | 立即调整 |
| **P1** | 下周继续处理 |
| **P2** | 观察记录 |
| **结项沉淀** | 转入结项报告 |
| **待确认** | 证据不足，需人工复核 |

## 14. 枚举与字典

### 14.1 复盘周期
7天 / 15天 / 30天 / 临时复盘。

### 14.2 当前执行状态
未开始 / 进行中 / 已完成 / 暂缓 / 待确认 / 未执行。

### 14.3 是否按建议执行
是 / 否 / 部分执行 / 待确认。

### 14.4 销售·毛利·库存·缺货变化（统一）
明显改善 / 小幅改善 / 持平 / 小幅恶化 / 明显恶化 / 待观察 / 数据不足。

### 14.5 是否需要 DisplayMap 复核
是 / 否 / 待陈列数据。

### 14.6 是否触发误判复核
是 / 否。

### 14.7 触发的误判类型（沿用 11 类误判防线，不新增）
① 新品保护期　② 季节/节日因素　③ 缺货导致低销　④ 陈列位置差导致低销　⑤ 价格异常导致低销　⑥ 规格不适配导致低销　⑦ 数据缺失/POS分类错误　⑧ 非显性需求　⑨ 供应商临时问题　⑩ 促销期异常高销　⑪ 负毛利/特价清仓误判。
> 详见 [[SKU动作建议表_模板定义_v0.1]] §8。**不新增防线。**

### 14.8 动作有效性
有效 / 部分有效 / 无效 / 待观察 / 未复盘。

### 14.9 复盘结论分级
P0 立即调整 / P1 下周继续处理 / P2 观察记录 / 结项沉淀 / 待确认。

### 14.10 下一步动作
保持原动作 / 放大 / 缩面 / 调价 / 换位 / 补货 / 观察 / 淘汰 / 引进 / 待确认 / 结束观察 / 转入结项 / 暂停动作。
> 说明：放大/缩面/调价/换位/补货/观察/淘汰/引进/待确认属 SKU动作表 10 类动作（沿用）；**保持原动作 / 结束观察 / 转入结项 / 暂停动作 属复盘处置，非 SKU动作枚举扩展，不回写扩展 SKU动作表枚举**。

## 15. 动作有效性判断口径

### 有效（任一）
- 销售/毛利/库存/缺货/陈列状态明确改善；
- 店长执行到位且现场符合预期；
- 原问题明显缓解；
- 动作可继续或放大。

### 部分有效（任一）
- 有局部改善未达预期；
- 执行不完全致效果不完整；
- 数据改善但现场未同步；
- 现场改善但数据尚未体现。

### 无效（任一）
- 执行到位但数据与现场均无改善；
- 原判断被误判防线推翻；
- 动作方向错误；
- 客户执行条件不具备。

### 待观察（任一）
- 时间不足 / 数据不足；
- 新品/季节/节日因素需延长观察；
- 非显性需求仍需验证。

### 未复盘（任一）
- 未到复盘时间 / 数据未提供 / 现场未反馈 / 责任人未完成复盘。

## 16. 复盘问题类型字典（20 类）

未按期执行 / 执行偏差 / 数据不足 / 现场反馈不足 / 销售未改善 / 毛利未改善 / 库存未改善 / 缺货未改善 / 陈列未调整 / 顾客反馈不明确 / 店员执行阻力 / 供应商未配合 / 原判断误判 / 促销影响未剔除 / 季节因素未剔除 / 价格异常未修正 / 清尾执行不到位 / 引进未落地 / DisplayMap数据缺失 / 需要老板决策。

## 17. 误判复核机制

任何"无效/恶化"结论，先过 §14.7 十一类误判防线复核：是否因缺货/季节/陈列/价格/促销/特价清仓等导致的**假无效**？若触发，标"延长观察"或"转待确认"，不直接判动作失败（防止把执行/外因问题误记为方法错误）。

## 18. 输出到结项报告的接口

> 下游收口 → [[结项报告_模板定义_v0.1]]（其 §16 周复盘结果总结直接承接本节 14 项）。

进入结项报告的内容：①本轮完成动作数 ②未完成动作数 ③有效动作数 ④部分有效动作数 ⑤无效动作数 ⑥待观察动作数 ⑦P0/P1 处理完成情况 ⑧SKU动作类型分布 ⑨问题类型分布 ⑩机会类型分布 ⑪典型有效案例 ⑫典型无效/误判案例 ⑬仍需老板决策事项 ⑭后续 30/60/90 天建议。

## 19. 老板摘要结构

①本周执行了什么 ②哪些动作有效 ③哪些动作无效 ④哪些问题需老板拍板 ⑤下周最重要的 3 个动作 ⑥需客户补充的数据/资源/决策。

## 20. 店长执行转译结构

①哪些动作继续做 ②哪些动作暂停 ③哪些 SKU 继续观察 ④哪些 SKU 需补货 ⑤哪些 SKU 需清尾 ⑥哪些 SKU 需改陈列 ⑦哪些数据/照片/反馈要补。

## 21. 风险边界

数据不完整/季节误判/促销异常/缺货误判/陈列误判/特价清仓·负毛利误判/非显性需求误判风险；**不承诺销售结果、不替代客户经营决策**；旧复盘模板需经本定义校准后才能对外使用。
