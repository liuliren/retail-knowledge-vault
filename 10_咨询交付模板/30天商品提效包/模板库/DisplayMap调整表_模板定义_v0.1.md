# DisplayMap调整表 模板定义 v0.1

> 本文件是「社区超市30天商品提效包」**陈列动作承接模板**的定义。**内部模板，不直接发客户。**
> **只引用、不重定义**：陈列动作类型≠SKU动作枚举（不新增/不改 10 类 SKU动作）；误判防线沿用 11 类（不新增）。
> 主链位置：商品诊断报告 + SKU动作建议表 → **DisplayMap调整表** → 周复盘表 → 结项报告。

## 1. 模板定位

DisplayMap调整表是 30天商品提效包的**陈列动作承接模板**——把商品诊断和 SKU动作建议中与陈列相关的动作，转成门店能执行、后续能复盘的陈列调整清单。

**它不是**：完整陈列系统 / 精细货架图工具 / 3D 陈列设计 / 真实客户陈列方案 / 表E·M-DEC-011 的替代品 / Excel 工具版 / 图片版陈列图。

## 2. 使用场景

商品诊断后 → SKU动作建议表完成后 → 门店执行前 → 周复盘前 → 陈列专项前置判断。

## 3. 与 30天商品提效包关系

四件套之间的**陈列动作承接表**（非完整陈列系统）：
```
商品诊断报告（陈列/货架问题）+ SKU动作建议表（换位/放大/缩面…）
   → DisplayMap调整表（本表：逐项陈列动作）
   → 周复盘表（陈列执行复盘） → 结项报告（陈列成果总结）
```
对应主定义 §13/§14、SKU动作表 §14、诊断报告 §10/§18 的下游引用。

## 4. 上游输入

| 字段 | 来源 |
|---|---|
| 门店 / 品类 / 货架·区域 / SKU·商品 | 诊断报告 + 现场 |
| 来自诊断的问题 | [[商品诊断报告_模板定义_v0.1]] §10/问题清单 |
| 来自 SKU动作表的建议动作 | [[SKU动作建议表_模板定义_v0.1]] G 区 |
| 销售/库存/毛利简要依据 | SKU动作表 B/C/D 组（分档表达）|
| 现场照片/货架数据是否具备 | 现场（决定能否给精确货位）|
| 数据可信度 | 诊断报告证据等级 A–E |

## 5. 是否进入 DisplayMap 的判断规则

**进入**：换位 / 放大 / 缩面 / 补货后陈列恢复 / 淘汰后腾位 / 引进新品上架位置 / 端架·堆头机会 / 现场复核。

**不进入**：纯调价 / 纯供应商调整 / 纯数据待确认 / 纯毛利修正 / 无陈列影响的后台动作。

**特例**：动作进入但**缺现场/货架数据** → 进表但标 **"待陈列数据"**（见 §9），不强行给货位。

## 6. 字段设计（5 组，38 字段）

### A. 基础信息（7）
门店 / 品类 / 子品类 / 货架·区域 / 调整日期 / 顾问 / 店长·执行人。

### B. 商品与问题来源（7）
SKU·商品 / 商品角色 / 当前 SKU动作 / 问题来源 / 问题类型 / 证据来源(A–E) / 数据可信度。

### C. 当前陈列状态（6）
当前区域 / 当前货架 / 当前层位 / 当前排面 / 当前是否缺货 / 当前陈列问题。

### D. 建议陈列动作（11）
是否进入DisplayMap / 陈列动作类型(§7) / 建议区域 / 建议货架 / 建议层位 / 建议排面 / 是否需要腾位 / 腾位来源 / 是否需要补货 / 是否需要现场复核 / **待陈列数据标记**。

### E. 执行与复盘（7）
执行责任人 / 计划完成日期 / 实际完成状态 / 周复盘日期 / 复盘结论 / 下一步动作 / 备注。

> 合计 38 字段。C/D 组的"层位/排面/精确货位"列在缺数据时留空，并在"待陈列数据标记"标注。

## 7. 陈列动作类型字典（12 类）

| 动作 | 含义 |
|---|---|
| 换位 | 移到更合适货架/区域 |
| 上移 | 移到更好层位 |
| 下移 | 移到较低层位 |
| 放大排面 | 增排面（需动销+库存+资源支撑）|
| 缩小排面 | 减排面 |
| 恢复陈列 | 补货后恢复正常陈列 |
| 清退腾位 | 淘汰品清出、腾位 |
| 新品上架 | 引进品上架位置（置换式）|
| 端架/堆头尝试 | 端架或堆头机会 |
| 维持不动 | 现状合理，不调 |
| 待陈列数据 | 缺数据，暂不给货位 |
| 现场复核 | 需现场人工确认 |

> ⚠ **这是陈列动作类型，不是 SKU动作枚举**——本文件**不新增、不修改** [[SKU动作建议表_模板定义_v0.1]] 的 10 类 SKU动作。两者是不同维度（SKU动作=对商品的处置，陈列动作=对货位的处置）。

## 8. 误判防线（沿用 11 类，不新增）

① 新品保护期 ② 季节/节日 ③ 缺货导致低销 ④ 陈列位置差导致低销 ⑤ 价格异常 ⑥ 规格不适配 ⑦ 数据缺失/POS分类错 ⑧ 非显性需求 ⑨ 供应商临时问题 ⑩ 促销期异常高销 ⑪ 负毛利/特价清仓。详见 [[SKU动作建议表_模板定义_v0.1]] §8。

**陈列场景重点**：④ 陈列位置差（先换位观察再判去留，勿误判商品差）；③ 缺货（恢复陈列前先确认非缺货假滞销）；① 新品（上架观察期不足勿急调）；②/⑩/⑪（季节/促销/清仓品的陈列不按常规放大）。**不新增防线。**

## 9. 待陈列数据处理规则

缺现场照片 / 货架信息 / 层位信息 / 排面信息时：
- **不给精确货位**；
- **不给具体排面数**；
- **不强行判断黄金层 / 动线 / 端架**；
- 统一标记 **"待陈列数据"**；
- 输出**补采清单**（需补：货架编号 / 层数 / 排面 / 现场照片 / 动线图）；
- 待数据补齐后**二次判断**。

> 锚定主定义 §13/§14 与全局口径"缺陈列数据统一标'待陈列数据'"。

| 本表汇总 | → [[结项报告_模板定义_v0.1]] §10/§19 |

## 12. 使用边界

- 不替代完整陈列规划；
- 不替代货架图软件；
- 不替代店长现场判断；
- 不在数据不足时生成精确货位；
- 不承诺陈列调整一定提升销售。

## 13. 安全检查（自检）

无真实数据/SKU/商品名/货架图；未改 SKU动作·商品角色枚举；未新增误判防线；缺数据强制"待陈列数据"；纯模板定义。
