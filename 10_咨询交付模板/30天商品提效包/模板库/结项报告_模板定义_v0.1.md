# 结项报告 模板定义 v0.1

> 本文件是「社区超市30天商品提效包」**结项报告**的模板定义（30 天阶段项目收口结构标准）。
> **不是**客户真实结项报告、不是对外销售页、不是长期陪跑方案。
> **只引用、不重定义**：动作枚举/误判防线沿用 [[SKU动作建议表_模板定义_v0.1]]，不新增。
> 主链收口位置：商品诊断报告 → SKU动作建议表 → 周复盘表 → **结项报告** →（承接）后续产品。

## 1. 模板定位

30天提效包的**项目收口报告结构标准**：向老板说明本轮做了什么、解决了什么、哪些有效、哪些未完成、下一步怎么接；产出成果总结/动作复盘/问题闭环/机会沉淀/后续建议。

## 2. 使用场景

30天提效包阶段项目结束时的结项汇报；老板决策下一阶段；多店复制各店结项汇总；花厅坊样板阶段总结（验证素材，非主定义）。

## 3. 与 30天商品提效包的关系

属 **S08 结项与案例化** 环节标准产物（见主定义 §9）；是 30 天阶段项目的终点交付件，承接第三阶陪跑/下一阶段产品。

## 4. 与商品诊断报告的关系

承接 [[商品诊断报告_模板定义_v0.1]] 的诊断结论/问题清单/机会清单/SKU动作输入/数据可信度/误判防线/老板·店长摘要（接口见 §接口.1）。

## 5. 与 SKU动作建议表的关系

承接 [[SKU动作建议表_模板定义_v0.1]] 的建议动作/优先级/角色/原因/执行状态/有效性/下一步动作/复盘备注（接口见 §接口.2）。

## 6. 与周复盘表的关系

承接 [[周复盘表_模板定义_v0.1]] §18 的 **14 项结项接口**（完成/未完成/有效性分布/P0·P1 完成/类型分布/典型案例/老板决策/90天建议）（接口见 §接口.3）。

## 7. 与后续产品承接的关系

输出到：商品力长期陪跑 / 多品类专项 / DisplayMap 陈列专项 / 商品数据库·配置优化 / 多店对比诊断 / 店长商品力训练（接口见 §接口.4）。

## 8. 报告使用对象

老板主读（一页摘要）；顾问汇报撰写；店长接收后续执行清单。

## 9. 报告边界

**做**：成果总结、动作复盘、问题闭环、机会沉淀、后续建议、老板决策事项。
**不做**：业绩结果承诺、替代客户经营决策、把 30 天阶段项目包装成长期经营成果、长期陪跑方案本身。

---

## 10. 标准报告结构（18 部分）

1. **封面信息**：项目名称/门店/项目周期/结项日期/诊断品类范围/数据周期/顾问/版本号/保密说明。
2. **一页式结项摘要**：本轮做了什么 / 发现什么 / 改了什么 / 哪些有效 / 哪些无效或待观察 / 需老板决策事项 / 下一阶段建议（老板一页看懂）。
3. **项目目标与边界回顾**：项目目标/诊断范围/不做什么/数据·现场输入边界/客户·顾问责任/不承诺事项。
4. **项目执行过程回顾**：启动与资料收集→数据检查→现场诊断→商品诊断→SKU动作建议→店长执行→周复盘→结项汇报。
5. **商品诊断结论回顾**：商品结构/销售毛利/库存缺货/价格带/需求匹配/陈列货架/供应商/数据质量 八类问题。
6. **SKU动作执行总结**（字段见 §15）。
7. **周复盘结果总结**（14 项，见 §16 / §接口.3）。
8. **商品结构变化总结**（见 §17）。
9. **销售/毛利/库存变化总结**（见 §18，**只述观察到的变化，不承诺因果收益**）。
10. **陈列与 DisplayMap 相关总结**（见 §19）。
11. **问题闭环情况**（字段见 §20）。
12. **机会沉淀清单**（字段见 §21）。
13. **未完成事项清单**（字段见 §22）。
14. **风险与误判复核**（11 类防线，见 §23）。
15. **老板决策事项**（字段见 §24）。
16. **店长后续执行清单**（字段见 §25）。
17. **后续 30/60/90 天建议**（见 §26）。
18. **下一阶段产品承接**（见 §27 / §接口.4）。

## 11. 结项输入要求

来自三上游模板的结构化结论（诊断报告/SKU动作表/周复盘表）+ 项目过程记录 + 数据质量分级。**不导入真实 POS 明细，只用结构化结论与分档表达。**

## 12. 结项摘要结构（一页式）

本轮做了什么 / 发现什么 / 改了什么 / 哪些动作有效 / 哪些无效或待观察 / 需老板决策事项 / 下一阶段建议。

## 13. 项目执行回顾

按 S01–S08 主链回顾：线索判断→首访→数据清洗→商品诊断→SKU动作→店长执行→周复盘→结项；每段记"做了什么+产出物+偏差"。

## 14. 诊断结论回顾

回顾八类诊断结论（§10.5），标注哪些已转动作、哪些进机会沉淀、哪些仍待确认。

## 15. SKU动作执行总结

**字段**：动作类型 / 建议数量 / 已执行数量 / 未执行数量 / 部分执行数量 / 有效数量 / 部分有效数量 / 无效数量 / 待观察数量 / 主要原因 / 后续处理建议。

**动作类型（沿用 10 类 SKU 动作，不新增）**：保留 / 放大 / 缩面 / 调价 / 换位 / 补货 / 观察 / 淘汰 / 引进 / 待确认。

## 16. 周复盘结果总结（承接周复盘表 14 项）

①完成动作数 ②未完成动作数 ③有效动作数 ④部分有效动作数 ⑤无效动作数 ⑥待观察动作数 ⑦P0/P1 处理完成情况 ⑧SKU动作类型分布 ⑨问题类型分布 ⑩机会类型分布 ⑪典型有效案例 ⑫典型无效/误判案例 ⑬仍需老板决策事项 ⑭后续 30/60/90 天建议。

## 17. 商品结构变化总结

SKU数量变化 / 子品类结构变化 / A·B·C·Z 结构变化 / 长尾压缩情况 / 待淘汰清尾情况 / 待引进补位情况 / 结构改善判断。（分档表达，不填真实数值）

## 18. 销售/毛利/库存变化总结

销售变化 / 毛利变化 / 库存变化 / 缺货变化 / 可售天数变化 / 负毛利·特价清仓处理情况 / 数据不足说明。
> **不得承诺因果性收益**，只能表述：本轮观察到的变化 / 与动作相关的变化 / 待继续验证的变化。

## 19. 陈列与 DisplayMap 相关总结

已执行陈列动作 / 未执行陈列动作 / 高销低位修正 / 低销高位修正 / 排面调整情况 / 换位动作情况 / 待 DisplayMap 复核事项 / 缺陈列数据事项（表E/M-DEC-011）。

## 20. 问题闭环情况

**字段**：问题编号 / 问题类型 / 原问题描述 / 是否已处理 / 处理动作 / 当前状态 / 是否需要继续跟踪 / 责任人 / 备注。

## 21. 机会沉淀清单

**字段**：机会编号 / 机会类型 / 机会描述 / 本轮是否执行 / 执行动作 / 当前结果 / 后续建议 / 是否进入下一阶段。

## 22. 未完成事项清单

**字段**：事项编号 / 未完成事项 / 来源 / 未完成原因 / 当前影响 / 建议处理方式 / 责任对象 / 建议完成时间。

## 23. 风险与误判复核（沿用 11 类误判防线，不新增）

① 新品保护期　② 季节/节日因素　③ 缺货导致低销　④ 陈列位置差导致低销　⑤ 价格异常导致低销　⑥ 规格不适配导致低销　⑦ 数据缺失/POS分类错误　⑧ 非显性需求　⑨ 供应商临时问题　⑩ 促销期异常高销　⑪ 负毛利/特价清仓误判。

说明：本轮触发了哪些误判复核 / 哪些判断被修正 / 哪些仍需观察 / 哪些结论不能下最终判断。详见 [[SKU动作建议表_模板定义_v0.1]] §8。**不新增防线。**

## 24. 老板决策事项

**字段**：决策事项 / 背景 / 可选方案 / 建议方案 / 影响 / 需要老板决定的时间 / 不决策的风险。

## 25. 店长后续执行清单

**字段**：执行事项 / 对应品类·SKU代号 / 执行动作 / 责任人 / 完成期限 / 检查方式 / 备注。

## 26. 后续 30/60/90 天建议

- **后续 30 天**：继续执行 / 修正 / 观察；
- **后续 60 天**：品类结构优化 / 供应商协同 / 陈列固化；
- **后续 90 天**：进入长期商品力陪跑 / 多品类复制 / 门店经营机制化。

## 27. 下一阶段产品承接

本报告可承接到：商品力长期陪跑 / 多品类专项提升 / DisplayMap 陈列优化专项 / 商品数据库·商品配置优化 / 多店连锁对比诊断 / 店长商品力训练。

已闭环 / 部分闭环 / 未闭环 / 待观察 / 转下一阶段。

继续执行 / 放大验证 / 修正动作 / 延长观察 / 进入下一阶段 / 暂停处理 / 需要老板决策 / 需要客户补数据。

已验证 / 初步有效 / 待观察 / 未验证 / 转下一阶段。

### 5. 老板决策优先级

## 接口

### 接口.1 来自商品诊断报告
诊断结论 / 问题清单 / 机会清单 / SKU动作输入清单 / 数据可信度 / 误判防线 / 老板摘要 / 店长执行摘要 → 进入 §5/§11/§12/§23 与一页摘要。

### 接口.2 来自 SKU动作建议表
建议动作 / 动作优先级 / 商品角色 / 动作原因 / 执行状态 / 是否动作有效 / 下一步动作 / 复盘备注 → 进入 §15 SKU动作执行总结 与 §20 问题闭环。

### 接口.3 来自周复盘表（14 项）
完成/未完成/有效/部分有效/无效/待观察动作数 + P0·P1 完成情况 + SKU动作类型分布 + 问题·机会类型分布 + 典型有效案例 + 典型误判案例 + 老板决策事项 + 30/60/90 天建议 → 进入 §16 周复盘结果总结。

### 接口.4 输出到后续产品承接
- 结构性/长期问题 → 商品力长期陪跑；
- 陈列类结论 → DisplayMap 陈列专项；
- 配置/数据类结论 → 商品配置·商品数据库优化；
- 可复制结论 → 多品类复制 / 多店对比诊断；
- 执行力/能力类结论 → 店长商品力训练。
