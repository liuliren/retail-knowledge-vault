# 一线执行工具_Excel工具版_使用说明

> 配套文件：`一线执行工具_Excel工具版_v0.1.xlsx`（同目录）。该 xlsx 受 `.gitignore` 忽略，本地可用、不入 git。

## 0. 工具定位

- 本工具服务社区超市30天商品提效包，用于**一线门店执行**；
- **不是**商品诊断表、**不是**采购单、**不是**正式销售数据表；
- **不含真实商品数据**（空模板 + 示例行）。

## 1. 工作簿结构

5 个 sheet：

1. **使用说明**——各 sheet 怎么用；
2. **今日动作执行卡**——店员当天做什么；
3. **7天动作追踪**——店长/主管一周跟踪；
4. **商品动作RACI**——角色责任参照；
5. **门店语言对照**——顾问语言转门店语言。

## 2. 今日动作执行卡怎么用

- **店长 / 主管预填**（商品名称、今天动作、位置、做完标准）；
- **店员只看**"今天动作"和"做完标准"，照着做；
- **调价 / 缩面 / 放大必须写清数量或价格**（如"缩到1个面""改价签→3.5元"）；
- **引进类动作待到货日再下发**（货没到不发卡）；
- 店员**不要自行判断引进 / 淘汰**；
- "验收人·状态"用下拉：待执行 / 待验收 / 已完成 / 待整改 / 暂缓。

## 3. 7天动作追踪怎么用

- **店长 / 主管每日勾选**；
- 周一到周日只填 **✔ / 未 / 暂缓**（下拉）；
- **未完成必须写原因**；
- 店长**每周确认一次**（"店长确认"下拉：已确认 / 待确认 / 需跟进）；
- 可按"责任人"列筛选，每人只看自己的。

## 4. 商品动作 RACI 怎么用

- **R = 负责执行 / A = 最终负责 / C = 参与协商 / I = 被告知**；
- **顾问退出后**，"确认商品动作清单"的 **R 由店长承接**；
- **无专职主管时**，主管列职责由**店长兼任**；
- **采购**只负责补货 / 引进 / 淘汰相关确认，**不负责现场摆货**；
- 该 sheet 是静态参照，不需要填。

## 5. 门店语言对照怎么用

- 用于把**顾问语言转成店员能懂的话**；
- 写执行卡时**必须用门店语言**；
- 避免使用 SKU、M-DEC、毛利、ABC 等内部术语。

## 6. 打印与使用建议

- **今日动作执行卡建议 A4 横向打印**；
- **每天只打印当天要做的动作**；
- **7天追踪建议店长保管**；
- **RACI 和语言对照可打印一份贴在办公室**。

## 7. 使用边界

- 不替代商品诊断；
- 不替代采购确认；
- 不替代周复盘；
- 不作为正式采购单；
- 不作为销售数据表；
- 不承诺动销结果；
- 不处理真实数据。
