# DisplayMap调整表 Excel工具版 使用说明 v0.1

> 配套文件：`DisplayMap调整表_Excel工具版_v0.1.xlsx`（同目录）。字段口径以 [[DisplayMap调整表_模板定义_v0.1]] 为准。
> ⚠ **internal_only，不发客户**；Batch 1 第二件表格类工具，复用周复盘 Excel 范式。

## 1. 文件定位

30天商品提效项目**陈列动作承接工具**；承接商品诊断报告 + SKU动作建议表，把陈列相关动作转成门店可执行、可复盘的陈列调整清单，输出到周复盘表 + 结项报告。
**它不是**：完整陈列系统 / 货架图工具 / 3D / 平面图 / 表E·M-DEC-011 替代品 / 图片版陈列图。

## 2. 使用前准备

- 已完成商品诊断报告与 SKU动作建议表；
- 已采集现场照片 / 货架·层位·排面信息（缺则按"待陈列数据"处理）；
- 复制本 xlsx 为项目副本，**先删除示例行**再填（真实数据不入 git，见 §11）。

## 3. 各 Sheet 说明

| Sheet | 用途 |
|---|---|
| README_使用说明 | 使用步骤与纪律 |
| ①DisplayMap主表 | 主填写表（38列/5字段组/16下拉）|
| ②陈列动作字典 | 12 类陈列动作含义/场景/前置 |
| ③待陈列数据字典 | 9 类缺数据处理"缺什么不做什么判断" |
| ④字段说明 | 38 字段含义/必填/来源 |
| ⑤脱敏示例 | 脱敏规则与示例说明 |

## 4. 主表填写步骤

1. **A 基础信息**：门店/品类/子品类/货架·区域/调整日期/顾问/店长·执行人；
2. **B 商品与问题来源**：从诊断报告/SKU动作表带入 SKU·商品角色·当前SKU动作·问题来源/类型·证据来源·数据可信度；
3. **C 当前陈列状态**：当前区域/货架/层位/排面/是否缺货/陈列问题（缺数据留空）；
4. **D 建议陈列动作**：是否进入DisplayMap → 陈列动作类型(12类) → 建议货位；缺数据时陈列动作类型标"待陈列数据"+对应"待陈列数据标记"，**不给精确货位**；
5. **E 执行与复盘**：责任人/期限/完成状态/复盘结论/下一步动作（"转入周复盘"的进周复盘Excel）。

## 5. 与商品诊断报告的接口

诊断报告 §10 陈列与货架诊断 / 问题清单（陈列类）→ 本表 B 组问题来源/问题类型 + C 组当前陈列状态。

## 6. 与 SKU动作建议表的接口

SKU动作表中"是否进入DisplayMap=是"的换位/放大/缩面/淘汰腾位/引进上架动作 → 本表逐项承接（当前SKU动作 + 陈列动作类型）。

## 7. 与周复盘表 Excel 的接口

本表 SKU·商品/当前SKU动作/陈列动作类型(=原建议动作)/实际完成状态/复盘结论/下一步动作 → 进 [[周复盘表_Excel工具版_v0.1]]（"是否来自DisplayMap"=是、"陈列动作"取此处）。脱敏导入，不自动化。

## 8. 待陈列数据规则（核心）

**缺什么，不做什么判断**（详见 ③待陈列数据字典）：
- 缺现场照片 → 不判黄金位/动线/端架；
- 缺货架编号 → 不给具体货架；
- 缺层位信息 → 不给上移/下移；
- 缺排面信息 → 不建议具体排面数；
- 缺库存数据 → 不建议放大排面；
- 缺销售数据 → 不基于动销做放大/缩面。
缺数据时：陈列动作类型="待陈列数据" + 待陈列数据标记=对应缺失项 + 输出补采清单 → 补齐后二次判断。

## 9. 脱敏规则

门店=示例门店A；SKU=SKU-DEMO-XXX；货架=示例货架A；人员=顾问A/店长A。**禁止**真实门店/客户/SKU/条码/品名/销售额/花厅坊/真实货架图。

## 10. 常见错误

- ❌ 缺货架数据硬填精确货位 → ✅ 标"待陈列数据"+缺失项；
- ❌ 把陈列动作类型当成新增 SKU动作 → ✅ 它是陈列维度（12类），SKU动作沿用 10 类；
- ❌ 缺库存/销售就放大排面 → ✅ 先补数据；
- ❌ 自创商品角色/误判防线 → ✅ 沿用 10/11 类。
