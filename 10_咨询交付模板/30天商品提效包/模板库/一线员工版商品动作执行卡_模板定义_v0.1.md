# 一线员工版商品动作执行卡_模板定义

## 0. 模板定位

- 这是给**店员 / 理货员 / 主管**看的**执行卡**；
- **不是**顾问分析表，**不是** [[SKU动作建议表_模板定义_v0.1]] 完整版（那是顾问/加强版 8 字段组）；
- 目标：让一线知道「**做什么、在哪做、怎么做、做完什么样、谁验收**」。

## 1. 使用对象

- **店员 / 理货员**：只看执行字段，负责动手（摆货/贴签/整理/上架）；
- **主管**：督导货架调整、现场检查、拍照；
- **店长**：分派、验收、收口。

> 店员只看执行字段；店长 / 主管负责判断和验收。

## 2. 使用场景

调整货架 / 放大排面 / 缩面 / 换位 / 补货 / 淘汰（卖完清掉）/ 引进到货后上架 / 价签整理 / 陈列整理 / 拍照验收。

## 3. 设计原则

- **每行一个商品动作**；
- **字段不超过 5 个核心字段**；
- **不出现顾问内部术语**；
- **不要求店员理解分析逻辑**；
- 店员只负责**执行和反馈异常**；
- 店长 / 主管负责**判断和验收**。

## 4. 门店语言转换规则

| 顾问语言 | 门店语言 |
|---|---|
| SKU 动作建议 | 这个货要怎么处理 |
| 放大排面 | 多摆几个面 / 给更大位置 |
| 缩面 | 少摆点 / 腾位置给好卖的 |
| 淘汰 | 卖完不再进 |
| 引进 | 新进这个货 |
| 误判防线 | 先别急着下架，看是不是缺货 / 换季 / 位置差 |
| 商品角色 | 这个货是招客的 / 赚钱的 / 凑数的 / 该清的 |
| DisplayMap | 货架怎么摆的图 |
| 周复盘 | 每周看一次做得怎么样 |
| 动销异常 | 这货突然卖得特别快 / 特别慢 |

## 5. 卡片字段定义

> 核心 **5 字段**（店员只填/只看这些）：

| 字段 | 门店叫法 | 填写说明 | 示例 | 是否必填 |
|---|---|---|---|---|
| 商品名称 | 什么货 | 写商品名，不写条码 | 示例饼干A | 是 |
| 今天动作 | 今天做什么 | 用门店语言（多摆面/少摆点/补货/卖完清掉…）| 多摆 2 个面 | 是 |
| 位置 / 货架 | 在哪做 | 哪个区、第几层 | 饼干区第2层 | 是 |
| 做完标准 | 做完什么样 | 看得见的标准（正面朝外/价签对齐/拍照）| 正面朝外、价签对齐、拍照 | 是 |
| 验收人 / 完成状态 | 谁验、做完没 | 主管/店长 + 待验收/已完成 | 主管 / 待验收 | 是 |

可选字段：未完成原因、照片编号。

## 6. 动作类型门店表达

> 沿用 30天包 10 类 SKU 动作，用门店语言表达（**不新增动作类**）：

| 动作（内部）| 门店表达 |
|---|---|
| 保留 | 照常摆，不动 |
| 放大 | 多摆几个面 |
| 缩面 | 少摆点、腾位置 |
| 调价 | 改价签（按店长给的价）|
| 换位 | 挪到指定货位 |
| 补货 | 补满、别断货 |
| 观察 | 先别动、再看几天 |
| 淘汰 | 卖完不再进、清掉 |
| 引进 | 新货到了上架 |
| 待确认 | 等店长确认再做 |

## 7. 验收规则

- **做完必须有人看**（店长 / 主管当天或次日验收）；
- **关键动作必须拍照**（换位/放大/引进上架/价签）；
- **未完成必须写原因**（缺货/没空/不会做/货没到）；
- **店员不能自行改动作**（要改先问店长 / 主管）；
- 验收人在"完成状态"标"已完成/待整改"。

## 8. 不给一线员工看的字段

条码 / SKU 编码 / ABC 分类 / 双ABC / PSD / 毛利率数值 / M1-M8 / M-DEC / 误判防线 11 类全称 / 证据来源 / 动作优先级 P0·P1·P2（改为"今天做/本周做/先别动")/ 内部裁决备注 / 供应商谈判信息 / 价格带建模。

## 9. 示例结构

> 示例（非真实商品）：

| 商品名称 | 今天动作 | 位置 / 货架 | 做完标准 | 验收人 / 状态 |
|---|---|---|---|---|
| 示例饼干A | 多摆 2 个面 | 饼干区第2层 | 正面朝外、价签对齐、拍照 | 主管 / 待验收 |
| 示例冲调B | 补满别断货 | 冲调区第3层 | 补到满层、先进先出 | 主管 / 待验收 |
| 示例薯片C | 卖完不再进、清掉 | 休食区端架 | 清空、撤价签 | 店长 / 待验收 |
