# 咨询作战卡片 · 流程×卡片矩阵查阅表 v1.0

> **用法**：出卡时查本表——当前在哪个环节(S)→该产哪些卡(受众L×卡型)→用哪张 mdcard 样例。**本表是查阅 reference，不进代码**（业务流变了改本 doc 便宜）。渲染一律交 `/mdcard`。
> **受众层**：L0内部六哥 / L1店方老板 / L2店长 / L3操作岗 / L4公域。**存放**：卡组就地跟源诊断件同目录，受众后缀区分（见 mdcard 存放规范）。

## 全流程卡片矩阵

| 环节 | 触发 | 产哪些卡（L层·卡型） | 装配武器(01_) | 备注 |
|---|---|---|---|---|
| **S01 线索/销售** | 获客/开局 | L0作战卡(销售策略/报价) | 三层销售打法/报价单 | ⚠️线索期**不出L4对外卡**(未成交·避授权风险) |
| **S02 首诊·档一** | 远程POS快诊 | L0作战卡(六哥判断) + L1老板诊断卡(半张·商品力+客流) | 首诊产品档一/7D D1D2D4 | 出"半张卡"·标待现场 |
| **S02 首诊·档二** | 到店全诊 | L0到店作战卡(采集清单) → L1老板整张诊断卡 | 首诊产品档二/7D全维 | 花厅坊已实跑(样板) |
| **S03 数据清洗** | POS到手 | L0口径卡(三层口径/失真声明) | POS清洗规范/数据治理 | 补v0.1漏项·口径是判断输入 |
| **S04 商品诊断/品类** | 数据量化完 | L1诊断成果卡 + L2店长执行卡 + L3采购操作卡 | 三件套/ABCZ/品类架构 | 多主题→卡组(1主+≤3细分) |
| **S05 陈列整改** | 诊断出错配 | L2执行卡 + L3货位操作卡 + L1前后对比成果卡 | 错配诊断/货位簇/DisplayMap | 前后对比卡=一个动作焦点(判据v2允许) |
| **S06 营销配套** | 诊断出短板 | L4营销卡(脱敏) + L2活动执行卡 | 场内营销体系/四大战术 | L4须过授权链 |
| **S07 复盘** | 阶段收口 | L1成果卡(战果) + L0作战卡(下阶段) | S07复盘/指标字典 | 老板卡可复用S04诊断卡(不重造) |
| **S08 案例对外** | 战果成熟 | L4营销卡(脱敏案例·竖幅hero) | 案例讲述/授权链 | 真名须✅授权,否则"社区生鲜样板店" |

## 分解判据（速记）
`一卡 = 一受众语言 × 一动作焦点`。同内容多受众→受众轴拆(L0/L1/L2)；同内容多岗位/多时刻复用→不拆复用；多主题→卡组；>10行清单不上卡配xlsx。
