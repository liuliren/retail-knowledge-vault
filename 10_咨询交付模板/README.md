# 10_咨询交付模板

## 1. 目录定位

本目录是**晟果新零售咨询 / 六哥零售**对外交付动作的**模板区**——沉淀「可被任何一次客户战役复用」的标准件，是 vault 四象限治理中**外延复制（象限 III）** 的主要落地容器之一。

定位口径：
- **模板 ≠ 项目过程材料**。本目录只放"下一个客户也能直接拿去用"的稳定件。
- **模板 ≠ 方法论本体**。方法论本体在 `01_科学零售方法论/`，本目录是把方法论"工具化"。
- **模板 ≠ 战役档案**。具体战役的章程 / 进展 / 风险 / 决策落 `16_客户与战役档案/<战役名>/`。

## 3. 适合放什么

- 销售打法模板（开局话术 / 报价 / 三层打法）
- 交付 SOP（数据清洗规范 / 拍摄规范 / 访谈表 / 入库标准 / 质控规范 / 首访手册）
- 战役治理 SOP（G01 Ingest / G02 Query / G03 Lint / G04 章程 / G05 阶段门 / G06 AAR / 等）
- 客户报告骨架（诊断报告 / 整改报告 / 经营月报 / 老板汇报 PPT 大纲）
- 老板看板骨架与执行凭证模板
- 30 天商品提效 / 90 天陪跑等标准项目套件

## 4. 不适合放什么

- ❌ 单个客户的过程材料、访谈原始记录、原始照片、原始 POS 数据 → 走 `90_素材暂存与待整理/` 或 `09_<门店>/99_原始素材/`
- ❌ 具体战役的章程 / 里程碑 / 风险 / 决策 / AAR → 走 `16_客户与战役档案/<战役名>/`
- ❌ 具体门店的诊断结论 / 整改清单 / 复盘 → 走 `09_<门店>/`
- ❌ 方法论主定义 → 走 `01_科学零售方法论/`
- ❌ 内部数据脚本与算法 → 走 `13_数据分析与工具脚本/`

## 5. 准入规则

模板进入本目录必须满足：

1. **可复用**：至少能服务"下一个不同客户 / 不同门店"的同类场景。
2. **可教学**：店长 / 顾问 / 新人能照着执行，不需要原作者口传。
3. **有版本号**：遵循 [[文件命名规范]] v0.1 / v0.5 / v1.0 / v1.1 / v2.0 阶梯。
4. **有客户验证证据**：v1.0 及以上的模板必须能追溯到至少一次真实客户场景验证（参 [[CLAUDE.md]] §21 24h 硬约束）。
5. **frontmatter 完整**：遵循 [[文档工程化标准]]。
