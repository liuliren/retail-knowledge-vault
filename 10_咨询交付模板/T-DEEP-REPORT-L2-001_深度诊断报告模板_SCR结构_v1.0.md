# L2 深度诊断报告模板 · SCR结构 v1.0（988产品标准）

> **用途**：988 深度报告的**标准骨架**,每个客户照此出。首个实例=[[2026-07-05_花厅坊深度诊断报告_L2_988样板_SCR结构_v1.0]]。
> **标杆依据**：McKinsey SCR + 金字塔原理([[EXT-CONSULT-BENCH-001_世界顶级咨询与零售咨询业务体系标杆_v1.0]])。**六哥 2026-07-05 认可 SCR 结构·固化。**

## 📐 五条写作铁律（L2 与 L1 薄卡的分界）
1. **结论先行(金字塔)**：第一屏=一句话结论 + 3-5 个必做(答案在前,适配老板自上而下读)。
2. **★方案占大头(60%+)**：现状(S)、问题(C)要**短**;解决方案(R)占全篇 60%+。**老板要解药不要现象复述**——L1 薄卡停在现象+问题,L2 深度报告的价值全在方案。
3. **每方案必带五件套**：`策略 → 具体怎么做(Step1/2/3) → 谁做 → 验收标准 → 预期效果`,再加排期。禁"腾现金"这类空话。
4. **证据分级**：现场B / POS推算C / 老板口头C-D;C/D级禁写精确结论(如不写"下降15%"写"老板反映转弱")。进价不准维度标C级。
5. **量化成果**：效益/效果/效率各挑指标,带 `基线→目标`,月底复盘(品类计分卡法)。

```

2. <第二件>

## 一、S·事实现状（短）

|---|---|

### 方案3·<...>（同五件套）
### 排期
| 周 | 动作 |
|---|---|
| W1-W4 | <逐周动作> |

## 四、成果预期（计分卡·可见结果）
| 类 | 指标 | 基线 | 目标 |
|---|---|---|---|
| 效益 | <毛利/损耗/现金> | | |
| 效果 | <动销/尾部/连带> | | |

## 五、待补（诚实边界）
```

## 🔗 配套（一次交付流水线）
1. **数据**：远程 POS(半张卡·D1/D2/D4) → 到店全诊([[KB-FIELD-KIT-001_现场巡店诊断工具包_v1.0]]·补D3/D5/D6/D7) → 进货单(D5)
2. **写报告**：套本模板·SCR·方案占大头
3. **派生卡**：报告摘要→L1老板卡(白话)+L2店长执行卡(走 [[mdcard]])
4. **对外版**：E层脱敏(去真名/条码·换"社区生鲜样板店")

## 分层定位（避免混用）
- **L1 免费初诊卡**：现象+红绿灯·**不给方案**(留付费墙)·引流
- **L2 本模板 988 深度报告**：SCR全·**方案占大头**·付费
- **L3 落地方案**：L2 + 排期 + SOP + KPI计分卡 + 阶段门·签约
