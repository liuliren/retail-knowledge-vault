# 多店综合诊断报告 模板定义 v0.1.1

> **本文是模板定义（字段与结构标准 + 判断基准），不是某组门店的真实诊断报告。** 实例化方法见 §0。
> **定位**：全店经营诊断卡（[[全店经营诊断卡_模板定义_v0.1]]，单店一张卡）的「上一级聚合层」——单店卡看一家店五模块全景，本报告把 **N 家店的单店卡横向焊成一张全景地图**，回答「先打哪家店、哪家是样板、跨店怎么调资源」。
> **复用铁律**：本报告**不重新诊断单店**——每店数据一律来自该店已出的 T-WHOLE-DX-001 全店诊断卡（或同口径单店诊断）。本报告只做**横向聚合、对标、排序、资源分配**，不在店内做 SKU/品类级裁决（那是单店卡和单品类卡的活）。
> **双用途铁律**：本报告既是**对客户的交付件**（场景A给集团老板 / 场景B给单一客户看位次），也是**对外能力展示的案例文件**（给下一个潜在客户看「六哥能做多店对标」）。两种用途的脱敏边界不同，见 §0.5 隐私条款——**用错边界即客户机密事故**。
> **状态**：draft（v0.1.1）。本模板含「标准/铁律/红线」字样，按全局宪法《标准审议铁律》须经≥3轮多角色论证方可升 stable；当前为草案，不得被其他 Agent 引用为已生效红线。**场景B（跨客户对标）的隐私规则未经六哥签字前，禁止实际对外交付。**

## §0 使用说明（顾问操作手册·实例化时整节删除）

> *【填写说明】本节不填入门店数据，是顾问操作手册。实例化交付给客户时整节删除。*

### 0.1 何时用（触发条件）

- **场景A（同客户多店）**：一个集团/连锁老板手上有 ≥2 家店，需要一张全景地图决定「钱和人先投哪家」。触发词：六哥说「多店诊断 / 多店对标 / 集团全景 / 哪家店最该投 / 几家店一起看」。
- **场景B（跨客户对标）**：单一客户（1家或几家店）想知道「我的店在同类门店里处于什么位置」，用其他客户（脱敏）做对标池。触发词：六哥说「对标 / 行业位次 / 我在同行里排第几 / benchmark」。

  > **⚠️ 场景B启用前置门（不过禁止向客户承诺场景B）**：使用场景B前必须先填写此核查项。当前同口径单店卡池共 ___ 家不同客户（须≥5家，且任一分位带≥2家）。若不满足，场景B整段替换为：「对标池积累中（当前 ___ 家/5），待满足k≥5后启用，预计 ___ 年 ___ 季度」，并删除§5.B全文。**禁止在对标池未满足k≥5时对客户做出场景B承诺——这不是技术问题而是产品能力边界，虚开支票将在交付时兑不出。**

- **不用本报告的场景**：单店全店体检 → 用 [[全店经营诊断卡_模板定义_v0.1]]；单品类货架 → 用 `/diagnose` 单品类诊断卡；陌生店 5 分钟快判 → 用 T-08。本报告**只在已有 ≥2 张同口径单店卡**时才启动（无单店卡 = 无聚合原料）。

### 0.2 谁来填

- **顾问（六哥/Agent辅助）** 填 §2 共性根因、§3 样板提炼、§5 优先级裁决、§6 资源调配——这些是 90/10 的 10%，跨店资源裁决与对赌式排序**必须六哥本人签字**。
- **Agent** 填 §1 一览表（从各店单店卡 §3.3/§4 机械汇总）、§4 各店摘要初稿、标准化指标换算。
- **集团老板** 提供 §6 资源约束（可投入的钱/人/时间总盘子），顾问据此做分配建议。

**预估工时参考（顾问报价与客户预期管理用）**：

| 阶段 | 执行方 | 预估工时（4家店）|
|---|---|---|
| §1/§4 初稿汇总（从各店单店卡机械取数）| Agent | 2-3 小时 |
| §2 根因分析/§3 样板提炼/§5.A 矩阵/§6 调配方案 | 顾问（六哥）| 3-4 小时 |
| 合计 | — | ~1 个工作日（不含等待客户补面积/补数据时间）|

> *注：店数每增加2家，顾问阶段增加约1小时。若单店卡数据不完整（红/黄），补数据等待时间不含在内。*

### 0.3 多久更新

- 一个诊断周期出**一张**（对齐各店单店卡节奏）；跨店调改执行后重出做前后对比。
- **同一报告内所有店的单店卡口径必须一致**（六大类口径、空销率N值、数据时段、证据等级体系）——否则横向对比失真。**口径不一致的店不得进同一张一览表**（§1 铁律）。

### 0.4 与单店诊断件的关系（聚合，不重叠）

| 诊断件 | 对象 | 与本报告关系 |
|---|---|---|
| T-08 五分钟卡 | 陌生店第一印象 | 本报告的「入池前哨」，不进多店交付 |
| 单品类诊断卡 `/diagnose` | 一个品类货架 | 单店卡的下钻；本报告不直接调用 |
| 全店经营诊断卡 T-WHOLE-DX-001 | **单店**六大类×五模块 | **本报告的唯一数据原料**——每店一张，本报告聚合其 §3.3 校准后总分 + §4 定量层 |
| **多店综合诊断报告（本报告）** | **N 家店横向** | **顶层聚合，向下复用单店卡，向上给集团/对标决策** |

> *【填写说明·张力】本报告是「全景指挥图的指挥图」。错误用法：拿本报告去某家店货架上做调整——那要回到该店的单店卡。本报告只回答「先打哪家店、哪家学哪家、资源怎么分」，不回答「这家店货架砍哪几款」。若用本报告做店内 SKU 级裁决，必然因颗粒度不足而误杀。*

### 0.5 隐私条款（铁律·两场景边界不同·实例化保留本节核心）

> *【填写说明·客户机密铁律】本报告含多店数据，是全库脱敏风险最高的交付件之一。两种用途的脱敏边界**完全不同**，用错即事故。本条款核心规则须在交付件 §0 对客户明示。*

**通用规则（两场景都适用）**：

1. **全员代号化**：所有门店一律脱敏代号（如「H店」「Y店」），真名一律不进云处理层。真名进入对外层（E层·client_shareable）前，须查 [[客户名称对外使用授权链清单]]（按代号 key）；✅ 且场景覆盖才可点名，否则脱敏泛化（fail-safe，查不到 = 最严格 = 不暴露）。
2. **反脱敏钥匙隔离**：代号↔真名映射表（`David-Liu-Vault/80_个人资料_敏感/客户代号映射表_DO_NOT_INGEST.md`）严禁上云、严禁进本报告任何角落。
3. **原始数据不入正文**：进价、条码、供应商、SKU 明细一律不进本报告（沿用 §6 C 层 client_confidential）。

**场景A 专属规则（同客户多店）**：

- 同一集团老板对自己旗下各店有知情权 → 各店之间**可见绝对值**（A店老板可看 B店周转天数），但仍用代号（防报告外泄后定位到具体门店）。
- 交付对象唯一 = 该集团决策层；**不得**把 A 集团的多店报告给 B 集团看。

**场景B 专属规则（跨客户对标·最严格）**：

| 规则 | 内容 | 违反后果 |
|---|---|---|
| **本店可见，他店只给位次** | 交付给客户 X 的报告中，**只有 X 自己的店出现绝对值**；对标池里其他客户的店**只以百分位/四分位分布出现，绝对数值一律不显示、不命名、不可反推** | 暴露他客户经营数据 = 客户机密事故 |
| **k-匿名阈值（铁律）** | 对标池有效样本须 **≥5 家不同客户的店**，且任一分位区间内 ≥2 家，方可展示分布；池 <5 家 → 禁止展示分布，只能给「样本不足，仅作定性参照」 | 池太小可反推单店真值 |
| **去再识别** | 不得同时披露「面积+地段+业态」三要素组合（组合可定位到唯一门店）；对标池只披露聚合后的分位带 | 组合披露 = 变相点名 |
| **入池授权** | 进入对标池的他客户门店数据，须经该客户授权或已充分聚合到不可还原；查不到授权 = 不入池 | 未授权使用客户数据 |
| **当前池状态（须填入每份交付件）** | 当前池共 ___ 家不同客户，预计满足k≥5时间：___ 年 ___ 季度 | 防止在池未满足时对客户承诺场景B |

> *【填写说明·张力·必读】场景B 的诱惑是「让客户 X 看到具体某家强店的数字更有说服力」——**这是红线，禁止**。客户 X 只能看到「您的周转天数 38 天，处于同类门店后 25% 分位」，**永远看不到**「那家 28 天的店是谁、具体多少」。若客户追问「那家最好的店多少天」→ 标准答复：「按对客户的保密承诺，我们只提供分位，不提供他店绝对值——这也是您的店在别人报告里受到的同等保护。」*

### 0.6 定义约定（全报告统一·填写前必读）

| 术语 | 定义 |
|---|---|
| **综合健康指数** | 直接取该店 T-WHOLE-DX-001 §3.4 的「校准后总分」（0-100）。**注意可比性陷阱见 §1 张力**：各店权重按各自生鲜占比 f 校准，故该指数衡量「每家店相对其自身最优结构的健康度」，不是「绝对谁更好」。**综合健康指数仅用于各店自比（§4 撕页），禁止进入跨店排序轴（见 §5.A 修正）** |
| **标准化指标** | 已除以规模分母、可跨不同面积门店直接比较的指标：坪效（元/㎡·月）、生鲜占比（%）、综合周转天数、空销率（%）、综合毛利率（%）、客单价（元）。**绝对销售额、绝对SKU数不是标准化指标，禁止直接横比** |
| **健康灯（文字版·不依赖颜色）** | 【优】=健康达标 / 【警】=警示需关注 / 【危】=危险须立即处理。打印/黑白/盲文友好，每格须带文字标签，不靠颜色单独表意 |
| **百分位/分位（场景B）** | 本店指标在对标池中的排名位置（如「后25%分位」=比75%的同类店差）。只给分位，不给他店绝对值 |
| **证据等级** | A=POS系统精确导出；B=顾问现场直接核查；C=老板/店长估算；D=顾问主观推断（对齐《诊断输出质量控制规范》及单店卡，全报告 A-D 四级不混用） |
| **口径一致性** | 进同一张一览表的所有店，六大类口径/空销率N值/数据时段长度/证据体系须一致；不一致的店标「口径待对齐」，不进横比表 |

# ━━━━━━ 执行摘要（一页纸·给忙碌的集团老板·结论先行）━━━━━━

> ⚠️ **模板状态提示（当前为5D模式）**：D6价格力/D7大湾区本地力（裁决项①）尚未签字激活，本报告全组对比基于D1-D5；D6/D7字段已灰化标注「待激活」，不参与雷达图横比。裁决项①签字并单店卡v0.2补D6/D7评分后自动升级为7D模式，届时删除本提示。

> *【填写说明·铁律】本页 ≤1 页，金字塔原理结论先行。集团老板 30 秒读懂「先投哪家、学哪家、做什么」。每个结论必须有 §1-§6 数据支撑，禁止「整体有待提升」类空话。预期改善只给方向+量级区间，不给承诺数字（质控规范红线④）。*

**一句话全景**：本次诊断 ___ 家门店（场景___），整体 ___（如「3家盈利2家失血、生鲜是共同强项、库存是共同病灶」），最该优先投入的是 **___店**，最值得全组学习的样板是 **___店**。

| 维度 | 结论（填一句话+关键数字）|
|---|---|
| **最需投入** | ___店（标准化指标：___触红线___项，最多；核心失血点 ___，引 §1/§5）|
| **最佳样板** | ___店（在 ___维度 得分 ___，全组最高；可复制做法 ___，引 §3）|
| **共性病灶 TOP1** | ___（≥___家店共有，根因 ___，方法论建议 ___，引 §2）|
| **资源调配建议（场景A）** | ___（如「样板店 Y 的生鲜采购长替 H 店带教 30 天」，引 §6；场景B填 N/A）|
| **本店位次（场景B）** | ___（如「您的店综合处于同类门店中位，生鲜领先、周转落后」，引 §5；场景A填 N/A）|
| **下一步（30天）** | 共同动作 ___ + 各店差异化首动作见 §7 |

> *【张力】若本页出现「全面提升经营水平」等无法对应到具体店、具体维度、具体动作的表述 → 本摘要作废重写。判据：找不到「哪家店→哪个维度→下一个动作」的链路，优先级最低，不进执行摘要。*

## §1 全景一览表（多店横向·一张表看全局）

> *【填写说明·本报告核心】列=各店代号，行=7D维度+综合健康指数+标准化指标。每格填「数值 + 健康灯」。健康灯用文字版（【优】/【警】/【危】），不依赖颜色。数据全部来自各店 T-WHOLE-DX-001 单店卡，本表只搬运+对标，不重新打分。*

### 1.1 七维度对标表（7D = 单店卡五模块拆解，映射见 1.4）

> *【7D 维度定义】D1生鲜运营力 / D2商品结构力 / D3陈列货位力 / D4客流运营力 / D5财务健康力 / D6价格力（待激活）/ D7大湾区本地力（待激活）。每维度填该店在此维度的「得分率%」（来自单店卡 §3.2 Step 0），后接健康灯。D6/D7 在裁决项①签字前灰化，不填数据。*

## §9 待六哥裁决（10%不可代签·模板治理·非门店字段）

> *【模板治理】本模板 v0.1.1 含「铁律/红线/标准」字样，按《标准审议铁律》尚未完成三轮论证，升 stable 前须六哥逐项签字。**当前任何引用本报告的 Agent 必须同时声明「依据为 draft v0.1.1，以下裁决项尚未确定」。场景B 隐私规则未签字前禁止实际对外交付。***

### 待裁决项①：7D 维度定义与 5模块映射锁定（⚠️ 第一优先级）

**六哥签字状态**：⬜ 待签字

### 待裁决项②：场景B 隐私规则与 k-匿名阈值（⚠️ 客户机密红线·升优）

**问题**：§0.5 设定 k≥5 家不同客户、任一分位带≥2 家为展示分布的阈值，属 inferred；去再识别规则（面积+地段+业态不同时披露）亦为顾问设定。**场景B在满足k≥5前为产品能力空白，已在§0.1增加显式启用前置门，禁止顾问向客户虚开承诺。**

**裁决选项**：① 采纳 k≥5 + 分位带≥2；② 调整阈值（六哥定 k 值）；③ 场景B 暂不上线，仅保留场景A，待客户池积累到足够规模再开（推荐，与当前实际相符）。
**六哥签字状态**：⬜ 待签字（**未签字前场景B禁止对外交付**）

### 待裁决项③：综合健康指数跨店可比性处置 + 健康灯阈值

**问题**：§1 张力指出各店综合健康指数按各自 f 校准，绝对横比有偏；健康灯 75/60 阈值为 inferred。v0.1.1已将综合健康指数从§5.A纵轴移除，仅限各店自比，本裁决项聚焦阈值验证。

**裁决选项**：① 采纳「综合健康指数仅自比、绝对排名用标准化指标」铁律（已采纳）+ 阈值待≥3店验证；② 设计一个跨店可比的统一权重指数（须新方法论，成本高）。
