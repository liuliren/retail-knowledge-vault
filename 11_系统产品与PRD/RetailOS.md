# RetailOS · 门店决策操作系统

> **结论先行**：RetailOS = 把科学零售体系的 L2 骨架 + L3 方法武器**封装成店员可照做、可打印、不依赖理论的门店决策 SOP** 的"可执行 / 可售卖"出口。它**不是新理论层**，是把现有层"压成动作句"的操作系统外壳；**只引用骨架与武器，不重定义**（术语表口径：不替代 M1-M8 / M-DEC）。
>
> 本页为 `[[RetailOS]]` 裸链的统一落点（概念主页 + 产品文档导航），**主定义正文在 [[00_科学零售知识体系总纲]] §6.5**，本页不另起炉灶。

## 1. 定义与定位（引用，不重定义）

- **主定义**：[[00_科学零售知识体系总纲]] §6.5 —— A-G 六层架构（v1.0 减法层：A库存→B SKU结构→C价格带→D陈列→E ABC裁决→F误判→G门店SOP），减法 / 加法双向。
- **体系位置**：纵向贯穿 L2-L4 的**封装层**；铁律 = 养料喂骨架、永不反客为主。
- **90/10 定位**：交付给**客户**的旗舰 App（马具 / 90%），跑在佐罗 Zero（个人 OS / 驭马人 10%）之上 —— 边界见 [[佐罗Zero宪章_ZERO-CHARTER-001_v0.2]] §4：交到客户手里 → RetailOS；发给公众 → 媒体程序；运行六哥自己 → Zero。
- **对外口径**：internal 叫 RetailOS；对客户交付场景按术语表（[[晟果科学零售术语口径统一表_v1.1]]）核验用名。
