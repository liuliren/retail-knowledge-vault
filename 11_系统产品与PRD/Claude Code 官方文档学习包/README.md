# Claude Code 官方文档学习包 README

## 1. 包定位

本目录是「晟果新零售咨询 / 六哥零售知识库」的 **Claude Code 操作手册中心**。它把 Claude Code 官方文档（`https://code.claude.com/docs/`）中与本知识库工作流强相关的机制，沉淀为可检索、可引用、可复用的内部参考。

服务三件事：
1. **三个月生存战的 token / 额度可控**：让每次会话有纪律。
2. **知识库长期工程化**：让 CLAUDE.md、`.claude/rules/`、subagent、skill、hooks、permissions 形成体系。
3. **复盘机制**：把 transcript 与 auto memory 转化为 `15_刻意练习与成长/` 的客观素材。

## 2. 入库来源

本包整体派生自暂存预览：

- 来源种子：`90_素材暂存与待整理/2026-05-04_Claude_Code官方文档学习包预览_v0.1.md`
- 抓取日期：2026-05-04
- 抓取页面：`llms.txt` 索引 + memory / context-window / commands / sub-agents / hooks / statusline / permissions / costs / claude-directory（共 10 页官方文档）
- 待核验状态：12 项待核验事项随同入库，未做静默消化（详见 [[10_待核验问题清单_v0.1]]）

## 4. 维护规则

1. 本包文件的 `status` 默认从 `review` 起。**待核验事项消化后**才能升 `validated`。
2. 待核验项的消化必须更新 [[10_待核验问题清单_v0.1]]，并在被引用文件中同步删除「待核验」标记。
3. Claude Code 官方文档持续迭代，**任一文件每 90 天必须做一次回查**（重点：commands、costs、hooks 三页）。
4. 不可在本包文件中编造未在官方文档出现的机制；新机制先入 [[10_待核验问题清单_v0.1]]，验证后再写入对应主题文件。
5. 重大版本升级（v1.0、v2.0）需在 [[版本更新记录]] 与 [[知识库总索引]] 同步登记。
6. `99_原始素材/` 仅放选摘片段、URL 与抓取日期，**不放官方文档原文全量**（避免版权与体量问题，以官方原页为准）。
