# 权限与安全红线

## 1. 文件定位

把 Claude Code 的权限模型、规则语法、模式选择讲清楚，并给出本 vault 的红线。**配置变更需单独授权**，本文件本身不实施配置。

## 2. 适用场景

- 配 `.claude/settings.json` / `~/.claude/settings.json` 之前
- 评估某条建议规则的副作用
- 排查"为什么 Claude 老在问我同一件事 / 没问就动手了"
- 审计第三方 / AI 生成的权限配置

## 3. 核心机制

### 3.1 评估顺序

**deny → ask → allow**（任何层 deny 一票否决）

设置层级（高到低）：管理策略 → 命令行参数 → 本地项目 (`.claude/settings.local.json`) → 项目共享 (`.claude/settings.json`) → 用户 (`~/.claude/settings.json`)

### 3.2 模式速选

| 模式 | 用途 |
|---|---|
| `default` | 首次使用提示（默认） |
| `plan` | 只读分析。**复盘 / 规划 / 扫库优先** |
| `acceptEdits` | 自动接受工作区内 file edit 与常见文件命令 |
| `auto` | 实验性背景安全分类器自动批 |
| `dontAsk` | 未预批一律拒（适合高度受控 SOP） |
| `bypassPermissions` | 全部跳过；仅在容器 / VM 用；`rm -rf /` 仍熔断 |

### 3.3 规则语法（要点）

- `Tool` / `Tool(specifier)`
- Bash 通配 `*` 任意位置；`Bash(ls *)` 与 `Bash(ls*)` 不同（前者强制词边界）
- `find -exec/-delete` / `xargs -flag` / `watch` / `setsid` 这类**不会**被 `Bash(find *)` 等覆盖
- `Read(./.env)` / `Edit(/src/**/*.ts)`（gitignore 风格；`/` 是项目根，**不是文件系统根**；要绝对路径用 `//`）
- `Bash` 内置只读集合（`ls / cat / grep / find / wc / diff / stat / du / cd` + 只读 git）默认免确认
- Read/Edit 规则**只对内置工具**生效，**不拦 Bash 子进程的 `cat .env`**；要 OS 级用 sandbox

### 3.4 工具粒度

- `Bash(npm run build)` 精确匹配
- `Bash(npm run *)` 前缀匹配
- `WebFetch(domain:example.com)` 域名约束
- `Read(./.env)` / `Edit(/src/**/*.ts)` 路径约束（gitignore 风格）
- `Agent(Explore)` 限子代理
- `mcp__server__tool` 限 MCP 工具

### 3.5 与 hook 的交互

Hook 决策**不绕过** deny / ask 规则：
- deny rule 永远先于 hook allow 生效
- ask rule 即使 hook 返回 allow 也仍会提示
- hook exit 2（阻断）会先于 allow rule 生效

## 4. 本 vault 红线（建议方向，不实施）

### 4.1 deny（硬阻断）

```
Edit(/99_归档/**)              防误改归档
Edit(./CLAUDE.md)              防误改主治理规则（必须人工编辑）
Edit(/00_入口与总索引/**)      防误改治理文件
Bash(git push *)               防误推
Bash(rm -rf *)                 不允许递归删（个别场景需手动确认）
```

### 4.2 ask（提示后放行）

```
Bash(rm *)                     一般删除
Bash(mv */99_*)                进归档操作
Edit(/.claude/**)              改 .claude 配置
```

### 4.3 allow（免提示）

```
Read(**)                       全 vault 可读（已是默认行为）
Bash(git status)
Bash(git diff *)
Bash(git log *)
Bash(ls *)（已是内置只读）
Bash(grep *)（已是内置只读）
```

### 4.4 永远不做

- 不要试图用 `Bash(curl http://x.com/*)` 限 URL——官方明确说不可靠
- 不要开 `bypassPermissions` 模式（除非未来在容器 / VM 跑全自动化）
- 不要把 `*` 当 `**` 用（gitignore 中 `*` 不跨目录）
- 不要把 `Read/Edit` 规则当成"OS 级文件保护"——它**不拦 Bash 子进程**

## 5. 操作规则

1. 个人 vault 的权限配置写到 `~/.claude/settings.json`（用户级）；**不**用管理策略级
2. 加新 deny 规则前，先用 `/permissions` 看现有规则
3. 见到"为什么 Claude 总问同一件事"——加 allow；见到"Claude 干了不该干的"——加 deny
4. 任何"给客户机器装 Claude Code 时配 deny 模板"的需求，先按 §4.1–§4.3 起步
