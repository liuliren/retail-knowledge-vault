## §6. 续约预案 ⭐

### 6.1 推荐方案 A（基础 + 分润）

```
基础费：    8 万/年（每月 6,667）
分润：      8% × 季度结算
计算基础：  增量销售额（vs 调改前同期）
KPI 门槛：  库存周转 -20% / 销售 +3%
预期总额：  13-23 万/年（含分润）

依据：
- 5/6 老板金句 #3："销量 + 客单价提升 → 拿 10% 利润"
- 老板金句 #4：拒绝全年陪跑 / 要快速结果
- 8% 比 10% 略低 / 表诚意 + 留续约空间
```

### 6.2 备选方案 B（纯固定 / 备份）

```
基础费：    12 万/年（每月 1 万）
分润：      0
适用：      老板抗拒分润 / 倾向稳定支出

依据：
- 4 店 1.5 亿 GMV / 12 万 = 0.08% 占比 / 极低
- 适用于"已经拿到 30 天数据 + 老板信心强"场景
```

### 6.3 备选方案 C（对赌 / 上限）

```
基础费：    5 万/年（每月 4,167）
分润：      15% × 季度结算
封顶：      30 万/年
门槛：      库存周转 -30% / 销售 +5%

适用：      老板倾向激进对赌
风险：      我方收入波动 / 仅当数据有把握时
```

### 6.4 续约红线（Anthropic Harmless）

```
❌ 不接受：
   - 纯销售提升对赌（无基础费）/ 我方风险过高
   - 12 个月以上锁定 / 缺乏退出灵活性
   - 4 店之外的免费扩展（必须分别签约）
   - 数据使用权限不清晰条款（必须 client_safety: confidential）

✅ 必须有：
   - 季度数据审计 + 老板 sign off
   - 我方有权终止（如客户不配合）/ 退还未发生月份费用
   - 知识产权归我方（vault 方法论 + 6 表 + 算法包）
```

## §8. 反幻觉红线（HHH / Anthropic Honest）⭐

### 8.1 数字承诺（[[CLAUDE.md]] §13.7）

```
❌ 禁止："3 个月销售提升 30%"
✅ 允许："30 天数据后提供 ROI 报告 / 基于真实数据测算"

❌ 禁止："保证续约"
✅ 允许："基于成果驱动 / 数据说话"

❌ 禁止："1 个月做完整改"
✅ 允许："Phase 1 30 天基线 + 快赢 / Phase 2 60 天稳定"
```

### 8.2 客户原话保留（§13.1）

```
所有老板 / 店长 / 一线员工原话：
✅ 一字不改记入客户验证日志
✅ 关键金句进入 [[2026-05-06_花厅坊_老板深度访谈日志_v0.1]] §H
❌ 不允许润色 / 不允许"客户表示认同"等空话
```

### 8.3 数据真实性（§13.2-§13.4）

```
✅ 销售提升 = POS 真实数据 / 同期对比 / 区分自然 vs 陈列驱动 vs 活动驱动（V4 ④ 4 类）
✅ 库存周转 = 真实库存 / 不"按估算"
✅ 毛利率 = 进价 + 售价 真实数据 / 不"按经验"
❌ 不允许编造 / 不允许"案例参考"等模糊表述
```

### 8.4 推断 vs 事实（§13.5）

```
报告中所有结论必须分类：
✅ 事实（POS 数据 / 现场观察）
✅ 分析（基于事实的推断 / 标注"分析"）
✅ 建议（基于分析的判断 / 标注"建议"）
✅ 待确认（不确定 / 标注"待确认"）
```
