## §2. P0-2：Week 1 必测 3 大 KPI 基线（阻断式）

### 2.1 v1.0 缺漏

KPI 表中 3 大基线是 [Week 1 测量值] 占位。

### 2.2 v1.1 新增 M1.0a 阻断里程碑

```
M1.0a：3 大 KPI 基线实测（阻断式 / 5/14 前必出）
─────────────────────────────────────────
必产出物：
  ✅ 库存周转天数（4 店分别 + 集团加权）
     公式：(期末库存 + 期初库存) / 2 / 日均销售成本
     
  ✅ 集团毛利率（12 个月历史平均）
     公式：(销售额 - 销售成本) / 销售额
     
  ✅ D1 类占比（实测 vs 估算 21.8% 校准）
     公式：D1 SKU 数 / 总 SKU 数

阻断规则：
  3 大基线未出 → 不允许进入 M2 阶段
  3 大基线出 → 凤凰计划 v1.1 → v1.2（更新真实数据 / 去 disclaimer）

签字确认：
  ✅ 老板签字
  ✅ 采购总监签字
  ✅ 4 店店长签字（数据真实性）
```

## §3. P1-6：4 店启动顺序保守化

### 3.1 v1.0 原计划（激进版）

```
Week 1-2：4 店并行采集
Week 5-6：4 店并行执行第一批快赢
```

### 3.2 v1.1 修订（保守版）

```
✅ Week 1-2 基线采集：4 店并行（轻量 / OK）
🔄 Week 5-6 第一波执行：仅花厅坊 + 1 店（不是 4 店）
🔄 Week 7-8 复制 1 店：基于花厅坊跑通经验
🔄 Week 9-10 复制其他 2 店

理由：
- 单兵带宽不够（45% 精力 + 主线 2/3/4 平行）
- 一线员工初中学历（金句 #5 / R17 风险）
- 失败风险高 → 必须先标杆店成功
- 花厅坊 5/7 已 lock 6 表 v1.0 → 直接启动
```
