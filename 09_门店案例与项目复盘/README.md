# 09_门店案例与项目复盘

## 目录作用

本目录沉淀**真实门店项目的全过程记录和项目复盘**，是知识库中最有价值的资产之一——因为它包含了方法论在真实场景中的验证结果。

每个门店独立建子目录，按项目阶段组织资料，确保可追溯、可复用、可交付。

---

## 适合存放

**每个门店子目录，标准结构如下：**

```
门店名称/
├── 00_项目总览/       项目基本信息、阶段进展、关键人
├── 01_商圈与客群/     商圈调研结论、竞品分析、客群画像
├── 02_生意诊断/       门店整体经营判断
├── 03_商品诊断/       SKU分析、ABCD分类结果
├── 04_品类规划/       品类整改方案
├── 05_陈列整改/       DisplayMap、现场图片说明
├── 06_营销活动/       活动方案与复盘
├── 07_经营复盘/       经营数据分析、阶段复盘
├── 08_交付报告/       客户版正式交付文件
└── 99_原始素材/       照片、Excel、录音等原始资料
```

## 不适合存放

- 通用方法论（属于 `01_科学零售方法论`）
- 通用模板（属于 `10_咨询交付模板`）
- 未经整理的原始照片或 Excel（先进 `90_素材暂存与待整理`）

---

## 与其他目录的关系

| 目录 | 关系说明 |
|------|----------|
| `01_科学零售方法论` | 所有项目结论应能追溯到方法论依据 |
| `02` 至 `08` | 各阶段的通用方法在对应目录，结论存在此处 |
| `10_咨询交付模板` | 本目录的 `08_交付报告` 引用交付模板 |
| `12_自媒体内容与表达转化` | 项目案例可转化为内容素材 |
| `99_归档` | 项目结束后不常用的资料可归档 |
