# 花厅坊休食区陈列资源命名与照片入库规范 v1.0

## 1. 文件用途

统一花厅坊休食区**陈列资源命名**，建立**商品 ↔ 陈列资源**一一匹配关系，支撑：

- 品效分析（SKU × 陈列位 × 销售）
- DisplayMap 制作
- 调改前后同机位对比
- 现场照片入库与升级
- 全店推广（后续可复制到酒水 / 调味 / 米面粮油 / 日化）

本文件是**花厅坊休食区项目级 v1.0**，与全店级 v0.3 平行；W22 后可回写 v0.3 → v0.4 / v1.0。

---

## 2. 适用范围

**适用**：
- 花厅坊休食区
- G1-G4 货架（4 排 × A/B 双面 = 8 货架面）
- 子货架 01-06
- 层位
- 通道
- 端架
- 主堆 / 侧堆 / 异型堆
- 休食区照片命名
- 调改前后复拍
- 商品 ↔ 陈列资源绑定

**不适用**：
- 酒水区
- 调味区
- 米面粮油
- 用品日化
- 沙埔大道店
- 花厅坊店内其他区域

## 3. 陈列资源主键总公式

### 3.1 公式

```
<区域>-<资源类型><编号>-<面位>-<子货架>-<层位>-<排面位>
```

### 3.2 字段表

| 字段 | 取值 | 必填 |
|---|---|:---:|
| 区域 | `HS` = 休食区 | ✅ |
| 资源类型 | `G` 货架 / `A` 通道 / `T` 端架 / `D` 主堆 / `SD` 侧堆 / `PD` 柱旁堆 | ✅ |
| 编号 | 排号（货架）/ 序号（通道/端架/堆）| ✅ |
| 面位 | `A` / `B`（仅货架）| 货架必填 |
| 子货架 | `01`-`06`（强制 2 位）| 货架按需 |
| 层位 | `1`-`7` 数字层 | 精分析必填 |
| 排面位 | `01`-`NN`（强制 2 位）| 重点 SKU 必填 |

### 3.3 三级粒度

| 粒度 | 示例 | 用途 |
|---|---|---|
| 子货架 | `HS-G1-A-01` | 普通 SKU 绑定 / 大部分照片定位 |
| 层位 | `HS-G1-A-01-3` | 重点 SKU / 层位诊断 |
| 排面位 | `HS-G1-A-01-3-02` | 精细品效分析 / 调改 SKU / 滞销 SKU |

## 4. 货架命名规范

### 4.1 休食区货架正式编号

| 排号 | A 面 | B 面 |
|---|---|---|
| G1 | `G1-A` | `G1-B` |
| G2 | `G2-A` | `G2-B` |
| G3 | `G3-A` | `G3-B` |
| G4 | `G4-A` | `G4-B` |

### 4.2 规则

1. G1-G4 = 4 排货架（不是 G1-G6）
2. A / B = 每排两面
3. 每面默认 6 个子货架
4. 子货架从左到右编号 01-06
5. 示例：`G3-B-04` = G3 排 B 面第 4 个子货架
6. **不再使用 G1-G6 描述休食区**
7. 旧文件出现 G1-G6（v0.3 §3 line 72）→ 标注为历史口径 / W22 订正

## 5. 子货架命名规范

### 5.1 公式

```
HS-G<排号>-<面位>-<子货架编号>
```

### 5.2 示例

```
HS-G1-A-01      # G1 排 A 面第 1 子货架
HS-G3-B-04      # G3 排 B 面第 4 子货架（方便速食 04 组）
HS-G4-A-06      # G4 排 A 面第 6 子货架（冲调 06 组 / 历史误投放点）
```

### 5.3 规则

1. 子货架编号**固定 2 位**（前导零）
2. **从左到右**编号
3. ❌ 不使用 `G3_B`（下划线）
4. ❌ 不使用 `G3B`（简写）
5. ✅ 入库统一使用短横：`G3-B`

## 6. 纵向层级命名规范

### 6.1 公式

```
HS-G<排号>-<面位>-<子货架编号>-<层位>
```

### 6.2 示例

```
HS-G3-B-04-1     # G3-B-04 子货架第 1 层
HS-G3-B-04-2     # 第 2 层
HS-G3-B-04-3     # 第 3 层
```

### 6.3 层位规则

1. 层位数字 = 该子货架**实际层数**（不强行补齐）
2. 常规货架 1-7 层 / 挂钩货架 1-6 层
3. **首次建档**必须记录该子货架实际层数
4. 5 层货架不得强行补成 7 层（v0.3 §6.2 已规范）
5. **层位方向 = 从上到下**（沿用 v0.3 §6.1：**1 = 顶层 / 末层 = 底层**）

### 6.4 层位方向裁定

| 来源 | 表述 | 采用 |
|---|---|---|
| v0.3 §6.1 line 151 | "第 1 层 = 永远顶层 / 末层 = 永远底层" | ✅ 本 v1.0 沿用 |
| 5/19 实测清单 | 用"上半区/下半区"语义（未直接编号）| 不冲突 / 上半区 ≈ 1-3 层 |
| 5/13 补拍清单示例 | `04-3_茶叶_黄金层` | 不冲突 / 第 3 层 = 黄金视线 |

→ **本 v1.0 锁定：1 = 顶层 / 末 = 底层（从上到下）**。

## 7. 商业层位语义

### 7.1 语义表

| 层位语义 | 典型物理层 | 离地高度 | 商业含义 | 适合商品 |
|---|---|---|---|---|
| 顶层囤货 | 7 层货架第 1 层 | 1.8-2.8m | 纯陈列 / 库存缓冲 | 整箱 / 形象品 |
| 形象 | 5/6 层第 1 层 或 7 层第 2 层 | 1.6-2.4m | 形象 / 轻包装 | 品牌墙 / 轻抛货 |
| 黄金视线 | 第 2-3 层（眼平 ~1.6m）| 1.5-1.7m | 最高可见性 | 主推品 / 品牌品 / 引流品 |
| 黄金手拿 | 第 3-4 层（~1.2m）| 1.1-1.3m | 最高拿取效率 | 高频刚需 / 引流品 |
| 中层 / 补充 | 第 4-5 层（~0.8m）| 0.7-0.9m | 补充销售 | 常规 SKU |
| 大包装 | 倒数第 2 层（~0.4m）| 0.3-0.5m | 蹲身拿大件 | 大规格 / 低频 |
| 围货 / 底层 | 末层（~0m）| 0-0.2m | 不取货 / 纯陈列 | 大箱 / 围货 |
| 挂钩 | 挂条区 | 视高度 | 小包装 / 冲动购买 | 小包零食 / 挂装商品 |

### 7.2 规则

1. 每个实际层位**可映射**一个商业语义
2. 商业语义用于陈列诊断
3. ❌ 不可只写"第 3 层"——应尽量补"黄金视线 / 黄金手拿"等语义
4. 无法判断时标 `[待现场确认]`
5. 命名时可作**后缀**附加（v0.3 §6.3 沿用）：
   ```
   HS-G3-B-04-3-黄金视线
   ```

### 7.3 与 v0.3 §6.2 5/6/7 层完整层位表的关系

完整 5/6/7 层映射沿用 v0.3 §6.2 表 / 不重写 / 双链：[[2026-05-06_门店陈列照片命名规范_v0.3]] §6.2。

## 8. 排面位命名规范

### 8.1 公式

```
HS-G<排号>-<面位>-<子货架编号>-<层位>-<排面位>
```

### 8.2 示例

```
HS-G3-B-04-3-01    # G3-B-04 子货架第 3 层第 1 个排面位
HS-G3-B-04-3-02    # 第 2 个排面位
HS-G3-B-04-3-01-03 # 同 SKU 占 3 个排面位（01 起 03 止）
```

### 8.3 规则

1. 排面位**从左到右**编号
2. **固定 2 位**前导零
3. 1 SKU 占 1 排面 → 绑定 1 个排面位
4. 1 SKU 占多排面 → 记录起止位（如 `01-03`）
5. 排面位用于**精细品效分析** / 不要求所有普通照片标注
6. **必须绑定排面位**的场景：
   - 重点 SKU（销售 / 毛利 top）
   - 滞销 SKU
   - 端架 SKU
   - 调改 SKU

## 9. 通道命名规范

### 9.1 休食区通道编号 v1.0 草案（待 5/25 现场或 CAD 实证）

| 通道编号 | 中文名 | 物理定义 | 相邻资源 | 主要用途 |
|---|---|---|---|---|
| `HS-A01` | 休食低温侧通道 | G1-A 与低温区之间 | G1-A / 低温区 | 低温区与休食区联动 |
| `HS-A02` | 休食 G1-G2 通道 | G1 与 G2 之间 | G1-B / G2-A | G1/G2 动线 |
| `HS-A03` | 休食 G2-G3 通道 | G2 与 G3 之间 | G2-B / G3-A | G2/G3 动线 |
| `HS-A04` | 休食 G3-G4 通道 | G3 与 G4 之间 | G3-B / G4-A | **方案 Y G3-B + G4-A 联动重点** |

### 9.2 规则

1. 通道编号**以货架关系定义** / 不用口语化名称作主键
2. "水果休食通道" / "垂直主通道 01" 等旧名 → 仅作**别名**（参 [[花厅坊门店通道命名表_v0.1]]）
3. 平面图显示方向不一致时 → 写入 §22 待核实
4. 方案 Y G3-B + G4-A 联动重点用 **`HS-A04`**
5. 通道编号用于：动线照片 / 顾客视角照片 / 调改前后同机位复拍

### 9.3 反幻觉边界

- HS-A01 至 HS-A04 的**几何逻辑**（4 排货架 → 至少 3 条排间通道 + 1 条侧通道）= ✅ 成立
- HS-A01 是否真的对应 G1-A 与低温区之间 = ❓ **待 5/25 现场或 CAD 0408 实证**
- 标注 `fact_layer: inferred / pending`

## 10. 通道机位命名规范

### 10.1 公式

```
HS-A<NN>-M<机位>
```

### 10.2 机位定义

| 机位 | 含义 |
|---|---|
| `M1` | 通道入口侧（顾客刚进入通道的视角）|
| `M2` | 通道中段（通道一半深度的视角）|
| `M3` | 通道末端（通道尾部的视角）|

### 10.3 示例

```
HS-A04-M1     # G3-G4 通道入口侧
HS-A04-M2     # G3-G4 通道中段
HS-A04-M3     # G3-G4 通道末端
```

### 10.4 规则

1. 机位用于**调改前后同角度复拍**
2. 每条通道**至少**保留 M1 / M2
3. 重点通道（如 HS-A04）建议 M1 / M2 / M3 全拍
4. 机位方向必须在**现场记录**中说明
5. G3-B + G4-A 联动复拍优先用 `HS-A04-M1/M2/M3`

## 11. 端架命名规范

### 11.1 沿用 v0.3 §8 C 方案

```
<P>-T<NN>-<关联G>[首/次/异][-L品类]_<内容>_<YYYYMMDD>_<时段>.jpg
```

### 11.2 花厅坊休食区端架 ID 表（v0.3 §8.2 锁定 + 本 v1.0 加 5/8 sign off 标注）

| 端架 ID | 关联货架 | 物理位置说明 |
|---|---|---|
| `HS-T01` | G1 首 | 休食 G1 货架首端架 |
| `HS-T02` | G1 次 | 休食 G1 货架次端架 |
| `HS-T03` | G2 首 | 休食 G2 首端架（异常仅 1 个 / 无次端架）|
| `HS-T04` | G3 首 | 休食 G3 首端架 |
| `HS-T05` | G3 次 | 休食 G3 次端架 |
| **`HS-T06`** | **G4 首** | **方案 Y 引流款 / 臭宝粉 + 统一桶面 G-4 端架（5/8 sign off ②）** |

### 11.3 硬约束

- `HS-T06` **= G4 首**（不是 G4 次）
- `HS-T07` **= G4 次**（不是 G4 首）
- 出现 `HS-T07-G4首` → **命名冲突** / 必须人工确认（参 §18）
- 跨区不冲突：`HS-T01` ≠ `SX-T01` ≠ `TL-T01`

## 12. 主堆 / 异型堆命名规范

### 12.1 公式

|---|---|

HS-D01-L方便速食_全景_20260523_常规.jpg

4. 不得凭空生成全店编号

## 13. 照片命名双层机制

### 13.1 现场快采命名（速度优先）

```
花厅坊_休食区_<资源编号>_<视角>_<序号>.jpg
```

示例：
```
花厅坊_休食区_G3-B_正面全景_01.jpg
花厅坊_休食区_G4-A_价格签细节_06.jpg
花厅坊_休食区_HS-A04_通道全景_01.jpg
花厅坊_休食区_HS-T06_端架全景_01.jpg
```

特点：
- **速度优先** / 现场佳妮 / 员工 / 顾问均可
- **不要求**现场填层位 / L 品类 / 时段
- ❌ 严禁"修饰区 / 洗护"等错误词
- ✅ 必须包含资源编号

### 13.2 入库标准命名（顾问升级）

| 资源类型 | 标准公式 |
|---|---|
| 货架照 | `HS-G<排号>-<面位>-<子货架>-<层位>[-L<品类>][-<层位语义>]_<内容>_<YYYYMMDD>_<时段>[_<序号>].jpg` |
| 通道照 | `HS-A<NN>-M<机位>_<内容>_<YYYYMMDD>_<时段>[_<序号>].jpg` |
| 端架照 | `HS-T<NN>-<关联G>[首/次/异][-L<品类>]_<内容>_<YYYYMMDD>_<时段>[_<序号>].jpg` |
| 主堆照 | `HS-D<NN>-L<品类>_<内容>_<YYYYMMDD>_<时段>[_<序号>].jpg` |

示例：
```
HS-G3-B-04-3-L方便速食-黄金视线_商品价签_20260525_常规_01.jpg
HS-A04-M1_通道全景_20260525_常规_01.jpg
HS-T06-G4首-L方便速食_调改后_20260525_夜灯后_01.jpg
HS-D01-L方便速食_全景_20260525_常规_01.jpg
```

### 13.3 双层机制硬约束

1. 现场快采名**可保留**在 `99_原始素材/`
2. 入库标准名用于**正式分析目录**

## 19. 入库操作流程（5 步 SOP）

### 19.1 流程图

```
[原始照片] → [初步 audit] → [生成 rename map] → [用户确认] → [正式入库升级]
   ↓             ↓              ↓                ↓               ↓
原文件名     统计 + 冲突      原名 → 标名       sign off       不覆盖原始
保留        分类           资源编号         冲突端架          复制 / 重命名
```

### 19.2 5 步详解

| 步 | 动作 | 输出 |
|:---:|---|---|
| 1 | 原始照片保存 | 保留原文件名 / 放入 `99_原始素材/04_现场照片/<日期>/` |
| 2 | 初步 audit | 统计张数 / 按货架/通道/端架分类 / 标命名冲突 |
| 3 | 生成 rename map | 原文件名 / 推荐标准名 / 资源编号 / 是否需人工确认 / 备注 |
| 4 | 用户确认 | 涉及冲突 / 端架 / 通道 / A/B 面 必须用户确认 |
| 5 | 正式入库升级 | 复制或重命名到正式照片目录 / **不直接覆盖原始素材** / 保留 rename map 作 audit trail |
