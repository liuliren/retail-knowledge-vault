> 起草：Claude Code · 2026-06-30 · status: draft（待六哥签字升 stable）

## 0. 系统一句话定义

> **每一平方米的空间都是成本，每一个陈列位都是收入机会。** 本系统把花厅坊的货架/堆头/端架从"感觉摆放"变成"数据经营"：盘点空间存量 → 绑定商品 → 追踪产出 → 优化配置 → 跟进调整。

**核心飞轮**：
```
空间盘点 → 商品绑定 → 产出追踪 → 坪效计算 → 调整决策 → 效果跟进
    ↑___________________________________________________|
```

## 1. 第一性原理推导（为什么这么设计）

### 1.1 空间的本质

```
空间 = 资产（需要回报）
       ≠ 背景（随意摆放）

成本端：租金 ÷ 净卖场面积 = 每平米每年租金成本
        → 花厅坊推算：总面积约 5550㎡，净卖场约 3000–3500㎡
        → 若月租 10万，年租 120万 → 每㎡/年租金成本约 343–400元

收入端：每㎡年销售额 × 综合毛利率 = 每㎡年毛利贡献
        → 社区超市健康坪效：¥8000–15000/㎡/年销售额
        → 生鲜区坪效通常高于非食品区
```

### 1.2 陈列位金字塔（价值分层）

```
价值最高
   ▲  堆头（Display Island）
   │  → 主通道出入口、收银前区、品类交界处
   │  → 销售提升：3–7倍（vs 同品在货架）
   │
   │  端架（End Cap）
   │  → 货架组的末端，双面可视
   │  → 销售提升：2–4倍
   │
   │  黄金货架区（Eye Level Zone）
   │  → 货架 80–150cm 高度，正面向通道
   │  → 销售提升：1.5–2.5倍 vs 底层/顶层
   │
   │  普通货架区（Regular Shelf）
   │  → 标准陈列位
   │
▼  死角区（Dead Zone）
   → 底层（< 50cm）、顶层（> 180cm）、背通道
价值最低
```

### 1.3 管理抓手：四类空间资产

| 空间类型 | 花厅坊现状（基于CAD） | 管理重点 |
|---|---|---|
| **货架** | 191节（47+56+38+50节+零散） | 品类归属、层位分配、SKU 密度 |
| **端架** | 每组货架末端，估计 20–30 个 | 供应商资源位、季节性品类 |
| **堆头** | 主通道节点，数量待现场盘 | 租金台账、促销效果追踪 |
| **冷链陈列** | 风幕柜/冰柜/玻璃门冷柜等 | 品类-柜型匹配、能效与销效对比 |

## 4. Module C：坪效计算体系

### C1. 三层坪效指标

```
L1 全店坪效：总月销售额 ÷ 净卖场面积
             → 健康标准：¥667–1250/㎡/月（年化 8000–15000）
             → 花厅坊目标：待六哥提供销售数据后填入

L2 品类坪效：品类月销售额 ÷ 品类占地面积
             → 用于跨品类比较，发现低效区域

L3 货位坪效：单货位月销售额 ÷ 货位面积
             → 用于货架内层位优化
```

### C2. 坪效数据采集流程

```
步骤1：确定净卖场面积（排除后台/通道/收银台）
  → 来源：CAD 空间地图（当前为推算值 ~3000㎡，需精算）
  → 行动：用 LINE 实体提取边界（tools/dxf_parser.py 下一版本功能）

步骤2：采集品类销售数据（来源：POS 系统/收银台报表）
  → 按品类分类统计月销售额
  → 频率：月度（首月建立基线）

步骤3：品类面积对应
  → 用货架盘点台账：节数×层数×节宽×层深 = 陈列面积
  → 堆头/端架单独统计

步骤4：计算 & 排行
  → 输出品类坪效排行榜（高→低）
  → 标记低效品类（坪效<行业均值50%）
```

## 5. Module D：堆头/端架盈利管理

### D1. 堆头 ROI 计算公式

```
堆头 ROI = (促销期增量销售额 × 综合毛利率 - 额外人工/物料成本) 
           ÷ 堆头机会成本
           
堆头机会成本 = 该位置改做其他用途的坪效 × 占地面积 × 促销期天数

实操简化版（供应商付费堆头）：
  ROI = (供应商堆头费 + 增量毛利) ÷ 场地成本
  场地成本 = 该位置坪效 × 面积 × 天数

判断标准：ROI > 1.5 = 值得 | 1.0–1.5 = 打平 | < 1.0 = 亏损
```

### D2. 堆头效果追踪（标准流程）

```
T-7天  布堆前：记录该SKU过去30天在货架上的日均销量（基线）
T+0天  布堆日：拍照存档（含产品/价格/位置/日期）
T+7天  一周后：统计堆头期日均销量 vs 基线
T+14天 两周后：再次统计（确认是否持续高位还是回落）
T+0'天 收堆日：拍照存档，计算总促销效果
T+7'天 收堆后7天：统计货架销量是否恢复基线（避免提前透支）

核心指标：
  销售提升率 = (堆头期日均销量 - 基线日均销量) ÷ 基线日均销量
  增量毛利 = (堆头期总销量 - 预期货架销量) × 综合毛利
  盈亏平衡销量 = 堆头成本 ÷ 综合毛利率 ÷ 促销天数（需每天卖多少）
```

### D3. 堆头效果评估矩阵

| 效果等级 | 销售提升率 | 判断 | 下次行动 |
|---|---|---|---|
| 爆款堆 | >5倍 | 超预期 | 复制至更多堆头位 |
| 优质堆 | 3–5倍 | 达标 | 续档/同品类深挖 |
| 合格堆 | 1.5–3倍 | 可接受 | 优化陈列/配合活动 |
| 低效堆 | <1.5倍 | 不合格 | 换品类/换供应商 |
| 负效堆 | <1倍（降销） | 反效果 | 立即撤换，分析原因 |

### D4. 端架管理规则

```
端架分配优先级（高→低）：
1. 当季主推商品（节假日/季节性）
2. 高毛利商品（毛利>35%）
3. 新品导入（帮助新品快速破零）
4. 供应商付费资源位（有费用支撑）
5. 清库存（低优先）

端架轮换频率：
  旺季（节前1个月）：每2周换一次
  平季：每月换一次
  滞销品：立即换（不等月度轮换）
```

### E2. 调整决策触发条件

```
必须立刻调整（立刻行动）：
  - 某货架品类月坪效 < 全店均值 30%（持续2月）
  - 某 SKU 连续30天动销<1件/天（死货）
  - 供应商资源到期且无续档意愿
  - 节假日前7天（主动布置节令陈列）

建议调整（下次月度例行）：
  - 品类坪效排名末20%
  - 端架销售提升<1.5倍
  - 新品导入2周后无动销

禁止随意调整：
  - 刚调整完不满14天（无数据基础）
  - 旺季前15天内（稳定期）
```

```
目标：把 POS 销售数据与空间台账关联

  ① 获取花厅坊 POS 系统品类月报（六哥向店主/运营确认数据权限）

```
