## 8. 系统导出表 → 自动汇总表 推荐流程(三档)

> **核心卡点(本轮新发现)**:POS 导出的 `.xls` 是 OLE2 老格式,Python 直读困难。**任何自动化的第一步必须是"转成 .xlsx/CSV"**。

### A. 手工极简版(现在马上能用)
```
1 每周从POS导出固定几张表(见§9 P0)
2 ★关键:用 Excel/Numbers 打开 → 另存为 .xlsx(把OLE2老.xls转新格式)
3 放入固定目录:09_.../01_原始数据/raw_exports/YYYY-MM-DD/
4 按§10 命名规范命名
5 Claude/脚本读取 .xlsx → 出汇总 Markdown
```
- 人工要做:导出 + 另存xlsx + 改名 + 放目录(约10分钟/周)。
- 可自动:读取/字段映射/汇总/缺口报告。

### B. 半自动脚本版(下阶段)
```
目录:13_数据分析与工具脚本/花厅坊_POS导出自动汇总/
  ├─ raw_exports/        # 每周拖入(git-ignore)
  ├─ field_map.yaml      # 原始字段→标准字段 映射表
  ├─ summarize.py        # 入口:读raw→标准化→出汇总
  └─ outputs/            # 汇总.md/.xlsx(git-ignore明细,只留脱敏汇总md)
脚本流程:
  ① 转换层:.xls→.xlsx(优先让用户先另存;或集成 ssconvert/pandas-calamine)
  ② 识别层:按 field_map.yaml + 表头模糊匹配 认表型
  ③ 标准化:统一列名/类型/单位
  ④ 汇总:小类/ABC/价格带/库存滞销/Item映射/字段缺口
  ⑤ 错误检查:缺关键字段→缺口报告;数据矛盾→不"圆",列假设(§13.20)
边界:raw_exports/ 与 outputs/明细 全 git-ignore;只脱敏汇总md入库。
```

### C. 稳定工具版(未来产品化,仅方向)
上传页面 → 自动识别表型 → 自动出 品类/SKU/陈列/库存诊断 → 导出 Excel/Markdown/PDF。(不展开开发)

## 10. 统一文件命名规范
```
花厅坊_<表型>_<导出日>[_<时段>].xlsx
例:
  花厅坊_商品档案_20260610.xlsx
  花厅坊_商品销售汇总_20260610_近90天.xlsx
  花厅坊_库存表_20260610.xlsx
  花厅坊_进货表_20260610_近90天.xlsx
  花厅坊_品类销售汇总_20260610.xlsx
  花厅坊_去年同期销售_20250610-20260610.xlsx
```
**为什么**:①门店前缀防多店混 ②表型固定→脚本认表 ③日期/时段写死→可追溯、可同比、不靠人记。
