## 使用流程

### 处理判断类任务前

1. **先读本表全表 18 条事实**（5-10 分钟）
2. 任何判断涉及 F-XX 项 → 直接引用 / 无需重新求证
3. 若任务输入与某条 F 冲突 → **不擅自接受任务输入** / 先 confront 用户
4. 若任务需补新事实 → 走 §"沉淀新事实流程"

### 沉淀新事实流程

1. 新事实必须有明确**确认来源**（客户验证日志 / sign off 文档 / vault 实测）
2. 新事实必须标 **fact_layer**（observed / client_told / inferred / pending）
3. 新事实必须列**适用范围 + 是否可被推翻 + 推翻条件**
4. 提交时编号顺延 F-19 / F-20 / ...
5. 每次新增需更新 §"版本记录"

## 反幻觉边界

- ✅ 本表只登记 vault 已有 / 客户口述已 confirm / 文件已观察的事实
- ❌ 不登记推断（推断走对应分析文件 / 标 inferred）
- ❌ 不登记 pending（pending 走待补清单 / 不能成"事实"）
- ❌ 不登记"佳妮已执行 53 SKU 中 X 个" 等未发生事项
