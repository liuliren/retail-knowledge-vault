# V4.0 品类表 vs A-GTM 数据源 匹配度+执行度分析 v0.1

> **本报告**回答两个问题：
> 1. **匹配度**：V4.0 品类表（308 L4）与 A-GTM 系统导出的食品动销数据（3,694 SKU / 56 cat_name）能 mapping 上多少？
> 2. **执行度**：实际门店运营的 SKU 是否按 V4.0 体系落地？品名/品牌与 V4.0 定义的契合度？
>
> **演示表**：`/tmp/V4.0_匹配度分析/V4.0_vs_POS_匹配度演示表.csv`（3,694 行 × 10 列 / 完整 SKU 级 mapping）

## §A. 数据基线

### A.1 V4.0 品类表（D018 唯一最终版）

| 维度 | 数据 |
|---|---|
| 总 L4 主数据 | **308** |
| L1 大类 | 3（休闲食品 175 / 基础食品 77 / 水饮料 56）|
| L2 中类 | 16 |
| L3 小类 | 63 |
| 字段 | 13（L1/L2/L3/L4/定义/品牌示例/角色/价格带/陈列/存储/场景/客群/大湾区）|

### A.2 A-GTM POS 食品动销源表（5/8 客户方提供）

| 维度 | 数据 |
|---|---|
| 总 SKU | **3,694** |
| 销额 30T | 372,464 元 |
| cat_name 数 | **56** |
| 数据切片 | 30T（4/8-5/8）|
| 字段 | 27（含货号/品名/单位/销售数量/销售金额/零售价/进价/类别/类别名称/库存/等）|

**结构差异**：
- V4.0 = **品类树**（L1-L4 4 层 / 308 L4 / 13 字段）
- A-GTM = **SKU 销售汇总**（按 cat_name 分组 / cat_name 是 L3 级别 / SKU 级粒度）
- A-GTM 的 cat_name ≈ V4.0 的 L3（不是 L4）

## §G. 核心结论

### G.1 匹配度结论

```
🟢 V4.0 与 A-GTM 食品动销数据匹配度：75-80%（综合）
   - 76.6% 销额能 mapping（28.5 万 / 37.2 万）
   - 81.6% SKU 能 mapping（3,015 / 3,694）
   - 73.2% cat_name 能 mapping（41 / 56）
   - 100% V4.0 L1 大类有 POS 数据触达
```

**结论**：V4.0 设计**高度契合**实际门店食品大类的 76% 销额 / 大湾区本地化做得好 / 主流品类全覆盖。

### G.2 执行度结论

```
🟢 V4.0 体系在花厅坊门店落地度：约 70-75%
   - Top 10 SKU 80% 完美匹配
   - 主流品牌（益力多/燕塘/蒙牛/粤盐/海天/千禾/甘竹）在 V4.0 示例中
   - 大湾区本地特色（粤盐/甘竹/燕塘/致美斋）落地良好
```

**结论**：V4.0 不是空中楼阁 / 而是**实际门店运营的真实写照**。

### G.3 不匹配缺口（25% 空间）

```
⚠️ 4 大缺口（按销额）：
   1. 冷冻类  41,526 元（11.2%）/ V4.0 设计时未纳入冷冻链
   2. 酒精类  35,408 元（9.5%）/ V4.0 水饮料 L1 不含酒精
   3. 火腿肠  10,019 元（2.7%）/ V4.0 没单独肉制品 L1
   4. 茶叶    109 元（0.0%）/ V4.0 不含干茶
```

**结论**：缺口不是"V4.0 错"/而是 V4.0 范围**未设计这 4 类**。需用户决定：
- 选项 A：扩 V4.0 → V4.1+（加冷冻/酒精/肉制品/茶叶 4 个 L1）
- 选项 B：4 类单独管理（不进 V4.0）
- 选项 C：等启明侧出"百货 V4.0 大湾区版" / 一并扩展
