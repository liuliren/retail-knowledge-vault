## §A.0 v2.2 → v2.2.2 客流口径订正（5/9 用户 2 次订正 / 矛盾完美闭合）⭐⭐

### 用户 5/9 第 2 次订正（订正 v2.2.1 错认）

```
❌ v2.2.1 错认：145,490 = 32 天客流（成交客户数）
✅ 用户 5/9 第 2 次订正：145,490 实际是「客单数」（订单数 / POS 笔数）/ 不是客流

原因：
  - 客流 = 进店人次（来过店的独立客户）/ 另外计算路径
  - 客单数 = POS 订单数 / 一个客户可能开多张单（购物多次结算 / 会员卡分开结）
  - 用户告知："保留客流计算路径 / 但先按客单数进行"
```

### 矛盾完美闭合（数学验证）⭐⭐

```
145,490 / 32 天 = 4,547 单/天（客单数 / 不是客流）
[推断] 但报表内可能双倍统计（待启明 5/10 验证）
真实客单数 ≈ 4,547 / 2 = 2,273 单/天（≈ 用户告知 2,300-2,500 范围）

✅ 数学验证（口径 B：销售 / 客单数 = 笔均）：
  日均销售 96,723 元 / 日均客单数 2,273 = 42.55 元/单
  ≈ 报表客单价 42.14 元（差 0.96%）⭐⭐ 完美对上！

✅ 客单价定义锁定：客单价 = 销售额 / 客单数（POS 笔均）
   每张订单（笔）平均成交金额 ≈ 42.14 元
   不是"销售 / 客流"（那是件单 / 不同口径）
```

### 数据三层口径 v1.0（vault 锁定）

| 口径 | 定义 | 数值 | 关系 |

## §D. 数据矛盾（必须 confirm）⚠️

### D.1 客流 × 客单价 ≠ 销售总额

```
30T 客流 145,490 × 客单 42.14 = 6,131,948 元
30T 类别销售占比总额 = 2,998,401 元
差异 = 2 倍多 ⚠️
```

**[推断]** 可能解释：
1. **客流 = 过店人数**（包括只逛不买）/ 客单价 = 实际成交单价 → 两者乘积无意义
2. **客流统计双倍**（进店+出店各算一次）→ 实际客流 ÷ 2 = 72,745（与 v2.1 一致）
3. **客单价是平均成交单价**（来自 POS）/ **客流量是过道流量**（来自门口传感器）→ 两个独立数据

→ **必须客户 confirm 客流定义** / 否则后续分析可能错。

### D.2 v2.1 计算可能用了 v2.2 的"客流 ÷ 2"

如果是双倍统计 → v2.1 客流 72,745 实际是"成交客户数" / 30T vs 60T 变化仍 +38.3%（除 2 后比例不变）/ 但绝对客流是 72,745 不是 145,490。

→ 需要客户 confirm。
