### 1.3 派生字段（dry-run 自动计算）

| 字段 | 算法 |
|---|---|
| gross_profit_30d | sales_amt_30d - cost_price × sales_qty_30d |
| gp_rate_30d | gross_profit_30d / sales_amt_30d × 100 |
| abcd_grade | 按 30 天销额累计 50% / 80% / 95% / 其他=D |
| z_flag | 多标签字段：Z_neg_inv（负库存）/ Z_zero_sales（零销售）/ Z_no_supplier（主供应商=0）|

## §3. ⚠️ 数据质量警告（启明必须解决 / 否则后续 SKU 级分析受限）

### 3.1 main_supplier 字段 99.5%+ 缺失

| 品类 | main_supplier=0 SKU | 占比 |
|---|---|---|
| 方便食品 | 198/199 | **99.5%** |
| 调味 | 745/747 | **99.7%** |

→ **影响**：
- 表 A 商品基础表 v1.0.1 字段 13（主供应商 FK）无法装填
- 表 B 字段 18（主供应商）= 全部空
- M-DEC-003（调改商品供应链落实节奏）失字段依据
- 采购体系（佳妮+文员+高彬+启明）4 个采购口的归因无法做

→ **建议**：启明 W19 内组织全店主供应商字段维护（先 A 类 SKU + 调改主线相关 ~500 SKU）。

### 3.2 ERP 进价不准（推断）

观察：牛肉 90.3% / 猪肉档 95.8% / 烘焙 85.5% 等品类毛利率虚高。
→ [推断] ERP 进价字段是上次进货价 / 不是加权平均 / 或部分 SKU 缺失（被视为 0）。
→ 影响：所有 gross_profit / gp_rate 字段实际不可信。

→ **建议**：启明 W19 内财务（佳妮）核对全店 ERP 进价 / 加权平均成本 / 修复后重装载。

### 3.3 文件名 vs 实际类目错位

5/8 客户方提供 4 个 120 天文件 + 1 个 2025 年文件**实际是粮油**（不是文件名标的方便食品/调味/茶叶/粮油）。
→ 影响：调味组缺 2025 年可用基线（A 阶段分析降级到 3 阶段 / 不是 4 阶段）。

→ **建议**：启明 W19 内全店重导（按真实大类）+ 文件命名 SOP 建立。
