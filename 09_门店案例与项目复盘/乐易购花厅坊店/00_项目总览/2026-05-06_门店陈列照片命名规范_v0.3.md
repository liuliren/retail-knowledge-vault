## §1. v0.2 → v0.3 五处升级总览

| # | 升级项 | v0.2(旧) | v0.3(新) | 触发原因 |
|---|---|---|---|---|
| 1 | **层数定义错误订正** | "第 7 层 = 顶层囤货层" | **统一规则:1 层永远是顶,末层永远是底**;7 层 = 顶部加 1 层 | v0.2 §3.3 自相矛盾(1=顶 vs 7=顶) |
| 2 | **引入三轴体系 P+L+G** | 仅物理区域(P 单轴) | **P 物理区域 + L 逻辑品类 + G 物理货架**(三维定位) | v0.2 无法表达"品类乱投放"等异常 |

## §2. ⭐ 核心 — 三轴体系(P + L + G)

### §2.1 三轴定义

| 轴 | 中文 | 含义 | 关系 | 主对接 |
|---|---|---|---|---|
| **P 轴** | 物理区域 | 门店实体空间分区(墙 / 通道 / 区域) | 1:1 物理位置 | 门店区域命名 |
| **L 轴** | 逻辑品类 | 商品品类树 | **跨 P 区** N:M 映射 | vault 标准品类表 |
| **G 轴** | 物理货架 | 货架编号(排 / 面 / 组 / 层 / 位) | 嵌入 P 轴下 | DisplayMap |

### §2.2 三轴交叉的 5 大优势

1. **三维定位**:任一照片精确定位 / 跨维度交叉检索
2. **品类一致性自动检查**:L 轴 grep 跨 P → 暴露品类乱投放
3. **门店区域调整可解耦**:P 轴改动不影响 L 轴
4. **vault 品类表强对齐**:L 轴是品类表的"实战外延" / 双向校对
5. **DisplayMap / 看板自动生成**:三轴 → Excel pivot → 自动生成各品类陈列分布图

## §7. 完整照片命名公式 v0.3(8 段)

### §7.1 主公式

```
<P>-<G>-<NN组>-<M层>[-<NN位>][-<L 主品类>][-<层位语义>]_<内容>_<日期>_<时段>.jpg
```

### §7.2 分隔符使用规则：字段内部用 `-`，字段之间用 `_`（v0.4 加 / 强制）

> **规则定位**：硬约束 / 强制 / 不可破例。
> **触发**：W20-Day1 W20-Day4 口袋卡 §6 命名示例不合规事件 / vault 反幻觉 §13 + 工程操作纪律 §17 共同要求。
> **适用范围**：仅"门店陈列照片命名"。Markdown 文件名、报告名、表格名仍可继续使用 `_` 作为主分隔符 / 不否定用户长期使用 `_` 的习惯。

#### §7.2.1 核心规则

门店陈列照片命名中，`-` 与 `_` 的职责必须分开：

- **`-`（短横）**：用于同一大字段内部的层级连接，尤其是位置编码字段内部。
- **`_`（下划线）**：用于分隔照片命名的四个大字段：位置编码 / 内容 / 日期 / 时段。

#### §7.2.2 标准结构

```text
<P>-<G>-<NN组>-<M层>[-<NN位>][-<L主品类>][-<层位语义>]_<内容>_<日期>_<时段>.jpg
```

第一个 `_` 之前是"位置编码字段"，内部统一用 `-` 连接。`_` 仅出现在 4 处大字段分隔位（位置编码 → 内容 → 日期 → 时段 → .jpg）。

#### §7.2.3 4 大字段拆解

标准示例：`HS-G3-A-04-3-L冲调-黄金视线_商品价签_20260513_常规.jpg`

| 大字段 | 内容 | 字段内部分隔符 | 与下一大字段分隔符 |
|---|---|---|---|
| 位置编码 | HS-G3-A-04-3-L冲调-黄金视线 | `-`（多层）| `_` |
| 内容字段 | 商品价签 | 单段 / 无内部分隔 | `_` |
| 日期字段 | 20260513 | 单段 / 数字 YYYYMMDD | `_` |
| 时段字段 | 常规 | 单段 / 锁定值 | `.jpg` 结尾 |

#### §7.2.4 4 大理由（为何这样定）

1. **G3-A 是完整货架编码**，不能写成 G3_A —— 货架编号 + 货架面（A/B）是一个整体单位，必须用 `-` 内部连接。
2. **`_` 是大字段分隔符**，如果货架内部也用 `_`，后续 Python / Excel / Claude 批量解析会把字段切碎（`split("_")` 切到 N 段而非 4 段）。
3. **`-` 用于位置编码内部层级**，可以稳定表达门店、货架、组位、层位、主品类、层位语义 7 层，互不冲突。
4. **`_` 用于分隔内容、日期、时段**，便于后续批量检索、照片归档、DisplayMap 对照和 AI 识别。

#### §7.2.5 禁止写法（5 错误点 / W20-Day4 口袋卡 §6 v0.2 反例）

```text
❌ HS-G3_A-04-3_茶叶_黄金层_冲调茶叶柜眼平_20260513_AM.jpg
```

5 错误点逐项：

| # | 错误 | 应为 | 错误所在 |
|---|---|---|---|
| 1 | `G3_A` | `G3-A`（货架编号内部用 `-` / `_` 是大字段分隔）| 位置编码内部 |
| 2 | `茶叶` | `L冲调`（必须使用 §4 L 主品类锁定值 / 必带 `L` 前缀）| 位置编码内部 |
| 3 | `黄金层` | `黄金视线` 或 `黄金手拿`（必须使用 §6.3 层位语义锁定值）| 位置编码内部 |
| 4 | `冲调茶叶柜眼平` | `商品价签` / `全景` / `调改前` / `24h复拍` 等 §9 内容字段锁定值 | 内容字段 |
| 5 | `AM` | `夜灯前` / `夜灯后` / `次晨白光` / `常规`（必须使用 §10 时段锁定值）| 时段字段 |

#### §7.2.6 正确写法

```text
✅ HS-G3-A-04-3-L冲调-黄金视线_商品价签_20260513_常规.jpg
```

拆解：
- 位置编码：`HS-G3-A-04-3-L冲调-黄金视线`（7 层 / 全部用 `-` 内部连接）
- 内容字段：`商品价签`（§9 锁定值）
- 日期字段：`20260513`（YYYYMMDD）
- 时段字段：`常规`（§10 锁定值）

#### §7.2.7 落地纪律（强制）

1. **Lint 强制扫**：每月 G03 lint 扫所有照片命名 / 第 1 个 `_` 之前不允许出现 `_` / 不合规标 ❌ 待修正。
2. **现场不得自创字段值**：G 货架 / L 主品类 / 层位语义 / 内容字段 / 时段字段全部锁定值（参 §3 / §4 / §6 / §9 / §10）/ 现场不得自创。
3. **AI 识别稳定性**：本规则保证 Python / Claude / Excel 用 `split("_")` 切分到 4 段（位置编码 / 内容 / 日期 / 时段）/ 不会误切。
4. **Markdown 文件名不受此规则约束**：报告名 / 表格名 / 章程名 / 备忘录名等 Markdown 文件继续用 `_` 作主分隔符 / 与照片命名解耦。

### §7.4 完整命名实例(覆盖 14 子文件夹场景)

```bash

# ⭐ 异常证据(品类乱投放 / L 轴标注)
```
