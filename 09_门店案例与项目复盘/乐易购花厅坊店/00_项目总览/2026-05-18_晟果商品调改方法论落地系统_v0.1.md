# 晟果商品调改方法论落地系统 v0.1

> **本文件定位**：晟果咨询的**顶层串联文件**。把 vault 内 12 份 01_方法论 + 商品库 v1.1 + 品类匹配算法包 + 花厅坊 6 表 + M-DEC-007/008 候选串成「**门店整体调研 → 商品调改决策 → 现场执行 → 复盘验证 → 方法论固化**」可复制系统。
>
> **不是**：单店项目记录 / 方便速食单点收口 / 治理审计文档
> **是**：晟果对外可交付的咨询产品 SOP 起点

---

## 1. 系统定位

### 1.1 是什么

晟果商品调改标准作业系统 = **3 个标准化产物**：
1. **方法论**（可教学）：从客群画像 → 商品决策 → 现场执行的完整链路
2. **工具包**（可复用）：27 张标准表单 + Excel 模板 + 算法包
3. **样板案例**（可对外讲）：花厅坊 → 洗护 → 标品 → 沙埔店复制全链路

### 1.2 面向客户

- 社区超市
- 标超 / 大社区超市
- 生鲜加强型社区店
- 中小连锁超市样板店

### 1.3 解决什么问题

| 问题 | 当前痛点 | 本系统解决路径 |
|---|---|---|
| 商品结构混乱 | 同质化高 / 滞销多 / 引流款失效 | TPOS + CDT 重构品类角色 |
| 淘汰决策不清 | 凭经验 / 无量化 / 无验收 | ABCD + 库存挤压 + 7/15/30 天复盘 |
| 采购无计划 | 凭门店补货习惯 / 无淘汰协同 | 淘汰清单 → 采购计划 → 供应商对接 |
| 陈列无逻辑 | 包装/品牌/促销混层 / 黄金层错配 | DisplayMap + 层板逻辑 + 端架决策 |
| 供应链缺口无闭环 | 缺货反复 / 新供应商引入慢 | 缺口清单 → 引进表 → 试销复盘 |

### 1.4 最终交付

- **客户侧**：诊断报告 + 整改报告 + 数据复盘报告 + 老板可对外讲样板
- **晟果侧**：3 层咨询产品（详见 §13） + 标准工具包 + 样板案例库
- **vault 侧**：M-DEC v1.0 入库（中层决策资产）+ 方法论持续迭代

## 4. TPOS 与 CDT 模块

### 4.1 TPOS 推导路径

```
客群画像（T-04）
  → 顾客 3-5 种主要"购物任务"分布
  → 每种任务对应的 SKU / 品类 / 价格带 / 包装
  → TPOS 矩阵：任务 × 品类 × 渗透率
```

**典型 TPOS（社区超市 / 待花厅坊数据校准 [推断]）**：
- 早餐场景：方便速食 / 冲调 / 蛋奶 / 烘焙
- 晚餐场景：生鲜 / 调味 / 米面粮油
- 周末家庭采购：礼盒 / 大包装 / 低价跑量
- 即时刚需补充：单包装 / 高频日用
- 节日 / 节令：节令礼盒 / 季节性 SKU

### 4.2 CDT 拆解

```
顾客在该品类前的决策树（CDT）：
  → 品牌？规格？价格？口味/功能？包装？
  → 各分支占比
  → 品类应保留几个分支 / 每个分支几个 SKU
```

**示例（冲调品类 / 待花厅坊数据校准 [推断]）**：
- 第 1 分支：**饮品场景**（蜂蜜 / 麦片 / 咖啡 / 奶茶 / 茶叶 / 早餐饮品）
- 第 2 分支：**包装大小**（个人即饮 / 家庭装 / 礼盒）
- 第 3 分支：**功能定位**（健康养生 / 提神醒脑 / 营养早餐 / 送礼）

### 4.3 TPOS / CDT 如何影响商品结构

| 影响 | 落地 |
|---|---|
| 商品结构 | 每个 TPOS 任务必须有"足够"SKU 覆盖 / 每个 CDT 分支必须有"代表"SKU |
| 价格带 | 每个分支低/中/高 3 个价格带 / 不能断档 |
| 品牌 | 1 个引流大品牌 + 1-2 个利润品牌 + 0-1 个补充本地品牌 |
| 规格 | 必须覆盖单包/中包/家庭装 |
| 功能 | 每个 CDT 分支至少 1 个 SKU |
| 陈列分区 | 同 CDT 分支 SKU 物理相邻 / 不分散 |

### 4.4 输出工具表

- **T-05 TPOS 矩阵**
- **T-08 CDT 拆解图**
- **T-09 品类角色判断表**（引流 / 利润 / 补充 / 便利）
- **T-10 品类规划表**（宽度 / 深度 / 价格带 / 品牌结构）

## 5. 商品基础资料库模块

### 5.1 为什么必须建商品基础库

| 痛点 | 不建后果 |
|---|---|
| 品牌字段缺失 99.5%（花厅坊实测）| 无法做品牌结构分析 / 无法做引流款决策 |
| 同名不同条码 / 同条码不同名 | 用名 join 必错 / 数据 ETL 全部失真 |
| 进价字段不全（生鲜 / 散称缺）| 毛利计算失真 / 淘汰判据失效 |
| 类别码与业务口径不一致 | 0206% vs 方便速食 vs 方便食品 vs 0206XX 4 子类混淆 |

### 5.2 字段（参 [[花厅坊商品基础档案Excel空模板_v0.1]] 13 sheet）

```
01_商品基础档案总表（核心 / 24-36 字段）
  ├─ 条码（主键）
  ├─ 商品名称
  ├─ 规格 / 单位 / 包装
  ├─ 品牌（清洗后）
  ├─ 类别码（020601 / 020602 等 4 级）
  ├─ 类别名（方便面 / 方便粉 / 即食类 / 等）
  ├─ 售价 / 进价 / 毛利率
  ├─ 主供应商 / 供应商名称
  ├─ 引进日期 / 末次销售日 / 末次进货日
  ├─ 是否停购 / 是否在售
  ├─ 数据质量字段（is_pending_check / dq_*）
  └─ ...
02-12 sheet（销售明细 / 库存 / 品类匹配 / 供应商信息 / 陈列位置 / 照片索引 / 动销衍生 / 商品动作 / 数据质量 / 阶段作业包 / 字段字典）
```

### 5.3 品牌清洗规则

参 [[_品类匹配算法包_v1.0/01_主算法_花厅坊POS清洗脚本_v0.1.2/README]]：
- 优先级 1：明确品牌字段（如有）
- 优先级 2：商品名前缀提取（如"统一红烧牛肉面桶" → "统一"）
- 优先级 3：关键词匹配字典（人工维护）
- 优先级 4：人工复核（置信度 < 80%）

### 5.4 单品匹配规则

- 主键 = 条码（不变）
- 同条码不同名 → 选 POS 系统最新名 + 标记冲突
- 同名不同条码 → 不合并 / 两条独立记录

### 5.5 品类匹配规则

- 主字段 = 类别码（如 020601 = 方便面）
- 不依赖类别名（"方便食品" vs "方便速食" vs "速食" 同义 / 但码不同）
- ⚠️ 0206 裸大类码（无子类）需人工复核（花厅坊 4 SKU / 7 销售记录 / 待启明 confirm 是否新品未分类）

### 5.6 置信度分层

| 置信度 | 判定 | 处理 |
|---|---|---|
| ≥ 95% | 自动入库 | 不复核 |
| 80-95% | 半自动 | 顾问抽检 10% |
| 60-80% | 待复核 | 顾问 100% 复核 |
| < 60% | 必须人工 | 标 dq_pending=true |

### 5.7 两天建档路径（5/18-5/19）

| 时点 | 动作 | 输出 |
|---|---|---|
| 5/18 EOD | 洗护品类 POS 全量导出 | raw csv |
| 5/19 AM | 跑 [[01_主算法_花厅坊POS清洗脚本]] v0.1.2 | _mapped.csv + 置信度报告 |
| 5/19 PM | 顾问复核 + 待办字段填实 | T-11 商品基础资料表（洗护版）|
| 5/19 EOD | sign off | 5/20 开始洗护品类分析 |

## 6. 商品分析与单品分析模块

### 6.1 必跑 10 项分析

| # | 分析 | 数据源 | 输出 |
|---|---|---|---|
| 1 | 品类销售分析 | POS 时段销售 | 各品类销售额 / 占比 |
| 2 | 毛利分析 | POS + 进价 | 各品类毛利率 / 毛利贡献 |
| 3 | 库存分析 | ERP 库存表 | 库存金额 / 周转天数 |
| 4 | 周转分析 | POS + 库存 | 各品类 / SKU 周转 |
| 5 | 缺货分析 | POS + 库存 | 0 库存 + 30 天有销售的 SKU |
| 6 | 价格带分析 | POS + 售价 | 各品类价格带分布 + 断档 |
| 7 | 品牌结构分析 | POS + 品牌（清洗后）| 各品牌销售占比 / 集中度 |
| 8 | 规格结构分析 | POS + 规格 | 各规格销售占比 / 大包装 vs 小包装 |
| 9 | ABCD 分类 | 销售额 + 毛利额双维度 | A / B / C / D 4 级 |
| 10 | 调改前后对比 | POS 时段对比 | 净增量 / 转化率 / 客单价变化 |

### 6.2 保留 / 淘汰 / 观察 / 调位 / 促销消化 / 新品替代判据

```
[决策树]
  SKU →
    ├─ A 类（销售额前 20% + 毛利率 ≥ 大类平均）→ 保留 + 黄金层
    ├─ B 类（销售额 20-50% / 毛利率 ≥ 大类平均）→ 保留 + 中层
    ├─ C 类（销售额 50-80% / 毛利率 中位）→ 观察 / 视情况调位
    ├─ D 类（销售额 80-100% / 毛利率 低）→
    │   ├─ 库存压力大（库存金额高 + 末次销售 > 30 天）→ 促销消化 / 后续淘汰
    │   ├─ 库存压力小 → 直接淘汰候选
    │   └─ 战略保留（民生基础 / 引流款）→ 调位 + 标"战略"
    └─ 新品替代候选 → 引进表（T-14）
```

## 10. 数据跟踪与复盘模块

### 10.1 三节点

| 节点 | 关注指标 | 输出 |
|---|---|---|
| **7 天观察** | 销量 / 30 秒找到率 / 拿取顺畅度 / 缺货 | T-23 7 天观察表 |
| **15 天复盘** | 销售 / 毛利 / 客单 / 大类 vs 子类 / 调改 SKU vs 未调改 SKU | T-24 15 天复盘表 |
| **30 天诊断** | 净增量 / 转化率 / 库存周转 / 供应商响应 / 客户反馈 | T-25 30 天诊断报告 |

### 10.2 指标口径

- **净增量 +10pp 阈值**（参 v0.5 方便速食阈值）：调改后销量 - 调改前同期 / 排除大盘波动
- **转化率**：销量 / 客流（用客单数近似 / 参 5/18 客流近似客单 [推断]）
- **库存周转**：库存金额 / 日均销售金额

### 10.3 复盘模板

参 [[方便速食组13天调改后效果分析报告_v0.5_数据填实]]（13 天版 / 可改造为 7/15/30 天通用）

### 10.4 调改有效性判断

```
有效 ⇔ 满足任一：
  - 销售额净增量 ≥ +10pp
  - 毛利率提升 ≥ +1pp
  - 30 秒找到率 ≥ +20pp
  - 客户反馈正面 ≥ 3 条
无效 / 待优化 ⇔ 至少一条 + 至少一条不满足
失败 ⇔ 所有指标都下降 → 立即回滚
```

## 11. 供应链优化模块

### 11.1 从诊断到供应链缺口


### 11.3 供应商对接清单（T-16）

- 现有主供应商：稳定性 / 响应速度 / 缺货率
- 新供应商引入：试单 / 测毛利 / 测周转 / 30 天评估
- 退出策略：连续 3 月低于阈值 → 启动替换

### 11.4 新供应商引入路径

```
顾问推荐 → 老板 sign off → 试单 → 30 天试销 → 复盘 → 长期合作 / 退出
```

### 11.5 持续优化

- 月度供应商评分（毛利 / 周转 / 缺货 / 配合度）
- 季度供应商盘点（淘汰末尾 10%）
