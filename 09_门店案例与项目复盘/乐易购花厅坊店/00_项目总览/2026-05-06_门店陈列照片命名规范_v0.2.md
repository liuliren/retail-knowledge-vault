# 门店陈列照片命名规范 v0.2(迭代版 / 待六哥审核)

> **目的**:把 v0.1 在花厅坊二访 322 张实战中暴露的 5 处不一致,升级为 v0.2 标准 + 5/6 之后所有门店复用。
>
> **范围**:门店陈列 / 货架 / 价签 / 端架 / 堆头 / 商圈 / 竞品 / 商圈环境 / 平面图等所有照片。
>
> **审核口径**:六哥逐字审 → 确定 → 落 `10_咨询交付模板/02_交付SOP/` 全局 SOP。

---

## §1. v0.1 → v0.2 五处升级总览

| # | 升级项 | v0.1(旧) | v0.2(新) | 触发场景 |
|---|---|---|---|---|
| 1 | **组号 2 位前导零** | `_6组_` | `_06组_` | G4-A 第 6 组命名不一致(其他 5 组都是 2 位)→ 排序错乱 / 自动化脚本失败 |
| 2 | **全景图统一术语** | "整排图" / "整体图" 混用 | `_整组图_`(单组全景)| G4-B 用"整排图"/ G3-B 用"整体图"→ 检索分类困难 |
| 3 | **整面级全景命名** | (缺) | `_整面_全景_`(整个货架的总览)| 缺货架级对比基线 → 无法做调改前后货架级对比 |
| 4 | **层位语义命名**(可选) | 只标 `_2层_` | `_2层_黄金视线_` | 跨货架对比时层号不可比(G3-B 第 6 层 ≠ G4-A 第 6 层)|
| 5 | **货架层数差异化标准** | 强制 5 层 | 5 层 / 6 层 / 7 层 各自标准 | G3-B 6 层 / G4-A 6 组特殊 7 层 — 物理差异需明示 |

## §2. 标准命名格式 v0.2

### §2.1 主公式

```
<门店简称>_<分店>_<区域>_<货架名>_<货架编号>_<NN组>_<层位>_<内容类型>_<日期>.jpg
```

### §2.2 11 字段定义表

| # | 字段 | 取值规则 | 是否必填 | 实例 |
|---|---|---|---|---|
| 1 | 门店简称 | 客户简称(中文 ≤ 4 字)| ✅ | 花厅坊 |
| 2 | 分店 | 主店 / 分店名 | ✅ | 主店 |
| 3 | 区域 | 大区域分类(休食区 / 生鲜区 / 加工区 / 散称区 / 收银区 / 商圈 / 竞品 等) | ✅ | 休食区 |
| 4 | 货架名 | 货架功能名(冲调货架 / 水饮货架 / 速食货架 / 端架 / 堆头 等) | ✅ | 冲调货架 |
| 5 | 货架编号 | G(货架前缀)+ 排号(数字)+ 面 A/B(可选 + 端架号)| ✅ | G4-A / G4-B / G3-B / G4-端架 |
| 6 | NN 组 | **强制 2 位前导零**(01 组 / 02 组 / ... / 06 组),整面级照片用 `整面` | ✅ | 06 组 / 整面 |
| 7 | 层位 | 物理层号 + 可选层位语义(见 §3) | 货架照必填 / 端架可空 | 1层_形象 / 2层_黄金视线 / 整组图 / 整面_全景 |
| 8 | 内容类型 | 商品价签 / 商品 / 价签 / 全景 / 局部 / 围货 / 异常 / 调改前 / 调改后 / 24h复拍 | ✅ | 商品价签 / 24h复拍 |
| 9 | 日期 | YYYYMMDD(8 位)| ✅ | 20260506 |
| 10 | 时段(可选) | 夜灯前 / 夜灯后 / 次晨白光(夜班 3 时段对比专用)| ⚠️ 调改时必填 | 夜灯前 / 次晨 |
| 11 | 序号(可选) | _NN 序号(同机位多张时)| 可选 | _01 / _02 |

## §3. ⭐ 层位语义对照表(核心升级)

### §3.1 6 层标准母版(CDT 顾客决策路径)

> 来源:vault `01_科学零售方法论/06_陈列科学/` + 花厅坊版本 B 设计

| 物理层 | 层位语义 | 角色 | 适用人群 / 场景 | 例 |
|---|---|---|---|---|
| **第 1 层(顶)** | 形象层 | 形象品 + 高端轻量品 | 远视 / 拍照 | 高端进口麦片(轻包装)|
| **第 2 层** | 黄金视线层 | 主销 + 高毛利 | 1.6m 视线水平 | 中端燕麦片(15-25 元)|
| **第 3 层** | 黄金手拿层 | 高转化(主销 + 利润) | 1.2m 自然手拿 | 主推 SKU |
| **第 4 层** | 补充层 | 二线 + 长尾 | 0.8m 蹲身 | 二线品牌 / 杂品 |
| **第 5 层** | 大包装层 | 大包装家庭装 | 蹲身拿大件 | 1.5kg 燕麦袋装 |
| **第 6 层(底)** | 围货层 | 整箱 / 库存缓冲 | 不取货,纯陈列 | 整箱整摞 |

### §3.2 5 层简化(底层无围货)

| 物理层 | 层位语义 | 与 6 层差异 |
|---|---|---|
| 第 1 层 | 形象层 | 同 |
| 第 2 层 | 黄金视线层 | 同 |
| 第 3 层 | 黄金手拿层 | 同 |
| 第 4 层 | 补充大包层 | 4 + 5 合并 |
| 第 5 层 | 大包装(底)层 | 直接落底,无围货 |

### §3.3 7 层扩展(G4-A 6 组特殊场景)

| 物理层 | 层位语义 | 触发条件 |
|---|---|---|
| 第 1-6 层 | 同 §3.1 | — |
| **第 7 层** | **顶层囤货层 / 围货层** | 货架顶部空间充足 + 大包装超量(如 G4-A 06 组)|

### §3.4 货架 → 层数对照(花厅坊实测)

| 货架 | 层数 | 触发条件 | 备注 |
|---|---|---|---|
| G3-B 速食 | 6 层 | 货架物理高(2.4m+)| 标准 6 层母版 |
| G4-A 冲调 | 5 层(常规)| — | 5 层简化版 |
| G4-A 06 组 | **7 层(特殊)** | ⚠️ 物理空间长 + 当前杂物堆积(干果 / 凉果 / 奶酪 / 即食 / 大包装混杂)| 调改后建议回归 5-6 层 + 6 组品类清理 |
| G4-B 水饮 | 5 层 | — | 5 层简化版 |

## §4. 全景图 3 级体系(v0.2 新增)

| 级别 | 命名字段 | 拍摄范围 | 数量 / 货架 | 用途 |
|---|---|---|---|---|
| **L1 整面_全景** | `_整面_全景_` | 整个货架(6 组合一)正面平拍 | 1 张 | 货架级总览 / 调改前后对比 |
| **L2 整组图** | `_NN组_整组图_` | 单组(单组 5/6/7 层) | 6 张(每组 1)| 组级总览 / 6 组结构对比 |
| **L3 单层 + 价签** | `_NN组_M层_X层位_商品价签_` | 单层 + 价签(可读 SKU + 价格) | 30-42 张(组×层) | SKU 级诊断 / 价格分析 |

### §4.1 拍摄完整性 Checklist(每个货架 / 调改时)

- [ ] L1 整面_全景 × 1 张
- [ ] L2 整组图 × 6 张(01-06 组各 1)
- [ ] L3 单层 + 价签 × 全层覆盖(5 层 → 30 张 / 6 层 → 36 张 / 7 层 → 42 张)
- [ ] 端架 × 全景(每个端架 1 张)
- [ ] 异常拍特写(乱投放 / 价签错 / SKU 错位 等)

## §6. 时段字段(夜班 3 时段对比 / v0.2 新增)

> 来源:`feedback_夜班作业纪律.md` D004 节奏

| 时段 | 时间 | 灯光条件 | 用途 |
|---|---|---|---|
| 夜灯前 | 22:00 调改前 | 店内日光灯(色温暖)| 调改前基线 |
| 夜灯后 | 02:00 调改完成 | 店内日光灯(色温暖)| 调改后即刻验收 |

## §7. 批量改名脚本模板(可复用)

### §7.1 第 6 组 → 第 06 组(组号 2 位前导零)

```bash
cd <目标目录>
for f in *_6组_*.jpg; do
  [ -f "$f" ] || continue
  mv "$f" "${f//_6组_/_06组_}"
done
```

### §7.2 整排图 / 整体图 → 整组图

```bash
for f in *_整排图_*.jpg; do
  [ -f "$f" ] || continue
  mv "$f" "${f//_整排图_/_整组图_}"
done
for f in *_整体图_*.jpg; do
  [ -f "$f" ] || continue
  mv "$f" "${f//_整体图_/_整组图_}"
done
```

### §7.3 加层位语义(可选 / 半自动)

```bash
# 警告:层位语义因货架不同,需先确认层数 + 标准 → 不建议无脑批量
# 5 层货架(G4-A / G4-B):
# 1层_形象 / 2层_黄金视线 / 3层_黄金手拿 / 4层_补充大包 / 5层_大包装
# 6 层货架(G3-B):
# 1层_形象 / 2层_黄金视线 / 3层_黄金手拿 / 4层_补充 / 5层_大包装 / 6层_围货
# 7 层(G4-A 06 组特殊):上 6 层同 + 7层_顶层囤货
```
