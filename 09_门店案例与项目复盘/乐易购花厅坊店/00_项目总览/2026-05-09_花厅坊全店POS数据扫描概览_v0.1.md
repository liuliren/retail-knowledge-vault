# 花厅坊全店 POS 数据扫描概览 v0.1（118 文件 batch / B 任务）

> **本文件性质**：5/8 客户方提供 118 个 .xls 全部 batch scan + 休食调改主线品类 30 天 vs 60 天差值法 + 全店客流上升识别。
>
> **关键反幻觉发现** ⚠️：**全店 30 天日均（4/8-5/8）vs 60 天日均（2/8-4/8）普涨 +20-40%** → 方便食品报告 v0.5 的 +29.7% 销额提升**不全是调改驱动** / 大概率 60-70% 是全店客流整体上升 / 30-40% 是调改净增量。**这是对前两份报告结论的重要校准**。

## §2. ⚠️ 关键反幻觉发现：全店 30 天 vs 60 天日均普涨

### 2.1 全品类 30 天日均 vs 60 天日均对比

| 品类 | 30 天日均（元）| 60 天日均（元）| 变化 | 备注 |
|---|---|---|---|---|
| 蔬菜 | 13,922 | 10,870 | **+28.1%** | 客流主驱动 |
| 猪肉档 | 21,521 | 15,045 | **+43.0%** ⭐ | 显著上涨 |
| 生鲜（含水产+肉禽）| 69,243 | 49,340 | **+40.3%** ⭐ | 显著上涨 |
| 休闲 | 1,201 | 970 | **+23.8%** | 食品类涨 |
| 酒水 | 4,336 | 3,561 | **+21.8%** | 食品类涨 |
| 散称 | 3,574 | 3,560 | **+0.4%** | 几乎持平 / 例外 |
| 糕点 | 167 | 286 | **-41.4%** ⚠️ | **唯一显著下滑** |
| **方便食品**（已通读） | 404 | 358 | **+12.8%**（30 天均 vs 60 天均 / 含调改前 17 天）| 与全店一致 |
| **调味**（已通读）| 2,385 | 1,793 | **+33.0%** | 与全店一致 |

### 2.2 方便食品 +29.7% 销额提升的重新归因（[推断] 校准）

之前 [[2026-05-09_方便速食组分阶段分析报告_v0.1]] 把方便食品调改后 13 天 +29.7% 销额视为"调改驱动"。

**校准后**（基于全店扫描）：
- 全店 30 天日均普涨 +20-40% → 推断**全店客流整体上升**
- 方便食品调改后 +29.7% 中：
  - **60-70% 可能是全店客流溢出**（被动 + 大盘红利）
  - **30-40% 是调改净增量**（主动 + 商品+陈列调整）
- 但方便粉 +135% / 即食类 +210% 仍**大幅超过全店 +30% 平均** → **结构升级是真**（不只是大盘）

**[推断] 校准后真实调改净效益**：
- 方便食品组：销额净增量约 +10-12%（不是 +29.7%）
- 方便粉子类：净增量 +100-110%（仍然爆款 / 螺蛳粉真有效）
- 即食类子类：净增量 +180-200%（仍然结构升级）
- 方便面子类：净增量约 -20%（销额涨被全店客流抬 / 实际本子类可能下滑）

这是**重要的反幻觉校准**：单看品类内数据 / 看不到大盘 / 容易误判调改贡献。

### 2.3 调味 +33.0% 校准

调味 +33.0% 与全店平均 +20-40% 持平 / **大概率是全店客流溢出 / 调味本身可能没调改 / 也可能小幅调改**。

→ 这强化了之前调味组健康度扫描报告里 H2（全店客流溢出）的可能性。

### 2.4 糕点 -41.4% 警告 ⚠️

唯一**显著下滑**的品类。可能原因（[推断]）：
1. **季节性**（夏季是糕点淡季 / 月饼/年货等下架）
2. **结构调整**（淘汰太多 / 新品没补上）
3. **竞争**（外部烘焙店 / 即食粥替代）

→ 建议：W19 内糕点深入 / 看是否需要调改主线纳入。

## §5. 数据质量问题（全店 P1 级）

### 5.1 主供应商字段缺失

[推断] 全店 99%+ SKU 主供应商字段未填（与方便食品 99.5% 一致 / 调味推断同等）。
→ 影响表 A 字段 13 装填 / M-DEC-003 失依据。

### 5.2 ERP 进价不准（多品类）

牛肉 90.3% / 猪肉档 95.8% / 烘焙 85.5% 等品类毛利率虚高 = ERP 进价字段不准 / 推断"进价"字段是上次进货价 / 不是加权平均成本 / 或字段缺失被视作 0。
→ 影响所有毛利分析 / 必须 W19 内核对。

### 5.3 多个文件名 vs 实际类目错位

启明导出时类别筛选错（已识别 / 5/9 已沟通过）：
- 120 天 "方便食品 / 调味 / 茶叶 / 粮油" 4 文件**实际都是粮油**（挂面+食用油+火腿肠+米+面粉）
- 2025 年 "调味" 文件实际也是粮油
- 2025 年 "百货 = 家清个护 = 用品"等部分文件类目混叠

→ 建议启明 W19 内全店重导（按真实大类）+ 文件命名 SOP 建立。

| 品类 | 负库存 SKU 数（30 天）|

[推断] 生鲜全部"负库存" = ERP 设计 = 称重商品/无库存追踪（不是 ERP 错）。其他品类负库存才是 ERP 数据问题（约 15-50 个 SKU 影响）。
