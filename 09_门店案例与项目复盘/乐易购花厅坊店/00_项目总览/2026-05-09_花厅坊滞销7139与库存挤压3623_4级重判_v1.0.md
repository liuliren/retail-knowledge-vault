## §4. 滞销 + 挤压重叠分析

### 4.1 重叠估算

```
全店滞销 7,139 SKU + 全店挤压 3,623 SKU
[推断] 重叠 ~2,000-2,500 SKU（严重滞销 → 必然挤压）

仅滞销不挤压 = 7,139 - 重叠 ≈ 4,640-5,140 SKU
  - 含义：销售低 + 库存少 → 自然末位 / 进货意愿小 / 即使长尾也低风险
  
仅挤压不滞销 = 3,623 - 重叠 ≈ 1,120-1,620 SKU
  - 含义：销售正常但进货过多 / 短期爆款进货过头 / 控制进货节奏即可
  
重叠 = ~2,000-2,500 SKU
  - 含义：销售低 + 库存高 → 双重问题 / 必须立即处置
```

### 4.2 处置优先级（基于重叠）

```
🔴 P0 重叠（双重问题 / 必处置 / ~2,000-2,500 SKU）
  - 90% 进表 D（淘汰）/ 立即停补 / 促销清库 / 损耗核销
  - 10% 战略保留（如老板品牌形象款）

🟡 P1 仅滞销不挤压（~4,640-5,140 SKU）
  - 50% 进表 D（自然末位淘汰）
  - 30% 节令休眠（等季节）
  - 20% 战略保留

🟢 P2 仅挤压不滞销（~1,120-1,620 SKU）
  - 控制进货节奏 / 不淘汰 / 等爆款回归
```
