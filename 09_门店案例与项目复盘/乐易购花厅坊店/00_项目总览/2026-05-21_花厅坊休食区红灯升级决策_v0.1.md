## 4. 三选一方案

### 4.1 方案 X：范围保持 / 时间顺延

**内容**：
- 继续保持 8 货架 + 糕点 + 大包装全量范围
- 验收顺延 6/3（与 10 天总攻原计划一致 / 不变范围）
- 5/22-5/24 继续紧急补救资料（追加 3 天补救期）
- 6/3 前完成全量闭环

**优点**：
- 保持休食区完整性 / 不下调原定范围
- 后续可对外讲完整样板

**缺点**：
- ❌ 放弃 5/31 节点（如老板坚持 5 月底 → 不可选）
- ⚠️ 现场协调压力继续存在 / 启明 / 顾问 / 老板配合度仍是变量
- ⚠️ 可能挤压 6/1 酒水区启动（顺延到 6/4+）
- ⚠️ M2 一周效果验证 + M3 14 天 G05 全部顺延 / 整体节奏推后 3 天

**适用条件**：
- 老板**不强求 5 月底**节点 / 重视完整质量
- 5/22 起启明 + 顾问 + 老板配合度提升
- 不影响 6 月酒水区启动节奏（或老板接受酒水推到 6/4+）

### 4.2 方案 Y：范围缩减 / 5 月底保局部成果 ⭐ 推荐

**内容**：
- 5/31 前主攻 **G3-B 方便速食 + G4-A 冲调**两个子组
  - G3-B：v0.5（13 天）→ v0.6（24 天）复盘 + 固化 + 53 积压执行
  - G4-A：5/30 G05 触发条件准备 + 5/13 试点预案 v0.2 局部调改
- **G1-A / G1-B / G2-A / G2-B / G3-A / G4-B** 顺延到 6/1 后（与酒水 / 调味同期推进）
- 糕点子组（5/10 已调改 / 5/13-5/17 观察）只做登记复盘 / 不做硬闭环
- 大包装区（M-DEC-002 候选）只做候选状态登记 / 不擅升 v1.0

**优点**：
- ✅ **能保住 5/31 可交付成果**（2 子组样板 / 不空仓）
- ✅ **基于已有资料推进 / 反幻觉风险低**（G3-B 53 张照片 + v0.5 + 9 P0 数据 / G4-A 62 张照片 + 5/13 试点预案）
- ✅ **可形成"方便速食 + 冲调"双样板**（5/9 v0.5 + 5/30 G05 G4-A）
- ✅ **不影响 6/1 酒水区启动准备**
- ✅ **6/1+ 接续 G1/G2/G3-A/G4-B + 酒水 + 调味 / 时间窗口仍合理**

**缺点**：
- ⚠️ 休食区全量闭环目标下调（8 → 2 货架本月底 / 其余顺延）
- ⚠️ G1/G2/G3-A/G4-B 顺延 / 6 月初配合酒水 / 工作量集中
- ⚠️ 需向老板说明范围调整原因（"卡资料不卡方案"）

**适用条件**：
- 老板**希望 5/31 必须有成果**
- 当前资料无法快速补齐（启明 / 顾问 / 老板配合不确定）
- 现场资源有限

### 4.3 方案 Z：整体顺延 6/3 / 不缩范围

**内容**：
- 不缩减范围（8 货架 + 糕点 + 大包装）
- 整体顺延到 6/3（与 10 天总攻原计划 5/27 + 7 天验收 = 6/3 一致）
- 5/22-5/24 继续补资料（追加 3 天补救期）
- 6/3 前完成全量闭环

**优点**：
- 质量较稳 / 保留完整休食区闭环

**缺点**：
- ❌ 放弃 5/31 目标
- ⚠️ 可能挤压 6/1 酒水区启动
- ⚠️ 项目节奏变慢

**适用条件**：
- 老板同意顺延
- 启明 + 顾问 + 老板 5/24 前能补齐

**与 X 区别**：
- X 是"自然顺延 6/3"（不主动设新目标）
- Z 是"主动设 6/3 为新目标"（更明确）
- 实际两者高度相似 / 区别仅在沟通口径
