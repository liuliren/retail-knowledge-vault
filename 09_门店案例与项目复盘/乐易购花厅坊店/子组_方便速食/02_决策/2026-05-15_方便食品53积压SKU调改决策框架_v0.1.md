# 方便食品 53 积压 SKU 调改决策框架 v0.1

## 1. 文件定位

本文件是 5/15 用户选路径 A 后 / 基于 `库存积压报表_方便食品_20260508.xls`（53 SKU / 25 字段）建立的调改决策 SOP。

**目的**：让六哥 / 店长能用 4 字段判据 / 半天内对 53 个积压 SKU 做出"淘汰 / 保留 / 降排面 / 换位置"判定 / 不依赖全自动数据底座（BUS-DATA-008）。

**本文件不包含**：
- 任何 SKU 名 / 品牌名 / 条码 / 真实库存数 / 销售金额
- 方便速食 v0.6 结论
- 客户原话

---

## 2. 适用范围

| 范围项 | 当前值 |
|---|---|
| 品类 | 方便食品（0206% / 待启明 sign off ⑥ 确认是否等同方便速食原口径） |
| SKU 数 | 53（积压子集 / 非全方便食品） |
| 时段 | 2026-04-08 ~ 2026-05-08（30 天 / 由报表打印时间倒推） |
| 数据状态 | C 级（含库存量但 0206% 口径未 confirm / 单一时段无对比） |

**不适用**：
- 全方便食品（非积压 SKU 部分 / 需启明追加拉数）
- 跨品类（休闲 / 冲调 / 调味等）
- 季节性品类（不在本表）

## 3. 决策核心 4 字段

从 25 字段中选出**最有决策力**的 4 个：

| #     | 字段名      | 列号     | 决策含义                         | 反幻觉提示                   |
| ----- | -------- | ------ | ---------------------------- | ----------------------- |
| **A** | 最近销售日期   | col 18 | **动销活性**指标 / 距今天数 = 死货程度     | 不能只看一个日期 / 要结合期间销量      |
| **B** | 期间销量     | col 12 | **30 天销售强度** / 0 或极低 = 强淘汰候选 | 销量 0 ≠ 一定淘汰 / 可能是新品在引入期 |
| **C** | 当前库存     | col 11 | **积压量** / 数字越大风险越大           | 大库存 + 健康动销 = 不算积压       |
| **D** | 库存售价合计金额 | col 33 | **占用资金** / 决定淘汰紧迫性           | 同等积压程度下 / 占用大的优先处理      |

**辅助字段（次要 / 决策时参考）**：
- 引进日期（col 27）— 新品 ≠ 死货
- 规格（col 28）— 大包装 vs 小包装影响淘汰判定
- 品牌（col 35）— 主力品牌 vs 长尾品牌
- 供应商（col 30）— 同供应商可批量处理

## 4. 四类动作判据

### A 类 — 淘汰（Delete）

**触发**（任一即可 / 但 P1 P2 同时命中更稳）：
- P1: 最近销售日期 ≥ 60 天前
- P2: 期间销量 = 0 且 引进日期 ≥ 90 天前
- P3: 库存售价合计 ≥ ¥500 + 期间销量 < 5 件

**动作**：
- 退仓 / 退供应商 / 清仓特价 / 下架
- 排面释放（释放数量 = 该 SKU 当前排面数）

**反幻觉边界**：
- 不能仅凭"最近销售日期 60 天"就淘汰 / 必须**同时**看：
  - 是否在淡季？（如方便食品冬季为旺季）
  - 是否近期断货？（断货 ≠ 滞销）
  - 是否在引入期？（< 90 天的新品给观察期）

---

### B 类 — 保留观察（Hold）

**触发**：
- 最近销售日期 ≤ 30 天 + 期间销量 ≥ 5 件 + 库存售价合计 < ¥500
- 或：新品（引进日期 < 90 天）

**动作**：
- 维持现状
- 标记"观察 30 天" / 30 天后重新判定

---

### C 类 — 降排面（Reduce）

**触发**：
- 最近销售日期 ≤ 30 天 + 期间销量 < 5 件 + 当前库存 ≥ 30 件
- 或：销售平稳但库存堆积 / 资金占用大

**动作**：
- 排面数 -50%（如原 2 排面 → 1 排面）
- 释放排面给 A 类淘汰后的位置 / 或给新品

---

### D 类 — 换位置（Relocate）

**触发**：
- 最近销售日期 ≤ 15 天 + 期间销量 ≥ 10 件 + 当前排面在边缘货架/低层
- 或：动销健康但摆错品类（如方便面摆到了米面区）

**动作**：
- 移到黄金货架 / 主通道
- 排面数 +50%（如原 1 排面 → 1-2 排面）

## 5. 判定 SOP（一步一步）

```
Step 1: 打开 库存积压报表_方便食品_20260508.xls
Step 2: 把 25 字段中的 4 核心字段（A/B/C/D）+ 4 辅助字段
        Copy 到新表 / 减少视觉噪音
Step 3: 按"最近销售日期"降序排序（最旧的在最上）
Step 4: 从上往下逐行判定 / 用本文件 §4 四类动作判据
Step 5: 每行打标签 / 写一行判定理由（10 字内）
Step 6: 检查 4 类动作分布
        - 健康分布: A 10-15% / B 50-60% / C 20-25% / D 5-10%
        - 异常: A > 30% → 检查阈值是否过严 / 或品类整体危机
        - 异常: A = 0 → 检查阈值是否过松 / 或数据不全
Step 7: 把判定表导出 / 拿到现场 walk through 一遍
Step 8: 与店长 sign off / 把 A 类淘汰单 给供应商沟通退/换
```

**单 SKU 判定时间**：熟练后 30-60 秒 / 53 SKU 约 0.5-1 小时

## 7. 反幻觉规则（强制）

按 [[CLAUDE.md]] §13 + §13.16-20：

1. **不能只凭一个数字判定** / 4 字段必须**联合判断**
2. **不能把"积压报表"等同于"全方便食品库存"** / 53 SKU 是子集 / 还有大量正常 SKU 不在表内
3. **不能跳过启明 sign off ⑥** / 0206% 是否等于方便速食原口径未 confirm / 影响品类边界
4. **不能假设阈值放之四海** / §4 中的天数 / 件数 / 金额阈值是**起步参考** / 真实店要根据：
   - 品类季节性（方便面冬季旺）
   - 客群（学校店 vs 社区店 / 销售节奏差很大）
   - 门店历史动销基线
5. **判定理由必须可追溯** / 不能写"感觉差" / 要写"60 天 0 销 / 库存 30 件"
6. **A 类淘汰前必须现场看一眼** / 防止报表数据滞后 / 比如断货导致销量 0 但其实是抢手货
