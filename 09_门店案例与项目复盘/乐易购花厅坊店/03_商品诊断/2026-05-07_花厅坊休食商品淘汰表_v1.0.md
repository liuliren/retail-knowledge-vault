# 花厅坊休食商品淘汰表 v1.0（淘汰执行 + 已下架 SKU 归档 + 淘汰复盘）

> **定位**：在售 SKU **淘汰执行 + 历史归档** 主表；与表 B 决策"淘汰"动作 sign off 后联动写入；与引进表 v0.1 形成"引进 → 在售 → 淘汰"全生命周期闭环。
>
> **设计依据**：
> - [[单品管理]] §10 滞销淘汰机制 + §11 候选商品池补位
> - [[M-DEC-007]] §3 S5 动作"淘汰" + §4 反幻觉红线
> - [[CLAUDE.md]] §13.13 SKU 淘汰决策必须六维综合 / §13.10 必须有依据链
>
> **使用方式**：vault `.md` 锁字段定义；本地 Excel 累积淘汰记录（永不删除 / 是淘汰复盘的依据）。
>
> **更新频率**：实时（每次淘汰决策 sign off 后写入）/ 周更（24h / 1 周观察填回）/ 月度复盘（lessons_learned 提炼）。
>
> **核心铁律**：本表 supersede [[2026-05-07_休食SKU去留表_v0.1]] §3 "淘汰"动作记录段；旧版已 mv → `99_归档/2026-05-07_supersede_by_v1.0/`。

## §1. 表与全生命周期闭环

```
[新引进表 v0.1] (候选池 / pre-在售)
       ↓ G-1 至 G-5 引进规则 + sign off
[表 A 商品基础表 v1.0] (sku_status='在售')
       ↓ M-DEC-007 §3 S2-S5 决策
[表 B 商品动销分析及跟进表 v1.0]
       ↓ decision_action='淘汰' + sign off (字段 33-35)
       ↓ 触发写入
[表 D 商品淘汰表 v1.0] (本表) ⭐
       ↓ 同步更新
[表 A 商品基础表 v1.0] (sku_status='历史归档' / 不删除)
```

> **铁律**：表 A 的 SKU `sku_status = '历史归档'` 后 **永不删除** — 是表 D 复盘的依据（[[CLAUDE.md]] §16.3 禁止删除重要资料）。

## §3. 淘汰原因分类（字段 4 / [[M-DEC-007]] §3 S4 归因决策树映射）

| `discontinue_reason` | 触发依据 | 复盘关注 |
|---|---|---|
| **真不需要** | M-DEC-007 §S4 决策树终点 / 6 维全部低 + 客群不匹配 | 是否选品时已可避免？|
| **客群不匹配** | snapshot_attribution_tag = "客群不匹配" / TPOS / CDT 不符 | 选品时门店画像评估是否到位？|
| **价格虚高** | 字段 16 retail_price > 市价 / 调价无效 | 供应链 / 进价是否有问题？|
| **活动期外低** | 仅活动期销，自然销量 = 0 / 配活动无效 | 是否本质就是促销品？|
| **缺货长期** | stockout_rate > 阈值 / 供货中断 ≥ 30 天 | 供应商替换可行？|
| **供货中断** | 供应商停供 / 进口商品断货 | 替代供应商？|
| **其他** | 兜底 / 必须在 lessons_learned 详述 | — |

## §4. 库存处置策略（字段 15 / 老板 sign off 必填）

| `inventory_disposal` | 适用场景 | 处置成本 |
|---|---|---|
| **清仓** | 库存仍可销售 / 折扣促销 | 让利 10-50% |
| **退货** | 供应商接受退货 / 临期商品 | 物流费 + 部分扣款 |
| **销毁** | 过期 / 损坏 / 不可销售 | 全损 |
| **让利赠品** | 满额送 / 客户福利 | 进价成本 |
| **拼盘搭售** | 与高动销 SKU 搭配 | 部分让利 |
| **其他** | 必须在备注详述 | — |

## §6. 与其他表的联动

```
表 B 商品动销分析及跟进表
   ├── 字段 29 decision_action='淘汰' + 字段 33 sign_off_required=Y
   └── 字段 34-35 sign_off_date / sign_off_evidence
        ↓ 触发
表 D 商品淘汰表（本表）
   ├── 字段 5-12 快照 from 表 B 同步冻结
   ├── 字段 13 decision_date = 表 B 字段 34
   └── 同步更新表 A
        ↓
表 A 商品基础表
   └── 字段 14 sku_status: '在售' → '历史归档'
   └── 字段 16 delisted_date 更新

引进表 v0.1
   ├── 候选 SKU sign off → 表 A 新增 sku_status='在售'
   └── 与表 D 字段 18 replacement_barcode 联动（淘汰释放排面 → 引进补位）
```

## §7. 数据装载工作流

```
S1 表 B sign off (淘汰决策)
   → decision_action='淘汰' + sign_off_evidence 完成
   ↓
S2 表 D 写入 1 行
   → 字段 1-12 from 表 B 自动复制（snapshot 冻结）
   → 字段 13 = 表 B sign_off_date
   ↓
S3 现场执行下架
   → 字段 14 delisted_date / 15 inventory_disposal / 16 amount / 17 facing
   ↓
S4 替换 SKU 决策（如有）
   → 字段 18-19（联动引进表）
   ↓
S5 同步更新表 A
   → sku_status='历史归档' + delisted_date 更新
   ↓
S6 24h / 1w 观察
   → 字段 22-23 文字观察
   ↓
S7 月度复盘
   → 字段 24 lessons_learned 提炼 + 喂回单品管理 §16 误区库
```
