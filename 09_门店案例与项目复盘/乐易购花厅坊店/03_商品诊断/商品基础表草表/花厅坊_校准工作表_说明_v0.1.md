# 花厅坊 校准工作表 · 使用说明 v0.1

> **一句话**：机器已把"铺台子"的 90% 做完——按品类把高价值 SKU 排好、在每个 SKU 旁放了一条**带 `?` 的机器弱建议**。六哥只需**一类一类扫过去，对的打 `✓`、不对的改一两个字**，不必再面对 2372 行裸 CSV。判断相（10%）由六哥独占。

> 脱敏声明：本说明 **0 条码 / 0 进价裸值 / 0 SKU 明细**，只给行数与方法。两张工作表 CSV 含品名 → `client_confidential`，落在本目录（`**/*.csv` 已 gitignore，不入库、不上云）。

## 2. 怎么用（打 ✓ / 改字）

两张表都留了一列 **`六哥裁定`（空白待填）**，紧挨着 **`机器弱建议…`** 列。操作只有两种：

1. **机器建议对** → 在 `六哥裁定` 填 `✓`（或复制机器那条值）。
2. **机器建议错/要调** → 在 `六哥裁定` 直接写最终值（覆盖机器建议，机器那列不用动）。

### 表①（角色）建议值词表
`引流` / `利润` / `形象` / `季节` / `补充观察`（与机器弱建议同口径，去掉 `?` 即可）。

### 表②（L4）填法
机器在 `机器弱建议L4决策组` 给了一个**聚类标签**（格式：`细分品类·价格带·品牌`，如 `啤酒·低价位·品牌待定`）。同一聚类的 SKU 已**排在相邻行**，六哥可**整段确认**：在该段每行 `六哥裁定` 写最终 L4 名（如把一段啤酒统一裁成 `国产工业啤酒` / `精酿`），或对个别行单独改。

> 排序规则：表①按**品类分组**（同品类连续成段，A 在 B 前，组内按库存降序；无 L3 的归 `(待归类)` 段，集中在表首）；表②按**宏品类 → 细分品类 → 价格带(高→低) → 品牌**聚类，方便一段一段过。

## 3. 机器弱建议的置信度边界（必须知道：这些都是"弱建议·待六哥裁定"，不是定论）

### 表① 角色——判定优先级（命中即停，机器只猜一条）
| 顺序 | 规则 | 给的弱建议 | 置信度边界 |
|---|---|---|---|
| 1 | 品名含强季节词（月饼/粽/中秋/年货/腊肠…）| `季节候选?` | 名义命中，**极保守**（仅 1 条命中）；"情人梅""初恋情人"等是商品名非节令，已排除 |
| 2 | A 类 + 非滞销 + 动销天数 ≥ 8 | `引流候选?` | 头部走量品的合理候选；动销天数受明细 join 覆盖率限制（约 37%），**缺明细的 A 类可能漏判** |
| 3 | 毛利率 ≥ 30% | `利润候选?` | **置信度最低**——进价不准（N1 坑⑤），毛利率全列 `仅参考`；这条最该被六哥推翻 |
| 4 | 价格带 = 高价位 | `形象候选?` | 仅按价格带粗判，未看陈列/客群定位 |
| 5 | 其余（含低动销有货、数据缺失）| `补充观察?` | 兜底，**最该被六哥升级**为更具体角色 |

> 所有值带 `?` = 机器弱建议；角色=经营判断，**机器从不替六哥下结论**。这是护城河红线。

### 表② L4 决策组
机器**只按结构字段（价格带×品牌×规格）做聚类预排**，给的是"候选分组锚点"，**不是 L4 终名**。品牌列约 2/3 为 `(品牌待定)`（长尾/无品牌前缀品），规格列部分 `(缺规格)`——这些不影响六哥按品名+价格带做 L4 切分。L4 切分维度（如白酒按香型/价位、啤酒按工业/精酿）是判断相，**不可下放**。

## 5. 填完回流路径（供后续 enrich 进商品基础表）

1. 六哥在两张 CSV 的 `六哥裁定` 列填完（✓ / 改字），存回**本目录**（仍 client_confidential）。
2. 下一轮脚本读 `六哥裁定`：
   - 表① → 回填商品基础表 `business_role`（由 `business_role_weak` 升级为已裁定角色），并清 `待校准` 里的「角色待定」。
   - 表② → 回填 `category_l4`（由 `需L4决策组` 升级为已裁定 L4），并清「争议类目(需L4决策组)」。
3. 六哥反复推翻的规则 → 反哺机器（如某价格带阈值、某毛利门槛），M-DEC 越跑越准（下一轮弱建议命中率提升）。
4. 回填后重出商品基础表 v0.3，角色/L4 两项覆盖率从机器弱建议升为**六哥签字的事实层**。

## 6. 机制 B 复盘（铺台子省了多少 / 边界 / 可改进）

- **机器铺了多少台子**：把 2372 行裸表收敛为 **841 + 268 = 1109 个真正需判断的高价值 SKU**，并**按品类预排 + 旁置弱建议**——六哥的动作从"逐行读 CSV、自己想角色"降为"扫一眼、打✓或改字"，判断相的输入成本压到最低。这正是 90/10 里"把可铺的台子全铺掉"。
- **机器为何只给弱建议、不下结论**：角色/L4 = 经营判断（驭马人层），机器下结论 = 越红线、污染基础表（铁律③）。故一律带 `?`、留 `六哥裁定` 空列。
- **数据边界**：① 进价不准 → `利润候选?` 置信度最低，毛利率仅参考；② 动销明细 join 覆盖约 37% → 缺明细的 A 类可能漏判"引流"；③ 季节判定纯靠品名强词，极保守（仅 1 命中），节令品多在 C 类长尾，本表不深挖；④ 表② 品牌 2/3 待定，不影响 L4 切分但提示品牌库仍可反哺。
- **可改进点**：① 动销维度补当期明细后重判"引流候选"（酒水/散装尤其缺明细）；② 毛利门槛待六哥给一次反馈后校准（现固定 30%）；③ `(待归类)` A/B 段（无 L3）可在角色裁定同时顺手补 L3；④ 六哥裁定回流后，把高频"机器猜错→六哥改对"的模式固化成规则，下一轮弱建议命中率应可观测上升（M-DEC 回路）。

> 复用链：N2 商品基础表草表 v0.1（含 `abc_class`/`category_l4=='需L4决策组'`/`business_role_weak`/`price_band` 等字段）→ 本派生脚本（纯只读草表，按规则排序 + 生成弱建议 + 留空裁定列）→ 两张 client_confidential 工作表。**草表一字未改。**
