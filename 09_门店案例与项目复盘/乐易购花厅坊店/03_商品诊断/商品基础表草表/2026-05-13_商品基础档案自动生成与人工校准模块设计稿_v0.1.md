# 2026-05-13_商品基础档案自动生成与人工校准模块设计稿_v0.1

## 1. 文件定位

- 本文件是**"商品基础档案自动生成与人工校准模块"**的设计稿。
- 用于**诊断阶段前置任务**。
- 目标：通过多源报表**自动生成**商品基础档案候选表 + 由人工**校准**形成终版。
- **不直接生成**真实商品数据。
- **不输出**商品调整结论。
- **不替代**现场调研和人工确认。

---

## 2. 模块建设背景

| # | 当前问题 | 表现 | 影响 |
|---|---|---|---|
| 1 | 人工整理商品基础档案效率低 | 5 v1.0 表（基础 / 动销 / 品类 / 淘汰 / 引进）按"休食"专项 / 全店级未统一 / 全店 8000+ SKU 人工拉表不可持续 | 治理债 + 跨子组口径不一致 |
| 2 | 多源报表分散 | 商品清单 / 销售报表 / 每日销售数据 / 库存表 / 供应商 / 进价售价 / 现有品类结构 各自独立 | 合并难 / 重复劳动 |
| 3 | 直接进入动销分析易因字段缺失导致分析失真 | 字段缺失 / 品类错配 / 供应商缺失 / 进价异常 / 等 | 跨子组分析不可靠 |
| 4 | 缺前置"自动生成 + 人工校准"两步骤 | 当前直接进入人工 / 没有 algo + human-in-loop 闭环 | W22+ 跨客户复用难 |

**触发**：诊断阶段需要**前置基础设施** / 形成"算法做粗筛 / 人工做精筛"的标准模式。

## 3. 模块目标

| # | 目标 | 输出 |
|---|---|---|
| 1 | 从多源报表**自动生成**商品基础档案候选表 | 候选表 |
| 2 | 建立**条码维度**商品主数据 | 主键唯一 / 8000+ SKU |
| 3 | 自动合并商品清单 / 销售明细 / 库存 / 供应商 / 进价售价字段 | 多表 inner / outer join |
| 4 | 自动匹配 V4.0 L1/L2/L3/L4 品类结构 | 算法 + 置信度评分 |
| 5 | 识别字段**缺失 / 重复 / 冲突 / 异常** | 异常清单 |
| 6 | 输出**人工标注与修订**入口 | 人工校准模板 |
| 7 | 形成商品基础档案**终版** | 终版主档 |
| 8 | 支撑后续动销分析 / 淘汰候选 / 商品结构表 / 诊断报告 / BI 流程 | 唯一主档来源 |

## 4. 输入数据

| 输入源 | 主要字段 | 用途 | 是否必需 | 备注 |
|---|---|---|---|---|
| **商品清单** | 条码 / 商品名 / 规格 / 售价 / 进价 / 供应商 / 系统品类 | 建主档 | ✅ 必需 | 供应商和进价已确认可导出（5/10 六哥 / fact_layer: client_told）/ **生鲜类进价例外** |
| **销售报表** | 条码 / 销量 / 销售额 / 毛利额 / 日期 | 动销指标 | ✅ 必需 | 建议至少 30-90 天 |
| **每日销售数据** | 日期 / 条码 / 销量 / 销售额 / 毛利额 | 时间趋势 | ⚠️ 建议 | 支撑 7 / 15 / 30 天复盘 |
| **库存表** | 条码 / 库存数量 / 库存金额 | 周转与异常 | ✅ 必需 | 识别负库存 / 高库存 / 缺货 |
| **品类结构表** | L1/L2/L3/L4 | 标准品类匹配 | ✅ 必需 | 对接 V4.0 大湾区品类表（[[2026-05-09_品类表治理决议_v1.0]]）|
| **供应商表** | 条码 / 供应商 / 供货方式 | 供应商分析 | ⚠️ 建议 | 若商品清单已含供应商 / 可合并 |
| **现场照片索引** | G货架 / L主品类 / 层位 / 照片 | 现场校验 | ⚠️ 建议 | 后续关联陈列位置（参 BUS-DATA-002 §8 + 003 §4）|

**5/12 启明协调拉数验证清单**（参 [[2026-05-10_花厅坊_供应商与进价字段口径confirm_v0.1]] §4.2）：
- 供应商填充率 ≥ 95% / ≥ 70% 可用 / < 70% 异常
- 进价填充率（排除生鲜后）同上
- 库存金额校验 = 进价 × 库存数量
- 90 天销售时段覆盖
- 活动标记字段是否存在 + 填充率

## 5. 自动生成流程（13 步）

```
1.  原始报表收集（商品清单 / 销售报表 / 每日数据 / 库存 / 供应商 / 品类结构 / 照片索引）
                    ↓
2.  字段识别（自动识别原始字段名 / 中英文映射 / 别名匹配）
                    ↓
3.  字段映射（→ BUS-DATA-002 字段模板 24+12+9+10+8+12+16+13+13 = 117 字段）
                    ↓
4.  条码标准化（去前导零 / 去空格 / 去非数字字符）
                    ↓
5.  商品名称清洗（去空格 / 去全半角差异 / 去无关后缀）
                    ↓
6.  多表合并（按条码 inner / outer join）
                    ↓
7.  去重（按条码 / 保留最新或最完整记录）
                    ↓
8.  价格 / 毛利校验（毛利率 = (售价-进价)/售价 / 异常标记）
                    ↓
9.  供应商匹配（如可获得 / 客户主供应商字段直读 / 模糊匹配回写）
                    ↓
10. 品类 L1-L4 匹配（系统品类 → V4.0 / 算法 + 置信度）
                    ↓
11. 库存异常识别（负库存 / 高库存 / 缺货）
                    ↓
12. 销售数据聚合（7 / 30 / 90 天 / ABC-Z 分类）
                    ↓
13. 生成「商品基础档案候选表」+「异常与待核验清单」
```

## 7. 自动识别异常规则（14 条）

| # | 异常类型 | 触发条件 | 处置 |
|---|---|---|---|
| 1 | 缺条码 | barcode 为空或非数字 | 进异常清单 / 散称 / 临时 |
| 2 | 重复条码 | 同条码多记录 | 合并保留最新 / 或单独 |
| 3 | 商品名冲突 | 同条码不同名 | 优先用最新 / 标 dq_name_conflict |
| 4 | 售价缺失 | price 为空或 ≤ 0 | 标 dq_price_anomaly |
| 5 | 进价缺失 | cost 为空 | **本次导出缺失**（非系统能力）/ 生鲜单独标记 |
| 6 | 毛利率无法回算 | 售价 / 进价 任一缺失 | 标 dq_margin_anomaly |
| 7 | 供应商缺失 | supplier 为空 | **本次导出缺失**（非系统能力）|
| 8 | 系统品类缺失 | sys_category 为空 | 标 dq_category_missing |
| 9 | 标准 L1-L4 无法匹配 | 算法置信度 < 0.5 | 进人工校准 / 标 待核验原因 |
| 10 | 负库存 | stock_qty < 0 | 单独处理 / 不参与正常周转 |
| 11 | 高库存低动销 | 库存周转 ≥ 阈值 + 90 天销量 ≤ 阈值 | 候选淘汰（不立即出 SKU 清单）|
| 12 | 销售有记录但商品档案缺失 | 销售表 join 商品档案 left only | 进异常清单 |
| 13 | 商品档案有记录但销售长期为零 | 商品档案 join 销售 left only / 90 天 0 销量 | 候选淘汰 |
| 14 | **生鲜类进价缺失** | 系统品类含生鲜 + cost 为空 | **单独标记**（不混入非生鲜诊断主线）|

## 8. 人工校准流程

> **核心原则**：自动生成候选表**不能直接作为终版** / 必须人工确认。

**人工校准 10 动作**：

| # | 动作 | 责任 |
|---|---|---|
| 1 | 确认条码是否正确 | 校准人 |
| 2 | 确认商品名称是否标准 | 校准人 |
| 3 | 确认规格 / 单位 | 校准人 |
| 4 | 确认系统品类与标准 L1-L4 | 校准人 + 品类负责人 |
| 5 | 确认供应商 | 校准人 + 启明（可疑时）|
| 6 | 确认进价 / 售价异常 | 校准人 + 启明（可疑时）|
| 7 | 确认是否在售 | 校准人 |
| 8 | 确认是否属于休食 / 冲调 / 方便速食 / 糕点 / 调味 / 大包装等子组 | 校准人 + 子组责任人 |
| 9 | 确认是否需要现场照片验证 | 校准人 + 现场责任人（佳妮 / 高彬 / 张力）|
| 10 | 确认待核验字段 | 校准人 |

## 10. 终版商品基础档案形成规则（7 条硬约束）

| # | 规则 | 含义 |
|---|---|---|
| 1 | **只有人工确认后的数据才能进入终版** | 候选表 → 人工 → 终版 / 不允许跳过 |
| 2 | **终版表必须保留来源字段和修订记录** | audit-log 完整 / 可回溯 |
| 3 | **待核验字段不得写成确定结论** | is_pending_check = true 时 / 进异常清单 / 不参与分析 |
| 4 | **终版商品基础档案作为后续唯一主档来源** | 动销分析 / 淘汰候选 / 商品结构表 / 诊断报告 / BI 流程 全部基于终版 |
| 5 | **新数据导入时先更新候选表** | 不直接改终版 |
| 6 | **候选表 → 终版需新一轮人工确认** | 不允许"自动覆盖" |
| 7 | **终版升版必须独立 commit** | 不与业务报告混提交 |
