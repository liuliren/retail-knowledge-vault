# 花厅坊 商品基础表 · 覆盖与待校准报告 v0.2

> **结论先行**：本轮做的是"再多 90% 的机械活"——**只扩两张映射/别名表（additive，不动 build/clean 核心）**，重跑后把 **标准 L3 覆盖率从 36.6% 拉到 81.1%**、**品牌识别率从 7.0% 拉到 32.1%**，规格保持 60.2%。**全集 2372 SKU**，机器已把可批量化的类目/品牌做掉，**剩余真·待人工校准已收敛到 448 个 SKU 的泛/杂类目 + 274 个复杂类 L4 + 角色裁定**——这些是机器无法替你判定、必须六哥签字的 10%。

> 脱敏声明：本报告 **0 条码 / 0 进价裸值**，只给汇总与百分比；裸值仍只在 client_confidential 草表 CSV（该目录 *.csv 已 gitignore）。进价不准 → 毛利率/积压成本/DOS 一律标 **仅参考**。

## 3. 剩余真·待人工校准清单（按类型聚合 · 不逐条堆）

### 3.2 复杂类 L4 决策组 — 274 SKU
落在复杂类（巧克力/酒水/冲调/果干蜜饯）的 SKU 已标 `需L4决策组`，需你定 L4 分层（价格带 × 品牌密度）。**这是判断相，不可下放。**

### 3.3 品牌确认 — 1610 SKU 仍 `品牌待定`（其中 A/B 类优先）
品牌库已从 ~22 拉到 ~104 条，命中 762。剩余多为长尾小品牌/无品牌前缀品。建议**只补 A/B 类**里的高频未命中品牌（持续反哺 brand_alias，M-DEC 越跑越全），C 类长尾可批量略过。

### 3.4 角色裁定 — 全集 2372 标 `角色待定`（核心 10%）
引流/利润/形象/补充/季节 = 经营判断，机器只在 `business_role_weak` 列给了带 `?` 的弱建议。建议**只对 A/B 类（841 个）**做角色裁定，C 类暂置。**这是护城河红线，不可委托。**

## 4. 数据边界与一处需你拍板的设计点

- **L3 命名双轨（需你裁决是否统一）**：休食 8 类沿用旧"snack schema"名（膨化薯片/饼干糕点/糖果果冻…），本轮新增非休食用的是 **V4.0 大湾区唯一标准** L3 名。两套命名在同一 `category_l3` 列**并存但不冲突**（覆盖不同大类）。若要全列统一到 V4.0，需再做一次休食类名的对齐（属下一轮，本轮按 additive 红线未动）。
- **精确映射可能"以源类目盖过品名推断"**：如「果汁/茶饮/雪糕」原由品名关键词粗猜，现由源类目精确归位——属**纠偏（更准）**，非回退。
- 进价不准（N1 坑⑤）：毛利率/积压成本/DOS 全部 `仅参考`。
- 酒水/散装无 03-06 明细 → 只入库存维度，动销待补当期明细。

## 5. 机制 B 复盘（提升多少 / 还剩多少真要人工 / 可改进点）

- **L3/品牌提升多少**：L3 **+44.5pt（36.6→81.1%）**、品牌 **+25.1pt（7.0→32.1%）**，全靠"扩两张表+重跑"，零改核心逻辑、零人工逐条标注——这正是 90/10 里"再多做 90%"的机械相打法。
- **还剩多少真要人工**：① 泛/杂类目 448 SKU（其中约 178 是"定个切分规则即可批量"的，约 175 是 V4 暂无对应类、约 95 需逐类一锤）；② 复杂类 L4 274；③ A/B 类品牌补漏与角色裁定。**真·不可委托的判断相已收敛清晰。**
- **机器为何不再往上冲**：剩余未映射多是"一个源类目对多个 L3"（文具/调味/玩具）或"V4 无干净对应"（其他用品/鞋类/饰品）——再自动猜 = 污染基础表，违背铁律③，故一律留 `待校准·待六哥确认类目`。
- **可改进点**：① 文具/调味/玩具可在 `category_mapper` 加"源类目+品名关键词"二级规则（如 文具+「笔」→笔类）再榨一轮，但需你先给切分维度；② A/B 类未命中品牌继续反哺 `brand_alias`（M-DEC 回路）；③ 与六哥确认是否把休食 L3 名统一到 V4.0，消除命名双轨；④ 补酒水/散装当期明细，提升动销 join（当前 37%）。

> 复用链：N1 `retail_clean`（读/清洗/join）→ `category_mapper`（L3/L4，本轮扩 `RAW_CAT_TO_L3`+`RAW_CAT_FALLBACK`）→ `sku_extractor`（品牌/规格，本轮扩 `brand_alias`）→ N2 `build_master.py`（编译草表，未改）。
