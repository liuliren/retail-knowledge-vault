# 2026-05-13_花厅坊商品数据底座与动销分析模块设计稿_v0.1

## 1. 文件定位

- 本文件是**花厅坊商品数据底座与动销分析模块**的设计稿。
- **不直接生成**商品调改结论。
- **不直接生成** SKU 淘汰 / 引进清单。
- 用于定义后续：① 数据清洗 ② 分析报表 ③ 商品基础档案 ④ 诊断报告 ⑤ BI 流程。
- 本模块是**方便速食 / 冲调 / 糕点 / 调味 / 大包装**等子组调改的**上游数据底座**。

## 2. 模块建设背景

> **🔵 audit log（5/10 W20-Day1 / B+ 修正）**：六哥 5/10 confirm 供应商 + 进价字段**系统可导出**（fact_layer: client_told）/ 生鲜类进价未入系统（非生鲜诊断主线不受影响）/ 之前"99.5% 缺失"判断 = 5/8 客户方 30T 食品 POS 旧导出（部分字段未拉）/ **不应再作为系统能力判断**。详参 [[2026-05-10_花厅坊_供应商与进价字段口径confirm_v0.1]]。
>
> **5/12 实际拉数待确认（B+ 边界）**：
> - 5/12 实际拉数是否包含**供应商**字段
> - 5/12 实际拉数是否包含**进价**字段
> - **生鲜类进价缺失**如何单独处理（不混入非生鲜诊断主线）
> - 本次导出字段**完整性**（≥ 95% 算 OK / ≥ 70% 可用 / < 70% 异常）

| # | 当前问题 | 表现 | 影响 |
|---|---|---|---|
| 1 | 商品基础档案尚未形成最终版 | 现有 5 个 v1.0 表（基础 / 动销 / 品类 / 淘汰 / 引进）按"休食"专项 / 全店级未统一 | 跨子组分析口径不一致 |

| 3 | 系统导出数据未形成稳定清洗流程 | 5/8 / 5/9 客户方 30T 食品 POS / 临时清洗 / 无 SOP | 每次重做 / 不可复用 |
| 4 | 动销 / 淘汰 / 结构 / 诊断报告未模板化 | v1.0 5 表是花厅坊休食专项 / 模板化层级不足 / 跨客户复用难 | 跨子组 + 跨客户重复造轮子 |

## 5. 数据清洗流程

```
原始导出（客户 POS / Excel / CSV）
    ↓
字段标准化（中英文字段名映射 / 数据类型转换）
    ↓
条码清洗（去前导零 / 去空格 / 去非数字字符）
    ↓
商品名清洗（去空格 / 去全半角差异 / 去无关后缀）
    ↓
去重（按条码去重 / 保留最新销售记录）
    ↓
异常值识别（负库存 / 0 售价 / 毛利率 > 100% / 等）
    ↓
销售 + 库存 + 商品档案三表合并（按条码 inner / outer join）
    ↓
品类匹配（系统品类 → V4.0 L1-L4 / 算法 + 人工核对）
    ↓
供应商匹配（如可获得）
    ↓
生成分析主表（SKU 级 / 全字段 / audit-trail）
```

**清洗硬约束**：

| # | 硬约束 | 反例 |
|---|---|---|
| 1 | **条码是主键** / 商品名不能替代条码 | 用商品名 join → 同名不同条码合并错位 |
| 2 | **没有条码的记录进入异常清单** | 散称商品 / 临时录入 / 单独处理 |
| 3 | **负库存必须单独标记** / 不参与正常库存周转计算 | 负库存当 0 库存处理 → 缺货误判 |
| 4 | **毛利率回算校验** | POS 字段毛利率与（售价-进价）/售价 不一致 → 标异常 |
| 5 | **活动 + 大盘客流影响不能直接归因于商品调改** | 调改后销售 +30% / 实际可能是节日效应 / 必须 control |

## 7. 商品淘汰候选逻辑

> **本节只设计逻辑 / 不输出具体商品**。

### 7.1 8 维度筛选

| # | 维度 | 阈值（候选示意 / 待 v0.5 校准）|
|---|---|---|
| 1 | 低动销 | 90 天累计销量 ≤ 阈值 X / 周转天数 ≥ Y |
| 2 | 低毛利 | 毛利率 ≤ Z% / 毛利额贡献 ≤ 阈值 |
| 3 | 高库存 | 库存周转天数 ≥ 阈值 / 库存金额占比高 |
| 4 | 长期不动销 | 30 / 60 / 90 天 0 销量 |
| 5 | 结构重复 | 同 L4 内 SKU 数过多 / 同价格带过密 |
| 6 | 陈列资源占用过高 | 排面数 / 陈列面积 ≥ 阈值 / 但销额占比低 |
| 7 | 与门店定位不匹配 | 主品类外 / 客群偏离 / 价格带不符 |
| 8 | 现场确认后仍无保留理由 | 店长 / 老板访谈 / 客户认可可淘 |

### 7.2 4 大硬约束（淘汰决策）

```
❌ 淘汰候选 ≠ 立即淘汰
✅ 必须经过现场确认（佳妮 + 6 哥）
✅ 必须经过店长 / 老板（启明）确认
✅ 必须考虑替代商品和新品候选池
```

## 8. 商品结构分析逻辑

### 8.1 多维结构

| 维度 | 分析层级 | 输出 |
|---|---|---|
| 品类层级 | L1 → L2 → L3 → L4 | 4 层树状结构 |
| SKU 数结构 | 各层 SKU 数 + 占比 | 数量树 |
| 销售额结构 | 各层销售额 + 占比 | 金额树 |
| 毛利额结构 | 各层毛利额 + 占比 | 金额树 |
| 毛利率结构 | 各层加权毛利率 | 比率分布 |
| 库存结构 | 各层库存金额 + 周转 | 金额树 |
| 价格带结构 | 各层价格带分布 | 直方图 |
| 供应商结构 | 各层供应商集中度 | 集中度图 |
| 子组结构 | 5 子组 SKU + 销售 + 毛利 | 子组对比 |
| P/M 域 | 物理区域 + 主品类 | 跨表透视（如数据支持）|

### 8.2 结构指标

- **L1 销售集中度**：前 3 大类销售占比
- **L4 SKU 数离散度**：避免某 L4 过载
- **价格带覆盖率**：低 / 中 / 高 三段是否齐
- **毛利率均匀度**：各层毛利率方差

## 10. BI 流程嵌入设计

### 10.1 10 步流程

```
1. 数据导入        ← POS / Excel / CSV / 客户提供
2. 字段映射        ← 中英文映射 / 数据类型 / 单位统一
3. 数据清洗        ← §5 清洗流程
4. 异常校验        ← §5 5 大硬约束
5. 主表生成        ← 销售 + 库存 + 档案三表合并
6. 指标计算        ← 毛利率 / 周转 / 客单 / 动销率
7. 分类分层        ← ABCD / L1-L4 / 价格带 / 子组
8. 清单生成        ← §6 9 类输出表
9. 报告生成        ← §9 10 段诊断报告
10. 复盘更新       ← 调改后效果 / 数据反推 / 模板迭代
```
