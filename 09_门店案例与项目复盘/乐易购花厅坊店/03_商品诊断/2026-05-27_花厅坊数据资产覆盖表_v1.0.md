# 花厅坊数据资产覆盖表 v1.0

## 文件用途

**强制**：以后判断"vault 内是否已有 X 数据 / 是否需启明回传 Y" 前必须先读本表。

**核心原则**（来自 5/26 第 6 次反幻觉失误教训）：
- ❌ 不能用"5/N+ 新文件 0 个"就判 "0 到位"
- ❌ 不能用"启明 N 天 0 回传"就判 "数据基础为 0"
- ✅ 必须**先查本表是否覆盖目标数据**
- ✅ **新文件为 0 不等于数据为 0** / 历史大表早已含目标时段

**适用范围**：花厅坊店 / 战役#1 / 所有 G3-B / G4-A / G4-B / 商品诊断 / 调改 / 验收。

**形成依据**：5/27 vault 数据目录全量实扫（85 个 xls + 5 个照片目录）。

## 使用流程

### 处理"数据是否已有"判断前

1. **先读本表全部 12 主数据资产 + 4 关键说明**（5-10 分钟）
2. 若目标数据有 ✅ active → 直接引用 / 不查启明
3. 若目标数据 🟡 近似 → 引用时明示"近似 / 严格定义 pending"
4. 若目标数据 ❌ pending → 列入待补清单 / 不擅自补
5. 若目标数据**不在本表** → 先扫 vault 数据目录全量 / 再决定是否补登

### 新增数据资产沉淀流程

1. 启明回传 / 用户补充新数据 → 入 `99_原始素材/01_门店数据材料/`
2. 立即补登到本表（顺延 D-XX 编号）
3. 必须列：路径 + 类型 + 覆盖品类 + 覆盖时间 + 可/不可用于 + 严格/近似 + 状态 + 注意
4. 每次新增更新版本记录
