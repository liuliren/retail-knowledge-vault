# Cluster G｜门店决策 SOP 操作系统（Retail OS v1.0）

> 把复杂系统判断压成**店员照做的动作清单**。可直接打印,不依赖分析能力/理论背景。
> 系统状态:M-DEC-009/010/011 均 candidate(不升active);本SOP用其规则但不改其状态。

---

## 一、门店案例结构（Case System）
**Case = 一个 SKU 群决策单元**
| 要素 | 内容 |
|---|---|
| SKU群 | 饼干类(T03) |
| 结构问题 | 中价带过密82% / SKU冗余 / 14滞销占库 / A类24%产60%(头部弱) |
| 决策路径 | A库存→B结构→C价格带→D陈列→E ABC裁决→F误判校正 |
| 最终动作 | A类保护 / C滞销清尾 / C非滞销重审 / B观察 / 冗余压缩 |

---

## 二、SOP 决策流程（5 步·店员照做）

### Step 1｜数据扫描（A/B/C）
- ☐ 库存异常?（滞销=有货>60天没卖 / 缺货=货架空）
- ☐ SKU冗余?（同一类多口味多款挤一起=冗余）
- ☐ 中价过密?（中间价位SKU占比 **>70%** = 过密,M-DEC-010）

### Step 2｜行为判断（D）
- ☐ A类(卖得最好的)在不在**黄金位**(视线0.6-1.8米/货架右上)?
- ☐ 占黄金位的卖得动吗?（黄金位卖不动=要换）
- ☐ 滞销品在哪层?（占黄金位→必下移）

### Step 3｜SKU裁决（E）
- ☐ 按销售额排ABC:累计**≤60%=A** / 60-90%=B / **>90%=C**
- ☐ A类→保留 / B类→观察 / C类→进Step4查

### Step 4｜误判校正（F）— 砍之前必答
> **C类/低销品砍之前，必须回答这4问，任一为"是"则禁止淘汰：**
- ☐ 是**需求问题**吗?（儿童/健康/送礼/节日=低频有需求→留）
- ☐ 是**结构问题**吗?（中价挤压/同质冗余→调结构不砍）
- ☐ 是**行为问题**吗?（在死角没被看到→先改陈列）
- ☐ 是**缺货假象**吗?（货架空导致0销量→补货不砍）
> **4问全"否" + 真有货积压 = 才允许进淘汰。**

### Step 5｜最终门店动作（G）
唯一输出 → **上架/下架 · 上移/下移 · 补货/清仓 · 保留/替换**

## 三、门店复制模板（打印即用）

### 模板1｜SKU 调整清单
| SKU(品名) | ABC | 当前货位 | 动作 | 原因 |
|---|---|---|---|---|
| (例)大嵘合桃酥 | A | 第3层 | **上移黄金位+保库存** | 核心利润 |
| (例)兜兜爱儿童饼干 | C | 底层 | **保留待现场确认** | 儿童需求(F) |
| (例)某负毛利威化 | C | — | **降价清尾** | 特价清仓(FC09) |
| ____ | _ | ____ | ____ | ____ |

### 模板2｜陈列调整 SOP
1. A类(卖最好)→货架**右上黄金位**(视线0.6-1.8米),给更多排面;
2. 形象品→左上 / 关联必需品→左下 / 畅销促销→右下(微观Z);
3. C类滞销→**下移底层**,排面5→3→2→1递减,仍差让位;
4. 同质SKU(多口味)→**集中纵向陈列**,留动销最好1-2;
5. 死角里的A类→**立即上移**主通道/端架。

### 模板3｜ABC 裁决表
| 类 | 判定线 | 处置 |
|---|---|---|
| A | 累计销额≤60% | 保库存+黄金位,**不准断货** |
| B | 60-90% | 观察,挑增长强的往A推 |
| C | >90% | 进滞销处理流程(模板4) |

### 模板4｜滞销处理流程（5道闸）
```
滞销品(有货>60天没卖)
 ① 货架空吗?(缺货)──是→补货,停 ✗淘汰
 ② 售价<进价?(负毛利)──是→降价清尾(特价清仓)
 ③ 是儿童/健康/送礼/节日?──是→留(需求点,F)
 ④ 在死角没被看到?──是→先上移黄金位观察
 ⑤ 以上全否+真积压──→淘汰/清仓
```

### 模板5｜价格带校正流程
1. 算中间价位SKU占比;**>70%→必须重构**(砍冗余中价/补两端);
2. 每个小类**必须有≥1个超低价走量品**(必要条件);
3. 高价品**必须有锚点**(配套形象/送礼场景),否则不留;
4. 低价**不能替代结构缺失**(缺强单品≠靠低价补)。

## 四、T03 标准案例（固化·对外可讲）

### Case：饼干类 SKU 系统重构
**1. 初始问题**:中价带过密82% / SKU冗余 / 14滞销长期占库 / A类过少(24%SKU产60%=头部缺强单品)。
**2. 系统路径**:A库存→B结构→C价格带→D陈列→E ABC→F误判。
**3. 关键发现**:
- A类=核心利润引擎(45个产60%,**必须保护**);
- **健康需求已在A类**(无糖黑芝麻酥/合桃酥/能量棒)=不是缺口;
- C类=结构问题≠低价值(中价挤压/同质冗余/弱位);
- 低销SKU中**大量误判项**(儿童/季节/弱位),14滞销经F收窄=3特价清尾+现场确认后真滞销;
- 6/12真数据确认14滞销**全积压/0缺货**(非缺货误判)。
**4. 最终动作**:
- A类→上黄金位+保库存;
- C类滞销→清尾(3负毛利已定);
- C类非滞销→结构重审(调价格带/改陈列,不砍);
- B类→观察转A潜力;
- 冗余SKU→压缩留代表。

## 五、门店 SOP（最终执行版·四条铁律）

**SOP-01 SKU进入规则**
- 无销量+无结构价值 → 淘汰;
- 有销量+结构支撑 → 保留;
- 低销量+高需求场景 → **进F层,不直接砍**。

**SOP-02 陈列规则**
- A类**必须**在黄金位;
- C类**禁止**占黄金位(验证期除外);
- 同质SKU**必须**集中。

**SOP-03 价格带规则**
- 中价过密(>70%)→**必须重构**;
- 低价**不能**替代结构缺失;
- 高价**必须**有锚点SKU。

**SOP-04 误判规则（关键）**
- 砍任何SKU前**必须过F层四问**(需求/结构/行为/缺货);
- **未通过F → 禁止淘汰**。
