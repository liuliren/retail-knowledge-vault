# B1条码分组统计去重排查

## 1. B1文件范围

- **物理位置**：`99_原始素材/00_客户原始材料/`(根层，非子目录)，文件名模式 `商品销售条码分组统计_<时间窗口>_<品类>数据_20260508.xls`。
- **实际计数**：该目录下匹配"条码分组统计"字样的文件 **118 个**（台账登记119，差1属于台账口径/README等边界计数差异，不影响结论）。
- **时间窗口覆盖**：30天 / 90天(3个月) / 120天(4个月) / 2025年(全年)，共4类窗口 × 约30余个品类切片，同一拉数日期 **2026-05-08**（系统性一次性拉数，非多次拼凑）。
- **字段结构**（SKU级，来自项目盘点报告的表头级扫描）：品名 / 单位 / 销售数量 / 销售金额 / 零售价 / 退货 / 主供应商 / 进价 / 库存。

### 附：另一批"相似命名"文件的归属判断

`99_原始素材/01_门店数据材料/2026-07_数据治理诊断批次/` 目录下另有 **11 个**文件命名含"条码分组统计"字样，经核对**不属于B1**，判断依据：

1. **物理位置不同**：B1 全部118个文件都在 `00_客户原始材料/` 根层；这11个文件在完全独立的 `01_门店数据材料/2026-07_数据治理诊断批次/` 目录下，且该目录本身混装了多个其他批次(B2/B5/B7/B9/B10等)的文件，是一个物理归档目录而非单一批次目录。
2. **命名规则不同**：B1 统一是"品类+窗口+20260508"单一拉数日期格式；这11个文件是"花厅坊\_<品类>\_商品销售条码分组统计\_<日期范围>"格式（如`20260101-0424`、`2026年3月-4月`、`2026年3月`），日期范围各不相同、且早于B1的5/8拉数。
3. **逐一比对文件名无重复**：11个文件里除1个同名`商品销售条码分组统计_30天_水果数据_20260508.xls`外，其余10个在B1的118个文件名单中均**不存在**；且经核实B1的118个文件里**没有**"30天\_水果"这一品类窗口组合（B1的30天窗口清单里没有水果），说明这个同名文件不是B1的重复件，而是B1系统性拉数遗漏的品类窗口，由早期临时拉数补上的独立文件。
4. **结论**：这11个文件是**独立的小批次**（早期、分散、按品类临时拉数的条码分组统计，非台账B1-B13已登记编号，属登记盲区），不计入B1的118个文件范围，**本次去重排查不覆盖它们**，如需处理需六哥另行确认是否登记为新批次并排期。

## 3. 独有字段

B1相对全部已核对批次的独有价值点：

1. **多时间窗口横向对比**：同一SKU/品类在30天/90天/120天/全年四个窗口下的销售表现，是全库唯一具备这种"趋势对比视角"的批次（其他批次要么是单一时间窗口的快照，要么是聚合到类别/门店层级，没有SKU级多窗口对比）。
2. **退货字段**：B1明确含"退货"列，目前已核对的其他批次（D1/B5/B10/商品档案/sku_registry）均未见退货字段，这是全库目前唯一潜在的退货数据来源。
3. **销售侧"主供应商"**：与静态商品档案里的供应商字段不同，B1的供应商字段挂在实际销售记录上，理论上可用于按供应商维度做销售贡献分析，静态档案做不到这一点。

## 4. 潜在价值

- 若能解决读取问题（见§5），B1可以补齐当前分析里缺失的"分品类×分时间窗口的动销趋势"视角，与已产出的D1(GMROI/库存)、B5/B10(门店整体客单)形成互补，而非替代。
- 退货字段若能落地使用，是全库唯一的退货侧信息，对判断"品类质量/顾客不满"有直接价值。
- 但价值实现的前提是先解决技术障碍（下条）。

## 5. 是否建议深读

**本轮不建议深读**，原因有二：

1. **不是重复劳动**——排查已确认B1不与已用数据重叠，具备独立价值，但这恰恰说明它是一项"新的、体量较大的(118个文件)专项工作"，而非可以顺手做的补充动作，不适合在"去重排查"这个小任务里顺带展开。
2. **技术前置障碍未解决**：项目盘点报告与6/19字段识别日志均记录，这批"假xls"（国产POS的OLE2/CDFV2格式）用标准xlrd读取会报`Expected BOF`错误，需要calamine引擎或转换层（ssconvert/pandas-calamine）才能读出行级数据；本次环境同样复现了该报错（抽样验证：`商品销售条码分组统计_30天_食品数据_20260508.xls`等3个样本文件读取均报此错误，且确认非加密文件，属POS导出格式兼容性问题，非B1专属问题——与posclean skill记录的"国产POS七坑"之一一致，但posclean skill正文目前尚未点名B1这一具体文件系列）。

## 6. 建议处理方式

1. **维持"已结构扫描"状态**，不升级为深读，避免占用当前聚焦花厅坊150天交付的窗口。
2. **技术前置任务**（优先于深读本身）：如未来决定深读B1，先跑一次posclean/calamine读取验证，确认118个文件能否批量转出行级数据，再评估深读工作量。
3. **11个独立小批次**（见§1附）建议六哥另行确认是否需要登记为新批次编号并排期核实，本次不做处理。
4. 台账 `2026-07-09_99原始素材深读总台账_v0.1.md` 中B1行的"建议去向"栏可后续更新为"确认不重复，价值点=多窗口趋势+退货+销售侧供应商；深读前需先过posclean假xls读取关"——本报告不直接修改台账，交六哥裁决后由其他动作回填。
