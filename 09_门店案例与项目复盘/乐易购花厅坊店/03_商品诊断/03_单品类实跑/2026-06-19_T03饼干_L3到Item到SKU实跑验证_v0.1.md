# T03饼干 L3→Item→SKU 单小类实跑验证 v0.1

## 1. 文件定位
- 本文件 = M2 四块二遍补强(B/C1/C2a/C2b-1)在**花厅坊 T03 饼干单小类**的实跑验证。
- **不是**正式标准、**不是**全店结论、**不新增 M-DEC**、不改 M-DEC-009~012/POS清单/品类表C、**不正式建需求点表**。
- 所有 SKU 级临界值与数字 **标 [推断] 或 pending**,等真实 POS/Excel/现场/老板 sign off。
- 目的:验证"L3→Item→SKU"方法能否落地 + 沉淀**可复制单小类模板**(§12)。

## 2. 小类选择理由(为何先 T03 饼干)
| 理由 | 说明 |
|---|---|
| 任务维度丰富 | 饼干天然覆盖 办公代餐/早餐/儿童/家庭分享/解馋/健康/礼盒/本地老人 多任务 → 最能验证"宽类窄品"+"Item拆分" |
| 比 T01 薯片更适合拆 Item | 薯片任务相对单一(解馋为主);饼干跨任务更广,拆 Item 信号更强 |
| 四链全覆盖 | 含低频刚需(健康/本地老人)、易同质(分享装品牌多)→ 同时压 B/C1/C2a/C2b-1 四链 |
| 可复制性 | T03 跑通 → 模板复制 T01薯片/T05果干/T08果冻/T02膨化 |
| 风险低 | 常规性/主角角色,非引流命脉,验证试错代价小 |

## 11. 初步动作建议(只输出规则,不点SKU名)
> 无真实 SKU 明细 → **不输出"砍掉某SKU"**,只给规则与待补数。

| 动作 | 适用(规则)|
|---|---|
| 保留 | 低频刚需 Item(I-T03-03/06/08)的代表SKU,即使C类 |
| 替换 | 衰退SKU若属必留Item → 好品替差品(非净减)|
| 瘦身 | 同质Item(I-T03-04/05)内重复SKU **待数据确认后** |
| 补齐 | 高价带/礼盒(I-T03-07)若漏档 → 补关键段 |
| 观察 | 双ABC衰退但未确认六桶的SKU |
| **待补数** | T03 绝大多数动作(SKU级数据缺)|

## 12. 可复制模板(L3→Item→SKU 单小类验证)
> 沉淀为模板,复制到 T01薯片/T05果干/T08果冻/T02膨化 等:
```
1 选小类(看任务丰富度+风险)
2 盘现有数据(markdown有什么/SKU级缺什么→pending)
3 L3→Item拆分(品类表任务+购物任务图谱画像;标低频刚需/必留)
4 C1 SKU数约束(角色/占比/ABC/价格带→超配欠配;缺数=待补)
5 C2a 宽类窄品(哪些Item宽/哪些窄品/低频刚需不砍/替换非净减)
6 B 六桶防误杀(ABC初筛+双ABC+售罄率;必留Item护栏)
7 M3联动(价格带过密/漏档)
8 M4联动(黄金位×ABC/Item混放/容量)
9 字段缺口清单(POS/表B/表C/表E/人工/现场)
10 动作=保留/替换/瘦身/补齐/观察/待补数(无明细不点SKU)
```
> 每小类约束值不同,**模板可复用,结论不可复用**。
