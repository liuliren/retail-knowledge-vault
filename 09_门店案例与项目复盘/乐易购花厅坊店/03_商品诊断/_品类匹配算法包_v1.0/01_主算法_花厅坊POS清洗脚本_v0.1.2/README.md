# 花厅坊 POS 清洗脚本 v0.1

## 1. 文件定位

本目录是花厅坊样板项目 POS 数据清洗的工程化资产，对应 P2-04C《POS数据清洗与商品标准化规范_v0.1》的首套实战实现。脚本设计为可追溯、可复用、可审计：原始文件全程只读，输入/输出 SHA256 全程校验，dry-run 与正式执行严格分离。

## 2. 适用场景

- 花厅坊样板首轮 POS 清洗（P1-03 任务，2026-05）
- 后续乐易购其他门店或客户 A/B 项目的 POS 清洗（v0.2 起脚本通用化）
- P2-04C SOP 的实战回写依据（v0.2 校准）

## 3. 输入

13 张文件全部位于花厅坊项目 `99_原始素材/`：

| ID | 类型 | 文件 |
|----|------|------|
| S01-S07 | POS 销售统计汇总 | 7 张 .xls/.xlsx |
| D01-D05 | 字典/档案 | 5 张 .xlsx |
| M01 | 4077 行基础匹配层 | 1 张 .xlsx（项目历史过程文档/）|

完整文件名见 `config.py` 中 `INPUT_FILES`。

## 4. 输出

### 4.1 dry-run 模式（默认，不写正式输出）

写入 `_dryrun_preview/`：
- `dryrun_summary.json` — 行数 / 异常率 / 冲突 / 匹配置信度
- `dryrun_sales_sample.csv` — 主表抽样 10 行
- `dryrun_archive_sample.csv` — 商品档案抽样 10 行
- `dryrun_inventory_sample.csv` — 库存快照抽样 10 行
- `dryrun_anomalies_sample.csv` — 异常清单抽样 10 行
- `dryrun_index_sample.csv` — 资料索引抽样 10 行
- `dryrun_conflicts_top20.csv` — 多源字段冲突 TOP 20

### 4.2 execute 模式（用户审阅 dry-run 后才允许）

写入 `09_门店案例与项目复盘/乐易购花厅坊店/03_商品诊断/01_清洗输出/`：
- 销售统计汇总_合并版_v0.1.xlsx
- 商品档案_合并版_v0.1.xlsx
- 库存即时快照_v0.1.xlsx
- 异常清单_v0.1.xlsx
- 资料索引表_v0.1.xlsx
- 清洗日志_v0.1.md

## 5. 运行方式

```bash
# Dry-run（默认）
bash run_dryrun.sh

# 正式执行（需用户预审 dry-run 后授权）
bash run_execute.sh
```

或直接：
```bash
python3 main.py            # dry-run
python3 main.py --execute  # 正式执行
```

## 6. 风险边界

### 6.1 原始文件保护

- 全部输入文件用 `read_only=True`（openpyxl）/ `xlrd` 默认只读模式
- 脚本内 grep `\.save\(|\.write\(` 必须返回 0 命中（针对原始路径）
- 执行前后对 13 张输入文件做 SHA256 校验，不一致即 `sys.exit(FATAL)`

### 6.2 输出路径隔离

- dry-run 仅写 `_dryrun_preview/` 与 `/tmp/htf_p1_03/`
- execute 仅写 `03_商品诊断/01_清洗输出/`
- 绝不写入 `99_原始素材/` 或任何治理文件路径

### 6.3 异常率守门

- 阻断异常 < 5% → 通过
- 阻断异常 5-15% → 警示，等待用户介入
- 阻断异常 > 15% → 阻断，不允许进入正式执行

### 6.4 库存表缺失边界

- 现有数据无独立库存导出，仅 POS 表"库存"列即时快照
- 输出 `库存即时快照_v0.1.xlsx` ≠ P2-04C §S6 标准库存表
- 不能做库龄 / 精确周转 / 缺货预警 / 双指标滞销淘汰
- 二访 P1-B B.6 必须补采

### 6.5 不做的事

- 不做经营诊断（ABC / 宽度 / 调改建议）
- 不修改 vault 治理文件（CLAUDE.md / 看板 / SOP / 方法论）
- 不修改 99_原始素材/ 任何文件
- 不输出客户敏感信息
