## 一、算法的输入是什么（POS 字段格式）

### 1.1 输入文件类型

13 张 Excel 文件，**全部需要由门店文员从 POS 系统导出**：

| 文件 ID | 类型 | 描述 | 必须 |
|---|---|---|---|
| **S01-S07** | POS 销售流水统计（7 张） | 商品销售条码分组统计（按品类 / 按时段 / 按门店） | ✅ |
| **D01** | 完整品类表 L4 | 晟果 V4.0 大湾区版品类系统（L1→L4 4 层标准） | ✅ |
| **D02** | 大湾区品类系统 V4.0 | 休闲食品品类系统表（定稿大湾区版） | ✅ |
| **D03** | 食品饮料品类匹配表 | 商品 → L1-L4 已知映射 | ✅ |
| **D04** | 商品条码名称统一表 | 品牌字段权威源（金标准）| ✅ |
| **D05** | 动销匹配品类表 | 品牌字段权威源（金标准） | ✅ |
| **M01** | 4077 行基础匹配层 | 已人工整理的条码 → L1-L4 映射（PK 主匹配层） | ✅ |

### 1.2 POS 流水必须字段（前置标准）

由 [[03_字段规范_POS数据清洗与商品标准化规范_v0.1.md]] §五 定义，关键字段：

| 标准字段 | 中文别名（POS 系统常见） | 类型 |
|---|---|---|
| `barcode` | 商品条码 / EAN码 / 货号 | STRING ≥ 8 位 |
| `sku_name_raw` | 商品名称 / 品名 / 商品全称 | STRING |
| `qty` | 销售数量 / 数量 / 件数 | DECIMAL |
| `sales_amount` | 销售金额 / 实收金额 | DECIMAL |
| `gross_profit` | 毛利额 / 毛利金额 | DECIMAL（可空 / 待确认）|
| `sale_date` | 销售日期 / 交易日期 | DATE (YYYY-MM-DD) |
| `cost_price` | 进价 / 成本价 / 采购价 | DECIMAL |
| `retail_price` | 零售价 / 售价 | DECIMAL |
| `category_raw` | 门店原始品类 / ERP 类别 | STRING（用于 4 级匹配兜底） |

### 1.3 数据周期

- **优先**：最近 90 天 POS 流水（如 2026-01-15 至 2026-04-15）
- **可选**：去年同期数据（用于环比 / 同比分析）
- **数据周期**：4 周 / 8 周 / 90 天 灵活（用户已确认花厅坊系统支持灵活时段导出）

### 1.4 输入位置（vault 内）

输入文件**全部位于**：
```
09_门店案例与项目复盘/乐易购花厅坊店/99_原始素材/
```

完整文件名映射见 `01_主算法_*/config.py` `INPUT_FILES`。

## 二、算法的输出是什么（V4.0 标准 L1-L4？）

### 2.1 主输出：6 张标准表

执行模式（`run_execute.sh`）输出到 `09_/.../03_商品诊断/01_清洗输出/`：

| # | 输出文件 | 内容 | 字段重点 |
|---|---|---|---|
| 1 | `销售统计汇总_合并版_v0.1.xlsx` | 主表（多文件合并去重）| 含 **L1-L4 品类标签** + 匹配状态 + 匹配置信度 + 匹配依据 + 匹配来源 |
| 2 | `商品档案_合并版_v0.1.xlsx` | 商品基础档案 | 条码 + 品名 + 品牌 + 规格 + L1-L4 |
| 3 | `库存即时快照_v0.1.xlsx` | 当前库存（POS"库存"列）| 条码 + 库存数量 + 金额 |
| 4 | `异常清单_v0.1.xlsx` | 13 类异常 | issue_type / impact_level / suggested_action |
| 5 | `资料索引表_v0.1.xlsx` | 文件追溯 | 原始文件 / 标准文件 / 状态 |
| 6 | `清洗日志_v0.1.md` | 决策记录 | 字段映射 / 异常处理 / 待确认 |

### 2.2 V4.0 标准 L1-L4 4 层品类编码

输出**完全对齐 V4.0 大湾区版**（`乐易购_品类系统V4.0_定稿_大湾区版.xlsx`）：

| 层 | 含义 | 示例 |
|---|---|---|
| **L1** | 大类 | 休闲食品 / 水饮 / 日用品 / 米面粮油 |
| **L2** | 中类 | 休闲零食 / 糖巧 / 果冻 / 肉脯 / 饮料伴侣 |
| **L3** | 小类（**12 子品类对齐表 C**）| 薯片 / 膨化 / 饼干 / 坚果炒货 / 果干蜜饯凉果 / 糖果 / 巧克力 / 果冻布丁 / 肉脯 / 辣条豆制品 / 饮料伴侣 / 其他 |
| **L4** | 细类 | 原味薯片 / 黑松露薯片 / 等 |

### 2.3 匹配置信度 4 级（POS 规范 §十）

| 置信度 | 触发条件 | 后续处理 |
|---|---|---|
| **高** | 条码精确命中 M01 + 品名一致 | ✅ 直接进入正式分析（A 级证据） |
| **中** | 关键词匹配 + 品牌一致 | 🟡 标"待复核"（B 级）|
| **低** | 三级组合 + 信息不全 | ❌ 必须人工复核（C 级）|
| **冲突** | 多种方法给出不同 L4 | ⛔ 暂停使用，人工裁决 |

### 2.4 dry-run 输出（预览模式）

执行 `run_dryrun.sh` 输出到 `_dryrun_preview/`（用户预审 → 授权后才能 execute）：

- `dryrun_summary.json` — 行数 / 异常率 / 冲突 / 匹配置信度统计
- `dryrun_*_sample.csv` — 6 张表抽样 10 行预览
- `dryrun_conflicts_top20.csv` — 多源字段冲突 TOP 20

## 八、使用流程（5 步）

```
Step 1: 文员从花厅坊 POS 系统导出 13 张 .xls/.xlsx
        → 放入 09_/.../99_原始素材/

Step 2: 顾问运行 dry-run 预览
        cd 01_主算法_花厅坊POS清洗脚本_v0.1.2/
        bash run_dryrun.sh
        ↓
        看 _dryrun_preview/dryrun_summary.json：
          - 总行数 + 异常率（应 < 5%）
          - 多源字段冲突 TOP 20
          - 6 张表抽样 10 行预览

Step 3: 审阅 dry-run，确认无重大问题
        ↓ 用户授权后

Step 4: 运行 execute 模式
        bash run_execute.sh
        ↓
        输出到 09_/.../03_商品诊断/01_清洗输出/ 6 张标准表

Step 5: 6 张标准表 → 装入 6 v1.0 表（_v1.0_交付包/）
        - 商品档案 → 表 A 商品基础表
        - 销售统计 → 表 B 动销分析（aggregate by 90 天）
        - 异常清单 → 表 B Sheet 4
        - 库存快照 → 表 B 字段 20-21
```
