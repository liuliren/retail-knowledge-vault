# 花厅坊库存订货样板 · Task Charter v0.1
> 套 [[任务前置契约协议_v0.1]]。六哥前置确定,AI 自主跑到底,末端批签。

1. **目标与范围**:用 [[社区超市库存订货优化包_主定义_v0.1]] 给花厅坊跑**首个库存订货诊断样板**,验证产品+回流为案例(闭合"方法论→产品→案例→证据")。**首品类=方便食品**(数据齐、已有单品类诊断基础)。**不做**:其余品类(本轮只方便食品);生鲜(负库存重灾区,数据需花厅坊加工后再跑)。
2. **数据与处理**:源=`库存积压报表_方便食品_20260508.xls`(当前库存/期间销量额/毛利/最近销售&进货日期)+ `方便食品_花厅坊商品明细_20260311-0612.xls`(日度流水→动销天数/日均)。**数据清洗(铁律)**:剔**负库存**、0/异常零售价、空货号行;**进价不准坑**→毛利/周转档标"仅参考"([[花厅坊数据质量坑]])。脚本中介、不把裸值堆进上下文。
3. **质量门**:套数量管理判据(有货率/安全库存Z值/ABC订货系数/售罄率/周转IR/可销天数);结论先行+红黄绿+fact_layer;crown jewel 不动;机制B复盘收尾。
4. **签字边界**:**AI 自签**=内部库存订货诊断报告(client_confidential·含品名数据)+ 填充模板 + 回流案例钩子;**末端批签**=诊断卡升 active;**永不代签(红线)**=发客户版(0术语脱敏版)发布、对赌背书 → 六哥本人 Z。
5. **打断条件**:仅当①方便食品库存数据也大面积异常无法清洗 ②触发红线节点 ③范围需变 —— 才找六哥;否则自主跑完。
