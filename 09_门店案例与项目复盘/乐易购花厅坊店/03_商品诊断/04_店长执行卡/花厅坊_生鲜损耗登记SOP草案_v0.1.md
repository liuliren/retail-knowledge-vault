## 0. 结论先行（读这段就能拍板）

1. **一句话 SOP**：生鲜员工每天闭店前，把当天扔掉/折价/加工损的货，**按小类记一行"数量 + 当天零售价金额 + 原因"**，一张纸、五分钟、当天填完——先跑纸质，别等系统。
2. **为什么这是最高杠杆**：花厅坊诊断有 4 条线（净品效 / SS分层 / 双毛利 / 净坪效）全部卡在同一个缺口——"没有生鲜损耗数"（W1诊断草案 §7 缺数据#5）。**一张损耗登记表，一次性解锁这 4 条线**，是当前投入产出比最高的一个动作。
3. **别一步到位上系统**：批0=纸质表本周起手（零成本、零培训门槛），跑满 2-4 周有了真实损耗率区间后，批1 再谈小程序/POS 打通。**先要"有数"，再要"好用"。**
4. **口径已定死（六哥 2026-07-04 拍板）**：采用**零售价口径** `损耗率 = 损耗零售额 ÷（销售零售额 + 损耗零售额）`；同时登记表要能折算出**损耗成本额**（喂净品效/净毛利）。口径已钉死，批0 即按此起手（§3）。

## 1. 登记什么（What）

### 1.1 登记粒度：三档就近取
| 商品类型 | 登记粒度 | 举例 |
|---|---|---|
| 有固定条码的预包装生鲜（盒装菜/预制/冷藏乳） | **SKU 级** | 扫码或抄码 |
| 散装称重、无固定条码（叶菜/根茎/水果/肉档） | **小类级**（就近归到 L3/L4 小类） | "叶菜类""根茎类""猪肉档"整体记一行 |
| 现制/加工（切配、熟食） | **小类级 + 标注"加工损"** | "净菜加工""熟食档" |

> **无固定条码怎么办**（散装称重品的核心痛点）：不追单品，**按小类记整体**。店里生鲜台秤/收银小类编码怎么分，登记表就怎么分——**登记口径必须与 POS 销售的小类口径对齐**（否则算不出该小类损耗率）。金额用"当天该小类零售单价 × 损耗重量"由员工估填（店员天天看价签，估得出）。

### 1.2 每行登记 4 个字段（缺一不可）
1. **小类/SKU**（就近取，见 1.1）
2. **数量**（斤/份/盒——用该品平时卖的单位，别换算）
3. **损耗零售额**（= 数量 × 当天零售单价；员工照价签填，不算成本）
4. **原因（4 选 1，圈一下即可）**：

| 代码 | 原因 | 定义（给员工的大白话） | 管理含义 |
|---|---|---|---|
| **A 自然耗损** | 失水、蔫、烂、自然减重 | 没人碰也会坏的 | 订货量/周转问题 |
| **B 加工损** | 切配去皮、修整、分装的损耗 | 加工台切下来扔的 | 加工出成率问题 |
| **C 过期** | 到期/临期下架 | 过了日子的 | 订货过量/动销问题 |
| **D 破损** | 磕碰、包装破、称错、丢失 | 弄坏弄丢的 | 操作/防损问题 |

> **折价不等于损耗、但要一起记**：临期打折卖出的货，另起一列记"**折价让利额**"（原价−折后价）。它不是扔掉，但**吃掉毛利**，双毛利/净毛利要用到。默认登记表把"报损（A-D）"和"折价让利"分两栏，别混。

## 2. 谁登 · 何时登 · 用什么登（Who / When / How）

| 维度 | 批0（本周起手·纸质） | 批1（数字化·2-4周后） |
|---|---|---|
| **谁登** | 生鲜区当班员工（各档口自己记本档），店长每日收表核签 | 同上，店长手机端复核 |
| **何时** | **每日闭店前统一填**（当天的记忆最准；别隔夜补） | 随手扫/随手录 |
| **用什么** | **一张 A4 纸质登记表**（模板见 §5），贴生鲜区操作台 | 极简小程序表 / 微信接龙 / POS 报损模块 |
| **摩擦控制** | 每天每档 ≤5 分钟、≤10 行；只圈选 + 填数，不算成本、不算率 | 下拉选小类、拍照留证 |

**最低摩擦铁律（防"登记本身把员工累跑")**：
- 员工只干三件事：**记数量、抄零售价、圈原因**。成本、损耗率、汇总，**全部由店长/后台每周算一次**，不占员工时间。
- 一档一表、当天当档，不搞跨档汇总（那是店长/周汇总的事）。

## 3. 口径定义（Definition · 必须先钉死）

> 这是全 SOP 的命门。口径不统一，四条诊断线拿到的是互相打架的数。**六哥 2026-07-04 已拍板：锁定"零售价口径"。**

### 3.1 损耗率口径（✅ 已定：零售价口径 · 六哥 2026-07-04 拍板）
- **已定采用（零售价口径）**：
  `损耗率 = 损耗零售额 ÷（销售零售额 + 损耗零售额）`
  含义 = "本可卖出的货里，损失掉的占比"。优点：分子分母同为零售价、员工填的就是零售额、口径直观、行业生鲜常用此口径。**与净品效/双毛利可打通**（见 3.2）。**批0 登记即按此口径起手。**
- **未采用·备选1（成本/进货口径）**：`损耗率 = 损耗成本额 ÷ 进货成本额`。更贴现金损失，但需要进货台账 + 成本额，花厅坊现无进货明细底座 → **不具备，未选**。（保留记录，供批1 有进货底座后回看。）
- **一致性红线（draft·待三轮审议）**：全店只用零售价口径，不得 A 品类用零售口径、B 品类用成本口径——否则跨品类损耗率不可比。*（含"红线"字样，升为门店正式执行/考核依据前须过标准审议三轮；批0 内部诊断取数即用此单一口径。）*

### 3.2 与四条诊断线的口径对齐（为什么要同时留"成本额"）
净品效、净毛利都吃的是**损耗成本额**，不是零售额。所以登记表虽让员工填零售额（省事），后台周汇总时须折算成本额：

- **损耗成本额 = 损耗零售额 ×（1 − 该小类毛利率）**（用该小类平均毛利率折算，花厅坊有品类毛利即可）
- **净品效 =（销售成本口径 or 销售额 − 损耗成本额）÷ SKU数**（对齐 Block 4 / C1批判 v0.2；口径细节随净品效定义定）
- **净毛利额 = 毛利额 − 损耗成本额 − 折价让利额** → **净毛利率 = 净毛利额 ÷ 销售额**（对齐"双毛利"）
- **净坪效 = 净品效 × 该品类陈列面积占比**（待货位桥表就绪后精化）

> 一句话：**员工填零售额（省事），后台用毛利率一步折成本额（喂模型）。** 这样既低摩擦、又不丢净口径所需的成本维。

## 4. 数据怎么回流（登记 → 周汇总 → 喂诊断）

```
每日纸质登记（各档）
   → 店长每日收表核签、留存
   → 【每周一次】店长/助理把一周表汇总进一张 Excel（小类 × 原因 × 周损耗量/额）
   → 计算：各小类周损耗率 + 损耗成本额
   → 回流三处：
      ① 净品效表：补上"损耗成本额"列 → Block 4 从"店长经验标注"升级为"算得出"
      ② SS分层/订货：A自然耗损高的小类 → 缩订货批量/提周转（Block 1 命题11 降压货）
      ③ 坪效/净坪效：高损耗小类的坪效重算为净坪效 → 揪出"毛坪效达标、净坪效已亏"的伪优等生
```

**回流节奏**：日记 → 周汇总 → 月度进诊断报告。**别日日算率**（那是过度工程、增摩擦）；周频足够支撑决策，月频进交付件。

**责任落点**：登记=生鲜员工；核签=店长；周汇总/折算/回流=店长或数据助理（六哥远程可代跑第一轮，做样板）。

## 5. 分阶段落地（批0 本周即动 → 批1 数字化）

### 批0 · 纸质起手（✅ 六哥授权 · 本周可部署 · 仅限内部诊断取数）
> 六哥 2026-07-04 授权：批0 纸质版本周即在花厅坊部署，用途**限定于内部诊断取数**（喂净品效/SS/双毛利/净坪效四线），**不作员工考核依据**（考核挂钩须先过三轮审议，且见 §6.2 防瞒报：初期与考核脱钩）。
- **交付物**：一张 A4《生鲜每日损耗登记表》——表头 = 日期 / 档口 / 小类 / 数量 / 零售额 / 原因(A自然·B加工·C过期·D破损) / 折价让利额 / 记录人。
- **动作**：打印贴台、生鲜员工闭店前填、店长每日核签。
- **周期**：连跑 **2-4 周**，攒出各小类真实损耗率区间。
- **成本**：≈0（一叠纸 + 5分钟/天/档）。
- **验收**：4 周后能回答"叶菜/肉/水果各档周损耗率大概几个点、主因是 A 还是 C"。

### 批1 · 数字化（批0 验证有效后再上·目标"更省力+可打通")
- 极简小程序表 / 微信接龙 / POS 自带报损模块择一，把纸质字段照搬电子化。
- 打通 POS 销售小类口径 → 自动算损耗率、自动折成本额。
- **触发条件**：批0 已稳定产出数、员工已养成习惯——**先习惯后系统**，别用系统去逼习惯。

> **别一步到位上系统的理由**：上来就上小程序/系统 = 员工要学新工具 + 口径没验证 + 一旦难用直接废弃。纸质零门槛先把"每天记"这个习惯和口径跑通，是最稳的路径。

## 6. 失效边界 / 防坑（这套数会在什么条件下变废）

1. **估算偏差**：散装按小类估重、零售额照价签填，天生有误差 → **不追求单笔精确，追求趋势可比**。用途是排序/找高损耗小类，不是财务对账，容忍 ±10-15% 估算噪声。
2. **防瞒报（最大的坑）**：损耗直接关联员工绩效时，员工会少报/漏报 → 数偏低、诊断失真。对策：**登记初期与考核脱钩**（只用于诊断、不扣钱），并用 §6.3 抽检校准；先建信任、后谈管理。
3. **抽检校准**：店长每周随机挑 1-2 天/1-2 档，闭店时实地清点损耗筐 vs 登记表，核对偏差；偏差大则复训。用抽检当"锚"校准整体登记可信度。
4. **口径漂移**：员工把"折价"记成"报损"、把"加工损"记成"自然耗损" → 原因分类失真。对策：原因表就贴在登记表旁边（4 类大白话定义），店长核签时顺手纠。
5. **小类编码不对齐**：登记的小类 ≠ POS 销售的小类 → 算不出该小类损耗率。**批0 前必须先对齐一次小类清单**（登记表的小类行 = POS 里生鲜小类）。
6. **别把损耗数当唯一诊断依据**（承接 C1批判 v0.2）：生鲜核心矛盾是鲜度/损耗/时段，损耗率是其一不是全部；高损耗可能是订货问题、也可能是陈列/温控/时段问题，需叠加第二问，不可只凭损耗率一刀切汰品。
