# Retail OS v1.1｜五步 SOP 接入版

> v1.0 五步流程 **+** v1.1 三项补深，织成一张可照做的门店决策 SOP。
> 🆕 = v1.1 新增/升级处。v1.0 原始件保持不动；本件为接入版，跨案例验证后并入商业交付版。

---

## 📋 商品决策 5 步（升级版·照做）

### Step 1｜数据扫描（A 库存）🆕升级
- ☐ **库龄分四级**（🆕替代单一60天）：0-30天正常 / 30-60预警 / 60-90滞销→进5闸 / >90重滞→优先清；
- ☐ 缺货？货架空 = 缺货 → 补货；
- ☐ **售罄闸**（🆕）：货架常空但不是没进货 = 卖太快 → **加订货量；毛利薄可试探提价**；
- ☐ **A类水位**（🆕）：A类剩到 `日均销量×到货天数×1.5` 就补，**永不等空**；
- ☐ 中价过密？中间价位SKU >70% = 重构（C层·M-DEC-010）。

### Step 1.5｜商品角色闸（B 结构）🆕新增
> 砍冗余/低销前，先给 SKU 定角色（4 类）：
- ☐ **重点品**（高销高贡献）→ 绝不动，多排面；
- ☐ **常规品**（稳定动销）→ 保留，标准排面；
- ☐ **一般品**（低频有特定需求）→ 看需求场景定汰留；
- ☐ **目的品**（引流/形象，自身不赚钱但引客撑形象）→ **绝不能按销量砍！**
> 🔑 一问定生死：**这SKU是引流品/形象品吗？是→不准砍，哪怕它自己不赚钱。**

### Step 2｜看位置（D 陈列）
- ☐ A类（卖最好）在黄金位吗（0.6-1.8m 右上）？
- ☐ 黄金位有卖不动的吗？滞销占黄金位 → 必下移；
- ☐ 同质多口味 → 集中纵向，留动销最好 1-2。

### Step 3｜SKU 裁决（E ABC）🆕升级为双维
- ☐ **销额ABC**（不变）：累计≤60%=A / 60-90%=B / >90%=C；
- ☐ **毛利ABC**（🆕）：按毛利额降序，累计≤60%=甲 / 60-90%=乙 / >90%=丙；
- ☐ **交叉裁决**（🆕）：

| 组合 | 身份 | 动作 |
|---|---|---|
| 销额A + 毛利甲/乙 | 核心引擎 | 死保+黄金位+不准断货 |
| 销额A + 毛利丙 | 走量低利 | 保留但控排面，找高毛利替代/搭售 |
| 销额C + 毛利甲 | 高利低销（金矿） | **不准砍**，上移/提曝光/促销拉动 |
| 销额C + 毛利丙 | 双低 | 进 Step4 误判4问 |

### Step 4｜误判校正（F）— 砍之前必答
> C类/双低 砍之前必答，任一"是"=禁止淘汰：
- ☐ 是**需求**吗？（儿童/健康/送礼/节日=低频有需求→留）
- ☐ 是**结构**吗？（中价挤压/同质冗余→调结构不砍）
- ☐ 是**位置**吗？（死角没被看见→先上移）
- ☐ 是**缺货**吗？（货架空导致0销量→补货不砍）
> 4问全"否" + 真有货积压 = 才能砍。
> （🆕 注：Step1.5 角色闸 + Step3 交叉裁决已前置拦截大部分误杀，本步是最后防线。）

### Step 5｜最终门店动作（G）
> 唯一输出 → **上架/下架 · 上移/下移 · 补货/清仓 · 保留/替换**

## ⚖️ v1.1 三条新铁律（并入 SOP-01~04 之后）
- **SOP-05 裁决双维**：砍/保看销量+毛利两维；高利低销不准砍，高销低利别堆资源。
- **SOP-06 A类水位**：A类按 `日均×到货×1.5` 补货，不等空；售罄太快=加量或提价。
- **SOP-07 角色优先**：先定角色再决汰留；目的品（引流/形象）不按销量砍。
