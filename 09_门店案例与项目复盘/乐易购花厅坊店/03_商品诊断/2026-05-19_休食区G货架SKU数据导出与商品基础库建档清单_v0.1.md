## 3. 必须导出的 5 张数据表

### 表 1：休食区 G 货架销售明细

**字段**（22 项）：
- 日期
- 条码
- 商品名
- 规格
- 单位
- 所属大类（如休闲 / 方便食品 / 冲调 / 等）
- 所属中类
- 所属小类
- 货架编号（如系统无 / 人工补）
- 子组
- 品牌
- 销量
- 销售额
- 毛利额
- 毛利率
- 售价
- 促销价
- 是否促销
- 供应商
- 门店编码
- 门店名称
- 备注

**时间范围**：
- ⭐ 最近 30 天（必）
- ⭐ 最近 60 天（必）
- ⭐ 最近 90 天（必）

如只能导一次 / 优先 **最近 90 天**（与 5/8 现有数据时段对齐 / 便于交叉验证）。

### 表 2：休食区 G 货架当前库存表

**字段**（15 项）：
- 条码
- 商品名
- 规格
- 单位
- 大类
- 中类
- 小类
- 货架编号
- 子组
- 当前库存数量
- 当前库存金额
- 最近进货日期
- 最近销售日期
- 库存天数（库存数量 / 30 天日均销量）
- 供应商
- 进价
- 售价

⚠️ **关键**：必须**全量**（不只是积压 / 不只是滞销）/ 含负库存 + 零库存 + 正常库存全部。

- 品牌

- 小类

- 联系方式
- 当前问题（响应慢 / 缺货频次 / 质量波动 / 等）

## 5. 货架映射规则

建立 7 层映射链：

```
条码 (主键)
  → 商品名 (POS 原始)
  → 品牌 (算法包清洗 + 人工复核)
  → 子类 (子类码 020XX)
  → 子组 (子类码 → 子组规则)
  → 货架 (子组 → 货架 / 现场实测 + 人工补)
  → DisplayMap (货架 → DisplayMap 版本)
  → 陈列动作 (调改决策)
```

## 6. 置信度分层

| 等级 | 判据 | 处理 |
|:---:|---|---|
| **A** | 系统字段完整 + 货架归属明确 + 人工确认 | 自动入库 |
| **B** | 系统字段完整 + 子类明确 + 货架待人工补 | 半自动 / 顾问抽检 10% |
| **C** | 商品可识别但品牌 / 子类 / 货架需复核 | 100% 人工复核 |
| **D** | 字段缺失严重 / 必须人工 | 标 dq_pending=true + 单列复核清单 |
