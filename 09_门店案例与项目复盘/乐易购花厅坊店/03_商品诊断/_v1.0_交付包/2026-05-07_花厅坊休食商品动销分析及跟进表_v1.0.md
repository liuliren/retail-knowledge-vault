## §3. ABCD 分级 + Z 异常算法（V4 ④ 标准 / v1.0.1 决策 3 微调 / 5/20 后实测校准）⭐

> [[M-DEC-007]] §3 S3 临界值（**本店本品类**校准 / 不可跨品类直接套）+ V4 ④ 诊断主表 ABCD 4 类：

### 3.1 ABCD 等级（字段 10 `abcd_grade`）

| 等级 | V4 ④ 定义 | 销售排序占比 | 累计销售贡献 | 调整方向 |
|---|---|---|---|---|
| **A** 重点维护 | 品类贡献 ≥ 15% + 动销 + 毛利 ≥ 10% | 前 20% | 70-80% | 黄金位 / 增排 / 不轻动 |
| **B** 稳定经营 | 品类贡献 5-15% + 动销 | 21-50% | 15-20% | 黄金辅助位 / 维持 |
| **C** 待优化 | 有动销但品类贡献 < 5% | 51-80% | 5-10% | 进 §S4 归因 / 调位 / 调价 |
| **D** 滞销整改 | 无动销 OR 周转 > 60 天 OR 毛利异常 | 后 20% | < 5% | **强制 §S4 归因** / 大概率淘汰 / 控采 / 调陈列 |

### 3.2 Z 异常标记（字段 39 `z_flag`）

| Z 异常情形 | 触发条件 | 处置 |
|---|---|---|
| 零动销 | sales_qty_90d = 0 | z_flag = Y / 进 §S4 归因 + 库存处置 |
| 数据异常 | match_confidence='冲突' / 价格 0 或负 / 条码错码 | z_flag = Y / 暂停决策 / 排查源数据 |
| 已停售 | sku_status='停售' 但仍在档 | z_flag = Y / 表 A 同步 status 修正 |

> **设计思路**（参 [[单品管理]] §7.2）：
> - **abcd_grade** = 当期贡献排名（A/B/C/D 4 类）
> - **z_flag** = 系统无法自动处理的异常状态（独立标记 / 与 abcd 配合）
> - 两者配合：`abcd_grade=D + z_flag=Y` → 最高优先淘汰候选；`abcd_grade=A + z_flag=N` → 重点维护

> **铁律**：单维度（销售）排序不够 — 必须 §1 字段 11-16 + 17-19 七维交叉后再分级（高销 0 毛利不能算 A）。

## §4. M-DEC-007 §3 S4 归因决策树（字段 28 取值规则）

```
低动销 SKU (字段 10 = C / Z)
  ├── 字段 9 shelf_position = 顶 / 底 / 边缘?
  │     ├── Y → attribution_tag = "陈列差" → S5 调位测试（不淘汰）
  │     └── N → 下一问
  ├── 字段 25 promotion_period_tag = 促销期? + 字段 14 sales_qty_90d 自然销量低?
  │     ├── 自然销量也低 → 下一问
  │     └── 仅活动期外低 → attribution_tag = "活动期外低" → S5 配活动 + 观察
  ├── 字段 16 retail_price (from 表 A) vs 市价 / 竞品 = 价格虚高?
  │     ├── Y → attribution_tag = "价格虚高" → S5 调价测试
  │     └── N → 下一问
  ├── 字段 27 stockout_rate_pct > 阈值 = 缺货导致?
  │     ├── Y → attribution_tag = "缺货" → 进缺货管理（不归因低销）
  │     └── N → 下一问
  ├── 字段 26 customer_segment_tag 与门店画像不符?
  │     ├── Y → attribution_tag = "客群不匹配" → S5 候选淘汰
  │     └── N → 下一问
  └── 真不需要?
        ├── Y → attribution_tag = "真不需要" → S5 淘汰
        └── N → attribution_tag = "健康" / 留观察
```

> **铁律**（[[M-DEC-007]] §4 / [[CLAUDE.md]] §13.6）：每个低动销 SKU 必须有归因标签，**不允许"销量低 → 直接淘汰"**。
