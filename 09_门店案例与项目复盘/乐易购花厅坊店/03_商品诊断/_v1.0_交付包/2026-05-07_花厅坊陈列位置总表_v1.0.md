# 花厅坊陈列位置总表 v1.0（统一货位主表 + 陈列资源单源真相）

> **核心定位**：vault 内**统一的陈列位置 + 陈列资源占有**单源主表（不嵌入表 A）。每行 = 1 个独立货位（含空置）。
>
> **解决的关键问题**（用户 5/7 阐述）：
> 1. **统一陈列位置编码**（5 段制 ZTGSL / 端架 / 堆头 特殊编码）
> 2. **陈列资源占有量化**（位置等级 / 可见性 / 客流量 / 物理面积 / 排面宽度）
> 3. **真正联动 6 表**（表 A × 表 E 商品-货位多对多 / 表 B 货位坪效 / 表 D 释放货位 / 引进表拟放货位）
> 4. **落到品类管理 4 步骤实际落地**（策略 → 战术 → 执行 → 复盘 / 货位作为资源决策单元）
>
> **设计依据**：
> - **V4 工具包 ⑫ 陈列编码总表**（16 字段 / 5 段制 ZTGSL / 4 级位置等级）— **完全复用**
> - V4 工具包 ⑬ 商品-陈列映射（join view / 不独立 vault .md）
> - V4 工具包 ⑭ 坪效分析（aggregate view / 不独立 vault .md）
> - [[花厅坊门店通道命名表_v0.1]]（通道编号底图）
> - [[M-DEC-006_陈列层数确认与区域落位SOP_v0.1_候选]]（层数实测纪律）
> - [[feedback_display_layout_confirmation]]（实测红线）
>
> **使用方式**：vault `.md` 锁字段定义 + 5 段制编码规范 + 反幻觉红线；本地 Excel 装实测货位数据（约 200-400 个货位）。
>
> **更新频率**：季度大盘点（物理属性）/ 实时（占用关系 / 调整任务）。

## §1. 5 段制货位编码规范（V4 ⑫ 标准 / vault 复用）

### 1.1 普通货架编码

```
位置编码 = 区域(Z) - 通道(T) - 货架组(G) - 货架号(S) - 层板(L)

示例：
A-T1-G1-S01-L2  ← 休食区A / 通道1 / 货架组1 / 货架01 / 第2层（眼平层）
B-T2-G1-S01-L3  ← 水饮区B / 通道2 / 货架组1 / 货架01 / 第3层
C-T3-G1-S01-L2  ← 日用品区C / 通道3 / 货架组1 / 货架01 / 第2层
```

### 1.2 区域编码（Z）

| 区域代码 | 区域名 | 备注 |
|---|---|---|
| A | 休食区 | 含薯片 / 饼干 / 坚果 / 糖巧 等 12 个 L3 子品类 |
| B | 水饮区 | 含矿泉水 / 牛奶 / 饮料 等 |
| C | 日用品区 | 含纸品 / 洗护 等 |
| D | 米面粮油区 | 含大米 / 食用油 / 调味 等 |
| E+ | 后续扩展 | 待门店实测确定 |

### 1.3 通道编号（T / 与 [[花厅坊门店通道命名表_v0.1]] 对齐）

| 通道代码 | vault 通道命名表对应 |
|---|---|
| T0 | 垂直主通道 00 |
| T1 | 垂直主通道 01 |
| T2 | 垂直主通道 02 |
| T3 | 垂直主通道 03 |
| TC | 垂直中央主通道 |
| TH | 水平主通道 02 |
| Tin | 进门 / 入门主通道 |
| Tout | 出口 / 出口主通道 |

> **铁律**：通道编号必须与 [[花厅坊门店通道命名表_v0.1]] 实地核实后的 v1.0 一致 / 当前 v0.1 confidence=low / 待二访核验后升 v1.0。

### 1.4 层板编码（L）

| 层代码 | 层位 | 高度参考 cm | 默认等级 |
|---|---|---|---|
| L1 | 顶层 | ≥ 160 | 普通位 / 弱势位 |
| L2 | 眼平层 | 120-160 | **黄金位** ⭐ |
| L3 | 腰部层 | 80-120 | 次优位 |
| L4 | 底层 | < 80 | 弱势位 |

> **铁律**（[[M-DEC-006]] / [[feedback_display_layout_confirmation]]）：货架层数**因货架不同而异 / 禁止假设全 7 层 / 必须现场实测**。L1-L4 是默认 4 层模型，遇异常货架需现场核实层数后扩展（如 L5 / L6）。

### 1.5 特殊编码（端架 / 堆头）

| 类型 | 编码格式 | 示例 |
|---|---|---|
| 端架 | `END-{区域}-{通道}-{方位}` | `END-A-T1-L`（休食区A 通道1 左侧端架）/ `END-A-T1-R` |
| 堆头 | `DH-{区域}-{序号}` | `DH-B-01`（水饮区B 第1个堆头）/ `DH-D-01` |
| 收银端 | `POS-{序号}` | `POS-01` / `POS-02`（5/6 二访读图 §6 已观察 = 肉脯专架 / 奇趣蛋专架）|
| 风幕柜 | `FW-{序号}` | `FW-B-01`（水饮区风幕柜）|

## §2. 字段定义（19 字段 / 6 维度）

> 复用 V4 ⑫ 16 字段 + 加 3 治理字段。字段命名英文 `snake_case`，与表 A/B/C/D/引进表 统一。

### A. 编码 + 类型（3 字段 / 主键）

| # | 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| 1 | `location_code` | STRING PK | ✅ | 位置编码（V4 ⑫ 5 段制 ZTGSL / 普通货架 + 端架 + 堆头 + 收银端 + 风幕柜统一编码）|
| 2 | `location_type` | ENUM | ✅ | 普通货架 / 眼平层 / 主通道货架 / 端架 / 堆头 / 收银端 / 风幕柜 |
| 3 | `area_name` | STRING | ✅ | 所属区域（休食区A / 水饮区B / 日用品区C / 米面粮油区D / 等）|

### B. 5 段制详细（5 字段）

| # | 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| 4 | `aisle_code` | STRING | ✅ | 通道编号（T0/T1/T2/T3/TC/TH/Tin/Tout / 与 [[花厅坊门店通道命名表]] 对齐）|
| 5 | `shelf_group` | STRING | ✅ | 货架组（G1/G2/...）|
| 6 | `shelf_number` | STRING | ✅ | 货架编号（S01/S02/...）|
| 7 | `layer_code` | STRING | ✅ | 层板代码（L1顶 / L2眼平 / L3腰 / L4底 / 端架 = "—" / 堆头 = "—"）|
| 8 | `layer_height_cm` | DECIMAL(5,1) | ✅ | 层板高度 cm（实测）|

### C. 物理属性（2 字段 / 实测）

| # | 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| 9 | `facing_width_cm` | DECIMAL(5,1) | ✅ | 排面宽度 cm（货位可容纳的最大宽度）|
| 10 | `area_m2` | DECIMAL(5,2) | ✅ | 货位面积 m²（坪效计算分母 / V4 ⑭ 用）|

### D. 等级评估（3 字段 / V4 ⑫ 4 级标准）

| # | 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| 11 | `position_grade` | ENUM | ✅ | **4 级**：黄金位 / 次优位 / 弱势位 / 普通位 |
| 12 | `visibility` | ENUM | ✅ | **4 级**：极高 / 高 / 中 / 低 |
| 13 | `traffic_flow` | ENUM | ✅ | **4 级客流量**：极高 / 高 / 中 / 低 |

> **位置等级判定参考**：
> - **黄金位**：眼平层（L2）+ 端架 + 堆头 + 主通道货架 + 收银端
> - **次优位**：腰部层（L3）+ 主通道边缘货架
> - **普通位**：顶层（L1）+ 普通货架
> - **弱势位**：底层（L4）+ 边缘弱客流货架

### E. 当前占用（3 字段 / FK→ 表 A / 多对多关系核心）

| # | 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| 14 | `current_barcode` | STRING FK→A1 | 否 | 当前商品条码（空置 = NULL / 待规划 = "TBD"）|
| 15 | `current_sku_facing` | INTEGER | 否 | 当前 SKU 占用排面数（如 1 / 2 / 3 / 0=空置）|
| 16 | `current_category_l3` | STRING | 否 | 当前商品 L3 子品类（FK→ 表 C / 冗余 from 表 A）|

> **多对多支持**：
> - 1 个 SKU 占多排面 → `current_sku_facing` ≥ 2
> - 1 个 SKU 占多货位（如主货位 + 端架）→ 1 行 / 货位独立 + `current_barcode` 都填该 SKU
> - 1 个货位空置 → `current_barcode` = NULL / `current_sku_facing` = 0
> - 1 个货位待规划（如端架待引进端架商品）→ `current_barcode` = "TBD"

### F. 治理（3 字段 / 反幻觉硬约束）

| # | 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| 17 | `measurement_status` | ENUM | ✅ | 待实测 / 已实测 / 已 sign off（参 [[feedback_display_layout_confirmation]]）|
| 18 | `measurement_date` | DATE | 否 | 实测日 |
| 19 | `notes` | TEXT | 否 | 备注（特殊情况 / 调整建议 / 实测异常）|

## §3. 6 表联动架构（用户 5/7 强调"真正联动"）

```
                    ┌────────────────────────┐
                    │ 表 E 陈列位置总表 v1.0  │  ← 货位单源真相

                          │      │      │
          ┌───────────────┘      │      └───────────────┐
          ↓                      ↓                      ↓
  ┌──────────────┐      ┌──────────────┐      ┌──────────────┐
  │ 表 A 商品基础│      │ 表 B 动销分析│      │ 引进表 v1.0   │
  │ (字段 25-28  │      │ (字段 5-9    │      │ (字段 26     │
  │ 货架冗余 /   │      │ 现场摸排 /   │      │ proposed_     │
  │ FK→ 表 E    │      │ FK→ 表 E)    │      │ location_code │
  │ location_   │      │ + 坪效计算)  │      │ FK→ 表 E)    │
  │ code)       │      └──────────────┘      └──────────────┘
  └──────────────┘             │
          │                    │
          ↓                    ↓
  ┌──────────────┐      ┌──────────────┐
  │ 表 D 淘汰表  │      │ 表 C 品类表  │
  │ (字段 17     │      │ (V4 ⑭ 区域   │
  │ released_    │      │ 坪效 aggreg) │
  │ facing_count │      └──────────────┘
  │ → 释放货位)  │
  └──────────────┘
```

### 3.1 联动 1：表 A 商品 ↔ 表 E 货位（V4 ⑬ 商品-陈列映射）

```sql
-- Excel 实操（VLOOKUP）：每个在售 SKU 在表 E 中找主货位
-- 联动键：表 A 字段 1 barcode = 表 E 字段 14 current_barcode
```

### 3.2 联动 2：表 E × 表 B 销售 = V4 ⑭ 货位坪效

```
货位坪效 = 表 B 90d 销售额 (字段 15) / 表 E 货位面积 m² (字段 10) / 90 天
        = 元 / m² / 天

坪效评级（V4 ⑭ 标准）：
  优秀 > 50
  良好 20-50
  一般 10-20
  差   < 10 / 零动销
```

### 3.3 联动 3：表 D 淘汰 → 表 E 释放货位

```
表 D 字段 14 delisted_date 触发：
  表 E 字段 14 current_barcode = NULL
  表 E 字段 15 current_sku_facing = 0
  表 E 字段 19 notes 加 "释放于 [日期] / 替换待规划"
```

### 3.4 联动 4：引进表 → 表 E 拟放货位

```
引进表字段 26 proposed_location_code = 表 E 字段 1 location_code
确认 sign off + 上架后：
  表 E 字段 14 current_barcode = 引进表字段 2 candidate_barcode
  表 E 字段 17 measurement_status = "已 sign off"
```

### 3.5 联动 5：表 C × 表 E aggregate = 区域坪效

```
表 C 字段 7-13 占比数据 = aggregate (表 E group by area_name) × 表 B sales

V4 ⑭ 区域坪效输出（如）：
  休食区A 28 m² × 6 货架 × 45 SKU → 90d 销售 2182 元
  → 区域坪效 0.9 元 / m² / 天 → 评级"差" → 优化方向：淘汰 D 类 + A 类扩排面
```

## §4. 品类管理 4 步骤实际落地（[[品类管理]] §3.5-§3.8）

| 步骤 | 货位资源决策点 | 表 E 字段 |
|---|---|---|
| **§3.5 品类策略** | 哪些品类资源加大 / 哪些收缩？ | aggregate area_m2 + position_grade by category_l3（联动表 C）|
| **§3.6 品类战术** | 陈列空间分配（货架面数 / 层位 / 端架 / 堆头）| 字段 7 layer_code + 字段 11 position_grade + 字段 2 location_type |
| **§3.7 品类执行** | 按 DisplayMap 完成货架陈列 | 字段 14-16 当前占用 / 字段 17 sign off |
| **§3.8 品类复盘** | 陈列效果是否影响动销 | 联动表 B 坪效（§3.2）+ 字段 19 调整建议 |

> **没有表 E 时**：上述 4 步只能定性谈"加大资源" / 无法量化执行 + 无法复盘。
>
> **有表 E 后**：每个货位是资源单元 / 决策可量化（"释放 0.16 m² 端架 / 让给 T08 果冻布丁 / 预期坪效 +50"）+ 复盘可追溯。

## §6. 数据装载工作流

```
S1 大盘点（首次装表 / 1-2 天）
   → 全店货位现场测绘 / 卷尺实测层板 + 排面 + 面积
   → 字段 1-13 全填
   ↓
S2 现状映射（首次 / 半天）
   → 现有 SKU 占用关系（字段 14-16）
   → measurement_status='已实测'
   ↓
S3 老板 / 店长 sign off（半天）
   → 字段 17 measurement_status='已 sign off'
   → 字段 18 measurement_date 填
   ↓
S4 联动激活
   → 表 A 字段 25-28 货架字段刷新（VLOOKUP from 表 E）
   → 表 B 字段 5-9 现场摸排 FK→ 表 E
   → 表 D / 引进表 货位字段联动
   ↓
S5 周更（动态字段）
   → 字段 14-16 当前占用关系
   → V4 ⑭ 坪效自动计算（联动表 B）
   ↓
S6 季度大盘点
   → 字段 8-10 物理属性复测
   → 字段 11-13 等级评估复盘
```
