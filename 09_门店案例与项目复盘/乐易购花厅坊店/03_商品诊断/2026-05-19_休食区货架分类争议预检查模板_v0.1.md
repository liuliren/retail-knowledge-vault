# 休食区货架分类争议预检查模板 v0.1

## 1. 文件用途

**强制使用场景**：每个货架调改前 / 顾问 + 佳妮 共同 ≤ 5 分钟完成本表 / 不完成 = 不进入正式调改。

**回答 1 个问题**：本货架本次调改的 SKU 池中 / 是否存在 ERP 分类 ↔ 商品名 / 实物位置 / 采购认知不一致的争议 SKU / 这些争议 SKU 必须暂缓现场动作。

**不解决**：
- SKU 是否下架（参治理规则 §6 现场核验 SOP 后再判）
- ERP 是否修改（vault 不改 ERP）
- 全品类清洗（本模板仅本次调改 SKU 池）

---

## 2. 触发场景

| # | 触发 | 说明 |
|:---:|---|---|
| 1 | 每个货架调改前 | 强制 / G4-A / G2-B / G3-A / G1-B / G2-A / G1-A 各一份 |
| 2 | 每次从 ERP / POS 按类目筛 SKU | 数据筛选环节即触发 |
| 3 | 每次发现商品名与类目不一致 | 现场即时填 |
| 4 | 每次现场商品不在预期货架 | 现场即时填 |
| 5 | 每次采购说法与 ERP 不一致 | 现场即时填 |

## 3. 预检查表（12 字段 / 每货架 1 份）

| 字段 | 取值 / 说明 |
|---|---|
| **日期** | YYYY-MM-DD（如 2026-05-19）|
| **货架** | G4-A / G2-B / G3-A / G1-B / G2-A / G1-A / G4-B / G3-B 复核 |
| **数据来源** | ERP（D-XX 编号）/ POS / 手工 / 现场观察 |
| **SKU 总数** | 本次检查 SKU 数（如 G4-A = 95 SKU）|
| **疑似争议 SKU 数** | 数量 |
| **争议 SKU 清单** | 条码 + 商品名（如 `6914973605688 德芙便携罐装小巧粒牛奶 42g`）|
| **争议类型** | ERP 分类冲突 / 陈列位置冲突 / 商品名冲突 / 跨品类 / 多重 |
| **现场动作** | 暂缓 / 核验（§6 SOP）/ 拍照 / 问采购 |
| **责任人** | 佳妮 / 启明 / 6 哥 / 顾问 |
| **是否影响调改** | 是 / 否（"否"= 不影响整体调改 / 仅个别 SKU 暂缓）|
| **后续动作** | 调整基础库 / 保留 ERP / 暂缓 / 待 §6 SOP 完成后决定 |
| **fact_layer** | observed / client_told / inferred / pending |

---

## 4. 预检查 6 项内容（按治理规则 §4 6 类争议判定）

| # | 检查内容 | 检查方法 |
|:---:|---|---|
| 1 | 当前货架 SKU 是否来自 ERP / POS 类目筛选 | 看 SKU 池来源（D-XX）/ 是 → 必跑 §2-§6 检查 |
| 2 | 是否存在商品名与 ERP 类目不一致 | 用商品名常识扫 SKU 名（如"德芙"在冲调 SKU 池 → 触发）|
| 3 | 是否存在商品实际陈列位置与 ERP 类目不一致 | 现场快走 / SKU 在非 ERP 归属货架 = 触发 |
| 4 | 是否存在跨品类商品 | 多场景商品（如红枣可冲调 / 可零食）= 触发 |
| 5 | 是否需要进入分类争议池 | 上 4 项任一命中 = 进入争议池 |
| 6 | 分类争议 SKU 是否暂缓现场动作 | 全部争议 SKU 必须**暂缓所有陈列动作**（不下架 / 不扩排 / 不重点保留 / 不弱动销处理 / 只核验）|

## 6. 输出与归档

### 6.1 即时输出（现场）

- 现场快采（纸笔 / 微信 / 现场表格）
- 必须**调改启动前**完成 / 否则该货架调改 NO-GO

### 6.2 EOD 归档

- 6 哥晚整理 → 本模板单页 markdown 文件 / 落档 `09_/03_商品诊断/<日期>_<货架>_分类争议预检_v0.1.md`
- 当日多货架 → 多份独立文件 / 不合并（便于检索）

### 6.3 W22 沉淀

- 多份预检表 → 累积成 [[2026-XX-XX_花厅坊商品分类争议池_v0.5]]（治理规则 §5.2 v0.5）

## 7. 红线

| # | 红线 |
|:---:|---|
| 1 | ❌ 预检未完成 = 不允许该货架进入调改 |
| 2 | ❌ 不擅自把"疑似争议"写成"分类错误" |
| 3 | ❌ 不未现场核验就调整基础库 |
| 4 | ❌ 不仅凭 ERP 子类码 / 仅凭商品名下结论（必须按治理规则 §3 6 级优先级）|
| 5 | ❌ 不在 vault 内改 ERP |
| 6 | ❌ 不擅自下架争议 SKU（必须按治理规则 §6 SOP）|
