# 商品基础库分类争议治理规则 v0.1

## 1. 文件用途

本文件用于处理 **POS / ERP 分类、商品名认知、实物陈列位置、采购认知**之间不一致的问题。

**不解决**：
- SKU 是否下架的最终决策
- ERP 系统数据是否修改
- 全品类商品归属清洗结果
- 商品基础库全字段治理

**只解决**：
- 出现"ERP 标 A / 商品名疑似 B / 实物可能在 C / 采购可能认为是 D"四方不一致时 / 如何登记 / 核验 / 暂缓处理 / 沉淀
- 建立可复用的"分类争议池"+ 字段表 + 现场核验 SOP + 应用边界

**适用范围**：花厅坊店全品类 / 后续扩展至沙埔大道店及其他门店。

## 3. 分类依据优先级

⭐ **核心规则**：同一 SKU 分类出现冲突时 / 按以下 6 级依据决策 / **不得越级**。

| 优先级 | 分类依据 | 说明 | 强度 |
|:---:|---|---|:---:|
| **P1** | **实物陈列位置** | 商品实际在哪个货架（必须现场拍照确认）| 🔴 最强 |
| **P2** | **商品名常识** | 品名是否明显属于另一品类（如"德芙"+"小巧粒"明显是巧克力）| 🟡 强 |
| **P3** | **采购责任人认知** | 佳妮 / 启明等对商品归属的判断（含历史采购意图）| 🟡 强 |
| **P4** | **标准品类表 V4.0** | 晟果大湾区品类表 V4.0（公司唯一标准 / 参 [[feedback_品类标准_大湾区品类表V4.0_唯一标准]]）| 🟢 中 |
| **P5** | POS / ERP 子类码 | 初始数据依据 / **不得单独作为最终归属** | 🟢 弱 |
| **P6** | 历史陈列习惯 | 可参考 / **不得优先于实物和标准品类** | ⚪ 最弱 |

### 3.1 决策口径

- ✅ P1 与 P5 冲突 → 优先 P1（实物为准）
- ✅ P2 与 P5 冲突 → 优先 P2（商品名常识为准）
- ✅ P3 与 P5 冲突 → 优先 P3（采购认知为准）
- ✅ P4 与 P5 冲突 → 优先 P4（公司标准为准）
- 🔴 P1 + P2 + P3 三者一致 → **强烈支持**调整商品基础库归属 / 但仍需 P4 标准品类表对照
- 🟡 P1 + P2 + P3 不一致 → 进入争议池 / 不强行下结论

## 4. 分类争议判定规则

出现以下**任一**情况 / 该 SKU **必须**进入争议池：

| # | 判定条件 | 示例 |
|:---:|---|---|
| 1 | ERP 子类码与商品名常识冲突 | 德芙小巧粒（巧克力名）/ ERP 标冲调 |
| 2 | ERP 归属与实际陈列位置冲突 | ERP 标冲调 / 实物在 G2-B 糖巧架 |
| 3 | 商品跨品类使用场景明显 | 红枣（可冲调 / 可休闲零食 / 可烘焙原料）|
| 4 | 采购认知与 POS 分类不一致 | 佳妮采购时按糖巧入库 / POS 标冲调 |
| 5 | 同一品牌 / 同一商品在不同报表中归属不一致 | D-06 标冲调 / D-08 休闲库存积压表也有 |
| 6 | 调改现场发现商品实际不在数据筛选对应货架 | D-06 95 SKU 中 X SKU 现场 G4-A 找不到 |

## 6. 现场核验 SOP（≤ 5 分钟 / 顾问 + 采购）

### 6.1 6 步动作

| 步 | 动作 | 责任人 | 输出 |

| 1 | 在 ERP 归属货架（如 G4-A 冲调）寻找该 SKU | 顾问 + 佳妮 | 在/不在 |
| 2 | 在商品名初判归属货架（如 G2-B 糖巧 / 相关休食货架）寻找该 SKU | 顾问 + 佳妮 | 在/不在 |
| 3 | 拍 ≥ 1 张实物陈列照片（命名 v1.0 资源位）| 顾问 | 文件名 |
| 4 | 记录货架位（命名 v1.0 三轴：如 `LYG-HTF-XS-G2-B-03-L02-P05`）| 顾问 | 文本 |
| 5 | 问佳妮该商品实际由谁负责采购 / 历史采购意图 | 顾问 | 文本 |
| 6 | 判断是否需要调整商品基础库归属 + 形成建议（**不现场修改 ERP**）| 6 哥 + 佳妮 | 文本建议 |
