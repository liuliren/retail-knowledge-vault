# 2026-05-13_花厅坊门店商品诊断与调改阶段作业包设计稿_v0.1

## 1. 文件定位

- 本文件定义**花厅坊门店商品诊断与调改各阶段的标准作业包**。
- **不是**具体数据分析报告。
- **不输出**商品结论。
- **不生成** SKU 调整清单。
- 用于明确每个阶段要准备哪些**表格 / 清单 / 照片 / 日志 / 报告**。

**上游依赖**：
- BUS-DATA-001 模块设计（[[2026-05-13_花厅坊商品数据底座与动销分析模块设计稿_v0.1]]）
- BUS-DATA-002 字段模板（[[2026-05-13_花厅坊商品基础档案字段模板_v0.1]]）
- BUS-DATA-002A 字段口径修正（[[2026-05-10_花厅坊_供应商与进价字段口径confirm_v0.1]]）

**下游路径**：BUS-DATA-004 数据导入与清洗任务单 / BUS-DATA-005 方便速食 v0.6 承接 / 等。

## 2. 总体原则（11 条）

| # | 原则 | 含义 |
|---|---|---|
| 1 | **新数据导入后优先更新表格** | 优先更新标准表格 / 清洗表 / 分析表 / 日志 / 复盘表 |
| 2 | **不应频繁修订原方法论文档或历史报告** | 方法论是上游 / 不被业务波动反复改 |
| 3 | **原始数据保留原貌** | 99_原始素材 + 原始 POS 导出 / 不动 |

| 6 | **结论进入报告** | 诊断 / 调改建议 / 复盘 |
| 7 | **流程问题才反哺 SOP 或方法论** | 升 v0.5 → v1.0 后再回写 |
| 8 | **照片必须遵守照片命名规范** | [[2026-05-06_门店陈列照片命名规范_v0.3]] v0.4 §7.2 |
| 9 | **不确定字段必须标记待核验** | _待核验 / is_pending_check |

## 3. 数据进入系统后的分层规则（4 层）

```
┌─────────────────────────────────────────────────────────┐
│ 层 1：原始数据层                                          │
│   - POS 导出（销售 / 库存 / 商品档案）                     │
│   - 原始照片（手机 / 相机 / 截图）                        │
│   - 访谈原始记录（录音 / 文字）                            │
│   - 客户原始材料                                          │
│   位置：99_原始素材/ 或对应原始素材目录                    │
│   规则：保留原貌 / 不修改                                  │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│ 层 2：标准化数据层                                        │
│   - 商品基础档案（按 BUS-DATA-002 字段模板）                │
│   - 销售明细导入表                                        │
│   - 库存导入表                                            │
│   - 照片索引表（按 §4 规则）                              │
│   位置：03_商品诊断/ 或对应业务模块目录                    │
│   规则：新数据进入则更新此层                              │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│ 层 3：分析输出层                                          │
│   - 动销分析表 / 商品结构表 / 淘汰候选表 / 异常数据清单    │
│   - 各子组分析子表                                        │
│   位置：03_商品诊断/ + 子组_<X>/                          │
│   规则：基于层 2 计算 / 自动衍生 / 可重跑                  │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│ 层 4：报告结论层                                          │
│   - 诊断报告 / 调改建议 / 复盘报告                         │
│   位置：03_商品诊断/ 或子组_<X>/06_复盘/                  │
│   规则：基于层 3 综合 / 含人工判断 / 含 fact_layer        │
└─────────────────────────────────────────────────────────┘
```

**核心规则**：

```
✅ 新数据进入 → 更新层 2 / 层 3 / 层 4
❌ 不直接改层 1 原始数据
❌ 不直接改方法论 SOP（仅当流程问题反哺）
✅ 历史报告保留 / 不反复修订（仅 audit log + 加注 / 不动正文）
```

## 4. 外部照片导入规则

### 4.1 5 条硬规则

| # | 规则 | 反例 |
|---|---|---|
| 1 | 外部照片可以导入 | — |
| 2 | 原始照片进入 99_原始素材 / 99_/00_客户原始材料 / 等原始素材目录 | 直接放分析报告里 |
| 3 | 用于分析的照片**必须建立照片索引表** | 用于分析但无索引 → 后续无法追溯 |
| 4 | 进入报告 / DisplayMap / 复盘的照片**必须按命名规范处理** | 用 IMG_8825.jpg 进 DisplayMap |
| 5 | 不允许用未命名 / 无索引的照片直接进入诊断结论 | "现场看到 X 缺货" 但无照片证据 |

**照片命名规则**（参 [[2026-05-06_门店陈列照片命名规范_v0.3]] v0.4 §7.2）：
- 字段内部用 `-`
- 字段之间用 `_`
- 标准示例：`HS-G3-A-04-3-L冲调-黄金视线_商品价签_20260513_常规.jpg`

**不立即重命名的处理**：
- 必须在**照片索引表**中记录 `原始文件名` + `标准文件名` 双轨
- 后续批量改名脚本可基于索引表回写

### 4.2 照片索引表字段（建议 16 字段）

| # | 字段 | 类型 | 说明 |
|---|---|---|---|
| 1 | original_filename（原始文件名）| string | 如 IMG_8825.jpg |
| 2 | std_filename（标准文件名）| string | 按命名规范 v0.4 |
| 3 | photo_date（拍摄日期）| date | YYYY-MM-DD |
| 4 | store（门店）| string | 花厅坊店 |
| 5 | physical_zone（区域 / P 物理区域代号）| string | HS / SX / SC / 等 10 代号 |
| 6 | g_shelf_code（G 货架编号）| string | G3-A / G4-B |
| 7 | group_position（组位）| string | 04（2 位前导零）|
| 8 | layer_position（层位）| int | 1=顶 / 末=底 |
| 9 | l_main_category（L 主品类）| string | L冲调 / L水饮 |
| 10 | content_type（内容字段）| string | 16 锁定值 |
| 11 | time_period（时段）| string | 4 锁定值（夜灯前 / 夜灯后 / 次晨白光 / 常规）|
| 12 | camera_position（机位 M1-M6）| string | M1-M6（参 SOP_6机位拍摄法 v0.5）|
| 13 | photo_purpose（照片用途）| string | 调改前 / 调改后 / 24h复拍 / 现场诊断 / 异常证据 / 等 |
| 14 | is_renamed（是否已重命名）| bool | true / false |
| 15 | is_pending_check（是否待确认）| bool | — |
| 16 | remark（备注）| string | 自由文本 |

## 9. 后期跟进循环周期（5 个）

### 9.1 7 天复盘

**目标**：
- 看执行是否到位
- 看陈列是否按方案落地
- 看缺货 / 价签 / 到货是否异常

**输出**：
- 7 天执行复查表
- 现场复拍清单
- 缺口整改清单

### 9.2 15 天复盘

**目标**：
- 看初步动销变化
- 看高风险商品是否改善
- 看新品 / 补充商品是否开始动销

**输出**：
- 15 天初步动销复盘表
- 商品动作跟踪表
- 异常商品清单

### 9.3 30 天复盘

**目标**：
- 看销售 / 毛利 / 库存结构变化
- 看调改是否进入稳定状态
- 判断是否进入下一轮优化

**输出**：
- 30 天阶段经营复盘报告
- 商品结构变化表
- 下一轮优化建议

### 9.4 2 个月复盘

**目标**：
- 看结构是否稳定
- 看供应商 / 库存 / 动销是否匹配
- 看品类角色是否需要调整

**输出**：
- 2 个月结构稳定性复盘表
- 供应商与库存复盘表
- 品类角色复核表

### 9.5 3 个月复盘

**目标**：
- 看是否形成长期经营改善
- 看是否具备复制到其他门店的价值
- 判断是否进入长期陪跑

**输出**：
- 3 个月长期效果复盘报告
- 可复制经验清单
- 长期陪跑建议

## 10. 表格前期加工标准

**所有表格在进入分析前必须完成 11 项**：

| # | 加工动作 | 字段 / 规则 |
|---|---|---|
| 1 | 字段标准化 | BUS-DATA-002 §3-§9 字段模板 |
| 2 | 条码主键确认 | barcode 唯一 / 去前导零 |
| 3 | 商品名清洗 | 去空格 / 去全半角差异 / 去无关后缀 |
| 4 | 品类匹配 | 系统品类 → V4.0 L1-L4 |
| 5 | 供应商匹配 | primary_supplier / 5/12 拉数后 |
| 6 | 进价 / 售价 / 毛利率校验 | 毛利率回算 = （售价-进价）/ 售价 |
| 7 | 库存异常识别 | 负库存 / 高库存 / 缺货 |
| 8 | 负库存标记 | dq_negative_stock |
| 9 | 活动 / 大盘影响标记 | is_promotion / 客流字段 |
| 10 | 数据来源记录 | data_source 字段 |
| 11 | 待核验字段标记 | is_pending_check / dq_* |

**核心警示**：

```
❌ 简单清洗不够（去空格 / 去重 / 等）
✅ 必须有字段模板（BUS-DATA-002）+ 校验规则 + 待核验机制
❌ 否则会导致分析失真 / 反复返工 / 跨子组口径不一致
```

| 字段类 | 含义 | 例 |
