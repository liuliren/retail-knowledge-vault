# 花厅坊新商品引进表 v1.0（综合分析结果性表格 / 2 来源闭环）

> **核心定位**：新商品引进表是**综合分析的结果性表格**（不是单源候选清单），输入来自 2 个来源，输出形成全生命周期闭环：
>
> ```
> 来源 1：内部诊断（淘汰反推）
>   品类分析（C）+ 单品管理诊断（B）→ 淘汰（D）→ 释放排面 + 补缺需求
>
> 来源 2：外部市场（综合扫描）
>   ① 市场最新变化（行业趋势 / 竞品截流）
>   ② 时令（节庆 / 季节 / 52周MD）
>   ③ 新品（供应商推荐 / 行业新品 / 顾客需求 / 老板战略）
>
> 综合考虑 → 引进表（本表）→ sign off + 试销 → 表 A（在售）→ 闭环
> ```
>
> **设计依据**：
> - **V4 工具包 ⑪ 新品引进清单**（13 字段 / 价格带缺口 + D 类替换 logic）
> - 旧版 [[2026-05-07_花厅坊新商品引进表_v1.0]]（20 字段 / 引进规则 G-1 至 G-5 + 5 反引进红线）
> - 用户 2026-05-07 logic 阐述（2 来源综合）
> - [[单品管理]] §9 新品引进机制 + §11 候选商品池
> - [[V4 ⑰ 任务生命周期跟进]] 新品 4 节点（第 7 / 15 / 30 / 60 天）
>
> **使用方式**：vault `.md` 锁字段定义 + 引进规则 + 反幻觉红线；本地 Excel 装实测候选池数据。
>
> **更新频率**：周更（候选评估）+ 决策日实时更新（sign off 状态）+ 试销节点（4 时点）。

## §1. 全生命周期闭环（5 表联动 + V4 工具包对齐）

```
┌── 来源 1：内部诊断（淘汰反推）──────────────────────┐
│                                                      │
│  [表 A 商品基础表]（在售档案）                       │
│         ↓ 品类分析（C）+ 单品管理诊断（B）           │
│  [表 B 动销分析]（六维 / ABCD / 归因决策树）         │
│         ↓ decision_action='淘汰' + sign off          │
│  [表 D 商品淘汰表]（释放排面 + 淘汰原因）            │
│         ↓ released_facing_count                      │
│         ↓ discontinue_reason → 反推同位补缺          │
│                                                      │
└──────────────┬─── 来源 1 输入 ──────────────────────┘
               │
               ▼
       ┌──── 综合考虑 ────┐
       │                  │
       │  本表（引进 v1.0）│  = 综合分析结果性表格
       │                  │
       └────┬─────────────┘
            ▲
            │
┌──── 来源 2：外部市场（综合扫描）────────────────────┐
│                                                      │
│  ① 市场最新变化（[[V4 ⑪]] 价格带缺口 + 竞品截流）   │
│  ② 时令（[[52周MD]] / [[大湾区全年节庆节奏表]] / 节庆）│
│  ③ 新品（供应商推荐 / 行业新品 / 顾客反馈 / 老板战略）│
│                                                      │
└──────────────────────────────────────────────────────┘
            ↓
      sign off + 试销期（30 / 60 / 90 天）
            ↓
      跟进生命周期（V4 ⑰ 4 节点：第 7 / 15 / 30 / 60 天）
            ↓
      回到 [表 A]（sku_status='在售'）← 闭环完成
            ↓
      进入 [表 B] M-DEC-007 7 步 SOP 评估
```

## §2. 字段定义（29 字段 / 8 维度）

> 字段命名英文 `snake_case`，与表 A/B/C/D 统一。

### A. 主键 + 关联（4 字段）

| # | 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| 1 | `introduce_id` | STRING PK | ✅ | 引进 ID 自增（如 `INTR-2026-001`）|
| 2 | `candidate_barcode` | STRING | 否 | 候选条码（pre-在售可空 / 正式上架后填）|
| 3 | `candidate_sku_name` | STRING | ✅ | 候选品名（含品牌 + 规格 / 不必同 raw 标准）|
| 4 | `candidate_brand` | STRING | ✅ | 候选品牌 |

### B. 2 来源分类（核心 / 4 字段）⭐

| # | 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| 5 | `source_category` | ENUM | ✅ | **来源 1 内部诊断** / **来源 2 外部市场** / 混合 |
| 6 | `source_subtype` | ENUM | ✅ | 7 类：排面释放反推（来源 1）/ 品类宽度补缺（来源 1）/ 价格带缺口（来源 1+2）/ 市场新品（来源 2）/ 时令节庆（来源 2）/ 战略必引（来源 2）/ 其他 |
| 7 | `discontinue_link_barcode` | STRING FK→D1 | 否 | 来源 1 时填 / 链接淘汰 SKU 条码 |
| 8 | `source_evidence` | TEXT | ✅ | 来源凭证：① 来源 1 = "替换 [淘汰条码] / 释放 [N] 排面"；② 来源 2 = "竞品 [赵一鸣 X] 同位 / 顾客需求 [X] / 时令 [春节 W04]" |

### C. 品类（4 字段）

| # | 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| 9 | `category_l1` | STRING | ✅ | 大类（休食饮料）|
| 10 | `category_l3` | STRING FK→C3 | ✅ | 12 子品类之一（T01-T12）|
| 11 | `spec` | STRING | ✅ | 规格（如 75g）|
| 12 | `unit` | ENUM | 否 | 袋 / 盒 / 瓶 / 罐 / 包 |

### D. 财务 + 价格带（4 字段 / 与 V4 ⑪ 对齐）

| # | 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| 13 | `cost_price` | DECIMAL(10,2) | ✅ | 进价（元）|

| 15 | `estimated_margin_rate` | DECIMAL(5,2) | ✅ | 预计毛利率 %（V4 ⑪ "预计毛利率%" / 应 ≥ 25% / 参 [[2026-05-06_新货整理框架_v0.1]] 评估标准 #3）|

| # | 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| 17 | `introduce_reason` | TEXT | ✅ | 1 句话引进理由（如 "T05 果干蜜饯凉果价格带高端缺档 + 配中老年常备客群任务"）|
| 18 | `composite_score` | DECIMAL(3,1) | ✅ | **综合评分 0-10**（4 维加权 / 见 §3 算法）|
| 19 | `seasonal_window` | STRING | 否 | 时令窗口（如 "全年" / "春节 W01-W05" / "中秋 W37-W40" / 与 [[52周MD]] 关联）|
| 20 | `priority` | ENUM | ✅ | P0（战略必引）/ P1（排面释放后即引）/ P2（候选池待机）|
| 21 | `competing_replacement_barcode` | STRING FK→A1 | 否 | 拟替换的在售 SKU 条码（V4 ⑪ "替换哪个商品"）|

### F. 试销（3 字段 / V4 ⑰ 4 节点对齐）

| # | 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| 22 | `trial_period_days` | INTEGER | ✅ | 30 / 60 / 90（默认 60）|

| 24 | `trial_target` | TEXT | ✅ | 试销目标（量 + 销售额 + 毛利合并 1 字段 / 如 "30 件/月 + 800 元 + 35% 毛利率"）|

### G. 供应 + 货位（3 字段 / V4 ⑫ 联动）

| # | 字段 | 类型 | 必填 | 说明 |

| 27 | `proposed_role` | ENUM | ✅ | 角色定位（vault 6 类）：目标性 / 常规性 / 季节性便利性 / 形象 / 流量 / 利润 |

| # | 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|---|

## §3. 综合评分算法（字段 18 / v1.0 起步 / 实战迭代）

```
composite_score = 紧迫度 × 0.35 + 时机 × 0.20 + 战略价值 × 0.25 + 毛利潜力 × 0.20

紧迫度（0-10）：
  - 来源 1 排面释放反推 + 同位空缺 = 9-10
  - 价格带缺口 = 7-8
  - 候选池待机 = 4-5

时机（0-10）：
  - 时令节庆窗口（如春节前 60 天）= 9-10
  - 自然销售旺季 = 7-8
  - 全年常态 = 5-6

战略价值（0-10）：
  - 老板战略必引（如儿童零食 / 中老年常备）= 9-10
  - 客群差异化（[[购物任务图谱_v1.1]] 高权重任务）= 7-8
  - 一般补强 = 4-6

毛利潜力（0-10）：
  - 预计毛利率 ≥ 35% = 9-10
  - 预计毛利率 25-35% = 6-8
  - 预计毛利率 < 25% = 1-5（接近反引进红线）
```

> **决策门槛**：
> - `composite_score ≥ 7.5` → priority = P0（战略必引）
> - `composite_score 6.0-7.4` → priority = P1
> - `composite_score < 6.0` → priority = P2 / 候选池待机

## §4. 引进规则（v0.1 G-1 至 G-5 继承 + V4 ⑪ 整合）

| 规则 | 来源 | 内容 |
|---|---|---|
| **G-1 战略必引** | 来源 2 | 老板战略品类 / 客群差异化 / 强需求未满 |
| **G-2 排面释放反推** | 来源 1 | 表 D 淘汰释放排面 → 同位 / 同品类 / 同价位补缺 |
| **G-3 价格带缺口补强** | 来源 1+2 | 表 C 字段 11-13 价格带分布 vs 字段 14 目标占比 / 漏档则补 |
| **G-4 候选商品池** | 来源 2 | [[单品管理]] §11 候选池 + 行业新品 + 供应商主推 |
| **G-5 竞品截流** | 来源 2 | 赵一鸣 / 百欢 / 山姆 同位 SKU 反向核（[[2026-05-06_读图分析待确认分桶派发表_v0.1]] C-05 桶）|
| **G-6 时令节庆** | 来源 2 ⭐ | [[52周MD]] / [[大湾区全年节庆节奏表_MC-GBA-05]] 春节/端午/中秋/清明拜山等节令商品 |
| **G-7 试销期纪律** | 通用 | 30-90 天 / 期间不淘汰 / 期满走完整 [[M-DEC-007]] 7 步评估 |

| # | 红线 | 出处 |
|---|---|---|
| 1 | ❌ 老板冲动决策 / 无客户需求支撑 | [[CLAUDE.md]] §13.7 |

| 7 ⭐ | ❌ `estimated_margin_rate < 25%` 强引 | [[2026-05-06_新货整理框架_v0.1]] 5 评估 #3 |

## §6. V4 ⑰ 任务生命周期跟进（试销期 4 节点 / 写入表 B 行动 sheet）

| 节点 | 日期 | 检查内容 | 触发动作 |
|---|---|---|---|
| 第 7 天 | listed_date + 7 | 陈列确认 + 价签 + 货位 sign off | 异常 → 立即整改 |
| 第 15 天 | listed_date + 15 | 动销初评（销量 / 客户反馈）| 0 销 → 调位 / 调价 |
| 第 30 天 | listed_date + 30 | ABCD 初分类（基于 30 天销售）| D 类 → 复核淘汰风险 |
| 第 60 天 | listed_date + 60 | 正式 ABCD 分类 + M-DEC-007 7 步评估 | 决定长存 / 淘汰 |

> **铁律**：试销期 60 天前不允许淘汰（[[单品管理]] §9 试销周期纪律）。
