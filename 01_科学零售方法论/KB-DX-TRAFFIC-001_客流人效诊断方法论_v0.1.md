## 核心论点

[[KB-DX-MATH-001_零售数学诊断工具体系_v0.1]] 的 RM-01 分解树解决了「问题在客数还是客单价」的**定位**问题，但当结论落到「客数↓」时，它只给方向（获客/留客），**不给归因**——为什么下滑、怎么办无系统工具。本页补这座桥：**客数↓ → 5 类根因归因树定位「为什么」→ 各路径领先指标给「怎么办」→ 人效三指标校验「人货场效率是否匹配客流」**。

> 调用顺序铁律：**先 RM-01 确认是客数问题（而非客单价），再进本页归因**。客单价问题误用客流归因树 = 南辕北辙。

## 一、客流归因树（5 类根因 · 客数↓ 的「为什么」）

> 归因原则：**先外后内、先快变量后慢变量**。外部竞争与季节时段是快变量（数周内可见），位置是慢变量（数月），商品与体验是可控变量（自身能改）。诊断时按「季节时段 → 外部竞争 → 商品 → 体验 → 位置」排查，**先排除不可控/伪信号，再深挖可控根因**。

| 根因 | 一句话定义 | 核心诊断指标 | 判断标准（触发该根因） |
|---|---|---|---|
| **R1 季节时段** | 客数波动是节令/天气/周期性正常起伏，非真实问题 | 同比客数（去年同期）、节令偏离度 | 同比基本持平（±8% 内[inferred，建议取花厅坊12个月同比数据标定实际波动带]）但环比↓ → **伪信号**，季节因素，不归因为经营问题 |
| **R2 外部竞争** | 新对手开业/老对手促销分流客流 | 商圈新增门店数、对手促销期重合度、客流流失时点 | 客数下滑**起点与某对手动作时点吻合**（同周内）+ 自身无变动 → 竞争分流 |
| **R3 商品问题** | 缺货/结构错配/鲜度差导致顾客「买不到/不想买」→ 不再来 | 生鲜日清达成率、A类断货率、动销率、顾客投诉品类分布 | A 类断货率 >5%[inferred] 或生鲜鲜度投诉↑ 或动销率<60%[inferred] → 商品驱动的流失 |
| **R4 体验问题** | 排队/脏乱/服务差/动线乱 → 来过不想再来 | 高峰收银排队时长、客诉率、复购率（会员）、停留时长 | 高峰排队 >5 分钟[inferred] 或 会员 30 天复购率↓ >10pct[inferred] → 体验驱动的流失 |
| **R5 位置/客源** | 商圈人口结构变化（拆迁/学校搬迁/人口外流） | 商圈常住人口、过店人流量（路过客流）、客源构成变化 | 过店人流量同步↓ 且非节令 → 位置/客源盘子缩小（最难逆转） |

### 归因决策树（操作流程）

```
客数↓（RM-01 已确认）
 │
 ├─ Q1 同比去年同期也↓吗？
 │     否 → R1 季节时段（伪信号），观察不整改
 │     是 ↓
 ├─ Q2 下滑起点撞上对手动作吗？
 │     是 → R2 外部竞争（防守反击：差异化锚点，见花厅坊调用）
 │     否 ↓
 ├─ Q3 过店人流量也同步↓吗？（门口数路过的人）
 │     是 → R5 位置/客源（结构问题，最后处理）
 │     否（路过的人没少，进店的少了）↓
 ├─ Q4 进店但商品力指标爆雷？（断货/鲜度/动销）
 │     是 → R3 商品问题 → 转 KB-DX-MATH RM-04/IM-04 + KB-FRESH 日清机制
 │     否 ↓
 └─ Q5 复购/停留/排队指标爆雷？
       是 → R4 体验问题 → 排队/动线/服务整改
```

**关键判据**：R5（过店人流量也↓）与 R3/R4（过店没少、进店少了）的分叉，是「**盘子缩小**」还是「**转化漏失**」的分水岭——**门口路过的人数是这棵树的第一道闸门**，没有它，R2/R3/R4/R5 无法区分。无设备时的路过客流估算见 §五。

## 二、人效诊断三指标（人货场效率是否配得上客流）

> 用途：客流稳定但**利润不行**时，问题常在「人效」而非客流本身——要么人多了（成本高），要么人少了（高峰漏单、体验崩→反噬客流 R4）。三指标互为交叉验证。

| 指标 | 公式 | 含义 | 大湾区社区生鲜参考基准（500-2000㎡）| 判断标准 |
|---|---|---|---|---|
| **人均产出（人效）** | 月销售额 ÷ 员工数 | 单兵产出，定编是否合理 | 月人效 **4–6 万/人**（健康）| <3.5 万 → 人浮于事（裁编/提销）；>7 万 → 可能高峰缺人(查 R4 排队) |
| **人时效率** | 销售额 ÷ 总工时 | 排班颗粒度效率，比人效更精细 | **50–80 元/工时** | <45 元 → 排班与客流波峰错配；优秀 >80 元 |
| **坪人比** | 营业面积(㎡) ÷ 员工数 | 单人看管面积，业态人力密度 | 生鲜重店 **40–60 ㎡/人**；工业品占比高 60–90 ㎡/人 | <35 → 人力过密；>90 且生鲜占比高 → 现场必失控(鲜度/补货跟不上) |

> ⚠️ `fact_layer: inferred`——以上基准为基于大湾区社区生鲜超市业态的**推断参考值**，非实测标定。花厅坊等门店实跑数据回流后须回验校准（开放问题①）。

### 三指标交叉判读（MECE）

| 人效 | 人时效率 | 坪人比 | 诊断 |
|---|---|---|---|
| 低 | 低 | 低(人密) | **冗员**：总人数超配，全面降编 |
| 正常 | 低 | 正常 | **排班错配**：总量够但波峰波谷没排对 → 高峰加人、平峰撤人 |
| 高 | 高 | 高(人稀) | **过载预警**：人效虚高来自人手不足，查高峰排队(R4) → 反而要加人保客流 |
| 高 | 高 | 正常 | **健康**：维持 |

**核心认知**：人效不是越高越好。**人效虚高 → 高峰体验崩 → 客流流失（R4）→ 下月人效跟着塌**，是社区生鲜最常见的隐性失血。坪人比是这条暗线的早期哨兵。

## 三、时段客流诊断（四时段 × 特征 × 诊断指标）

> 数据接口：直接复用 [[KB-GBA-TPOS-001_大湾区TPOS本地化_v0.1]] 的「T—时机」时段表与 [[KB-SHOPPER-TASK-001_购物任务图谱_v0.1]] 的购物任务时段分布。本节把「时段特征」翻译成「时段诊断指标」。

| 时段 | 主力客群/行为（TPOS）| 健康特征 | 时段诊断指标 | 判断标准（该时段失血） |
|---|---|---|---|---|
| **早市 06:00–08:30** | 老人散装买菜（高频刚需）| 生鲜唱主角，客数全天峰之一 | 早市客数占比、生鲜陈列完成时点 | 06:00 前未完成生鲜陈列上架 → 错过早市峰（早市不等人，开门即要货在位）；早市客数占比<20%[inferred] → 早市经营缺失 |
| **午间 11–13** | 午休便利/即食驱动 | 客单低、连带弱，即食/半成品为主 | 午间笔数、即食品类动销 | 笔数<早市 30% 属正常；若即食缺货 → 漏掉便利客 |
| **晚高峰 17:00–19:30** | 下班顺路补货（冲动比例高，全天最高峰）| 当日晚餐食材 + 偏好性休食冲动 | 晚高峰客数占比、收银排队时长、连带率 | 排队>5 分钟[inferred] = 漏单+体验崩(R4)；连带率<早市 → 关联陈列未到位 |
| **周末全天** | 家庭计划性大采购 | 客单显著高于工作日、家庭装/聚会任务激活 | 周末客单/工作日客单比、家庭装动销 | 周末客单未高于工作日 1.2×[inferred] → 大采购承接失败(家庭装/聚会SKU缺位) |

### 时段诊断铁律

1. **波峰错配是头号杀手**：晚高峰 17:00–19:30 是全天峰，**收银/补货/促销资源必须压到这两个半小时**——平峰加人、波峰缺人是排班最常见错误（与 §二人时效率联诊）；19:00–19:30 仍在高峰窗口，提前撤人将直接触发排队>5 分钟漏单(R4)。
2. **早市生鲜上架时点 = 早市客数的天花板**：06:00 前未完成生鲜陈列上架，早市 06:00–07:00 整一小时客流在货不在位状态下白白流失，当天不可追回。
3. **时段占比异常 = 定向归因入口**：哪个时段塌了，就查那个时段的对应任务（TPOS）有没有被商品/陈列承接。

## 四、客流增长三路径（新客 / 复购提频 / 客单提升）

> 对应 RM-01 客数 = (既存客 × 来频) + 新客。三路径正交，**领先指标（leading）先于结果指标（客数/销售）变动**，是「还没涨销售前就能看出对不对」的早期信号。

| 路径 | 杠杆 | 结果指标（滞后）| 领先指标（提前预警）| 判断标准 |
|---|---|---|---|---|
| **P1 拉新客** | 触达新客源、首单转化 | 新客数、新会员注册数 | 进店转化率(过店→进店)、首单优惠核销数、地推/社群曝光量 | 转化率↑但新客数未动 → 留客漏(转P2)；曝光↑转化平 → 钩子无吸引力 |
| **P2 复购提频** | 提高既存客来店频率 | 会员月均到店次数、复购率 | 社群活跃度、会员券打开率、30天回头率、生鲜日清稳定度 | 30天回头率<40%[inferred，待会员数据回验] → 复购盘漏；日清不稳→鲜度信任流失(最伤复购) |
| **P3 客单提升** | 提高单次购买额（连带/升级）| 客单价、1客点数 | 连带率、关联陈列触达率、M域(高价偏好品)动销 | 连带率↓ → 关联陈列失效；M域动销↓ → 高价品没卖动(接RM-01 1点单价↓) |

### 路径选择判断标准（资源有限，先打哪条？）

```
诊断输入：归因树结论 + 商圈盘子
 │
 ├─ R5 盘子缩小（过店客流↓）→ P1 拉新优先级低（盘子就这么大），主攻 P2 复购榨存量
 ├─ R2 竞争分流 → P2 复购（锁客）+ P3 客单（差异化锚点），P1 暂缓（对手在抢，拉新成本高）
 ├─ R3/R4 自身问题 → 先修商品/体验（堵漏），漏没堵住做增长 = 漏水的桶里加水
 └─ 盘子健康、自身无硬伤 → P1 拉新 + P3 客单 双轮（增量市场）
```

**铁律**：**有 R3/R4 硬伤时不做增长投放**——体验/商品没修好就拉新，等于花钱让更多人来体验差评，加速口碑崩塌。**先堵漏、后增长**是不可逆顺序。

## 五、客流数据获取方法（无客流计数设备时的替代估算法）

> 痛点：社区生鲜超市绝大多数**没有客流计数器**，归因树第一道闸门「过店客流 / 进店客流」缺数据就断链。本节给无设备时的替代估算法，按精度/成本排序，**关键是『进店客流』与『过店客流』要分别有近似量**。

| 方法 | 估算对象 | 操作 | 精度 | 适用条件 / 误差来源 |
|---|---|---|---|---|
| **A POS 笔数法** | 进店成交客数 | 交易笔数 ≈ 成交客数（1 单≈1 客）| 中高 | **最易得**。误差：拼单/分单、只逛不买者不计 → 系统性低估进店客数 |
| **B 提袋率反推** | 进店总客数 | 成交客数 ÷ 提袋率 = 进店客数；提袋率按业态经验取 60–80% | 中 | 提袋率需一次性人工标定（数100个进店者有几个结账）|
| **C 人工抽样外推** | 进店/过店客流 | 选 2–3 个代表时段(早峰/晚峰/平峰)各人工点数 15 分钟 × 外推全天 | 中 | 低成本可同时测过店与进店；误差来自时段代表性 |
| **D 监控回放点数** | 过店/进店客流 | 调门口监控，定时段人工/AI 回放点人头 | 高 | 几乎零增量成本（已有监控）；耗人工，建议抽样不全量 |
| **E 支付商户后台** | 成交客数 | 微信/支付宝/收钱码后台笔数 → 同 POS 笔数 | 中高 | 现金客漏计 → 老人多的社区店现金占比高，须叠加现金系数 |
| **F WiFi/门磁/电子哨兵** | 进店客流 | 免费 WiFi 连接数 / 门磁触发数 / 红外计数器(百元级) | 中 | 低成本设备；WiFi 漏未连网者，门磁双向需 ÷2 |

### 估算法判断标准（数据够不够用）

1. **最低可用配置**：A（POS 笔数）+ C（人工抽样过店客流）= 进店与过店都有近似量 → 归因树第一道闸门可运行。**只有 A 没有过店数据 → R5 与 R3/R4 无法区分，归因树断链**。
2. **趋势优先于绝对值**：替代估算法**绝对值不准但趋势可信**——诊断客数↓只需「环比/同比变化方向 + 量级」，不需精确人头。**同口径连续测**比单次精测更有价值。
3. **标定一次、复用长期**：提袋率(B)、现金系数(E)、抽样外推系数(C)各做一次人工标定即可长期套用，成本一次性。
4. **失效提示**：促销日/节令日客流结构异常，估算系数（提袋率/抽样外推）会失真，须单独标定或剔除，不可套用平日系数。

## 花厅坊直接调用

> **当前（6–7 月）客流诊断动作清单**：
> 1. **先取过店客流近似量**：花厅坊有监控 → 用方法 D，抽早峰(06:00–08:30)/晚峰(17:00–19:30)/平峰各 15 分钟回放点头，建立过店基线——这是归因树能跑的前提，**当前最缺的一块数据**。时段边界须与 [[KB-GBA-TPOS-001_大湾区TPOS本地化_v0.1]] 一致（以实测结果标定后回填本页）。

> 3. **若客数↓**：走归因决策树——花厅坊紧邻小学+老社区，R5 客源盘子是否变化（学校/拆迁动向）要先排除；若盘子没变，重点查 R3（生鲜日清达成率，接 [[KB-FRESH-001_生鲜护城河经营方法论_v0.1]]）与 R4（晚高峰收银排队）。
> 4. **增长路径**：花厅坊属「综合体+小学+老社区」综合盘，盘子健康 → P2 复购（儿童零食/中老年常备的高频任务锁客）+ P3 客单（连带）优先；R2 若出现对手分流，以差异化锚点（凉果陈皮、儿童专项）防守。
> 5. **人效校验**：用方法 §二三指标核花厅坊定编是否压在晚高峰——若晚高峰排队>5 分钟同时白天人效虚高，是「排班错配」非「缺人」，先调班再谈增编。

## 失效条件（张力）

1. **归因树的「先外后内」顺序在自身刚大改后失效**：门店刚完成 SKU 调改/陈列重构 30 天内，客数波动可能是新布局适应期摩擦（顾客找不到货），此时套「先排除季节、再查竞争」会误导——**调改后 30 天内，优先归因 R3/R4（自身变动），而非外部**。与 [[KB-DX-MATH-001_零售数学诊断工具体系_v0.1]] 的调改保护期规则一致。
2. **人效基准对「夫妻店/极小店」失效**：本页人效基准基于有雇员的 500–2000㎡ 标准社区超市；家庭自营、雇员<5 人的店，人效/坪人比指标失去管理意义（人力不是可调变量）。
3. **客流增长三路径在「位置硬伤（R5）」前全部失效**：商圈人口结构性外流（拆迁/产业搬迁）时，P1/P2/P3 都是在缩小的盘子里内卷，正确动作是**评估迁址或业态收缩**，而非增长投放——这是本方法论无法解决、必须上交六哥判断（驭马人层）的边界。
