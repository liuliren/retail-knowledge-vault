# 门店全景诊断框架 · v0.1（draft·待签字）

## 核心论点（结论先行）

> **一家店的经营天花板，在开业前的硬件决策里就被锁死了一半。** 市调（L0）回答「这个位置周边有没有生意」——商圈客群是外部环境，选址落定后**基本不可改**；7D 经营诊断（L2）回答「这门生意跑得好不好」——经营效率靠运营，**低成本可改**。中间缺一层：**这家店本身的「硬件底子」怎么样**——落位、动线、冷链、门头。这就是本框架的 L1 定位：**门店全景 = 硬件条件层**，它的处置方式介于 L0（接受）和 L2（运营优化）之间——**需要资本改造才能改，有明确成本门槛**。
>
> L1 的诊断价值不在自身打分，而在**它为 L2 设定天花板**：冷链温区不够，D2 生鲜运营再怎么日清也顶不到满分；动线有死角，D3 陈列技巧救不回被浪费的曝光；停车不可达，D4 客数上限被物理锁死。**先诊断硬件天花板，再诊断经营效率**——否则会把「硬件短板导致的低指标」误判为「运营不努力」，开错药方。本框架四维（落位 A / 动线 B / 硬件 C / 形象 D-H）各 25 分满分 100，D 维中的**运营形象项（整洁度/异味等 D-O）单独记录但不计入 L1 总分**（归 L2 D7 运营诊断处理）。

## 0. 定位与边界（必读）

### 0.1 科学零售诊断三层结构（本框架的坐标）

| 层 | 名称 | 回答什么 | 可改性 | 处置方式 | 权威 KB |
|---|---|---|---|---|---|
| **L0** | 外部环境（市调/商圈/客群） | 这个位置周边有没有生意？客是谁、要什么？ | **基本不可改**（选址已定） | 接受约束，据此定位 | [[商圈分析标准框架]]（**需新建**）+ [[KB-GBA-TPOS-001_大湾区TPOS本地化_v0.1]] |
| **L1** | **门店全景（硬件条件）← 本框架** | 这家店的**硬件底子**怎么样？ | **资本可改**（有成本门槛） | 分「资金改造 vs 运营弥补」处置（§5） | **KB-STORE-OVERVIEW-001（本文件）** |
| **L2** | 经营诊断（7D 七维度） | 这门生意**跑得好不好**？ | **运营可改**（低成本） | 运营调改，进 TOP3 行动 | [[KB-DX-FULLSTORE-001_全店7D诊断框架2.0_v0.1]] |

> *【三层为何要分开·必读】把三层揉成一张表会犯两类错：① **把不可改的当可改**——花大量诊断精力去「优化」一个选址已经错了的商圈，L0 的约束只能接受不能诊断掉；② **把硬件短板当运营懒惰**——L2 看到「生鲜损耗高」就派日清 SOP，但真因是 C 维冷链温区覆盖不足（L1 硬件），SOP 救不了。三层分开，才能让每个问题落到**正确的处置通道**：L0 接受、L1 算改造账、L2 派运营动作。*

## 1. 四维总览与权重（满分 100）

> *【权重哲学·必填】四维等权（各 25）不是「都一样重要」，而是「四维各锁死 7D 的不同天花板，缺一维就漏掉一类不可逆约束」。等权是 v0.1 的**保守起点**——真实撬动性排序（落位 > 硬件 > 动线 > 形象）尚未经多店验证，故先不做差异化加权，避免用单例经验压出假精确。差异化加权列 §9 待裁决项①。*

> **【D 维计分边界·v0.1 铁律】** D 维满分 25 分**仅计 D-H 硬件形象子项**（门头结构/固定导视/外立面固定件）。D-O 运营形象子项（整洁度/异味/陈列秩序）在首访走店清单中**单独记录、单独出结论**，但**不计入 L1 总分**，归 L2 D7 顾客体验运营项处理。这样 L1 总分 100 分真正反映「需要资本才能改的硬件」，老板看「硬件 65 分低」= 硬件需要花钱改，逻辑自洽。

| 维度 | 名称 | 满分 | 计分范围 | 一句话定位 | 主要锁死的 7D 天花板 |
|---|---|---|---|---|---|
| **A** | 落位与位置 | **25** | 全计入L1 | 门店在商圈中的物理位置、可见性、可达性 | **D4 客流人效**（客数上限）+ D5 坪效 |
| **B** | 动线与空间 | **25** | 全计入L1 | 顾客动线、面积配比、空间利用率 | **D3 陈列空间**（曝光上限）+ D7 连带 |
| **C** | 硬件条件 | **25** | 全计入L1 | 冷链、照明、收银、仓储 | **D2 生鲜运营**（品质上限）+ D5 成本 |
| **D-H** | 品牌与形象（硬件件） | **25** | 仅D-H计入L1 | 门头固定结构、分区固定导视、外立面固定件 | **D7 顾客体验**（第一印象上限）+ D4 新客转化 |
| **D-O** | 品牌与形象（运营件） | —— | **不计L1** | 整洁度/异味/陈列秩序（纯运营，SOP可改） | 归 L2 D7 运营项，首访单独记录 |
| **L1 合计** | | **100** | A+B+C+D-H | | |

**总分计算**：`门店全景总分 = A + B + C + D-H`（D-O 不入总分）。

**分档标准（每维 0–25，全局宪法《标准审议铁律》draft 态）**：

| 得分区间 | 定级 | 含义 | 处置信号 |
|---|---|---|---|
| **25–20** | 行业标杆级 | 硬件不构成约束，7D 该维天花板 ≈100% | 保持，不投改造资金 |
| **19–15** | 合格水平 | 硬件基本够用，天花板 ≈85–90% | 运营弥补为主，小额优化 |
| **14–10** | 有明显改善空间 | 硬件拖累经营，天花板 ≈70–85% | 算改造 ROI，分「资金 vs 运营」 |
| **9 及以下** | 影响经营的硬件短板 | 硬件是天花板本身，天花板 <70% | **优先立项**，运营救不动 |

> *【张力·必填】此分档在什么条件下失真？① **社区生鲜店 vs 便利店**：便利店对 A 落位（即时可达）的容忍度远低于社区店（顾客愿为生鲜多走 300m），同一落位分对两种业态含义不同——分档阈值应随业态浮动，v0.1 未做，列 §9①。② **加盟 vs 直营**：加盟店硬件已由品牌方锁定，C 维低分未必可改（改造权不在店主手上），此时低分应转为「选址复盘」而非「改造立项」。③ 分档的「天花板百分比」映射（如 14–10→70–85%）来自花厅坊单例反推，**非行业规律**，跨店前须 ≥2 店验证（§9①）。*

## 2. 四维详细定义

> 每维按六要素展开：①定义 ②评估子项（含分值分配）③关键判读指标 ④工具/采集方式 ⑤评分锚点 ⑥硬件短板类型（接口 → §4 天花板 + §5 改造优先级）。
> **可得性标注**：硬件维度多为「现场可核（B 级）」——这是本框架相对 7D（多 C 级）的证据优势。

---

### A 落位与位置｜锁死 D4 客数天花板

**① 定义**：门店在商圈中的物理落位质量——它站在人流的哪个位置、路过的人看不看得见、想来的人到不到得了、周边是什么人群组合。这是**选址决策的物化结果**，接店后基本不可改（除非搬迁），是四维中「最接近 L0、最不可逆」的一维。适用条件：500–2000㎡ 社区生鲜超市的实体落位。

**② 评估子项（合计 25）**：

| 子项 | 分值 | 判读 |
|---|---|---|
| **A1 商圈位置** | 8 | 主干道临街（8）/ 支街/次干道（5）/ 商场内/社区内街（3）/ 隐蔽位（1）。社区生鲜的最优位≠最贵位：临**社区主入口/回家动线**上 > 纯商业主干道 |
| **A2 可见性** | 6 | 路过 3 秒能否识别为「生鲜超市」：门头无遮挡+正对人流（6）/ 侧对或部分遮挡（4）/ 需绕行才见（2） |
| **A3 可达性** | 7 | 入口数量+停车（电动车/自行车停放为社区店关键，非汽车位）+ 无台阶/坡道障碍。骑车停放便利在社区店权重高于汽车位 |
| **A4 周边人群组合** | 4 | 居民楼（复购基本盘）/ 学校（时段客流）/ 写字楼（工作日午晚）/ 医院/菜场（截流风险）的组合质量与匹配度 |

**A3 三档锚点（7/4/1 场景）**：
- **7分**：≥2 个入口（主次分设）+ 骑车位 ≥10 个且有规划（划线/设架）+ 零台阶/坡道无障碍
- **4分**：1 个主入口 + 骑车可停放但无规划（随意停放无秩序）+ 无台阶
- **1分**：单入口且偏侧 + 骑车停放侵占出入通道（造成拥堵）+ 有台阶/坡道障碍

**③ 关键判读指标**：
- 门店 500m 半径内居民户数（复购基本盘规模）
- 到店主动线是否在居民「回家路径」上（社区生鲜的黄金落位判据）
- 电动车/自行车停放位数量与秩序（社区店真实交通方式）
- 周边同业截流点数量与距离（菜场/竞品的分流强度，与 L0 竞调交叉）

**④ 工具/采集方式**：高德/百度地图热力 + 现场步测（`/read-floorplan` 建店周环境图）；周边人群组合对接 [[KB-GBA-TPOS-001_大湾区TPOS本地化_v0.1]] 客群画像。**现场必采**。

**⑤ 评分锚点（0–25）**：满分=主入口回家动线上 + 门头正对人流无遮挡 + 骑车停放便利 + 周边高密度居民楼。证据：A1/A2/A3 现场步测 **B 级**；A4 需交叉 L0 人口数据。

**⑥ 硬件短板类型（→ §4 / §5）**：
- `SO-A-P1` 落位偏离回家动线（客数基本盘物理受限 → 锁死 **D4 客数天花板**，运营救不动，改造=搬迁不现实 → 转「选址复盘/接店评估」）
- `SO-A-P2` 可见性差/门头被遮（到店转化前置漏斗窄 → D4 新客 + 可低成本改门头，属运营弥补区）
- `SO-A-P3` 骑车停放缺失/混乱（社区店可达性硬伤 → D4 来频 + 可低成本划线/加架，运营弥补）
- `SO-A-P4` 截流点密集（菜场/竞品分流 → D4 客数 + 属 L0 竞争约束，靠差异化经营而非硬件）

### B 动线与空间｜锁死 D3 曝光天花板

**① 定义**：卖场内部的空间组织质量——顾客怎么走（主/次动线、死角）、各区面积怎么分（生鲜/食品/百货配比）、关键功能区（入口/收银/高频区）位置合不合理、空间用得满不满（有效陈列面积占比）。这是**建筑格局 + 布局决策的结果**，承重结构不可改、隔断/货架布局可中成本改。适用条件：全店卖场空间。

**② 评估子项（合计 25）**：

| 子项 | 分值 | 判读 |
|---|---|---|
| **B1 顾客动线规划** | 8 | 主动线是否引导全场回游 + 次动线是否覆盖高毛利区 + **死角面积占比**（死角>15%=严重）。社区店主动线宜「生鲜先导」（进门先见生鲜护城河） |
| **B2 各区面积配比** | 7 | 生鲜区/食品区/百货区实占比 vs 最优配比。社区生鲜基准：生鲜 35–45% / 食品 30–40% / 百货 15–25%（**⚠️ 若 f>65% 高生鲜密度门店，生鲜区基准上调至 40–50%，见张力**） |
| **B3 功能区位置合理性** | 6 | 入口缓冲区（不堵）+ 收银动线（出口顺、不逆流）+ **高频区（生鲜）落位**（宜在动线深处拉回游，非门口即取即走） |
| **B4 空间利用率** | 4 | 有效陈列面积 ÷ 总营业面积。过低=空间浪费（通道过宽/闲置区）；过高=拥挤压迫（<1.2m 通道伤体验） |

> *【B2 f>65% 特化注脚】花厅坊 f≈70.74% 属高生鲜密度门店。在这类门店中，通用基准（生鲜区 35%）会错误判断「生鲜区 32% 属合格」。评分锚点建议：f>65% 时 ≥40%=7分 / 35–39%=5分 / 30–34%=3分 / <30%=1分。首访须先确认 f 值，再选对应锚点。（来源：花厅坊单例 inferred，§9① 须多店验证）*

**③ 关键判读指标**：
- 主动线总长 vs 磁石点间距（每 15–20m 一驻留点，接 [[KB-RETAIL-DISPLAY-001_商品陈列体系_v0.1]] 动线效率公式）
- 死角区面积占比及其品类价值
- 生鲜区是否在「先导位」（进门 5 步内可见/可闻烟火气，接 [[KB-FRESH-001_生鲜护城河经营方法论_v0.1]]）
- 有效陈列面积占比（行业健康区间待验证，v0.1 参考 55–70%）

**④ 工具/采集方式**：`/read-floorplan`（门店平面图读取与空间建图·**本维度核心工具**）+ 现场走店动线记录；面积配比对接 L2 的 D2/D1 品类结构。**现场必采 + 平面图**。

**⑤ 评分锚点（0–25）**：满分=主动线全场回游无大死角 + 面积配比匹配 f（见 B2 注脚）+ 生鲜先导 + 收银顺流 + 利用率 55–70%。证据：现场 + 平面图 **B 级**。

**⑥ 硬件短板类型（→ §4 / §5）**：
- `SO-B-P1` 动线死角>15%（曝光被物理浪费 → 锁死 **D3 陈列曝光天花板**；承重墙造成=高成本改，货架布局造成=中成本改）
- `SO-B-P2` 面积配比失衡（生鲜区过小 → 压制 D2 生鲜 + D1 商品宽度；靠布局调改，中成本）
- `SO-B-P3` 生鲜非先导位（护城河没在动线上被先看见 → D2 来频 + D7 连带；布局可改）
- `SO-B-P4` 通道过窄/利用率异常（体验压迫或空间浪费 → D7 + 坪效；通道靠布局，中成本）

### C 硬件条件｜锁死 D2 生鲜品质天花板

**① 定义**：支撑经营的物理设备条件——冷链（冷柜数量/温区覆盖/设备状态）、照明（整体/生鲜专区/促销位）、收银（POS 台数/排队/自助）、仓储（后场面积/冷库容量/入库效率）。这是**四维中最「花钱能解决、但门槛最高」的一维**——冷链/冷库是重资产，改造 ROI 必须算清。适用条件：全部社区生鲜店。

**② 评估子项（合计 25）**：

| 子项 | 分值 | 判读 |
|---|---|---|
| **C1 冷链设备** | 9 | 冷柜延米匹配生鲜 SKU（4分）+ **温区覆盖完整**（3分）+ 设备状态（2分）。**冷链是生鲜护城河的物理底座** |
| **C2 照明** | 5 | 整体照度充足 + **生鲜专区专用光**（暖光衬肉色/生鲜冷柜内补光）+ 促销位重点照明。照明直接影响生鲜「卖相→品质感知」 |
| **C3 收银系统** | 6 | POS 台数 vs 高峰客流（下午4–6点是否排长队）+ 是否有自助/扫码补位 + 系统能否导出诊断所需数据（关系 7D 证据等级） |
| **C4 仓储** | 5 | 后场面积占比（收货/暂存/加工空间）+ 冷库容量（生鲜周转缓冲）+ 入库效率（收货动线是否顺畅） |

**C1 子项细化分值分配**：
- **冷柜延米匹配（4分）**：4=生鲜全品类 SKU 有柜可放、无须裸放常温；2=部分 SKU 需裸放（10–30%）；0=大量 SKU 无法冷藏（>30% 裸放）
- **温区覆盖（3分）**：3=四温区齐全（冷藏 0–4℃ / 冷冻 -18℃ / 鲜货保鲜区 / 常温生鲜区）；1.5=缺 1 个温区；0=缺 2 个及以上温区
- **设备状态（2分）**：2=制冷达标、无故障记录、维保正常；1=偶有故障但仍运行中；0=带病运行/制冷持续不达标（见损耗记录）

**③ 关键判读指标**：
- 冷柜延米数 vs 生鲜 SKU 数（是否够放，缺口=温区外裸放=损耗源）
- 温区覆盖清单（缺哪个温区→哪类生鲜无法经营/损耗高）
- 生鲜区照度与色温（暖光/冷白光对肉菜卖相的影响）
- 高峰时段收银排队长度（C3 收银瓶颈=丢单，与 D4-P2 交叉）
- 后场/前场面积比（社区店参考 15–25% 后场，过小=补货频繁扰动前场）

**④ 工具/采集方式**：现场设备清点（冷柜/温区/POS 数量·**现场必采**）+ 照度目测/测光 + 高峰收银目测 + 后场走查。设备状态可请店主提供采购/维保记录。

**⑤ 评分锚点（0–25）**：满分=冷链温区齐全+设备健康 + 生鲜专区照明到位 + 收银高峰不排长队 + 后场仓储充足顺畅。证据：设备清点 **A/B 级**（客观可数，本维度证据最硬）。

**⑥ 硬件短板类型（→ §4 / §5）**：
- `SO-C-P1` 冷链温区缺失/冷柜不足（生鲜品质无物理保证 → 锁死 **D2 生鲜品质天花板**，日清 SOP 也救不了；**高成本改造，须算 ROI**）
- `SO-C-P2` 生鲜照明不足/色温错（卖相压制品质感知 → D2 + D7；**低成本改造，优先做**——补光灯 ROI 极高）
- `SO-C-P3` 收银瓶颈（高峰排队丢单 → D4 客数 + D7 体验；加 POS 中成本 / 自助补位）
- `SO-C-P4` 仓储不足（周转缓冲小、补货扰动 → D2 鲜度 + D5 周转；后场扩容高成本，流程优化低成本）

### D 品牌与形象｜D-H（硬件件·计入L1）+ D-O（运营件·不计L1）

**① 定义**：门店品牌信号与形象质量，**强制分为两层**：
- **D-H 硬件形象**（门头固定结构/分区固定导视/外立面固定件）：这些是资本或中等成本才能改的物理固定件，**计入 L1 总分（满分 25）**，是本框架对 D7/D4 天花板的判据。
- **D-O 运营形象**（整洁度/异味/陈列秩序）：这些是 SOP 问题，**不计入 L1 总分**。首访走店时单独记录，标注「L2 D7 运营项」，归入 L2 经营诊断处理——不改变 L1 硬件总分，但仍出结论、仍驱动 SOP 行动。

> *【为何分拆·必填】如果整洁度计入硬件分，老板看到「L1 硬件底子 60 分」会理解为「需要花钱改硬件」；但若 60 分的主要原因是整洁度 SOP 问题，改造信号就是误导。D-H/D-O 分拆后，L1 低分 = 纯硬件需投资，整洁度低 = SOP 立即改，两个信号分开传递，不再混淆。*

**② D-H 评估子项（计入L1，合计 25）**：

| 子项 | 分值 | 判读 |
|---|---|---|
| **D-H1 门头硬件** | 12 | 招牌固定结构清晰可识（远处认得出是生鲜超市）+ 店名/LOGO 醒目 + **主推定位信息牌**（生鲜/社区/性价比，一眼可读，而非信息堆砌）。这些是招牌框架/灯箱/固定门牌等物理件 |
| **D-H2 固定分区导视** | 8 | 品类指示吊牌/分区墙贴/固定导视系统是否安装齐全，顾客能否 30 秒找到目标品类。接 [[KB-CAT-TIMEEFF-001_时间效率主轴评价指标体系_v0.1]] 找到率 |
| **D-H3 外立面与入口固定件** | 5 | 橱窗/外立面墙体状态（无严重破损/脱漆）+ 自动门/推拉门功能正常 + 入口固定装饰件整体 |

**D-H1 三档锚点（12/7/2 场景）**：
- **12分**：远处 50m 可识别为生鲜超市 + 招牌醒目无遮挡 + 定位信息清晰（≤1 个核心定位信号）
- **7分**：招牌可见但有遮挡 / 信息堆砌（>3 个并列信号）/ 仅近距可识别
- **2分**：门头老旧/脱色/模糊 / 无明显生鲜超市品类识别标志

**② D-O 运营形象项（不计L1，首访单独记录）**：

| 子项 | 记录要点 | 归属 | 行动 |
|---|---|---|---|
| **D-O1 门店整洁度** | 进门 3 秒第一印象：地面/货架/生鲜区卫生 + 无异味 + 陈列不凌乱 | **L2 D7 运营项** | 出结论：SO-D-O-P1 整洁度差 → 进 §5 象限二「立即做」；纯 SOP，零改造成本 |
| **D-O2 异味** | 是否有腐烂/污水味（社区生鲜信任杀手） | **L2 D7 运营项** | 同上，异味=生鲜区清洁 SOP 直接驱动 |

> *【D-O 与 §5 的接口】D-O 项（整洁度/异味）虽不计 L1 分，但仍进改造优先级矩阵 §5 象限二（经营影响高 + 改造成本低/零 → 立即做）。这与 L1 总分分开传递，对老板的沟通结构是：「L1 硬件 65 分，要花钱改的方面有 XXX；另外整洁度/异味是运营层问题，和硬件无关，立即用 SOP 处理。」*

**③ 关键判读指标（D-H）**：
- 门头信息负载（是否 ≤ 一个核心定位信号，避免堆砌）
- 固定导视完整性（各主要品类区是否有吊牌/墙贴）
- 外立面物理状态（招牌灯箱是否正常工作/门体是否完好）

**④ 工具/采集方式**：现场走店观察（门头/导视/外立面·**现场必采**）+ 30 秒找到率快测。D-H 全部现场可核，**B 级**；D-O 同步记录但不入 L1 计分。

**⑤ 评分锚点（D-H 0–25）**：满分=门头定位清晰醒目 + 固定分区导视齐全 + 外立面状态良好。证据：现场观察 **B 级**。

**⑥ 硬件短板类型（→ §4 / §5）**：
- `SO-D-P1` 门头硬件弱/定位模糊（新客识别前置漏斗窄 → D4 新客 + D7；**中低成本改门头固定件**，重新设计灯箱/招牌）
- `SO-D-P2` 固定导视缺失（找不到品类 → D7 时间效率 + 连带；**低成本加固定吊牌/墙贴**，运营弥补）
- `SO-D-P3` 外立面破损（信任感/第一印象负分 → D7；视程度，轻微=运营弥补，严重=中成本修缮）
- `SO-D-O-P1` 整洁度差/有异味（**D-O 项·不计L1·归L2 D7**；纯 SOP 弥补，零改造成本，象限二立即做）

## 4. 四维 → 7D 天花板映射（本框架的核心接口）

> **【v0.1 天花板边界声明·必读】本框架 v0.1 不提供天花板量化百分比。天花板量化系数（校准系数）尚未经多店验证（§9①），无法给出精确数值。当前 L1 诊断产出为「定性硬件状态红/黄/绿」，供 7D 判读时定性调整判断尺度。精确量化接口（如「C 维 14 分 → D2 满分按 82% 折算」）须待 §9① 裁决后激活。在此之前，7D 诊断引用 L1 结论时，须在报告中声明「L1 为定性红黄绿判断，天花板数值待量化」，防止误读为精确约束。**

> *【接口铁律·必读】L1 门店全景不替代 L2 的 7D，而是**为 7D 每个维度设定可达上限（天花板）**。7D 诊断时，若某维经营指标低，须先查本框架对应天花板：**天花板低→是硬件锁死（走 §5 改造判断），不是运营懒惰（不派运营 SOP 白费力）**。*

| L1 维度 | 主锁 7D 维度 | 天花板机制 | 若硬件短板，7D 该维应如何判读 |
|---|---|---|---|
| **A 落位** | **D4 客流人效** | 落位决定客数基本盘上限 | D4 客数低 + A 维低 → 判「客数受落位物理锁死」，非「引流不努力」；不派大额引流预算 |
| **A 落位** | D5 财务（坪效） | 商圈级别决定坪效基准 | D5 坪效对标须按 A 维落位级别调整基准线，非一刀切行业均值 |
| **B 动线** | **D3 陈列空间** | 死角/布局决定曝光上限 | D3 陈列转化低 + B 维死角多 → 判「曝光被动线浪费」，陈列技巧救不回，先改布局 |
| **B 动线** | D7 顾客体验（连带） | 动线决定连带机会 | 连带率低 + 动线不回游 → 布局问题，非促销组合问题 |
| **C 硬件** | **D2 生鲜运营** | 冷链温区决定生鲜品质上限 | D2 损耗高 + C 维冷链缺 → 判「品质被冷链锁死」，日清 SOP 救不了，先补冷链 |
| **C 硬件** | D5 财务（成本） | 设备能耗/收银效率影响成本 | 老旧设备高能耗、收银瓶颈丢单，计入 D5 成本与 D4 客数 |
| **D-H 形象** | **D7 顾客体验** | 门头/固定导视决定第一印象硬件上限 | D7 体验低 + D-H 维形象件缺 → 门头/导视是硬件问题，须花钱改。若 D-H 满分但 D7 体验仍低 → 查 D-O 运营形象项（整洁度/异味） |
| **D-H 形象** | D4 新客转化 | 门头/定位决定新客识别 | 新客弱 + 门头硬件模糊 → 改门头灯箱/招牌（中低成本），非狂投广告 |

> **【多维叠加规则·必填】若同一 7D 维度被多个 L1 维度同时设限（如 D4 被 A 和 D-H 同时约束；D7 被 B 和 D-H 同时约束），取最严（最低）天花板——最严约束胜出**。例：A 维 22 分（绿·不设限）+ D-H 维 8 分（红·<70% 天花板）→ D4 天花板取最严 = 红（<70%），不因 A 维绿而放宽。顾问在报告中须明确标注哪个 L1 维度是 7D 该维的「主约束」。

> *【天花板量化·v0.1 版裁决声明】v0.1 天花板映射为定性（20+ 绿/不设限、10–19 黄/部分设限、<10 红/硬顶）；精确量化系数（校准系数）须 §9① 多店验证后方可给出。在量化系数到位之前，L2 诊断**不得**以「L1 XX 分 = D{n} 打折 XX%」的方式做精确计算，只用定性约束。（这是有意降级，防止假精确）*

## 5. 改造优先级判断：资金改造 vs 运营弥补（核心决策工具）

> *【处置铁律·必读】L1 硬件短板的处置分两条通道：**资金改造**（需资本投入，算 ROI）vs **运营弥补**（低/零成本，靠 SOP 与布局）。判断门槛 = **「这个短板不花钱能不能补上 80%」**——能→运营弥补先行；不能→算改造 ROI。误判方向 = 要么把该改造的硬伤用 SOP 硬扛（扛不住），要么把能运营弥补的小事上马大改造（浪费资本）。*

### 5.1 改造优先级矩阵（改造成本 × 经营影响）

| | **经营影响高**（锁死高权重 7D 天花板） | **经营影响低** |
|---|---|---|
| **改造成本高**（重资产/承重结构） | **象限一·立项算 ROI**：冷链温区补齐（SO-C-P1）、承重墙死角（SO-B-P1）、后场扩容（SO-C-P4）、**活水缸（SO-C-GBA）须用复合 ROI（§3）** → 须算「改造投入 vs 生鲜损耗降低/客数提升」回本周期 | **象限四·接受/不改**：非承重的次要空间浪费、边缘区照明 → 硬件接受，不投资本 |
| **改造成本低/零**（运营可弥补） | **象限二·立即做（最高优先）**：生鲜补光灯（SO-C-P2）、整洁度 SOP（SO-D-O-P1·D-O项）、门头改造（SO-D-P1）、加固定导视（SO-D-P2）、骑车划线（SO-A-P3）→ ROI 极高，**先做这些** | **象限三·顺手改**：小额优化，排在象限二之后 |

> **注**：SO-D-O-P1（整洁度）虽不计 L1 分，但仍进象限二执行——不计分 ≠ 不处理，整洁度对 D7 的影响极大，SOP 弥补是最高优先。

### 5.2 处置通道判定

| 短板类型 | 通道 | 依据 |
|---|---|---|
| **A 落位类**（SO-A-P1 回家动线偏离） | **多数不可改** → 转「选址复盘/接店评估」，为未来选址积累判据 | 搬迁不现实，接受约束 |
| **C 冷链类**（SO-C-P1 温区缺失） | **资金改造·象限一** → 算 ROI（损耗降低回本） | 生鲜品质物理底座，SOP 救不了 |
| **B 布局类**（SO-B-P2 面积配比） | **中成本改造** → 货架/隔断调整 | 非承重可改，算调改成本 |
| **D-H 形象硬件类**（SO-D-P1 门头 / SO-D-P2 固定导视） | **中低成本改造** → 招牌灯箱/吊牌系统 | 物理固定件需投资安装 |
| **D-O 运营形象类**（SO-D-O-P1 整洁 / 异味） | **运营弥补·象限二** → SOP/零成本，立即做 | 不计 L1 分，但 D7 影响大，必须处理 |
| **C 照明类**（SO-C-P2 补光灯） | **运营弥补·象限二** → 小额，立即做 | 低成本高 ROI，运营主战场 |

> *【张力·必填】① 「不花钱补 80%」的判据在**加盟店**失效——加盟店改造权/整洁标准归品牌方，店主想运营弥补也无权；此时短板转为「与品牌方协商项」。② 冷链 ROI 计算依赖**准确的损耗数据**，而多数社区店无损耗台账（见 KB-DX-FULLSTORE-001 D2-4），故 ROI 常为估算 C 级，改造决策须标「基于估算损耗」。③ 「运营弥补优先」不等于「硬件短板可以永远不改」——整洁度 SOP 能弥补一时，但若 C 维冷链是真短板，运营弥补只是拖延，须在报告明确区分「弥补」与「治本」。*

## 6. 门店全景诊断编排（与首访对齐）

诊断动作嵌入 [[S02_首访资料采集]] 首访流程，**在 7D 之前先扫 L1**：

```
L0 市调（商圈客群·接店前/首访前案头）
  → L1 门店全景四维扫描（首访现场·本框架）
      A 落位（步测店周） → B 动线（/read-floorplan 平面图 + 走店）
      → C 硬件（设备清点） → D-H 形象（门头/固定导视/外立面）
      同步记录 D-O 运营形象（整洁度/异味·不入L1分）
  → 产出①「硬件天花板红黄绿表」（§4·定性）
     + ②「D-O 运营形象问题清单」（归L2 D7·独立）
  → L2 7D 经营诊断（读天花板后判读经营指标是否触顶）
  → 改造优先级矩阵（§5）+ 7D TOP3 行动
```

**编排铁律**：
1. **L1 先于 L2**：7D 打分前必须先有 L1 天花板（定性红黄绿），否则会把硬件锁死的低指标误判为运营问题。
2. **硬件维度证据优势**：L1 多为现场可核 B 级（甚至设备清点 A 级），比 7D 的 C 级证据硬——**L1 结论可信度高于 L2**，可作为 7D 判读的锚。
3. **产出三清单**：L1 产出「硬件改造清单」（资本决策）+ 「D-O 运营问题清单」（SOP 立即做）+ L2 产出「TOP3 运营动作」——三清单分开给店主，别混（第一个要花钱、第二个靠SOP、第三个靠管理）。

## 7. 花厅坊直接调用（落地锚点）

> *【KB 质量铁律·直接调用性】下次首访/复诊花厅坊时，本框架可直接生成「→ 下一个具体动作」：*
> - **A 落位**：先步测花厅坊 500m 居民户数 + 是否在回家主动线上 + 骑车停放秩序（按 A3 三档锚点打分）→ 定 D4 客数天花板红黄绿。花厅坊 30T vs 90T 日均 +14.2% 若为客单数驱动（健康），须先确认 A 维未把客数物理锁死。
> - **B 动线**：用 `/read-floorplan` 读花厅坊平面图 → 测死角占比 + 生鲜是否先导位 → **f≈70.74% 时 B2 基准上调至 40-50%**，若生鲜区<40% 即为配比失衡 → 若 T03 饼干等品类错配（M-DEC-010）叠加动线死角，先判「是布局问题还是陈列问题」再派动作。
> - **C 硬件**：清点花厅坊冷柜延米/温区覆盖（按 C1 三子项打分）→ **这是判读 D2 生鲜的前置**：花厅坊 f≈70.74% 生鲜占比极高，若 C 维冷链温区不全，D2 高占比是「危险的高」（品质无保障的高损耗），补光灯（SO-C-P2·象限二）优先立即做，活水缸考虑复合 ROI（§3）。
> - **D-H 形象**：查门头灯箱/招牌结构（D-H1）+ 固定导视完整性（D-H2）→ 进 D-H 硬件计分；同步记录整洁度/异味（D-O·不计L1·立即出 SOP 结论）。
> - **改造判断**：花厅坊硬件短板按 §5.1 矩阵分流——冷链缺口进象限一算复合 ROI；补光/固定导视/门头进象限二立即做；整洁度/异味作为 D-O 独立清单驱动 SOP。
