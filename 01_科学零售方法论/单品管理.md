# 单品管理

## 1. 文件定位

本文件是科学零售体系中**单品管理**的主定义文件，属于商品力下游并列模块，是 `04_商品诊断与商品力提升` 的方法论上游依据，同时支撑 `05_品类管理与商品规划`、`06_陈列规划与DisplayMap`、`08_经营分析与数据看板` 的落地作业。

单品管理与 [[品类管理]] 并列，均属于商品力的下游模块：

- **品类管理** 解决**结构层**问题：品类边界、品类角色、价格带、宽度深度、资源配置、策略方向。
- **单品管理** 解决**SKU 层**问题：SKU 进退、动销效率、毛利贡献、库存健康、引进、淘汰、替换。

商品配置理论、52周MD 同为商品力下游并列模块，文件尚未建立，相关引用以文字说明代替。

## 2. 核心定义

**单品管理**是以 SKU 为最小经营单元，对在架商品进行持续评估、分类、决策和动作管理的方法。

单品管理的本质是 **SKU 级经营控制系统**，而不是商品清单管理。它回答的核心问题是：

> 品类内的每一个 SKU，当前是在贡献价值还是在消耗资源？下一步应该加码还是退出？

单品管理是商品力落地的最细颗粒度，是连接品类策略与货架现场的执行节点。

### 2B. 单品管理的理论根（零售老刘逐字批25/26回填·六哥 2026-06-25 签）

> 来源:零售老刘《品类建模》165/168/172(转述)。**给"为什么单品=最小经营单元"补上理论根**。

- **单品 = SKU = 最小可执行/可度量/可优化的原子管理单元**。**"以单品为战略单元"**(优于"以品类为战略单元"——品类管不到颗粒,精力不能只盯几个单品)。单品四逻辑层:**①识别(商品→数据元/SKU唯一身份)②分析(总体概览→微观归因·"利润增长具体是哪几个单品贡献的")③决策(经验→数据驱动)④协同(部门墙→统一语义·采购/运营/财务都围绕同一 SKU)**。
- **"管理 = 度量"推导**:单品管理=单品+管理=单品+度量 ⟹ **管理=度量**。所以单品管理落地=抓对的度量(9维ABCZ就是这套度量)。**"数量"指标不可替代**:先有数量再有销售额,任何单品业务表必须有数量列(报货/库存/退换货都是数量形态的流转)。
- **单品 = 零售的"毛细血管"**:**"零售总利润 = 所有单品利润之和"**——利润回不到单品,精细化管理就没有闭环(很多店利润只算到大类/区位=失去毛细血管=易"洪灾")。单品四现实场景:**财务(可度量利润单元)/运营(空间陈列+时间生命周期承载者)/效率(客流/周转/动销/坪效晴雨表)/消费(顾客决策最终触点)**。
- 与 [[品类管理]] 分工:品类管"结构层"(Item/需求),单品管"SKU 层"(选择/进退);链条=**Item 定需求 → [[商品配置理论]] HS 选 SKU → 本模块管 SKU 进退**。

## 3. 单品管理解决的问题

以下问题由单品管理负责，品类管理不深入到这一颗粒度：

| 问题类型 | 具体问题 |
|----------|----------|
| SKU 进退决策 | 这个 SKU 该不该继续经营？是淘汰、替换还是加大力度？ |
| 动销效率 | 这个 SKU 在架多长时间没有销售记录了？动销率是否正常？ |
| 毛利贡献 | 这个 SKU 销量尚可，但是否在侵蚀毛利？ |
| 库存健康 | 这个 SKU 的库存是否积压？周转天数是否异常？ |
| 新品引进 | 哪些新 SKU 值得引进？进店后多久评估？评估标准是什么？ |
| 滞销淘汰 | 哪些 SKU 应该被淘汰？淘汰的判断标准和执行流程是什么？ |
| 候选商品池 | 被淘汰 SKU 的替代候选从哪来？如何维护候选池？ |
| 缺货响应 | 哪些 SKU 出现缺货？缺货期间如何避免误判动销数据？ |

## 4. 单品管理与品类管理的关系

**两者不是上下级关系，而是不同颗粒度的协同关系。**

| 维度 | 品类管理 | 单品管理 |
|------|----------|----------|
| 颗粒度 | 品类级 | SKU 级 |
| 核心问题 | 这个品类的结构是否合理？角色是否清晰？ | 这个 SKU 在品类内是否在创造价值？ |
| 主要工具 | 品类角色、价格带、CDT、品类评估 | ABCD 分类、ABC-Z 模型、动销分析、候选商品池 |
| 信息流向 | 为单品管理提供 SKU 宽度、深度、价格带目标 | 为品类管理提供单品贡献数据、结构健康状态 |
| 动作触发 | 品类策略调整时，通知单品管理执行 SKU 进退 | 单品诊断结果反馈品类评估，修正品类判断 |

**协同流程：**

```
品类管理（确定品类宽度/深度/价格带目标）
    ↓
单品管理（在目标框架内执行 SKU 评估和进退决策）
    ↓
品类管理（收到单品评估结果，修正下一轮品类策略）
```

## 5. 单品管理的核心对象

在本知识库中，**单品 = SKU**，以商品条码（EAN/GS1/UPC）作为唯一识别方式。

| 概念 | 定义 | 说明 |
|------|------|------|
| 单品 | 经营管理的最小商品单元 | 本知识库中等同于 SKU，是所有决策和动作的基本单位 |
| SKU | Stock Keeping Unit，库存管理单位 | 同一商品不同规格（如 500ml 和 1L）视为不同 SKU |
| 条码商品 | 有唯一条码的物理商品 | 是单品的识别手段，但条码数据质量问题需通过数据清洗处理 |

**条码数据的质量风险：**

一个 SKU 理论上对应一个主条码，但实操中可能存在：
- 一品多码（同一 SKU 有多个条码版本）
- 无码商品（散装/生鲜）
- 条码录入错误
- 进口商品原产地条码与国内流通条码并存

单品管理依赖**经过清洗的商品档案数据**作为基础，未清洗的条码数据不可直接用于 SKU 分类和诊断。商品档案数据清洗规则由 `13_数据分析与工具脚本` 承接（待完善）。

## 6. 单品管理的关键指标

单品管理依赖以下指标对每个 SKU 进行持续评估：

| 指标 | 分类 | 含义 | 说明 |
|------|------|------|------|
| 销售额 | 金额类 | 该 SKU 在统计周期内的销售总金额 | 单独看销售额不充分，需结合毛利额和动销率综合判断 |
| 毛利额 | 金额类 | 该 SKU 在统计周期内贡献的毛利总金额 | 毛利额比毛利率更直接反映单品价值贡献 |
| 毛利率 | 结构类 | 该 SKU 的商品利润率 | 高毛利率但销量极低的 SKU，未必是值得保留的 SKU |
| 动销天数 | 效率类 | 统计周期内有销售记录的天数 | 连续无销售天数是滞销预警的核心指标 |
| 动销率 | 效率类 | 统计期内有销售记录天数占总天数的比例 | 与动销天数配合使用（待确认：以哪个作为主指标） |
| 库存数量 | 库存类 | 仓库+货架该 SKU 当前库存总量 | 结合日均销量判断库存是否合理 |
| 库存周转天数 | 库存类 | 当前库存可支撑的销售天数 | 过长→积压风险；过短→缺货风险；各品类合理上限待确认 |
| 品类贡献占比 | 结构类 | 该 SKU 销售额/毛利额占所在品类总量的比例 | ABCD 分类和陈列资源分配的核心依据 |
| 陈列面数 | 资源类 | 该 SKU 当前在货架上占据的排面数 | 结合销售贡献判断陈列资源是否匹配 |

## 7. 单品分类方法

### 7.1 ABCD 分类

ABCD 分类是单品管理的**基础分类工具**，按品类内的贡献水平将 SKU 分为四类，用于驱动差异化的管理动作。

| 类别 | 定义 | 管理动作方向 |
|------|------|------------|
| A 类 | 品类内高贡献 SKU，是品类销售额/毛利额的核心支撑 | 保证货源、保证陈列面、稳定定价，持续强化 |
| B 类 | 品类内中等贡献 SKU，销售稳定但贡献适中 | 维持现状，关注趋势；有改善空间时可适度强化 |
| C 类 | 品类内低贡献 SKU，销售有但贡献有限 | 收缩陈列，纳入观察期；若持续无改善则进入淘汰候选 |
| D 类 | 品类内极低/零贡献 SKU | 淘汰候选，启动候选商品池替代流程 |

**分类方法说明（待确认）：**
- 分类的**主要维度**：以销售额贡献为主？毛利额为主？还是多维度综合评分？权重待确认
- 各类的**划分阈值**：固定比例（如各类占20%/30%/30%/20%）还是按品类内相对排名浮动？待确认
- **评估周期**：建议月度更新，活动前后做临时复盘；具体频率待确认

**分类工具：** Excel 模板或系统自动分类，具体分类维度与权重待确认。模板由 `13_数据分析与工具脚本` 承接（待建立）。

### 7.2 ABC-Z 模型

ABC-Z 模型是在 ABCD 分类基础上的**管理动作框架扩展**，补充了 Z 类异常识别和处置机制。

| Z 类情况 | 典型表现 | 处置方向 |
|----------|----------|----------|
| 零动销商品 | 统计周期内完全无销售记录 | 核实原因后启动淘汰或调仓决策 |
| 数据异常商品 | 条码错误、库存负数、价格为零等 | 先修正数据档案，再重新纳入分类评估 |
| 已停售商品 | 实际已下架但系统仍在档 | 从在档清单中移除，不纳入评估统计 |

**关系说明：** ABCD 分类是当期贡献评估工具，ABC-Z 是动态全生命周期管理框架。ABCD 给出排名，Z 类识别系统无法自动处理的异常状态，两者配合使用。

> **商品 7 大状态·生命周期状态机(零售老刘逐字批16回填·六哥 2026-06-25 签)**:**建档 → 新品 → 正常 → 停购 → 停售 → 淘汰**(进销存生命周期;另有 6 态分法:建档/新品/正常/暂停/淘汰/删除)。状态联动:停购→采购系统自动屏蔽订货;状态三维=基础销售状态(可售否)/基础业务状态(库存陈列·补货逻辑)/临时特殊状态(样品/召回)。**ABCZ 汰换前先读商品状态**(新品给爬坡期、不按低销砍;停售/淘汰才出统计)。数据源=adapter 的状态字段。
> **状态机不可逆 + 状态=熵增(逐字批20回填)**:转档**只能由前往后**(正常←新品转正,不可逆),系统强约束。**商品状态本身就是一种"熵增",抗熵增=提升商品管理效率的必由之路**(呼应失真库哲学根「失真=熵增」)。状态管理混乱=熵值飙升=管理失效。

**待确认：** ABC-Z 模型是否需要独立建立主定义文件？当前仅在本文件中建立骨架定义。

### 7.3 9 维评分与量化阈值（解决 §7.1 "权重待确认"）

> 来源：v3.2 SKU 分层逻辑 §2-3（打捞 SALVAGE-BACKLOG #2）。**这就是 §7.1 "权重待确认" 的答案。**

**9 维度评分（0-100，与品类均值比较）：**

| 维度 | 权重 | 数据来源 |
|------|----:|---------|
| ① 销售额贡献 | 0.25 | 近28天销售额 |
| ② 毛利额贡献 | 0.25 | 近28天毛利额 |
| ③ 动销率 | 0.15 | 近8周动销周数/8 |
| ④ 库存周转速度 | 0.15 | 周转天数（目标≤15天） |
| ⑤ 价格带匹配 | 0.10 | 售价 vs P域/M域 |
| ⑥ 规格带合理性 | 0.03 | 规格带标签 |
| ⑦ 关联销售贡献 | 0.03 | 连带购买（度量=连带率/品单数,数据源=小票购物篮;**计算用关联三度:支持度(A∩B/总订单)/置信度(A∩B/A)/提升度**;零售老刘B4+逐字批04回填) |
| ⑧ 供应稳定性 | 0.02 | 近8周断货次数 |
| ⑨ 门店适配度 | 0.02 | 门店定位匹配 |

综合评分 = Σ(维度得分 × 权重)。**评分区间**：85-100 核心款 / 70-84 A类主力 / 50-69 B类维护 / 30-49 C类观察 / **0-29 D类淘汰候选**。

**ABCZ 量化阈值（帕累托累计 + 零销识别）**：零销（近8周=0）直接 D 类；非零销按销售额累计 0-70%→A / 70-90%→B / 90-99%→C / 末1%或评分<30→D。典型 SKU 占比 A 15-25% / B 30-40% / C 20-30% / D ≤10%。
> **标准ABC基准(零售老刘逐字批10回填·30:70递归)**:教科书标准比 = **品种数 A10%/B30%/C60% · 销售额 A50%/B40%/C10%**(约10%A品实现50%销售)。用作 ABCZ 分布的**对照基准**:实测显著偏离(如A品种>25%或销售额<40%)= 结构异常信号(头部过宽/长尾过厚),非硬阈值。

> **汰换串联**：先过 [[SKU角色层与目的品保护机制_v0.1]] 防误杀闸（引流/形象/GBA锁定豁免）→ 剩余跑本 9 维评分 + ABCZ → D类 + 补充角色 + 无锚点 = 汰换候选。状态 `candidate` 待战役验证。

### 7.3B 可得性防误杀 + 保利期判据（零售老刘吸收回填·六哥 2026-06-25 签）

> 来源：零售老刘《品类建模》选品配置/商品组合系列(转述)。见 [[2026-06-25_零售老刘吸收_方法论回填清单_v0.1]]。**不入 9 维权重表**——可得性=判据闸、保利期=Z 汰换触发器,比硬加权重维更准。

1. **可得性防误杀(汰换前必查)**：低销 SKU 先问"是真滞销,还是**缺货导致的伪滞销**"。`实际可用 = 理论 × 现货率 × 履约率`,**现货率低的低销 = 缺货,不是该砍**。与角色闸并列为第二道防误杀。
   - **缺失机会抓手(逐字批04回填)**:**孤品篮子**(单商品小票·占约1/3=顾客想买更多没买)、**孤儿商品**(放入篮但没买/丢角落·建"孤儿商品登记表"找不买原因)、**零事务**(无销售/零毛利/周转0)——这些都藏着"想买而没买/没买够"的机会,是可得性/选品改进信号,不是简单砍。
   - **转身层配置(逐字批11回填)**:为高频痛点品配"替代痛点"的商品(转身层),缺货时顾客能转身取替代品——防"缺货=痛点无解=流失"。替代品本身是连带/防缺货角色,不因单看低销砍。
2. **保利期汰换判据(Z 类增强)**：**商品保利期 = 从购进到售出能实现目标利润的最长允许库龄**(毛利被持有成本消耗完的天数)。**库龄 > 保利期 = 隐性亏损 → Z 汰换,即使毛利率高**。把静态毛利率升级为"毛利 × 周转时间"的动态判断。
   - **持有成本构成(逐字批08回填)**:**超龄库存费(>180天)、弃置费(临期/报损/损耗·生鲜大)、资金占用、仓储费**——这些就是吃掉毛利的"持有成本"。弃置费率还反映门店管理水平。
   - **超保利期处置(逐字批09回填)**:① **折扣触发规则:库龄 > 保利期 → 打折(库龄越长力度越大)**,在保利期内提示预警;② **剩余有效期定价**:保质期临近=价值下降,**临期应按"剩余使用期/全周期"折价**(防"按原价卖临期"误导顾客=一种价值失真)。
   - **生鲜"一日三价"(逐字批26回填)**:原则=**"在单品最有价值的时候卖出去"**——早市最新鲜→正价(最贵);午后稍蔫(不影响售卖)→低于原价;闭店前→打折出清。这是保利期判据在生鲜短保品上的"日内"版:库龄按小时计,定价随剩余价值递减。**零售追"效率"的核心原因=几乎所有单品都有保质期,效率决定单品能否在最有价值时售出**。
3. **效率/贡献度量(逐字批07回填)**:**GMROI = 毛利率 × 周转率**(筛"高毛利高周转"核心品,优先补货陈列);**贡献率 = 品类结构占比 × 毛利率**(识别主力品类)。两者比单看毛利率/周转更准。
4. **到位率非越高越好(逐字批07回填)**:订货到位率(服务率)合理区间 90-95%(Z 1.68-1.74),**增 1 点到货率可能需 2-3 倍库存**;盲目追 98-100% = 库存积压。与可得性配对:可得性看"够不够",到位率看"补到几成最划算"。
5. **极致SKU=主做"短尾"(逐字批22回填·砍长尾的正向逻辑)**:硬折扣(奥乐齐<2000·甚至<1000;一般店3500-5000)的精简逻辑=**减少选择 / 提升周转 / 压缩成本 / 高动销→供应链反应快**(二八法则)。OEM 自有品牌覆盖约 80% 需求、便宜 2-3 成。**给"砍长尾"一个正向理由**:不是单纯防误杀(§7.3B-1)或防超龄(§7.3B-2),而是短尾本身=高动销=供应链快=成本低的主动战略。**硬折扣(系统性重构供应链)≠ 软折扣(只压价)**;"简约不简单"=最优配比,非越少越好。⚠️ 砍长尾仍须先过角色闸 + 可得性防误杀两道闸,短尾逻辑是"砍后"的正向收益,不是"乱砍"的许可。
   - **宽类窄品的有效性前提(逐字批26回填)**:**"宽类窄品"=窄掉非畅销 SKU(1万+→5000-8000·70%新品)、腾笼换鸟换生鲜/熟食/烘焙的宽类空间**(沃尔玛"单品驱动+EDLP+净成本"同源)。但"窄品 ≈ 电脑重启/重装"=**治标不治根**:**同样窄品有的成功有的失败,区别只在"是否基于自身目标顾客"**——窄品前必须先答"我的目标客群要哪 300 个小类需求",否则砍完一段时间又臃肿。社区店落地:窄品=阶段性"重启",不是一劳永逸,配套需求洞察才有效。
6. **JIT 准时制=可得性的供货侧(逐字批22回填)**:**JIT(Just-In-Time)=在需要的时间、按需要的量供应**(丰田 TPS 核心),接近**零库存**、**小批量高频次补货**(如面包每日 2-3 配)、与供应商深度协作共享需求数据。**售完即补,非提前囤积**。与 §7.3B-1 可得性配对:**可得性看"货架够不够"(需求侧),JIT 看"如何补到刚好够而不囤"(供货侧)**;与 §7.3B-4 到位率配对:JIT 是把"到位率最划算点"落到补货节奏上的供应链手段。生鲜/短保品尤其受益(高频次锁鲜)。
> ⚠️ 上述依赖**准进价 + 库龄/周转数据**;花厅坊进价不准是已知坑,故 `candidate`,数据到位才硬用。第 5-6 点的硬折扣/JIT 是**大店重构级**能力,社区超市落地需量力,先作判据参照而非照搬。

## 8. 单品动作机制

单品管理的日常运作包含以下四类核心动作：

| 动作类型 | 触发条件 | 执行内容 |
|----------|----------|----------|
| **引进** | 品类有空缺、有替换需求、季节波段或活动需要新品 | 从候选商品池中确认进店 SKU，完成档案录入和陈列配置 |
| **淘汰** | SKU 被评为 D 类或 Z 类（零动销），或供货中断 | 启动淘汰流程，释放陈列面和库存资金，由候选商品池补位 |
| **替换** | 主力 SKU 停产/供货不稳/同类更优品进入市场 | 引进替代品，同时淘汰被替换品，保持品类 SKU 数稳定 |
| **调整** | 陈列面与销售贡献不匹配（A 类缺陈列、D 类占过多） | 重新分配货架排面，与 DisplayMap（未建立）联动执行 |

## 9. 新品引进机制

新品引进是单品管理的主动输入动作，解决品类 SKU 来源问题。

**引进来源（待确认是否完整）：**
- 供应商主动推介
- 竞品门店同类爆品观察
- 顾客反馈或投诉中反映的缺失商品
- 季节/节令波段的计划性补充
- 品类结构调整的主动选品

**引进流程（骨架）：**

```
候选商品池提名
    → 商品基础信息确认（条码/规格/进价/供货条件）
    → 试销上架
    → 试销期评估
    → 正式引进 / 退出决策
```

**试销周期（待确认）：** 建议按品类差异设定，高频消耗品和低频品的评估节点不同，具体天数待通过实际案例验证后确认。

---

## 10. 滞销淘汰机制

滞销淘汰是单品管理的清退动作，解决低效 SKU 占用陈列和库存资源的问题。

**淘汰触发条件（待确认）：**

| 条件类型 | 典型判断 |
|----------|----------|
| 连续零动销 | 在统计周期内连续 X 天无销售记录（X 待确认，建议按品类差异处理） |
| ABCD 长期 D 类 | 连续 N 个评估周期均为 D 类（N 待确认） |
| 库存周转严重超标 | 库存周转天数超过各品类合理上限（上限待确认） |
| 供货条件恶化 | 主供应商停止供货、价格严重偏移市场 |

**淘汰流程（骨架）：**

```
D 类 / Z 类识别
    → 人工复核（排除干扰因素）
    → 确认淘汰
    → 候选商品池补位选品
    → 下架处理
```

**重要约束：** 滞销淘汰决策必须先排除活动、陈列、缺货、价格异常等干扰因素，再判断商品是否真正滞销。活动停止后的销量回落、陈列调整期间的临时数据、缺货期间的零销售记录，均不能作为滞销的直接依据。

**淘汰周期（待确认）：** 建议按品类分组差异处理，短保商品、食品饮料、日用品的合理判断周期不同，具体数值待通过实际案例验证后确认。

## 11. 候选商品池机制

候选商品池是单品管理的**后备供给机制**，解决淘汰 SKU 的替补来源问题，保证淘汰动作有商品可进，不造成品类空洞。

**候选商品池定义：**
已完成基础信息评估、具备引进条件、尚未正式上架的 SKU 清单。

**候选池来源（待确认是否完整）：**
- 供应商推介但尚未决策的商品
- 市场调研或竞品观察中发现的强势 SKU
- 历史上曾销售、表现不错但因档期下架的商品
- 季节性波段提前锁定的备选商品

**候选池管理规则（骨架，待细化）：**
- 候选池每条 SKU 须有基础档案：品名、规格、条码、供应商、含税进价、建议零售价、品类归属
- 须标注：推荐原因、预期替换的 SKU 类型、推荐来源
- 定期清理过期/停供候选 SKU，保持候选池有效性

**待确认：** 候选商品池是否需要独立建立主定义文件？当前作为单品管理的一个节建立骨架定义，待规则复杂度提升后再判断是否独立。

## 12. 缺货管理机制

缺货是单品管理中需要识别和响应的异常状态，同时也是动销数据评估的重要干扰源。

**缺货的主要类型：**

| 类型 | 含义 | 处置方向 |
|------|------|----------|
| 货架缺货 | 库存有货但货架无货（补货不及时） | 及时补货上架，检查补货频率设置 |
| 库存缺货 | 门店库存已耗尽，无法补货 | 紧急下单补货，评估安全库存水位 |
| 供应链缺货 | 供应商断货或延迟发货 | 启动候选商品池替代，或临时调货 |

**缺货对单品数据的干扰（重要）：**

缺货期间的零销售记录**不等于无需求**。在对 SKU 进行 ABCD 分类或滞销判断时，必须识别并排除缺货区间的数据，否则可能将正常需求商品误判为滞销商品并错误淘汰。

**归属说明（已确认口径）：**
- 本文件负责：缺货对 SKU 动销评估的数据影响和排除规则
- 运营力侧后续承接：缺货发现、补货执行、供应链协调的操作 SOP（文件待建立）

## 14. 在社区超市中的应用

社区超市的单品管理不能照搬大型连锁标准，必须考虑以下约束：

- 社区超市 SKU 规模因面积、业态定位、系统管理水平不同而差异较大，具体范围待通过样板门店数据确认。
- 员工执行能力有限，数据录入和档案维护能力较弱。
- POS 系统数据质量参差不齐，条码匹配率可能低于理想水平。
- 采购决策常由老板或采购负责人独立执行，缺少系统化的引进/淘汰流程。

**第一阶段优先解决（建议）：**

1. **商品档案清洗：** 确保在档 SKU 有准确条码和品类归属，这是所有分类和诊断的前提
2. **首次 ABCD 分类：** 先做一次全品类分类快照，识别 D 类和 Z 类
3. **启动首批淘汰：** 从 D 类中有把握的商品开始清退，释放货架资源和库存资金
4. **建立候选商品池：** 维护一份可进店的备选 SKU 名单，为淘汰补位做准备

乐易购花厅坊店可作为后续测试案例，但真实数据和结论待现场录入，不得编造。

## 16. 常见误区

- **误区1：** 只看销量高低判断 SKU 好坏——需同时看毛利额、动销率、库存周转，不能单维度判断。
- **误区2：** 把零动销直接等同于滞销——需先排除缺货、陈列问题、活动后效应等干扰因素，再下结论。
- **误区3：** 引进新品不设试销周期和退出机制——无退出机制的引进，会导致 SKU 数持续膨胀，拖累品类效率。
- **误区4：** 淘汰只看当前表现，不看历史周期和季节性——部分 SKU 是季节性商品，非旺季的低动销不代表可以淘汰。
- **误区5：** 把单品管理等同于商品诊断报告——商品诊断是一次性项目输出，单品管理是持续运行的经营机制。
- **误区6：** 单品管理归品类管理管理——两者并列，单品管理有独立的问题域和工具体系，不是品类管理的附属执行。
