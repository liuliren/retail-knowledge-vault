# 运营力总纲

## 1. 文件定位

本文件定义科学零售体系中**运营力**的核心概念、构成框架和典型表现，是 `06_陈列规划与DisplayMap`、`07_营销活动策划与实施`、`08_经营分析与数据看板` 等模块的方法论上游依据。

## 2. 核心定义

**运营力**是门店把商品策略和组织执行转化为持续经营结果的能力，是通过陈列、活动、日常运营、异常处理和经营复盘，持续改善门店经营表现的能力。

运营力不是简单的"日常杂事管理"。运营力的核心是**持续运转、动态调整和闭环改善**。

三力分工：商品力决定"卖什么"，组织力决定"谁来做、怎么做"，运营力决定"如何持续跑起来、如何不断改善"。

## 3. 运营力的核心构成

| 构成模块 | 核心内容 |
|----------|----------|
| 陈列优化 | 按 DisplayMap 落地货架陈列，持续维护陈列标准，根据动销数据调整排面 |
| 营销活动 | 按 52周MD 节奏策划和执行堆头、端架、促销活动，并复盘效果 |
| 门店日常运营 | 开关店流程、补货、价签、卫生、损耗管理等日常执行闭环 |
| 异常管理 | 缺货、滞销、价格异常、设备故障、顾客投诉的快速识别和处理 |
| 库存与缺货处理 | 库存水位监控、缺货上报与补货追踪、滞销商品及时处置 |
| 周/月复盘 | 定期对经营数据进行结构性分析，识别问题根因，输出下一周期调整动作 |
| 经营改善闭环 | 把复盘结论转化为可执行任务，追踪执行，验证改善效果 |

## 4. 运营力在社区超市中的典型表现

**强运营力的门店：**
- 陈列有标准，货架整洁、分区清晰，顾客能快速找到商品
- 活动有节奏，提前规划、有序执行、结束后复盘效果
- 异常有响应，缺货当天补、滞销定期处置、顾客投诉有闭环
- 复盘能转化为动作，数据不只是看，而是驱动下一步调整

**弱运营力的门店：**
- 陈列靠感觉，随意摆放，无标准可执行
- 活动随意上，没有节奏，复盘缺失，反复踩坑
- 缺货滞销无人跟踪，只到问题严重时才发现
- 复盘只看数据，不输出改善动作，会议结束一切照旧

## 5. 与商品力、组织力的关系

三力协同，缺一不可：

- **商品力**提供经营内容和商品结构，决定门店卖什么、卖给谁、如何卖
- **组织力**提供执行主体和流程保障，决定谁来做、按什么标准做
- **运营力**负责把商品和组织转化为持续经营结果，决定如何持续跑起来、如何不断改善

商品策略再好、组织配置再合理，没有运营力持续落地和动态调整，门店经营结果无法持续改善。

## 6. 与52周MD的关系

52周MD 是运营力在年度商品节奏、活动节奏、季节波段和门店节奏上的重要工具。运营力通过 52周MD，把品类策略、商品配置、季节波段和营销活动，转化为年度、月度、周度的运营动作。

具体体现为：提前规划备货时间、陈列切换节点、活动执行周期和复盘频率，让运营动作从被动响应升级为主动节奏管理。

## 7. 与数据与系统能力底座的关系

数据与系统能力底座为运营力提供数据支撑，主要体现在：

- 经营看板：销售额、毛利额、客单价、客流实时可见
- 异常预警：缺货、滞销、毛利异常、库存超期自动提醒
- 活动复盘：区分自然动销与促销拉动，评估活动 ROI
- 销售日报/周报/月报：自动生成，降低人工整理负担
- 库存与缺货预警：提前发现风险，减少被动处理
- 老板看板：简化版核心指标，支持快速决策
- 任务执行跟踪：运营动作布置后的追踪与验收

**注意：** 系统和数据不能替代运营判断，只能提高运营动作的可见性、可追踪性和复盘效率。

## 8. 常见误区

- **误区1：** 把运营力等同于"店长忙不忙"——忙不代表有效，运营力的核心是结果改善，而非工作量。
- **误区2：** 只做活动，不做复盘——活动结束不复盘，下次活动重复踩坑，无法积累有效经验。
- **误区3：** 只看销售额，不看毛利、库存、缺货和执行质量——销售额高不等于运营健康，需多维度综合判断。
- **误区4：** 把运营力和组织力混为一谈——组织力是"谁来做、怎么做"，运营力是"如何持续跑起来、如何改善"，两者层次不同。
- **误区5：** 把数据看板当成运营改善本身——看到数据只是起点，把数据转化为执行动作才是运营力的体现。
