# CDT 应用与子格落位方法

## 1. 核心论点（结论先行）

子格落位必须先画 CDT（顾客决策树）——货架从左到右的子格顺序是顾客视觉扫描路径和购买决策路径的物理表达，跳过 CDT 直接落位等于跳层操作，会系统性拉低面位效率，即使短期销售数据看起来正常。

## 3. 原文主体内容

### 一、CDT 是什么、为什么重要

CDT（Customer Decision Tree，顾客决策树）是品类管理中描述顾客购买决策路径的工具。核心问题：**当顾客走到某个货架前，他是按什么顺序做出"买哪个"的决策的？**

**子格落位为什么必须先画 CDT：**

货架上的子格顺序（从左到右、从上到下）本质上是顾客视觉扫描路径和决策路径的物理表达。

| 情况 | 结果 |
|---|---|
| 子格顺序与顾客决策路径**对齐** | 快速找到、快速决策、快速完成购买；30 秒找到率达标；销售自然跟上 |
| 子格顺序与顾客决策路径**错位** | 顾客需扫多遍才能找到；决策成本上升；销售被惯性带，没发挥面位潜力 |

### 二、CDT 的标准画法（六步）

**第一步：列出该面位所有候选品类（L3 级）**
按面位任务锁定表中的 L3 清单列。

**第二步：找出顾客的第一层决策维度**
第一层决策通常是属性差异最大的维度，判断标准是"顾客在脑海里第一个分类的维度"：
- 材料/口味（如甜口 vs 巧克力的"巧"）
- 价位带（基础款 vs 高端款）
- 场景（即食 vs 家庭装 vs 节令）
- 客群（儿童 vs 成人）

**第三步：按第一层决策切大格**
将所有 L3 按第一层决策维度分组，每组对应一个"大格"。

**第四步：找出第二层决策维度**
在每个大格内部找第二个决策维度，通常是：
- 包装规格（小包装 vs 家庭装）
- 品牌（主力品牌 vs 跟风品牌）
- 风味子分（如薯片原味 vs 烧烤味）

**第五步：按第二层决策细分**
在每个大格内按第二层决策细分，对应到具体子格。

**第六步：验证视觉扫描路径**
把整个落位顺序从左到右、从上到下读一遍——如果某个跳跃感觉别扭，就是 CDT 还没对准。

---

### 三、子格落位的层位规则（纵向 L1-L6）

CDT 切横向格位顺序，层位规则管纵向。

**6 层结构标准任务分配：**

```
L1 形象层      —— 形象品 / 品牌墙 / 视觉锚点
L2 黄金视线层  —— 主力品（销售大头）
L3 黄金手拿层  —— 主力品 + 促销装
L4 补充/特色层 —— 长尾补充 / 小众特色
L5 袋装/便携层 —— 小包装 / 便携装 / 独立装
L6 围货/家庭层 —— 家庭装 / 大袋装 / 围货装
```

**四条铁律：**
- **纪律 1：主力居中层（L2-L3）**——黄金视线和黄金手拿是销售核心层；任何把主力放到其他层的设计都是错的。
- **纪律 2：家庭装下置（L6）**——家庭装重、体积大，下置便于拿取；购买决策较慢，下层不影响快购客流。
- **纪律 3：便携居中（L4-L5）**——小包装、独立装放 L4-L5，便于晚高峰快购客一手抓走。
- **纪律 4：形象层做品牌墙**——L1 不是放"包装最好看的商品"，而是承担形象输出任务。

---

### 四、CDT 应用的常见错误

**错误 1：把第二层决策当作第一层**
例子：某面位先按"清凉糖 vs 软糖 vs 棒棒糖"切大格——这是糖类内部子分类（第二层），第一层应先切"糖 vs 巧克力"。
纠偏：问自己"哪个维度是顾客最直觉、最自然的分类"——通常是材料、价位、场景三选一。

**错误 2：用销售数据切 CDT**
例子：销售大的品类放主格，销售小的品类放边角格——这是销售集中度，不是 CDT。
纠偏：CDT 切顾客决策路径，不切销售数据。销售数据用于错配诊断（A4），不用于 CDT。

**错误 3：CDT 画得太细**
例子：试图把所有 SKU 都进 CDT。
纠偏：CDT 通常切到 L3（小品类）级即可，L4（具体品牌/规格）由层位规则承接。

**错误 4：忽略空间约束做 CDT**
例子：CDT 切出 8 个分组，但物理货架只有 6 个子货架。
纠偏：当 CDT 切的组数 > 物理子格数时，必须合并相邻组（合并依据是第二层差异最小的两组）。

### 五、案例：花厅坊 G2-B 挂钩面的 CDT 应用（内部，已脱敏结构）

**候选品类（L3 级）：** 清凉功能糖 / 软糖棒糖 / 巧克力 / 果干蜜饯 / 凉果陈皮 / 瓜子花生 / 坚果

**第一层决策切分（按味觉直觉 + 材料属性）：**
```
组1：甜口糖类（糖 + 巧克力）
组2：酸口（果干 + 蜜饯 + 凉果陈皮）
组3：咸脆（瓜子花生 + 坚果）
```

**第二层决策切分：**
- 甜口糖类：糖 vs 巧克力（材料 + 价位差异）
- 酸口：果干 vs 凉果陈皮（全国通货 vs 大湾区地方品牌）
- 咸脆：瓜子花生 vs 高端坚果（数据显示是两个完全不同的客群）

**物理映射到 6 个子格：**
```
第1格：糖（清凉糖+软糖+棒糖合并）
第2格：巧克力
第3格：果干蜜饯
第4格：凉果陈皮
第5格：瓜子花生
第6格：坚果（缩）
```

**视觉扫描路径验证：**
糖 → 巧克力 → 果干 → 凉果 → 瓜子花生 → 坚果
味觉过渡：**甜 → 甜 → 酸甜 → 酸 → 咸脆 → 咸脆**——平滑自然。

**CDT 反向校验的标准动作（V1→V1.1）：**
V1 草图按"清凉糖 / 软糖棒糖"切第一层，违反 CDT 原则。V1.1 修订：合并清凉糖+软糖+棒糖为 1 格"糖类整合"，释放出的 1 格用于"瓜子花生独立"。

### 六、CDT 的局限与替代

| 局限 | 应对 |
|---|---|
| CDT 假设顾客决策是理性的（忽略冲动购、惯性购、连带购） | CDT + 销售数据双轨——CDT 决方向，销售数据决格位强度 |
| CDT 受陈列习惯影响（顾客决策路径被现有陈列"训练"出来的） | 参考"理想决策路径"（行业标杆店、顾客访谈），而非当前实际路径 |
| 小面位（2-3 个子货架）CDT 意义不大 | 小面位直接按层位规则 + 销售数据落位 |

---

### 七、CDT 应用工作流模板

```
1. 列出该面位的所有候选品类（L3 级）
2. 找出顾客的第一层决策维度（材料/价位/场景三选一）
3. 按第一层决策切大格
4. 找出第二层决策维度
5. 在每个大格内按第二层决策细分到子格
6. 验证陈列顺序是否与顾客视觉扫描路径一致
7. 应用层位规则（L1-L6）填充子格内容
8. 用销售数据校验子格强度（参见 A4 错配诊断）
```

可做成 Excel 或在线表单，每个面位填一份。

## 花厅坊直接调用

> 花厅坊 G2-B 挂钩面（零食品类 7 个 L3）的 CDT 案例已在 §五 完整展示（甜→酸→咸 味觉过渡路径）。
> **下次进店快判**：站在零食区正面，30 秒内能否感知到"甜 → 酸 → 咸"的味觉过渡？能感知 = 子格顺序合理；无法感知 = 子格落位需重新按 CDT 六步检查。
> 若 G2-B 实际格位与 CDT 映射不符（如凉果陈皮混在糖类区），即为典型 A4 错配——用 [[KB-CAT-DX-MISMATCH-001_销售陈列错配诊断_v0.1]] 做错配定量诊断。

## 失效条件（张力）

| 失效场景 | 原因 | 替代方案 |
|---|---|---|
| 冲动品 / 便利品（如饮料、口香糖）| 顾客无预先决策路径，进店随机取 | 按视觉冲击力 + 动线位置配置，不画 CDT |
| 面位只有 2–3 个子格 | 切分过细执行成本大于收益 | 直接按层位规则（L1–L6）+ 销售数据落位 |
| 新兴 / 引进中品类 | 顾客决策路径未稳定，CDT 样本量不足 | 先试销 3 个月观察购买行为，再画 CDT |
| 价格带高度单一的专卖陈列 | 无需分层决策，顾客按品牌 / 口味直觉选 | 按品牌陈列 + 层位规则处理 |
