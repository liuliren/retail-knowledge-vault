# 组织力最小可用模块补建任务卡 v0.1（P1）

## 1. 背景

组织力为晟果科学零售三力之一，但当前体系中组织力内容相对薄弱。v0.4 审计已将"组织力盲区"列为开放项。用户已拍板：组织力需要原创补建，但本阶段只补"商品经营所需的最小组织力"，不扩展为完整组织管理体系。

## 2. 为什么要补

商品力、选品、陈列、定价、库存、复盘等方法要落地，必须有组织执行支撑：

- 谁负责？
- 谁确认？
- 谁执行？
- 谁复盘？
- 谁训练？
- 谁对结果负责？

如果缺少组织力，方法论会停留在报告里，无法稳定进入门店动作。

## 3. 本阶段只补什么

本阶段只补 5 个最小组织力模块：

1. 门店商品执行责任机制
2. 店长 / 主管 / 采购 / 理货员 RACI
3. 周度商品复盘会议机制
4. 店长商品力训练路径
5. 多店复制时的总部商品能力雏形

## 4. 暂不补什么

本阶段暂不补：

- 招聘体系
- 薪酬体系
- 完整绩效考核体系
- 企业文化体系
- 完整组织架构设计
- 非商品相关管理体系
- 大型培训学院体系

## 5. 后续输入材料

后续补建该模块时，优先读取：

- 花厅坊组织现状
- 店长 / 主管 / 采购访谈
- 现有绩效经营簇 M7
- 周复盘模板
- 门店执行照片与整改日志
- 多店复制需求
- 30天商品提效包执行反馈
