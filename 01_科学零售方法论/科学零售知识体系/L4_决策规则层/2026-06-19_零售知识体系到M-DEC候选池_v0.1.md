# 零售老刘知识体系 → M-DEC 决策规则候选池 v0.1

## 1. 文件定位
- 本文件是"**外部知识 → 晟果科学零售 → M-DEC 决策规则**"的**中间转化池**;**不是正式 M-DEC、不占编号、不代表规则已验证**。
- 用途:承接零售老刘高质量知识点 → 判断哪些可转决策规则 → 形成 **M-DEC future_candidate** → 标验证条件 → **等真实门店案例重复出现后,再决定是否正式新增 M-DEC**。

## 2. 转化原则
1. 外部知识**不能直接变成 M-DEC**;
2. 只有能形成"**判断条件 → 触发信号 → 必要字段 → 决策动作 → 反误判边界**"的内容,才有资格进池;
3. **单一小类出现 = 案例**,不直接固化为通用 M-DEC;
4. **至少两个小类/两个案例重复**,才考虑正式新增;
5. M-DEC 服务**客户诊断与门店动作**,不做理论收藏;
6. 零售老刘内容作 source_attribution,**对外表达必须转成六哥方法语言 + 花厅坊实跑证据**;
7. 所有候选标 **evidence_status**:`theory_only` / `single_case` / `multi_case` / `ready_to_create`。

## 7. 与花厅坊实跑的关系
- **T03 饼干已验证**:FC04(库存防误判)、FC01线索(需求点防误砍)、FC09(负毛利特价)、FC02部分(尾部厚)、FC03(价格带过密漏档)。
- **T01 薯片初步验证(结构共性)**:FC02(尾部42%产10%)、FC03(中价72%+高价0)。
- **只在 T03 出现,不能上升规则**:FC01(低销必留 T01弱)、FC09(负毛利 T01未见)、FC07。
- **T03/T01 都出现,优先级高**:**FC02、FC03**(价格带过密/尾部过厚 = 共性)。
- **门店现场核查 = 挂项,不阻塞候选池建设**。

**重点结论**:
1. **价格带过密 / 尾部过厚 = T03/T01 共性** → FC02/FC03 优先级最高;
2. **低销必留在 T03 明显、T01 不明显** → **FC01 不急建**;
3. 因此:**FC02/FC03 > FC01**(但 FC03 并入 010、FC02 待临界值)。

## 8. 正式新增 M-DEC 的门槛(满足 ≥4 条,且必含跨案例)
1. **跨两个小类或两个案例重复出现**(硬条件)；
2. 有明确触发条件；
3. 有明确必要字段；
4. 能输出动作建议；
5. 有反误判边界；
6. 能被 Excel/POS 数据支持；
7. 不与现有 M-DEC 重复；
8. 有客户实跑价值。
> 当前**无一 FC 满足"跨案例 + 不重复"两条同时**:FC02/FC03 跨案例但 FC03 与 010 重复、FC02 待临界值;FC01/FC07/FC09 仅单案例。**故本阶段 0 新增。**

## 9. 下一步路线图
**P0**:维护候选池;把 T03/T01 证据挂到 FC01/FC02/FC03;**不新增正式 M-DEC**。
**P1**:T03 现场核查回来 → 更新 FC01/FC05/FC09;T01 用 6/12 数据重跑 → 更新 FC02/FC03。
**P2**:继续吸收 M5/M7/M8 → 第二批 FC;待跨案例重复后再考虑建 M-DEC-013。

## 10. 结论
- 本轮**不是新增 M-DEC**,是**建立 M-DEC 候选池**;
- 零售老刘体系要继续吸收,但须走"**知识点 → 模块 → 候选规则 → 实跑验证 → 正式 M-DEC**"的转换;
- 当前最优先**不是扩数量,而是提高决策规则的 可验证性 / 可落地性 / 反误判能力**;
- **0 新增 M-DEC、0 占用 M-DEC-013**;FC01-FC10 候而不建,分 A/B/C 三层管理。
