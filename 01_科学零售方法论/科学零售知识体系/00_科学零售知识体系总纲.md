# 科学零售知识体系总纲

## 1. 文件定位

本文件是**科学零售知识体系**的入口与组织图。它不重新定义任何方法论,而是把四个东西**编织成一张可检索、可落地的网**:

1. **你的骨架**(象限 I 内核)= [[科学零售总纲]] 的三力 + 经营控制链 + 数据底座
2. **外部养料**(象限 IV)= 零售老刘体系的量化武器(8 个吸收模块)
3. **落地 IP**(实战沉淀)= 花厅坊 M-DEC
4. **操作系统封装**(可执行/可售卖)= **RetailOS** 门店决策操作系统(把骨架+武器封装成店员可照做的 SOP,见 §6.5)

> **铁律**:养料喂骨架,**永不反客为主**。本体系只**引用** `01_科学零售方法论/` 的既有定义,绝不另起炉灶重定义三力/经营控制链(守 [[CLAUDE.md]] §14 一致性:同一概念一个主定义)。

## 2. 四层架构

```
┌─────────────────────────────────────────────┐
│  L1 痛点统领层  —— 为什么做(全局透镜)          │  → [[痛点框架]]
│     顾客痛点 / 经营痛点 = 一切方法论的归因起点    │
├─────────────────────────────────────────────┤
│  L2 骨架理论层  —— 用什么想(内核 / 只引用)      │  → 01_科学零售方法论/
│     三力(商品力·组织力·运营力)                  │
│     经营控制链(金额→数量→品质→复盘)+ 数据底座   │
├─────────────────────────────────────────────┤
│  L3 方法武器层  —— 怎么算准(养料 / 8 模块)      │  → L3_方法武器层/
│     零售老刘 8 个量化吸收模块                     │
├─────────────────────────────────────────────┤
│  L4 落地IP层    —— 怎么变现(实战 / 可复制)      │  → 09_/16_ 花厅坊 M-DEC
│     通论 → 花厅坊临界值 → M-DEC → 跨战役复制      │
└─────────────────────────────────────────────┘
```

**层间关系**:L1 痛点定方向 → L2 骨架给框架 → L3 武器算准数 → L4 实战出 IP → 反哺 L2(升级方法论)。

> **封装贯穿**:**RetailOS**(§6.5)纵向贯穿 L2-L4——把 L2 骨架 + L3 武器封装成**店员可照做的门店决策 SOP**,是体系的"可执行/可售卖"出口。它**不是新层**,而是把现有层"压成动作句"的操作系统外壳。

## 6.5 RetailOS 门店决策操作系统(封装层·2026-06 升级编入)

### A. 架构(A-G 六层 + 减法/加法双向)
```
v1.0 减法层(汰换): A库存→B SKU结构→C价格带→D陈列→E ABC裁决→F误判→G门店SOP
v1.1 三项补深: E毛利交叉ABC / A库龄+售罄+水位 / B商品角色
```

## 6.6 客户管理成熟度务实降级(横切原则·2026-06 升级编入)

> 贯穿 L2-L4 的**横切约束**:生活性零售客户(社区/便利/生鲜超市)**管理成熟度普遍不高**——过程字段(如采购周期/MOQ)常空或不准。

**原则**:方法论与工具**务实降级**——优先用**结果性、稳健、现成**的数据(销售/库存/进货/停购/ABC),**少依赖需要客户高管理水平才填得准的过程字段**。
- 实例:RetailOS「供得上」闸**不用采购周期**,改用**在购+近期进货+采购规模+是否本地**四代理判可得性;
- 适用:任何 M 模块在落地时,遇到客户数据缺/不准,优先找结果性代理,而非要求客户先补管理。
- 详见 [[客户管理水平约束-花厅坊]]。

## 6.7 顾客 / 营销横切能力口径(2026-06 决策编入)

> 决策(LifeRetail-v0.4-Decision-001 / v0.5-System-001):**不设"顾客力 / 营销力"第四力**,维持"三力 + 数据与系统能力底座"。顾客与营销作为**横切能力**,嵌入商品力、运营力、组织力,不另起支柱。

| 横切能力 | 嵌入商品力 | 嵌入运营力 | 嵌入组织力 |
|---|---|---|---|
| 顾客 | 客群选品、需求单元、小商圈品、M5 | 顾客动线、门店体验、购买路径 | 店员服务、顾客反馈、门店执行责任 |
| 营销 | 促销选品、价格带、定价机制、M3 | 促销陈列、活动执行、M4 | 促销执行责任、门店人员协同 |

补充口径(精确归属,**不简单归为 M3/M5**):
- **定价**主要归入 **M3**(价格带毛利);
- **促销**同时涉及 **M3** 价格机制、**M4** 陈列执行、**M5** 顾客需求;
- **会员**暂作为顾客识别与运营触达工具,横切商品力、运营力和组织力,不单独设支柱。

> 理由:生活性零售的顾客与营销不独立成业务线,而是渗透在"卖什么 / 怎么摆 / 谁执行"里。设独立支柱会割裂三力,也违背"养料喂骨架、只引用不另起炉灶"的铁律。
