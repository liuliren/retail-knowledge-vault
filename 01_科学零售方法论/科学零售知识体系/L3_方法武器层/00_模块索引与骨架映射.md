# L3 方法武器层 — 8模块索引与骨架映射

## 1. 定位

L3 是"算得准"层。零售老刘 322 篇 Clippings = 量化弹药库,提炼成 **8 个武器簇**,每簇精确喂给骨架(L2)的特定部件。

> **主定义声明(2026-06 决策编入)**:本表是 M1-M8 模块**编号、边界、定义的唯一主定义**。其它文件中的模块描述,如 P0-1 Clippings 能力地图、总纲摘要、项目索引等,均为**引用 / 导航视图**。凡出现冲突,**以本表为准**。新增或调整模块,必须先更新本表,再同步其它引用位置。

**源头分工**:
- **本层(模块卡)**= 骨架映射 + 武器清单 + 调用入口(象限 I,薄)
- **`14_外部案例与行业研究/零售老刘体系/`**(深度吸收笔记)= 养料原文消化(象限 IV,厚)
- **`Clippings/`**(322 篇原文)= 一手来源

## 4. 重建 SOP(每个模块怎么吸收)

1. 从 `Clippings/` 挑出该簇相关原文(本卡 §Clippings源 已列)
2. 逐篇精读 → 提炼:核心思想 / 公式卡 / 反常识点
3. 映射骨架:每个武器喂哪个部件
4. 花厅坊落地 [推断]:能治哪个痛、怎么算
5. 落 1 份吸收笔记进 `14_外部案例/零售老刘体系/`
6. 回本层模块卡填实 + 总纲状态更新

> **吸收纪律**:标来源(归属零售老刘)/ 区分通论 vs 花厅坊临界值 / 不反客为主。
