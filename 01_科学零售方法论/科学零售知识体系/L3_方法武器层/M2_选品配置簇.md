# M2 选品配置簇

> ✅ **已吸收 v0.1**。深度笔记见 → [[2026-06-19_零售老刘体系吸收_02_选品配置簇_v0.1]](`14_外部案例/零售老刘体系/`)

## 1. 主治痛点
**滞销**(经营)/ **买不对**(顾客)。反常识:按销量末位淘汰 = 极端错误,要引时间维度 + 双ABC。

## 2. 武器清单(速查)

| 武器 | 一句话 |
|---|---|
| **商品 ABC** | 单/双/3ABC;三步法(降序→累加%→求值);**17%A产60%**;小类ABC更实际 |
| **双ABC 汰换** ⭐ | 去年A今年C/消失 = 衰退汰换候选 |
| **PSD 单店单品效率** | Per Store Per Day = 单店单品日均销售;因品施策 |
| **宽类窄品** | 类要宽品要窄;"经营问题最后都体现在SKU数上";SKU精简 |
| ABC 修订订货 | AA类 ×1.8 系数(接 M1)|

## 3. 骨架映射
- 商品ABC/双ABC/层级ABC → **商品力·单品管理 / ABCD**(升级多维)
- PSD → **商品力·单品管理**(效率标尺)
- 宽类窄品/合理SKU → **品类管理** + **单品管理·SKU精简**

## 4. 花厅坊落地 + M-DEC 候选

- 新品引进 → 宽类窄品找类缺位 + 小类ABC 定空白

## 6. 二遍补强(2026-06-19 / B 簇 5 篇精读:03/零模02/25/26/27)
> 增量,不推翻 §2。深度 → [[2026-06-19_零售老刘体系二遍吸收_B_ABC售罄率与老品淘汰簇_v0.1]]

| 补丁 | 口径 |
|---|---|
| ABC=初筛非终判 | 用**小类内综合值ABC**(销售/销量/毛利加权);ABC 含假A类+无趋势,不能直接淘汰 |
| 双ABC=衰退识别 | 去年A今年C/消失,**4分型**(断货/替换/生命周期/系统假象)→ 预警非淘汰 |
| 售罄率=时间校验 | 售罄率②=销量÷订货量;平衡点**46.7%**=毛利率/(1−费用率);<46.7%且非缺货=真亏损 |
| 老品淘汰**四重防误杀** | 库存(缺货M1)+订货(售罄率)+陈列(空间系数M4)+价格带(M3)→ 才淘汰 |
| 淘汰**六桶** | 真滞销/缺货/陈列差/价格带错位/生命周期衰退/系统假象;仅真滞销进M-DEC-007 |
| M2 联动 | 初筛(M2 ABC)→查缺货(M1)→查错位(M4)→查错配(M3)→真滞销才淘汰 |

> 对 M-DEC-009 补双ABC分型/售罄率×ABC/六桶(**仅建议,不改临界值**);POS 建议增补 去年销额量毛利3 + replacement_sku_flag + shelf_position_change_flag + stockout_days。ABCD 定义仍以动销表§3为准,老刘"15-17%产60%"仅作健康参考**不替换**。

## 7. 二遍补强(2026-06-19 / C1 选品配置31-38 共8篇精读:SKU数约束)
> 增量,不推翻 §2/§6。深度 → [[2026-06-19_零售老刘体系二遍吸收_C1_SKU数约束专题_v0.1]]

| 补丁 | 口径 |
|---|---|
| SKU数=经营约束非陈列喜好 | "一切零售问题就是SKU问题";合理SKU数=多约束动态最优解,非拍脑袋固定值 |
| 7 大约束 | 需求/物理空间/资金效率/边际效益/供应链/综合分析/动态优化(花厅坊优先前4)|
| 合理SKU语义 | 匹配度×单品宽度×单品深度;品类SKU=总目标×(销售占比×深度系数/Σ权重)|
| 关键公式 | 物理上限=(陈列米×层×系数)/单品面宽;品效=总销售/SKU数;边际品效<平均品效→精简;GMROI=毛利率×周转率 |
| 瘦身承接B簇 | 先判小类**超配**(当前SKU vs 合理SKU)→ 超配才砍,按ABC/售罄率排序,过六桶防误杀 |
| SKU减≠商品力升 | 错误瘦身→顾客任务缺口/价格带漏档;须与价格带(M3)/陈列容量(M4)联动 |

> 花厅坊落地:**休食SKU数约束判断链v0.1**(10步,见C1笔记§12)。字段多为品类表C新增计算列,**几乎不增POS原料**。不出"全品类SKU数标准答案"。

## 8. 二遍补强(2026-06-19 / C2a 选品配置39-44 共6篇精读:宽类窄品核心)
> 增量,不推翻 §2/§6/§7。深度 → [[2026-06-19_零售老刘体系二遍吸收_C2a_宽类窄品核心专题_v0.1]]

| 补丁 | 口径 |
|---|---|
| 宽类窄品≠简单砍SKU | 是**结构重分配**;不是只砍尾货、不是品类越多越好 |
| 三级语义 | 宽类(Category需求骨架)→**Item(中间类/顾客心智的类)**→SKU;无Item层做不成 |
| 宽类=任务覆盖,窄品=单任务去冗余 | 类在前品在后;一个类最多2价格带;同类相斥/类间相吸 |
| 承接C1+B簇 | C1判超配/欠配→C2a判扩类/窄品;C/D只是窄品候选,过六桶防误杀,低频刚需必留 |
| 与M3/M4联动 | 漏档不窄关键段/过密先窄重复段/黄金位错配先查陈列/目标性引流不过度窄 |
| 替换非净减 | 好品替差品;窄品狠心、宽类温柔 |

> 花厅坊落地:**休食宽类窄品判断链v0.1**(10步,见C2a笔记§14)。**前置缺口**:花厅坊缺 Item/中间类(需求点)层(仅有L3小类)→ 待立项。不改POS清单/品类表C,不出"宽类窄品标准答案"。

## 9. 二遍补强(2026-06-19 / C2b-1 选品配置45-54 共9篇精读:类为先+可得性)
> 增量,不推翻 §2/§6/§7/§8。深度 → [[2026-06-19_零售老刘体系二遍吸收_C2b-1_类为先与可得性专题_v0.1]]

| 补丁 | 口径 |
|---|---|
| 类为先 | 先判需求点(Item)是否需存在,再判SKU;顾客买的是类不是品(希克定律) |
| Item/中间类=关键层 | 连接品类结构与SKU(顾客心智的类);花厅坊缺此层=宽类窄品拦路石 |
| 窄品保任务可得性 | 窄品≠让顾客买不到;每关键任务≥1代表SKU;可得性不足时停止瘦身 |
| 低频刚需防误杀 | must_keep_task_flag(老人/儿童/特殊口味/节庆/应急/本地习惯);ABC的C/D仅候选 |
| 属性继承可得性 | 宽类定骨架/窄品填血肉;非物理属性(感官)常比保质期更早不可售 |
| 平衡 | 宽类窄品=需求点覆盖(宽)与SKU去冗余(窄)的平衡 |

> 花厅坊落地:**休食 Item/需求点层建表草案v0.1**(21字段,见C2b-1笔记§13)。**仅草案待立项**;Item层每店不同、无通用标准答案。不改POS清单/品类表C、不建表。
