# M5 顾客需求簇

> ✅ **已吸收 v0.1**。深度笔记 → [[2026-06-19_零售老刘体系吸收_05_顾客需求簇_v0.1]]

## 1. 主治痛点
**客流降 / 需求错判**(经营)/ 买不到·买不对(顾客)。

## 2. 武器清单(速查)
| 武器 | 一句话 |
|---|---|
| **顾客需求指数** ⭐ | 销售下沉到**小类×每天**(一小类=一需求);大类粒度不够 |
| **需求变化指数** | 追踪小类需求随时间升/降/稳(联动M6增长率)|
| **小票客群法** ⭐(E2回填) | 一张购物小票=一次顾客购物行为→**不靠会员卡也能做客群/购物篮分析**;社区超市(会员弱)首选抓手 |
| **痛点洞察** ⭐(逐字批12回填) | **痛点=需求×情感²**(情感平方级);**找痛点4法=市调/场景/情绪/顾客抱怨**(差评=改进良药);痛点四层(大众/分众/小众/微众)。痛点=需求的具体化,接卖点3级维度(功能/情感/社会) |
| **顾客五类型**(逐字批14回填) | 忠诚购买者/竞品忠诚者/**游离者(摇摆·已成多数→促销重点)**/价格敏感者/非购买者。不同类型对应不同手段(忠诚→提频/连带;游离→促销策反) |
| 类别RFM / 以顾客为中心 | 小类需求强度分层;一切从顾客需求出发(**RFM需会员数据·花厅坊到位后进阶**) |

## 6. 数据底座补强（2026-06-30 · 源自 [[数据与系统能力底座]] v0.5.4/v0.5.5/v0.5.8）

### 6A 卖场三指标 = 超市经营诊断核心指标集（§4E）

| 指标 | 定义 | 诊断价值 |
|---|---|---|
| **⭐ 转化率** | 来店客流 ÷ 有效购物客流（目标 ~90%） | 超市最关键指标·所有经营动作反映于此·含商圈三率（入店率/市占率/购买率）+ 货位转化率（坪效/陈列/区位） |
| **驻停时间** | 顾客在卖场停留时长 | 首要→动线/环境布局；次要→商品气氛/音乐；等待时间越短越好 |
| **价值指数** | 见 [[品类管理]] §3.1D（SSOT·此处不重复承载） | 衡量客群质量·与小票五度量互为表里（一看质·一给量） |

> 建模三方向：历史同比 / 即时实时 / 洞悉未来。**转化率 = report-export 卖场健康卡首要诊断切口**。
> ⚠️ **花厅坊限制**：转化率分母（来店客流）需客流统计摄像头，花厅坊当前无此设施 → 暂用客单价+品单数替代诊断，转化率作参照指标待客流系统到位后启用。

---

### 6B 小票客群五度量（§4F · DAX 口径 · 无会员也能做）

> **前提**：一张小票 = 一个顾客一次购物行为。社区超市会员弱 → 小票比会员更可靠、全量。

| 度量 | DAX 表达式 | 含义 |
|---|---|---|
| **客流数** | `DISTINCTCOUNT(小票号)` | 基础度量·Power BI 易做 Excel 难做即时 |
| **客单价** | `销额 ÷ 客流` | 每客贡献额 |
| **客单量** | `销量 ÷ 客流` | 每客购买件数（含称重·≠品单数） |
| **品单数** | `销售表行数 ÷ 客流` | **连带率** = 每客购买品类宽度 |
| **品单价** | `客单价 ÷ 品单数` | 每个品类的平均价格层 |

> **花厅坊落地**：M5小票客群法 → Data Adapter 客群度量底座 → 进 report-export 客群健康卡。品单数（连带率）呼应购物篮 [[品类管理]] §3.1E。

---

### 6C 购买率（成交率）= 六大购物指标总绩效（§4E 补充）

六大顾客购物指标：**客流量 / 进店率 / 成交率 / 客单价 / 连带率 / 复购率**

- **购买率 = 有购物小票顾客数 ÷ 进入卖场总顾客数** = 综合绩效指标
- **盲点**：进店量（分母）难获取 → 解法：客流统计摄像头对接 POS
- 进店后约 87% 是否关注商品 → 取决于环境/陈列科学性/显眼度

### 6D 百货课诊断范本六步链（§4H · 可直接套花厅坊单品类诊断）

```
① 坪效（如 22元/㎡ 偏低）
② 动销率：在销动销率 = 有销SKU÷总SKU · 在库动销率 = 有销÷有库存（60% 正常线）
③ A类识别
④ 货架数测算：应配组数 = 有销SKU ÷ 每组25单品 → 实际超配 = 减组
⑤ 销售/单品/陈列 三占比对比
⑥ 去年A类畅销丢失 + 对"有销无库"A类即时补货
```

> 定位：`/diagnose` skill 的方法骨架 · 可进 report-export 单品类诊断卡 · **SSOT 在 [[数据与系统能力底座]] §4H，此处为调用入口**。
