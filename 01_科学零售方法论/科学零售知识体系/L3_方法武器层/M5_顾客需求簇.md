# M5 顾客需求簇

> ✅ **已吸收 v0.1**。深度笔记 → [[2026-06-19_零售老刘体系吸收_05_顾客需求簇_v0.1]]

## 1. 主治痛点
**客流降 / 需求错判**(经营)/ 买不到·买不对(顾客)。

## 2. 武器清单(速查)
| 武器 | 一句话 |
|---|---|
| **顾客需求指数** ⭐ | 销售下沉到**小类×每天**(一小类=一需求);大类粒度不够 |
| **需求变化指数** | 追踪小类需求随时间升/降/稳(联动M6增长率)|
| **小票客群法** ⭐(E2回填) | 一张购物小票=一次顾客购物行为→**不靠会员卡也能做客群/购物篮分析**;社区超市(会员弱)首选抓手 |
| **痛点洞察** ⭐(逐字批12回填) | **痛点=需求×情感²**(情感平方级);**找痛点4法=市调/场景/情绪/顾客抱怨**(差评=改进良药);痛点四层(大众/分众/小众/微众)。痛点=需求的具体化,接卖点3级维度(功能/情感/社会) |
| 类别RFM / 以顾客为中心 | 小类需求强度分层;一切从顾客需求出发(**RFM需会员数据·花厅坊到位后进阶**) |

## 3. 骨架映射
- 顾客需求指数 → **商品力·客群** + **品类管理·需求**
- 需求变化/RFM → 商品力 + 数据底座
- 以顾客为中心 → 反哺总纲顾客视角

## 4. 花厅坊落地 + M-DEC 候选
- **小票客群法(E2回填·六哥 2026-06-25 签)**:花厅坊会员管理弱→**用 POS 小票做客群/购物篮分析**(单张=一次购物),RFM 留作会员到位后进阶。来源零售老刘,见 [[2026-06-25_零售老刘体系吸收_E2_顾客需求侧簇_v0.1]]。
- 需求洞察 → 休食销售下沉小类×日,看真实需求强弱/节奏
- **M-DEC 候选**:休食小类需求指数「连续下降N期」→ 需求衰退预警(联动汰换)
