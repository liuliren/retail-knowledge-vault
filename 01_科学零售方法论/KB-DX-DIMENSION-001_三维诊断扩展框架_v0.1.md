# 三维诊断扩展框架（品类族 × 周期 × 节令）· v0.1（draft·待签字）

## 核心论点（结论先行）

> **7D 全店诊断给的是一张「快照」，本框架把它展开成一条「连续诊断流」。** KB-DX-FULLSTORE-001 回答「此刻这家店作为一门生意健不健康」；但一家社区生鲜超市不是均质体——**生鲜每天在变、快消每周在变、百货每月才变**，且**同一指标在节令窗口内会系统性偏移**。用一套频率、一套阈值、一套工具去诊断所有品类所有时点，必然「生鲜看得太慢、百货查得太勤、节令误报成异常」。
>
> 本框架沿**三条正交轴**展开 7D：
> - **维度 1 · 品类族**（分辨率轴）：按变化速度把全品类分三族，每族配**不同的核心指标 / 主诊断周期 / 专属工具**。
> - **维度 2 · 周期**（节奏轴）：日 / 周 / 月 / 季 / 年各有**不同的关注问题、工具、产出决策、责任人**。
> - **维度 3 · 节令**（上下文轴）：节令窗口内对**状态机阈值做校正**，使节令带来的正常波动**不误触危险状态**，并前移诊断重心。
>
> **最大张力（本框架的第一性约束）**：三轴全铺开 = 3 族 × 5 周期 × N 节令 = 指数级诊断动作，**直接违反「少而深」**。故本框架的价值不在「把矩阵填满」，而在**§4 收敛铁律**：每族只跑**一条主周期线**，其余周期与节令仅在**触发条件**下加诊。**不设收敛规则的三维展开是伪科学——它制造了工作量，没制造判断力。**

## 0. 定位与边界（必读）

| 项 | 说明 |
|---|---|
| **本框架是什么** | KB-DX-FULLSTORE-001（7D）的**正交展开层**。7D 定义「诊断哪 7 个维度」，本框架定义「这 7 个维度在**不同品类 / 不同周期 / 不同节令**下如何差异化执行」。 |
| **本框架不是什么** | **不新增诊断维度**，不改 7D 权重，不替代任何单维度工具。它是**调度层 / 分辨率层**，不是评分层。 |
| **何时调用** | 从「一次性全店体检」升级为「常态化经营诊断节奏」时；给客户设计**诊断日历 / 巡店 SOP** 时；触发词「多久看一次 / 这个品类要不要天天盯 / 节令要不要调」。 |
| **何时不用** | 单次陌生店快判（走 7D §11 方案 A）；只测一个货架（单品类诊断卡 `/diagnose`）。 |
| **与品类架构的关系** | 品类族划分（维度 1）**继承** [[KB-CAT-ARCH-001_全品类双投影架构_v1.2]] 的 L2/L3 分类，不另造分类轴；只是按「变化速度」把 L2 重新聚合成三族。 |
| **与节令矩阵的关系** | 维度 3 的节令数据源是 [[KB-GBA-TPOS-001_大湾区TPOS本地化_v0.1]] 六节令表；**节令乘子与提前准备天数**是本框架新增的量化层（inferred·待标定），未来收口进拟建的 KB-SEASON-MATRIX-001 与 [[52周MD]]。 |
| **与状态机的关系** | 节令校正跨「调度层（本框架）」与「计算层（KB-CAT-STATE-001）」，两层分工明确——本框架负责人工判断输入，CAT-STATE-001 的 V4/V6 负责自动计算修正。详见 §3 三套机制说明框。 |
| **证据分级** | 沿用 7D §7 的 A-E 五级；本框架多数参数为 **C-E 级（inferred）**，须多店 / 多年实测后方可升 evidence-based。 |

## 1. 维度 1 · 分品类：三族差异化诊断

> **划分第一性原理**：品类的**变化速度**决定诊断频率——变化越快，越要高频盯（否则问题暴露太晚、损失不可逆）；变化越慢，越不该高频盯（浪费注意力、噪声当信号）。故按「日变 / 周变 / 月变」把全品类聚合成三族，**每族的核心指标、主诊断周期、专属工具随之不同**。

### 1.1 三族总览

| 族 | 覆盖品类（L2 聚合） | 变化速度 | 核心指标 | 主诊断周期 | 专属工具 |
|---|---|---|---|---|---|
| **生鲜族** | 蔬菜 / 水果 / 肉禽 / 水产 / 蛋品豆制品 / 熟食烘焙 | **日变**（保质期以小时-天计） | 损耗率 · 日销速 · 鲜度（保质期余量）· 天气敏感度 | **日维度优先** | 损耗控制诊断 · 时段销售诊断 |
| **快消食品族** | 饮料 / 零食 / 方便食品 / 调味品 / 冲饮 / 粮油 | **周变**（周转以周计） | 周转天数 · 促销响应率 · 陈列效率 · SKU 活跃率 | **周维度** | 销售陈列错配诊断 · 价格带分析 |
| **百货非食族** | 日化 / 清洁 / 纸品 / 百货 / 烟 / 散装 | **月变**（周转以月计） | 月周转 · 坪效 · 毛利率 · 渗透率 | **月维度** | 品类贡献度分析 · 毛利结构分析 |

> *【张力·必读】此三族划分在什么条件下失真？① **熟食烘焙**放生鲜族（日清逻辑），但其**毛利结构**更接近快消（须月度看毛利），属跨族品类——诊断时损耗看日、毛利看月，**双周期挂靠**。② **散装**（干货 / 坚果 / 凉果）放百货非食族按月周转，但大湾区**凉果陈皮**是差异化锚点、且有**节令性**（见维度 3），高峰期须临时升为周诊。③ **烟**周转快但价格锁死、无促销弹性，放百货族仅因监管与毛利属性，其诊断退化为「有无断货 + 证照合规」两项，不套用百货全套指标。→ **跨族品类以「主属性定族、次属性挂靠周期」处理，不强行归一。***

### 1.2 生鲜族 · 诊断细则

**核心指标（日维度优先）**：

| 指标 | 判读门槛 | 数据来源 | 承接 7D |
|---|---|---|---|
| 损耗率 by 子品类 | 叶菜 ≤8% / 水果 ≤5% / 肉类 ≤5%（KB-FRESH-001）；损耗每降 1% → 综合毛利 ≈+0.3% | 损耗台账 / 盘点差 | D2-4 |
| 日销速（动销速度） | 当日进货当日清占比；肉类日清率 = 当日清零无隔夜肉 | POS + 现场 | D2-2 |
| 鲜度（保质期余量） | 叶菜看水灵度 / 短保品（豆制品）日清率 ≥85% | 现场 + 短保台账 | D2-2/D2-3 |
| 天气敏感度 | 高温日叶菜损耗↑、汤料↓；雨天客流↓生鲜备货须下调 | 天气 + POS 对比 | D2-4 / D4-3 |

- **专属工具**：**损耗控制诊断**（挂 [[KB-FRESH-001_生鲜护城河经营方法论_v0.1]] 损耗管理 + 修复顺序：日清机制→品种结构→鲜度陈列）；**时段销售诊断**（挂 [[KB-CAT-TIMEEFF-001_时间效率主轴评价指标体系_v0.1]]，生鲜属目的性区域→销售毛利主轴 + 时段错配检测，下午 3-5 点肉类区烟火气快判）。
- **大湾区特殊性（横切进入，非可选）**：① **潮汕口味**——卤味 / 潮汕牛肉丸 / 鱼露腌制，熟食烘焙族须含潮式;② **时令蔬菜**——通菜 / 苋菜 / 马蹄 / 芥兰 / 夏季苦瓜冬瓜,时令主推位是「本地新鲜货」信号;③ **生猛海鲜偏好**——活鲜水产日均销量达标率 ≥80%，清远鸡必备（缺 = 严重失分，D2-P4）。这些是生鲜族诊断的**必查项**，不是加分项。

> *【直接调用性检验】生鲜族诊断读完能立刻生成「花厅坊→下一步」：花厅坊 f≈70.74%（护城河稳固）但 ERP 进价不准 → **损耗率用盘点差口径实测、不用 ERP 推算**；日诊先站下午 3-5 点肉类区听剁肉声（30 秒快判），无烟火气即 D2-P2 隔夜肉风险，当日整改。*

### 1.3 快消食品族 · 诊断细则

**核心指标（周维度）**：

| 指标 | 判读门槛 | 数据来源 | 承接 7D |
|---|---|---|---|
| 周转天数 | 社区店快消基准 15-30 天；> 结算周期即现金流风险 | POS + 库存 | D5-2 |
| 促销响应率 | 促销期销量 ÷ 常态销量；引流品（鸡蛋 / 可乐）弹性高 | POS 对比 | D4 / D5 |
| 陈列效率 | 销售-陈列比（>1.5 挤位 / <0.5 占位）；黄金层是否放 A 类 | 走店 + POS | D3-1/D3-2 |
| SKU 活跃率 | 30 天动销 SKU ÷ 在档 SKU（= 1 − 空销率） | POS | D1-5 |

- **专属工具**：**销售陈列错配诊断**（挂 [[KB-CAT-DX-MISMATCH-001_销售陈列错配诊断_v0.1]]）；**价格带分析**（三档大众 / 主流 / 高端是否齐全、有无中价过密，挂 [[KB-SKU-ROLE-001_SKU四维角色体系_v0.1]] 价格角色）。
- **主诊断动作**：每周复盘品类销售趋势 + 空销率 + 价格执行（是否照价签卖）。
- **大湾区专项（GBA 必查·V1 验证修订补入）**：
  - ① **凉茶饮料**——王老吉 / 加多宝 / 本地凉茶品牌（如陈皮王 / 健力宝凉茶系列）是大湾区快消区核心差异化品类；周诊须**独立追踪**其 SKU 活跃率和陈列效率，不得与全国统一饮料口径混算；夏季节令期（6-9 月）按乘子 ×1.3-2 扩陈列面（见 §3.2）。凉茶品类缺位或陈列效率低是大湾区社区店快消区的高失分项（D3-P2）。
  - ② **本地 vs 全国品牌价格带结构**——大湾区消费者对本地零食品牌（如徐福记广东区定位、盐津铺子 GBA 渗透率）与全国大品牌有地域偏好差异；周诊的价格带分析须识别「本地品牌 vs 全国品牌」结构比例，不得只用全国统一的大众 / 主流 / 高端三档口径，否则漏诊本地品类的真实竞争力。

> *【张力】快消族「周诊」的失效边界：**促销周**与**节令周**内，周转 / 促销响应率会剧烈偏移，常态阈值不可用。须切换到节令校正口径。「切换」不是模糊指令，须落实为**具体替代阈值**（以端午周为例，inferred·待标定）：*
> - *周转天数告警线：从常态 15 天下调至 7 天（周转加速属节令快销，不触发 P0 缺货误报）*
> - *SKU 活跃率：节令品类活跃率上升视为正常，不套常态基准；节令结束后活跃率回落不触发 SKU 清退*
> - *促销响应率：节令期不做促销拉动判断（购买是需求驱动，非促销驱动）*
> - *节令结束，上述替代阈值须显式回归常态（以回归日为准，见 §3.2）；未设回归日 = 切换无效。节后回落不得被误判为「品类衰退」。*

### 1.4 百货非食族 · 诊断细则

**核心指标（月维度）**：

| 指标 | 判读门槛 | 数据来源 | 承接 7D |
|---|---|---|---|
| 月周转 | 日化 / 纸品基准 30-60 天；超期 = 库存挤压 | POS + 库存 | D5-2/D5-4 |
| 坪效 | 月销售额 ÷ 品类占用面积；低坪效 = 空间错配 | POS + 平面图 | D3 / D5-4 |
| 毛利率 | 百货是毛利补充盘，综合毛利须托住全店 20-25% | POS | D5-1 |
| 渗透率 | 买该品类的客单数 ÷ 总客单数；反映品类必要性 | POS 篮分析 | D1 / D4 |

- **专属工具**：**品类贡献度分析**（该品类销售 / 毛利占全店比，判角色）；**毛利结构分析**（挂 [[KB-DX-FULLSTORE-001]] D5 + RM-04：提高高毛利品类占比优于直接提毛利率）。
- **主诊断动作**：月末评品类结构 + 毛利贡献 + 慢周转清理，**不做日诊 / 周诊**（变化慢，高频查是噪声）。
- **大湾区高价值诊断项（正文必查·V1 验证修订升入）**：**新会陈皮**是大湾区百货区最重要的差异化高毛利品类，月诊中须单独分析：
  - 按价格层（普通陈皮 / 5 年陈 / 10 年陈）分析毛利贡献和销售结构（不得与其他百货混算均值）
  - 渗透率：购买陈皮的客单数 ÷ 总客单数（反映本地顾客文化黏性，是大湾区社区店区别于内地的核心差异化指标）
  - 节令性：冬季（秋冬煲汤季）保健需求上升，是差异化引流锚点；窗口期须临时升为周诊（见维度3 §3.2）
  - 此项是大湾区百货族区别于内地的高价值判断项，须进入月诊正文，不得仅在张力注中被动提及。

> *【张力】百货「月诊」的例外：**烟**须周查断货（监管 + 现金占用）；**散装凉果**在节令窗口（冬至 / 春节送礼）须临时升周诊；**新会陈皮**在秋冬煲汤季须临时升周诊。即「主属性定月诊、节令 / 监管属性触发加诊」。*

## 2. 维度 2 · 分周期：五周期诊断矩阵

> **每个时间周期，诊断的「关注问题、工具、产出决策、责任人」都不同。** 高频周期看**运营异常**（快变量、止血），低频周期看**结构战略**（慢变量、布局）。混淆会导致「日会上讨论年度定价」或「年度复盘才发现天天在漏损耗」。

| 周期 | 关注什么 | 使用什么工具 | 产出什么决策 | 谁负责 |
|---|---|---|---|---|
| **日**（每日必看） | 生鲜损耗 · 当日客流 · 当日销售达成 | 损耗控制诊断 · 时段销售诊断（生鲜族为主） | 今日打折 / 补货 / 下架 / 加人（即时止血） | 店长 / 生鲜主管 |
| **周**（每周复盘） | 品类销售趋势 · 库存状态 · 空销率 · 价格执行 | 销售陈列错配诊断 · 价格带分析 · 五态宽度诊断（快消族为主）· **V5 双口径**（见下注） | 本周调价 / 补空缺 / 清空销 / 端架换品 | 店长 / 采购 |
| **月**（月末诊断） | 品类结构评估 · 毛利结构 · 新品试销结果 · 库存清理 | 品类贡献度分析 · 毛利结构分析 · 7D 全店诊断（百货族为主） | 汰换 SKU / 调品类面积 / 新品转常规或退场 | 店长 / 顾问 |
| **季度**（季度审视） | 品类角色重评 · 选品结构调整 · 供应商评估准备 | 五分法品类角色 · 双投影架构复核 · 供应商 KPI 台账 | 品类角色升降 / 换供应商 / 结构性增删品类 | 顾问 / 老板 |
| **年度**（年度战略） | 全品类宽度重设 · 定价策略 · 店型升级评估 | 品类宽度诊断（重设建议区间）· 定价体系复盘 · 竞品 / 商圈重扫 | 全店品类宽度基线 / 定价打法 / 是否升级店型 | 老板 / 顾问（**六哥签字**） |

> **【V5 双口径说明·V2 验证修订】** 快消族**周线**空销率诊断须同时看两个 V5 口径，不可混用：
> - **快信号口径**（POS 直算）：30 天 0 动销 SKU ÷ 在档 SKU（来自 KB-DX-FULLSTORE-001 D1-5，瞬时快照，含所有近期零销品）
> - **慢确认口径**（状态机读取）：KB-SKU-STATE-001 的 ⚪退市候选 ÷ 总在档 SKU（滞后指标，要求连续 N 周期🔴且通过横比闸）
> - **两个口径设计上不会收敛**（差值 = 已达🔴但未满 N 期阈值的 SKU 集合，属正常时间滞后），**不得用「不收敛 = 数据陈旧」诊断**。当快信号口径 > 慢确认口径 超过 5% 时，说明有大量 SKU 即将进⚪，须手动触发状态机更新，而非等待超时自动降级。

**周期递进铁律（金字塔）**：
- **下层为上层提供证据**：日损耗数据 → 汇成周趋势 → 汇成月结构 → 汇成季角色 → 汇成年战略。**跳层无据**：没有日 / 周数据积累，季度「品类角色重评」就是拍脑袋。
- **上层为下层设定约束**：年度定的品类宽度基线 → 约束季度增删；季度定的品类角色 → 约束月度汰换方向。**下层不得越权改上层**（店长不能在周会上改年度定价策略）。
- **90/10 落点**：日 / 周 / 月为**执行相**（Harness，可 AI + 店长跑）；季 / 年为**判断相**（驭马人，含角色升降、定价、店型——**不得下放，六哥签字**）。

> *【张力·必读】此矩阵在什么条件下失真？① **新店 / 大改后前 90 天**——数据未稳，须临时把「月诊」提到「双周诊」，否则错误结构固化太久。② **危机态**（生鲜占比跌破 30% / 现金流红线）——**冻结常规周期表，转入 7D §5 的 TOP3 止血编排**，一切慢变量诊断让路。③ 责任人栏假设门店有「店长 + 采购 + 生鲜主管」分工；**纯夫妻店**下全部落到老板一人，须把日 / 周 / 月诊压缩成「一张每日 5 分钟清单 + 每月一次顾问远程」，否则无人执行。*

> *【夫妻店压缩版清单·V1 验证修订实质化】可直接使用，无需签字（A 档可逆动作）：*
> *日诊 3 项（每日 5 分钟）：① 生鲜损耗目测——叶菜水灵/蔫？今日扔了几斤？② 今日客流 + 日销达成（估算可）③ 今日异常——缺货 / 投诉 / 过期品（没有就写「无」）*
> *周诊 2 项（每周五 10 分钟）：① 本周空销 SKU——哪些商品一周没动（商品名 + 大约几天没卖）② 本周客流趋势——比上周高还是低（估算%或感受即可）*
> *汇总方式：每周五以微信语音 / 文字发给顾问，作为月诊数据基础（见 §6 花厅坊日诊最小数据包模板）。*

## 3. 维度 3 · 分节令：节令诊断调整

> **节令期间，正常的销售异常不应触发危险状态。** 端午粽子销量暴涨不是「结构失衡」，冬至后腊味回落不是「品类衰退」，夏季汤料下滑不是「护城河失守」。节令诊断做三件事：**① 确保状态机计算层参数已正确录入 → ② 偏移诊断重点 → ③ 前移准备窗口。** 与 [[KB-GBA-TPOS-001_大湾区TPOS本地化_v0.1]] 六节令表联合使用。

> **【三套节令校正机制·层级与分工（V2 验证修订·必读）】**
> 节令校正横跨「调度层（本框架）」与「计算层（KB-CAT-STATE-001）」，三套机制分工明确，不冲突：
>
> | 层级 | 机制 | 执行者 | 做什么 |
> |---|---|---|---|
> | **调度层（本框架）** | DIMENSION-001 §3 节令调整 | 顾问 / 人工判断 | 决定「哪个节令关注哪个品类」；确保 V6 值已正确录入 CAT-STATE-001；设置回归日 |
> | **计算层·自动** | KB-CAT-STATE-001 V4 节令豁免闸 | CAT-STATE-001 自动运行 | 季节性品类 V6<0.5 时，自动把「空缺-紧急🔴」降级为「健康-季节性休眠」，防误报 |
> | **计算层·自动** | KB-CAT-STATE-001 V6 乘性修正 | CAT-STATE-001 自动运行 | V6_adj = 1+(V6−1)×k_季，对健康分 H 做有界微调（±10~15%），自动缓解节令期误判 |
>
> **分工结论**：DIMENSION-001 不直接操作宽度5态阈值或周转红线的参数（CAT-STATE-001 对此无外部接口）。DIMENSION-001 的节令操作 = 确保 V6 正确录入 + 回归日已设置 + 备货量决策参考（节令乘子）。V4/V6 在计算层自动处理误报，无需 DIMENSION-001 重复干预。

### 3.1 三件调整（机制定义）

**① 调度层动作（节令窗口期内）**——DIMENSION-001 的节令校正是**调度层的人工输入**，不直接修改 CAT-STATE-001 的硬编码阈值（该层无对外参数接口）；实际误报防护由 CAT-STATE-001 的 V4 豁免闸和 V6 乘性修正自动完成。DIMENSION-001 的操作职责：
- **确保 V6 值已正确录入**：节令期内受影响品类的 V6 系数须已在 CAT-STATE-001 中录入（值源自 KB-GBA-TPOS-001 六节令表）；V6 未录入 = V4/V6 保护机制无法生效，自动计算仍会误报。
- **设置回归日**：每个节令窗口须显式设置**回归日**（= 节令结束日 + 回落期，见 §3.2 回落期列）。护栏②「节令结束必须回归常态」依赖此参数——未设置回归日 = 永久放水风险（CAT-STATE-001 不会自动判断节令结束，须人工触发回归）。
- **备货量决策**：受提升品类的备货上限 = 常态备货量 × 节令乘子（§3.2 乘子列）。**阈值校正幅度参考**（inferred·待花厅坊实测标定）：节令乘子 ×N → 相关备货类决策容限 ×N（如：周转天数「缺货」告警线缩短至常态的 1/N；V6 系数 ≈ min(乘子, 1.8)，精确换算待 KB-SEASON-MATRIX-001 建成后标定）。
- **受降低品类**：确认 V6 已录入较低值（V6<0.5 将触发 V4 豁免闸，把🔴空缺自动降为季节性休眠，无需人工干预）。
- **未受影响品类**：无操作（防节令一刀切放水）。

**② 诊断重点偏移**——节令窗口内，把诊断注意力从常态指标移到节令敏感指标：
- 节令前：重点查**备货充足度 + 陈列扩面到位 + 提前引进的波段品是否入场**。
- 节令中：重点查**日销速跟不跟得上（缺货丢单）+ 损耗（备多了）+ T×S 双触发扩陈列是否兑现**。
- 节令后：重点查**退场清理（节令品不留库存挤压）+ V6 回归录入 + 回归日触发是否执行**。

**③ 提前准备窗口**——节令前 N 天启动准备诊断：查供应商锁定、首批到货、陈列位规划、价格带设定。**N 天 = 提前准备天数**（见 §3.2，inferred·待标定）。

### 3.2 节令 × 品类 × 影响 调整表（v0.1 草案·inferred·待标定）

> 节令与高需求品类映射源自 KB-GBA-TPOS-001；**影响方向 / 调整幅度 / 提前准备天数 / 回落期**为本框架新增量化层，均为 inferred，须花厅坊多年同比数据标定后方可升 observed。

**两套量化数字的角色区分（V2 验证修订·不可混用）**：
- **节令乘子（×N）** = **需求倍率**，用于**备货决策**（订多少货 / 扩多大陈列面）；量纲 = 节令期预期销量 ÷ 常态销量。
- **V6 系数**（在 KB-CAT-STATE-001 使用）= **健康分校正因子**，用于**状态判断**（品类在节令期波动是否正常）；量纲 = 健康分 H 的有界修正系数（0.3~2.0，实际影响 ±10~15%）。
- 两套数字来自同一底层销售数据，但量纲不同，**不可直接类比或混用**。暂定换算参考（待花厅坊实测后标定）：V6 ≈ min(乘子, 1.8)，即乘子超过 1.8 后 V6 封顶，V6 对健康分的影响有界。精确值须录入 KB-SEASON-MATRIX-001 后生效。

| 节令（月份） | 品类类型 | 影响方向 | 节令乘子（inferred·备货决策用） | 提前准备天数（inferred） | 回落期（天·inferred） |
|---|---|---|---|---|---|
| **冬至（12 月）** | 烧腊 / 腊味 / 糯米 / 汤料 / 礼盒 | 提升 | ×1.5–2.5（全年最重家庭聚餐节点） | 14–21 天 | 7–10 天（12月24日前后回归） |

> *【张力·必读】节令校正最大的风险是**「放水掩盖真问题」**：阈值一放宽，节令期内的真实经营恶化（如损耗管理失控、供应商断供）会被归因到「节令正常波动」而漏诊。**护栏三条**：① 校正**只对映射表内的受影响品类**生效，其余品类阈值不动；② 校正是**临时窗口**，须显式设置回归日（见上表回落期列），节令结束必须触发回归，防止「永久放水」；③ 现金流红线（周转 > 结算周期）**不受节令校正**——节令囤货占现金反而更该盯（横切进 D5，见 KB-GBA-TPOS-001 §2）。*

## 4. 三维合成：收敛铁律（本框架的执行核心）

> 三维正交展开 = 3 族 × 5 周期 × 7 节令 ≈ 上百个诊断组合。**全铺开违反「少而深」，必然无人执行、沦为纸面 SOP。** 本节定义如何把三维压回**可执行的少数几条诊断线**。

**收敛规则（默认诊断路由）**：

1. **品类族定「主周期线」**：每族只跑一条主诊断线——生鲜族→日线、快消族→周线、百货族→月线。**其余周期不默认跑**（生鲜不做月线常态诊、百货不做日线）。
2. **周期矩阵定「加诊触发」**：季 / 年为全店级低频诊断，按日历固定跑（季度末 / 年度末），不因品类族重复。
3. **节令定「事件触发的临时加诊」**：仅在**节令窗口 [节令日 − 提前准备天数, 节令日 + 回落期]** 内，对**映射表内受影响品类**临时叠加节令口径诊断；窗口外不激活。

   **双窗口并发裁决规则（V1 验证修订）**：当两个节令窗口在同一月份并发且覆盖相同品类（如 12 月冬至 + 腊月均覆盖腊味），执行以下规则：
   - ① **备货上限取较高乘子**（冬至 ×1.5-2.5 vs 腊月 ×2-3 → 取腊月 ×2-3 作为备货上限）
   - ② **阈值放宽取最宽松值**（两窗口各自校正效果中取更宽松的一方）
   - ③ **回归独立执行**：两窗口各自有独立回归日（冬至回归日 = 约 12 月 24 日 + 回落期 7-10 天；腊月回归日 = 农历正月初十），较早窗口回归后，较晚窗口继续有效，不共用回归日
   - ④ 并发期间，顾问须在月诊中显式确认「哪个窗口主导本月采购决策」，确保采购员面对两套乘子有明确执行依据

4. **危机优先级压倒一切**：任一 7D 的 P0（生鲜失守 / 现金流断裂 / 称重诚信）触发 → 冻结上述常规路由，转 TOP3 止血。

**合成后的「一家店默认诊断日历」（收敛示例）**：

| 频率 | 默认跑什么 | 谁 |
|---|---|---|
| 每日 | 生鲜族日线（损耗 + 日清 + 时段） | 店长 / 生鲜主管 |
| 每周 | 快消族周线（趋势 + 空销双口径 + 价格执行）+ 生鲜周汇总 | 店长 / 采购 |
| 每月 | 百货族月线 + 全店 7D 快照 | 店长 / 顾问 |
| 每季 | 品类角色重评 + 选品结构 + 供应商 | 顾问 / 老板 |
| 每年 | 全品类宽度重设 + 定价 + 店型 | 老板 / 顾问（六哥签字） |
| 事件触发 | 节令窗口内受影响品类临时加诊（按 §3.2） | 采购 / 顾问 |

> *这张日历就是本框架的「可交付物形态」——它把 7D + 三维展开落成客户店长看得懂、跑得动的一页巡店节奏表。落地载体未来收口进 [[52周MD]]（52 周活动日历 + 商品经营计划模板）。*

## 6. 花厅坊直接调用（落地锚点）

> *【KB 质量铁律·直接调用性】下次为花厅坊设计诊断节奏时，本框架可直接生成「→ 下一个具体动作」：*
