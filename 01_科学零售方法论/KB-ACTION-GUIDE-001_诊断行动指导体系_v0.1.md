## 核心论点（结论先行）

> **诊断的产物不是"问题清单"，而是"动作清单"。** 一条发现如果不能立刻回答"谁、何时、怎么动、动完怎么验收、不动的代价"，它就还停在诊断卡上，没有进入交付。本页是诊断卡（七维评分）与现场执行之间缺的那座桥——把每一类典型问题钉死到一个**带触发阈值的具体动作模板**上。

三条设计铁律（贯穿全页）：

1. **触发要有数值阈值**——"生鲜损耗高"不是触发条件，"叶菜损耗率 >8%"才是。无阈值不进问题库。
2. **动作要有执行步骤**——"优化商品结构"是空话；"按 POS 销额降序，退出尾部 40–60 个连续 6 周 0 动销的中价重复款"才是动作。
3. **可逆性决定执行档位**——对照全局宪法操作四档：可逆增量直接做（A/B 档），不可逆/跨架构动作先列清单等签字（C/D 档）。本页每个动作标注建议档位。

## 0. 如何使用本页（30 秒上手）

```
诊断卡出分 → 看哪个维度(D1–D7)失守/拉低 → 查 §A 问题库定位问题ID
        → 问题ID 的"对应动作"列指向 §B 动作模板
        → 动作模板给出 谁/何时/步骤/资源/验收/不做代价
        → 多个动作并存时 → 过 §C 优先级矩阵排时序（P0先过铁律1，不进矩阵）
        → 有前置依赖的 → 查 §D 连锁关系，先做前置再做后续
        → 顾问离场前 → 每个>7天动作必填「顾问缺席协议」
```

## A. 问题类型库（35 个典型问题 · 7D × 严重程度）

> **格式说明**：问题ID / 维度 / 问题描述 / 诊断触发条件（数值阈值）/ **证据级** / 紧急程度 / 对应动作。
>
> **紧急程度**：**P0=护城河/现金流红线，即时止血**；**P1=结构性失分，30天内动**；**P2=效率优化，60–90天或数据积累后动**。
>
> **证据等级（使用点可见）**：**A**=多店验证或行业标准；**B**=有限验证（方法论/行业基准）；**C**=花厅坊单例推导（待验证假设，不得向客户呈现为基准，须≥2家验证后升B）；**D**=专家判断（无数据基础，须保守使用）。C/D级触发只能立"待验证假设"，不得直接作为客户承诺依据。

## B. 动作库（6类 × 各≥5动作模板）

> 每个模板字段：**触发**（问题ID）→ **执行步骤**（不允许空话）→ **谁**（顾问C/老板B/店长S/员工E）→ **何时**→ **资源**（时间·人力·资金 三轴：低/中/高）→ **验收**（量化指标）→ **不做代价** → **档位**（A/B/C/D）→ **顾问缺席协议**（>7天动作必填）。

### 类型一 · 调品（选品：增/减/替换）

**AC1-1 退出尾部滞销SKU**
- 触发：PB-D2-01 / PB-D5-02 / PB-D5-05
- 步骤：① POS拉该类30天（季令品取窗口期）SKU销额降序；② 标记连续6周0动销 或 销额落在尾部20%且非目的性角色（KB-SKU-ROLE-001 待淘汰品）；③ 剔除"销弱因陈列差"的伪滞销（目的性/偏好性品先改陈列观察30天，KB-CAT-DX-MISMATCH-001局限1）；④ 列退出清单+残值处理（清仓价/退供/捆绑）；⑤ 下架腾出面位交 AC2-2。
- 谁：顾问出清单 → 店长执行 → 老板签退出（涉及退供谈判老板出面）
- 何时：清单7天 / 清仓执行30天
- 资源：时间中·人力低·资金低（仅清仓让利）
- 验收：滞销SKU占比降至<10%；释放面位数；该类GMROI回升
- 不做代价：滞销持续占款占位，周转拖长，挤压新品引进空间，现金沉淀
- 档位：清单C档（不可逆下架，列清单确认）；清仓执行B档
- 顾问缺席协议：离场前交接物=退出SKU清单+清仓价授权表给店长；推进主体=店长；上报节点=Week2店长报去化SKU数和剩余库存额，Week4报完成率；解锁条件=Week4数据达标后顾问确认是否追加促销力度，店长不得自行加码

**AC1-2 引进价格带空缺补位品**
- 触发：PB-D2-02 / PB-D2-03 / PB-D2-05
- 步骤：① 画该类价格带分布直方图（按销额/SKU数双轴）；② 定位空缺档（如花厅坊T03中高价12–22元空缺）；③ 按"形象品/高毛利承接品"角色选2–3款补位，不超量；④ 设进场试销期30天考核；⑤ 配 AC3-1 锚点定价 + AC2-3 黄金位。

- 验收：空缺档销售占比从~0升至该类5–10%；综合毛利率+1–2pp（高毛利承接）

- 档位：C档（引进=增SKU，列清单确认）

### 类型二 · 调陈列（货位/排面/动线）

> **格位计数法（V1-缺陷3修正 · D3全维度通用）**：本章所有涉及「销售-陈列比」的步骤①，均使用以下快速估算法，约15分钟完成，由店长或顾问助手执行：
> 1. 走货架，以「1层板×约1米宽=1格位」为单位记录各L3品类格位数
> 2. 填入【格位记录简表】（三列：L3品类 / 实际格位数 / 本周POS销额）
> 3. 格位占比 = 该类格位数 ÷ 货架总格位数
> 4. 销售-陈列比 = 该类销售占比 ÷ 该类格位占比
>
> 无需测量实际面积，格位数即可替代。若货架结构特殊（弧形/立体柜）无法简单计数，降为D级定性快判。

**AC2-1 增格（强势品扩面）**
- 触发：PB-D3-01 / PB-D2-06
- 步骤：① **【格位计数法】**走店15分钟，店长填写【格位记录简表】（L3品类/格位数/本周销额），算各L3格位占比；② 计算销售-陈列比（销售占比÷格位占比），比值>1.5且角色为目的性/偏好性→列增格候选；③ POS销额降序对比现场格位数，差值最大前5个=第一批；④ 从缩格候选（AC2-2）调出空间补给；⑤ 30天复测比值回到0.7–1.5。
- 谁：顾问诊断 → 店长走店+填表 → 陈列员执行
- 何时：7天（局部）/ 配合区域调改30天
- 资源：时间低·人力低·资金低（仅理货）
- 验收：增格品销售-陈列比回落至匹配区间；该品销额↑
- 不做代价：强势品被憋在小面位，售罄断货损失隐性销售，A类未最大化
- 档位：A/B档（理货可逆，局部A，多区B）
- 顾问缺席协议（30天配合调改阶段）：离场前交接物=格位记录简表+增格任务清单给店长；推进主体=店长；上报节点=Day7完成第一批格位调整后发照片，Day30复测比值；解锁条件=顾问收到Day30数据确认效果

**AC2-2 缩格（弱品压缩）**

- 步骤：① **【格位计数法】**同AC2-1步骤①，标比值<0.5且角色偶发性的子格；② 排除"销弱因陈列差"的目的性品（先改陈列观察）；③ 缩为半格/SKU级补充，让出空间给增格品；④ 季令品避开旺季前夕缩（KB-CAT-DX-MISMATCH-001失效条件）。
- 谁：顾问诊断 → 陈列员执行
- 何时：7–30天
- 资源：时间低·人力低·资金低
- 验收：释放面位数；整体货架销售-陈列匹配度提升
- 不做代价：弱品长期占黄金空间，坪效拖累，挤压强势品
- 档位：A/B档
- 顾问缺席协议（30天阶段）：离场前交接物=缩格清单+季令例外名单给店长；推进主体=店长；上报节点=Day15面位调整完成情况；解锁条件=顾问Day30远程查看格位记录对比表

**AC2-3 黄金视线层换A类**
- 触发：PB-D3-03 / PB-D2-06
- 步骤：① 核查1.4–1.6m视线层现陈列品角色；② 非A类/非形象品的下移，A类/高毛利承接品/形象品上移；③ 端架=主货架放大器，放引流或利润品（KB-CAT-ZONE-SOP-001第四步）；④ 30天看动销变化。

### 类型三 · 调价（定价/促销 · 大湾区价格敏感特别注意）

> 🦐 **大湾区价格敏感铁律**：城中村/社区客群对刚需引流品（米油蛋菜肉、包装水）极度敏感，这些必须价格对标到位甚至引流定位；但特色差异化品类（煲汤料/凉果/陈皮/烧腊）折扣竞品偏弱，是**差异化定价的最大阵地**，不必跟价、可保高毛利。调价的纪律是：**引流品卷价格，特色品卷价值**，两者绝不混淆。

**AC3-1 价格带三档锚点设置**
- 触发：PB-D2-02
- 步骤：① 该类设低/中/高三档锚点（如T03饼干15/18/22.8元）；② 锚点品负责价格感知，承接品做正价主体，活动敏感品做促销；③ 标价对比突出锚点；④ 30天观察价格带是否被填满、有无新空档（四线复盘·定价复盘）。
- 谁：顾问定价格架构 → 老板审 → 店长改标价
- 何时：7天
- 资源：时间低·人力低·资金低
- 验收：三档结构成形；客单价结构改善（高档承接出现）
- 不做代价：无锚点=顾客无价格参照，高价品卖不动，价格带塌成单档
- 档位：B档

**AC3-2 生鲜日清阶梯打折SOP 🦐**
- 触发：PB-D1-02 / PB-D1-03 / PB-D1-06
- 步骤：① 定阶梯：下午4点7–8折→打烊前5折（肉类，KB-FRESH-001）；② 豆制品/熟食按日清率目标设折扣节点（豆制品日清率≥85%）；③ 上岗培训+授权店员执行（不必请示）；④ 日清率与损耗每日记录。
- 谁：顾问写SOP → 店长培训 → 员工每日执行
- 何时：即时（当天可上）
- 资源：时间低·人力低·资金低（让利换日清）
- 验收：肉类日清率→100%无隔夜；叶菜损耗≤8%；损耗降1%≈综合毛利提升（见RM-04有效域注）
- **RM-04有效域注（V2-缺陷5修正）**：「损耗降1%≈综合毛利+0.3%」（RM-04）的成立条件为生鲜占全店销售约40%。当生鲜占比<30%（D1-P0状态）时，实际改善约0.19–0.22pp而非0.3pp；请以「实际生鲜占比×0.75」作修正系数。在生鲜P0场景下向客户承诺改善幅度时，用定性表述替代此数字，精确量化须待AC5-2完成4周数据后计算。
- 不做代价：隔夜肉=品质信任崩塌（护城河第一层根基），损耗吃毛利，鲜度差赶客
- 档位：A档（即办，可逆）

**AC3-3 特色品差异化定价 🦐**
- 触发：PB-D5-03
- 步骤：① 锁定大湾区特色项（煲汤料/凉果/陈皮/龟苓膏/海产干货）；② 按毛利25–32%定价（折扣竞品在此偏弱）；③ 不跟折扣店价，靠品质/本地化/现场体验支撑；④ 做品类故事陈列强化价值感。
- 谁：顾问定策略 → 老板定价 → 店长执行
- 何时：30天
- 资源：时间低·人力低·资金低
- 验收：特色项毛利率达25–32%；综合毛利率被拉高1–2pp
- 不做代价：把差异化阵地也卷成价格战，丢掉唯一的高毛利安全区
- 档位：B档
- 顾问缺席协议：离场前交接物=特色品定价清单+毛利核查方法给老板；推进主体=老板；上报节点=Day15反馈各特色品当前定价和销量；解锁条件=Day30顾问复核毛利率

**AC3-4 引流品价格对标（止血客流）**
- 触发：PB-D4-01（客流跌+客单涨危险信号）
- **步骤0（前置排除检查·V1-缺陷1修正，执行此步才可进入步骤①）**：确认以下所有问题：（a）本次触发是否已满足PB-D4-01中≥4周持续条件；（b）近期是否主动撤SKU/缩品类/调整收银流程/搬迁？若有可解释的结构性原因→记为"待观察警示"，不执行AC3-4，继续追踪4周后重判。无可解释原因方可进入步骤①。
- 步骤：① 锁定刚需高频引流品（米油蛋菜肉/包装水/乳品）；② 对标周边折扣店/对手价：顾问或老板当天/次日步行走访门店1公里内主要竞争折扣店/生鲜超市，比价清单固定约15个刚需引流品（5kg大米/5L食用油/30枚鸡蛋/2–3款叶菜/1.5L包装水/常温乳品），用手机拍价签，20–30分钟完成，每月更新一次；③ 把价格调到有竞争力；④ 用引流品定位（毛利可薄甚至引流），靠连带利润品回收；⑤ 周度复盘客流是否回升。
- 谁：顾问诊断+前置排除 → 老板定价（涉及毛利让渡，老板拍）
- 何时：即时–7天（止血优先，但需先完成步骤0排除误报）
- 资源：时间低·人力低·资金中（让利）
- 验收：客单数（客流代理）止跌回升；危险信号（连续下滑趋势）解除
- 不做代价：客单价涨是"消耗老客"的假繁荣，客流持续流失，3–6个月后崩盘
- 档位：B/C档（涉毛利结构，老板确认）

**AC3-5 活动促销机制（冲量/连带/清库存）**
- 触发：PB-D4-02 / PB-D4-03 / PB-D5-02
- 步骤：① 按目标选机制——拉客流用引流品特价、提连带用搭配满赠、清积压用捆绑清仓；② 选活动角色品（引流/冲量/连带/利润品，KB-SKU-ROLE-001）；③ 设活动期+目标+预算；④ 活动后做四项复盘（销量/毛利/连带/周转）。
- 谁：顾问设计 → 老板批预算 → 店长执行
- 何时：7–30天（按节奏滚动）
- 资源：时间中·人力中·资金中
- 验收：活动期客流/客单件数/积压SKU去化达预设目标
- 不做代价：无活动=经营无节奏，需求不被激活，积压不去化，结果无放大
- 档位：B档
- 顾问缺席协议：离场前交接物=活动方案（机制/角色品/预算/考核标准）给老板；推进主体=老板；上报节点=活动中期（Day7/Day15）反馈客流件数变化；解锁条件=活动后顾问复盘四项数据，下一轮活动须顾问设计后启动

### 类型五 · 调流程（运营SOP改写）

**AC5-1 生鲜日清SOP改写**
- 触发：PB-D1-02 / PB-D1-03 / PB-D6-04 / PB-D7-04
- 步骤：① 写日清全流程SOP（订货量上限/到货验收/分时打折节点/打烊清零/隔夜禁上架）；② 明确责任人+时间点；③ 上墙+培训；④ 日清率每日记录复盘。
- 谁：顾问写 → 店长落地 → 员工执行
- 何时：7天
- 资源：时间低·人力低·资金低
- 验收：肉类日清率100%；豆制品≥85%；损耗下降
- 不做代价：日清靠人记忆=时好时坏，损耗与信任不稳
- 档位：B档

**AC5-2 损耗记录与归因流程**
- 触发：PB-D1-02 / PB-D5-03
- 步骤：① 建每日损耗台账（品类/数量/金额/原因码）；② 按原因归类（订货过量/鲜度管理差/陈列损伤/客损）；③ 周度归因找前三大损耗源；④ 对症（订货量/技能/陈列）。损耗是生鲜问题先导指标，早毛利2–4周（KB-RETAIL-KPI-001）。
- 谁：顾问建表 → 店长/员工每日记 → 顾问周度分析
- 何时：7天建表 / 需数据（4周后可归因）
- 资源：时间中·人力低·资金低
- 验收：损耗有数可查；前三损耗源识别并下降；损耗降1%→毛利改善（见RM-04有效域注）
- **RM-04有效域注（V2-缺陷5修正）**：同AC3-2的RM-04注。在生鲜占比<30%时，向客户报告毛利改善时须用「实际生鲜占比×0.75」修正系数替代0.3%固定值，或仅用定性表述，精确量化在4周台账完成后计算。
- 不做代价：损耗黑箱，毛利漏在看不见处，无法对症
- 档位：A/B档
- 顾问缺席协议：离场前交接物=损耗台账模板+记录SOP给店长；推进主体=店长/员工每日填；上报节点=每周五发当周台账给顾问；解锁条件=顾问周度分析后回复下周改进重点，店长不得擅自停记

**AC5-3 客流/客单数据采集流程**
- 触发：PB-D4-04 / PB-D4-05 / PB-D7-02
- 步骤：① 优先装客流计数（门口红外/摄像头AI）或先用POS订单数作客流代理；② 建每日/周数据看板（客单数/客单价/件数/前5单品）；③ 训练老板每周看数习惯；④ 进店转化率=客单数÷客流（设备到位后）。
- 谁：顾问设计 → 老板投入设备 → 店长维护看板
- 何时：30天（设备）/ 数据习惯持续
- 资源：时间中·人力低·资金中（设备）
- 验收：客流可观测；转化率可算；老板能说出前5单品
- 不做代价：客流是X光机，没数据=盲诊，所有客流维度结论只能停在估算（C级）
- 档位：B/C档
- 顾问缺席协议：离场前交接物=看板模板+数据填写说明给店长；推进主体=店长；上报节点=每周一发上周数据看板截图；解锁条件=顾问确认数据完整后继续，缺2周数据=预警

**AC5-4 动销驱动订货流程**
- 触发：PB-D5-02 / PB-D7-01 / PB-D5-05
- 步骤：① 按SKU周转天数设安全库存与补货点；② 高周转生鲜小批量高频，低周转工业品大批低频（对照KB-RETAIL-KPI-001各类周转基准）；③ 滞销品停止自动补货；④ 月度对账库存周转。
- 谁：顾问设规则 → 采购/店长执行
- 何时：30天
- 资源：时间中·人力低·资金低
- 验收：综合周转天数<账期；断货率↓；积压↓
- 不做代价：拍脑袋订货=要么断货丢单要么积压压款
- 档位：B档
- 顾问缺席协议：离场前交接物=订货规则表（各品类安全库存+补货点）给采购/店长；推进主体=采购；上报节点=Month1末发当月周转数据；解锁条件=顾问复核后调整规则

**AC5-5 四线复盘机制（30/90天）**
- 触发：PB-D4-05 / 任何调改动作收尾
- 步骤：① 固定30天/90天复盘节点；② 四线复盘——选品（新品是否如预期）/定价（价格带空档重叠）/陈列（错配是否消除）/活动（量·毛利·连带·周转达标否）；③ 出复盘结论→下一轮动作；④ 沉淀进M-DEC。

- 何时：30/90天周期
- 资源：时间中·人力低·资金低
- 验收：每轮调改有闭环复盘；动作有数据反馈
- 不做代价：只调不复盘=系统只能分析无法驱动结果（KB-METHOD-4TACTICS-001），无法迭代
- 档位：B档
- 顾问缺席协议：离场前交接物=复盘模板（四线数据收集表）给老板；推进主体=老板收集数据；上报节点=Day28/Day88前老板汇总四线数据发顾问；解锁条件=顾问收到完整数据后安排复盘会（远程/现场均可），未收到数据不得进入下一轮调改

### 类型六 · 调供应链（供应商/频次 · 含门槛条件）

> 🦐 **门槛条件**：调供应链动作多数依赖采购量级与谈判筹码，社区单店议价能力弱。**先评估门槛**：账期谈判需稳定采购量背书；引备选货源需达起订量；采购频次调整需供应商配送能力。门槛不足时，这些动作降为"中长期目标"或借小连锁集采，不强行推。

**AC6-1 生鲜到货时间前移 🦐**
- 触发：PB-D1-02 / PB-D1-03 / PB-D7-04
- 步骤：① 与生鲜供应商谈配送时间提前到开市前；② 不行则改自提/换近场货源；③ 到货验收前置，保证上午高售罄时段有货；④ 跟踪到货准时率。
- 谁：顾问诊断 → 老板/采购谈
- 何时：30天
- 资源：时间中·人力低·资金低
- 门槛：供应商配送能力/有近场替代
- 验收：到货准时率↑；上午缺货↓；日清更顺
- 不做代价：到货晚=上午无货丢早市客流、下午积压拉高损耗
- 档位：B/C档
- 顾问缺席协议：离场前交接物=谈判要点+替代货源备选清单给老板；推进主体=老板；上报节点=Day15反馈谈判结果；解锁条件=顾问收到结果后确认下一步（成功则监测准时率，失败则启动换货源程序）

**AC6-2 ERP进价数据清洗**
- 触发：PB-D7-02
- 步骤：① 抽样核对生鲜/重点品类ERP进价与实际采购单（优先抽前30大SKU）；② 批量订正错误进价；③ 建进价维护流程（每次采购更新）；④ 重算毛利/周转。这是所有财务维度诊断的前置（花厅坊已知盲区）。
- 谁：顾问指出 → 店长/采购清洗 → 顾问复核
- 何时：30天
- 资源：时间高·人力中·资金低
- 验收：进价准确率↑；毛利/周转数据可信（从C级升B级）
- 不做代价：进价脏=毛利周转全不可信，财务诊断全建在沙上
- 档位：B档（数据维护，不动客户敏感原值结构）
- 顾问缺席协议：离场前交接物=抽样清单+进价维护SOP给采购；推进主体=采购；上报节点=Week2发已订正的前30大SKU进价清单；解锁条件=顾问核验后确认可进入全量清洗，未核验不得扩大范围

**AC6-3 账期结构优化**
- 触发：PB-D5-01 / PB-D7-03
- 步骤：① 盘点供应商账期结构（现结/30/45/60天占比）；② 对高占比品类谈延长账期或换账期友好供应商；③ 优先处理拖累现金流的大额现结；④ 复核综合周转天数vs账期。
- 谁：顾问诊断 → 老板谈判（需采购量筹码）
- 何时：60–90天
- 资源：时间中·人力低·资金低
- 门槛：稳定采购量作谈判背书
- 验收：综合周转天数<结算周期（RM-06红线）；现金流缓解
- 不做代价：周转>账期=资金链紧张，扩品/备货无弹药，最坏断裂
- 档位：C档

- 资源：时间中·人力中·资金低
- 门槛：达备选供应商起订量

- 不做代价：单一货源断货=货架开天窗，顾客扑空转投对手
- 档位：B/C档
- 顾问缺席协议：离场前交接物=必备品备选货源候选清单（含联系方式+起订量）给老板；推进主体=老板；上报节点=Day30反馈已接触的备选供应商情况；解锁条件=顾问确认第二货源到位

## C. 优先级裁决矩阵（影响力 × 执行难度）

> **使用前必读两项前置声明：**

> **【P0 除外声明】（V2-缺陷2修正）**：**本矩阵仅适用于 P1/P2 问题排序。P0 问题触发§D裁决铁律1，直接绕过本矩阵按即时档执行，不参与矩阵排队。** 矩阵中标注P0的动作（AC3-4、AC3-2、AC5-1等）仅供P0解除后的持续优化阶段参考；若问题仍为P0状态，铁律1优先，此格不适用。

> **【影响力判断三步决策树（V2-缺陷2修正）】**：使用矩阵前，先用以下三步判断动作的影响力等级（这是顾问的输入判断，不是矩阵输出）：
> - **Step 1**：这个动作对「客单数×客单价」公式的核心变量有直接数值撬动吗（如客流、转化率、件数、定价）？**有 → 高**

## D. 动作连锁关系（依赖图 · 先A后B）

> 有些动作有硬依赖：前置没做，后续做了也无效甚至有害。诊断后排序必须尊重连锁。

### 多链并发裁决规则（V2-缺陷3修正 · 2026-06-30加入）

> 当两条连锁链共用同一动作（如AC1-4同时出现在链1和链5），且两链给出矛盾指令时，按以下规则裁决：

**P0链优先于P1/P2链的序列依赖**：P0级连锁链（链5：客流危险信号止血）的即时并行指令，优先于P1/P2级连锁链（链1：生鲜结构序列依赖）的等待条件。

**但P0链并行时须附加守护条件**：P0客流危险信号触发时，AC1-4可并行执行而不等损耗达标，但**必须同步启动AC5-1（日清SOP）+ AC5-2（损耗台账）双轨并行作为守护条件**。单独执行AC1-4而不配守护条件，则链1警告的「放大损耗」风险仍然成立。

**实操判断**：当PB-D4-01（P0）与PB-D1-02（P1）同时触发时：执行 AC3-4 + AC5-3 + AC1-4 并行（链5指令），同时强制启动AC5-1 + AC5-2（守护条件），待4周后评估损耗是否达标再决定是否继续扩品种结构。

---

### 连锁链 1：生鲜——先稳损耗，再优结构
```
AC5-2 损耗台账 + AC5-1 日清SOP + AC3-2 日清打折（先稳损耗与日清）
        ↓ 损耗/日清达标后
AC1-4 补品种结构 + AC2-4 生鲜扩面（再做结构与空间）
        ↓
AC1-5 长尾撤退评估（最后做减法）
```
理由：损耗失控时扩面/补品种=放大损耗；必须先让生鲜"不漏"，再让它"长大"。KB-FRESH-001修复顺序：日清机制→品种结构→鲜度陈列→大湾区特化。

**⚠️ AC1-4多链冲突注**：若同时触发链5（P0客流危险信号），AC1-4的序列等待条件按上方「多链并发裁决规则」处理（P0链优先+守护条件并行）。

### 连锁链 2：数据——先有数，再诊断财务/客流
```
AC6-2 ERP进价清洗 + AC5-3 客流数据采集（先建数据底座）
        ↓ 数据可信后
AC5-4 动销订货 + AC6-3 账期优化 + AC6-5 采购频次（财务动作）
AC3-4/AC3-5 客流活动动作（客流动作）
```
理由：进价脏（PB-D7-02）则毛利/周转全不可信，基于脏数据的财务动作会错杀；客流无观测则客流动作无法验收。**无数据的财务/客流动作=C/D级证据，不得直接执行。**

**⚠️ 脏数据P0降级处理规则（V1-缺陷2修正 · 2026-06-30加入）**：

当PB-D7-02（ERP进价脏）已确认时，PB-D5-01（周转>账期P0）触发降级流程，而非直接执行全量财务动作：

1. PB-D5-01从P0降为「P0-待确认」状态
2. 立即启动「7天快速抽样核验」版AC6-2：抽前30大SKU核进价（而非等待全量30天清洗）
3. 同时发起现金流主观问询（直接问老板：「本月15/30天内到期账款金额是多少？现金是否够付？」）作为临时替代红线信号
4. 若快速抽样核验后周转仍明显超账期，升回P0执行AC6-3；若临时问询显示即时现金流告急，亦升回P0即时处理
5. 在完成步骤2–3前，AC6-3（账期优化）和AC6-5（采购频次）不得基于脏数据直接执行

### 连锁链 3：陈列——先框架，后SKU（五步法纪律）
```
AC2-6 区域角色定位（说明书）
        ↓
错配诊断（AC2-1增格/AC2-2缩格工具）
        ↓
CDT+子格落位 → AC2-3黄金位
        ↓
AC1-1退滞销 + SKU上下架（最后动SKU）
        ↓
AC5-5 30/90天复盘
```
理由：KB-CAT-ZONE-SOP-001铁律"先框架后SKU"。未定区域角色就动SKU=没有评判标准的瞎调。

### 连锁链 4：商品结构——先减后增，腾笼换鸟
```
AC1-1 退尾部滞销（腾面位+回笼资金）
        ↓
AC2-2 缩格（释放空间）
        ↓
AC1-2 补价格带空缺 / AC1-6 引特色品（用腾出的面位+资金）
        ↓
AC3-1 锚点定价 + AC2-3 黄金位
```
理由：不先减就增=SKU净膨胀、库存进一步积压、现金流更紧（PB-D5-02恶化）。

### 连锁链 5：客流危险信号止血（P0特例·并行不串行）
```
PB-D4-01（客流跌+客单涨，已满足≥4周+前置排除）触发时，三动作并行立即上：
AC3-4 引流品对标（止客流血）
AC5-3 客流数据采集（看止血是否生效）
AC1-4 补生鲜必备（修护城河根因）
        + 守护条件：AC5-1 + AC5-2 同步启动（见多链并发裁决规则）
        ↓ 30天客流企稳后
AC3-5 活动 + AC2-5 端架（提连带提客单数）
```
理由：危险信号是"正在消耗老客基础"，是慢性失血，止血动作要并行抢时间，不能串行等。

**⚠️ AC1-4多链冲突注**：若同时存在PB-D1-02（损耗超标），AC1-4并行执行时须附加守护条件（AC5-1+AC5-2），而非单独扩品种。详见「多链并发裁决规则」。

### 连锁总览（依赖优先级）
```
数据底座(链2) ─┬─→ 财务动作（脏数据时先走降级规则）
               └─→ 客流动作
生鲜稳损耗(链1) ──→ 生鲜扩张（P0客流时走多链并发规则+守护条件）
框架(链3/链4减法) ──→ SKU增法/陈列优化
                            └─→ 复盘(AC5-5) ──→ 回流M-DEC ──→ 下一轮
```
