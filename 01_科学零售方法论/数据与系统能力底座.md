# 数据与系统能力底座

## 1. 文件定位

本文件定义**数据与系统能力底座**的核心概念、构成和对三大支柱的支撑关系。它是 `08_经营分析与数据看板`、`11_系统产品与PRD`、`13_数据分析与工具脚本` 等模块的方法论上游依据。

## 2. 核心定义

**数据与系统能力底座**不是独立于商品力、组织力、运营力之外的第四力，而是支撑三大业务能力运行的基础设施。它让科学零售从经验判断升级为可观察、可分析、可执行、可复盘、可复制的管理体系。

底座本身不创造经营目标，也不替代业务判断，它的价值在于让商品、组织、运营三大能力更高效、更准确、更可持续地运行。

## 3. 为什么不是第四力

| 判断维度 | 第四力（错误认知） | 底座（正确定位） |
|----------|------------------|----------------|
| 是否独立产生经营价值 | 是 | 否，依附于三大支柱 |
| 是否有独立业务目标 | 是 | 否，服务于商品/组织/运营目标 |
| 是否可以独立考核 | 是 | 否，以三大支柱的落地质量为衡量标准 |
| 核心作用 | 独立驱动 | 支撑放大 |

**结论：** 把数据与系统能力单独列为第四力，会导致为了数字化而数字化，脱离真实业务问题；正确做法是把它嵌入三大支柱的执行链路中，让每项业务能力因底座而更强。

## 4. 数据与系统能力底座的核心构成

| 构成模块 | 核心内容 |
|----------|----------|
| 数据采集 | 销售数据、库存数据、客流数据、价格数据、陈列现场数据的采集规范 |
| 数据清洗 | 异常数据识别、条码匹配、字段标准化、历史数据修复规则 |
| 指标体系 | 销售额、毛利额、动销率、库存周转、客单价、坪效、人效等核心指标定义 |
| 分析模型 | SKU ABCD分类、ABC-Z模型、品类贡献分析、价格带分析、活动效果分析 |
| 管理系统 | 门店经营管理软件、任务系统、巡店工具、报表平台 |
| AI工具 | Claude Code辅助分析、智能报告生成、异常预警、辅助决策 |
| 报告输出 | 周报、月报、老板看板、品类报告、商品诊断报告的标准化模板 |
| 执行闭环 | 数据分析结果转化为任务清单、跟踪执行、验证结果的完整链路 |

## 5. 对商品力的支撑

- SKU 动销分析、ABCD 分类、滞销品识别
- 品类贡献额、贡献率、品类宽度与深度分析
- 价格带覆盖分析、主力价格段识别
- 库存周转率诊断、缺货预警
- DisplayMap 联动：根据陈列结构和动销数据提供排面调整依据

## 6. 对组织力的支撑

- 岗位权限的系统化定义和管理
- 任务在系统中布置、追踪、验收，实现任务闭环
- 巡店检查工具：现场问题记录、汇总、推送整改
- 培训进度记录、员工能力评估数据化
- 日报/周报自动生成，降低人工整理负担，让复盘聚焦于判断

## 7. 对运营力的支撑

- 经营看板：销售额、毛利额、客单价、客流、动销率实时可见
- 异常预警：缺货、滞销、毛利异常、库存超期等自动提醒
- 活动复盘：区分自然动销与促销拉动，评估活动 ROI
- 52周MD 数据支撑：历史同期数据支持商品波段规划
- 老板看板：简化版经营核心指标，支持快速决策

## 8. 在社区超市中的典型应用

**当前阶段（数据能力薄弱型门店）：**
- 先做好数据采集规范：POS 数据、库存数据、商品档案的基础整理
- 建立最小指标集：动销率、库存周转、品类销售额
- 用 Excel 或简单工具完成 ABCD 分类和品类分析

**进阶阶段（系统化提升）：**
- 引入门店管理系统，实现任务闭环和巡店工具
- 建立自动化周报和异常预警
- 用 AI 工具辅助商品诊断和报告生成

**注意：** 不同规模和阶段的社区超市，底座建设的优先级和工具选择差异较大，具体方案待确认。

## 10. 常见误区

- **误区1：** 把数据与系统能力当成独立目标——"数字化转型"本身不是目的，让三大支柱跑得更好才是目的。
- **误区2：** 以为有了系统就有了能力——系统是工具，不替代判断；数据出来了，还需要人解读和执行。
- **误区3：** 在基础数据不准确的情况下追求高级分析——数据清洗和字段标准化先于分析模型，基础不牢则分析结论不可信。
- **误区4：** 只建报表，不建执行闭环——数据到看板就停了，没有转化为行动清单，等于没用。
- **误区5：** 忽视社区超市的实际数字化能力——照搬大型连锁的系统方案，会超出门店承载能力，反而形成负担。
