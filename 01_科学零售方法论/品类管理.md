# 品类管理

## 1. 文件定位

本文件是科学零售体系中**品类管理**的主定义文件，是 `05_品类管理与商品规划` 的上游方法论依据，同时支撑 `04_商品诊断与商品力提升`、`06_陈列规划与DisplayMap`、`07_营销活动策划与实施` 等模块的落地作业。

## 2. 核心定义

**品类管理**是以顾客购买任务和门店经营目标为基础，把品类作为战略业务单元进行规划、配置、执行和复盘的管理方法。

品类不是系统里的商品分类。品类是顾客购买任务、商品结构、经营目标和门店资源配置之间的连接单元。

在社区超市场景中，品类管理必须从"老板经验选品"升级为**有目标、有结构、有数据、有执行、有复盘**的经营动作。

## 3. 品类管理完整流程

品类管理包含以下八个环节，构成从定义到复盘的完整闭环：

### 3.1 品类定义

明确品类的边界、服务的顾客购买任务和决策路径。

- **品类边界：** 哪些商品属于这个品类，哪些不属于
- **消费者购买任务：** 顾客进店买这个品类是为了解决什么生活任务
- **顾客决策树 CDT：** 顾客在购买时最先关注什么（品牌/口味/规格/价格），依次决策路径是什么

### 3.2 品类战略

确定品类在门店整体经营中的定位、发展方向和资源分配优先级。

- **品类定位：** 这个品类是门店的强项、弱项还是空白
- **发展方向：** 扩张、维持、收缩还是退出
- **资源投入优先级：** 相对于其他品类，这个品类值得投入多少陈列、活动和采购资源

### 3.3 品类角色

根据门店经营目标，为每个品类分配角色，驱动资源配置决策。

| 角色类型 | 核心作用 | 典型特征 |
|----------|----------|----------|
| 目标性品类 | 吸引主要客流，建立门店形象 | 高频、高关注、客流驱动 |
| 常规性品类 | 满足日常购买需求，维持销售基盘 | 稳定、高复购 |
| 季节性/便利性品类 | 季节带动、一站式满足 | 季节波动明显或补充需求 |
| 形象品类 | 建立门店品质感知 | 顾客感知强，未必销量高 |
| 流量品类 | 吸引客流入店 | 价格敏感，竞争激烈 |
| 利润品类 | 贡献毛利额 | 毛利率高，动销稳定 |

### 3.4 品类评估

定期评估品类经营健康度，判断策略是否需要调整。

| 评估维度 | 评估内容 |
|----------|----------|
| 销售额 | 品类销售额及同比/环比趋势 |
| 毛利额 | 品类毛利贡献及毛利率水平 |
| 动销率 | 在架 SKU 中有销售记录的比例 |
| 库存周转 | 库存消化速度，判断积压风险 |
| 价格带覆盖 | 是否覆盖目标顾客主力消费区间 |
| 客群匹配度 | 品类顾客画像与门店主力客群的吻合程度 |

### 3.5 品类策略

根据品类角色和评估结果，制定具体的经营策略方向。

- **增长策略：** 扩大 SKU 宽度、加大陈列资源、引入新品带动
- **利润策略：** 优化价格带结构，引入高毛利 SKU，收缩低贡献单品
- **引流策略：** 强化价格竞争力，做促销活动，提升客流带动效应
- **防守策略：** 守住基本盘，防止客流被竞品截流，稳定核心 SKU
- **季节策略：** 提前备货，波段式陈列调整，节令商品强化

### 3.6 品类战术

把品类策略转化为可落地的具体商品动作。

- **商品组合：** 确定品类内的 SKU 结构（宽度×深度），哪些保留、引进、淘汰
- **价格带：** 确认价格区间覆盖，主力价格段清晰
- **陈列空间：** 分配货架面数、层位、端架和堆头资源
- **促销活动：** 制定品类促销节奏，与 52周MD 对齐
- **供应商协同：** 核心 SKU 的供货稳定性、账期、新品联合推广

### 3.7 品类执行

把品类战术落地到门店现场。

- **选品：** 按商品配置理论确定进店 SKU 名单
- **上架：** 按标准完成商品档案录入、条码匹配、价签制作
- **陈列：** 按 DisplayMap 完成货架陈列，符合品类陈列规则
- **补货：** 按动销节奏和库存水位执行补货，避免缺货和积压
- **活动落地：** 按活动方案完成堆头、端架、促销标签的现场执行

### 3.8 品类复盘

评估本周期品类执行结果，驱动下一轮策略调整。

- **目标达成：** 销售额、毛利额是否达到目标
- **策略偏差：** 本期策略方向是否正确，哪里出现偏差
- **单品贡献：** 哪些 SKU 贡献了销量/毛利，哪些拖了后腿
- **陈列效果：** 陈列是否符合规范，是否影响了动销
- **下一轮调整：** 明确下一周期的 SKU 调整、价格调整、陈列调整、活动计划

## 4. 社区超市品类管理本土化

社区超市品类管理不能照搬大型连锁的流程和标准，必须结合本地实际条件落地。

关键约束条件包括：门店面积、主力客群半径、社区购买任务、老板资源和决策风格、员工执行能力、供应链可及性。

**第一阶段优先解决的核心问题：**
- 品类边界不清：同一品类商品散落在不同区域，顾客找不到
- SKU 结构失衡：某些品类 SKU 过多积压，某些品类严重断货
- 价格带断层：缺失主力价格段，顾客购买意愿受阻
- 陈列资源错配：高贡献品类陈列面不足，低贡献 SKU 占据黄金位置
- 复盘机制缺失：没有定期品类评估动作，靠感觉做决策

乐易购花厅坊店可作为后续测试案例，但真实数据和结论待现场录入，不得编造。

## 5. 与商品配置理论的关系

品类管理和商品配置理论回答不同层次的问题，两者是战略与落地的关系：

- **品类管理** 回答：这个品类为什么存在？承担什么角色？应该如何发展？
- **商品配置理论** 回答：在具体门店、具体区域、具体时间，应配置哪些商品、多少 SKU、哪些价格带、哪些陈列资源？

品类管理是上游决策依据，商品配置理论是下游落地工具。

## 6. 与52周MD的关系

品类管理确定品类角色和策略方向，52周MD 把品类策略转化为年度、月度、周度的商品经营节奏。

季节性品类（如冷饮、暖饮、节令礼品）和活动性品类（如促销堆头商品）尤其需要通过 52周MD 提前规划备货节点、陈列切换时间和活动商品波段。

## 7. 与数据与系统能力底座的关系

数据与系统能力底座为品类管理提供数据支撑，主要体现在：

- 品类销售额与趋势分析
- 品类毛利贡献分析
- 动销率监控
- 库存周转诊断
- 价格带覆盖分析
- 陈列效果分析（结合 DisplayMap）
- 活动效果分析（区分自然动销与促销拉动）
- 品类复盘报告自动生成

**注意：** 系统和数据不能替代品类判断，只能提高品类判断的准确性、可追踪性和复盘效率。最终判断仍需依赖对顾客、门店、竞品和经营目标的综合理解。

## 8. 常见误区

- **误区1：** 把品类管理等同于商品分类——系统里的分类是商品档案管理，品类管理是以顾客购买任务为核心的战略决策。
- **误区2：** 只看销售额，不看毛利、库存、动销和客群任务——销量好不一定品类健康，需要多维度综合判断。
- **误区3：** 只做选品，不做陈列、价格带规划和复盘——选品只是品类管理的一个环节，割裂执行会导致策略失效。
- **误区4：** 照搬大连锁品类管理流程，忽视社区超市执行能力——复杂流程在5-10人的社区超市无法落地，必须简化和本土化。
- **误区5：** 把品类角色写在文件里，但不落实到 SKU、陈列、活动和复盘——品类角色只有转化为具体执行动作才有价值。
