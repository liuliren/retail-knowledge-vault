# 品类管理

## 1. 文件定位

本文件是科学零售体系中**品类管理**的主定义文件，是 `05_品类管理与商品规划` 的上游方法论依据，同时支撑 `04_商品诊断与商品力提升`、`06_陈列规划与DisplayMap`、`07_营销活动策划与实施` 等模块的落地作业。

## 2. 核心定义

**品类管理**是以顾客购买任务和门店经营目标为基础，把品类作为战略业务单元进行规划、配置、执行和复盘的管理方法。

品类不是系统里的商品分类。品类是顾客购买任务、商品结构、经营目标和门店资源配置之间的连接单元。

在社区超市场景中，品类管理必须从"老板经验选品"升级为**有目标、有结构、有数据、有执行、有复盘**的经营动作。

## 3. 品类管理完整流程

品类管理包含以下八个环节，构成从定义到复盘的完整闭环：

### 3.1 品类定义

明确品类的边界、服务的顾客购买任务和决策路径。

- **品类边界：** 哪些商品属于这个品类，哪些不属于
- **消费者购买任务：** 顾客进店买这个品类是为了解决什么生活任务
- **顾客决策树 CDT：** 顾客在购买时最先关注什么（品牌/口味/规格/价格），依次决策路径是什么

### 3.1A 维度 vs 度量·元框架（零售老刘吸收回填·六哥 2026-06-25 签）

> 来源:零售老刘《品类建模》理论根(02品类语义,转述)。见 [[2026-06-25_零售老刘吸收_方法论回填清单_v0.1]]。

- **维度** = 看问题的角度(不可计数):品类角色 / 价格带 / 复杂度 / 陈列层 / 失真型 —— 决定"从哪个角度看"。
- **度量** = 可计数可量化:销额 / 毛利 / 动销 / 可得性 / 保利期 —— ABCZ 各项都是度量。
- **方法纪律:先定维度,再用度量算分**(不是堆指标)。角色闸/失真库=维度层,9维ABCZ=度量层。
- **哲学根**:品类管理本质=用**结构化逻辑**应对**非结构化、动态、不确定**的零售;数据建模(品类树+复杂类分层+ABCZ)是解这个矛盾的路径(零售老刘04 印证本方法论方向)。
- **方法论理论根:需求论 > 目标顾客论(逐字批24回填)**:商品定位两种方法论——**目标顾客论**(研究顾客画像→指导经营·供应链前端·"产品语义"·传统主流)vs **需求论**("用户不是人,而是需求的集合"·供应链后端·"单品语义"·互联网产品观)。**品类管理 = 需求论的落地**:与其研究"顾客是谁"的画像共性,不如深挖"不同顾客在需求上的相似性"——**用一个小类满足一个需求("一个小类=一个需求")**,以品类(需求集合)为强关联单元做画像。这是本方法论"以需求为中心"而非"以客群为中心"的根:诊断不从"我的客群是谁"起,从"这块货架在替顾客完成哪个需求单元"起。

### 3.1C JTBD 任务单元 + 目标侧（零售老刘吸收回填·六哥 2026-06-25 签）

> 来源:零售老刘《品类建模》任务系列(233-238)+非财务预算(转述)。见 [[2026-06-25_零售老刘吸收_方法论回填清单_v0.1]]。
> **补强来源(逐字批36-38·六哥 2026-06-26「先回填」签)**:零售老刘《品类建模》228-240(商品陈列35-47)逐字精读。见 [[批36_类POP纵向陈列落位JTBD任务_v0.1]]、[[批37_隐性任务类业务任务魔咒熟食关系链_v0.1]]、[[批38_任务交换成本信任库存管理_v0.1]]。

- **品类 = 顾客要完成的"任务"(JTBD),不是商品集合**。CDT(决策树)之上的"why"层:顾客买这个品类要解决什么任务/痛点。**隐性任务**:顾客不直说,只知"一件事没做好/一种感受没满足"——需求侧要洞察(接 [[2026-06-25_零售老刘体系吸收_E2_顾客需求侧簇_v0.1]])。
  - **JTBD 完整模型:顾客"雇佣"商品完成任务(逐字批36-38回填·零售老刘·六哥2026-06-26「先回填」签)**:顾客要的是"洞"(任务)不是"钻头"(商品)——`顾客不在乎你(门店/业务),只在乎他自己的任务`(=零售的解放:别把资源浪费在自我感动;但你得在乎他的任务=信任与复购之源)。**六层业务链(自下而上):属性→组合→品类→需求→宽类→任务**——任务是宽类窄品的"终极上层 why"(品类=管理视角,宽类=顾客视角,任务=跨宽类的上层聚合,如"完成孩子学校午餐"=早餐+便当盒+水果+饮料)。**任务三层:功能层(好不好用·入场券)/情感层(满不满意·复购理由)/社会层(我是谁·溢价来源)——"多一层,多一层钱"**(只满足功能层=只能拼低价红海;星巴克 38 元=功能+情感+社交)。**任务类两公式**:①`任务类 = 宽类 + 任务场景约束`(只选强相关窄品)②`任务类 = 宽类(商品集) + 窄品(选择约束) + 任务(情境+社会)`;功能层=补偿性的"货架数学题",情感/社会层=非补偿性(一票否决)的"人心选择题"。**⚠️ 卖任务 ≠ 卖体验**(体验是任务完成后的附加结果,先治好感冒才谈体验)。这是 §3.1C JTBD 段的理论骨架:诊断从"这块货架在替顾客完成哪个任务、补齐了哪几层"起,而非"我摆得好不好看"。
- **KANO 三型需求 + 价值层↔需求层映射(逐字批24回填)**:需求分三型——**基本型(="既要"·缺失则极度不满、有了不额外加分:收银能结账/生鲜要新鲜)/期望型(="也要"·满足越多越满意·竞争主战场:更全/更便宜/更快)/魅力型(="还要"·期望之外的惊喜·带来忠诚:免费包装/限量联名)**。落选品配置:基础价值品→**引流品**(鸡蛋米水面包·成本价微利·**缺则关联客流齐降·须无短板全覆盖**);功能/情感价值品→核心/贡献品(PB/进口);社会价值品→形象品。**需求会下沉**:期望型被满足后降为新基本型(顾客永不被完全满足→品类须持续创新)。**PB 开发=瞄准期望型空白**(NB太贵+白牌品质差→高品质合理价 PB)。这给 JTBD 的"任务"配上**分型尺**:诊断时先问"这个品类的基本型有没有短板(致命)→ 期望型够不够强(主战场)→ 有没有一个魅力型抓手(差异化)"。
- **关系思维(农耕vs狩猎)**:社区超市做"顾客生活任务协作者",非"商品搬运工";交易结束关系才开始(接复购/小票客群)。
  - **卖货 vs 任务关系链 = 狩猎 vs 农耕 · 价值交换四步 + 三种交换(逐字批36-38回填·零售老刘·六哥2026-06-26「先回填」签)**:卖货思维=**狩猎**(一次性成交·"进什么货怎么卖快"→2-3 年关门怪电商),任务思维=**农耕**(长期关系/复购·"顾客买了生活会变什么"→开新店、老顾客主动介绍)。价值交换=零售顶层艺术(用我所长换你所需、创造非算计)。**践行四步**:①重新定义你的"类/商品"(不问"我卖什么",问"帮顾客完成什么任务",任务类与小类建一对多)②找到愿意交换的顾客(共同痛点/渴望/价值观三圈交集)③建"让交换变容易"的机制(降首次门槛·超预期交付·一次性→可持续会员社群)④把顾客变成"宣传员"(NPS·价值清晰/显著/体面)。**三种交换:认知交换(用解决方案换顾客决策时间)/情绪交换(用仪式感共鸣换情感认同·价格非首要)/时间交换(替顾客做到极致换其时间成本解放与溢价)**——外加信任交换(坦诚承诺/试吃/无条件退款替顾客承担试错成本→换信任成常客)。本质:把"卖"字从零售字典删掉,利润=为顾客创造足够价值后的自然回响。与上行"关系≠关联""农耕vs狩猎"同源,此处补"怎么换"的可操作四步。
  - **零售本质=维护多元关系 / 关系≠关联(逐字批26回填·哲学根)**:零售经营本质=**构建和维护正向、良性的多元性关系**(与顾客/供应商/供应链/内部组织);零售是"生命体"(诞生→成长→成熟→衰退),关系=其"经络",管理者=园丁/管家(非冰冷耕作员)。**关系 ≠ 关联**:关系决定关联的价值方向,关联(关联陈列/捆绑促销/购物篮Apriori"啤酒尿布")=关系在运营层的量化表达与落地抓手。**铁律:先有关系再有关联(关系包含关联)**=先有 Item 需求再有 SKU 选择=陈列先定 Item 需求(落位)再配 SKU。由此**品类管理可重述为:以单品为单元、以生命性关系为中心**——选品=在已有单品关系网上加"能发生关联"的新节点("选品选不对,后续全白费");**杂乱陈列 vs 关联陈列的销售差异 = 关系价值**。这是 [[品类管理]]"以需求为中心"的更深一层:需求的本质是关系。
  - **服务价值链 + 信任=可量化资产(逐字批21/24回填)**:零售两核心=**商品价值链(效率/成本·物理属性)+ 服务价值链(关系/个性/体验·心智)**,后者常被忽略且缺结构体系=长期主义战略。其终点=**"信任"**:信任=新一代零售的核心价值/可量化"硬通货"(顾客"闭眼入"),`确定性价值 = 每个顾客的信任`。**两个度量:毛利额占比(信任产出多少)+ PI 值/顾客喜欢程度(信任厚度/长期性)**。四大信任范式=Costco 会员信任(稳定)/奥乐齐低价信任(好品质够低价矛盾体平衡)/山姆精选信任(90% 续费)/胖东来真诚信任。三大根本问题→共同指向"确定性":**①有没有我要的 ②是否货真价实 ③真的还是假的**。**信任崩塌=初心失守**(几年建立、几分钟废掉:背离最初承诺)。社区店落地=把"善良真诚"拆成可感知、可验证、可持续的确定性输出(承诺与交付一致)。优先中:作关系/服务维的总纲级概念,暂不入诊断硬指标。
  - **信任成本等式 + 任务三层↔三种信任 + 信任价值=NPS + 对抗诅咒三问(逐字批36-38回填·零售老刘·六哥2026-06-26「先回填」签)**:**关键等式——你越想绕开"信任"这个高成本(长期主义)的构建,就越被迫陷入功能任务层(暂时毛利)的无休止极致低价/高效率竞争**(成本最终最高昂、利润无限趋薄);反之投情感/社会层成本=积累"信任资产"(降未来交易成本、创溢价)。**任务三层对应三种信任**:功能层→**契约信任**(履约·动销率/评分)/情感层→**关系信任**(非促销复购率/响应率/好评)/社会层→**认同信任**(NPS/口碑/社群活跃);可用强势层弥补劣势层。**把"信任价值"变核心经营指标**(别只盯销售额毛利),最高级度量=**NPS 净推荐值**(顾客忍不住主动告诉别人)——与上行"毛利额占比 + PI 值"两度量互补,NPS 是信任长期性的对外验证口。**对抗"管理者诅咒"三问(自检)**:①这个动作让顾客更省时还是更费解(功能)②让顾客被尊重还是被当麻烦(情感)③顾客会怎么向朋友描述——"没办法它便宜"还是"我喜欢这里、很懂我"(社会)。**治理侧互指**:"零售管理者的诅咒"(把工具当员工/把功能当任务·知识诅咒的零售变体)完整机制由治理页 [[零售管理者的诅咒]] 承接,此处只取与品类/任务/信任价值直接相关的三问与映射,二者 wikilink 互指、不重复承载。
    - **科斯交易费用论 = 信任成本的经济学理论根(逐字批42回填·零售老刘《数据分析架构系列》04-05·六哥2026-06-26「先回填」签)**:**结论先行——上一行"信任成本等式"不是孤立的零售直觉,它的经济学根是科斯交易费用论:信任成本本质=交易费用(交易成本)的一种**。**① 定义**:交易费用由**科斯(Coase, 1937)** 提出 = **价值获取成本**(获得顾客价值所付出的成本),存在于零售价值链(制造商→零售商→消费者)的商品流传环节,且**双向**(买方卖方都付)。三种构成——**搜寻费用**(顾客找一家又好又近又便宜的店)/**谈判费用**(讲价、询问)/**执行费用**(称重、验货、完成交易);本质=买卖双方花费的**时间总和 + 信息成本**(非货币支出,故常被忽略却最贵)。**② 竞争焦点**:同质化时代,商品本身拉不开差距,**竞争很大程度上体现在"交易费用的经营"上**——谁让顾客更省时省心,谁赢。**③ 零售核心规律(通用分析框架·醒目)**:`宏观经济发展(GDP)带来新消费需求 → 新技术实施降低交易费用 → 交易费用降到极限 → 形成新商业形态`。两次业态变迁印证:大型超市(一站购齐降搜寻费用)、线上电商(搜索/比价/到家三费用同时压到极低)。**用法**:这是判断"某新技术/新业态该不该上(直播、社群团购、即时零售、AI 选品…)"的**通用尺**——只问一句:它把搜寻/谈判/执行哪一项交易费用降到了"新极限"?降不动的=噱头,真降的=可能催生新形态。**衔接信任成本等式**:上一行"投情感/社会层成本 = 降未来交易费用、创溢价"正是此论的微观映射——**信任资产 = 把三费用一次性降到极低的存量**(顾客"闭眼入"=搜寻费用≈0、"货真价实"=执行/验货费用≈0、"很懂我"=谈判费用≈0);所以绕开信任去拼极致低价,本质是放弃了"用信任永久压低交易费用"这条护城河,只能在功能层反复支付高昂的当期交易费用。
- **目标侧(品类指标分解)**:企业目标(销售/毛利)→ 按品类树分解到 L2/L3 → 部门承担;诊断现状之上加"该品类目标 vs 现状缺口"(forward-looking)。销售预测=目标系数×需求变化趋势。
  - **经营公式骨架(逐字批21回填)**:**GMV − TCO ≈ Gross Profit(总毛利)**,其中 **GMV=目标(过程,全渠道交易总额)/ TCO=结果(全部成本)**;系统化供应链 = 一个需求计划 + 一个成本计划。把"目标侧"从单看销售额升级为"GMV/TCO 双计划"——目标不只是冲交易额,还要同时压总成本,否则 GMV 涨而毛利不涨(注意:GMV/TCO/效率构成"不成立三角",三者不可同时极致)。优先低,作目标分解的概念骨架,数据未到位前不硬算。
- **品类四大战术 4P(逐字批13回填)**:**选品 / 定价 / 陈列 / 促销**(麦肯锡4P,零售一切基本问题)。其中**选品=商品力中影响力最大、是所有业务的起点**(选品错→补货/订单/定价/促销/陈列/库存全被影响→加速熵增);木桶短板比优势更危险。**先品类战略,再4P战术**(避免"战术勤快掩盖战略懒惰")。
  - **木桶短板的数学根:经营效果=价值×效率(乘法非加法·逐字批24回填)**:`价值+效率=维度+度量`,经营效果是各维度效率的**连乘积**,不是相加。**5 个维度各只做到 0.95 → 0.95⁵ = 0.78**(不是相加的 4.75);效率单维最大值=1。这就是"每步差一点→结果无限接近 0"的数学根:**短板是乘上去的,一个 0.5 维度直接把整体砍半**。诊断推论:① 4P/各维不能"平均都还行",一个致命短板(如选品错)会乘垮全盘;② 资源应先补最低分维度(把 0.5 拉到 0.8 的收益 > 把 0.9 拉到 0.95)。与 §3.1A "先维度后度量" 同源。

### 3.1D 生鲜按生活场景选品（生鲜专题 · 逐字批45回填·六哥 2026-06-26「先回填」签）

> 来源：零售老刘《商品库存管理分析建模》4 逐字精读。（逐字批45回填·零售老刘《商品库存管理分析建模》4·六哥2026-06-26「先回填」签）
> **定位**：本节是 §3.1C JTBD 任务单元在**生鲜**上的落地——**场景 = 任务的生鲜表达**。六哥深耕 500–2000㎡ 社区生鲜超市，此节直接服务该定位，并直击 SaaS 第三阻碍「生鲜验证」。

**结论先行：生鲜选品不能按常规销售/综合值/ABC 排名排队砍，要按"生活场景"组货——"烟火味"。仅基于孤立单品考虑，往往没效果。**（逐字批45回填·零售老刘《商品库存管理分析建模》4·六哥2026-06-26「先回填」签）

1. **核心命题——按场景组货而非按单品排名**：生鲜选品**不能按常规销售额 / 综合值 / ABC 排名**机械排队、低分即砍。要围绕**生活场景**组货，问的是"烟火味"四问：① 围绕**一顿饭**需要哪些商品；② 一个**家庭厨房**需要哪些东西；③ 一张**餐桌**主要有哪些单品；④ **打个火锅**需要什么配料。**只盯单品做加减，往往没效果**——因为顾客雇佣的是"做一顿饭"这个任务，不是孤立单品（直接呼应 §3.1C JTBD：顾客要的是"洞"不是"钻头"，场景=任务的生鲜表达）。
2. **生鲜「五品」重定义（老刘原称，逐字批45列四类）**：**鲜活品 / 加工品 / 熟制品 / 附属品**——这是比"按 SKU 看"更贴生鲜本质的分类，**可据此做布局设计与陈列**（按品的形态与作业属性分区，而非只按品类树拆）。
3. **生鲜 7 特点（决定生鲜管理与常规快消不同）**：① **价格变化大、调价频繁**；② **季节性强、陈列调整快**；③ **分级评级专业**（生鲜要分级，非一品一价）；④ **采购流程变化大、不确定多**；⑤ **保质期短 = 日配、需每天精管**；⑥ **高粘性 = 差异化商品**（做好能增顾客信任与价值）；⑦ **订货周期短、频率高**。
4. **定位——生鲜是社区生鲜超市的护城河**：生鲜 = **高粘性 + 差异化**，是 500–2000㎡ 社区生鲜超市区别于电商/便利店的**护城河**。本节是**生鲜诊断方法论的起点**：诊断不从"这条生鲜小类销额排第几"起，而从"这块生鲜货架在替顾客完成哪个生活场景、补齐了哪几道菜"起——与 §3.1C 的 JTBD / 任务类**同源打通**（场景=任务的生鲜表达，任务三层的功能/情感/社会同样适用：新鲜=功能入场券，本地时令/会做菜的信任=情感与社会层）。
5. **⭐⭐ 定品 = 场景营造 + 集群完整性（深化"按生活场景选品"·逐字批46回填）**：选品（定品）**不能只靠末位淘汰**——销售排名太粗糙，砍掉的可能正是别人离不开的那一环。两步走，**先底线后抓手**：**① 先搭"商品完整性"= 集群概念（底线）**——关联商品不能缺，**"店里有面粉却没酵母 = 和顾客过不去"**：集群里少一环，整个场景任务就完不成（呼应 §3.1C JTBD：顾客雇的是"做一顿饭"，缺了酵母，面粉这单生意也一起黄）。**② 再做"场景营造"（组货抓手）**——围绕早市 / 厨房 / 烹调等生活场景组货，可把**分属不同结构类的商品按场景自定义功能性类别**：例「葱姜蒜韭」分属不同小类，按"调味蔬菜"自定义类放在一起，顾客一次买完。**一句话：集群完整性 = 底线（不能缺），场景自定义类 = 抓手（组得巧）。** 这是对本节"按生活场景组货"的可操作深化——场景不止决定"选什么"，还包括"怎么把选出来的货按场景重新归类、摆在一起"。（逐字批46回填·零售老刘《商品库存管理分析建模》12·15·六哥2026-06-26「先回填」签）

### 3.1E 以顾客为中心的精细化零售三步（客群方法论 · 逐字批50回填·六哥 2026-06-26「先回填」签）

> 来源：零售老刘《商品库存管理分析建模》05 逐字精读。见 [[批50_以顾客为中心假A类陈列时空系数_v0.1]]。（逐字批50回填·零售老刘《分析建模》05·六哥2026-06-26「先回填」签）
> **补强来源（逐字批51·六哥 2026-06-26「先回填」签）**：零售老刘《分析建模》09「卖场经营指数」逐字精读——顾客价值指数（RMF / PCA / 矩阵法 / 负价值指数）+ 7-11 小票客群采集法，深化第三步"价值量化"侧（落点见本节第 4 条）。（逐字批51回填·零售老刘《分析建模》09卖场经营指数·六哥2026-06-26「先回填」签）
> **补强来源（逐字批52·六哥 2026-06-26「先回填」签）**：零售老刘《分析建模》12「顾客需求与类别属性」逐字精读——⭐⭐⭐ 类别 RFM（绕开会员缺失，把顾客 RFM 转化为小类 RFM）+ 客流 × 客单 9 宫格，给第三步"价值量化"配上**无会员系统也能跑**的小类价值分级工具（落点见本节第 5 条）。（逐字批52回填·零售老刘《分析建模》12顾客需求与类别属性·六哥2026-06-26「先回填」签）
> **补强来源（逐字批62·六哥 2026-06-26「先回填」签）**：零售老刘《零售经营终极问题探索》逐字精读——⭐⭐⭐ NPS 净推荐值 / 终极问题（"是否愿意推荐本企业"→量化客户忠诚度），作客群方法论（乃至全体系）的 **capstone 收束**：客户需求=一切零售问题根源、NPS=口碑价值量化、陈列终极定义、NPS 标准线 20%–85% 作客群健康标尺（落点见本节第 6 条）。（逐字批62回填·零售老刘《零售经营终极问题探索》·六哥2026-06-26「先回填」签）
> **定位**：本节是 §3.1C JTBD（需求侧「为什么买」）在**客群维度**的落地——把"以需求为中心"补上"哪些顾客的需求值得优先满足"这一层。与 §3.1A「需求论 > 目标顾客论」并不矛盾：需求论解决"按需求而非画像组货"，本节解决"按**带利润的价值顾客**的需求来排优先级"，二者互补不互斥。呼应客群分析（[[M5_顾客需求簇]]）。

**结论先行：粗犷经营→精细化经营 = 以顾客为中心地更清晰管理。关键不是服务好所有顾客，而是维系**带利润的价值顾客**(含潜在)，从**商品主导型企业 → 顾客需求型企业**。顾客需求 = 买「什么」+「为什么」买（两者结合才出高绩效，只看"买什么"=有数据无洞察）。**（逐字批50回填·零售老刘《分析建模》05·六哥2026-06-26「先回填」签）

1. **核心理念——经营黑洞 = 对顾客不了解**：经营两大黑洞之一 = **不懂顾客**——流失了多少顾客 / 流失的是哪类 / 为何流失，全是黑洞 → 只能拍脑袋决策。精细化的反面不是"不努力"，而是"努力用错了对象"：把资源平均撒给所有顾客，等于没维系真正贡献利润的那部分人。
2. **以顾客为中心三步（客群方法论主轴）**：
   - **① 找出真正的价值顾客**：商圈研究 + 从**销售额与边际利润**两维评估每一类顾客的潜在价值（不只看销售额，要看谁带利润）。⚠️ **99% 的企业不做商圈分析**——这正是社区超市的低垂果实（接 §4 本土化：主力客群半径 + 社区购买任务）。
   - **② 以价值顾客需求为依据的改变**：**一个小类 = 一类顾客的一个需求**（同源 §3.1A "一个小类=一个需求"）；**至少每半年重新调整一次小类**（顾客侧的结构维护纪律）。每次调整前过**两问判据**：**（a) 为的是不是目标价值顾客？（b) 是否让他们更愿意来、买得更多？** 两问都"否"=别动。
   - **③ 目标顾客价值量化**：用**小类销售占比 / 动销率 / 客流 / 客单**等行为数据**反推**顾客价值（行为数据 > 老板感觉）。纪律金句：**"赶在顾客之前改变"**——半年不调、或从不调整价值类别 = 危险信号。
3. **衔接与互指（打通已有锚点）**：
   - **↔ §3.1C JTBD**：JTBD 答"任务 = 为什么买"，本节答"哪些顾客的任务优先满足"——任务三层（功能/情感/社会）叠加价值顾客筛选，先锁定带利润客群，再补齐他们的任务三层。
   - **↔ §3.1D 生鲜场景**：生鲜按生活场景组货时，场景的取舍依据正是"目标价值顾客的厨房/餐桌"，而非全客群最大公约数。
   - **↔ 批49「每半年梳理结构」**：批49 给的是**结构/商品侧**的半年维护依据，本节"至少每半年调小类"给的是**顾客侧**依据——同一个半年节拍，双侧对齐（见 [[批49_品类结构维护ABC实践参数货位商品状态_v0.1]]）。
   - **两问判据 = 诊断自检清单**：任何选品/汰换/陈列调改动作，落手前都先答这两问；答不出"目标价值顾客"的动作，多半是自我感动（同源 §3.1C「顾客不在乎你，只在乎他的任务」）。
4. **⭐⭐⭐ 顾客价值指数 = 量化「价值顾客」的标尺（深化第三步"价值量化"侧）**（逐字批51回填·零售老刘《分析建模》09卖场经营指数·六哥2026-06-26「先回填」签）：**结论先行——第三步"目标顾客价值量化"不能停在"看销售占比/动销/客流/客单"，要进一步算出每个顾客的「价值指数」，按价值分层施策——这才是精细化运营的真正前提。**
   - **顾客价值指数 = 衡量从历史到当前每个顾客贡献的收益**（生命周期价值，含对未来贡献的预估）。**精细化运营的前提 = 不同价值的顾客采取不同策略**（高价值重维系、低价值控成本），而非一视同仁平均撒资源（呼应本节核心理念"努力别用错对象"）。两种算法：**① RMF 模型**——R 最近消费时期 / M 累计总金额 / F 消费频次，三维交叉划分顾客群体（社区超市可落地的轻量算法）；**② 主成分分析 PCA**（多指标降维，数据底座成熟后用）。
   - **矩阵法（价值指数 × 流失指数 → 四象限）**：**高价值 × 高流失 = 积极唤回**（最该抢救，掉一个少一块利润）；**低价值 × 高流失 = 平衡成本**（别为留住低价值客投入过头）。这给第三步"赶在顾客之前改变"配上可操作的优先级排序。
   - **⭐ 负价值指数（最易被忽略的诊断切口）**：价值指数有正也有负——顾客**不喜欢来 / 在卖场里找不到东西 / 不知道该买哪个 / 找不到想买的 / 服务态度差**，这些都是"负价值"。经营的努力 = **把负价值转化为正价值（产生购买）**。所以**价值指数的本质 = 以顾客为中心**：每一个负向触点都是一个待修复的诊断切口（接 §3.1C「顾客不在乎你，只在乎他的任务」——你没接住他的任务，就在制造负价值）。
   - **⭐ 7-11 小票客群案例（直接可落地的低成本采集法）**：7-11 收银员在结账前**看顾客一眼 → 按一个收银键**（少 / 中 / 青 / 壮 / 老，记录年龄段）。于是**一张小票 = 一个顾客 → 背后挂一个用户标签维度**（性别 / 年龄段 × 时段 → 反推客群结构）。**10 个按键，折射的是整套精细化管理**。这是不依赖会员系统、不依赖人脸识别的**低成本客群数据采集法**，可直接落花厅坊等社区超市——收银台加一道"看一眼按一键"，一周就能积出"客群 × 时段"的真实结构（补上本节第①步"找价值顾客"缺的原始客群数据入口）。
   - **衔接**：本条深化 §3.1E 第三步"目标顾客价值量化"——从"看行为数据"升级到"算价值指数 + 分层施策"；7-11 小票客群呼应 [[M5_顾客需求簇]] 的小票客群方法论，是客群数据的采集源头。
5. **⭐⭐⭐ 类别 RFM + 客流客单 9 宫格 = 绕开会员缺失的小类价值分级（深化第三步"价值量化"侧）**（逐字批52回填·零售老刘《分析建模》12顾客需求与类别属性·六哥2026-06-26「先回填」签）：**结论先行——第 4 条的顾客价值指数依赖会员数据，可超市的会员体系本就缺失、对顾客分析意义有限；真正的破局点是"顾客与商品的关联"——顾客买的是商品，一个小类 = 一类顾客的一个需求，所以把"顾客 RFM"整体转化为"小类 RFM（类别 RFM）"，无会员系统也能给每个小类做价值分级。**
   - **① 换思维：顾客 RFM → 小类 RFM（类别 RFM）**：RFM（R 最近消费 / F 频次 / M 金额）原本是给会员顾客分级的，超市会员缺失 → 直接对顾客算 RFM 意义不大。但**顾客与商品的关联才最有意义**（顾客买的就是商品）。既然一个小类对应一类顾客的一个需求，就把三个维度平移到小类身上：**R = 该小类最后一次被购买距今天数（越近越好）· F = 客流（引用客流数据，谁来买的人多）· M = 销售额**。**按平均值计分**（某小类该维度 > 全店平均 = 1，否则 = 0），三个维度 0/1 组合 → **8 种小类类别属性**（借用客户 RFM 的 8 分类框架，把"重要价值 / 重要保持 / 重要发展 / 一般价值……"从客户层平移到小类层）。一句话：**会员缺失不是终点，把分级对象从"人"换成"小类"，RFM 照样能跑。**
   - **② 诊断切口：小类客流 × RFM 分级的错位对比**：把每个小类的客流分级与它的类别 RFM 属性两两对照，找**该匹配却没匹配**的异常。例：一个本该是"重要价值类别"的小类，理想上应对应"良好"的经营状态，实际却落进"**促销过度类别**"（靠促销硬撑出来的 R/M）——**合理吗？为什么会这样？该怎么改？** 这三问就是诊断的落手处：错位即问题，问题即调改清单（同源本节"两问判据 = 诊断自检清单"）。
   - **③ 客流 × 客单 9 宫格（两期对比驱动业务动作）**：取**两期数据**，分别看每个小类的**客流升 / 平 / 降**与**客单升 / 平 / 降**，交叉 → **9 种可能、4 种主要类型**（客流客单双升 = 健康放大 / 双降 = 危险收缩 / 客流升客单降 = 引流没接住转化 / 客流降客单升 = 靠少数人高客单撑、根基在流失）。关键是给 9 宫格**加一列「分类判断」（IF 规则）**——把每个落格直接翻译成"该做什么"（补客流 / 提客单 / 守 / 撤），**用分类驱动业务动作，而非停在描述现象**（呼应 §10.1 So-What：每条发现必挂动作）。
   - **衔接**：本条与第 4 条互补——第 4 条（价值指数）解决"有会员时怎么算顾客价值"，本条（类别 RFM）解决"没会员时怎么用小类反推价值"，两条一起补全第三步"目标顾客价值量化"的双路径；**类别 RFM 的 M 维呼应批48「毛利贡献值」、整体价值分级呼应批51「价值指数」**（都是给"价值"找可量化标尺）；**与小类内 ABC 互补**——ABC 看的是销售集中度（防误杀见 §3.3B），类别 RFM 看的是价值新鲜度（R 维 = 最近是否还在被买），两把尺子一个测"大不大"、一个测"新不新鲜"，叠用才不误判（接 §3.4 品类评估的多维健康度）。
6. **⭐⭐⭐ NPS 净推荐值 / 终极问题 = 客群方法论（乃至全体系）的收束与升华**（逐字批62回填·零售老刘《零售经营终极问题探索》·六哥2026-06-26「先回填」签）：**结论先行——前 5 条把"以顾客为中心"层层拆细（找价值顾客→按需求改变→价值量化→价值指数→类别 RFM），但所有这些工具最终都收敛到一个问题、一个数字：「你是否愿意把本企业推荐给别人？」=「终极问题」→ 量化出 NPS（净推荐值）。客户需求才是一切零售问题的根源和基础，NPS 是这个根源唯一够格的对外标尺——它既是本节客群方法论的收束，也是 §3.1C JTBD 乃至整套方法论的总纲升华。**
   - **① 终极问题 → NPS（《终极问题》赖克霍德）**：终极问题 = **"你是否愿意把本企业推荐给别人？"**——一个问题，量化出 **NPS 净推荐值**，衡量客户忠诚度。客户据此分三类：**推荐型 / 消极满意型 / 贬低型**（NPS = 推荐型占比 − 贬低型占比）。它的厉害在于：把"顾客到底买不买账"这件最虚的事，压成一个可比、可追踪的数。
   - **② ⭐核心命题：客户需求 = 一切零售问题的根源和基础**：很多经营决策出问题，根上都是**缺了"客户需求"这个基础**（顾客效用 / 口碑价值没想清楚就动手）。**NPS = 客户口碑价值的量化**——而口碑价值是零售里**最难量化**的一件东西，终极问题给了它一把尺。这条直接呼应 §3.1C JTBD「顾客不在乎你，只在乎他的任务」：NPS 高 = 你接住了他的任务、他愿意替你背书；NPS 低 = 你在制造负价值（接第 4 条「负价值指数」）。
   - **③ 找根源三问（5W1H 的第一个 W）**：动手前先答三问——**① 你知道客户是谁吗？（Who = 5W1H 第一个 W）② 了解他的需求吗？③ 应该提供什么商品 / 服务？** 三问答不出，后面所有选品 / 陈列 / 促销都是无根之木（同源本节「两问判据 = 诊断自检清单」，此处把判据再往根上推一层到"客户是谁 + 要什么"）。
   - **④ ⭐陈列的终极定义（基于终极问题反推）**：陈列 = **基于顾客「易看到、易选择、易拿取」的需要，处理商品在卖场的位置和数量**。推论极锋利：**"陈列没有死原则"才是最大的标准**——例如"先进先出"在卖场就**违反终极问题**（把临期品前置 = 顾客不愿买，硬塞 = 损口碑 = 拉低 NPS）。一切陈列规则都要回到终极问题前接受审判：它让顾客更愿意推荐，还是更不愿意？
   - **⑤ ⭐NPS 标尺（可落地的客群健康线）**：**胖东来 NPS ≈ 80%**；**超市 NPS 标准值线 = 20%–85%，中值 45% 以下 = 缺乏"创造好利润、促进真成长"的条件**——低 NPS 之下的利润增加 = **暂时的、非真成长**（靠促销 / 压价 / 透支口碑挤出来的，迟早还回去）。这给本节"客群健康"配了一条**可直接量化的标尺**：NPS 落 20%–85% 区间看健康度，跌破 45% = 红灯（增长不真，先修口碑再谈扩张）。
   - **衔接（capstone 收束）**：NPS = 客户口碑的量化，与前文三把"价值尺"同源互指——**呼应第 4 条批51「价值指数」（顾客价值的量化）、批06「信任价值 = NPS」（信任的对外验证口）、第 5 条批52「类别 RFM」（无会员时的小类价值）**：四者都在回答"价值 / 口碑怎么量化"，NPS 是它们最顶层、最对外的那一个。**客户需求 = 根源 = §3.1C JTBD 的总纲收束**（JTBD 答"为什么买"，NPS 答"买完愿不愿替你说话"，一前一后闭合需求闭环）；**NPS 标准线 20%–85% 可作客群健康标尺**，与本节第①步"找价值顾客"、第④条"负价值指数"一起，构成"以顾客为中心"从入口（是谁）到出口（愿不愿推荐）的完整度量链。

### 3.1B 品类树 3+1 结构与 L4 启用层

> 来源：v3.2 `02A_3+1层级架构方案` + `02_品类体系`（打捞 SALVAGE-BACKLOG #1 + #6）。治"选品过度拆分 + 品类表乱分支"两个现役老问题。

**3+1 结构**：三层默认 + 一层专项。
- **L1 经营板块层**：大经营盘、哪些板块占资源。
- **L2 经营组织层**：中类组合、组内失衡。
- **L3 经营分析与动作层**：结构问题、动作落点（**普通类停在 L3**）。
- **+ L4 复杂类决策层**：只给复杂类。**L4 不是"更细分类"，而是"更细决策"**（决策路径/价格带/场景/陈列/活动的内部差异）。

**为什么 3+1**：只做三层 → 复杂类分析过粗、价格带与路径混判；全做四层 → 树太深、管理成本高、复杂度失控。一句话：**让普通类保持轻，让复杂类有地方做深。**

**L4 启用判据（满足任一才进，#1 核心）：**

| # | 进 L4 条件 |
|---|-----------|
| 1 | 顾客决策路径明显不止一条 |
| 2 | 价格带差异巨大，混在一起判断会失真 |
| 3 | 场景差异明显 |
| 4 | 陈列表达差异明显 |
| 5 | 活动激活方式差异明显 |

> **铁律**：不是商品多就进 L4，而是"**决策逻辑明显分叉**"才进 L4。（如 G4-A 冲调多品牌多价格带 → 适合 L4 分层算，避免 IP-Chart 形状失真。）

**主树 vs 经营属性线（#6，治"乱分支"）**：品类树（L1-L4）是**分类骨架**；选品/定价/陈列/活动四条经营属性 + 地域风土时令，用**经营属性线挂载**，**不混进主树拆分**。
> **外部印证(零售老刘逐字批06·六哥 2026-06-25 签)**:商品两大属性=**固有属性**(品牌/功能/产地/规格/口味·进主树)+ **经营属性**(ABC/价格/促销/会员/库存·进属性线)。我的"主树vs属性线"= 老刘"固有属性vs经营属性"同构。"真商品群判据=基于顾客需求+符合经营角色定位"(说不清用途的组合≠商品群)。
- ✅ 正确：一个"零食"L3，下挂"散称/预包装""家庭装/儿童装""节庆"等**经营标签**。
- ❌ 错误：把"预包装·零食"和"散称·零食"拆成两条主树分支（分类与经营维度混用 → 品类表乱）。

> **品牌=经营属性线上的战略单元(零售老刘逐字批22回填·BRAND 原理)**:**四大品牌类型 = NB(全国性/一线) / LB(区域性/地方) / SB(限定性/店内独有) / PB(自有/白牌)**。BRAND 不是固有属性里的"牌子名",而是**零售商主导的多品牌生态组合**——每个品牌在品类里担一个战略角色(引流/利润/形象/价格锚)。**BRAND 配置图**=按品牌角色铺面,目标=最大化单位面积产出 + 强化定位/忠诚度。落地:品牌作为**经营属性线的一条**(与 ABC/价格/促销/会员/库存并列),不进主树拆分;诊断时可看"一个 L3 内 NB/LB/SB/PB 的密度配比是否合理"(印证 [[多品类压力测试报告_v2.0]] 品牌密度型)。

### 3.2 品类战略

确定品类在门店整体经营中的定位、发展方向和资源分配优先级。

- **品类定位：** 这个品类是门店的强项、弱项还是空白
- **发展方向：** 扩张、维持、收缩还是退出
- **资源投入优先级：** 相对于其他品类，这个品类值得投入多少陈列、活动和采购资源

### 3.3 品类角色

根据门店经营目标，为每个品类分配角色，驱动资源配置决策。

| 角色类型 | 核心作用 | 典型特征 |
|----------|----------|----------|
| 目标性品类 | 吸引主要客流，建立门店形象 | 高频、高关注、客流驱动 |
| 常规性品类 | 满足日常购买需求，维持销售基盘 | 稳定、高复购 |
| 季节性/便利性品类 | 季节带动、一站式满足 | 季节波动明显或补充需求 |
| 形象品类 | 建立门店品质感知 | 顾客感知强，未必销量高 |
| 流量品类 | 吸引客流入店 | 价格敏感，竞争激烈 |
| 利润品类 | 贡献毛利额 | 毛利率高，动销稳定 |

### 3.3B 商品角色闸与防误杀铁律（SKU 层 · 治"汰换误杀"）

> §3.3 是**品类层**角色；本节是**商品（SKU）层**角色——两层独立，**缺第二层会在汰换时误杀引流/形象/目的品**。这是汰换决策前必过的闸。
> 来源：经营型 Taxonomy 升级任务书 §5.4 经营角色表（五角色词表 + GBA 防误杀双保险）+ 品类体系综合版 v1.2。
> 📖 **完整机制（SSOT）**：[[SKU角色层与目的品保护机制_v0.1]]（含 5 步决策流程 + ABC/长尾/M-DEC 对接）；本节为品类管理内的操作摘要。

**商品主角色（五选一，辅角色可空）：**

| 商品角色 | 作用 | 汰换态度 |
|----------|------|----------|
| 引流 | 带客流入店（价格敏感/高关注） | 🔒 防误杀：不得仅凭销量砍 |
| 形象 | 建品质/信任感知（未必高销） | 🔒 防误杀：不得仅凭销量砍 |
| 利润 | 贡献毛利额 | 看毛利额，非单看销量 |
| 补充 | 满足长尾/一站式需求 | 低销 + 无角色 → 优先汰换候选 |
| 季节 | 节令/节庆带动 | 按季节日历，非看常态销量 |

**防误杀铁律（沙埔大道双保险）：**

1. **GBA 锁定品类**（大湾区差异化锚点：凉果/陈皮/煲汤料/中式糕点/烧腊等）带**锁定标记**，进入**所有淘汰判断的防误杀通道**——默认**保留 + 浅库存**，汰换须**举证**（不得仅凭销量/动销排名）。
2. **Role × ABC 默认处置**：锁定品/引流/形象默认保留；仅"**补充角色 + C/D 档 + 无锚点**"才进汰换候选。
3. **汰换前必填**：每个待汰 SKU 先标商品角色 + 是否锁定；**未标角色不得汰换**。

> **实战接口（G4-A 冲调）**：81% 低销 SKU 汰换前先过本闸——冲调里的引流/形象款、煲汤冲饮类 GBA 锚点，即便低销也走防误杀，不一刀切。状态 `candidate`，待战役验证升 active。

### 3.4 品类评估

定期评估品类经营健康度，判断策略是否需要调整。

| 评估维度 | 评估内容 |
|----------|----------|
| 销售额 | 品类销售额及同比/环比趋势 |
| 毛利额 | 品类毛利贡献及毛利率水平 |
| 动销率 | 在架 SKU 中有销售记录的比例 |
| 库存周转 | 库存消化速度，判断积压风险 |
| 价格带覆盖 | 是否覆盖目标顾客主力消费区间 |
| 客群匹配度 | 品类顾客画像与门店主力客群的吻合程度 |

### 3.5 品类策略

根据品类角色和评估结果，制定具体的经营策略方向。

- **增长策略：** 扩大 SKU 宽度、加大陈列资源、引入新品带动
- **利润策略：** 优化价格带结构，引入高毛利 SKU，收缩低贡献单品
- **引流策略：** 强化价格竞争力，做促销活动，提升客流带动效应
- **防守策略：** 守住基本盘，防止客流被竞品截流，稳定核心 SKU
- **季节策略：** 提前备货，波段式陈列调整，节令商品强化

### 3.6 品类战术

把品类策略转化为可落地的具体商品动作。

- **商品组合：** 确定品类内的 SKU 结构（宽度×深度），哪些保留、引进、淘汰
- **价格带：** 确认价格区间覆盖，主力价格段清晰
- **陈列空间：** 分配货架面数、层位、端架和堆头资源
- **促销活动：** 制定品类促销节奏，与 52周MD 对齐
- **供应商协同：** 核心 SKU 的供货稳定性、账期、新品联合推广

### 3.7 品类执行

把品类战术落地到门店现场。

- **选品：** 按商品配置理论确定进店 SKU 名单
- **上架：** 按标准完成商品档案录入、条码匹配、价签制作
- **陈列：** 按 DisplayMap 完成货架陈列，符合品类陈列规则
- **补货：** 按动销节奏和库存水位执行补货，避免缺货和积压
- **活动落地：** 按活动方案完成堆头、端架、促销标签的现场执行

### 3.8 品类复盘

评估本周期品类执行结果，驱动下一轮策略调整。

- **目标达成：** 销售额、毛利额是否达到目标
- **策略偏差：** 本期策略方向是否正确，哪里出现偏差
- **单品贡献：** 哪些 SKU 贡献了销量/毛利，哪些拖了后腿
- **陈列效果：** 陈列是否符合规范，是否影响了动销
- **下一轮调整：** 明确下一周期的 SKU 调整、价格调整、陈列调整、活动计划

## 4. 社区超市品类管理本土化

社区超市品类管理不能照搬大型连锁的流程和标准，必须结合本地实际条件落地。

关键约束条件包括：门店面积、主力客群半径、社区购买任务、老板资源和决策风格、员工执行能力、供应链可及性。

**第一阶段优先解决的核心问题：**
- 品类边界不清：同一品类商品散落在不同区域，顾客找不到
- SKU 结构失衡：某些品类 SKU 过多积压，某些品类严重断货
- 价格带断层：缺失主力价格段，顾客购买意愿受阻
- 陈列资源错配：高贡献品类陈列面不足，低贡献 SKU 占据黄金位置
- 复盘机制缺失：没有定期品类评估动作，靠感觉做决策

乐易购花厅坊店可作为后续测试案例，但真实数据和结论待现场录入，不得编造。

## 5. 与商品配置理论的关系

品类管理和商品配置理论回答不同层次的问题，两者是战略与落地的关系：

- **品类管理** 回答：这个品类为什么存在？承担什么角色？应该如何发展？
- **商品配置理论** 回答：在具体门店、具体区域、具体时间，应配置哪些商品、多少 SKU、哪些价格带、哪些陈列资源？

品类管理是上游决策依据，商品配置理论是下游落地工具。

## 6. 与52周MD的关系

品类管理确定品类角色和策略方向，52周MD 把品类策略转化为年度、月度、周度的商品经营节奏。

季节性品类（如冷饮、暖饮、节令礼品）和活动性品类（如促销堆头商品）尤其需要通过 52周MD 提前规划备货节点、陈列切换时间和活动商品波段。

## 7. 与数据与系统能力底座的关系

数据与系统能力底座为品类管理提供数据支撑，主要体现在：

- 品类销售额与趋势分析
- 品类毛利贡献分析
- 动销率监控
- 库存周转诊断
- 价格带覆盖分析
- 陈列效果分析（结合 DisplayMap）
- 活动效果分析（区分自然动销与促销拉动）
- 品类复盘报告自动生成

**注意：** 系统和数据不能替代品类判断，只能提高品类判断的准确性、可追踪性和复盘效率。最终判断仍需依赖对顾客、门店、竞品和经营目标的综合理解。

## 8. 常见误区

- **误区1：** 把品类管理等同于商品分类——系统里的分类是商品档案管理，品类管理是以顾客购买任务为核心的战略决策。
- **误区2：** 只看销售额，不看毛利、库存、动销和客群任务——销量好不一定品类健康，需要多维度综合判断。
- **误区3：** 只做选品，不做陈列、价格带规划和复盘——选品只是品类管理的一个环节，割裂执行会导致策略失效。
- **误区4：** 照搬大连锁品类管理流程，忽视社区超市执行能力——复杂流程在5-10人的社区超市无法落地，必须简化和本土化。
- **误区5：** 把品类角色写在文件里，但不落实到 SKU、陈列、活动和复盘——品类角色只有转化为具体执行动作才有价值。
