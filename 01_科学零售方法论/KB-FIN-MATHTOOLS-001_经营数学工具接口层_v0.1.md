# 经营数学工具接口层：GMROI / TL-TH / TF函数 / Si值 / CTM 统一定义

> **本文件的唯一目的**：`商品配置理论.md` §20 待完善事项 #17 明确写过"建立 GMROI、TF函数、TL/TH、Si值、CTM 的正式定义文件"——这是库里自己挂了很久的欠账，不是新发明的模块。本文件把散落在 `商品配置理论.md`/`单品管理.md`/`数据与系统能力底座.md`/`KB-DX-MATH-001`/`KB-DX-FULLSTORE-001` 五处的同一批工具收拢成一份权威口径，避免同一个词在不同文件里有轻微不一致的公式表述。**不是从零造知识，是查重去重。**
>
> **反幻觉红线**：以下五个工具中，只有 GMROI 有跨文件一致、可直接使用的公式和基准值；其余四个在全库范围内都标注"待确认/待建立独立工具定义"。本文件如实保留这个差距，**不替 TF函数/Si值/CTM 杜撰计算公式**——杜撰比留空更危险，会被当成"已验证方法论"用到客户交付上。

## 1. GMROI（已成熟，可直接调用）

**定义（全库一致，三处独立定义互相印证）**：

> GMROI = 毛利额 ÷ 平均库存金额 = 毛利率 × 库存周转率

出处交叉验证：`商品配置理论.md` §21（毛利额÷平均库存金额）、`单品管理.md` 逐字批07回填（毛利率×周转率）、`KB-DX-MATH-001` RM-06"交差率"（毛利率×周转率，同一公式的诊断工具化命名）。

**社区店健康基准**：GMROI ≥ 2.5（`KB-DX-FULLSTORE-001` D5-3 评分锚点、`KB-ACTION-GUIDE-001` PB-D5-04 触发线，两处独立引用一致）。**⚠️ 出处未溯源**：2.5 这个数值本身查不到原始行业出处（不是零售学堂课程数字，两处引用文件均为 `status: draft`），目前只是库内自设的内部锚点，非外部权威基准——引用时按"内部经验值，待验证"对待，不要对外包装成"行业标准"。

**用途**：跨部门统一的"这个 SKU/品类值不值得占这块库存资金"判据。`单品管理.md` 明确定位它为"防既当球员又当裁判"的财务立项门槛。

**术语对齐提醒**：`KB-DX-MATH-001` 里把同一公式叫"交差率"（零售现场惯用语），本文件起 GMROI 是同一件事的国际通用叫法——**两个词不要同时在同一份对客文件里出现，选一个，建议对内用交差率（团队熟悉），对外/写标杆案例用 GMROI（行业通用，便于对标国际案例）**。

**可直接调用判断句**：某品类 GMROI < 2.5 → 检查是"毛利率低"还是"周转慢"拖累了它 → 毛利率低查价格带/促销结构，周转慢查动销/库存深度 → 分别转 `金额管理.md` §15/§16 或 `KB-DX-MATH-001` IM-04 处理。

**张力/失效条件**：GMROI 是库存投资回报率，不是唯一裁决指标。目的品/引流品（低毛利高客流）GMROI 天然低，不能套用 2.5 基准直接判死刑——须先过 `SKU角色层与目的品保护机制` 的角色豁免，再看 GMROI。

## 2. TL / TH（现金流安全边界，候选阶段——框架已立，阈值未定）

**定义**（`商品配置理论.md` §21 原文）：
- TL（理论最小周转天数）：当前商品结构在理想情况下能达到的最快周转速度
- TH（最大周转天数/供应商账期）：企业对供应商的结算账期，即商品必须变现的最晚天数
- 判断：TL/TH < 1 → 现金流较安全；TL/TH ≥ 1 → 可能存在现金流风险

**本文件新增的唯一实质内容**：把 §21 的框架和 `KB-DX-MATH-001` RM-06"综合周转天数必须 < 结算周期（通常30-45天）"这条独立铁律对齐——**两者是同一件事的两种表述**：RM-06 给了行业通用的账期锚点（30-45天），TL/TH 给了"理论最小周转天数"这个更精细的分子。合并读法：

> TL/TH < 1 的更可操作版本 = 当前实际周转天数 < 30-45天结算周期锚点（先用这条粗筛，TL 的"理论最小"精算留待有真实供应商账期数据后再做）

**fact_layer: inferred，尚未被任何真实门店数据验证**。`商品配置理论.md` 原文明确写"v0.1 不写入固定公式和阈值"——本文件同样不新造阈值，只做框架合并。

**下一步验证路径（不在本轮做，留痕）**：拿到客户真实供应商账期数据后，用"实际周转天数 vs 账期"跑一次，才能把 TL 从"理论"变成"本店实测"。

## 3. TF函数（已采纳定义）/ Si值 / CTM（仍未定义，本文件明确不杜撰）

**TF函数 — 状态更新（2026-07-10 六哥裁定采纳）**：此前本节把 TF函数与 GMROI/TL-TH 一并列为"未定义、优先怀疑是重复投影"的观察项。**该判断已被推翻**——零售学堂候选资产总审工程（`14_外部案例与行业研究/零售学堂课程材料/`）核实 TF函数是一套独立、完整、可操作的方法论（价格→理论周转天数的指数回归拟合 y=a·e^bx，前置步骤是目标客群收入分配模型），六哥判断"整个推导和逻辑完全可以借用，数据上可能有小出入"，正式裁定采纳。

**权威定义位置**：`商品配置理论.md` §20A（"TF函数 / TL / TH 完整方法论"）。本文件不重复承载公式，只做状态索引。

**fact_layer: inferred**——方法论逻辑采纳，但课件案例的具体拟合系数（如胖东来案例 y=0.3162e^0.0406x）是案例特定值，**客户使用前须用本店真实价格-周转天数数据重新拟合，不可直接套用课件系数**。

**术语澄清（与 MC-GBA-03 的关系）**：`04_商品诊断与商品力提升/大湾区IPC与TF函数本地化_MC-GBA-03.md` 原先内部也用"TF函数"称呼其大湾区收入校准公式，与零售学堂原版 TF函数是两个不同工具（分域上限估算 vs 价格-周转天数回归）。2026-07-10 已将 MC-GBA-03 内部公式改称"P/M域收入校准公式"，让出"TF函数"这个名字给零售学堂原版定义，文件名/ID (`MC-GBA-03`) 不变（15+处 wikilink 依赖，不做文件改名）。

Si值 / CTM 两者状态不变，仍是**观察项**：

| 工具 | 全库现有描述(仅此而已) | 出处 | 状态 |
|---|---|---|---|
| Si值 | "衡量陈列面积与销售贡献匹配度"，判断排面配置是否过多/不足 | `商品配置理论.md` §20经营数学接口层表 | 待确认 |
| CTM相乘积 | "综合判断贡献质量"，用于六分仪某格子的综合贡献 | 同上 + §9 六分仪章节 | 待确认，具体算法待确认 |

**本文件的处置**：不新造公式。两者维持观察项，理由：
- **Si值**：陈列面积×销售贡献的匹配度，本质接近 `01_.../数据与系统能力底座.md` 里的"坪效前加维度→有效/无效坪数"元方法（`金额管理.md` §18）——**同样怀疑是坪效元方法的一个特化版本**，建议下次要定义时先读 §18 再决定要不要另立新工具。
- **CTM**：六分仪综合评分格子专用，依赖六分仪矩阵本身先稳定（六分仪目前也是 draft），暂不具备独立定义的前提条件。
