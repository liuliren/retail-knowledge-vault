# 组织力总纲

## 1. 文件定位

本文件定义**组织力**的核心概念、构成框架和典型表现，是 `02_零售企业生意诊断`、`08_经营分析与数据看板` 等模块在组织维度的方法论上游依据。

## 2. 核心定义

**组织力**是门店持续把正确的人、正确的事、正确的标准，落地到日常经营动作中的能力。

组织力的本质是**执行落地能力**，而非人员数量或组织架构图。有商品策略、没有执行力，商品力无法落地；有运营目标、没有任务闭环，运营力无法持续。

## 3. 组织力的核心构成

| 构成模块 | 核心内容 |
|----------|----------|
| 岗位职责 | 每个岗位的职责边界、权限范围和汇报关系是否清晰 |
| 流程标准 | 日常经营动作是否有可执行的标准流程（收货、陈列、盘点、补货等）|
| 任务闭环 | 任务从布置到完成到复盘的全链路是否可追踪 |
| 培训机制 | 新员工上岗和在岗人员能力提升是否有系统性支撑 |
| 复盘机制 | 日/周/月经营数据能否驱动真正的行动修正，而不只是开会 |
| 激励机制 | 薪酬、考核、激励是否与经营目标挂钩 |
| 组织文化 | 门店是否形成"主动发现问题、主动解决问题"的行为习惯 |

## 4. 组织力在社区超市中的典型表现

**强组织力的门店：**
- 店长能把品类规划和陈列规则落地到每个货架
- 员工知道补货时间、陈列标准、缺货上报流程
- 日常复盘不只是对数据，而是能识别问题根因并给出下一步动作
- 新商品引进时，有标准流程控制进场、陈列、动销跟踪

**弱组织力的门店：**
- 有方案，无落地；有会议，无结论；有结论，无跟踪
- 依赖老板一人盯，缺乏标准流程
- 数据看了，但改变不了执行动作
- 商品和陈列标准因人而异，无法复制

## 5. 数据与系统能力在组织力中的位置

**数据与系统能力是科学零售体系的底座，支撑商品力、组织力、运营力三大支柱，而不是独立的第四支柱。**

在组织力维度，数据与系统能力的体现是：

- **角色与权限管理：** 系统支持岗位职责的数字化定义和权限边界
- **任务布置与闭环：** 任务在系统中布置、追踪、验收，不依赖口头通知
- **巡店与现场检查：** 用工具记录现场问题，自动汇总、推送、跟踪整改
- **数据驱动复盘：** 日报/周报自动生成，减少人工整理负担，让复盘聚焦在判断而非汇总
- **培训记录：** 系统化记录员工培训进度和能力评估

数据与系统能力让组织动作**可记录、可追踪、可复盘、可复制**，但系统本身不能替代人的判断和执行意愿。

## 6. 与商品力、运营力、数据与系统能力底座的关系

- **上游：** [[科学零售总纲]]——组织力是三大支柱之一，与商品力、运营力协同
- **与商品力的关系：** 商品力决定"卖什么"，组织力决定"谁来做、怎么做到"；商品策略必须通过组织力才能落地到货架
- **与运营力的关系：** 运营力负责持续改善，组织力是运营力持续运转的载体；没有组织力，运营动作无法稳定复现
- **数据与系统能力底座：** 底座支撑三大支柱，在组织维度体现为任务系统、巡店工具、数据看板、报表自动化；底座不是独立支柱，不应被当成独立的经营目标来管理

## 7. 适用场景

- 门店组织能力诊断
- 岗位职责梳理与流程标准化
- 任务管理和闭环机制设计
- 店长、员工培训体系建立
- 门店复盘机制设计
- 系统产品中与组织力相关的功能模块规划

## 8. 常见误区

- **误区1：** 把组织力等同于"人多"——人员冗余不等于执行力强，岗位职责不清反而降低执行力。
- **误区2：** 把流程建设等同于"写文件"——流程文件写了不执行，等于零；真正的流程标准需要培训、演练和检查闭环。
- **误区3：** 把数据系统当成组织力本身——系统是底座，不是能力；有了系统，执行者不执行，系统毫无价值。（此误区的完整机制——错把工具当员工、把功能当任务、用工具思维管人——见 [[零售管理者的诅咒]]。）
- **误区4：** 把复盘等同于"开会"——会议不等于复盘；真正的复盘要有数据、有根因分析、有行动清单、有下次跟踪。
- **误区5：** 忽视组织文化——制度只能约束行为，文化才能驱动主动性；社区超市的组织力核心往往是店长文化和团队习惯。

## 补充：绩效精细化（绩效必须回到「小类×员工×货位」）

> ⭐⭐ 本段为「激励机制 / 复盘机制」（§3）的方法论展开，回答**绩效到底要细到什么颗粒度才不是伪管理**。
> （逐字批51回填·零售老刘《分析建模》08·09绩效建模·六哥2026-06-26「先回填」签）

**结论先行：绩效必须回到小类（每个单品）+ 每个员工 + 货位关联三者绑定，否则就是不完善、不科学的绩效。** 员工的销售业绩来自其辖管货位上的商品；货位—员工—小类不挂钩，绩效就既算不清账，也压不到人。

1. **绩效的最小颗粒度 = 小类/单品 × 员工 × 货位。** 不到小类、不到单品、不绑货位的绩效，都只是「看起来在管」。员工销售来自其辖管货位的商品，所以「谁管哪片货位 → 哪些小类/单品」必须先对应清楚，绩效才有归属。

2. **⭐伪管理：粗糙绩效 = 伪管理员的伪管理。** 主管说一句「粮油大类下降了」——他说了个 A，员工**听了个 B**（到底哪个小类、哪个单品出问题？不知道），自己**想成 C**，**做出 D**，最后**结果 E**——早就不是 A 了。绩效设得太粗，就是这条 A→B→C→D→E 的失真链；颗粒度不够的绩效=鸡肋，挂了等于没挂。

3. **⭐倒金字塔绩效：压力应在上一级，不在最底层员工。** 绩效责任顺着层级**向上压**，越往上担越多，而不是把账全摊到一线理货员头上。底层员工只对「能用手指头数过来」的具体动作负责，复杂判断与下降归因的压力留给主管/店长。

4. **岗位量化是绩效能不能落地的关键卡点。**
   - 岗位 = 把「评价一个员工表现」这件抽象的事**具体化**；标准是**「能用手指头数过来」**——数得过来的具体动作才可作为分析维度。
   - **抽象行为量化 = 绩效做不下去的最大卡点**：「态度好」「积极」这类数不过来的东西，一进绩效就崩。
   - 落地用**计分（工分）**：完美记 1，稍差记 0.8，按可数的动作累计，不靠印象打分。

5. **人事费用率 = 薪酬额 ÷ 销售额。**
   - **半年算一次**；用「模拟配人」反推费用率，来判断人员是多还是少。
   - **增人的最大理由 = 销售提高**（不是「忙不过来」的体感）；费用率守不住，加人就是亏。
   - **绩效五层级**：公司 → 门店 → 部门 → 课组 → 员工，逐层往下拆，每层都能对回自己那一段的小类/货位。

6. **价值：绩效不止是发钱的依据，更是检验工具。** 精细到小类×员工×货位的绩效，还能**反向检验各经营环节的漏洞与流程衔接**——哪段货位长期不出业绩、哪个交接环节断了，绩效数据会先暴露出来。这正呼应 90/10 的「**可解释 = 可负责**」：颗粒度够细、归属够清，结论才讲得清、责任才签得下。

## 补充：人效五法（门店人员配置·社区超市直接可用）

> ⭐⭐ 本段为「岗位职责 / 流程标准」（§3）与上段「人事费用率」（绩效精细化 §5）的方法论展开，回答**一家店到底该配几个人、怎么倒推**。
> （逐字批60回填·零售老刘《人员配置与人效》2018·六哥2026-06-26「先回填」签）

**结论先行：人效 = 销售额 ÷ 某个维度（某时段）；门店配几个人不靠拍脑袋，用五种方法交叉倒推，社区生鲜超市优先用「人均面积 25-40㎡/人 + 费用率 4-4.5%」两把尺子互相校准。**

1. **人效的定义 = 销售额 ÷ 维度（某时段）。** 维度可以是人、面积、货架米、单品数；同一个销售额换不同分母，就得到不同口径的人效。配人之前先想清楚算的是哪个人效。

2. **法一·人均销售（费用率法，最硬的一把尺）。**
   - **人力成本占销售额：大卖场 3-3.5% / 标超 4-4.5% / 便利店 5-7%。**
   - 平均人效 **1200-1500 元/天·人**。
   - **人数 = 费用率 × 预估销售 ÷ 平均工资**——这是从「养得起几个人」倒推，而不是从「忙不忙」正推。

3. **法二·人均面积。** 大卖场 **50+ ㎡/人**、标超 **40 ㎡/人**、**社区店 25-40 ㎡/人**。六哥 500-2000㎡ 社区生鲜超市直接落这把尺：按营业面积 ÷（25~40）得人数区间。

4. **法三·岗位比例。** 管理 **15-20%** / 理货 **40-50%** / 收银客服 **15-20%** / 防损 **5-10%**。总人数定了，再按比例切到各岗，避免「理货堆人、管理虚胖」。

5. **法四·工作量。** 每人货架 **5-15 米** / 单品约 **100 个** / 收银机 **1-3 名收银员**。这把尺从「干得过来吗」校验前三把尺算出的人数。

6. **法五·添加条件（修正项）。** 男女比、人员流动率、固定工/临时工结构等，作为前四法的现实修正——同样的面积和销售，高流动率或临时工多的店要多留冗余。

7. **⭐定位:社区生鲜超市怎么用。** 花厅坊这类 500-2000㎡ 店,先用**人均面积 25-40㎡/人**框一个人数区间,再用**费用率 4-4.5% 倒推（人数 = 费用率 × 预估销售 ÷ 平均工资）**校准；两把尺打架时,以**养得起（费用率）**为准——守不住费用率,人均面积再合理也是亏。岗位比例切岗、工作量兜底、添加条件修正。

## 补充：企划定位（战略整合层·呼应 90/10 驭马人）

> ⭐⭐ 本段把「组织力」往上接一层：组织力解决「谁来做、怎么做到」（§2），企划解决**「整个企业的资源往哪个方向拧成一股绳」**。它是组织力的上游战略框，呼应 90/10 护城河的驭马人层。
> （逐字批60回填·零售老刘《再谈企划真言》2023·六哥2026-06-26「先回填」签）

**结论先行：企划 = 综合性交叉学科，定位在企业高端（战略层、定方向），核心动作是把企业所有资源（人财物）逐一分析 → 计划整合 → 分解实施 → 聚合成型——其中「计划整合」是核心；企划用木桶原理看企业，盯的是没短板，不是单点最长。**

1. **企划的学科属性 = 综合性交叉学科。** 它不是某个单点职能（不是「做活动」「做海报」），而是横跨经营各环节的整合性工作。

2. **企划的定位 = 企业高端（战略层）。** 企划负责**战略与方向定位**，对**战略目标负责**；它整合企业**所有资源（人、财、物）**，理想状态是**全员企划**——人人在自己环节里对总方向负责，而不是只有一个企划部在画图。

3. **企划盯的是「线」和「面」，不是「点」。** 经营的单点（一个单品、一次促销）做得再漂亮，不等于经营好；企划的重点是把点连成线、线铺成面，看的是整体经营结构。

4. **⭐企划思维 = 木桶原理。**
   - 注重的是**有没有短板**，而不是某块板最长。
   - **短板只有从企划（整体）视角才看得见**——站在单点上永远发现不了自己是短板。
   - 企业瓶颈，**多半源于长期存在、却没被企划发现的短板**。这给组织力诊断一个上游观点：诊断不是找最强项，是从整体扫最弱的那块板。

5. **企划的四步法（核心是「计划整合」）。** 逐一分析 → **计划整合** → 分解实施 → 聚合成型。前有逐一分析（拆），后有分解实施与聚合成型（落地与拢回），但**枢纽是中间的「计划整合」**——把分散的资源和动作拧成一个有方向的整体。

6. **⭐定位:企划 = 战略整合层,呼应 90/10 驭马人。** 企划「整合所有资源、对战略目标负责、整体判断」这套,正是 90/10 护城河里**驭马人层**做的事——**整体判断 = 可解释 = 可负责**,讲不清整体方向的结论不签字。而**木桶原理 = 短板诊断观**:组织力/7D 诊断的底层视角,不是夸最强项,是从企划全局发现并补长期被忽视的那块短板。
