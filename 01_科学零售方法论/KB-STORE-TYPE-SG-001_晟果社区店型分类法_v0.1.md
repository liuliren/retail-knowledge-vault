# 晟果社区店型分类法 v0.1（S1-S4）

> [!important] 标准审议状态声明
> 本页属"分类法/标准"类文件，触发《标准审议铁律》。当前 **status: draft · 审议轮次: 1（起草即第1轮输入稿）**，六哥签字前**不具约束力，不得被其他 Agent 或文件引用为执行依据**。全部阈值与型别定义 `fact_layer: inferred`——**零实证**，待花厅坊归型演练（N=1）与后续客户店实测校准。本页不得自行升级轮次或状态。

---

## 0. 结论先行

1. **"店型" ≠ 面积段。** 大湾区 500-2000㎡ 社区生鲜超市在同一面积段内至少存在四种生存逻辑互斥的店型（S1-S4）；面积桶与店型是**正交二维**，归桶须按"面积桶 × 店型"二维执行，否则同面积异型店横比与基准回填都会系统性失真（推断，待实证）。
2. **归型靠可采集判据，不靠宏观叙事。** 消费阶段等宏观变量在 1.5km 商圈粒度不可测；本分类法改用三条 POS/现场可采集判据（促销日客流比 / 时段客流结构 / 500m 竞对锚）归型——三条判据的阈值目前全部 inferred，第一优先级校准样本是花厅坊。
3. **混合商圈禁止取中间值。** 城中村×高端盘一街之隔时，"加权平均店型"是两头挨打的错误答案；正确输出是"主型 + 副型"双标签，副型以时段/品类模块表达。
4. 大连锁案例（山姆/ALDI/唐吉诃德等）只提供机制方向，背景见 [[KB-RETAIL-STORE-001_店型研究框架_v0.2|KB-RETAIL-STORE-001]]（行业研究背景层）；其数值一律不得代入本分类法。

## 1. 四店型定义（S1-S4 · 全部 inferred）

> 适用域：大湾区 500-2000㎡ 社区生鲜超市**内部再分型**。四型是否 MECE 尚未压测（见 §5 开放问题），以下画像与重心均为待验证假设，非基准。

### S1 · 刚需菜场平替型

- **一句话定义**：在价格高敏商圈里做"比菜市场更便宜、更方便"的生鲜刚需站，靠周转不靠毛利。
- **典型特征（inferred）**：城中村/老旧小区商圈；主竞对为菜市场、钱大妈类折扣生鲜；促销日客流可达平日 3 倍以上；客单低区间（约 25-35 元，inferred）。
- **该型经营重心**：生鲜陈列面积占比拉高（方向：45%+，inferred）；Destination 品类押叶菜/猪肉/豆制品，价格带以 P 域（低价域）为主；毛利率下限运营，损耗靠日清纪律；周转速度优先于一切。
- **该型不该做什么**：不做品质溢价与氛围感装修升级；不扩 Preferred 精选比例（方向：压至 5-8%，inferred）；不投入"信任信号/选品纪律可见化"类建设——价格高敏客群下此类投入边际影响趋近零 [draft互引·不作执行依据：KB-RETAIL-STORE-001 CRITIC 补充命题3 失效条件]。

### S2 · 家庭厨房主力型

- **一句话定义**：在成熟居住区做家庭主理人计划性购物的一站式主力店，用完备性满足 80% 日常餐桌需求。
- **典型特征（inferred）**：成熟居住区；家庭主理人为主力客群，计划性购物；客单中区间（约 40-55 元，inferred）；生鲜占比约 55-70%（inferred）。
- **该型经营重心**：品类**完备性优先**——[[KB-CAT-ROLE-GBA-001_大湾区品类角色特化_v0.1|KB-CAT-ROLE-GBA-001]] §4"完备性 > 专精度"断言的成立域即本型（张力待第3轮正面裁，见 §5）；GBA-001 五角色比例基本原样适用；广府客群加煲汤料/腊味节拍。
- **该型不该做什么**：不照搬山姆式收窄 SKU 卖筛选权（筛选权逻辑属 S4 成立域）；不为单点极致牺牲全品类完备；不盲目扩鲜食便利模块挤占家庭餐桌原料区。

### S3 · 便利鲜食复合型

- **一句话定义**：在高外食率/双职工社区替代"今晚吃什么"的即时决策，卖时间不卖囤货。
- **典型特征（inferred）**：新城白领/双职工社区，外卖密度高或近写字楼；购物频次高、客单低；工作日 18-21 时为客流峰。
- **该型经营重心**：鲜食+熟食+半成品合计营收占比为本型准入线（方向：≥35%，inferred）[draft互引·不作执行依据：该判据升格自 KB-RETAIL-STORE-001 CRITIC 补充命题1]；营业时长、结账效率、动线快捷优先于品类宽度；晚市运营是主战场。
- **该型不该做什么**：不按传统备货逻辑铺全品类宽度；不做囤货型大包装；不用品类完备性指标考核本型（与 S2 逻辑互斥）。

### S4 · 品质精选型

- **一句话定义**：在高端盘/中产社区卖"筛选权与确定性"——消费者付溢价买"有人替我筛过、且筛选有底线"。
- **典型特征（inferred）**：高端盘/中产社区，理性-品质消费段；促销敏感度低（促销日客流比 <1.5 倍，inferred）；复购与会员黏性重于客流量。
- **该型经营重心**：SKU 收窄 + 选品纪律**可见化**（方向：一品 2-3 选 + 选品理由卡，inferred）；Preferred 比例上调（方向：20%+，inferred），M 域（中价域）为主；毛利率上限运营（方向：22-26%，inferred）。
- **该型不该做什么**：不打价格战；不追求品类完备性稀释选品纪律；不做无选品理由的 SKU 扩张——"可解释的选择"是本型护城河来源之一（非"唯一"，其溢价窗口正被平台 AI 导购侵蚀，见第2轮审议问④）。

## 2. 三条 POS 可采集判别判据（阈值全部 inferred · 待花厅坊实测校准）

> 采集成本：判据 1、2 从 POS 流水直接算；判据 3 需 1 次 500m 现场普查。**下表所有阈值均为推断值，未经任何门店实证，禁止当基准引用。**

| 判据 | 度量口径 | S1 | S2 | S3 | S4 |
|---|---|---|---|---|---|
| ① 促销日客流比（价格敏感度代理） | 促销日客流 ÷ 平日客流（POS 即有） | ≥3 倍 | 1.5-2.5 倍 | <1.5 倍 | <1.5 倍 |
| ② 时段客流结构（客群结构代理） | 工作日分时客流分布（POS 即有）：早市 7-9 时 / 午后 15-17 时 / 晚峰 18-21 时 | 早市主导 | 各时段均衡 | 晚峰（18-21 时）主导 | 均衡偏晚 |
| ③ 500m 竞对锚（商圈画像代理） | 1 次现场普查：500m 内主竞对类型 | 菜市场 / 折扣生鲜 | 无强锚或大卖场较远 | 外卖密度高 / 近写字楼 | 高端盘物业 / 精品超市 |

**校准计划（inferred → observed 的路径）**：第一样本 = 花厅坊（POS 数据已有，采集成本≈0）；三条判据各自的阈值在 N≥3 家实测店后才允许讨论固化，在此之前每次引用必须携带 inferred 标注。

**已知盲区（起草即声明）**：判据未纳入"即时零售渗透率 / 500m 内前置仓密度"维度（第2轮审议问④指出线上维度缺席）；也未含"商圈消费阶段迁移速度"变量（旧改/新盘交付场景）。两者留待第3轮裁定是否增补为第 4、5 判据。

## 3. 归型规则（inferred）

1. **主型判定**：三判据中 ≥2 个命中同一型 → 该型为**主型**。
2. **副型判定**：剩余判据命中他型 → 挂**副型**标签。副型不另立店型，以**时段/品类模块**表达（例：主 S2 + 副 S3 = 保持家庭厨房主力盘面，晚市增设鲜食/半成品模块）。
3. **混合商圈强制规则**：三判据三型分裂 → 判"混合商圈"，**强制主型+副型双标签**，运营方案按时段×品类拆分；**禁止取中间值**做"加权平均店型"——中档折中店在价格上打不过 S1 竞对、在品质信任上打不过 S4 竞对，是两头挨打的结构性错误答案（推断，源自第2轮审议反例 A）。
4. **再判定触发器（占位，待第3轮细化）**：商圈发生旧改交付/大型竞对进驻/客流结构连续 8 周偏移时，须重跑三判据归型。
5. **花厅坊：疑似主型 S2，待归型演练验证。** 依据（observed 数据点）：客单 42.14 元、生鲜占比 70.74%、荔湾广府核心客群——三者与 S2 画像方向一致，但**尚未按 §2 三判据正式演练归型**，在演练完成前"花厅坊=S2"只是假设，不得写入任何交付件。

## 5. 开放问题（留给第3轮及六哥裁决）

### 5.1 与 KB-CAT-ROLE-GBA-001（stable）的张力——第3轮正面裁

GBA-001 §4 无条件断言"社区超市生存逻辑 = 品类完备性 > 品类专精度"，但该断言**缺消费阶段/店型前提标注**；本分类法的 S2 即其成立域，S4（筛选权逻辑）与 S3（便利优先逻辑）则是其失效域。若花厅坊外的某店归型为 S4，GBA-001（stable）与本页（draft）将给出相反操作指令——按状态机 stable 压过 draft，但 GBA 赢在行政级别而非逻辑。**需第3轮正面裁决**：① GBA-001 是否随本分类法落地补注"成立域=S2 型社区"前提；② 五大角色 SKU 比例是否按 S1-S4 分化出修正系数。

### 5.2 第2轮审议自留的 4 个批判靶子（照录）

1. 四型是否 MECE？S3 与 S4 在"新城中产社区"可能重叠——需要第3轮用真实商圈样本压测。
2. 判别阈值（3 倍 / 35% / 40%）全部 inferred，一个实证都没有——与 MULTISTORE placeholder 同病，须防"伪精确"复发。
3. 是否该有 S5"乡镇/低线大量消费期型"（对应日本三时代的第二阶段）？大湾区外扩客户（惠州/江门县域）可能落在四型之外。
4. 更省的替代方案：直接采用业界现成分类（日本 SM 业态分类 / 永辉店型分级）再本地化，可能比自建快——请第3轮专业角色 steelman。

### 5.3 替代方案 steelman："直接本地化日本 SM 业态分类可能更省"

**正方（为替代方案辩护，供第3轮对抗）**：
- 日本 SM（食品超市）业态分类经数十年行业协会数据沉淀，型别边界与 KPI 基准都有公开统计口径支撑；中国生鲜零售研究大量借道日本模型（三消费时代即一例），认知迁移成本低。
- 自建分类法 = 自担校准成本：S1-S4 的每个阈值都要自己攒 N≥3 实证，而日本分类自带基准区间，"借来再打折"可能比"从零实测"快一个数量级。
- 永辉/钱大妈等本土连锁的店型分级已在大湾区真实市场里被验证过存活率，其分级变量（商圈档次×面积×生鲜占比）与本页判据高度同构——存在"抄作业"空间。

**反方（自建派的回应，同样待检验）**：
- 日本 SM 分类以连锁体系为默认单位，判别变量（商势圈人口统计、连锁采购结构）对单体店不可得；本页判据的设计约束恰恰是"POS 即有 + 1 次普查"，这是咨询作业成本决定的，借来的分类未必满足。
- 大湾区特有变量（城中村混合商圈、外卖密度、广府饮食节拍）在日本分类中无对应槽位。
- **裁决交第3轮**：若专业角色能找到判别变量可在单体店采集的现成分类，本页应降格为该分类的本地化映射层，而非坚持自建。
