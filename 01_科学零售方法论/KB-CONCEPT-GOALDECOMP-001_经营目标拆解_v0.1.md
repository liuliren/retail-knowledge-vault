# 经营目标拆解 v0.1（candidate）

## 结论先行
**经营目标拆解 = 把老板嘴里一句"今年要做到 X 万"，沿"销售额分解树"逐层拆成品类目标 → 单品动作 → 谁在什么时候做什么**。它是《五层经营链》里"顾客→配置→品类→单品→数学"落到现场的第一道桥：不拆解，诊断结论就停在"你毛利低"这种正确的废话，落不到动作。

## 拆解链（社区生鲜超市版）
1. **总目标锚定**：营收目标 → 用根公式反推 `销售额 = 客数 × 客单价`（口径见 [[KB-METRIC-DICT-001_经营指标字典_v0.2]] ①业绩规模）。
2. **品类分解**：按生鲜/休食/日配/标品的销售占比与毛利结构，把总目标摊到品类（生鲜占比 f 是护城河变量，见字典⑥）。
3. **单品分解**：品类目标 → ABC-Z 分层 → A 类守量、C/Z 类腾空间换新品（[[KB-DX-MATH-001_零售数学诊断工具体系_v0.1]] IM-04）。
4. **动作分解**：每个单品目标挂 What/Who/When/验收点，落到店长可执行的调改清单。

## 花厅坊可直接调用
- 老板给"休食区月营收目标" → 拆：休食占比×总盘 → 休食内 A 类（TOP 款守面数）+ 尾部 39% SKU（产 10% 销额）清理换新 → 输出"退 40-60 款 + 补 X 款"的动作卡。**下一个动作 = 生成休食区目标拆解表**。

## 张力（何时失效）
- **老板目标本身不实**（拍脑袋翻倍）→ 拆解会把错误目标精确化，须先用保本销售额（字典②）校验目标是否够本，再拆。
- **数据口径未定**（如动销率分母含烟/不含烟未裁，字典⑦冲突①）→ 品类分解的基数会错，须先锁口径再拆。
- 目标拆解**只解决"往哪走"，不解决"走不走得动"**——组织力/供应链跟不上时，拆得再细也落不了地。
