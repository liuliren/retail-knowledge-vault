## 核心论点

销售-陈列错配诊断的本质是**用一个比值（销售占比 ÷ 陈列空间占比）快速识别哪里需要动手**——比值 >1.5 增格、<0.5 缩格，再结合品类角色（A2 五分法）做最终裁决，是区域调改五步法中第二步与第三步之间的必经桥梁。

## 一、什么是销售-陈列错配

货架上每个 L3（小品类）/ 每个子格占据一定的**陈列空间份额**，同时贡献一定的**销售份额**。理想状态下两者大致匹配——销售大的占空间多，销售小的占空间少。

实际门店常见三种错配：

```
类型 A：销售大但空间小  → 增格（空间扩张）
类型 B：销售小但空间大  → 缩格（空间压缩）
类型 C：销售一般空间合理 → 保留
特殊：  销售小空间小但战略重要 → 重新评估角色
```

错配诊断的核心是**用最少的数据快速识别需要动手的地方**——不是优化所有面位，而是先动错配最严重的几处。

## 二、诊断的标准步骤

### 第一步：把面位上的每个 L3 / 子格销售数据汇总

输入：30 天动销数据，按 L3 或子格分组求和。

输出表格：

```
| L3 / 子格 | 销售额 | SKU 数 | 销售占比 |
| L3-A | ?? 元 | ?? | ??% |
| L3-B | ?? 元 | ?? | ??% |
| ... | ... | ... | ... |
```

### 第二步：把面位上每个 L3 / 子格的陈列空间份额计算出来

输入：现有落位图（每个 L3 / 子格占多少格、多少层）

简化算法：
- 一个标准子格占据全面位空间份额 = **1 / 子格总数**
- 例如 6 格货架：每个子格占 16.7%
- 半格/扩展格按比例计算

输出：

```
| L3 / 子格 | 陈列空间占比 |
| L3-A | 16.7% |
| L3-B | 33.3% |
| ... | ... |
```

### 第三步：计算"销售-陈列比"

```
销售-陈列比 = 销售占比 / 陈列空间占比
```

- 比值 ≈ 1：匹配，无需动作
- 比值 > 1.5：销售强、空间小 → **增格候选**
- 比值 < 0.5：销售弱、空间大 → **缩格候选**

### 第四步：标记所有错配点

把面位上所有"比值 > 1.5"和"比值 < 0.5"的 L3 标出来。**这些就是优先要动的地方**。

### 第五步：根据角色（A2 五分法）做最后判断

错配不是绝对动作信号——还要结合品类角色：

- **目的性 / 偏好性 + 销售-陈列比 > 1.5** → 强烈增格信号
- **目的性 + 销售-陈列比 < 0.5** → 不缩格，反而要补强（可能是陈列差导致销售弱）
- **偶发性 + 销售-陈列比 > 1.5** → 慎重增格（增了反而长尾）
- **偶发性 + 销售-陈列比 < 0.5** → 强烈缩格信号
- **季节性** → 看窗口期数据，不看全年平均

## 三、案例：花厅坊 G3-A 的错配诊断

> 注：以下为脱敏内部案例，仅供方法论说明用，数据不对外。

### 数据输入

V1 落位下，G3-A 6 个子格平均每格占 16.7%（不计层位变化）。30 天动销数据：

| L3 | 销售额 | 销售占比（占 G3-A 总销售） | 陈列空间占比 | 销售-陈列比 |
|---|---:|---:|---:|---:|
| 肉类零食 | 1898 元 | 56.5% | **16.7%（1 格）** | **3.4** |
| 海味零食 | 670 元 | 19.9% | **16.7%（1 格）** | **1.2** |
| 豆干素食 | 665 元 | 19.8% | 16.7%（1 格） | 1.2 |
| 魔芋 | 130 元 | 3.9% | 16.7%（1 格） | 0.2 |

> 注：肉类零食在 V1 草图中其实占 3 格（卤味即食 + 肉干肉脯 + 高蛋白分享），不是 1 格，所以这个数据需要按子格颗粒度重算。下面用子格级数据重算。

### 子格级错配诊断（更精细）

| 子格 | 销售估算占比 | 陈列空间占比 | 销售-陈列比 | 判断 |
|---|---:|---:|---:|---|
| 第 1 格 豆干素食 | 19.8% | 16.7% | 1.2 | 匹配 |
| 第 2 格 魔芋 | 3.9% | 16.7% | 0.2 | **缩格候选** |
| 第 3 格 海味（满格） | 19.9% | 16.7% | 1.2 | 匹配（但角色是偶发性，仍可缩） |
| 第 4 格 卤味即食 | 25%+ | 16.7% | 1.5+ | **增格候选** |
| 第 5 格 肉干肉脯 | 20%+ | 16.7% | 1.2+ | 匹配（强势） |
| 第 6 格 高蛋白分享 | <10% | 16.7% | <0.6 | **缩格候选** |

### V1 → V1.1 的错配诊断驱动修订

- **海味（偶发性 + 比值 1.2）**：虽然比值不极端，但角色是偶发性，**主动缩半格**释放空间
- **卤味（偏好性 + 比值 1.5+）**：**扩为 2 格**（第 4 格 + 部分第 6 格）
- **高蛋白分享（比值 < 0.6）**：**缩为 SKU 级补充**，第 6 格形象重定位为"卤味升级"
- **魔芋（比值 0.2）**：暂保留作为差异化年轻客群补位，不缩——但要在第五步严控 SKU 数

## 四、错配诊断的两种用法

### 用法 1：调改前的诊断输入

在区域调改五步法的第二步（面位任务锁定）之后、第三步（CDT + 子格落位）之前，对 V1 现有落位做一次错配诊断，识别需要重点处理的错配点。

### 用法 2：调改后的效果验证

调改后 30 天 / 90 天，再做一次错配诊断，看：

- 原错配点是否消除（销售-陈列比是否回到 0.7-1.5 区间）
- 是否出现新的错配（说明新方案有问题）

## 五、错配诊断的局限

### 局限 1：销售数据可能被陈列错配本身扭曲

某品类销售弱可能是因为陈列差，而不是真的卖不动。这种情况下"销售-陈列比 < 0.5"的判断会错——不是缩格而是要先改善陈列再观察。

**应对**：错配诊断 + 角色判断（A2）双轨。如果是目的性/偏好性品类的"销售-陈列比 < 0.5"，先试改善陈列 30 天再决定。

### 局限 2：30 天数据期短，季节性误判

30 天数据在节令商品上完全不可信。

**应对**：节令商品单独看窗口期数据；非节令商品 30 天数据可用但结论保守。

### 局限 3：陈列空间份额的简化计算

实际陈列空间不只是"格数"——还包括层位、面数、端架等。简单按格数算会有误差。

**应对**：颗粒度足够时用"格×层"的总面数；做大方向判断时用"格数"即可。

### 局限 4：忽略毛利贡献

销售大不等于毛利大。真正的"陈列价值"应该是"毛利贡献"。

**应对**：当能拿到毛利数据时，把"销售占比"替换为"毛利占比"做更精准的诊断。

## 六、错配诊断的标准工作流

```
1. 抽取面位上 L3 或子格级 30 天销售数据
2. 计算每个 L3 / 子格的销售占比
3. 测量每个 L3 / 子格的陈列空间占比
4. 计算销售-陈列比
5. 标记 > 1.5 和 < 0.5 的错配点
6. 结合角色（A2）做最终动作建议（增/缩/保留/再观察）
7. 把动作建议输入到 CDT + 子格落位（A3）作为强度参考
```

## 七、错配诊断在区域调改五步法中的位置

```
第二步 面位任务锁定
   ↓
   错配诊断（A4）  ← 这里
   ↓
第三步 CDT + 子格落位（A3）
   ↓
第四步 专项嵌入
   ↓
第五步 SKU 上下架（用错配诊断验证）
```

错配诊断**不是独立的一步**，而是穿插在第二、三、五步中的工具。

## 花厅坊直接调用

> T03 饼干错配诊断步骤：用 M-DEC-010 中价 82% 数据 + 实际格位数对比，识别两类错配：
> - **第一类**（高销额/低陈列）：TOP20 主力款若陈列面数 < 2 → 立即扩面
> - **第二类**（低销额/高陈列）：尾部 40–60 款若各占 ≥1 格 → 压缩让出面位
>
> 执行门槛：POS 销额降序 → 对比现场格位数 → 差值最大的前 5 个 = 第一批调改对象。

## 失效条件

当L3/子格处于新品引进观察期（<30天动销数据）或季节旺季前夕，销售-陈列占比比值会系统性低估真实需求，此时按比值缩格会过早砍掉潜力品——须在新品满30天后或节令结束后再执行陈列压缩决策。
