# 商品配置理论

## 1. 文件定位

商品配置理论是商品力体系中用于**拆解、设计、诊断和优化商品结构**的核心方法论。

它不是选品清单，不是陈列图，也不是 SKU 数量规定，而是连接以下要素的**商品结构设计工具**：

- 品类管理（[[品类管理]]）
- 单品管理（[[单品管理]]）
- 价格结构（[[价格带管理]]，待建立）
- 品种结构（HS矩阵）
- 顾客购买任务（[[顾客决策树CDT]]，待建立）
- 货架空间与陈列（[[DisplayMap]]，待建立）
- 年度商品波段规划（[[52周MD]]，待建立）

在商品力主链路中的位置：

```
商品力总纲
├── 品类管理      → 定义品类战略方向和品类角色
├── 单品管理      → 管理 SKU 进退和单品贡献
├── 商品配置理论  → 设计商品结构（本文件）
└── 52周MD        → 规划时间维度的商品节奏（待建立）
```

四个模块并列，均为商品力总纲的下游，互相协同但分工不同。

## 2. 核心定义

**商品配置**：

在明确顾客购买任务、品类战略和货架资源的前提下，决定一家门店（或一个品类区域）**应该卖哪些商品、各价格带放多少 SKU、哪些品种必须配齐、哪些 SKU 可以精简**的系统方法。

商品配置的核心输出是一张可操作的**商品结构图**，而不仅仅是 SKU 清单。

商品配置要同时回答：
- 这个品类的价格带分布是否合理？
- 这个品类的品种覆盖是否完整？
- 顾客购买路径在现有商品结构中是否清晰？
- SKU 的投入与产出是否匹配？

### 2B. 商品四大价值层（选品/卖点拆解尺·零售老刘逐字批24回填·六哥 2026-06-25 签）

> 来源:零售老刘《品类建模》155「商品价值层」(转述)。**给"选哪些商品"配一把价值拆解尺**——CDT/HS/IPC 解的是结构,这把尺解的是单品"凭什么被选进来"。

任一商品(单品)可拆为四层价值,**配置时自下而上、不可越层**：
1. **核心层**(基本功能效用)：买牙刷是为"刷牙",不是为牙刷本身。**商品"应该卖什么"的根·产品开发核心依据·卖点的第一来源**。
2. **形式层**(外在表现)：陈列正面/POP/包装/卖场环境——顾客"以貌取物"的购买决策依据。但避免"**买椟还珠**"(过度包装喧宾夺主)。
3. **附加层**(服务/承诺/惊喜)：海底捞、胖东来 400+ 免费服务——制造满意+顾客粘性。
4. **延伸层**(稀缺/匠心/文化/软实力)：价值放大器(胖东来"善良真诚一点")。

- **价值层公式**：`价值 =（售价 − 进价）× 满意度的平方`(满意度是放大器,故平方)。
- **越层价值=禁忌**：没做好核心层就想做价值层 = 空中楼阁。**配置纪律:核心层商品配置数最多(70%以上),依价值层递减**;形象/延伸层占比小而精。
- **落选品**:每引一款先问"它的核心层卖点是什么、卖给谁",再看形式/附加/延伸能加多少分——而非反过来用包装/故事掩盖核心层空洞。接 [[SKU角色层与目的品保护机制_v0.1]] 角色判定。

## 3. 商品配置理论解决的问题

商品配置理论解决以下三类结构性问题：

| 问题类型 | 典型表现 | 对应工具 |
|----------|---------|---------|
| 价格结构失衡 | 高价品太多、价格断层、波峰错位、大众品不足 | IPC 商品配置图 |
| 品种结构失衡 | 同质商品扎堆、需求盲区存在、低效SKU占位 | HS矩阵 品种结构表 |
| 顾客决策路径断裂 | 顾客找不到商品、品类骨架不清晰、选择逻辑混乱 | CDT 消费者决策树 |

商品配置不合理的常见后果：
- 高动销区排面不足，低动销区占用过多货架资源
- 价格带断层导致中间价位顾客流失
- 品种冗余但关键需求点遗漏
- 持续引进新品但未清退冗余 SKU，整体 SKU 数量越来越臃肿
- 商品结构与顾客购买任务错位，动销率持续低迷

## 4. 理论来源与参考

本方法论主要参考日本零售实务专家**渥美俊一**的商品配置思想。其著作日文原名已确认为**《商品構成》**，中文译名及是否有正式中文出版版本尚未核验。IPC、HS矩阵等工具的日文原文全称仍需进一步核验，本文件使用中文意译命名，不对原文术语作最终确认。

渥美俊一商品配置思想的核心贡献包括：
- 以商品配置图（IPC，即 Item & Price Chart）分析价格带分布结构
- 以品种结构表（HS矩阵）分析品种的需求覆盖与投入产出结构
- 将消费者决策树（CDT）与商品配置结合，形成三工具组合方法论
- 提出新店规划与老店诊断采用不同的工具作业顺序
- 将商品结构划分为 P域（Popular，大众品）与 M域（Moderate，趋势品）两大区域

> - IPC 暂按 Item & Price Chart / 商品配置图或价格结构图理解。
> 
> 上述术语是否与原著、日文原文或可靠文献完全一致，待后续查阅原著或可靠资料后确认。
> 待确认：著作中文译名及是否有正式中文译本；IPC 和 HS矩阵在原著中的完整日文术语；"六分仪"命名是否来自原著。

## 5. 商品配置与品类管理的关系

两者不是重复关系，分工明确：

| 维度 | 品类管理 | 商品配置理论 |
|------|----------|-------------|
| 核心问题 | 这个品类往哪个方向走？ | 这个品类应该配什么商品、怎么配？ |
| 核心工具 | 品类角色、品类评估、品类策略、品类战术 | IPC、HS矩阵、CDT、商品配置全表 |
| 主要输出 | 品类战略方向、品类策略、品类战术调整建议 | 商品结构图、价格带分布、品种需求矩阵 |
| 时间跨度 | 通常以季度或年度为规划周期 | 可单次诊断，也可周期性更新 |

协同方式：
- 品类管理确定品类角色（目标性/常规性/季节性等）→ 商品配置理论按品类角色设计对应的结构深度与 SKU 数量上限
- 品类策略确定增长/利润/引流方向 → 商品配置理论在 IPC 中优先布局对应价格带

## 6. 商品配置与单品管理的关系

两者不是重复关系，分工明确：

| 维度 | 单品管理 | 商品配置理论 |
|------|----------|-------------|
| 核心问题 | 这个 SKU 是否保留？何时引进/淘汰/替换？ | 这个品类/价格带应该配几个 SKU？什么结构？ |
| 核心工具 | ABCD分类、ABC-Z模型、候选商品池 | IPC、HS矩阵、商品配置全表 |
| 主要输出 | SKU 进退动作清单 | 商品结构设计图、SKU 数量框架与结构要求 |

协同方式：
- 商品配置理论确定各价格带/品种应有的 SKU 数量框架 → 单品管理在该框架内执行具体 SKU 的引进/淘汰/替换/调整决策
- 单品 ABCD 分类结果 → 反馈到 HS矩阵，识别哪些品种 SKU 贡献偏低、哪些品种 SKU 冗余

## 7. 商品配置的三大核心工具

在本知识库中，暂将 CDT（消费者决策树）、HS矩阵（品种结构表）、IPC（商品配置图）作为采购选品的**三类核心工具组合**使用。是否采用某一固定命名（如"三神器"等）作为本知识库正式称谓，待确认。

三工具各自视角与分工：

```
CDT 消费者决策树   →  顾客怎么选？（购买决策路径）
HS 矩阵 品种结构表 →  品种配置是否完整？（需求覆盖检查）
IPC 商品配置图     →  价格结构是否合理？（价格带分布校验）
```

三工具互补：
- **CDT** 是顾客视角，解决"顾客路径"问题
- **HS矩阵** 是需求 × 属性视角，解决"品种完整性"问题
- **IPC** 是经营 × 价格视角，解决"价格结构合理性"问题

三工具输出合并，可生成"**商品配置全表**"（另见 §11）。

## 8. IPC：价格结构分析

**IPC（商品配置图）** 站在**供给方/经营者视角**，分析一个品类内不同价格带的商品分布结构。

### 核心作用

- 判断各价格带 SKU 数量分布是否合理
- 识别价格断层（某价格段无商品覆盖）
- 识别价格堆积（某价格段商品过于集中）
- 判断价格波峰位置是否与顾客主流需求匹配
- 平衡大众品与趋势品的 SKU 比例，兼顾现金流与利润率

### P域与M域

| 区域 | 全称含义 | 价格参考范围 | SKU 占比参考 | 财务作用 |
|------|---------|------------|------------|---------|
| P域（大众品/基础品） | Popular，主流需求商品，大多数顾客能高频购买的实用商品 | 约为目标顾客月薪的 1%～30% | 参考值：60%～80% | 现金流保证，走量，高周转率 |
| M域（趋势品/差异化品） | Moderate，差异化商品，满足较小众或低频较高端需求 | 约为目标顾客月薪的 30%～360% | 参考值：20%～40% | 利润流保证，走价，高毛利率 |

P域保现金流和销售额基础，M域布局未来利润空间和差异化竞争力。

> 注：上述价格参考范围和 SKU 占比参考值来自渥美俊一框架，适用于目标顾客月薪已知的品类配置场景。具体品类的价格边界需结合本地客群消费水平和样板门店数据验证，不可直接套用。
> 待确认：各品类 P域/M域 价格边界的具体划分标准；各品类最优 SKU 占比区间；样板门店验证结果。

### IPC 与现金流安全

IPC 不只是价格带分布图，也可视为商品结构的**现金流心电图**。

如果 P域商品不足、M域商品过多，商品结构可能出现三个问题：
- 大众高频需求覆盖不足，销售基础变弱
- 高价低频商品占用库存资金，整体周转变慢
- 顾客形成"这家店偏贵"的价格感知

因此，IPC 调整通常有两个方向：

- **抬高 P域顶点**：在主力大众价格带增加适销 SKU 或陈列面，提高走量能力，加快现金流回流速度。
- **左移 P域顶点**：当整体价格结构偏高、现金流压力较大时，将主力价格点向更低价格带迁移，强制加快库存周转。

无论采用哪种方式，调整后的 IPC 曲线都应避免高价区域 SKU 过度堆积。理想形态是主价格点之后，SKU 数量随价格升高逐步减少，形成**右下倾斜**结构。

> 注：抬高/左移 P域顶点、右下倾斜等判断方法属于 IPC 应用规则，具体计算方法和样板门店验证待确认。

## 9. HS矩阵：品种结构分析

**HS矩阵（品种结构表）** 站在**需求方/消费者视角**，将商品物理属性（Hard）与消费需求类型（Soft）交叉，形成品种需求矩阵。

### 核心作用

- 检查品类内品种配置是否完整
- 发现需求遗漏（某类需求无对应商品）
- 排除冗余滞销 SKU（多个 SKU 服务同一需求，贡献重叠）
- 衡量各品种 SKU 投入与销售/毛利产出是否协调

### Hard × Soft 维度说明

| 维度 | 含义 | 示例（待按品类设置） |
|------|------|-------------------|
| Hard | 商品物理属性 | 规格、口味、包装形态、功能、品牌、容量 |
| Soft | 消费者软性需求 | 消费场景、购买任务、人群偏好、价格敏感度、便利性 |

示例矩阵结构（以冲调饮品为例，仅作说明用，具体维度待确认）：

| Hard ↓ / Soft → | 早餐场景 | 办公提神 | 家庭待客 | 下午茶 |
|-----------------|---------|---------|---------|-------|
| 速溶咖啡 | ● | ● | | |
| 袋泡茶 | ● | | ● | ● |
| 固体饮料（含乳） | ● | | ● | |
| 麦片/营养粉 | ● | | | |

> 矩阵中空格代表潜在需求遗漏，需结合客群数据判断是否值得填入，不可直接判定为必须补充。

> 待确认：
> - 不同品类的 Hard 维度具体包含哪些属性
> - 不同品类的 Soft 维度具体包含哪些需求类型
> - HS矩阵标准模板的归属目录（候选：`05_品类管理与商品规划` 或 `10_咨询交付模板`，待确认）

### 投入产出差异系数

HS矩阵可以通过差异系数识别冗余单品和扩充机会：

**差异系数 = 毛利额占比 ÷ SKU数占比**

| 差异系数范围 | 诊断含义 | 建议动作 |
|------------|---------|---------|
| 系数 < 1 | 高投入低产出，存在冗余风险 | 检查是否可以删减 SKU |
| 系数 ≈ 1 | 投入与产出大致匹配 | 维持现状，定期复核 |
| 系数 > 1 | 低投入高产出，存在扩充机会 | 评估是否值得增加该品种 SKU |

**重要限制：** 差异系数不能机械使用。以下情况不可仅凭系数判断删减：
- 某 SKU 毛利低，但承担顾客购买完整性（缺货会导致顾客转店）
- 某 SKU 毛利低，但属于价格形象商品（用于传递低价信号）
- 某 SKU 毛利低，但具有连带购买拉动效应

此类商品必须结合 CDT 和品类角色复核，避免误杀战略性商品。

> 注：差异系数的计算需要完整的 SKU 级销售数据和毛利数据，具体计算口径和诊断阈值待通过样板门店验证确认。

## 10. CDT：顾客决策路径分析

**CDT（消费者决策树）** 描绘顾客在一个品类中做出购买决策时的路径和优先顺序。

### 核心作用

- 明确顾客在该品类中先按哪个维度选，再按哪个维度筛
- 揭示品类骨架的合理层级（品牌优先还是口味优先、功能优先还是价格优先）
- 指导货架陈列的分区逻辑和商品排列方式
- 作为 HS矩阵设定 Soft 维度的依据

### 关键区分

- CDT 是顾客视角，不是企业内部商品分类逻辑
- CDT 应服务品类定义、品类结构和商品配置
- 不同门店的 CDT 可能因客群差异而不同，不能直接套用

> 注：CDT 的完整主定义和方法论待单独建立文件（[[顾客决策树CDT]]，待建立）。本文件仅说明 CDT 在商品配置理论中的协同角色。

## 11. 商品配置全表 / 六分仪

将 IPC + HS矩阵 + CDT 三工具的输出合并，可生成全景式的商品配置判断工具，暂称"**商品配置全表**"。

"六分仪"为另一候选命名，其来源及是否作为本知识库正式术语，待确认。

### 商品配置全表的六个核心指标

"六分仪"名称来源于在每个"价格带 × 需求类型"交叉格中同时观察六个指标（命名来源及是否为原著术语待确认）：

| 指标 | 类型 | 含义 |
|------|------|------|
| SKU数占比 | 投入指标 | 该格子占用了多少单品名额 |
| 陈列面数（饰面数）占比 | 投入指标 | 该格子占用了多少货架陈列空间 |
| 销售额占比 | 产出指标 | 该格子贡献了多少销售额 |
| 毛利额占比 | 产出指标 | 该格子贡献了多少毛利 |
| 毛利率 | 效率指标 | 该格子的利润率水平 |
| CTM（相乘积） | 综合指标 | 综合判断贡献质量，具体算法待确认 |

### 通过全表可以识别

- **高投入低产出**：SKU数/陈列面占比高，但销售额/毛利额占比低 → 可能存在冗余，需检查
- **低投入高产出**：SKU数/陈列面占比低，但销售额/毛利额占比高 → 可能存在扩充机会
- **高毛利低周转**：毛利贡献高但库存周转慢 → 可能拖慢现金流，需结合 TF函数判断
- **低毛利高周转**：可能是 P域现金流商品 → 不能简单因低毛利率而淘汰
- 价格断层或价格堆积
- 需求空白或需求重叠

> 注：商品配置全表依赖销售数据、毛利数据、商品档案的完整性，具体字段依赖和数据清洗规则见 [[数据与系统能力底座]]。

## 12. 商品配置的四类配置方式：标配 / 选配 / 季节配 / 活动配

商品配置可进一步拆分为四类配置方式：

| 配置类型 | 含义 |
|---------|------|
| **标配** | 长期稳定经营、必须保留的基础商品配置 |
| **选配** | 根据门店客群、商圈、面积和竞争环境选择性配置的商品 |
| **季节配** | 围绕节令、气候、消费季节变化配置的商品 |
| **活动配** | 围绕促销、主题活动、节点营销临时配置的商品 |

四类配置方式的协同逻辑：
- 标配构成品类 SKU 的稳定基础，不宜随意调整
- 选配体现门店差异化，需结合商圈调研和客群分析制定
- 季节配需与 [[52周MD]]（待建立）的波段规划联动
- 活动配通常是临时性占位，活动结束后应及时清退，避免占用稳定排面

> 以上四类配置方式为 v0.1 骨架定义，具体判断标准、各类 SKU 占比建议和执行模板待确认。

## 13. 新店规划中的应用顺序：CDT → HS → IPC

**适用场景：** 新开门店品类规划、全新品类引进、品类从零建立。

**作业顺序：**

```
① CDT
   画出该品类的顾客购买路径和决策层级
   ↓
② HS矩阵
   在决策路径下，逐一检查各品种需求是否覆盖，确定应配 SKU 范围
   ↓
③ IPC
   检验拟配 SKU 的价格带分布，校正波峰位置，调整 P域/M域比例
```

**逻辑：** 先建顾客认知骨架，再填商品，最后校验价格结构是否可经营。

从顾客出发 → 从顾客路径中找出品种需求 → 用价格结构校验商业可行性。

## 14. 老店诊断中的应用顺序：IPC → HS → CDT

**适用场景：** 既有门店商品诊断、品类整改、商品调改。

**作业顺序：**

```
① IPC
   读取现有商品的价格带分布，识别价格结构问题（断层/堆积/波峰错位）
   ↓
② HS矩阵
   在现有 SKU 中逐一对照品种需求，筛除冗余、识别遗漏、评估各品种贡献
   ↓
③ CDT 重构
   根据前两步结论，重新梳理顾客决策路径，确认品类骨架是否需要调整
```

**逻辑：** 先从数据找结构问题，再筛 SKU，最后确认顾客路径是否重新清晰。

从现状数据出发 → 识别结构问题 → 筛选 SKU → 重建顾客逻辑。

> 注：两种顺序目标相同（建立合理商品结构），起点不同（新店从顾客路径出发，老店从现有数据出发）。最终都需三工具输出互相印证。

## 21. 商品配置的现金流安全边界

商品配置不仅影响顾客选择，也直接影响现金流安全。

如果一个品类配置了过多高价、低频、低周转商品，即使毛利率较高，也可能导致库存资金长期沉淀，形成现金流风险。因此，商品配置必须同时关注：

- **毛利率**：单品或品类的利润能力
- **周转率**：库存变成现金的速度
- **库存金额**：资金占用规模
- **供应商账期**：企业实际可承受的资金周转周期

GMROI 可以作为商品配置的上位经营目标：

> GMROI = 毛利额 ÷ 平均库存金额
>
> 也可理解为：GMROI = 毛利率 × 库存周转率

注：以上公式作为经营数学接口写入，具体口径、数据来源、计算周期和适用范围待确认。

在商品配置诊断中，可用 TL/TH 作为现金流安全判断框架：

- **TL（理论最小周转天数）**：当前商品结构在理想情况下能达到的最快周转速度
- **TH（最大周转天数/供应商账期）**：企业对供应商的结算账期，即商品必须变现的最晚天数
- 当 TL/TH < 1 时，理论上现金流较安全
- 当 TL/TH ≥ 1 时，可能存在现金流风险，需要调整商品结构

注：TL、TH、TF函数的具体计算方法、适用边界和样板门店验证结果待确认。v0.1 不写入固定公式和阈值。

## 22. 在社区超市中的应用

### 适用场景

商品配置理论在社区超市中优先适用于以下品类（待通过样板门店验证）：
- 冲调饮品（SKU 数量多、价格带宽、同质化严重，结构诊断价值高）
- 休闲食品（品种结构复杂、季节性强、需求分层明显）
- 方便速食（品类角色重要、价格带分层清晰、进退频繁）

### 社区超市的特殊约束

- 总货架资源有限，SKU 总数受物理空间约束，不宜无限扩充
- 顾客以周边固定社区居民为主，CDT 相对稳定，但需本地化校准，不可直接套用外部标准
- 高周转率要求下，M域（趋势品）比例不宜过高，否则影响整体动销效率
- 社区客群对价格敏感度通常较高，P域主力价格带的设定需结合本地客群消费水平

## 23. 常见误区

| 误区 | 正确理解 |
|------|---------|
| 商品配置 = 选品清单 | 商品配置是结构设计，清单是其输出之一 |
| SKU 越多越好 | SKU 数量有合理上限，冗余会降低整体动销和陈列效率 |
| 只看销量决定 SKU | 需同时看价格结构、品种完整性和顾客购买路径 |
| IPC 是陈列图 | IPC 是价格结构分析图，同时反映价格感知和库存现金流风险，与货架排布图不同 |
| CDT 是商品分类表 | CDT 是顾客决策路径，不是企业内部分类逻辑 |
| HS矩阵空格就是遗漏 | 矩阵空格是潜在遗漏，需结合客群数据判断，不可直接填入 |
| 新店和老店用同一顺序 | 新店：CDT→HS→IPC；老店诊断：IPC→HS→CDT，顺序相反 |
| 商品配置做一次就够 | 应随季节、客群结构、竞品变化定期更新 |
| 商品配置可脱离现场条件 | 必须结合真实货架尺寸、包装规格和门店动线校验 |
| 毛利率高的商品一定应该多配 | 高毛利但低周转商品可能拖慢现金流，必须结合库存周转和 GMROI 综合判断 |
| 低毛利商品一定应该淘汰 | 部分低毛利商品承担 P域现金流、价格形象、顾客完整性或连带购买任务，不能只看毛利率 |
| IPC 只用于价格分层 | IPC 同时反映价格结构、顾客价格感知和库存现金流风险，需综合解读 |
| 六分仪只看销售占比 | 六分仪要同时比较 SKU数、陈列面数、销售额、毛利额、毛利率和综合贡献指标，不能只看单一维度 |
