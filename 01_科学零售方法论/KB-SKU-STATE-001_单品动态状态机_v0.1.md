## 1. 核心论点（结论先行）

**SKU 有两套正交的标签：角色（它应该做什么）是相对稳定的战略定位，状态（它现在做得怎么样）是频繁变化的运营健康。** 状态机的任务是：用可从 POS 直接算出的少数几个指标，把每个 SKU 实时归到五态之一，五态直接输出四个运营动作（留/退、促/不促、增排面/缩排面、备货/控库），并向上提供两个独立信号供品类状态机交叉验证（见 §8）。

**v_median 基准铁律（v0.1r2 修正核心）**：速度达成率 R 的分母永远用**历史固定基准 v_median_base**（滚动6个月，对齐同比同期），而非当期品类中位——当品类整体下滑时，用当期 v_median_live 会导致分母随之缩小，原本🔴的 SKU 读数虚高而漏报。v_median_live 只用于§7 第二关横比闸，判断"是品类整体问题还是单 SKU 的问题"。

**一句判断门槛**：状态永远是"数据触发、自动迁移"的；角色永远是"人工定性、相对锁定"的。二者不得混为一谈——不能因为一个 SKU 这周卖得差（状态🔴），就改它的角色（把形象品降成待淘汰品）。角色的调整是另一套决策，须单独判断。

## 2. 角色 vs 状态：正交双轴（本页的地基）

| 维度 | 角色（Role · 静态定位） | 状态（State · 动态健康） |
|---|---|---|
| 回答什么 | 它**应该**做什么 | 它现在**做得怎么样** |
| 变化频率 | 相对稳定（季度级） | 频繁（周级/日级） |
| 谁决定 | 人工定性判断（六哥/顾问） | 数据自动触发迁移 |
| 载体 | 见 [[KB-SKU-ROLE-001_SKU四维角色体系_v0.1]] | 本页五态 |
| 举例 | 引流品 / 利润品 / 形象品 / 补缺品 | 🟢健康 / 🟡预警 / 🔴滞销 / 🔵待调整 / ⚪退市候选 |

> **兼容声明**：本页采用 [[KB-CAT-ARCH-001_全品类双投影架构_v1.2]] 裁决的 per-SKU 角色属性口径——引流(`is_traffic`)/利润(`is_margin`)/形象(`is_image`)/补缺(`is_fill`)四个布尔字段，加必填字段 `primary_role`（见 §4.1 多角色规则）。

**为什么必须正交**：角色决定"期望值"，状态度量"达成度"。同样是每天卖 2 件——对形象品是健康（它本就不指望走量），对引流品是滞销（它的存在意义就是拉量）。**脱离角色谈状态没有意义**，所以状态判定的第一步永远是"先读它的 primary_role，再取对应的期望基准"。

## 3. SKU 五态定义（含图标 · 进入/退出条件）

| 状态 | 图标 | 一句话定义 | 核心运营含义 |
|---|---|---|---|
| 动销健康 | 🟢 Healthy | 销售符合角色预期，库存合理，近14天活跃 | 维持现状，正常销售，排面不动，正常备货 |
| 动销预警 | 🟡 Warning | 销售低于角色预期 10–30%，或库存偏高（周转 1.2–1.5×基准） | 观察 + 轻促销 / 微缩排面，控制补货 |
| 滞销危险 | 🔴 Stagnant | 销售严重低于预期（>30%），或库存周转 >1.5×基准 | 强促销 / 缩排面 / 停止备货，进入倒计时 |
| 待调整 | 🔵 In-Adjustment | 已挂**有效**改造计划（满足§3.2四要素门控），状态被人工挂起 | 按改造方案执行，暂停自动降级，计时观察 |
| 退市候选 | ⚪ Candidate-Out | 连续 N 周期🔴 + 无持续改善迹象，进入淘汰评估 | 停止备货 + 清库存，等人工确认退市（不自动执行） |

### 3.1 状态迁移图

```
                   ┌──────── 有效改造计划(四要素门控) ──────┐
                   ▼                                     │
  🟢 Healthy ──速度/库存恶化──► 🟡 Warning ──继续恶化──► 🔴 Stagnant
     ▲   ▲                        │  ▲                    │   │
     │   └──── 数据持续回暖 ───────┘  └── 数据持续回暖 ──────┘   │
     │                                                         │
     └─── 改造见效(持续达成率回到🟢) ─ 🔵 In-Adjustment ◄────────┤
                                          │  (改造无效/超窗)     │
                                          ▼                    ▼
                                连续N周期🔴且无持续改善 ──► ⚪ Candidate-Out
                                                            │
                                                过三关+六哥签字(§7)
                                                            ▼
                                                       退市/下架
```

### 3.2 迁移守护条件（Guard，写死）

- **只能逐级降级或跨级升级**：🟢→🟡→🔴 逐级下滑；任一环节数据**持续**回暖（须满足§6改善判定新规则）可**直接跳回🟢**（不必逐级爬）。
- **🔵是人工挂起态，须四要素门控**（v0.1r2新增）：有效改造计划须同时包含以下四要素，缺一不满足🔵进入条件：
  - ① **具体动作**（调价/调陈列/引入替代SKU/推荐位申请，不得写"综合改善"）
  - ② **量化验收目标**（如"R ≥ 0.85"或"OSA > 90%"，须可自动核查）
  - ③ **责任人**（姓名/职位，不得写"门店"）
  - ④ **截止日期**（具体日期，不得写"下个月"）
  - 四要素缺一：🔵不生效，SKU继续计🔴周期，`consecutive_red_periods`继续累加。进入🔵后改造观察窗 N 个周期（生鲜2周/快消4周）。改造窗内持续改善则回🟢；窗满仍🔴则转⚪。
- **进⚪必须同时满足**：连续 N 周期🔴 + 无持续改善迹象（见§6）+ 通过§7三关。**⚪不等于退市**，只是"进入退市评估队列"；真正退市须六哥/顾问签字（D档，不可自动执行）。
- **数据质量闸**（复用 KB-DX-FULLSTORE-001 证据等级 A–E）：POS 数据证据等级 < C（仅访谈/单项现场，无连续动销流水）时，SKU **不得进⚪**，最多停在🔵待验证——数据不足时状态机自动降级为"疑似"，不触发不可逆动作。

## 4. 核心指标（含公式 + 数据来源）

> 全部指标以 **POS 动销流水**为主源，**库存快照**为辅源；口径统一"近30天"（与 KB-DX-FULLSTORE-001 D1-5 空销率 30天口径一致），活跃率用"近14天"。

### 4.1 指标一：销售速度达成率 R（速度维度 · 核心）

```
实际速度 v_actual = 近30天销量(件) / 30                    [数据源: POS 动销流水]

【基准拆分·v0.1r2 修正核心】
v_median_base = median(同L4品类活跃SKU · 滚动6个月历史数据)  [历史固定基准，用于R分母]
v_median_live = median(同L4品类活跃SKU · 当期近30天数据)     [当期基准，仅用于§7第二关横比闸]

角色期望速度 v_expected = v_median_base × k_role           [分母永远用历史基准，不随品类整体下滑漂移]
达成率 R = v_actual / v_expected
```

> **为什么必须用 v_median_base 而非 v_median_live**：梅雨季（4–6月）湿叶菜全类滞销时，v_median_live 从8件/天跌至5件/天；一个实际只卖3件/天的 SKU 用 v_median_live 算出 R=0.60（🟡），但用历史基准算是 R=0.375（🔴）。状态机若在品类最需要预警时自动调低标准，与设计意图正好相反。v_median_live 保留但只做§7横比闸的"品类整体下滑判断"用途，不入 R 公式。

**角色系数 k_role（v0.1r2 修正：multi-role 规则重写）**：

| 角色（is_* 布尔）| k_role | 依据 |
|---|---:|---|
| 引流品 is_traffic | 1.5 | 存在意义=拉量，应显著快于品类中位 |
| 利润品 is_margin | 1.0 | 走量与毛利平衡，对齐品类中位；同时受"贡献毛利等级"门槛约束（§4.3） |
| 形象品 is_image | 0.5 | 走量非其职责，低速属正常；主指标已改为 OSA（§4.6） |
| 补缺品 is_fill | 0.6 | 填价格带/规格空白，低速可接受，主看"缺口是否仍存在" |

**多角色 SKU 规则（v0.1r2 重写，废弃旧版"取最高k_role"）**：

1. SKU 设置角色时，人工**指定 `primary_role`**（主职能，单选），对应布尔值置 true。
2. k_role 按 `primary_role` 取上表对应值，非主职能角色作**辅助评估维度**（不影响状态判定）。
3. **强制约束**：`is_image = true` 时，k_role 上限为 **0.8**——无论 `primary_role` 是什么，形象品不得被要求以引流标准衡量走量（防止大湾区进口高价海鲜等"形象+引流"双职能品被误判为滞销进而强促销，损伤价格锚点）。
4. 如六哥/顾问认为某 SKU 兼任引流须高标准 → 须人工 override `k_role` 字段并**记 DEC**（不可静默覆盖）。

### 4.2 指标二：库存周转倍数 M（库存维度 · 核心）

```
库存天数(动销天数) = 当前库存量(件) / 日均销量(件/天)     [库存快照 ÷ POS]
       日均销量 = 近30天销量 / 30
周转倍数 M = 实际库存天数 / 品类基准周转天数
```

**品类基准周转天数**直接调用 [[KB-DX-MATH-001_零售数学诊断工具体系_v0.1]] RM-06 大湾区社区超市健康基准：

| 品类 | 基准周转天数 | | 品类 | 基准周转天数 |
|---|---:|---|---|---:|
| 生鲜(叶菜) | 2–3天 | | 熟食烘焙 | 1–2天 |
| 生鲜(肉禽) | 3–5天 | | 工业食品 | 15–30天 |
| 酒水饮料 | 10–25天 | | | |

> **铁律沿用**：SKU 库存天数不得使整品类综合周转天数 > 结算周期（通常30–45天），否则占压现金流（RM-06 铁律）。

### 4.3 指标三：贡献毛利额 GM$（不只是毛利率）

```
贡献毛利 GM$ = 近30天销量 × (售价 - 进价)     [POS销量 × (售价 − D层进价)]
```

- **为什么看金额不看率**：一个毛利率 40% 但每天卖 0.5 件的 SKU，贡献毛利可能远低于毛利率 18% 但日销 10 件的 SKU。退市评估看**它到底给门店赚了多少绝对毛利钱**。
- 利润品(is_margin)额外门槛：GM$ 低于该 L4 品类利润品 GM$ 中位数的 50% → 即使速度达标也降🟡（它没尽到利润职责）。
- 进价属 D 层 `raw_sensitive`，只在本地只读环境参与计算，**不写入正文、不入对外交付**（§隐私）。

**贡献毛利等级（v0.1r2 新增·退市签字可见字段）**：D 层计算完成后脱敏为字母等级，写入 E 层退市⚪队列，签字人看等级不看原始进价：

| 等级 | 定义 | 退市签字含义 |
|---|---|---|
| S | GM$ > 品类利润品 GM$ 中位数 | 高毛利贡献款，退市前须重点确认 |
| A | 50%–100% 中位数 | 中高贡献，退市风险中等 |
| B | 20%–50% 中位数 | 中低贡献，可退但留意替代方案 |
| C | < 20% 中位数 | 低贡献，退市优先候选 |

> 等级映射在本地只读环境（D层）完成，字母进 E 层，不违反 D 层 `raw_sensitive` 隔离规则。

### 4.4 指标四：陈列效率 DE（销售陈列比）

```
陈列效率 DE = 该SKU销售占比 / 该SKU陈列排面占比     [直接调用 KB-CAT-DX-MISMATCH-001]
```

- 复用 [[KB-CAT-DX-MISMATCH-001_销售陈列错配诊断_v0.1]] 的销售-陈列比：DE > 1.5 = 增排面候选；DE < 0.5 = 缩排面候选。
- **DE 是"排面动作"的唯一裁决指标**，与状态正交：一个🟢健康 SKU 若 DE > 1.5，动作是"增排面"；一个🔴滞销 SKU 若 DE < 0.5，动作是"缩排面 + 促销/停备货"。状态管"留退促备"，DE 管"排面增缩"。

### 4.5 指标五：SKU 活跃率 / 活跃闸（近14天）

```
活跃 = (近14天销量 > 0) ? true : false     [POS 动销流水]
```

- 近14天无销售 → 状态**至少🟡**（无论速度库存多好，14天0动销本身即预警）。
- 近30天无销售 → 命中 KB-DX-MATH-001 空销/Z类雏形，状态**至少🔴**。
- 与 IM-04 ABC-Z 衔接：连续3个月 0 动销 = Z 类 = ⚪退市候选的最强触发条件（见 §7 保护规则的时间尺度）。

### 4.6 指标六：在架完整性 OSA（形象品专属 · v0.1r2 新增）

> **仅 is_image = true 时触发**，替换速度 R 作为形象品的主要🔴触发判据。

```
OSA = (近14天在架天数) / 14
     等效：OSA = 1 - (近14天断货天数 / 14)     [数据源: 库存快照·日级或每日盘点记录]
```

**形象品健康判定规则（与 is_image=false 的 SKU 不同）**：

| 触发类型 | 条件 | 含义 | 状态结果 |
|---|---|---|---|
| 主触发① | OSA < 80%（近14天断货 ≥3天） | 没货=没形象，形象职能失效 | → 🔴 |
| 主触发② | GM$ 等级 C（<品类形象品中位数20%） | 连形象职能的毛利支撑也没有 | → 🔴 |
| 辅助参考 | R < 0.7（速度严重低于预期） | 形象品设计为低速，单独低R不触发🔴 | 仅记录 |

> **设计理由**：形象品的职能是"塑造品质感知"，核心失效场景是"没货"而非"卖得慢"；进口高价海鲜日销1–3件但OSA=100%，是正常运营的形象品，不应触发强促销（促了损伤价格锚点）。只有当它连货都没得卖、或者连毛利支撑都没有时，才定为🔴。

## 5. 状态判定函数（把五个指标合成一个状态）

**取"最差维度"原则（fail to worst）**：一个 SKU 的状态 = 速度维度状态、库存维度状态、活跃闸三者中**最差的那个**，再叠加形象品 OSA 逻辑、人工挂起（🔵）与退市队列（⚪）逻辑。

### 5.1 速度维度状态（由 R 得）

**节令系数修正（v0.1r2 新增，集成 KB-SEASON-MATRIX-001）**：R 的判定阈值须随节令调整，避免旺季误判过速健康、淡季深度误杀季节品。

```
f_season = 来自 KB-SEASON-MATRIX-001 §1 接口协议的节令修正系数
           f = 1 + v × K（其中 v=节令强度，K=品类敏感系数，详见 SEASON-MATRIX-001）

R_threshold_green = 0.9 × f_season    [健康基准线]
R_threshold_red   = 0.7 × f_season    [滞销基准线]

注意边界：f_season ∈ [0.5, 1.1]；
  旺季(f>1.0)：R_threshold升高，防止旺季过速被误判为健康
  深度淡季(f<0.6)：R_threshold降至不触发🔴阈值，防止深度淡季误杀季节品
```

| 达成率 R 与节令调整后阈值关系 | 速度状态 |
|---|---|
| R ≥ R_threshold_green | 🟢 |
| R_threshold_red ≤ R < R_threshold_green | 🟡 |
| R < R_threshold_red | 🔴 |

> 当 KB-SEASON-MATRIX-001 未落地时（数据不足），f_season=1.0，阈值退化为原始值 0.9/0.7。

### 5.2 库存维度状态（由 M 得）

| 周转倍数 M | 库存状态 |
|---|---|
| M ≤ 1.2 | 🟢 |
| 1.2 < M ≤ 1.5 | 🟡 |
| M > 1.5 | 🔴 |

### 5.3 合成规则（伪代码）

```python
# 1. 基础状态：取最差维度
state_speed     = get_speed_state(R, R_threshold_green, R_threshold_red)   # §5.1
state_inventory = get_inventory_state(M)                                     # §5.2
state_active    = get_active_state(活跃14天, 活跃30天)                        # §4.5

state = worst(state_speed, state_inventory, state_active)   # 🟢<🟡<🔴

# 2. 形象品专属 OSA 修正（is_image=true 时覆盖速度R的🔴判定）
if is_image:
    if OSA < 0.80 or gm_grade == 'C':
        state = max(state, 🔴)   # 主触发条件满足，至少🔴
    # 单独 R<0.7 不升级state（辅助参考，不触发🔴）

# 3. 人工挂起态（须满足四要素门控）
if has_valid_adjustment_plan and adjustment_window_not_expired:
    state = 🔵    # 暂停自动降级

# 4. 退市候选队列
if (consecutive_red_periods >= N
        and not has_sustained_improvement()    # §6 改善判定新规则
        and evidence_grade >= 'C'              # 数据质量闸
        and passed_three_gates()):             # §7 三关
    state = ⚪

# 5. 记录时间戳（解决"停留多久"缺口）
```

### 5.4 五态 → 四个运营动作（直接可执行）

| 状态 | ①留/退 | ②促销 | ③排面（叠加 DE 判定） | ④备货 |
|---|---|---|---|---|
| 🟢 | 留 | 正常销售 | DE>1.5 增 / 否则不动 | 正常备货 |

## 6. 状态时间戳与超时规则（补现有体系空缺）

> 现有体系无"状态进入时间"字段——无法计算 SKU 在某态停留多久。本页补齐：

- 每个 SKU 持久化字段：`state`（当前态）、`state_entered_at`（进入本态时间戳）、`consecutive_red_periods`（连续🔴周期计数）、`R_entry_baseline`（进入🔴时的 R 值，用于改善判定基线）。
- **超时降级**：🔴 停留达 N 个周期（生鲜N=2周/快消N=4周）且无持续改善 → 自动转⚪队列。

**改善判定（v0.1r2 重写·废弃旧版单期点对点比较）**：

旧规则（废弃）：本周期 R 较上周期回升 ≥ 20% 相对幅度即归零。问题：大湾区生鲜自然波动±15–30%，单次好周末即可触发，慢性滞销品可靠自然噪音无限规避⚪。

新规则：
```
持续改善判定（须同时满足）：
  ① 最近 ceil(N/2) 期的 R 均值（滚动平均）比 R_entry_baseline 高出 ≥15%（绝对幅度）
  ② 连续 ≥2 期 R 均值高于 R_entry_baseline
  → 同时满足①②：视为持续改善，consecutive_red_periods 归零，记录新的 R_entry_baseline

单次自然反弹（如一个好周末）：R 点值回升但不满足②（仅1期）→ 不归零，继续计🔴
```

**生鲜 SKU 额外规则**：v_actual 改为工作日均速加权值，减少周末聚集效应干扰：
```
v_actual_adjusted = (工作日日均销量 × 5 + 周末日均销量 × 2) / 7
（按实际日历比例加权，避免把周末聚集销售摊入日均时虚高）
```

- **周期定义**：生鲜以"周"为周期（高频），快消以"周"计数但阈值 N 更长（4周），与 §7 退市保护 N 值一致。

## 7. 退市决策保护规则（重要 · 三关 + 签字，不可自动执行）

> **红线：不能因为一个 SKU 一时状态差就自动退市。** ⚪只是"进入评估队列"，退市动作必须过以下三关且经人工签字。此为 KB-DX-MATH-001 IM-09 删品框架在状态机层的强化。

**第一关 · 时间持续性（连续 N 周期🔴）**

| 品类类型 | N（连续🔴周期） | 依据 |
|---|---:|---|
| 生鲜 | 2 周 | 高周转、坏得快，2周足以定性 |
| 快消/工业品 | 4 周 | 低周转、季节波动大，需更长窗口避免误杀 |

> 与 IM-04 Z 类（连续3个月0动销）的关系：Z 类（3个月0动销）是**绝对退市触发**，直接入⚪无需再等；本关的 N 周期🔴 是**相对速度滞销**的触发，须叠加第二关横比。两条触发并行，取先到者。

**第二关 · 同品类横向对比（用 v_median_live 判断是不是"这个 SKU 的问题"）**

```
判断基准：v_median_live 与 v_median_base 的差值

若 v_median_live 较 v_median_base 同期下滑 >20%
   → 品类整体下滑（客流/季节/门店问题），不归咎单 SKU
   → 不进⚪，升级为品类级诊断（由 KB-CAT-STATE-001 处理）

若 v_median_live 平稳（与 v_median_base 差值 <20%），而该 SKU 独自下滑
   → 确认是这个 SKU 的问题 → 可进第三关
```

> **注意**：v_median_live 在此处是"品类整体健康探针"，与 R 公式中的分母完全分离。两个数值各司其职，不混用。

> 季节性上下文闸：季节品在**非窗口期**（且 f_season < 0.6，见§5.1）的低速属正常，不触发🔴/⚪。须先确认"当前是否窗口期"再判退市（复用 KB-SEASON-MATRIX-001 窗口期定义）。

**第三关 · 人工确认（六哥/顾问签字，D档）**

- ⚪队列清单由状态机自动生成 → 提交六哥/顾问 → **逐个签字**方可退市。
- **队列清单必须包含贡献毛利等级字段**（§4.3 S/A/B/C，v0.1r2 新增）：签字人须知道每款退市候选的毛利量级，防止把 S/A 级高毛利款批量签掉。等级=C 者优先退市，等级=S 者须重点说明为何退市。
- 保留条件（IM-09 沿用，满足任一可暂保）：CDT 中独特占位无替代品 / 上架 <3个月新品 / 季节品待旺季 / 厂商谈判战略品。
- **状态机永远不自动执行退市/下架**——这是 90/10 护城河的判断相，不下放（治理 §3④、§7 模型分层调度）。

## 8. 汇总规则（SKU → 品类层交叉验证信号）

> v0.1r2 修正：本层向品类状态机提供两个**独立补充信号**，用于交叉验证，不作为 KB-CAT-STATE-001 的直接数据源输入（品类层V1/V5由CAT-STATE独立从POS计算，见§11接口说明）。

### 8.1 SKU 滞销压力分布（品类内退市候选与滞销占比）

```
退市候选率(⚪/总) = 品类内 ⚪退市候选 SKU 数 / 品类总在架 SKU 数
滞销压力率       = (🔴 + ⚪) SKU 数 / 总SKU数   [领先信号，比⚪/总更早预警]
```

> **V5口径澄清（v0.1r2 修正）**：本层的"退市候选率（⚪/总）"与 KB-DX-FULLSTORE-001 D1-5 的"30天0动销率"是**两个独立信号，口径不同、数值不会相等**：
> - 本层⚪：需连续N周期🔴+无持续改善+通过三关才入⚪，是**慢信号**（时间尺度：周～月）
> - D1-5 空销率：30天截面，只要0动销即触发，是**快信号**（时间尺度：月）
> - 两者差值 = "近期新进入🔴但尚未到⚪的SKU"/总，差值越大说明品类正在快速积累潜在退市压力（比⚪/总更领先的预警）
> - KB-CAT-STATE-001 V5 由其自身从 POS 独立计算（30天口径，见CAT-STATE §8.2口径分工铁律），本层的⚪/总作为**补充领先指标单独追踪**，不作为 CAT-STATE V5 的数据源

### 8.2 SKU 层健康品贡献趋势（辅助交叉验证视角）

```
                  Σ_i ( 🟢SKU_i 的销售趋势_i × 贡献毛利_i )
健康品趋势 = ─────────────────────────────────────────────────
                         Σ_i ( 🟢SKU_i 的贡献毛利_i )

其中 销售趋势_i = (近14天日均速度 − 前14天日均速度) / 前14天日均速度
求和范围 i ∈ {该品类内所有 🟢健康 SKU}
```

- **只取🟢健康 SKU** 算趋势：滞销 SKU（🔴/⚪）的趋势噪音大且即将退出，纳入会污染品类真实动能判断。
- **按贡献毛利加权**：大毛利贡献的 SKU 趋势对品类的意义更大，避免长尾小品把趋势带偏。

> **V1接口澄清（v0.1r2 修正，选B方案）**：本指标是 SKU 层的健康品贡献趋势视角，与 KB-CAT-STATE-001 的品类层 V1（由 POS 直接计算4周销额环比/同比，所有SKU全量）**是两个独立计算的信号**，时间窗口（14天 vs 28天）、SKU范围（仅🟢 vs 全量）、计算路径均不同。两者结论可互为交叉验证（若两者方向相反，说明品类内部有结构分化，值得诊断），但不互为数据源，不得混用。

> **7D接口（待建能力）**：品类状态分布理论上可作为7D框架D1/D2维度的动态数据源（替代静态打分）。该能力需 KB-DX-FULLSTORE-001 在 D1 评分定义中建立对应接收接口后方可生效；当前7D D1/D2仍按独立指标（D1-1至D1-5）独立打分，本层不声称已成为7D动态数据源。该接口列为 planned 能力，待KB-DX-FULLSTORE-001 v0.2 实现。

## 10. 失效条件（这条状态机什么时候会误判）

1. **新品适应期**：上架 <30天的 SKU 无稳定动销基线，R/M 均不可信 → 强制进🔵待验证，不进🔴/⚪（对齐 KB-DX-MATH-001 IM-09 新品保留条款）。v_median_base 也不应纳入新品数据。
2. **季节品非窗口期**：f_season < 0.6 的深度淡季，R_threshold_red 自动降低，大多数季节品不触发🔴（见§5.1 节令修正）。若节令数据不可用，须手工核查"当前是否窗口期"再判退市。
3. **品类整体下滑被 v_median_base 正确过滤**：本版已将 R 的分母改为历史固定基准，品类整体下滑时个别 SKU 不再因分母同步缩小而虚高。但 v_median_base 须定期更新（建议按季，重大品类调改后立即更新）——若 v_median_base 过期（超过12个月未更新），自动降级为"参考值"并标注⚠️，不触发不可逆退市。
4. **数据质量不足**：POS 流水缺失/证据等级 <C 时，状态机只能标"疑似"，最多🔵，不触发不可逆退市（§3.2 数据质量闸）。
5. **大规模调改后 30 天内**：门店刚做完 SKU 大改，全品类速度处于磨合期摩擦，此期间的🔴不代表真实滞销 → 参照 KB-DX-MATH-001 §失效条件的调改保护期，30天内 R 下滑 <15% 不触发降级。
6. **OSA 数据缺失（形象品）**：门店未落实日级库存快照时，OSA 无法计算 → 形象品退化为用速度 R 判断（与非形象品相同规则），须在诊断报告中注明"形象品 OSA 不可用，评估精度降级"。
