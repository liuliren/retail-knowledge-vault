# 脚本缺口评估报告 draft（Y3 · 任务书10问逐答）

**Q1 现有脚本覆盖了哪些环节，断在哪里？**
POS原始→标准表（pos_clean）→三件套分析（trio_engine）→诊断卡（diagnose/mdcard）→提交门禁（pre-commit hook）→vault体检（lint_v2）已成链。**断点**：①治理侧（M-DEC promote、frontmatter状态、证据包索引、RESUME生成）零脚本，全靠 LLM 手工；②品类表→POS 映射刚由 category_pos_cleaner v0.1-0.3 补上但仍 mock 阶段。

**Q2 文档与代码是否一致？**
不一致实锤一处：`G03_Lint_v2/README.md` 写"六查体检仪表盘"，`lint_v2.py` 实做约13类检查（signoff审计、SCHEMA豁免区、execute前置登记表存在性等均超出六查）。README `updated: 2026-06-22` 停更，代码已迭代到 07-06。另 publish skill 正文版本号漂移（v0.4 vs v0.2）同属此类。

**Q3 测试覆盖真实状况？**
两极分化：知识库自动化_v1 有 9 个测试文件（但该包疑似闲置）；生产主干 lint_v2 仅 2 个测试文件（test_excludes、test_canonical_link_target），**敏感正则（客户真名/裸值检测）零测试**；tests/test_engines_regression.py 仅导入级+2断言；mdcard/build_master/7D/脱敏链路零测试。测试投入与资产重要度倒挂。

**Q4 pre-commit hook 防线是否有洞？**
有。扩展名硬停名单含 xls/xlsx/csv/db/ppt/key/pdf/dwg/tar；魔数嗅探只认 zip/OLE/PDF 三类签名。**rar、7z、gzip/tar、sqlite 改后缀为 .md/.txt 可绕过**内容嗅探。属红线件，本 Phase 只建议不改：补 `Rar!`、`7z¼¯'`、`\x1f\x8b`、`SQLite format 3` 四签名，走六哥 C 档。

**Q5 是否存在双入口/重复造轮子？**
三处：①花厅坊POS清洗脚本_v0.1（旧管道）vs POS清洗库_v0.1（新权威入口）——旧件文件头虽声明"不重叠"，实际功能被取代大半；②脱敏测试链路 sanitize/analyze vs /diagnose 的 diagnose.sh；③知识库自动化_v1 的 inventory/dashboard vs lint_v2。均需消歧或归档（C档清单见Y9）。

**Q6 口径 SSOT 是否落实？**
部分落实：pos_schema.py 是正面样板（07-06 归一零漂移）。未落实：ABCZ 阈值（A≤70/B≤90）在 trio_engine 硬编码且 skill 自标"draft标准待审议"；gate_checker G字母口径与验收框架文档不逐字重合（README已声明但仍是双解释源）。

**Q7 哪些脚本可跨客户复用，哪些绑死单店？**
高复用：pos_clean、trio_engine、retail_clean、sku_extractor、mdcard、lint_v2、category_pos_cleaner。绑死单店：花厅坊POS清洗脚本（config 硬编码）、selftest_huachangfang、trio skill 里的花厅坊默认路径。趋势正确：新脚本（barcode_gap_classifier 带 --store 参数）已参数化。

**Q8 安全边界执行情况？**
好于预期：sanitize.py（裸值不进上下文）、build_master（裸值只进 gitignore 目录）、照片预处理（只读源）、convert_xls（输出/tmp）均内置边界声明。缺口：这些边界**只有声明没有测试**——如 sanitize 输出是否真 0 裸条码，无断言保障。

**Q9 疑弃/闲置件几何？**
4 项：花厅坊POS清洗脚本_v0.1、G03_Lint_双链巡检脚本_v0.1（仅README无代码）、知识库自动化_v1（launchd 挂载状态待核实）、selftest_huachangfang.py。均不可删（D档），归档移动是C档，本 Phase 只列清单。

**Q10 最该新增什么脚本？**
按"治理侧零脚本"缺口排：①frontmatter状态checker（status污染/active缺signoff 是lint消债主要弹药来源）②M-DEC promote 就绪度checker（promote skill 的机械相）③RESUME/LOOP_STATE 骨架生成器（handoff 半人工）④证据包索引器（16_/花厅坊样板证据件散落）。四件已在本 Phase tool_drafts/ 落地为 v0.1（全部只读+dry-run）。
