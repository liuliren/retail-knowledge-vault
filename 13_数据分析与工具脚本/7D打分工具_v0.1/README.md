# 7D 现场打分工具 v0.1（draft·未经校准）

> 补 988 首诊产品交付链的硬断点：[[KB-DX-PRODUCT-001_零售企业首次诊断产品_装配框架_v1.0]] §三 缺口①「7D 诊断打分件」。原设想为 Excel，本 v0.1 以 Python 脚本先跑通（零第三方硬依赖，Excel 版可后续再议）。

## 使用三步

1. **复制模板**：`cp 打分单模板.yaml 打分单_<门店代号>_<日期>.yaml`（门店一律用代号，不写真名）。
2. **到店填单**：每个子项填 `score`（0-5，5=满分锚点达标）+ `evidence`（A-E 证据级）+ `note`（证据一句话：观察→锚点→判定）。没数据的子项整个留空。`meta.fresh_ratio` 填 POS 生鲜占比（触发 §8 校准），`employee_mode` 按雇员数填 full/gray/none。
3. **跑脚本出分**：`python3 score_7d.py 打分单_XX.yaml`
   → 控制台出 7 维得分表 + 证据分布 + 总判定；同目录生成 `7D打分结果_<代号>_<日期>.md`（ASCII 条形 + 证据链明细）；matplotlib 可用时附雷达 PNG（失败不阻断）。

示例：`python3 score_7d.py 示例_DEMO.yaml`（DEMO 为虚构数据，非真实客户）。

## 铁律

1. **权重来源 = 装配框架链**：七维固定权重（20/25/10/15/15/10/5）与生鲜占比校准公式照抄 [[KB-DX-FULLSTORE-001_全店7D诊断框架2.0_v0.1]] §1 权重表、§8 校准公式、§3-D6② 雇员再分配——**一字未改**。该框架本身 status=draft（§9① 权重为花厅坊单例 inferred，未过三轮审议、未签字），故**本工具全部输出同为 draft，未经多店校准，不得被引用为已生效标准**。
2. **C 级证据禁精确%**（2026-07-04 花厅坊首诊卡 v1.1 修订纪律）：证据 C/D/E 级的维度只输出档位（强/中偏强/中/中偏弱/弱）+ 证据级，自动标「仅方向性参考·禁写精确%」；7 维证据级不齐时不凑对外精确总分，只出「结构化判定」——B 级以上维给"半张卡"得分率，全维加权分仅标内部参考。
3. **判断相在六哥**：工具只算加权算术。打分裁量、维度证据级覆盖（`override_grade`）、D8 顾问综合判断、最终签字均归六哥（90/10 的 10%，不可代签）。D8 待补状态下产出不得作为正式交付件对外输出（框架 §5）。
4. **客户机密**：打分单与结果文件一律用门店代号；示例只用虚构数据。

## 已知局限（v0.1）

1. 维度证据级默认取子项**最弱**一级（fail-safe 最严格），可能偏保守——D2 有多个 B 级子项时仍可能被一个 C 级子项拖成 C 维；六哥可用 `override_grade` 裁量覆盖。
2. 灰区（1-4 雇员）D6 让出的 5 分并入 D2:D5=6:4 是**工具层推断**（框架只写了夫妻店 10 分的去向），待校准。
3. 档位切点（80/65/50/35）与子项 0-5 打分制为工具层约定，非框架原文，未经审议。
