# category_table_v1.csv 说明 · draft

> **声明**：本 csv 是**结构示例/实验件**，L4 行为虚构示例，**非 V6.0 替代品，V6.0 仍是唯一 SSOT**。C 码取自 KB-CAT-ARCH-001 公开正文。

## 必答 8 问

1. **本轮目标**：把"只存在于 xlsx 和 md 表格里的品类结构"变成一份机器可读、可 JOIN、可 diff 的开放 CSV 骨架（评估报告问题 #1/#4/#10 的解）。
2. **改了什么**：① 每节点一行、显式 `category_code`/`parent_code` 父链；② L1 收敛为 生鲜/食品/非食 三大投影（FRESH/FOOD/NONFOOD）；③ L2 给了显式编码（A1/C1/G1…）；④ 新增 `X99 未分类` 官方 fallback 节点；⑤ 91 节点覆盖 休食/冲调/方便食品/生鲜/非食 五域。
3. **解决什么**：程序第一次可以直接 `read_csv` 拿到完整父链做映射与校验；穷尽性有了显式兜底。
4. **新风险**：① 与 V6.0 真实 385 L4 不对齐——若被误当主数据引用会造成口径分裂（已用 status=draft + 本声明防护）；② L1 三投影与表A 九区（A–I）是两种切法，需在说明层讲清楚"三投影=顶层汇总轴，A–I=顾客动线轴"；③ 生鲜 A 系仍是过渡口径。
5. **POS 映射适配**：仅提供结构，**尚无 keywords/alias**——映射器只能做"原POS分类字段→模糊对表名"，命中率低，脏数据无防护。
6. **社区超市适配**：节点粒度按 500–2000㎡ 社区生鲜超市裁剪（无百货商场级深度）；家居百货只保 I1/I5 两组，符合 I 域防扩张纪律。
7. **诊断适配**：无诊断字段；诊断仍需人工圈品类。
8. **是否推荐进下轮**：**是**。骨架成立，v2 在其上加映射知识字段。

## 字段

`L1,L2,L3,L4,category_code,category_name,parent_code,status,notes`（9 列；L1–L4 为路径名文本，code 为机器键）。
