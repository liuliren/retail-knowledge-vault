# 三件套引擎 v0.1

单文件 CLI（`trio_engine.py`），四个子命令，是 4 个 skill（/trio /abcz /movement /priceband）的共享执行层——**一份代码，四个入口，杜绝副本漂移**。

## 子命令

| 子命令 | 输入 | 输出 | 说明 |
|---|---|---|---|
| `clean` | 分品类销售xls目录（POS「商品-分部汇总」34列打印式报表） | `sales_master.csv` | 逐表与系统合计行对账，偏差>0.5% 报警；自动排除生鲜大表/食品大类合表（冗余/截断） |
| `abcz` | master.csv + 档案xls目录 | ABCZ_SKU级.csv + Z类清单.csv | **小类内口径**（签字铁律），A≤70%/B≤90% 累计净额 |
| `movement` | 同上 | 动销率_小类.csv | 分母=在册未停购；停购字段未维护时输出预警 |
| `priceband` | master.csv | 品类概览_价格带毛利.csv | P25/中位/P75 + 差价率（生鲜侧自动标注不可信） |
| `all` | master.csv + 档案目录 | 全部四件 | 单次加载，顺序执行 |

## 内置口径（改口径=改代码=须六哥签字）

1. ABC 小类内，禁全场（[[SKU角色层与目的品保护机制_v0.1.6]] §2.3）
2. Z 类 = 在册未停购 × 期间零动销
3. 毛利双轨：标品=POS差价率；生鲜=档口月度倒算（引擎只提示，不硬算）
4. 校准参照：A 类健康区 ≈15-17% SKU→60% 销售（零售老刘）

## 已知边界

- 列映射绑定当前 POS 导出版式（34列打印报表）；店方改用标准导出后需重映射
- 档案表自动识别两种版式（r2紧凑/r3标准）；新版式会打 ⚠️ 跳过
- 输出 csv 均落客户工作区，受 .gitignore 保护不入 git
