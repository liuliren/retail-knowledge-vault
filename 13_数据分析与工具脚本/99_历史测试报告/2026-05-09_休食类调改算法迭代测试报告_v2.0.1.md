# 休食类调改算法迭代测试报告 v2.0.1

> **本报告**：5/8 客户方提供原始 POS 数据 → v1.0 算法 → 7 项问题识别 → v2.0 算法（5 项改进）→ 5 项进一步问题 → v2.0.1 算法（共 12 项改进）/ 多次测试通过。
>
> **核心成果**：流失 SKU 真实命中率从 v1.0 全算 480 个 → v2.0.1 三分类（真流失 134 + 季节性 34 + 噪音 367）/ **误杀降低 72%**。

---

## §1. v1.0 算法精准度测试 — 7 项已知问题

| # | 测试项 | v1.0 表现 | 问题 |
|---|---|---|---|
| 1 | 缺失值处理 | cost_price=0 / NaN 不区分 | 影响毛利计算 |
| 2 | 重复 barcode | 不去重 | 可能双计 |
| 3 | 毛利率分布 | -68.9% 到 33.6% / 异常未标注 | 亏本卖未自动标 |
| 4 | 流失 SKU 辨别 | 全算 480 / 不区分真假 | **60% 是低销噪音** ⚠️ |
| 5 | ABCD 阈值敏感 | 50/80/95 固定 / 未小样本 fallback | 体量小子组 A 类被挤出 |
| 6 | 子组 vs 合并分级 | 单视图（合并）| 体量小子组无 A 类 |
| 7 | 60T 日均估算 | 差值法（90-30）| 30T 异常时偏 4-15% |

详见 §1.1 测试输出。

### §1.1 v1.0 测试实测数据（方便食品 199 SKU）

```
【v1.0 测试 1】缺失值/异常值检测：
  cost_price 为 0/NaN: 0 SKU （此组数据干净）
  gross_profit 为负值（亏本）: 3 SKU ⚠️
  
【v1.0 测试 2】重复 barcode: 0 / 唯一 199 / 干净

【v1.0 测试 3】毛利率分布：
  均值 18.9% / 中位 19.1% / 最低 -68.9% / 最高 33.6%
  毛利率 < 0%（亏本）: 3 SKU
  
【v1.0 测试 4】流失 SKU 真假辨别：
  流失总数: 40
  90 天销量 ≥ 5 件（真流失）: 16 (40%)
  90 天销量 < 5 件（噪音）: 24 (60%) ⚠️
  
【v1.0 测试 5】ABCD 阈值敏感性：
  阈值 (0.5,0.8,0.95): A=22 B=56 C=65 D=56
  阈值 (0.6,0.85,0.95): A=35 B=58 C=50 D=56
  阈值 (0.7,0.9,0.97): A=53 B=61 C=45 D=40
  → 阈值不同结果差距大 / 需要小样本 fallback
  
【v1.0 测试 7】60T 日均估算：
  差值法: 357.66 元/天
  90T 加权法: 372.99 元/天
  差异: -4.11%（小子组 / 大组差异更大）
```

## §2. v2.0 算法 — 5 项改进设计

| # | 改进 | 设计 |
|---|---|---|
| 1 | 数据清洗加强 | 全字段 fillna(0) + 显式分类 |
| 2 | 重复 barcode 去重 | drop_duplicates('barcode') |
| 3 | data_quality_flag split | 与 business_anomaly_flag 拆开 |
| 4 | NS_new 新品识别 | 30T 有 / 90T 无 → NS_new 类 |
| 5 | 双视图 ABCD | 子组独立 + 全合并 双坐标 |

### §2.1 v2.0 测试实测数据（方便食品+调味+散称+休闲 / 4 子组 / 糕点 90 天文件路径错）

```
【v2.0 测试 1】data_quality vs business_anomaly split：
  data_quality_flag 非空: 2,134 SKU （v1.0 全部混在 z_flag）
  business_anomaly_flag 非空: 90 SKU
  → split 后真业务异常仅 90 个（v1.0 报 943 个 / 提升 90%）

【v2.0 测试 2】NS_new 新品识别：
  4 子组 NS_new = 0 SKU
  → 5/8 没新品引入 / 与 4/25 调改"做减法"模式一致

【v2.0 测试 3】流失 SKU 真假辨别（threshold=5 件）：
  流失 SKU 总数: 480
  Z_real_lost: 148 (vs v1.0 全算 480 / 精度提升 69%)
  Z_low_sales_noise: 332

【v2.0 测试 4】子组独立 vs 合并分级：
  子组独立 A 类总和: 264
  合并分级 A 类: 132
  → v2.0 给两个视角 / 体量小子组也有 A 类
```

## §3. v2.0.1 算法 — 5 项进一步改进

| # | 改进 | 设计 |
|---|---|---|
| 1 | 糕点 90 天路径修复 | "_90天_3个月_糕点"（中间下划线 / v2.0 漏） |
| 2 | 流失三分类 | 真流失 + 季节性休眠 + 噪音（火锅底料案例） |
| 3 | 清仓模式识别 | BA_low_stock_5（库存 ≤ 5 + 90T 销量足） |
| 4 | 价格弹性维度 | 30T 单价 vs 90T 单价对比 |
| 5 | 小样本 fallback 阈值 | SKU<200 用 60/85/95 阈值（避免过严） |

| 表 | v1.0 数字 | v2.0.1 数字 | 变化 |
