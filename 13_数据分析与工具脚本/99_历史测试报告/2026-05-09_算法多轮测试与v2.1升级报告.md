### 改进 20：滞销 4 级（基于 [[2026-05-09_滞销与库存挤压判定标准_v1.0]] §B.4）

```python
def sluggish_flag_v21(sku):
    """
    返回 sluggish_flag ENUM:
      P0_immediate_remove / P1_30day_observe / 
      P2_seasonal_dormant / P3_strategic_keep / healthy
    """
    days_no_sales = (today - sku.last_sales_date).days
    inventory = sku.current_inventory
    
    # 按品类 L1 阈值（v1.0 §B.1）
    threshold_map = {
        '01_生鲜': {'P0': 5, 'P1': 3.5},        # 生鲜 5 天 / P1 = 0.7×P0
        '02_食品_短保': {'P0': 30, 'P1': 21},   # 冷冻食品 / 休闲食品（中保）
        '02_食品_长保': {'P0': 90, 'P1': 63},   # 调味 / 粮油
        '03_百货': {'P0': 120, 'P1': 84},       # 百货 / 日化（耐用品）
        '04_散装': {'P0': 45, 'P1': 32},
        '05_烟': {'P0': 60, 'P1': 42},
        '06_专柜': {'P0': 7, 'P1': 5},          # 烘焙 / 熟食（短保 / 现烤）
        '酒水': {'P0': 90, 'P1': 63},
    }
    
    cat = sku.category_l1
    th = threshold_map.get(cat, {'P0': 60, 'P1': 42})  # 默认
    
    # 排除项（不判滞销）
    if sku.is_new_product and sku.days_on_shelf < 30:
        return 'healthy'  # 新品观察期
    if sku.is_strategic_kept_by_boss:
        return 'P3_strategic_keep'  # 老板/启明 sign off
    if sku.is_seasonal and not sku.in_season:
        return 'P2_seasonal_dormant'  # 季节外
    
    # 主标准
    if days_no_sales >= th['P0'] and inventory > 0 and inventory_value > value_threshold:
        return 'P0_immediate_remove'
    elif days_no_sales >= th['P1']:
        return 'P1_30day_observe'
    else:
        return 'healthy'
```

### 改进 21：挤压 4 级（基于 [[2026-05-09_滞销与库存挤压判定标准_v1.0]] §C）

```python
def overstock_flag_v21(sku):
    """
    三维联合判定（金额 + 周转 + 库存月数）
    返回 overstock_flag ENUM:
      severe_overstock / moderate_overstock / 
      light_overstock / healthy
    """
    inventory_value = sku.current_inventory * sku.cost_price  # 库存金额
    inventory_turnover_days = sku.current_inventory / (sku.daily_sales + 0.001)
    inventory_months = sku.current_inventory / (sku.monthly_avg_sales + 0.001)
    
    # 按品类 L1 阈值（v1.0 §C.2）
    threshold_map = {
        '01_生鲜': {'value': 1000, 'days': 7, 'months': 0.5},
        '02_食品_休闲': {'value': 3000, 'days': 60, 'months': 3},
        '02_食品_冷冻': {'value': 5000, 'days': 60, 'months': 3},
        '02_食品_调味': {'value': 5000, 'days': 90, 'months': 4},
        '03_百货': {'value': 5000, 'days': 120, 'months': 6},
        '酒水': {'value': 10000, 'days': 90, 'months': 4},
        '05_烟': {'value': 10000, 'days': 30, 'months': 1},
    }
    
    cat = sku.category_l1
    th = threshold_map.get(cat, {'value': 5000, 'days': 60, 'months': 3})
    
    # 排除项
    if sku.is_new_product and sku.days_on_shelf < 30:
        return 'healthy'  # 新品观察期
    
    # 严重挤压：所有维度 × 2
    severe_count = sum([
        inventory_value > th['value'] * 2,
        inventory_turnover_days > th['days'] * 2,
        inventory_months > th['months'] * 2
    ])
    
    # 中度挤压：所有维度 ≥ 阈值
    moderate_count = sum([
        inventory_value > th['value'],
        inventory_turnover_days > th['days'],
        inventory_months > th['months']
    ])
    
    if severe_count >= 2:
        return 'severe_overstock'
    elif moderate_count >= 3:
        return 'moderate_overstock'
    elif moderate_count >= 1:
        return 'light_overstock'
    else:
        return 'healthy'
```

### 改进 22：流失三分类 × 滞销四级整合

> **背景**：v2.0 的"流失 SKU 三分类"（z_real_lost / z_seasonal_dormant / z_low_sales_noise）与 v2.1 的"滞销四级"在概念上有重叠。v2.1 整合规则：

```python
def integrate_z_flag_with_sluggish(sku):
    """
    整合 v2.0 流失三分类 与 v2.1 滞销四级：
    
    流失三分类（v2.0 / 关注"是否流失"）       滞销四级（v2.1 / 关注"如何处置"）
    -------------------------------------    -------------------------------
    z_real_lost          (真流失)           → P0_immediate_remove
    z_seasonal_dormant   (季节性休眠)       → P2_seasonal_dormant
    z_low_sales_noise    (低销噪音)         → P1_30day_observe
                                            P3_strategic_keep（老板 sign off）
                                            healthy（不滞销）
    """
    z = sku.z_flag_v20  # v2.0 三分类
    is_strategic = sku.is_strategic_kept_by_boss
    
    if z == 'z_real_lost':
        return 'P0_immediate_remove' if not is_strategic else 'P3_strategic_keep'
    elif z == 'z_seasonal_dormant':
        return 'P2_seasonal_dormant'
    elif z == 'z_low_sales_noise':
        return 'P1_30day_observe' if not is_strategic else 'P3_strategic_keep'
    else:  # 无 z 标记 = 健康
        return 'healthy'
```

→ **效果**：v2.0 流失三分类 + v2.1 战略保留通道整合后 / 误杀率进一步降低 / 老板战略 SKU 不被误杀。
