# vault 商品库字段字典 v1.1

> **single source of truth**：所有商品库 CSV 字段一律以本文件为准 / 05_/商品库索引仅引用 / 不重复定义。
>
> **v1.1 关键升级**：
> 1. 字段优先级 P0/P1/P2 分级（防字段过载）
> 2. 经营属性三层归属（B 品类级 / A 商品级 / C 门店级）
> 3. keyword_rule 治理字段升级（13+ 列）

---

## §1. 字段优先级定义（P0/P1/P2）

| 优先级 | 含义 | 入库要求 |
|---|---|---|
| **P0** | 必填字段 | 缺失则不得入库 |
| **P1** | 推荐字段 | 有则填 / 没有可空 |
| **P2** | 未来字段 | 占位 / 不纳入 v1.1 验收 |

---

## §2. 经营属性三层归属（v1.1 核心）⭐⭐⭐

```
品类级经营属性（最稳定）→ B_category_skeleton.csv
  例：生抽 = 民生刚需 / 价格敏感 / 价格带划分

商品级相对稳定属性 → A_core_sku_seed.csv
  例：家庭装 / 礼盒装 / 高端款 / 包装类型

门店级经营判断属性（会变）→ C_client_specific/*_mapped.csv
  例：是否端架 / 是否流量品 / 是否毛利品 / business_action
```

**防"商品天然属性"和"门店经营策略"混层** / 后期同一商品在不同门店判断会冲突。

## §10. P0/P1/P2 字段汇总（v1.1 验收用）

```
P0 必填字段（全表必填）：
  通用：sku_id / skeleton_id / rule_id / category_l4_code / source_type / status / created_date / updated_date
  B：category_l1-l4 / l4_code（FK V4.0）
  A：standard_product_name / brand_name
  C raw：raw_sku_id / raw_product_name / source_file / import_date
  C mapped：raw_sku_id（FK raw）/ category_l4_code（FK V4.0）

P1 推荐字段（有则填）：
  品牌别名 / 规格 / 包装 / 经营属性（按三层归属）/ 价格带 / 备注

P2 未来字段（占位）：
  embedding_id / vector_version / external_source_id / cross_client_score
```

## §11. 单源原则

⛔ **不要把同一字段在多个文件中重复定义为不同含义**：
- `category_l4_code` 在 B/A/C/keyword_rule 中含义一致 / 都是 FK→V4.0
- `status` 在所有表中含义一致 / 都是 active/dormant/archived
- `source_type` 在 B/A 含义略异（B 仅 v4_skeleton/manual / A 含 off/manual/client_feedback/gs1）/ 已在各表说明
