# /posclean · POS 数据清洗

## 执行(机械相)

```bash
L=13_数据分析与工具脚本/POS清洗库_v0.1/pos_clean.py
python3 $L probe   --src <原始xls目录>                       # 先体检:格式/截断/期间
python3 $L sales   --src <目录> --out <工作区> --prefix <前缀>  # 销售明细→master
python3 $L archive --src <目录> --out <工作区>                # 档案→sku_registry
```

## 纪律

1. **永远先 probe 后 clean**——假xls/截断在体检期暴露,不带病进管道
2. 任何 🔴/⚠️(截断/对账偏差>0.5%/假xls)→ 停,报六哥,不出下游结论
3. 原始文件只读;输出只落客户工作区(csv 不入 git)
4. 下游衔接:master → /trio 三件套;字段级硬化 → 数据清洗匹配_v0.1(不重叠)

## 判读边界(判断相)

清洗是机械相;但"缺口归因"(如烟草缺表=系统分离还是漏导)、"截断文件弃用还是补导"归六哥/主线程。
