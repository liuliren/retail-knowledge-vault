# /mdcard · 图卡生成体系 v1.0（正式稿 · 六哥 2026-07-03 收口）

> **目的**：把"数据→卡片"的方法固化为本地代码——**引擎(matplotlib)零 token 秒级出图是主干**；Image-2 只是门面件的可选精修。设计细节保持开放演进；方法、规则、流程以本稿为准，需更正时走版本更新。

## 一、五步操作流程

```
① 定受众层(L0-L4) → 打 chip
② 过三级生成规则(自动/询问/不出)
③ 单主题=单卡;多主题诊断=卡组(1主+≤3细分)
④ 从 md 提炼 spec JSON → 引擎渲染(真值版):
   python3 13_数据分析与工具脚本/图卡引擎_v0.1/mdcard.py --spec <spec.json> --out <同名.png>
⑤ 门面件且时间允许 → Image-2 精修(见§六),否则引擎版即定稿
```
spec 结构见引擎 docstring；PNG 与 md 同目录同名；spec 过程件放 scratchpad 不落库。

## 二、五层受众矩阵（每卡必打 chip）

| 层 | chip | 卡型 | 语言纪律 |
|---|---|---|---|
| L0 | 内部·六哥 | 作战卡(决策点/KPI/跟进) | 专业·判断导向·So-What 前置 |
| L1 | 店方·老板 | 诊断/成果卡 | **内联括注式**：瓦片 label=`指标名(白话注)`，如"动销率(在卖占比)"；note 小字行专职放上下文数字；常规术语可用，深术语(ABCZ/SKU)不用；每个数字翻成钱或动作 |
| L2 | 店方·店长/科组主管 | 执行卡 | 任务话：做什么/谁做/多久/怎么算完 |
| L3 | 店方·采购/店员/文员 | 操作卡 | 一卡一件事，零判断 |
| L4 | 公域 | 营销卡(hero 竖幅) | 故事+数字钩子；**脱敏铁律：客户真名须过授权链(P1-GOV-ClientNameAuth-001)，清单未填前默认"社区生鲜样板店"** |

## 三、三级生成规则（判据=传播风险，非重要性）

| 级 | 触发 | 动作 |
|---|---|---|
| 自动生成 | 诊断性结论(硬数字)/内部件/已签批行动件 | 不询问，随 md 同步出卡 |
| 询问后生成 | ①对外发送件 ②draft/未签草案 ③含供应商/价格明细 | 先问六哥；草案出卡必打 DRAFT 角标 |
| 不生成 | 纯过程件(审计/台账/代码)、>10 行长清单 | md 即交付；清单类配 xlsx 附件 |

> 原理：图片传播性>md。草案图片化=给草案传播载体，最高危；生成≠发送，内部结论卡零风险。

## 四、卡组模式 + 双版本制

- **双版本制**：内部=md 全量报告(要 PDF/Word 走 make-pdf 管道)；店方=卡组
- **卡组结构**：1 张总览主卡(全貌+分带+"后面 N 张卡"指引) + ≤3 张细分卡(**一卡只推一件事、带一个动作**)；编号入 kicker(细分 1/3)
- 分带用白话：强项/最需瘦身/跟着瘦身/单独定去留/换季甄别(判例见§八)

## 五、语言与数据规范

**数据三纪律（数据是事实，凌驾所有卡型）**：
1. 无数字不判断——定性词必挂数字("很旺"→"628万/占7成")
2. 指标带翻译不删指标——白话化是加注解，不是藏指标
3. 口径上卡——期间/样本量/数据源必须在来源行，数字可复核才叫事实

**方法论隐身**：店方卡禁出现框架维度名(What/Price 等)/口径治理词(draft/ABCZ/KB)——后厨的事不上菜单，直接给结果。

## 六、双轨渲染：引擎真值 + Image-2 美化档

**引擎版=数字权威(SSOT)**：本地零 token、秒级、数据变即刷、完全确定——**默认与兜底**。
**Image-2 版=门面件可选精修**（老板卡/营销卡，时间与额度允许时）：
1. 引擎先出真值版 → 2. `~/.claude/skills/huashu-gpt-image/scripts/gen_via_codex.py --ref 引擎版.png --quality high`（中文短句 prompt："参照参考图全部文字与数字原样保留，Apple Keynote 风格…中文无错字"）→ 3. **强制逐数校验**——AI 读图 vs spec 逐个对，一个错数即废弃该图退回引擎版。
数据变了必须从步骤 1 重走，禁止只改图。内部卡/高频重渲染卡不走 Image-2。

## 七、设计语言（当前基线 · 保持开放演进）

引擎 v0.5 基线：纯白底/近黑墨 #1d1d1f/品牌绿 #0a7d54/行动红 #bf1d2c；数值 30 号大字主角、标题即结论(折行防孤字)、留白分隔仅表头一条发丝线、hero 竖幅 1080×1350、来源行必带；数值默认墨色，红=要行动的数，绿=正向亮点，一卡一个视觉重心。
> 设计层优化不设终稿——有更好的字阶/版式随时升引擎版本，规则不动。
