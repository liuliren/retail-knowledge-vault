# /mdcard · md 配套图卡

## 三级规则(六哥 2026-07-03 升级·核心判据=传播风险而非重要性)

| 级 | 触发 | 动作 |
|---|---|---|
| **自动生成** | 诊断性结论(硬数字)/内部件/已签批行动件 | 不询问,随md同步出卡,打「内部」角标 |
| **询问后生成** | ①对外发送件 ②draft/未签草案 ③含供应商/价格明细 | 先问六哥;草案出卡必打「DRAFT」角标 |
| **不生成** | 纯过程件(审计/台账/代码)、>10行长清单 | md即交付 |

> 原理:图片传播性>md(md在库里,图片进微信)。草案图片化=给了草案传播载体,是最高危场景。生成≠发送,内部结论卡零风险故免询问。

## 五层受众矩阵(六哥 2026-07-03 定调 · 每卡必打chip)

| 层 | chip | 卡型 | 语言纪律 |
|---|---|---|---|
| L0 | 内部·六哥 | 作战卡(决策点/KPI/跟进) | 专业·判断导向·So-What前置 |
| L1 | 店方·老板 | 诊断/成果卡 | **内联括注式**(六哥07-03两版A/B实测定标):瓦片label=`指标名(白话注)`括注跟术语走,如"动销率(真的在卖的占比)";note小字行专职放上下文数字;常规术语可用,深术语(ABCZ/SKU)不用;每个数字翻成钱或动作 |
| L2 | 店方·店长/科组主管 | 执行卡 | 任务话:做什么/谁做/多久/怎么算完 |
| L3 | 店方·采购/店员 | 操作卡 | 一卡一件事,零判断 |
| L4 | 公域 | 营销卡(hero布局) | 故事+数字钩子;**脱敏铁律:客户真名须过授权链(P1-GOV-ClientNameAuth-001),清单未填前默认"社区生鲜样板店"** |

## 数据三纪律(六哥 2026-07-03 补充 · 数据是事实,凌驾所有卡型)

1. **无数字不判断**:卡上每个定性词必须挂数字("很旺"→"628万/占7成"),形容词不许裸奔
2. **指标带翻译,不删指标**:白话化是给指标加注解,不是把指标藏掉——L1写法:`指标名 数值(白话注)`;L0内部卡指标全量出
3. **口径上卡**:期间/样本量/数据源三要素必须在卡面(来源行),数字可复核才叫事实

## 设计语言 v0.4(Apple系极简·信息传递漂亮>装饰漂亮)

纯白底 #ffffff / 近黑墨 #1d1d1f / 单品牌绿 #0a7d54 / 行动红 #bf1d2c;留白分隔(仅表头一条发丝线);标题即结论(折行防孤字);hero布局=竖幅1080×1350大数字营销卡;来源行必带。

## 执行(机械相)

```bash
# 1. 从 md 提炼 spec(标题/关键指标瓦片≤8/核心表格≤10行/口径脚注)写 JSON
# 2. 渲染:
python3 13_数据分析与工具脚本/图卡引擎_v0.1/mdcard.py --spec <spec.json> --out <同名.png>
```

spec 结构见引擎 docstring。PNG 与 md 同目录同名;spec(过程件)放 scratchpad 不落库。

## 图卡纪律

1. **一卡一结论**:瓦片放"老板 10 秒要知道的数",表格≤10行,超出的进 md 不进卡
2. client_shareable 卡:零术语零条码,红色仅用于需要行动的数字(warn)
3. 数字必须与 md 正文一致(同源生成,禁手抄)
4. 生图类配图(公众号封面等)不归本 skill,走 huashu-gpt-image / /publish
