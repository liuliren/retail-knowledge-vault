# /trio · 三件套整体分析

## 执行(机械相·脚本)

```bash
E=13_数据分析与工具脚本/三件套引擎_v0.1/trio_engine.py
# 首次或数据更新时先清洗:
python3 $E clean --src <分品类xls目录> --out <工作区> --prefix <文件名前缀>
# 三件套(单次加载全跑):
python3 $E all --master <工作区>/sales_master.csv --archive <档案xls目录> --out <工作区>
```

花厅坊默认路径:src=`09_门店案例与项目复盘/乐易购花厅坊店/99_原始素材/01_门店数据材料/2026-07_数据治理诊断批次/`,out=`09_.../03_商品诊断/01_清洗输出/`。

## 流程纪律

1. **clean 对账偏差>0.5% → 停,先查数,不出结论**(反幻觉:结论必须锚定系统合计)
2. 原始 xls 只读,输出只落客户工作区(csv 不入 git,gitignore 已盖)
3. 引擎口径=已签铁律,改口径须六哥签字改代码

## 判读(判断相·不得下放给脚本)

脚本只出数;以下判断留给主线程+六哥:动销率结构判读、A类宽窄与单品力、SKU效率洼地定位、汰换候选取舍(必须过 IssueTag+角色闸防误杀)、生鲜毛利倒算安排。参照首验判读样例:[[2026-07-02_花厅坊三件套首算_v0.1]]。
