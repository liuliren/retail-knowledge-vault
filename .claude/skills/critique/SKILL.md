# /critique — 批判审议

> **一句话**：给每件待审文件派一个不认识作者的"魔鬼代言人"，带着四问和反例任务去拆它；拆不动的才配升级。

## 何时用 / 何时不用

- **用**：draft 标准/框架/SOP 进第 3 轮前的第 2 轮审议；六哥对任何方案说"批判一下"；外部 Prompt/方案采纳前的对抗检验。
- **不用**：纯事实性文件（台账/日志/数据表）；已 stable 件的日常小修（走正常修订）；<50 行的小 note（用 lite 版：**独立单 Agent** 四问，不开多 Agent——仍须独立上下文，不得用携带起草上下文的主线程；对齐审议细则 §1 lite 口径）。

## 执行五步（细则 §3 的操作化）

### Step 1 · 锁定与设角
- 确认对象文件清单（1-N 件）；每件派**一个独立 general-purpose Agent**，多件并行发出。
- Agent prompt 必含：角色=批判/魔鬼代言人；**不携带起草上下文**；原文件绝对路径。

### Step 2 · 四问模板（每个 Agent 必答）
1. 这套规则/公式/流程在**什么具体场景下产生错误结果**？（给反例，不给"可能"）
2. 与库内**现有规则的逻辑张力**？（指名文件，grep 验证引用真实性）
3. **遵守成本**是什么？有没有更轻量的替代方案？
4. **五年后还成立吗**？（哪些是耐久内核、哪些是速朽参数——参考 90/10：马具会贬值）

### Step 3 · 反例与实算（硬纪律）
- ≥3 个边界失效案例；
- 凡公式/权重/阈值：**代入数字实算**（文字自洽≠数字自洽）；
- 凡引用：grep 回源验证出处真实、原义未被扭曲；
- 凡"自称已修"（双 Pass/自检）：逐条复核是否真修干净。

### Step 4 · 判定分级
`放行 / 有条件放行（必修项 R1…）/ 打回（阻断项 R1…）/ 降格重定位` —— 判定针对文本不针对方向；每条判定带行级证据。

### Step 5 · 汇总裁决单
全部 Agent 回传后，主线程出一份汇总：判定总表 + **跨文件发现**（同词异义/构念重叠/互引死锁等，往往比单件修订更值钱）+ 依赖拓扑修订排期（先修被引者）+ 六哥裁决点编号。

## Agent Prompt 模板（复制改路径即用）

```
你是《标准审议铁律》第2轮的批判角色（逆向/魔鬼代言人）。对象：<绝对路径>
任务：通读全文，写第2轮审议记录。必答四问：①什么场景产生错误结果（给具体反例）②与库内现有规则的逻辑张力（指名文件并grep验证）③遵守成本与轻量替代 ④五年后还成立吗。
另外：找≥3个边界失效案例；公式/阈值代入实算；引用回源验证；"自称已修"逐条复核。
输出：写入 <输出区审议目录>/<日期>_第2轮批判审议_<文件ID>.md
frontmatter：status: draft、审议轮次: 2、角色: 批判-魔鬼代言人、summary。
正文：结论先行（放行/有条件放行/打回/降格）→ 四问逐答 → 反例清单 → 编号修订建议。
纪律：绝不修改原文件；不读 xlsx/csv/99_原始素材/_client_private；每条批判给行级证据。
```

## 出口纪律（红线）

- **原文件零改动**——审议只产记录，修订是裁决后的另一动作；
- **裁决权在六哥**——skill 产出判定建议，放行/打回由六哥签；判断相不下放（模型分层调度红线）；
- **自检不算轮次**——本 skill 的运行本身才构成一次正式轮次（细则 §1）；
- 审议记录落输出区审议目录，汇总单登缺口总账对应行。

## 复盘 D + 迭代 E（Skill 闭环铁律）

每次运行后追问两句并回写本文件：①哪个 Agent 的批判空转了（四问答得泛）→ 收紧该问的模板措辞；②有没有跨文件发现漏网 → 补 Step 5 检查角度。lessons 记于下方：

### Lessons
