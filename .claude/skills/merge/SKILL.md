# /merge · 双文档冲突裁定 Skill v0.1

> **一句话**：两份相近文档放进来，出来的是"留哪个/怎么合/生成新正本"的裁定结论+可用草稿。

---

## 触发条件

遇到以下任意情况时触发本 skill：
1. 同名或高度相似的文档在不同目录各存一份
2. 精读评估发现两份文档共享核心概念但版本不明
3. 归档内容与当前活跃库存在潜在覆盖重叠

---

## 输入

| 参数 | 说明 |
|---|---|
| `file_A` | 第一份文档的绝对路径 |
| `file_B` | 第二份文档的绝对路径 |
| `context`（可选） | 补充说明：这两份文档应该回补到哪个 wiki 页，以及比较的目的 |

---

## 执行步骤

### Step 1 · 全文读取

- 读取 file_A 完整内容
- 读取 file_B 完整内容
- 记录：字数、结构（有无 frontmatter、章节数）、修改时间（如可读）

### Step 2 · 相似度判断（快速先验）

**判断维度**：
- 标题是否相同
- 前 200 字是否相同
- 结构（章节标题列表）是否相同
- 核心论点是否一致

**分类**：
| 类型 | 特征 | 触发动作 |
|---|---|---|
| **IDENTICAL** | 内容 >90% 相同 | 选一保留，另一进 D 批次 |
| **OVERLAP** | 核心相同，各有独特段落 | 提取独特部分合并为单一正本 |
| **DIVERGENT** | 结构或观点有实质差异 | 以更完整版本为底，吸收另一份的独特洞见，生成新正本 |

### Step 3 · 深度分析

**如果 IDENTICAL**：
- 确认哪份文件路径更"规范"（命名/位置）
- 输出：`verdict: IDENTICAL · 保留 A / 删除 B`（或反之）

**如果 OVERLAP**：
- 列出 A 独有段落（B 中无对应）
- 列出 B 独有段落（A 中无对应）
- 列出共同内容（取版本更完整的）
- 合并顺序建议（A 的结构为主 or B 的结构为主）

**如果 DIVERGENT**：
- 分析哪份是"更新版本"或"更完整版本"
- 列出两份各自的核心贡献
- 起草新正本（以较完整版为骨架，吸收另一份的独特洞见）

### Step 4 · 输出文件

写入 `98_AI协作中枢/01_Claude_Code/Claude输出区/YYYY-MM-DD_merge/<slug>_裁定报告.md`

## 后续接口

- 裁定完成后，若 verdict=MERGE → 合并草稿可直接作为 `/ingest` 的 source 输入
- 若 verdict=IDENTICAL → 被删文件路径加入当前轮次 D 批次等待六哥确认删除
- 每次 /merge 结果写入 `98_AI协作中枢/01_Claude_Code/Claude输出区/YYYY-MM-DD_merge/`

## 典型案例参考

| 场景 | 预期结论 |
|---|---|
| 同一文件被拷贝到两个归档目录，内容完全相同 | IDENTICAL → 删除冗余副本 |
| v1 和 v2 同主题文档，v2 是 v1 的扩展 | OVERLAP → v2 为主干，吸收 v1 的独特段落 |
| 同主题但角度不同（如操作层 vs 原理层） | DIVERGENT → 合并为双层结构正本 |
