# 运营力诊断检核清单 v0.1

> 🚧 **校准待花厅坊8月实测**——当前所有🟢🟡🔴阈值为**初稿(inferred)**,尚未经门店数据回归验证。**禁进客户交付件**,仅供内部现场判断与自我训练。

## 0. 结论先行

本清单**派生自 [[运营力总纲]] §3(运营力核心构成)**,把总纲定义的 **7 个构成模块**逐一转成"怎么现场判一家社区生鲜超市运营力强弱"的检核项。

- **它不是什么**:不重新定义运营力,不新造第二套运营力模型。运营力的**定义、边界、与商品力/组织力的分工**一律以 [[运营力总纲]] 为准(SSOT)。
- **它是什么**:总纲的**下游诊断工具**。总纲回答"运营力是什么",本清单回答"怎么看一家店有没有运营力"。
- **适用**:500–2000㎡ 大湾区社区生鲜超市,现场巡店 30–60 分钟 + 调阅基础报表即可完成一轮。
- **判据锚定**:🟢🟡🔴 三档的强/弱信号取自总纲 §4(社区超市典型表现)与 §8(常见误区),再落到社区生鲜场景具体化。

**用法**:逐模块勾判,7 模块各出一个红绿灯 → 汇成运营力画像。红项即优先改善靶点,配 [[经营复盘]] 回路闭环。

## 1. 检核主体(对应总纲 §3 七模块)

### 模块1 · 陈列优化
> 总纲§3:按 DisplayMap 落地货架陈列,持续维护陈列标准,根据动销数据调整排面。

| # | 检核问题 | 看什么数据/现象 | 判据 | 数据来源 |
|---|---|---|---|---|
| 1.1 | 有没有可执行的陈列标准并被维护? | 是否存在 DisplayMap / 货架陈列图;抽 3–5 组主货架看实物与图的一致度 | 🟢 有陈列图且落地,抽查基本一致 / 🟡 有图但执行走样、多处凭感觉 / 🔴 无任何陈列标准,全靠店员随手摆 | 现场巡店 + DisplayMap 文件 |
| 1.2 | 黄金层(1.4–1.6m)放的是不是 A 类动销品? | 主通道 3 个货架黄金层的商品动销等级 | 🟢 黄金层基本为 A 类品 / 🟡 出现 1–2 个明显 C 类占位 / 🔴 黄金层普遍错配、慢销品占黄金位 | 现场巡店 + 单品 ABC 动销表 |
| 1.3 | 生鲜区(叶菜/猪肉)陈列是否整洁、随鲜度维护? | 叶菜蔫菜数、猪肉断档/空盘、散装区凌乱度 | 🟢 分区清晰、无明显蔫菜、随卖随补 / 🟡 局部蔫菜或空盘、补货滞后 / 🔴 蔫菜多、空盘长时间不补、脏乱 | 现场巡店(生鲜区) |
| 1.4 | 排面是否随动销调整,还是常年不动? | 近期是否有按动销扩/缩排面的动作;空排面(断货位)数量 | 🟢 有据动销调排面的痕迹,空排面≤个位数 / 🟡 少量调整,空排面成片出现 / 🔴 排面常年不动,大量空排面无人管 | 现场巡店 + 动销数据 |

### 模块2 · 营销活动
> 总纲§3:按 52周MD 节奏策划和执行堆头、端架、促销活动,并复盘效果。

| # | 检核问题 | 看什么数据/现象 | 判据 | 数据来源 |
|---|---|---|---|---|
| 2.1 | 活动有没有节奏,是否挂 52周MD? | 是否有活动日历 / 52周MD;近 4–8 周活动是否提前规划 | 🟢 有 [[52周MD]]/活动日历,活动提前排 / 🟡 有活动但零散、临时起意 / 🔴 无节奏、想起才做 | [[52周MD]] / 活动日历 + 访谈 |
| 2.2 | 堆头/端架是否紧扣当季节令主题? | 堆头端架主题与当季大湾区节点(端午/中秋/开学/春节等)的匹配度;是否常年同一组货 | 🟢 堆头端架切当季主题、定期轮换 / 🟡 部分应景但多数长期不换 / 🔴 堆头端架常年一套、无主题 | 现场巡店 + [[KB-SEASON-MATRIX-001_大湾区节令品类影响矩阵_v0.1]] |
| 2.3 | 活动结束后有没有复盘? | 是否有活动效果/ROI 记录;能否区分自然动销与促销拉动 | 🟢 每场活动有复盘、能算拉动 / 🟡 偶尔复盘、只看总销 / 🔴 从不复盘,反复踩同一坑(对应§8误区2) | 活动复盘记录 + 经营看板 |

### 模块3 · 门店日常运营
> 总纲§3:开关店流程、补货、价签、卫生、损耗管理等日常执行闭环。

| # | 检核问题 | 看什么数据/现象 | 判据 | 数据来源 |
|---|---|---|---|---|
| 3.1 | 价签是否齐全、准确? | 抽查主货架价签缺失数、标价与系统/收银价是否一致 | 🟢 抽查基本无缺、无错价 / 🟡 零星缺签或个别错价 / 🔴 大面积缺签或多处标价≠收银价 | 现场抽查 + 收银/系统价 |
| 3.2 | 补货是否及时闭环? | 巡店时段的空排面数、后仓积压未上架情况 | 🟢 补货及时、空排面极少 / 🟡 高峰后补货滞后、空排面成片 / 🔴 长时间断货不补、后仓货压着不上 | 现场巡店 + 补货台账 |
| 3.3 | 卖场卫生与生鲜损耗是否受控? | 地面/冷柜/生鲜操作区卫生;生鲜损耗率量级(店内自比,不设跨店绝对值) | 🟢 整洁、损耗在门店合理区间且有记录 / 🟡 卫生一般、损耗无稳定记录 / 🔴 脏乱、生鲜损耗明显失控且无人管 | 现场巡店 + 损耗报表 |

### 模块4 · 异常管理
> 总纲§3:缺货、滞销、价格异常、设备故障、顾客投诉的快速识别和处理。
> ⚠️ 本模块与模块5在"缺货/滞销"上有交叠(见§3待完善);诊断时**模块4看"响应机制有没有、快不快",模块5看"库存结果指标"**,避免重复计分。

| # | 检核问题 | 看什么数据/现象 | 判据 | 数据来源 |
|---|---|---|---|---|
| 4.1 | 异常有没有响应机制(谁发现、多久处理)? | 是否有异常上报路径;缺货/价格异常从发现到处理的时效 | 🟢 有明确上报响应、缺货当天补 / 🟡 靠人临时发现、时效不稳 / 🔴 无机制、问题严重才发现(对应§4弱店) | 访谈 + 异常/补货台账 |
| 4.2 | 顾客投诉是否有闭环? | 投诉是否有记录、有回应、有防再犯动作 | 🟢 投诉有记录+回应+改进 / 🟡 有回应无沉淀 / 🔴 投诉无跟踪、口头了事 | 投诉记录 + 访谈 |
| 4.3 | 冷链/关键设备异常是否被及时发现? | 冷柜/冻库温度是否巡检、有无带病运行 | 🟢 有设备巡检记录、异常即处理 / 🟡 偶查、无固定巡检 / 🔴 设备带病运行、无人监测(生鲜店高风险) | 设备巡检记录 + 现场看温度 |

### 模块5 · 库存与缺货处理
> 总纲§3:库存水位监控、缺货上报与补货追踪、滞销商品及时处置。

| # | 检核问题 | 看什么数据/现象 | 判据 | 数据来源 |
|---|---|---|---|---|
| 5.1 | A 类品与主力生鲜的缺货情况如何? | 高动销 SKU / 主力生鲜的缺货频次(店内自比) | 🟢 A类与主力生鲜少缺货 / 🟡 缺货偶发、集中在个别品类 / 🔴 A类高频缺货、生鲜断档常态 | 缺货记录 + ABC 动销表 |
| 5.2 | 库存水位与周转是否健康? | 库存周转/超期库存量、有无积压 | 🟢 周转健康、超期库存少 / 🟡 局部积压、超期识别慢 / 🔴 大量积压/超期库存沉淀 | 库存报表 |
| 5.3 | 滞销商品有没有被及时处置? | 滞销 SKU 是否被识别并有清退/降价/换位动作 | 🟢 滞销定期识别+有处置动作 / 🟡 能识别但处置拖延 / 🔴 滞销无人处置、常年占排面(对应§4弱店) | 滞销/动销(ABCD)表 + 现场 |

### 模块6 · 周/月复盘
> 总纲§3:定期对经营数据做结构性分析,识别问题根因,输出下一周期调整动作。

| # | 检核问题 | 看什么数据/现象 | 判据 | 数据来源 |
|---|---|---|---|---|
| 6.1 | 有没有固定的周/月复盘机制? | 是否有周/月复盘记录、会议纪要或复盘表 | 🟢 有固定节奏的复盘并留痕 / 🟡 不定期、偶尔做 / 🔴 无复盘机制 | 复盘记录 + [[经营复盘]] |
| 6.2 | 复盘是不是多维,还是只看销售额? | 复盘是否覆盖 销售额+毛利+库存+缺货+执行质量 | 🟢 多维综合看 / 🟡 主要看销售额、少量毛利 / 🔴 只看销售额(对应§8误区3) | 复盘记录 + 经营看板 |
| 6.3 | 复盘是否找到问题根因,而非只对数字? | 复盘结论是否落到"为什么"而非只罗列涨跌 | 🟢 能定位根因 / 🟡 停在现象描述 / 🔴 只念数字不归因 | 复盘记录 + 访谈 |

### 模块7 · 经营改善闭环
> 总纲§3:把复盘结论转化为可执行任务,追踪执行,验证改善效果。

| # | 检核问题 | 看什么数据/现象 | 判据 | 数据来源 |
|---|---|---|---|---|
| 7.1 | 复盘有没有产出可执行的动作清单? | 复盘后是否形成"谁做/何时/验收标准"的任务 | 🟢 每次复盘出动作清单、责任到人 / 🟡 有动作但无责任/时限 / 🔴 只开会不产出动作(对应§8误区5) | 任务追踪表 + 复盘记录 |
| 7.2 | 上一期的改善动作落地了吗? | 上期动作的执行率、完成情况 | 🟢 上期动作多数落地 / 🟡 落地率一般、拖延多 / 🔴 布置了没人做、会后照旧 | 任务追踪表 |
| 7.3 | 改善效果有没有被验证? | 是否用调改前后数据对照,确认动作有效 | 🟢 有前后对照、验证闭环 / 🟡 做了没回看 / 🔴 从不验证、无从积累经验 | 前后对照数据 + [[经营复盘]] |

## 3. 与上游关系(SSOT 声明)

- 本清单是 **[[运营力总纲]] 的下游诊断工具**,不是独立方法论。**运营力的定义、7 模块的划分与命名、与商品力/组织力的边界**,全部以 [[运营力总纲]] 为唯一权威(SSOT);本清单若与总纲不一致,以总纲为准。
- 7 个检核模块**逐一对应总纲 §3 原表的 7 行**,未新增、未合并、未改名。
- 本清单同时是总纲 §9 待完善项「建立社区超市运营力诊断表(对照花厅坊验证可操作性)」的**首个落地物**,校准回流后应反哺总纲。
- 节奏/复盘口径分别引用 [[52周MD]](模块2)与 [[经营复盘]](模块6/7),不重复定义。

## 4. 待完善(批判性陪练 · 发现的上游缺口)

> 以下为编译本清单时发现的 **[[运营力总纲]] §3 本身**的问题,按治理护栏"不自行补一套模型",在此列缺口交六哥裁决,不擅改总纲。

1. **模块4(异常管理)与模块5(库存与缺货处理)在"缺货/滞销"上语义交叠**:总纲§3两行都含"缺货""滞销",易造成诊断重复计分。本清单临时以"4看响应机制、5看结果指标"切分止血,但**根因在总纲 §3 模块划分未做 MECE**,建议总纲明确二者边界(或合并)。
2. **总纲 §3 全部为"做什么"的定性描述,无任何量化阈值**:本清单所有🟢🟡🔴阈值均为 inferred 初稿,需花厅坊8月实测回归校准(尤其模块3损耗率、模块5缺货率、模块1空排面数的具体数值锚点)。
3. **"经营改善闭环"(模块7)与组织力的执行追踪职责边界待厘清**:总纲§8误区4提醒运营力≠组织力,但"任务追踪、责任到人"在实操中易与组织力交叠,诊断时需注意归属,建议总纲补一句边界说明。
4. **缺现场取数口径**:多项"数据来源"依赖门店台账/报表,但社区超市普遍报表缺失,现场靠访谈+目视。总纲未定义"无系统门店"的降级取数法,校准时应补。
