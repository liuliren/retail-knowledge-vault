# 现场巡店诊断工具包 v1.0（通用·档二到店全诊）

> **定位**：[[KB-DX-PRODUCT-001_零售企业首次诊断产品_装配框架_v1.0]] 档二"到店全诊"的现场工具。**任何店通用**（泛化自花厅坊 2026-07-03 首个实跑）。
> **配套工具**：照片预处理 `13_数据分析与工具脚本/现场诊断工具_v0.1/现场照片预处理.py`（HEIC/RAW/TIFF等→JPG·压诊断分辨率）。
> **能力分层（护城河）**：本工具包固化"拍什么/对应哪维/打分锚点"（**耐久结构层**）；"读图打分"是 **AI 视觉判断（进化能力层，不固化）**。别把读图写成代码。
> **用法**：到店前打印本表 or 存手机；拍照按 §一协议；现场按 §二顺序采集；回来按 §三打分。渲染成手机卡走 `/mdcard`。

---

## 一、拍摄规范（拍什么·标准机位·每维张数）

> 命名建议：`D{维号}_{区域}_{序号}.heic`（如 `D2_猪肉柜_01`），回来预处理脚本批量转 JPG。

| 维 | 拍什么（对象） | 标准机位 | 张数 |
|---|---|---|---|
| **D1 商品力** | 各大类货架全景 / 价格带代表段 / 端架堆头 / 尾部滞销/空销区 | 平视正对货架 | 3-4 |
| **D2 生鲜** | 肉类分档柜 / 禽类 / 叶菜岛台(拍出品种数) / 水果 / 冷柜温度牌 / 损耗临期 / **下午3-5点高峰烟火气** | 俯45°拍岛台+平视拍柜 | 5-6 |
| **D3 陈列** | 主通道动线全景 / 黄金层(1.4-1.6m视线层) / 端架堆头推什么 / 价签规范特写 / 空缺货位 / 明显错配对比 | 站通道口拍全景+特写价签 | 4-5 |
| **D4 客流** | 收银台排队(**标时段**) / 入口客流 | 站收银侧拍队列 | 2 |
| **D5 财务** | 进货单据 / 促销牌 / 价签 | 特写正对·可读数字 | 按需 |
| **D6 人员** | 各岗位在岗(生鲜/收银/理货) / 班次表·公告栏 / 后场 | 平视·不侵扰 | 3 |
| **D7 体验** | 收银/称重流程 / 会员社群物料 / 店内卫生 / 顾客客群 | 平视·不拍顾客正脸 | 3-4 |

**拍摄纪律**：① 价签/单据必须拍清数字（后续要读）；② D2 等生鲜高峰时段（下午3-5点）；③ 不拍顾客正脸（隐私）；④ 全景+特写配对（一个区域至少一全景一特写）。

## 二、现场采集顺序（5步·约1.5h·泛化自花厅坊）

| 步 | 时长 | 动作 | 覆盖维 |
|---|---|---|---|
| ① 进门快筛 | 5min | 5分钟诊断卡+4D映射·红绿灯初判 | 全维初扫 |
| ② 生鲜现场 | 15min | 烟火气/品种/日清/损耗·拍D2组 | D2 |
| ③ 走店陈列 | 20min | 错配/黄金层/动线/价签·拍D1D3组 | D1·D3 |
| ④ 要数据 | 5min | 进货单据/账期/保本点(拍D5) | D5 |
| ⑤ 访谈 | 30min | 人员岗位/单点依赖(填运营力检核表)·体验收银熟客·老板访谈 | D6·D7 |
| ⑥ 回来 | — | 照片预处理→读图→§三打分→出整张7D卡 | 收口 |

> 现成表现场填：[[运营力诊断检核清单_v0.1]] / [[组织力诊断访谈表_v0.1]] / [[T-INTERVIEW-BOSS-001_老板首访访谈提纲_v0.1]]。

## 三、7D 打分卡（现场/回来填·锚点见框架）

> 满分锚点/证据分级口径以 [[KB-DX-FULLSTORE-001_全店7D诊断框架2.0_v0.1]] 各维⑤为准。得分率用销售/动销口径（生鲜进价不准时禁用毛利口径）。

| 维 | 权重(通用) | 满分锚点(速记) | 证据级 | 得分率 | 加权 |
|---|---|---|---|---|---|
| D1 商品力 | 20 | A类占好位·五态无空缺·三档价格带·空销<30% | POS+现场B | | |
| D2 生鲜 | 25 | f≥40·肉日清·叶菜≥10种·损耗达标·烟火气 | 现场B | | |
| D3 陈列 | 10 | 无大错配·黄金层放A类·30秒找到率高 | 现场B | | |
| D4 客流 | 15 | 客单稳增·无危险信号·高峰人手够 | POS/访谈C | | |
| D5 财务 | 15 | 毛利20-25%·周转<账期·GMROI≥2.5 | 需进货单·否则C | | |
| D6 人员 | 10 | 人工8-12%·关键岗有合格人·无单点依赖 | 现场/访谈C | | |
| D7 体验 | 5 | 收银快·有复购经营·老板叫得出熟客 | 现场C/D | | |

**权重注**：高 f 生鲜店（如 f≈70%）必走框架 §8 POS 校准（D2 校准回升）；纯夫妻店 D6 权重并入 D2/D5。
**诚实纪律**：C/D 级证据禁写精确结论（"老板反映…"不写"下降15%"）；进价不准的 D5 标"P0候选·待单据"，不进 TOP3 确定结论。

## 四、输出（回来后）
- 照片：`现场照片预处理.py` 批量转 JPG → AI 读图打 D1/D2/D3 + 客群(D7)

- 出整张 7D 卡（L0内部）→ 派生 L1老板诊断卡 / L2店长执行卡（走 [[mdcard]]）
