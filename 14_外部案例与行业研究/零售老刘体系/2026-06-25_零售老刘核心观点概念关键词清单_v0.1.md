# 零售老刘体系 · 核心观点/概念/关键词清单 v0.1

> **诚实声明**:本清单来自**三层阅读法吸收**(skim 全量 + 28 簇/B1-B5 批 extract 核心命题),**非 321 篇逐字精读**。反映真实吸收的命题层。

## 一、十大核心观点（提炼)
1. **品类管理本质 = 用结构化逻辑应对非结构化、动态、不确定的零售**;品类建模(引数据建模)是解这个矛盾的路径。
2. **品类 = 顾客要完成的"任务"(JTBD),不是商品集合**;隐性任务(顾客不直说)是需求侧的真目标。
3. **单品(SKU)是战略单元**;Item(需求单元/窄品)≠ SKU(最小可售单元),区分决定分析精度。
4. **类要清晰**:管理类名 ≠ 顾客类名;POP/类标必须用顾客认可的类名。
5. **陈列 = 基于注意力的漏斗系统**(非装饰);纵向陈列把"类优先"投影成视线垂直扫描。
6. **保利期**:毛利会被库存时间消耗;超保利期 = 隐性亏损(即使毛利率高)。
7. **可得性**:低销可能是缺货(伪滞销),不是真滞销;实际可用=理论×现货率×履约率。
8. **购物篮/连带率(品单数)= 最重要经营指标**;研究"如何让顾客往篮里多放商品"。
9. **熵增 = 零售底层逻辑**;系统化(品类树/分层/陈列)= 持续抗熵增。
10. **商家视角 ≠ 顾客视角 = 失真根**(卖点≠买点 / 类名失真 / 管理者诅咒同源);先需求后选品。

## 三、核心关键词谱（速查)
- **结构层**:维度·度量 / 宽类窄品 / 类·Item·SKU / 需求单元 / 主树·经营属性线
- **顾客层**:JTBD·任务类 / 隐性任务 / 买点·卖点·痛点·痒点·爆点·爽点 / 顾客需求指数 / 小票客群 / RFM / 连带率·品单数
- **陈列层**:VMD / 3P黄金角 / IP / Z定律·F定律 / POP / 注意力漏斗 / 纵向陈列 / 类要清晰 / T型技能
- **量化层**:ABC·ABCZ / 30:70 / 价格带·IP-Chart / GMROI / 售罄率 / 安全库存 / 库龄·账龄 / 保利期 / 可得性(现货率·履约率)
- **数据层**:商品信息表 / PIM / 属性·标签·字段 / 属性▶标签▶需求▶业务 / 万物皆维度·万事可量化
- **经营层**:盈亏平衡 / 销售预测 / 品类指标分解 / 卖场经营指数 / 三条主线(库存·品类·数据)
- **哲学层**:熵增 / 结构化矛盾 / 商家视角≠顾客视角 / 农耕vs狩猎(关系思维) / 长期主义
