# Cluster C 价格带与毛利 v6.0 证据对齐卡 v0.1

> v6.0纪律:逐篇读+证据对齐+冲突检查+门店可用性。**不造新框架、不新增R-规则、不抽象总结。**
> 来源归「零售老刘」(external_reference)。

## 卡 C1 — 14 小类价格带建模 ★（M-DEC-010 方法核心）

**1. Fact Layer**:不同小类用不同价格带(休食≠厨房用品);价格段分 0-3/3-5/5-10 区间;核心区间(销售/毛利占比高)细分、差段粗分;小类×9段(笛卡尔积);输出"小类×价格带×销售/客流"矩阵判合理性。
**2. Application Layer**:门店可建"小类×价格带"矩阵,直接看每小类价格分布是否过密/漏档。✅可用于**定价/价格带诊断**。
**3. System Alignment**:支持 **M-DEC-010(价格带预警=此建模法)**;支持 **FC03(过密漏档=矩阵里某段SKU%畸高/相邻跳空)**;对齐 Cluster B R-SKU-13(双峰)。
**4. Store Impact**:🔧调价/价格带——T03饼干建"价格带×销售"矩阵→看中价段是否>70%(过密)。
**5. Confidence**:**高**(方法可落地,缺当期数据)。

## 卡 C2 — 商品组合11 毛利目标模型

**1. Fact Layer**:盈亏平衡毛利率=费用÷销售(例月销500万费用100万→20%);+利润率3%→毛利率23%;毛利分解大→中→小→货位;毛利波动±0.5%基准;三级预警(黄单项超5%/橙连续2月/红净利1%);自动调价引擎可提1.5-5个点。
**2. Application Layer**:门店可算自身盈亏平衡毛利率,判某品类毛利是否够本。✅可用于**定价/品类毛利目标**。
**3. System Alignment**:支持 **M-DEC-010(毛利结构错配=毛利率达不达盈亏线)**;对齐 Cluster A(滞销1天+0.3-1%资金成本)。
**4. Store Impact**:🔧调价——T03饼干毛利~20%≈盈亏线=**薄利品类**,贡献毛利有限。
**5. Confidence**:**中**(盈亏公式硬,部分AI泛框架)。

## 卡 C3 — 商品组合12 毛利模型成本

**1. Fact Layer**:成本8项(采购/头程/推广/经营/FBA/仓储/退货/其他);超龄库存费(>180天);食品周转≤30天,每滞销1天+0.3-1%资金成本;坪效0.8-1.2万/㎡/年;人效20-50万/年;好位置毛利权重1/较好0.8/一般0.5。
**2. Application Layer**:门店可按位置分配毛利权重、算滞销资金成本。✅可用于**陈列/淘汰决策(资金成本)**。
**3. System Alignment**:支持 M-DEC-010(成本结构);对齐 **Cluster A(超龄库存费>180天=呆滞R-INV-14)**;对齐 FC05(位置毛利权重)。
**4. Store Impact**:🔧淘汰——T03 14滞销×0.3-1%/天资金成本=持续失血,印证清货。
**5. Confidence**:**中**(成本8项具体,偏财务)。

## 卡 C4 — 商品组合13 毛利业务语义

**1. Fact Layer**:毛利率=(销售-成本)/销售;三把斧(毛利率/净利率/ROE);毛利率多维(行业基准/趋势/规模/财务联动/商品组合/时段性);高毛利商品 vs 低毛利走量品;毛利率方向标=商品组合优化。
**2. Application Layer**:门店可多维看毛利率(非单数字),判经营含金量。✅可用于**品类毛利诊断**。
**3. System Alignment**:支持 M-DEC-010(毛利结构);对齐 Cluster B(高毛利 vs 走量品=超低价R-SKU-12)。
**4. Store Impact**:🔧调价/SKU——T03需分"走量低毛利"与"形象高毛利",非一刀切。
**5. Confidence**:**中**(三把斧公式硬,多维偏分析)。

## 卡 C5 — 商品组合21 折扣的逻辑 ★（解释3负毛利特价）

**1. Fact Layer**:折扣=消费趋势(反向消费/性价比)+库存管理(临期/保利期/库龄);临期(1年保质期→到期前45天);**DOS=平均库存额÷日均销售**;DOS超标(快消30-45天)→折扣;有效库存比<70%→滞销加大折扣;保利期内卖掉,库龄达保利期50%启动阶梯折扣。
**2. Application Layer**:门店可按DOS/库龄/保利期触发折扣清仓。✅可用于**淘汰/折扣清货**。
**3. System Alignment**:支持 **M-DEC-009(临期/库龄折扣=滞销处理)**;对齐 **Cluster A(R-INV-13清滞销长保质期)**;对齐封存包"3特价清尾"。
**4. Store Impact**:🔧折扣清货——**T03 3个负毛利特价清仓 = 此逻辑**(临期/库龄→阶梯折扣)。
**5. Confidence**:**高**(DOS/45天/70%/50%阶梯可执行)。

## 卡 C6 — 选品配置12 商品硬折扣

**1. Fact Layer**:硬折扣(ALDI)=极简SKU(<2000甚至<1000)+PB化70-80%+规模采购+极致成本;主短尾非长尾(二八);一般超市动销SKU 3500-5000;软折扣=营销驱动(锚定/促销/关联/紧迫);硬=成本驱动、软=营销驱动。
**2. Application Layer**:门店可判自身SKU是否过多(对标动销SKU数)。✅可用于**SKU结构精简**。
**3. System Alignment**:支持 **FC02(SKU数约束)**;对齐 Cluster B R-SKU-07(精而准);对齐 R-SKU-12(超低价走量=硬折扣低价)。
**4. Store Impact**:🔧SKU结构——花厅坊=社区店,刚需品可借硬折扣(少SKU+超低价走量)。
**5. Confidence**:**中**(SKU数对标具体,偏战略)。
