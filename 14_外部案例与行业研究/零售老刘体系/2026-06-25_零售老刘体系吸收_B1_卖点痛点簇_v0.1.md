# 零售老刘体系吸收 · B1 卖点痛点簇

> [[2026-06-25_零售老刘全量深度吸收实施计划_v1.0]] B1 批(P1·商品力)。三层阅读法 extract 级。转述非引用。

## 一、核心命题（转述)
| 篇 | 命题 |
|----|------|
| 62 卖点提炼 | **客单=购物小票**(客流/购物篮语义,非客单价);错把卖点当买点 |
| 65 痛点属性 | **痛点可量化**(痛点万能公式);**情感对痛点起"平方级"作用**;痛点可逐渐清晰化 |
| 66 痛点组合 | 找到顾客需求痛点;痛点与门店经营协合;欲望与需求博弈 |
| 67 卖点量化 | **卖点=痛点的综合**;卖点4量化指标;需求词谱=卖点/买点/痛点/痒点/爆点/爽点 |
| 72 商品线体系 | 商品线=商品分类 + 属性定义 + **消费者认知**的语义化表达 |

## 二、最大收获
1. **买点/痛点 = 需求侧的量化抓手**(接 [[2026-06-25_零售老刘体系吸收_E2_顾客需求侧簇_v0.1]]):E2 给了"小票客群法",B1 给"痛点量化"——需求侧从"有没有需求"(需求指数)深化到"痛在哪、可量化"(痛点公式)。
2. **再坐实元原则「商家视角≠顾客视角」**:卖点(商家说好)vs 买点/痛点(顾客真要)。与 类名失真型 + 卖点≠买点(收尾簇)合流。
3. **客单=购物小票** 印证 E2 小票法(客单不是客单价,是购物篮/客流)。
4. **商品线 = 分类+属性+认知** 接 [[品类管理]] 经营属性线(认知是属性的一层)。

## 三、回填候选（记入回填清单·正文须签)
1. **需求侧深化**:M5 / 诊断加"痛点"视角(顾客买这个小类解决什么痛点);痛点可量化(配小票法)。
2. **需求词谱**:卖点/买点/痛点/痒点/爆点/爽点 作选品/媒体选题的需求颗粒度参考。
3. **元原则**入失真库总则(已在收尾簇提出,B1 加证)。
