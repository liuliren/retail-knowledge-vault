## 本簇必答(用户指定 A–G)
| 问 | 答(精读结论)|
|---|---|
| **A. ABC 怎么做?** | **小类内 ABC**(原文明示比全店更实际)+ **综合值 ABC**(销售/销量/毛利加权,非单指标)。花厅坊休食优先:**小类内综合值 ABC**(休食小类差异大)。**ABC 不能直接淘汰**——它是当期快照,含假A类(断货掩盖)、不含衰退趋势,只能作初筛。 |
| **B. 双ABC衰退怎么定义?** | 去年A/B 今年C/D 或消失 = 衰退信号,**不直接淘汰** → 进观察,先分型(断货/替换/生命周期/系统假象)。双ABC = M-DEC-009 查2,是**提前一周期的衰退预警**,非淘汰指令。 |
| **C. 售罄率×ABC 联动?** | 售罄率高≠好(可能订货太少/断货致假高);售罄率低≠该淘汰(可能缺货/陈列差/价格错位)。须与 订货量+库存+周期+毛利 联动。**46.7% 平衡点应进老品淘汰判断链**(低于46.7%且非缺货=真亏损滞销)。 |
| **D. 老品淘汰防误杀?** | **六桶分类**:真滞销 / 假滞销-缺货 / 假滞销-陈列差 / 假滞销-价格带错位 / 生命周期衰退 / 系统-条码-替换假象。每桶不同处置,仅"真滞销"进淘汰。 |
| **E. M-DEC-009 增强?** | 建议(不改临界值):①双ABC衰退4分型 ②售罄率×ABC联合判据 ③淘汰桶 4→6 扩 ④防误杀优先级 ⑤回填字段。详 §10。 |

| **G. M2 回填?** | ABC=初筛非终判 / 双ABC=衰退识别 / 售罄率=时间校验 / 淘汰四重防误杀 / M2↔M1·M3·M4 联动。详 §12。 |

## 2. 原文核心命题
- **ABC 是系数不是结论**:ABC 值用于修正业务(订货/陈列/促销),不是淘汰判决书。
- **小类内 ABC 更实际**:各小类各自排序,避免大类掩盖小类的 A 品。
- **销量≠需求**:断货掩盖真实需求 → "假A类"普遍,按销量末位淘汰=极端错误。
- **售罄率必带时间维度**:用"达50%售罄的速度"判畅销,而非绝对销量。
- **需求管理 > 需求预测**:预测必错,重点是即时纠偏。

## 3. ABC 分类方法(原文)
| 维度 | 内容 | 来源 |
|---|---|---|
| 单ABC | 销售/销量/毛利/综合值/敏感度 各自一套 | 03 |
| **综合值ABC** | 销售×w1+销量×w2+毛利×w3 加权后排序 | 03 |
| 双ABC | 两时段对比(去年×今年 / 两度量组合)| 03 |
| 3ABC | 类+课+位 三属性组合 | 03 |
| 粒度 | **小类内ABC > 全店ABC**(更贴业务)+ 层级粒度(大/中/小类ABC)| 03 |
| 标准比 | 10%A产50% / 30%B产40% / 60%C产10%;**实践15-17%A产60%** | 03 |
| ABC系数 | 订货AA×1.8;陈列A+2~3倍面/C−20~30%;缺货率A+20%/B−10%/C−25% | 03/零模02 |

## 4. 双ABC / 衰退判断链
```
双ABC = 前位(去年) + 后位(今年)
├ _A:去年无、今年A → 新晋(关注成长)
├ AA/AB:持续强 → 维持
├ A_ / AC / BD:去年A/B 今年C/D或消失 → 衰退信号(不直接淘汰!)
│   └ 分型:
│       ① 断货致衰(库存=0占比高)→ 补货,非淘汰
│       ② 替换品上位(同需求新SKU顶替)→ 正常换代,汰旧留新
│       ③ 生命周期结束(品类整体退潮)→ 计划性淘汰
│       └ ④ 系统/条码假象(改码/合并/停采未删)→ 数据修正,非淘汰
└ 衰退确认 + 真滞销 → 才进淘汰
```
> 双ABC 的价值是**提前一个销售周期预警**(等 days_no_sales 触发已晚)。

## 5. 售罄率与 ABC 的关系
| 联动 | 说明 | 来源 |
|---|---|---|
| 售罄率高 ≠ 好 | 可能订货太少/断货 → 假高(真实需求被压制)| 25 |
| 售罄率低 ≠ 淘汰 | 可能缺货/陈列差/价格错位 → 假低 | 25/零模02 |
| 速度判畅销 | 相同周期"达50%售罄"越快越畅销 | 25 |
| 平衡点 | 售罄率 < 46.7%(=毛利率/(1−费用率))且非缺货 → 真亏损滞销 | 25 |
| 与ABC联合 | A类低售罄=优先查缺货/陈列;C类低售罄+有货=淘汰候选 | 综合 |

## 6. 老品淘汰的误杀风险(六桶)
| 桶 | 特征 | 处置 | 主武器 |
|---|---|---|---|
| ① 真滞销 | 有货、售罄率<46.7%、双ABC衰退确认、非缺货 | **淘汰**(进M-DEC-007 SOP)| 售罄率+双ABC |
| ② 假滞销-缺货 | 无销售期内 库存=0占比高 | 补货 | M-DEC-009查1 |
| ③ 假滞销-陈列差 | 在非黄金层/低空间系数位 | 换位(M-DEC-011)| 空间系数 |
| ④ 假滞销-价格带错位 | 落在过密/断档/漏档段 | 调价/调结构(M-DEC-010)| 价格带 |
| ⑤ 生命周期衰退 | 双ABC衰退+品类整体退潮 | 计划性汰换 | 双ABC |
| ⑥ 系统/条码/替换假象 | 改码/合并/替换品/停采未删 | 数据修正,不算淘汰 | replacement_flag |
> 现 M-DEC-009 是"四桶"(真滞销/缺货/季节/衰退);本簇建议扩到**六桶**(补陈列差③、价格带错位④、系统假象⑥)。

## 7. 关键公式 / 计算口径(每条带来源)
| 公式 | 表达式 | 来源 |
|---|---|---|
| 售罄率② ⭐ | **销量 / 订货量 ×100%** | 25 |
| 平衡点售罄率 | **毛利率/(1−费用率)** = 0.35/0.85 = **46.7%** | 25 |
| ABC标准比 | 10%A产50% / 实践15-17%A产60% | 03 |
| ABC订货系数 | AA类 ×1.8 | 03 |
| 缺货率系数法 | A+20% / B−10% / C−25%(合计−15%)| 零模02 |
| ABC陈列系数 | A类+2~3倍面 / C类−20~30%面 | 零模02 |
| 综合值ABC | Σ(销售/销量/毛利 加权)后降序累计 | 03 |
| 动销率 | 动销品数/总进货品数(卖场例51.5%)| 25 |

## 8. 反常识点
- **销量高/销额高 ≠ 畅销品**;假A类普遍("畅销的才是缺货的")。
- **ABC 不能直接淘汰**:它是当期排名,无时间维度、含假A类。
- 末位淘汰(按销量倒序砍)**极端错误**。
- 双ABC 都缺货时合计缺货率相同(−15%),但 A+20/B−10/C−25 业务逻辑完全不同。
- 售罄率 100% 可能是**订货不够**(满足不了需求),不是完美。

## 9. 对花厅坊的可用部分
- 休食 7139 滞销:用**六桶**重分类,只让"真滞销"桶进淘汰,堵误杀。
- 小类内综合值 ABC:对 12 个 L3(薯片/饼干/巧克力…)各自排 ABC,接动销表 abcd_grade。
- 售罄率②:对休食 P0 淘汰候选先算 销量/订货量,<46.7% 且非缺货 → 才淘汰。
- 双ABC:对德芙/乐事等去年A今年掉级品做衰退预警。

## 12. 对 M2_选品配置簇.md 的增量回填(§6 二遍补强)
- **ABC = 选品/汰换初筛,非最终决策**(小类内综合值ABC)。
- **双ABC = 衰退识别工具**(提前一周期预警,4分型)。
- **售罄率 = 时间维度校验工具**(②式 + 46.7%平衡点)。
- **老品淘汰四重防误杀**:库存(缺货)+ 订货(售罄率)+ 陈列(空间系数)+ 价格带 → 才能淘汰。
- **M2 联动**:初筛(M2 ABC)→ 库存查缺货(M1)→ 陈列查错位(M4)→ 价格带查错配(M3)→ 真滞销才淘汰。
