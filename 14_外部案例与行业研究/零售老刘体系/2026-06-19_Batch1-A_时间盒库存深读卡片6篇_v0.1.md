## 卡片 ① 再谈零售企划真言（A组·判断纪律）

> ⚠️ 真实正文校准:本篇**不讲"时间盒"技术**,讲的是 **企划=战略整体性 + 长期主义 + 系统一致性**。按正文归档,不硬套时间盒。

| 段 | 内容 |
|---|---|
| **1 判断链** | 经营好坏从单一财务/营运角度 ≠ 从企划(整体)角度 → 单点(单日销售/单部门流程)正常不代表系统正常 → 必须看"线"(部门间制度/流程/绩效连接)和"面"(系统闭环) → 找木桶短板 → 战略层长期纠偏 |
| **2 经营命题** | **零售经营不能用单点指标代替系统判断;局部正常 ≠ 全局健康,短期结果 ≠ 长期能力。** |
| **3 模型结构** | 「点-线-面」判断分层模型:点(单SKU/单日/单部门)→线(跨部门连接)→面(系统闭环)。配「木桶原理」:整体由最短板决定。 |
| **4 输入变量** | 非数据型,是判断视角变量:诊断单元(点/线/面)、时间跨度(短期/长期)、一致性基准(是否符合定位)。 |
| **5 错误来源** | ①把"点"当"面"(改一个SKU/一个流程当解决了问题);②短期主义(月末冲数);③无章法累加(每个POP/布局各行其是→形象模糊);④伪企划(夸大单一手段=POP/促销=企划)。 |
| **6 修正机制** | 任何局部动作回到「是否符合企业定位/系统一致性」;诊断从单点上升到线/面;以战略长期目标而非短期单点为准绳。 |
| **7 M1-M8归档** | **跨模块元纪律**(不归单一模块)。直接约束 M6品类结构、M7绩效经营的"系统视角";约束顾问诊断方法论本身。 |
| **8 FC映射** | 不进具体 FC(无触发条件/字段)。**作为 FC 池的上位判断纪律**:任何 FC 不能只看单点(如只看销量)→ 支撑 FC01/FC04 的"反单点误判"立场。 |
| **9 M-DEC映射** | 不增强具体 M-DEC。**是 M-DEC-009"防误杀"的哲学根**:不可凭单点(低销)杀品,须看线面(缺货/陈列/价格带/生命周期)。 |
| **10 花厅坊影响** | T03 诊断纪律:**不能因"长期无销售"就单点判滞销**,必须看线(缺货/价格带/陈列)和面(品类角色)——长期无销售商品中同时存在有库存与零库存两类，需要分别判断。直接支持 v0.2 简报"先六桶细判再杀"。 |
| **11 反误判风险** | 若误读为"企划=务虚":会忽视系统性短板,继续单点救火(砍一个SKU补一个SKU),门店结构问题永不解决。**被误砍**:被单点低销判死、实则缺货/陈列差的品。 |

## 卡片 ② 提示内容-系列的三条主线 告知（A组·阅读框架锚点）

| 段 | 内容 |
|---|---|
| **1 判断链** | 零售建模须按业务语义主次/顺序/层级展开 → 三条主线递进:需求单元线 → 商品单元线 → 商品业务线 → 缺前面语义则后面概念无根 |
| **2 经营命题** | **零售数据体系有固定语义骨架:先需求(点线面/人货场)、再商品(SKU/Item/品类)、再业务建模(维度+度量)。脱节读=缺语义地基。** |
| **3 模型结构** | 三主线模型:①需求单元线(人货场结构/货位/购物篮需求分层/会员/品类细分 + 度量:基础类指标+购物篮指标);②商品单元线(SKU→Item→品类→商品线→区域,商品维度/元素体系);③商品业务线(维度+度量数据建模驱动业务)。 |
| **4 输入变量** | 维度:人/货/场、货位、购物篮分层、会员、品类细分;商品元素:SKU/Item/品类/商品线/区域;度量:基础类指标、购物篮指标。 |
| **5 错误来源** | 选择性跳读 → 缺语义铺垫 → 后续概念(如售罄率/安全库存/ABC)无地基 → 误用。 |
| **6 修正机制** | 连续性阅读;先建需求维度→再商品维度→再建模。本系列=Batch 全程的"语义索引"。 |
| **7 M1-M8归档** | **全局框架**:需求线→M5顾客需求/M6品类结构;商品线→M2选品配置;业务线→M8数据建模。是 M1-M8 之上的"语义排序根"。 |
| **8 FC映射** | 不进具体 FC。**定义 FC 的语义层级**:需求点(FC01)在需求线、SKU约束(FC02)在商品线、建模规则(FC06/FC10)在业务线。 |
| **9 M-DEC映射** | 不增强 M-DEC。**为 M-DEC 提供语义坐标**:每条 M-DEC 应能定位到三主线的哪一层。 |
| **10 花厅坊影响** | T03 三级语义(宽类→Item中间类→SKU)正是"商品单元线"的落地;需求点属"需求单元线"。**确认 T03 方法骨架与零售老刘语义同构。** |
| **11 反误判风险** | 若跳过此框架:会把"需求侧问题"(顾客不需要)和"商品侧问题"(SKU配置错)混为一谈,误判滞销根因。 |

## 卡片 ③ 零售超市库存管理系列（01）（B组·库存底座)

| 段 | 内容 |
|---|---|
| **1 判断链** | 库存=闲置资源占资金 → "不合理"是缺参照/标准/系统的代名词 → 绝对数量无意义 → 改看相对增量(以时点为基数看业务量变化下库存变化是否合理) → 进货95%在订货周期内卖出且补回=合理库存 |
| **2 经营命题** | **离开需求谈库存合理与否=脱裤子放屁;库存合理性只能从"相对增量+周转"判断,不能从绝对数量判断。** |
| **3 模型结构** | 「相对增量」库存合理性模型:以时点库存为基数,业务量变化Δ → 判断库存变化Δ是否匹配。配套参照:损耗率 0.4% 合理线、边际周转率(业务增15%库存增<15%)、ABC优化结构、库存资金上限(浮动超10%警戒)。 |
| **4 输入变量** | 日均销量DMS、送货周期、安全库存、周转率、损耗率、业务增长率、库存增量、库存资金额。 |
| **5 错误来源** | 拍脑袋订货"猜不准"三因:①需求多样化从未分析判断;②采购渠道单一+缺库存结构分析("似乎都不好卖又都能卖一点");③订货越细而技术停在5-10年前。 |
| **6 修正机制** | 用销售计划+预估销量分析、订货系统、商圈需求指导的结构调整、ABC、定期盘点、边际周转率考核 → 把"猜"变"算"。 |
| **7 M1-M8归档** | **M1库存订货**(底座)。损耗率/周转率→M7绩效;ABC结构→M2选品配置/M6品类结构。 |
| **8 FC映射** | **FC10(库存积压联动)直接来源**:相对增量模型=库存增量是否匹配业务增量的触发逻辑。**部分支持**(有判断逻辑,缺字段化阈值)。 |
| **9 M-DEC映射** | 不直接增强。为 M-DEC-009 提供"周转/相对增量"补充视角(老品是否该汰,看相对增量而非绝对库存)。 |
| **10 花厅坊影响** | T03"疑似滞销商品中同时存在有库存与零库存两类":**库存=0 不能用绝对库存判滞销**(本篇:绝对数量无意义);要看有货品的相对周转。直接支持 §18 库存join 结论。 |
| **11 反误判风险** | 若用绝对库存数判滞销:库存=0 误判为"卖光=好"或"没货=滞销"两头错;库存高 误判为滞销实则刚到货。**被误砍/误留**:刚补货的高库存品被当滞销;缺货断档品被当淘汰。 |

## 卡片 ④ 零售超市库存管理系列（02）（B组·两核心+闭环+订货公式)

| 段 | 内容 |
|---|---|
| **1 判断链** | 优秀库存管理=组织+系统+流程(能力,非只奔结果) → 两核心:最优库存量 + 最大限度解决动态需求 → EOQ解第一、安全库存解第二 → 需求计划须闭环(以销定采→以采定销) → 否则销售对库存不负责 |
| **2 经营命题** | **库存管理的本质是"需求计划闭环"+"两个核心数学问题",不是仓库管理;销量≠需求,采购回的不全是顾客要的。** |
| **3 模型结构** | ①EOQ:理想订货量 Q=DMS×订货周期;②安全库存(正态分布,90%=1.28/95%=1.64);③**订货公式 Q=d(T+L)+Zσ√(T+L)−I**(d=DMS,T+L=采购周期,Z=服务水平系数,σ=日销标准差,I=现有库存);④闭环需求计划:以销定采↔以采定销。 |
| **4 输入变量** | DMS(d)、采购周期(T+L)、服务水平Z(通常1.64)、日销标准差σ、现有库存I、持有费用Holding、订单费用Ordering、周转天数 vs 月结30天。 |
| **5 错误来源** | ①只奔结果不改能力(组织/系统/流程缺一不行);②需求计划开口(不与销售达成共识)→销售对库存不负责、采购成替罪羊;③把库存管理误解为"仓库管理/先进先出/货位陈列"。 |
| **6 修正机制** | 闭环需求计划(采购按计划供应+销售按计划销售,两头凑);培养1-2个懂信息的计划员用系统;周转天数控制<月结天数(否则货没卖现金已付)。 |
| **7 M1-M8归档** | **M1库存订货**(核心公式所在)。需求闭环→M5顾客需求/M7绩效;周转天数vs月结→M8数据建模/财务。 |
| **8 FC映射** | **FC06(售罄速度/安全库存)核心来源**:订货公式+安全库存=售罄/补货节奏的数学根。**FC04(缺货防误判)**:σ大→需求波动大→不能用单期缺货判滞销。**支持**(有公式/字段/触发,可字段化)。 |
| **9 M-DEC映射** | 增强 M-DEC-009:"销量≠需求"是防误杀第一原理(采购回的不全是顾客要的→低销可能是配错不是顾客不要)。 |
| **10 花厅坊影响** | T03:**库存=0=缺货(I=0),不是顾客不要**;按本篇 Q=d(T+L)+Zσ√(T+L)−I,I=0 时只说明该补货,绝不能判滞销。**这是"须先看有货品才可判滞销"最硬的公式依据。** |
| **11 反误判风险** | 若忽略闭环/σ:把缺货(I=0)当滞销→误砍真实有需求品;把高σ(波动)当稳定→安全库存设错→断货或积压。**被误砍**:波动型刚需(如节令送礼饼干)在淡季被当滞销。 |

## 卡片 ⑤ 零售超市库存管理系列（03）（B组·周转率真相+1.5倍+内仓零库存)

| 段 | 内容 |
|---|---|
| **1 判断链** | 传统周转率=销售成本/((期初+期末)/2)被误导 → 点库存可造假(期初末为0→周转率∞) → 改用每日库存现金系数 → 单品合理库存=日均销售额1.5倍 → 陈列量=1.5倍订货单位 → 内仓零库存 |
| **2 经营命题** | **点库存(期初末平均)会骗人,真实效率看"每日库存现金系数";单品合理库存≈日均销量×1.5,陈列=1.5×订货单位可实现内仓零库存。** |
| **3 模型结构** | ①**库存现金系数 IDSM=AVG(D1..Dn)/AVG(S1..Sn)**(D每日库存,S每日成本);②单品合理库存=DMS×1.5(1.5≈(1.64+1.28)/2);③陈列量=1.5×订货单位;④内仓零库存逻辑(订货单位12→陈列18→进24=18上架+6内仓→剩6再进12→全上架);⑤边际周转率(总库5周+分库2周=7周→52/7=7.4;新增5周→52/5=10.4)。 |
| **4 输入变量** | 每日库存D、每日成本S、DMS、订货单位、陈列量、Z值(1.64/1.28)、总库/分库周数、业务增长率、库存增量上限。 |
| **5 错误来源** | ①传统周转率→月末/季末疯狂控库存(压供应商/不入系统/期末清仓);②点库存掩盖每日真实变化(销售/报损/领用/退货/调拨);③大超市追求"广全/一次性购足"→库存效率持续下降。 |
| **6 修正机制** | 用每日库存现金系数替代点库存周转率;按 DMS×1.5 设单品库存;陈列=1.5×订货单位做内仓零库存;边际周转率考核新增业务(增量库存周转须高于存量)。 |
| **7 M1-M8归档** | **M1库存订货 + M8数据建模**(现金系数=建模指标)。陈列量=1.5订货单位→M4陈列货位;精简商品数→M2选品配置。 |
| **8 FC映射** | **FC06(售罄速度)+FC10(库存积压)双来源**:1.5倍=合理库存上限基准(超出=积压信号FC10);现金系数=资金占用预警。**支持/可字段化**。 |
| **9 M-DEC映射** | 增强 M-DEC-009:单品库存>DMS×1.5 远超→积压(可能滞销);但 DMS=0 且 I=0→缺货非滞销。提供"汰换"的库存量化线。 |
| **10 花厅坊影响** | T03:可对**有货品**算"实际库存÷(DMS×1.5)",远大于1=积压验证滞销;**I=0的品**无DMS基础,本模型不适用→再次证明不能判滞销。给当期库存核查一把量化尺。 |
| **11 反误判风险** | 若用传统点库存周转率:被月末造数误导,看不出真实积压;把刚到货(高库存)当积压,把缺货(I=0)当健康。**被误留**:长保质期滞销品(仓库常储存这类,与零库存背道而驰)。 |

## 卡片 ⑥ 不成立三角 01 高库存的逻辑（B组·供应链整体平衡)

| 段 | 内容 |
|---|---|
| **1 判断链** | 高库存=7大异常之一,但表象 → 任何"不合理"库存本质=供应链"效率"问题 → 一叶知秋(背后藏门店经营问题) → 大采购是"果"非"因",高流通性(高周转)才是因 → 须整体/全局平衡,非局部最优 |
| **2 经营命题** | **高库存只是症状,根因是供应链效率失衡;局部最佳化 ≠ 全局最佳化,"门店报货次日配送到位"只是单环节最优,非整体平衡。** |
| **3 模型结构** | ①不成立三角(要素与核心目标内在冲突,无法同时最优);②道生一(商品维度)→二(需求维度)→三(业务维度)→万物(建模);③库存三类:结构性存货(布局)/运作存货(周期)/应对波动存货(生产/需求/供应波动);④零库存配置三关键维度:满库存量、订货单位、DMS;⑤长鞭效应(信息互通解)。 |
| **4 输入变量** | 生产周期、等待周期、供应链节点时间、需求波动(季节/促销/团购)、供应波动(货期/质量/价格)、满库存量、订货单位、DMS。 |
| **5 错误来源** | ①把高库存当孤立商品问题(实为供应链);②局部最优当全局最优(单环节"准时配送"自满);③产供销各自为政(各自视角→长鞭效应→各环节都造高库存);④仓库储存大量滞销长保质期品(与零库存初衷背道)。 |
| **6 修正机制** | 整体/全局/长期主义平衡(而非局部);信息流物流互通破长鞭;明确责任主体;聚焦高流通性(因)而非大采购(果)。 |
| **7 M1-M8归档** | **M1库存订货 + M6品类结构**(整体平衡)。供应链效率→M7绩效/M8建模。 |
| **8 FC映射** | **FC10(库存积压联动)语义根**:积压=供应链效率问题,须联动订货/需求/周转多维,非单看库存。**部分支持**(思维框架,缺可执行阈值)。 |
| **9 M-DEC映射** | 不直接增强。**为所有库存类 M-DEC 提供"整体观"边界**:不可单维度(单看库存/单看销量)下杀品结论。 |
| **10 花厅坊影响** | T03:**滞销不是单SKU问题,是该小类供应链效率(订货/需求匹配/周转)的具象**。"长期无销售商品群"应整体看(价格带过密/无高端/中价拥挤=结构问题),不是逐个SKU救火。呼应卡片①点线面纪律。 |
| **11 反误判风险** | 若把高库存当孤立SKU处理:头痛医头(砍一个又冒一个),7大异常天天搞月月做不了了之。**被误砍/误留**:供应链/订货问题造成的"假滞销"被当真滞销砍掉,根因(订货不准)仍在。 |

## Batch 1-A 总结构图

### 一、时间盒→库存→SKU→误判链(主链)

```
[判断纪律层] 卡片①点线面/长期主义 + 卡片②三主线语义骨架
        │ (元纪律:不能用单点代替系统判断)
        ▼
[库存认知层] 卡片③ 离开需求谈库存=无意义 / 相对增量才有效
        │
        ▼
[库存数学层] 卡片④ Q=d(T+L)+Zσ√(T+L)−I  ← 销量≠需求的硬公式
        │      I=0=缺货≠滞销 / σ大=波动需高安全库存
        ▼
[库存效率层] 卡片⑤ 库存现金系数IDSM / 单品库存=DMS×1.5 / 内仓零库存
        │      (点库存会骗人 / 1.5倍=积压判断线)
        ▼
[供应链整体层] 卡片⑥ 高库存=供应链效率症状 / 局部最优≠全局最优
        │
        ▼
[误判出口] 缺货(I=0)误判滞销 ✗  /  刚到货误判积压 ✗  /  单SKU救火 ✗
```

### 二、销量≠需求 的三重证据(本批最强结论)

| 证据来源 | 论点 | 对花厅坊T03的判定 |
|---|---|---|
| 卡片③ | 绝对库存数量判合理性=无意义 | 库存=0的品不能凭库存判滞销 |
| 卡片④ | Q=d(T+L)+Zσ√(T+L)−I,I=0=缺货 | 库存=0的品=缺货,须补货非淘汰 |
| 卡片⑤⑥ | 仓库常存滞销长保质期品;1.5倍线 | 只有有货品才算"实际库存÷(DMS×1.5)"判积压 |
| **合流** | **销量≠需求,库存=0时尤其不可判滞销** | **T03"须先剔除零库存品才可判滞销"= 公式级成立** |
