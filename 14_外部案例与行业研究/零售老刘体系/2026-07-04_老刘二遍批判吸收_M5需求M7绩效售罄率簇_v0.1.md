# 老刘二遍批判吸收 · M5顾客需求 + M7绩效 + 售罄率簇

> **status: draft · 审议轮次 1/3 · 未签字前不具约束力。**
> 本批 6 篇：10顾客需求指数 / 12顾客需求与类别属性 / 09卖场经营指数 / 25聊聊售罄率 / 26售罄率建模运用01 / 27售罄率建模运用02。
> **批判焦点**：这些量化模型都建在「有规范 POS 小类明细 + 会员/客流数据」的假设上——社区生鲜小店有没有这个数据基础？没有时怎么退化用？与现有 KB-RETAIL-KPI-001 / CONSUMER-001 命题哪里冲突？

## 0. 一句话结论（结论先行）

这 6 篇的真金 = **4 组可搬公式**；但 4 组模型的数据门槛从高到低差异极大，**社区生鲜小店只有 2 组能直接落地（售罄率第2公式 + 类别RFM 需求侧），另 2 组（卖场转化率、EOQ/三线库存建模）在社区小店是"知道有、但低 ROI / 退化成经验估算"**。老刘自己在原文里已多处承认"很少用于建模""国内卖场很少关注"——这批的批判价值，主要在**把每个模型的数据前提和失效边界钉死**，防止顾问把大企业/系统化语境的模型硬搬到 500–2000㎡ 社区店。

## 1. 四组可搬公式（原文萃取·转述非引用）

### A. 售罄率 4 公式（25/26/27）⭐核心payload

| # | 公式 | 语境/用途 | 社区生鲜可用性 |
|---|---|---|---|
| 公式1 | 售罄率 = 进货成本 / 销售额 | 成本回收视角 | 中 |
| 公式2 | 售罄率 =（某段时间内）销售数量 /（同时间段内）订货数量 ×100% | 高频日配品·每天订货 | **★最可落地** |
| 公式3 | **售罄率 = 毛利率 /（1 - 费用率）** ＝ 盈亏平衡售罄率（指导标准）| 批次管理·服装/精品 | **低·不适用生鲜** |
| 公式4 | 售罄率 = 动销单品数 / 总进货单品数 | 动销率视角 | 中 |

- **公式3 推导实例（原文）**：毛利率 35%、费用率 15% → X = 0.35 /（1 - 15%）= **46.7%**（此售罄率=成本平衡点）。实际还要减去折扣率、损耗率、退货率再换算。
- **公式2 生鲜实操（原文牛肉例）**：18 公斤牛肉，早市须卖 70%（售罄率70%）、晚市 30%；早市完不成 70% → 晚市促销/折扣；早市提前到 95% → **说明订货量不够，缺失销售机会**。⚠️**反直觉铁点**：当日完成率 100% ≠ 好——很可能是订货不足，一直照此订货会系统性丢销售。

### B. 顾客需求指数 = 销售粒度下沉到「小类 × 天」（10）

- **一个小类 = 一个顾客需求**（老刘体系的公理，已在 E2/KB-KPI 命题1收）。
- **周系数** = 当前每个星期几的平均销售 / 该星期几的最低销售。
- **小类系数** = 按「年月×小类」求平均销售与系数（把周系数公式里的时期列换成小类码列）。
- **三种运用**：① 周系数判商圈类型（**7天系数合计越接近 7 = 越稳定 = 住宅区**；周六周日系数明显高 = 商业区）；② 小类周系数 → 订货/选品/绩效；③ 每日系数 → 每日指标制定。
- **增量（现 KB 系数分解法未含）**：用"7天系数合计是否接近7"反推商圈类型——**这是社区店可直接调用的小洞见**（花厅坊：拉近8周各星期几销售，算周系数合计，判断是纯社区盘还是带商业外溢盘，决定周末备货加成）。

### C. 类别 RFM = 把顾客 RFM 换主体到「小类」（12）⭐绕开会员数据

- **R** = 小类"最后被购买时间"距当前的天数（新建列：最后被购买时间 → 度量：当前时间相减）。
- **F** = 客流（直接引用原有客流度量 = 不重复小票数）。
- **M** = 销售额（直接引用销售额度量）。
- 各求平均值 → 大于均值计 **1**、否则 **0** → 得 8 类小类属性（借用客户 RFM 8 分类）。
- **可交叉验证**：把小类的 RFM 分级 × 客流/客单对照——如某小类是"重要价值类别"理想应对应"良好"，若实际对应"促销过度类别"→ 质疑：合理吗？为什么？怎么改。
- **核心批判价值**：**这是对"RFM 依赖会员数据"这一门槛的正面破解**——RFM 主体从顾客换成小类，只需 POS 小票 + 小类明细，**不需要会员卡**。（详见 §3 张力 T1）

### D. 卖场经营指数 = 转化率 + 驻停时间 + 价值指数（09）

- **转化率** = 有效购物客流 / 来店客流（原文例：1600/2000 = 88%，一般应控制在 **90%** 左右）。反映商品/货位/陈列/服务/商圈综合好坏。
- **购买率** = 购买顾客数 / 入店人数；购买顾客数 = 不重复小票数（POS 可得），入店人数需计数器/人工。
- **驻停时间**：分区停留（货架/商品区/收银台），与购买正相关**但有反例**（收银等待、投诉处理、找货时间越短越好）。
- **价值指数（顾客价值指数）**：两种算法——① RFM 法；② 主成分分析 PCA 法。**负价值指数**概念（易忽略）：顾客不知东西在哪 / 不知该买哪个 / 找不到想买的 / 服务差 → 都是负价值，须经营努力转正。
- **三方向数据思维**（已在绩效簇07收）：历史（同比）/ 即时（动态，"黄花菜都凉了"反例）/ 洞悉未来。
- **7-Eleven 收银键盘案例**（少/中/青/壮/老）= 低成本人工采集顾客画像的经典手法（详见 §4 遗漏）。

### E. 库存三线 + EOQ（26/27）配套payload

- **三种库存量**：安全库存量、管道库存量、最佳采购量。
- **管道库存 = 提前期 × 日均需求**（原文例：日均 100 × 提前期 5 天 = 500）。★极简可搬。
- **三条库存线**：安全库存线 / 再订购点线 / 库存上下限。**最大库存 = 再订购点 + 最佳定量**。
- **触发逻辑**：低于安全库存 → 紧急处理；低于再订购点 → 做采购订单；高于最大库存 → 清理库存。（多数店没设这三线 → 紧急处理退化成普通订单、清库存形同虚设）
- **需求管理取代需求预测**：预测总是错的（静态假设），要即时动态修正当前实际与预测的偏差。
- **EOQ 经济订货量**：总成本 = S + V×Q + D×i×T/2；EOQ = √(2DS / v·i)。原文例：D=3650、S=5、v=10、i=20% → EOQ=135（取 50 的倍数 ≈150）。**老刘自评：EOQ 很少用于建模，只是理论/粗估工具**；批量单品不多时用 Excel 经验估算即可。

## 2. 社区生鲜落地命题（每条带"无数据时怎么办"失效条件）

> 过 §1.2 三检验：直接调用性 / 场景具体性 / 张力（失效条件）。不过检验的降为参考储备，不占命题位。

### 命题 M5-1：生鲜日配品用「售罄率公式2 = 当日销量/当日订货量」，不用盈亏平衡售罄率
- **可调用**：花厅坊生鲜（肉禽/叶菜/日配）每天订货、天然有"订货量 vs 销量"两个数——直接把售罄率指标加进生鲜台账，按早市/晚市两段看售罄节奏（早市目标 70%、当日目标 95%+）。**当日完成率 100% 要警惕订货不足，不是达标**。
- **失效条件**：① 无 POS 或称重品当日订货量凭手记 → 用"进货重量 - 剩货重量"倒推售罄率，精度低但可用；② 公式3 的盈亏平衡售罄率（46.7% 这类）是**服装/精品批次管理语境，对当日清零的生鲜无意义，禁止套用**（张力 T2）。

### 命题 M5-2：社区店做需求侧 RFM 用「类别 RFM（小类为主体）」，绕开会员数据缺口
- **可调用**：花厅坊无成熟会员体系 → 不做顾客 RFM，改做**小类 RFM**：只需 POS 小票 + 小类明细即可给每个小类打 F（客流/小票数）、M（销售额）两维分级，识别"促销过度类别""重要价值类别"，接品类结构调整。
- **失效条件**：① **R 维（最近被购买时间）对高频生鲜几乎无区分度**——生鲜小类天天动销、R≈0-1 天，全部同分 → 生鲜主力上类别 RFM **退化为 FM 两维**（老刘未点破，见张力 T1）；② 若称重生鲜混录、小类编码不规范（社区店常见），F/M 本身失真，需先做小类编码规范再谈 RFM。R 维只对中低频标品/百货小类有意义。

### 命题 M7-1：卖场转化率对社区生鲜店是"低优先级参考指标"，不投人工专测
- **可调用**：诊断花厅坊时**知道有转化率这个指标即可，不建议投人工/装计数器专门测**——社区生鲜是目的性高频购买场，进店即买率天然高（≈90%+），转化率区分度低、测量成本高、ROI 低。要看"经营好坏"优先用小票数（客流）趋势 + 客单，而非转化率。
- **失效条件**：老刘自承"国内卖场很少关注或没有这个指标"、90% 基准是**大卖场语境**；若花厅坊是大店型/带强逛店属性（生活广场内），转化率才回升为有效指标。

### 命题 M7-2：EOQ / 安全库存三线是"思维工具"，社区店落地=经验估算 + Excel，不做全套建模
- **可调用**：借 EOQ"批量增大摊薄单位成本 vs 抬高库存持有成本，找平衡点"的思维，对**批量单品不多**的社区店用简单 Excel 表估最佳订货批量即可；管道库存用极简公式（提前期×日均需求）估到货前需留量。
- **失效条件**：EOQ 需要订货成本 S、库存持有成本 i、提前期偏差统计——社区店这些参数多为拍脑袋，**全套 EOQ 建模 ROI 为负**（老刘自评"很少用于建模"）；生鲜提前期短、需求波动大，安全库存趋近于当日周转量，三线库存线的"再订购点/最大库存"对当日清零的生鲜意义弱（与 KB-KPI 命题6 的"超短保安全库存趋近0"一致）。

## 5. KB 回填清单（列清单 → 六哥确认 → 再动主体，档位 C）
