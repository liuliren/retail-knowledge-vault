# 零售老刘体系吸收 · 收尾簇（绩效/数据建模/卖点买点）

> 老刘深读工程**收尾批**。覆盖绩效经营(07)/数据建模(08)/商品组合卖点买点。/ingest 深读·转述·从紧。

## 一、核心命题（转述)
| 篇 | 命题 |
|----|------|
| 61 **卖点与买点** | **错把卖点(商家说的好)当买点(顾客真要的)是误区**;"先需求后选品"的逻辑依据 |
| 08/09 绩效建模 | **后台系统主管"管理数据"非"分析数据";绩效分析应系统外搭建**(更灵活);建模维度=商品/部门/人员 + 销售/毛利度量 |
| (数据建模) | 行为→可记录维度 + 可量化度量;指标=分析的"参照物" |

## 二、最大收获
1. **卖点≠买点 = 类名失真型的同源**:两者都是"**商家视角 ≠ 顾客视角**"的失误。
   - 卖点≠买点 → 选品从买点(需求)出发,先需求后选品(接 E2 需求侧 + [[品类管理]] 选品)。
   - 类名失真型(论类要清晰)→ POP 用顾客类名。
   - **可合并为一条元原则:凡"商家视角替代顾客视角"= 失真根**(诊断时统一审视)。
2. **绩效建模"系统外搭建" = 印证我 Data Adapter 架构**:老刘"后台管数据/分析系统外"= 我的"ERP raw → adapter → 分析层"(raw 系统只管数据,分析在 adapter 之上)。独立同构。

## 三、回填（So-what·留签字批次)
1. **元原则「商家视角≠顾客视角=失真根」**:统辖 类名失真型 + 卖点≠买点,写进失真库总则。
2. **选品方向**:[[品类管理]] 选品强调"先需求(买点)后选品",非"先有货(卖点)再卖"。
