# 零售老刘体系二遍吸收 G 陈列货位与空间系数簇 v0.1

> 象限 IV 养料 → 喂 M4 骨架。对外引用须标来源。

## 本簇必答(用户指定)
| 问 | 答(精读结论)|
|---|---|
| **货位配置/陈列系数/空间系数三者关系?** | 三层递进:**货位配置**=哪个货位放哪些SKU、放几个(销售占比×毛利占比分配货位数+单品数);**空间系数**=每个货位/每层的固有好坏权重(不变,黄金位高);**陈列系数**=空间系数×时间系数(随调位/时段变)。配置是"放什么",系数是"这个位值多少"。 |
| **黄金位×ABC错配是否加空间权重?** | 应该。现 M-DEC-011 只用 position_grade(4级)定性判黄金位;可加 `space_score`=空间系数(由 等级×可见性×客流量 量化),把"黄金位喂C/D"从定性升级为"高空间系数货位却低ABC/低坪效"的连续量化错配。 |
| **表E 是否需新增计算列?** | 建议新增 1 计算列 `space_score`(空间系数,派生自字段11/12/13),不新增原始字段。其余(坪效/位置等级)已有。 |
| **M-DEC-011 是否需更清晰错配分型?** | 需要。由现 3 型细化为带空间权重的分型(见 §7):①高系数喂低ABC ②高ABC窝低系数 ③高系数低坪效(选品问题)④货位内层位错配(占比高却非黄金层)。 |

## 2. 原文核心命题
- **人货场关联**:只分析商品本身(销量/毛利)而忽略"场"(货位)和"人"(顾客)是严重错误;同样毛利,占黄金位 vs 角落不一样。
- **空间系数**:把"黄金位"从口头说法变成可计算权重;每货位/每层赋系数。
- **空间系数 + 时间系数 = 陈列系数**:空间不变、时间随调位/时段变。
- **货位配置**=销售占比×毛利占比(交叉比)分配货位数与单品数。
- **陈列规划5要素**(时间/商品/空间/陈列手法/顾客,林光涛《陈列规划》)。
- **货位单品状态**:有销无库/A类无货/已选无库/有库未陈/畅销缺货/此货暂缺。

## 3. 关键公式/计算口径(每条带来源)
| 公式/口径 | 表达式 | 来源 |
|---|---|---|
| 货位数/单品数分配 | **销售占比 × 毛利占比**(交叉比)| 04 |
| 货号在货位内占比 | 该货号销额 / 该货位合计销额(非全店)| 06 |
| 陈列系数 | **空间系数(不变) × 时间系数(变)** | 06 |
| 空间系数赋值 | 各货位/各层手动或建模赋权(黄金位高)| 06 |
| 陈列调整触发 | 货号销额占比高却在非黄金层 → 上移黄金位 | 06 |
| 排面分配 | 销售占比/毛利占比大的小类/SKU → 更多排面 | 15 |

## 4. 业务判断链
```
货位 × 商品(join)
├ 货号在货位内占比高 但 在非黄金层(如第2层) → 上移黄金位中间(原文案例:19.3%占比→应移第5层)
├ 高空间系数货位 放 低ABC/低销 SKU → 错配,换A类
├ 高ABC SKU 窝在 低空间系数货位 → 上移
└ 高空间系数货位 坪效却低 → 该位选品有问题
配置:货位数/单品数 = f(销售占比×毛利占比)
```

## 5. 反常识点
- 同样销量毛利,**占大地堆+2组货架 vs 角落随便摆,价值完全不同**(未来潜力也不同)。
- **整体平均分摊成本/毛利是错的**:每件商品付出的场成本不同。
- "黄金位"人人会说,但**几乎没人精细量化**;空间系数就是把它算出来。
- 货号占比要算**货位内占比**,不是全店占比(粒度陷阱)。

## 6. 对花厅坊的可用部分
- 表E v1.0 已有 位置等级(4级)/可见性/客流量/坪效 → 可立即合成 `space_score` 空间系数。
- M-DEC-011 黄金位×ABC错配:用空间系数把定性"黄金位"升级为连续量化。
- 休食端架/堆头(GP/DH/END):用"货位内占比"+空间系数判断该位 SKU 是否值这个位。

## 7. 对 M-DEC 的增强点(M-DEC-011)
| 错配分型(二遍细化)| 判据 |
|---|---|
| ① 高系数喂低ABC | space_score 高 且 该位 abcd∈{C,D} → 换A类 |
| ② 高ABC窝低系数 | abcd=A 且 space_score 低 → 上移 |
| ③ 高系数低坪效 | space_score 高 且 货位坪效<均值 → 选品复查 |
| ④ 层位错配 | 货号货位内占比高 但 非黄金层(L2)→ 层内上移 |
> **不改 M-DEC-011 临界值**,仅把"黄金位"从 4 级定性升级为 `space_score` 连续权重 + 增 ④层位错配型。

## 8. 对 POS 字段回填清单的影响
- **无需新增 POS 原始字段**。`space_score` 是表E计算列(派生自等级/可见性/客流量),不依赖 POS。
- 清单中 011 部分已含 `pos_abc_mismatch`;建议其取值细化为 §7 四型 → 仅口径,不增字段。

## 9. 可转成 Excel 工具的部分
- **花厅坊空间系数表**(表E 加 `space_score` 列):等级×可见性×客流量 → 货位权重。
- **黄金位×ABC错配扫描表**:表E×表B join,按 §7 四型亮灯出换位清单。
- **货位内占比×层位**视图:揪"占比高却非黄金层"的 SKU。

## 10. 与现有科学零售体系的映射
M4 陈列货位簇 → 运营力·陈列 + 数据底座(空间/时间系数)。M4 已有"货位空间系数(量化黄金位)/人货场/假A类",本簇补足其计算口径(配置=销售×毛利占比、货位内占比、空间×时间=陈列系数)。

## 11. 与既有 M-DEC/标准/表格的冲突风险
- ⚠️ **space_score vs 表E v1.0 position_grade(4级)**:space_score 是连续量化、position_grade 是 4 级定性,**互补**(连续值可映射回4级)。处理:新增 space_score 为**派生计算列**,不改 position_grade 定义。→ M4 卡 + (未来)表E 标注。
- 与 M-DEC-006(落位SOP)不冲突:006管流程,011管诊断(已在 011 §7 明示)。
