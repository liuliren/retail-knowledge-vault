## 2. C2a 原文核心命题
- **"既要类宽、又要品窄、还要落地"**:横向类目宽(覆盖多需求)、纵向SKU浅(单类1-3品)。
- **商品结构(类)比选品更前更重要**:顾客"怕买不到 > 不怕没得挑";"有没有"远比"有多少"重要。
- **三级语义**:宽类(Category 需求骨架)→ **Item(中间类/顾客心智的类)**→ SKU(实物唯一单元);**没有 Item 中间类做不成宽类窄品**。
- **基于一个个"1"**:1需求=1小品类=1货位BAY=1-3 SKU;不能批量操作。
- **温柔一刀 vs 狠心一刀**:对宽类(类)温柔(宁缺毋滥地补全),对窄品(SKU)狠心(同质坚决砍)。

## 3. "宽类窄品"的定义
> [推断·属老刘原义,非六哥定义]:在覆盖顾客多任务/多场景的前提(宽类)下,每个需求类只配置少数(1-3个)精选SKU(窄品),并落到陈列场景。
- 本质是 **"类+品"模型**:先有类(需求),再配品(SKU);类在前、品在后,绝不倒置。
- "宽类+窄品"实为"宽类 + 窄 + 品":**窄=对品数量的限制词**(尽量少),不是字面的"窄品"。

## 4. "宽类"到底宽在哪里
- 宽在 **顾客任务/需求场景的覆盖广度**(饮料:解渴/补能量/正餐佐餐/健康),不是大类越多。
- 宽类的"类"≠结构表小类,通常是**小类与SKU之间的"中间类(需求点/Item)"**;类数≈小类数 **5-8倍**≈月SKU动销数(经验值)。
- 判断维度:顾客任务数 / 场景数 / 价格带是否必要覆盖 / 规格口味品牌带是否必须 / 社区刚需低频任务 / 当前任务缺口。

## 5. "窄品"到底窄在哪里
- 窄在 **单个需求类内的 SKU 数**(同质坚决去重),不是全店砍品。
- 同类 SKU **同性相斥**(5种矿泉水顾客只买1种,其余4种滞销);类间 **同性相吸**(没农夫买怡宝)→ 窄品减内耗、扩客群。
- 判断维度:同质SKU过多 / 价格带重复 / 规格口味品牌重复 / PSD过低 / ABC尾部过厚 / 陈列承载不了 / 补货复杂。

## 6. 与 C1 SKU 数约束的关系
```
C1:判小类 超配/欠配(当前SKU vs 合理SKU区间)
C2a:判方向 —— 扩类 还是 窄品?
├ 超配 → 窄品(单类去冗余),但可能是"从重复段转向缺口类"(非全局砍)
├ 欠配 → 扩类/补关键(补任务缺口/关键价格带,非补很多SKU)
└ 目标 = 结构重分配(总SKU可能不降,质变),不是减总SKU
```

## 7. 与 B 簇 ABC/老品淘汰的关系
- ABC 的 C/D **只是窄品候选**,不是淘汰判决;
- 真淘汰前过 **B簇六桶防误杀**(缺货/陈列差/价格带错位/衰退/系统假象);
- **低频刚需不能因C类就砍**(原文:抗敏牙膏/老人儿童特定规格——代表一个需求类,必留);
- 生命周期衰退SKU(可汰)与 任务占位SKU(必留) **分开**;
- 窄品动作常是 **替换(好品替差品)非净减**。

## 8. 与 M3 价格带的关系
- 价格带 **漏档** → 不能窄掉关键价格段(交 M-DEC-010);
- 价格带 **过密** → 优先窄重复价格段(窄品先砍这里);
- 老刘原则:**一个类最多覆盖 2 个价格带**(高端+大众),别搞 ≥3;
- 价格带从 宽类(设范围)→Item(限制)→SKU(定价)逐层落地。

## 9. 与 M4 陈列货位的关系
- 宽类窄品**最终落在陈列**;人货场货位 4 步:落位(平面布局)→ 黄金视线+面位(ABC给A类多面)→ 品类关联(啤酒+花生)→ 动态调整(好品替差品/整类没人买撤整类)。
- 货位定位:入口端架(促销/季节)/黄金视线(高频冲动)/中段(目的性如水)/次通道(低频必需)/收银端架(便利小包高毛利)。
- 黄金位错配 → 先查陈列(M-DEC-011),**不直接砍SKU**;货架容量不足 → 窄品受陈列约束;**目标性/引流类不能过度窄品**。

## 10. 哪些小类适合宽类窄品
- 同质化高、SKU冗余、价格带重复的小类(如碳酸饮料案例60%同质)。
- 便利性/常规性角色、顾客认牌不认款的小类。
- 花厅坊休食候选(待实证):T01薯片/T03饼干(品牌口味多易同质)、T06糖果、T10辣条。

## 11. 哪些小类不适合简单宽类窄品
- **目标性/形象/引流**角色小类(过度窄品损商品力):花厅坊 T05果干蜜饯(大湾区差异化形象)、T08果冻布丁(儿童目标性)。
- 含**低频刚需**的小类(老人/儿童特定规格)。
- 价格带漏档/任务缺口的小类(该补不该窄)。
- 需求宽度本身就窄的小类(已是窄类,再窄=缺口)。

**动作规则**:
```
超配+同质过密 → 窄品(狠心砍同质),过六桶防误杀

欠配+价格带漏档 → 补齐关键段(非窄)
角色=目标性/形象/引流(T05/T08) → 保持深度,不过度窄
含低频刚需 → 必留任务SKU(替换不净减)
黄金位错配 → 先查陈列(M-DEC-011)再定
```

## 15. 对 M2_选品配置簇.md 的增量回填(§8 二遍补强)
- **宽类窄品 ≠ 简单砍SKU**——是结构重分配。
- **宽类=顾客任务覆盖,窄品=单任务内去冗余**;类在前品在后。
- **必须承接 C1 SKU数约束**(先判超配/欠配)。
- **必须过 B簇 六桶防误杀**(低频刚需/任务占位SKU 必留)。
- **必须与 M3 价格带、M4 陈列容量联动**(漏档不窄/过密先窄/目标性不过度窄)。
- **前置缺口**:需先建 Item/中间类(需求点)层(花厅坊目前只有L3小类)。
