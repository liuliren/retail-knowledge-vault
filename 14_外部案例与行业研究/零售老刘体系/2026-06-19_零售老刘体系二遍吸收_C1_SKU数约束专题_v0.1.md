## 2. C1 原文核心命题
- **"一切零售问题就是SKU问题"**(38):进销存、财务、绩效最终都落到 SKU 数量增减。
- **合理SKU数 = 多维约束下的最优解,非固定值**(31/32):不是拍脑袋的静态数,是刚性+柔性约束下持续校准的动态平衡点。
- **容器语义**(31/36):门店=容器,先定容量(约束)再装SKU;选品三部曲=需求确定→品类定位→SKU选择。
- **不盲目求多、不片面求少**(32/35):SKU多≠商品力强(稀释效率);过度精简→流失顾客;用约束求解黄金区间。
- **一进一出 / 动态优化**(31/37):新品占比上限 + 老品强制淘汰比例,保 SKU 总数良性循环,防"无效熵增"。

## 3. SKU 数为什么不能拍脑袋定
- 单看"销售额排序"定 SKU 数 = 与多重约束不符(32);
- SKU 总量"过大"本身就预示系统性管理问题(缺一进一出机制)(31);
- "今天方便面卖500包 vs 昨天420包"无意义——完成销售的 SKU 数(动销率)不同(36);
- SKU 由"生/养/管/算"四端协同决定,非任何单一部门意志(33)。

## 4. SKU 数与品类角色的关系
- **品类深度系数**:目标性品类取高值 **0.8-1.5**,便利性品类取低值 **0.3-0.8**(34,经验值待校准)。
- **品类上下限**(35):目标性品类必须足够深度(原文例 ≥15 SKU);便利性品类 2-3 SKU 即可,过深是浪费。
- **需求覆盖法**(34):总SKU目标按 品类权重(销售占比×深度系数)分解到各小类。
- 角色决定深度方向:浅而宽(偶尔需求)vs 深而窄(专业需求)(34)。

## 5. SKU 数与 小类销售/毛利/周转/陈列面积 的关系
| 维度 | 关系/公式 | 来源 |
|---|---|---|
| 销售 | 品类SKU数 = 总销售 / 品效;品效=总销售/SKU数 | 34 |
| 毛利 | GMROI=毛利率×周转率,跨部门统一评 SKU 健康;每米货架毛利额排序 | 33/35 |
| 周转 | SKU越多单个周转越慢;经济承载=(月销×周转天)/(30×单品成本)| 34 |
| 陈列面积 | 物理上限=(陈列米数×层数×陈列系数)/单品面宽;**陈列面积是刚性天花板** | 34 |

## 6. SKU 数与 ABC/售罄率/老品淘汰 的衔接(承接 B 簇)
```
B簇:ABC/售罄率 → "哪些SKU表现差"(单品级)
C1:SKU数约束 → "这个小类该容纳多少SKU"(小类级)
衔接逻辑:
① 先判小类是否超配(当前SKU数 vs 约束算出的合理SKU数)
② 小类不超配 → C/D类SKU 只是结构替换(汰旧引新,净数不减)
③ 小类超配 → 才进入瘦身:按ABC+售罄率(46.7%)排淘汰优先级
④ 淘汰前仍过 M-DEC-009 六桶防误杀(缺货/陈列/价格带/衰退/系统假象)
```
> **关键**:不能"见C/D就砍";砍多少由小类超配程度决定,砍谁由 B 簇 ABC/售罄率决定。

## 7. SKU 数与供应链约束的关系
- **实际可用SKU = 理论规划SKU × 仓库现货率 × 供应商履约率 × 经营良性率**(35)。
- **宽品低库 vs 窄品高库**(34):补货快(本地日配/生鲜)→低库存深度+宽SKU;补货慢(进口/酒)→高库存+窄SKU。
- **MOQ 约束**(35):整箱起订消化不了(日均2-3瓶 vs 24瓶/箱)→ 不引进或替换。
- SKU 增减 = 全供应链单据流转(37/38):增1个SKU≈7类单据,减1个≈6类;复杂度成本随SKU指数上升。

## 8. SKU 数过多的风险(7 型)
| 型 | 表现 |
|---|---|
| ① 小类内过密 | 单小类SKU畸高,品效低于阈值 |
| ② 同质品牌过多 | 多品牌同质,内耗 |
| ③ 价格带重复 | 同价格段堆SKU,价格形象模糊 |
| ④ 陈列承载不了 | 超物理上限→拥堵,顾客难识别 |
| ⑤ 低效占高价值位 | C/D类占黄金位(交 M-DEC-011)|
| ⑥ 周转被拖慢 | SKU越多单个周转越慢,资金稀释 |
| ⑦ 补货复杂度高 | 单据激增/拣货效率降/盘点难 |

## 9. SKU 数过少的风险(6 型)
| 型 | 表现 |
|---|---|
| ① 顾客任务缺口 | 某购物任务无对应SKU |
| ② 价格带漏档 | 核心价格段缺失(交 M-DEC-010)|
| ③ 品牌带缺失 | 顾客认牌缺主力 |
| ④ 核心规格缺失 | 如大包装/家庭装缺 |
| ⑤ 季节场景未覆盖 | 年货/节令缺 |
| ⑥ 过度瘦身致损失 | 砍过头→流失顾客/丢机会 |
> 老刘提醒(34):近30年罕见正常营运下"SKU过少",实际几乎都是过多;但社区店瘦身仍须防①-⑥。

## 10. 关键公式 / 计算口径(每条带来源)

**动作决策规则(承接 D/E 必答)**:
```
超配 + 价格带过密 → 优先砍重复价格段(交M-DEC-010定段)

角色=便利性 → 可瘦到2-3 SKU
```

## 13. 对 M2_选品配置簇.md 的增量回填(§7 二遍补强)
- **SKU数是经营约束,不是陈列喜好**——由 7 约束综合决定。
- **SKU数 = 角色×需求宽度×销售毛利×库存周转×陈列容量 共同决定**(非单指标)。
- **SKU瘦身必须承接 ABC/售罄率/防误杀**(先判小类超配,再排淘汰优先级,再过六桶)。
- **SKU减少 ≠ 商品力提升**——错误瘦身造成顾客任务缺口/漏档。
- **SKU数判断必须与价格带(M3)、陈列空间(M4)联动**。
