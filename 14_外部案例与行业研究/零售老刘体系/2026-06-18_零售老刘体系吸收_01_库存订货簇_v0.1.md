## 1. 五大核心思想(可直接用于客户对话 / 培训)

| # | 思想 | 一句话 | 反常识点 |
|---|---|---|---|
| 1 | **高库存是果不是因** | 高库存本质是供应链效率/平衡问题;大采购的前提是高周转(高流通性),不是反过来 | 大采购压成本 = "果"不是"因";局部最优 ≠ 全局最优 |
| 2 | **追加式订货陷阱** | 按销量补货 = "越畅销越缺货,越滞销一大堆"。畅销品当天卖光后整周断货 → 销量被低估(**假需求**);滞销品没卖完 → 销量"真实"。继续多订滞销少订畅销 = **双倍损失** | 你眼里的 A 类/畅销品 ≠ 顾客眼里的畅销品 |
| 3 | **闭环需求计划** | 以销定采 → 以采定销。开口式需求计划 = 销售永远不对库存负责,采购永远背锅 | 营采合并解决不了;根子是流程开口 |
| 4 | **需求预测 ≠ 安全库存** | 两者分开处理:预测对付平均需求,安全库存对付不确定性。**不可把安全库存分摊进预测**,不可一刀切统一系数 | 一刀切 ×1.65 系数 = 越畅销越缺货的元凶 |
| 5 | **判断畅销必须引时间维度** | 越短时间达到 50% 售罄 = 越畅销。售罄率 100% ≠ 完美,可能是订货不足缺货 | 按销量倒序末位淘汰 = 极端错误 |

## 2. 关键公式卡(花厅坊可直接套用)

| 公式 | 表达式 | 用途 |
|---|---|---|
| **标准订货公式** | Q = d×(T+L) + Z·σ − I | d=日均销量DMS / T+L=采购周期 / Z=服务水平系数 / σ=日销量标准差 / I=现有库存 |
| **合理库存** | 库存 = 日均销量 × 1.5 | 单品/小类/中类/大类逐层算 |
| **内仓零库存配置** | 陈列量 = 1.5 × 订货单位 | 例:订货单位12 → 陈列18;首次进24(18上架+6内仓);卖到剩6再进12=18上架,内仓归零 |
| **有货率→Z值** | NORMSINV;95%→1.64 / 90%→1.28 / 99%→2.33 | 服务水平转安全库存系数 |
| **安全库存→有货率** | +1σ→84.1% / +2σ→97.7% / +3σ→99.9% | 投资回报递减,别盲目追100% |
| **库存现金系数** | IDSM = AVG(每天库存) / AVG(每天成本) | 真实周转,破"期初期末平均"造假 |
| **周转天数硬约束** | 周转天数 < 月结账期(30天) | 否则现金先付货没卖 = 现金流出血 |
| **售罄率盈亏平衡** | X = 毛利率 / (1−费用率) | 例:35%/(1−15%)=46.7% |
| **边际周转率** | 业务增15% → 库存增 <15% | 规模效益考核 |
| **损耗率合理值** | 0.4% | 盘亏红线 |

## 3. 花厅坊落地映射(6/22 + 后续)[推断]

| 花厅坊现存问题 | 用哪个武器 | 落地动作 |
|---|---|---|
| **采购与动销矛盾 / 货架缺货** | 思想2+3 + 订货公式 | 还原真实 DMS(不被缺货"假需求"骗)+ 推闭环需求计划(以销定采)|
| **采购订单逻辑(六哥6/10原话)** | 订货公式 + 思想4 | "动销数量跟进算法" = Q=d×(T+L)+Z·σ−I;安全库存分 ABC 不一刀切 |
| **老品淘汰清单** | 思想5 售罄率 + 双ABC | 售罄率达50%天数排名 + "去年A今年C"双ABC |
| **品牌单品优化** | 价格带建模 + ABC | 休食核心小类价格带分布,找9.9过密/缺段;A类17%产60%抓重点 |
| **现金流 P3** | IDSM + 周转<账期 | 算花厅坊真实周转天数 vs 供应商账期,挤现金 |
| **内仓/陈列量** | 内仓零库存配置 | 陈列量=1.5×订货单位,定各 SKU 满陈列参数 |

## 4. 映射科学零售骨架(养料 → 内核)

| 零售老刘工具 | 喂给六哥骨架的哪个零件 |
|---|---|
| 订货公式 / 安全库存 / DMS / 内仓零库存 | **经营控制链 → 数量管理**(SKU/补货/缺货数量动作)|
| 周转率 / 售罄率 / 动销率 / 边际周转率 | **商品力 → 库存周转**(算得准)|
| 价格带建模 | **品类管理 → 价格带管理** |
| IDSM / 周转天数<账期 / 损耗0.4% | **金额管理 → 现金流/损耗** |
| 闭环需求计划(以销定采→以采定销) | **组织力 → 流程闭环**(销售对库存负责)|
| 双ABC / 综合ABC / 层级ABC | **商品力 → ABCD分类**(升级为多维/层级/组合)|

> **转化正道**:每把一个工具用到花厅坊真实场景跑通 → 产 1 条 **M-DEC**(带花厅坊临界值)。
> 例:零售老刘讲"售罄率46.7%盈亏平衡"=通论;六哥讲"**花厅坊休食方便速食售罄率临界X%触发淘汰**"= IP。
