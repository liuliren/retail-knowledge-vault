# 零售老刘全量深读执行标准:三层阅读法、五张表与批次节奏 v0.1

> ⚠️ 口径:"零售老流"=零售老刘/老木匠(同公众号),不新建实体。

## 1. 文件定位
- 本文件是《[[2026-06-19_零售老刘全量深读工程启动包_v0.1]]》的**执行标准补丁**:回答"315 篇 MD 到底**怎么读才深、怎么读才能串联、怎么读才能进晟果科学零售体系**"。
- **启动包**解决"读什么、按什么范围读";**本文件**解决"怎么读、怎么建卡、怎么串联、怎么吸收"。
- 本文件**不是文章总结、不是 M-DEC、不替代单篇深读卡片**。

## 2. 总原则:不按文章顺序读,按零售经营问题链读
> 315 篇**不能按文件顺序逐篇总结**,要围绕**经营问题链**读。

**问题链**:库存为什么不准? → 销量为什么≠需求? → SKU 为什么过多? → 老品为什么不能一刀切淘汰? → 价格带为什么断档/过密? → 陈列为什么让好商品卖不动? → 顾客需求如何变 Item/品类结构? → 这些判断如何变 FC/M-DEC? → 这些规则如何在花厅坊验证?

**每篇必须回答**:① 解决哪个经营问题 ② 对应 M1-M8 哪个模块 ③ 能否转 字段/公式/判断链/FC/M-DEC/工具表 ④ 在花厅坊/未来客户有无验证机会。

## 3. 三层阅读法
### 第一层:地图阅读(全 315 篇)
目标:不深读全部,把 315 篇放进知识地图。每篇只做 **6 判断**:① 主题 ② 解决问题 ③ 对应 M1-M8 ④ 是否可转 FC/M-DEC ⑤ 是否关花厅坊/客户 ⑥ 优先级。
> 输出:315 篇全量知识地图(=台账 + 本标准的状态口径)。
> 纪律:skimmed ≠ deep_read;extracted ≠ integrated;名相似不合并正文;保留作者/来源/原始路径。

### 第二层:簇内深读(一组一起读,成逻辑链)
目标:不单篇孤立总结,**一组文章一起读,形成簇内逻辑链**。
- **库存簇**:销量≠需求 → 缺货扭曲销量 → 还原真实需求 → 安全库存 → 售罄速度 → ABC边界 → 老品汰换防误杀 → **M-DEC-009 / FC06 / FC10**
- **SKU/选品簇**:小类角色 → 顾客任务 → SKU数约束 → 品效拐点 → 宽类窄品 → 重复SKU压缩 → 低频刚需保护 → **FC02 / FC07 / FC01**
- **价格带簇**:价格段划分 → SKU占比 → 销额占比 → 毛利占比 → 过密 → 漏档 → 断档 → **M-DEC-010 / FC03**
- **陈列簇**:货位配置 → 空间系数 → 层板/排面 → 黄金位 → ABC → 商品差vs陈列差 → **M-DEC-011 / FC05**
> 输出:每簇一份"簇内知识关系图"(表3)。

### 第三层:体系吸收(每读完一簇必出 5 件)
① 单篇深读卡片 ② 簇吸收笔记 ③ M1-M8 更新建议 ④ FC/M-DEC 更新建议 ⑤ 花厅坊/客户案例验证点。
> **铁律**:只做文章总结 = 仍是外部资料;**只有进 M1-M8/FC/M-DEC/花厅坊案例/工具字段,才算变成晟果知识资产。**

## 5. 批次阅读顺序
| Batch | 范围 | 目标 | 产出 |
|---|---|---|---|
| **1** | 时间盒+库存+ABC | 判断纪律/防误杀/库存口径/售罄速度/ABC边界 | 库存判断链·老品防误杀链·售罄率链·ABC边界·**M-DEC-009增强**·FC06/FC10 |
| **2** | SKU数+选品+宽类窄品 | 一小类该多少SKU/怎么砍/怎么补 | SKU数约束模型·品效拐点·宽类窄品操作链·**FC02/FC07**·T03/T01对比 |
| **3** | 价格带+毛利+促销 | 价格带从感觉变结构诊断 | 过密/漏档/断档判断·毛利贡献·负毛利识别·**M-DEC-010增强**·FC09 |
| **4** | 陈列+货位+棚格 | 区分商品差vs陈列差 | 陈列系数·黄金位×ABC·空间效率·货位配置·**M-DEC-011补强**·表E字段/拍照标准 |
| **5** | 顾客需求+品类角色+购物任务 | 把需求点/Item层做扎实 | 顾客任务图谱·Item层定义·低销必留判断·**FC01补强**·T03需求点确认标准 |
| **6** | 绩效+预算+数据建模 | 方法论变管理指标/经营看板 | 品类绩效·非财务预算·人效/坪效/货效·数据字段体系·**M7/M8补强** |
| **7** | 案例与综合 | 沉淀话术/交付/案例表达 | 客户沟通话术·案例库·销售材料·诊断报告表达 |

## 6. 每批五步闭环(每批 ≤12 篇)
1.锁定本批文件 → 2.逐篇建深读卡片 → 3.生成簇内知识关系图 → 4.输出 M1-M8/FC 更新建议 → 5.挂接花厅坊/客户案例验证点。
> 要求:**不读完再整理**(边读边建卡、边读边进体系);**一批 ≤12 篇**;**不允许只做泛泛总结**。

## 7. 每篇必答 8 问
1.解决哪个零售问题? 2.反对什么常见错误? 3.给出什么判断链? 4.有无 公式/字段/阈值/时间盒? 5.对应 M1-M8 哪个? 6.能否转 FC/M-DEC? 7.花厅坊有无验证机会? 8.有什么边界、不能乱用在哪?

## 8. 知识点节点化规则
> 每篇不能只摘要,必须拆出**知识点节点**:

| 字段 | 内容 |
|---|---|
| 知识点名称 | |
| 来源文章 | |
| 对应模块 | |
| 相关字段 | |
| 影响规则 | |
| 花厅坊案例 | |
| 风险边界 | |

**示例(真实)**:
| 字段 | 内容 |
|---|---|
| 知识点名称 | 销量 ≠ 需求 |
| 来源文章 | 可怕的订货量 / 安全库存建模 / ABC 相关 |
| 对应模块 | M1 库存订货、M2 选品配置 |
| 相关字段 | sales_qty / stock_qty / stockout_days / days_no_sales |
| 影响规则 | M-DEC-009 / FC04 |
| 花厅坊案例 | T03 14 个>60天无销售 → 4 有货 / 10 库存=0 |
| 风险边界 | 缺库存时不能判滞销;库存口径过时仅近似 |
