# 逐字精读卡片集 · 批55 数量为先·单品数状态·库存周转率两组度量

> 📌 **本批 = 分析建模19-21**(2022年11月)。库存周转率建模实现(19-20)+奢侈的货位棚格(21)。**金矿:数量为先(这是"数量管理"方法论页的根命题)+各类单品数状态**。
> ⚠️ 下一批续读 分析建模22-27(货位棚格2/货位棚格建模/售罄率)等。

## 1. 分析建模(19) 库存周转率建模(1) ⭐⭐
- **核心论点**:时间属性=库存建模第一原则;**流程闭环=时期闭环**(报货→要货→补货→订货→验货→进库串时期·时期不闭环则流程总觉有问题)。事实表=出入库明细表(货号/时期/出入库类型/成本价)。
  - **DAX**:`IR_库存系数=库存成本÷销售`;`库存周转天数=DIVIDE(时期,IR)`。
  - **两组度量(各有用途)**:① **库存系数IR+周转天数**(实际运用建议用这组)② **年化库存周转率(次数)+年平均持有天数**(=成本率×365÷天数)。区分:年化周转率=一年库存转化销售的次数(如9.14次/40天周转一次,不是转40次);库存系数=销售与库存的关系(每月库存≤销售多少倍·如需1.76倍库存支撑)。
- **关键词**:时间属性=第一原则 / 流程闭环=时期闭环 / IR=库存成本÷销售 / 两组度量(系数用这组·年化次数) / 9.14次vs40天

## 2. 分析建模(20) 库存周转率建模(2) ⭐⭐
- **核心论点**:5个建模问题——① 销售表无成本价→建成本价表(时期+条码关联·移动平均成本不一定真实)② 透视必须有时期维度筛选(否则结果错)③ 两组周转天数有差异不影响判断(作系数非主度量)④ **周转天数≠订货周期**(仅参考·周转率常用于粗略查询很少精细化);更好的订货周期=按供应商/采购区域/类别属性设·体现在商品信息表**【采购周期】列**;缺采购周期→订货建议表/基于日均销量的安全库存全失效 ⑤ **库存及绝大多数业务,首先应关注的是数量**。
- **关键词**:成本价表(移动平均≠真实成本) / 周转天数≠订货周期 / 采购周期列缺失→安全库存失效 / 以销定采 / 首先关注数量

## 3. 分析建模(21) 奢侈的货位与棚格图表(1)·数量为先 ⭐⭐⭐
- **核心论点(数量管理的根命题)**:**被忽略的最重要度量=单品数量(SKU)**。
  - **各类单品数状态**:总单品数 / 库位单品数(一SKU一家) / 货位单品数(卖场陈列) / **满陈列单品数** / 在销单品数 / 选品配置数 / 当前库存数 / **此货暂缺数**(可订可销需陈列但没陈列出来·≠无库存)。
  - **⭐7大异常商品**(卖场货位形态):负库存 / 高库存 / 负毛利 / 畅缺货 / 零现象(销售/库存/毛利/进货为零) / 有销未订 / 终止商品。+单品组合形态(有陈列无库存/有库存无销售/有销无订货/此货暂缺/临过期/满陈列量/再订货点)+系统库位形态(新品→试销→评估→正常品→禁止下单→淘汰→删除)。
  - **⭐核心:最原始的衡量度量是数量,然后才是金额/毛利**。商品经营两大错误:① 只重金额毛利衍生度量、不重数量变化(先后顺序错乱→流程/绩效/决策失误且不易发现)② **本末倒置**(陈列18原则里几乎无数量·只"排面丰满"含糊·丰满到什么程度?)。
  - **⭐陈列最简定义=解决商品在卖场的数量和位置**;**过度陈列**=重位置忽略数量。"陈列漂亮不是第一位,第一位是顾客想要的商品不缺货,首先研究数量"。
  - **⭐商品配置应先数量后销售**:先商圈调研/货位布局/商品结构/定位→预估结构单品数+各类单品状态数量→再决定各维度(供应商/品牌/规格);而非先价格带/规格/品牌/销售占比。奢侈的棚格图表定式(想A说B做C成D果E);广州某连锁极致量控(找不到此货暂缺·加货频率时间数量恰到好处)。
- **关键词**:**数量为先(最原始度量·先数量后金额)** / 各类单品数(总/库位/货位/满陈列/在销/此货暂缺) / **7大异常商品** / **陈列=解决数量+位置·过度陈列** / **商品配置先数量后销售**
