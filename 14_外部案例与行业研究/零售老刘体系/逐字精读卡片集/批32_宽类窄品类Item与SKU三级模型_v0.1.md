# 逐字精读卡片集 · 批32 宽类窄品/类Item与SKU三级模型

## 187. 品类建模(203) 供应链SKU ⭐
- **核心论点**:再强化"**一切零售问题=SKU 问题**"(进货点数/卖货扫数/盘点数数/财务摊成本·全绕 SKU;"零售就是跟 SKU 在过日子")。SKU 数变化 ↔ 经营变化(双向):任何折腾(促销/换陈列/引新品)→ 体现为 SKU 数增减;SKU 数一变 → 库存/资金/陈列/人力/供应链全跟着变。**合理 SKU 数(供应链定义)=使各类业务单据流量与复杂度可控、同时满足市场需求、实现供应链整体成本最优的 SKU 水平**。反推法:二八分析(20%SKU贡献80%单据)/边际拐点(SKU vs 订单行数)/仓储效率临界。
- **关键词**:一切零售问题=SKU问题 / SKU数↔经营变化双向 / 合理SKU=单据流量可控+成本最优 / 单据反推

## 188. 品类建模(204) 论宽类窄品 ⭐⭐⭐
- **核心论点**:**宽类窄品=既要品类宽(覆盖多需求)+又要SKU窄(选择快/货架效率)+还要符合业务场景**(横向类宽、纵向品浅)。**底层逻辑=能赚钱**(好处多/省事)。四大好处:**①符合顾客心理("有还是没有">"有多少选择"·怕买不到>怕没得挑→商品结构比选品更前更重要)②提高货架效率(宽类窄品的货架产出>窄类宽品)③降库存风险(同类SKU相斥"九子夺嫡"·跨类相吸)④供应链简化**。落地4步:**①划分"类"颗粒度(小类/子品类/Item·"类"=无限接近SKU的不可再分单位=最高解释权)②做减法(每类留1-3 SKU·按销量/品牌/价格带≤2/特殊功能保留)③做加法(查"类"缺口·建类必备表·宽类是前提)④定期轮换测试**。
- **关键词**:宽类窄品=宽类+窄品+符合场景 / 底层逻辑=能赚钱 / 有没有>有多少·结构>选品 / 同类相斥跨类相吸 / 落地4步(分类/减法/加法/轮换)

## 189. 品类建模(205) 解宽类窄品 ⭐⭐
- **核心论点**:**"类+品"模型**(类=顾客心智需求"解渴/饱腹"/品=SKU"罐装红牛330ml")**·必须先类后品(绝不倒过来·"宽类窄品"=宽类+窄+品)**。**三个基本单元:①宽类窄品(类多品少·主流)②窄类宽品③窄类窄品**(无"宽类宽品"=时代过去·现多数店是"天然宽类宽品"=待优化)。**宽类窄品+宽类=横向跨类组合**(垂直继承 vs 水平对比·关联陈列/场景陈列的逻辑·如"饮用水+苏打水"=解渴场景)。**三级语义层角色:宽类做"宽"(Category·总有一类解决你的问题)→ Item 做"精"(1-2个·避免选择困难)→ SKU 做"准"(规格/包装/口味)**;对应"上帝视角(需求)→生意视角(运营)→后台视角(供应链)"完整闭环。
- **关键词**:类+品模型·先类后品 / 三个基本单元(宽类窄品/窄类宽品/窄类窄品) / 宽类窄品+宽类=跨类组合 / 宽做宽·Item做精·SKU做准

## 190. 品类建模(206) 扩宽类窄品 ⭐⭐
- **核心论点**:**标准宽类结构 = 品类(Category)→ 小品类(Item)→ 商品(SKU)** 三级。衍生结构(部门/课位→大类/区位→SKU)须几次"转化"(中间隔几层转几次·第一层任意高维、第二层必是"小品类"=SKU上一级不可再分类)。**核心执行原则="4 合 1":1个需求 = 1个小品类(Item)= 1个BAY陈1-2类 = 1个小品类配1-3 SKU**——宽类窄品**总是基于一个个"1"操作**(不能批量·像棚格图表每次只做1个BAY)。"只有立起一个个'1',后面的'0'才有意义"。**"宽类"的类≠小类**:小类×5-8倍≈"类"数≈门店月 SKU 动销数;顾客买的是"类"不是 SKU(供应链的类 vs 顾客心中的类两套语义,最理想=供应链类无限适配顾客类)。
- **关键词**:标准宽类结构(Category→Item→SKU) / 4合1单元(1需求/1小品类/1BAY/1-3SKU) / 基于一个个"1"·1立0才有意义 / 宽类≠小类(×5-8倍)

## 191. 品类建模(207) Item与宽类 ⭐⭐⭐
- **核心论点**:**Item 中间类 = 宽类窄品成败的关键**(没有 Item 中间类,执行必陷被动甚至无法实施)。**Item = SKU 前不能再细分的最小"类"(中间类/过渡类)·是顾客心智中的"商品种类"**(西瓜是 Item,1/2 西瓜才是 SKU)。三层属性继承:**宽类定通用属性组(骨架)→ Item 继承+加特有属性 → SKU 再细化(继承链 SKU▶Item▶类)**。**从品名拆类/Item/SKU**:"高露洁 M-08 美白 牙膏 120g"——主语"牙膏"=类;围绕主语的修饰定语(品牌/型号/功能/规格)=Item;定语限定的唯一值组合=SKU。**80% 门店死于/正死于"SKU 熵增"**(SKU 太多的负向)。Item 在门店常被隐藏=零售短板。
- **关键词**:Item中间类=宽类窄品成败关键 / Item=顾客心智的商品种类(西瓜=Item·1/2西瓜=SKU) / 三层属性继承(宽类骨架→Item→SKU) / 80%死于SKU熵增

## 192. 品类建模(208) 解宽类语义 ⭐⭐
- **核心论点**:三级基本语义结构定名:**第一层=宽类(Category)/第二层=单品或品项(Item)/第三层=商品(SKU)**。**"宽类+窄品"实为"宽类+窄+品"**("窄"=尽量少的限制词·非直接的"品"·故窄品≠Item·但 Item 是其桥接中间类)。**基于顾客永远只有"类+品"(无宽窄之分·宽窄是门店搞出来的)**;顾客心中的类=Item(先选 Item 类种再选 SKU 规格)。核心问题分层:宽类="我为什么买/解决什么需求"·Item="要买什么商品"·SKU="选哪个具体实物"。**属性继承核心价值="宽类定骨架,窄品填血肉"**(命名规范/自动/唯一·属性模板化/复用/级联更新·降维护成本+保数据一致)。
- **关键词**:三级=宽类Category/Item品项/SKU商品 / 宽类+窄+品(窄≠品) / 顾客只有类+品 / 宽类定骨架窄品填血肉
