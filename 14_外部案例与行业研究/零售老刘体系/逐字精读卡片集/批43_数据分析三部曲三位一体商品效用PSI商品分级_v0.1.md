# 逐字精读卡片集 · 批43 数据分析三部曲·三位一体·商品效用PSI·商品分级

> 📌 **本批 = 「数据分析架构系列」07-12**。比 01-06 实得多:从 BI 工具(07)→数据分析三部曲本质(08)→实战报表系统(09-10)→三位一体货位管理(11)→**商品好坏论+综合效用PSI+商品分级(12·本批最强)**。多条直接呼应已建的 SKU角色层/Data Adapter/满陈列。
> ⚠️ 序号以系列编号(07-12)为准。本批后续 13-14 收官本系列,再转零散库存建模篇。

## 1. 系列07 BI工具与数据分析危机 ⭐
- **核心论点**:Excel2003 仅 6万行 → Excel2016+Powerpivot 可处理**1亿行**(取决内存);Excel 最权威普遍,2013+ 可导入几乎任何数据源;Power BI=商务智能(BI)。**"确保至少80%的BI需求由业务用户自身实施"**(Forrester)。**数据分析危机**:管理者越爱分析、IT 越不能胜任 → BI 解。"维"(维度)分级(大类/中类/小类/细类/系列),维度组合=千变万化。**后台系统=数据管理,不能代替数据分析**(固化、实用性差)。
- **关键词**:Excel 1亿行 / Power BI=BI / 80%BI需求业务自助 / 数据分析危机 / 后台=管理≠分析

## 2. 系列08 数据分析三部曲=维度×度量×输出 ⭐⭐
- **核心论点**:**Power BI 三步法=数据分析本质**:① 加载数据→数据模型 ② 数据建模 ③ 拖透视/度量输出。**一句话本质:数据分析=从哪个维度(角度)去了解(度量),然后展示(输出)**。
  - **零售常用维度**:时期/门店/仓库/类别/商品/供应商/客户;**常用度量**:销售额/销量/毛利/毛利率/单品数/客单/客流/交叉率/动销率。分析=**维度×度量的组合**(透视表)。例:`6月份(时期)×001店(门店)×休闲食品(类别) → 动销率52%(度量)`。
- **关键词**:数据分析三部曲(加载→建模→输出) / **维度×度量=分析本质** / 7维度+9度量清单 / 透视组合=业务需求

## 3. 系列09 实战分析系统(模版化) ⭐⭐
- **核心论点**:四个商品管理常用分析系统(全做成**模版系统**:更新数据自动出结果·动态可维护):① **商品ABC**(销售占比二八·单品在全场ABC+在自身类别占比)② **商品属性表系统**(日均销量/可销天数/销量→算库存状态/销售状态/**配置调整属性:必需品/必选品/结构品**)③ **商品配置表系统**(自动算每小类建议合理配置数)④ 畅销选品分析。再次强调后台系统=数据管理不能代替数据分析。
- **关键词**:模版系统(更新即出) / 商品ABC / 商品属性表(必需品·必选品·结构品) / 小类建议配置数自动算

## 4. 系列10 数据化管理定义 + 常用模板 ⭐⭐
- **核心论点**:**数据不会骗人,关键是运用和解释**(顾客嘴里说的≠真打算做的→看购物小票验证)。先弄清想从数据得到什么,否则无止境追寻。管理者困境(销售不好只能怪天气客流/忙琐事没空追销售/数据混乱拍脑袋/销售员想当然编故事不分析)。
  - **数据化管理定义**:运用分析工具对**客观、真实**的数据科学分析,并将结果运用到销售/业务流程各环节的管理方法;**三阶段=数据收集整理 → 分析 → 运用**;关键差异:数据化管理会**区分数据真假、剔除错误数据**(常规软件无此功能,销售数据常有水分)。
  - 常用模板:价格带区间销售占比/RFM/VIP级别变化/AARRR/**周系数分析**(判商圈特点·预估销售的度量系数)。
- **关键词**:数据不骗人·关键在解释 / 数据化管理三阶段 / 客观真实·剔除错误数据 / 周系数(判商圈预估销售)

## 5. 系列11 三位一体:库位+货位+人位 ⭐⭐
- **核心论点**:商品货位管理=电脑里设置(品类架构+配置)+卖场里配置(数量+位置)。**商品数据库管理三步**:① 制定合理商品品类(部门/大类/中类/小类·4级,绝大多数超市都做)② **「三位」设置:库位(系统数据库位)、货位(卖场BAY)、人位(员工类别课组)相互关联** → 解决商品定位优化/人员区域管理/绩效考核 ③ 实际运用(可用 Power BI/Excel 建表关系做库位分析)。
  - 货位例:食品组/生鲜组→各组下分 BAY → 每商品都有货位"家"(编码);人位:食品组→休闲散货→老刘。
- **关键词**:**三位一体=库位(系统)+货位(卖场)+人位(课组)** / 4级品类 / 三位关联解决定位·绩效 / 承接批41货位三级

## 6. 系列12 没有绝对好坏商品·商品综合效用PSI·商品分级 ⭐⭐⭐
- **核心论点**(本批最强·三块):
  - **① 没有绝对的好商品/坏商品**:销量好但毛利低?毛利高但周转慢占库存?周转高但占排面大库存费高?坏商品淘汰后老客户流失、其他商品也不好卖(强关联)→ **单看销售额/销量/毛利率不够**。
  - **② 商品综合效用(PSI商品重要度)**:商品在卖场承担不同作用,按作用做不同侧重评估。例:取毛利的商品考核销售额/毛利额/毛利率;促销商品考核人气度/销量/周转/关联度。**指标动态**(随季节/营销/商圈变)。**判据金句:"只要某商品发挥了预期的作用(价值),就是好商品;没发挥作用,就不是。"** "如何让有效用的好商品越来越多=超市永恒课题。"——**这是 [[SKU角色层与目的品保护机制_v0.1]] 的理论根**(目的品/毛利品/人气品=不同效用角色)。
  - **③ 商品分级(精细化效用落地)**:价签 020325+25,前段=品类,末尾=**商品分级=排面总陈列量÷日均销量=排面放满能卖几天**。三大用途:**(a) 判排面是否合理**(10天卖1个却10排面=浪费;每天10个却1排面=日补10次);**(b) 按级别设不同安全库存**(A类4天满排面销完→安全库存5天)→电脑自动下单;**(c) 加速汰换**(等级向中间靠拢,畅销排面增/滞销缩,系统按滞销状态自动淘汰)。
  - 附:缺货**关联损失**(强关联A缺货→B也不买,损失须与关联商品一同算);王老吉效用例(A家3.5/B家3.2敏感价格战/C家3.8衬托加多宝);SKU加法vs减法(本质=好商品占比;加法=依赖后台利润因循守旧);品类做宽vs做深(整体做宽/部分精选/部分做深·按效用精细化)。
- **关键词**:**没有绝对好坏商品** / **商品综合效用PSI(按作用动态评估)** / 发挥预期作用=好商品(SKU角色层理论根) / **商品分级=排面总陈列÷日均销=能卖几天** / 分级三用途(排面/安全库存/汰换) / 缺货关联损失 / 效用赋予(王老吉例)
