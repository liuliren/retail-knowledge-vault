# 逐字精读卡片集 · 批45 生鲜订货建模 · 小类内ABC · 生活场景选品

> 📌 **本批 = 「商品库存管理分析建模」4/6/7**(2022年5月·技术定量·细读)。聚焦 Excel/Power BI 把生鲜与食品百货的**订货/台账/选品配置**工程化。**与六哥两件大事强相关**:① 生鲜选品法→直击 SaaS 第三阻碍「生鲜验证」;② 小类内ABC→精化 SKU 防误杀。
> ⚠️ 系列编号 4/6/7(5 篇被微信豆隐藏/缺号)。下一批续读 商品库存管理分析建模 10/12/14/15。

## 1. 建模(4) 生鲜台账运用 + 生鲜订货建模准备 ⭐⭐⭐
- **核心论点**:生鲜陈列台账=解决陈列两核心(**位置+数量**)+衍生管理(促销/卫生/保鲜/价格/损耗/交接班)→把生鲜销售·陈列·日常管理**串成闭环**。生鲜订货=解决量(基本量+动态需求量)。
  - **⭐生鲜选品全新理念=按生活场景(非销售排名)**:不能按常规销售/综合值排名选品,要按**生活场景**——"烟火味":围绕一顿饭/一个厨房/一张餐桌/打火锅需要哪些商品。仅基于单品考虑往往没效果。生鲜"五品"重定义:**鲜活品/加工品/熟制品/附属品**。
  - **生鲜7特点**:价格变化大调价频繁 / 季节强陈列调整快 / 分级评级专业 / 采购流程变化大不确定多 / 保质期短=日配需每天精管 / **高粘性=差异化商品(做好增顾客信任与价值)** / 订货周期短频率高(几乎天天在订货)。
  - **⭐选品 ABC 不能只看全场占比**:要算**每个小类(=一个顾客需求)内每个商品的 ABC**——某商品全场 C 类但在小类内是 A 类(佼佼者/结构必需),不能据全场 ABC 淘汰它。
  - **动态订货量**:4周平均/上周/本周/安全量/昨日销量**多参考量综合加权**→动态建议订货量。**实践能满足70%商品订货准确率就很准了**。
  - **订货的痛**:建模容易、**难在前期准备**(数据/信息准备);不愿前期花时间→后面每环节耗大量时间。**补货与配货分离=缺货根源**(应统一关联)。系统设置:满陈列量→**"内仓为零"**(订货围绕满陈列)。
- **关键词**:生鲜台账=位置+数量闭环 / **生活场景选品(烟火味·非排名)** / 生鲜五品+7特点·高粘性差异化 / **小类内ABC(全场C小类A不可淘)** / 动态订货量加权(70%即很准) / 难在前期准备 / 内仓为零

## 2. 建模(6) 生鲜订货建模之二(连锁多门店Excel版) ⭐⭐
- **核心论点**:门店与采购同时处理信息(永续订单·联合编辑 APP:OA/石墨/钉钉)。Excel 公式实现(VLOOKUP 引各门店订货/INDEX/IFERROR 容错)。
  - **采购四过程**(把采购的"分货"与算价工程化):① 定是否采购+实际采购量+**分货**到各门店;② 算进价成本(20公斤120元→6元/公斤,分店金额);③ 进价售价→**自动算毛利率**(偏低偏高调售价;**进价高于平时则不适合做促销**);④ 配货表**贯穿送货/验货/入库同一张表**(统一条码信息·不重复敲键盘)。
  - **核心方法论**:把订货作业**每个环节统一**起来(否则各环节各部门各自图方便→费时费力出错);技术含量不高,**每环节效率叠加才是意义**。
- **关键词**:永续订单·联合编辑APP / 采购四过程(采购→算进价→自动毛利→配货贯穿) / 进价高于平时不促销 / 环节统一=效率叠加

## 3. 建模(7) 食品百货台账建模 ⭐⭐
- **核心论点**:食品百货不像生鲜需频繁调整,但几年不优化也是弊病(缺模型难坚持)。
  - **⭐货位编码内单品数=5类**:① **系统信息数**(最初配置·最大)② **在陈单品数**(已陈列)③ **在销单品数**(有销售)④ **能配单品数**(货位合适陈列数)⑤ **参考BAY/单品数**(同类店/竞争店)。**诊断判据**:系统信息数大但能陈列少=久未维护(该淘汰没淘汰);能配数远大于在陈数=拉排面厉害缺单品(畅销品)=**"良品驱赶劣品"现象**。
  - 建模步骤:加载(信息/结构/销售+货位表+满陈列量指标表)→建关系+度量→**ABC分类(关键参考)**→透视→加陈列棚格列(层/面/位/深/位码/代码/满陈列量)。
  - **⭐再次强调小类内ABC**(与建模4同):只有小类代表一个顾客需求,只有单品在自身类别占比才真实反映 ABC 状态,全场 C 类但小类 A 类不能淘汰。
- **关键词**:**货位5类单品数(系统/在陈/在销/能配/参考)** / 良品驱赶劣品 / 建模步骤(载表→关系度量→ABC→透视→棚格列) / 小类内ABC(再次强调)
