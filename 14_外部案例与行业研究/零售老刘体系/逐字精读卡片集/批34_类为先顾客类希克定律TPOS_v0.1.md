# 逐字精读卡片集 · 批34 类为先/希克定律/TPOS

## 199. 品类建模(215) 类为先前言 ⭐
- **核心论点**:**"纸上谈兵"零售版批判**——零售门店极度排斥理论(理论=地图/实践=走路·要协调非二选一);"**用战术的勤奋掩盖战略的懒惰**"。零售门店=理科"严重偏科"(只要能干的手脚·排斥思维工具方法论)→ "**越勤劳越穷**"的死循环。该拒绝的是"只会画饼不会干活"的人,不是拒绝方法论。"先活下来再谈优化"的惯性=多数门店永远停在活着、不谈优化。
- **关键词**:纸上谈兵零售版 / 战术勤奋掩盖战略懒惰 / 理科偏科=排斥理论=越勤劳越穷 / 理论=地图实践=走路

## 200. 品类建模(216) 顾客类为先 ⭐⭐⭐
- **核心论点**:**顾客购买的是"类"而非"品"**(任务型行为·"我渴了"→先想饮料类·SKU 是完成任务后的选择结果)。人脑**层次化决策**(确定类→缩到子类→比较具体品)。**"心智货架"**=顾客脑中按类排列的导航地图;**最好的陈列=陈列出的"类货架"无限靠近顾客"心智货架"**(陈列做不好最大原因=没按类陈列,非SKU不漂亮)。**需求购物篮**(进店前脑中的类标签组合·缺这个类则不进行下一步)。信息过载→顾客用"类"作过滤器简化决策。**三层类结构**(官方标准GB→供应链商品结构表→顾客心中的类·搞出皆大欢喜的"宽类的类")。**把繁杂留给自己,把简洁留给顾客**。
- **关键词**:顾客买的是类非品·任务型 / 心智货架·陈列=类货架靠近心智货架 / 需求购物篮 / 三层类结构 / 把繁杂留给自己

## 201. 品类建模(218) 零售类为先 ⭐⭐⭐
- **核心论点**:类优先于品=从"产品导向"转"场景导向"·零售商=替顾客管理"生活采购清单"的**代理人/管家**。**希克定律(Hick's Law)**:决策时间随选择数对数增长 `T = a + b·log₂(n)`→**类优先把"一次面对 N 选项"转为"多次面对少量选项"**(对数增长降为线性累加)。**TPOS**(Time/Place/Occasion/Style·日本零售从消费场景倒推商品组合·替代按品类/品牌选品)。范式:卡斯美(先定类再筛品·五段分位法·**满足 80% 核心需求非 100%**)/伊藤洋华堂(52周MD+PDCA·两年半做到"**几乎没有滞销品**"=每 SKU 都是顾客需要的类)/Lopia(肉类品类杀手·深做透一个类)/唐吉诃德(反例·海量SKU探宝)。
- **关键词**:类优先=产品导向→场景导向·管家 / 希克定律T=a+b·log₂(n) / TPOS场景倒推 / 卡斯美80%核心需求 / 伊藤几乎没有滞销品

## 202. 品类建模(219) 课类的宽类 ⭐
- **核心论点**:**生鲜/食品/百货三课宽类语义不同**:生鲜窄品=稳定活体类别(宽类需引入品质/时间维度)/食品=稳定标准品类(纯层级分类)/百货=可能跨多宽类(功能性集合·迭代快)。库存保质期紧迫性=生鲜>食品>百货。**生鲜的"宽类"反而最好做**(类少而清晰·更接近顾客心智),难在"**窄品=控品/品控**"(动态即时控 SKU·结构化窄品不盲目缩量·相同质量产地规格只留 1-2 主力);"**初品比标品难做**"。生鲜与食百:战略上是统一总目标的整体,战术上是互补协同的分体。
- **关键词**:三课宽类不同(生鲜活体/食品标准/百货跨类) / 生鲜宽类最好做难在控品 / 初品比标品难做

## 203. 品类建模(220) 解先类后品 ⭐⭐
- **核心论点**:**52周MD = 落地"类优先"的操作系统**("**作、演、调**"闭环·战略单元=场景需求类·铃木哲男·52周=TPOS 的 Time 维度·持续把"生活提案"呈现给顾客)。传统误区=单品思维(只盯 TOP100 SKU→同质化挤占+重要子类缺失)。**宽类窄品首先解"类缺失"**(不是顾客不买,是类缺失了)。选品顺序=**先定类清单再填品数量**;货架按"类"分区非按品牌(同子类集中陈列·清晰分界线·按心智决策顺序排:解渴→提神→习惯→健康→享受)。**类优先=顺序非重要性**(不同场景品敏感度不同:目标性品类深度 2-5 SKU/便利性 1 品牌)。**铁律:类缺失的代价 > 品过度细分的代价**(增一个缺失小类的回报 > 同小类加第 5 口味)。
- **关键词**:52周MD=类优先操作系统·作演调 / 宽类窄品首解类缺失 / 先定类清单再填品 / 类优先=顺序 / 类缺失代价>品细分代价

## 204. 品类建模(221) 宽类TPOS ⭐⭐⭐
- **核心论点**:**TPOS(Time时间/Place场所/Occasion场合/Style生活方式)= 从消费场景倒推商品组合·替代按品类/品牌选品**·按"用来做什么"分而非"是什么"分。四步法(画生活场景地图→定每场景需求类→每场景配 1-2 SKU→动态调整)。**TPOS 定义"宽类"(方向盘),宽类窄品=执行(车轮)**;三步流程:**TPOS 定场景 → 映射为宽类 → 宽类配窄品**。野餐=一个宽类(把四散商品聚合·提客单价)。卡斯美(从"配齐品类"到"配齐场景")。核心价格带附近配更多 SKU;同一窄品不同场景不同角色("吃饭不重要,重要的是跟谁吃")。**TPOS 让选品从"我有什么"转向"顾客在何时何地、为何需要"**。
- **关键词**:TPOS=场景倒推·按用来做什么分 / TPOS=方向盘·宽类窄品=车轮 / 三步(定场景→映射宽类→配窄品) / 配齐品类→配齐场景
