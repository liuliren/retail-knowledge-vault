# 逐字精读卡片集 · 批35 宽类语义汇/求类/类要清晰

## 205. 品类建模(222) 宽类语义汇 ⭐⭐
- **核心论点**:**宽类窄品线性逻辑 = "以类优先 → 属性继承 → 可得性约束 → 供应链协同"四递进环节**(每步是下一步前提)。核心三问:① TPOS 重定义"宽类"=让分类成为全链共享语言+顾客购物地图 ② 属性继承让规则自动传给窄品=低成本高效精细化 ③ 窄品可得性约束打通采购/仓储/门店/促销/财务。三大心法:**宽类负责"聚人气"(TPOS场景聚商品)/窄品负责"保生命"(属性继承+可得性管生命周期)/供应链协同负责"提效率"**。MVP 试点 1-2 个高流量宽类(早餐/火锅)·基于"4 个 1"单个单元作业,不需一步到位。
- **关键词**:线性逻辑四环节(类优先→属性继承→可得性→协同) / 三问(共享语言/低成本精细化/打通全链) / 三心法(聚人气/保生命/提效率)

## 206. 品类建模(223) 求宽类的类 ⭐⭐
- **核心论点**:**陈列必有对错**("陈列无对错"=最大谎言·"只有适合与否"=漂亮废话→成了"想咋摆咋摆"的借口);陈列对=有效(达成目标)、错=无效(与目标/顾客习惯脱节)·**80% 门店陈列没有"目标"**。**Item 类表/宽类表 = 宽类窄品的前提**(没有=无法进行)。**"类"=顾客决策单元**(生活场景+功能的结合体·比小类细、比 SKU 粗·边界由顾客决策习惯定)。属性层级=类(第一属性品类)→Item(属性组)→SKU(第四属性·规格口味);**中间类=几个属性的组合**(天然在小类与 SKU 之间)。健康度诊断(ALDI/Costco/山姆第一档·国内头部堪堪及格·**80% 门店从没执行过宽类窄品、甚至无逻辑基础**)。
- **关键词**:陈列必有对错·80%门店陈列无目标 / Item类表=前提 / 类=顾客决策单元(比小类细比SKU粗) / 中间类=属性组合

## 207. 品类建模(224) 宽类表实例 ⭐⭐
- **核心论点**:**得宽类的类=从顾客"购物任务"倒推**(顾客不说"买酱油",说"做红烧肉")。**"宽类的类"语义公式 = 消费者在特定场景下的"最小可识别任务单元"**(比小类细/比 SKU 粗/边界由顾客决策习惯定,非商品属性)。四步:① 画顾客"任务地图"(按时段拆+看商品包装/使用说明了解任务)② 筛高频/高敏感/难替代任务(才独立成类)③ 对照合并现有分类表(横跨多小类的建"虚拟单元")④ 数据验证微调(问卷>70% 区分则拆类)。**"宽类的类=顾客用脚和嘴投票投出来的"**(故"十店十色");你不需专家现成表,每天听顾客问"有没有…"就能画出自己的宽类地图。
- **关键词**:从购物任务倒推 / 宽类的类=最小可识别任务单元 / 四步(任务地图/筛任务/对照合并/验证) / 顾客用脚和嘴投票

## 208. 品类建模(225) 门店调整例 ⭐⭐
- **核心论点**:老李社区超市(210㎡)饮料区**五步落地实例**:① 摸现有类有多乱(A4 纸画现状·一看吓一跳)② 找 5 约束(物理空间/顾客认知/供应商/资金/品类角色)③ 简易工具(ABC 排名砍低效类 + 竞品对标补缺类 + TPOS 按场景加类)④ 动手调成 8 个"类专区"(SKU 32→13)⑤ 验证(+18%·补货次数少了)。**陈列线单元 = 货位总长 ÷ 类数**。**"约束不是挂墙上的东西,是调货架前必算的几笔账"**(货架多长/顾客认不认/钱够不够/供应商稳不稳/这类该不该做)。全程纸笔+常识,不搞复杂模型,效果立竿见影。
- **关键词**:五步落地(摸类/找约束/简易工具/调专区/验证) / 陈列线单元=货位总长÷类数 / 约束=调货架前必算的账 / 纸笔+常识

## 209. 品类建模(226) 视觉POP例 ⭐
- **核心论点**:沃尔玛实地观察:① **主通道无中间堆头**(空出来=保整体视野·反"寸土必争见缝插针")② **全包装陈列**(生鲜包装化=工业化靠近标品·但失去顾客"挑选乐趣"=客群选择问题)③ **商品清晰化(不遮挡/视线开阔/易看易找易选)=陈列最核心原则·POP 只是辅助**(过多过杂 POP 喧宾夺主·"让商品开口说话"前提是商品本身够漂亮)④ **沃集鲜把 POP 印在包装=极致标准化** vs 胖东来极致亲切=POP 两条极致路径。蔬菜竖摆(=自然生长样子+保鲜·看着也舒服)。
- **关键词**:主通道无堆头 / 全包装=工业化但失挑选乐趣 / 商品清晰化=核心·POP辅助勿喧宾夺主 / 标准化vs亲切两路径

## 210. 品类建模(227) 论类要清晰 ⭐⭐⭐
- **核心论点**:**"类要清晰"= 供应链(门店)的"类"无限接近、适配顾客心智的"类"·主要体现在陈列 + 陈列 POP**。核心="**一眼能辨,三步能懂**"(3 秒判断当前是哪个类·1.5 米外看清类边界)。理论根=类优先 + 希克定律 + 认知减负。三层:宏观(大类空间区隔·主通道作天然分界)/中观(子类视觉标识 PP·分类 POP 牌)/微观(同类纵向陈列"类柱"+分隔条+POP 插条)。注意:① **避免过度分类**(每层 1 个维度·复合属性越多类越不清晰)② 分类一致性(全店统一·优先按场景/功能分)③ 动态调整慢过渡("搬家告示"POP 留 1-2 周)④ **标识与位置精确对应**("一物一签"·补货不得跨类乱放)。**类 POP 牌必须用顾客听懂的语言**("普通/精品猪肉"=门店管理语→换"骨头类/五花肉类/肉馅类"=顾客语·手绘 POP 更亲切)。验证=顾客问"X 在哪"的次数越来越少。**"90% 门店只有陈列、没有落位"**。
- **关键词**:类要清晰=供应链类靠近顾客类 / 一眼能辨三步能懂(3秒/1.5米) / 三层(宏观/中观/微观) / 类POP用顾客语言 / 90%门店只有陈列没落位
