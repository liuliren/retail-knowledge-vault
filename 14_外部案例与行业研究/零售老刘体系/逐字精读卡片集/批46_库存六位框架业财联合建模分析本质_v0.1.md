## 1. 建模(10) 业务财务联合建模 ⭐⭐
- **核心论点**:超市业务与财务流程常**各成一套**,应统一建模(多事实表+维度表)。商品流每环节(要货/订货/送货/验货/入库/对单/结账)各产一单据→**单据维度表**。财务现金流四状态:**应付/待付/未付/已付**;操作只两个:**对账+付账**。
  - 建模关键点:付款状态条件列(未付/已付=未付金额为0/待付)、从对账单号提取时期、账单类型列(单号前缀判类型)。**财务要金额度量,业务要数量度量**(同一模型同时设两类)。
- **关键词**:业财必须统一建模 / 现金流四状态(应付/待付/未付/已付)·对账+付账 / 单据维度表 / 金额(财务)+数量(业务)双度量

## 2. 建模(12) 库存管理"六位"框架 ⭐⭐⭐
- **核心论点(本批金矿)**:商品在卖场的位置和数量,体现为**六位:定位 / 定数 / 定品 / 定架 / 定价 / 定管**:
  - **定位**:每个商品都有"家"(货位编码)。狭义=卖场位置;广义=商品定位(新店商圈调研定类别结构落位图;老店按需求先调小类)。**小类货位BAY分配=综合权重占比×总BAY数**(例:某小类综合占比0.2%×300组=6组BAY)。货位管理→随时监控(BAY单品数/缺货/临期/畅销品在否/动销数/某货架为何越来越差/调整后效果)。
  - **定数**:确定合理数量(各类别数量分配·选品简化优化的前提)。
  - **定品(选品)**:末位淘汰(销售排名)太粗糙;选品依据品类管理。**先搭商品完整性(集群概念)**——"店里有面粉却没酵母=和顾客过不去";再**场景营造**(早市/厨房/烹调场景;调味蔬菜葱姜蒜韭放一起=自定义"调味蔬菜"类·虽分属不同结构类)。
  - **定架**:确定货架位;卖场调整多是**减货架**(原定位错+没按商圈调+没维护BAY销售状态)→应按集群/场景管理。
  - **定价**:价格是最好的经营工具;有生鲜的一般超市要"**时刻表演便宜·烟火味·离顾客近**";不能让顾客觉得贵("开业就觉得贵=一开业就失败一半")。
  - **定管**:以货位为业务主线定流程规范/绩效考核。
- **关键词**:**六位框架(定位/定数/定品/定架/定价/定管)** / BAY分配=综合权重占比×总BAY / 定品=集群完整性+场景营造(面粉没酵母) / 定价=表演便宜烟火味

## 3. 建模(14) 区位+货位+课组关联建模 ⭐⭐
- **核心论点**:引入**商品区域位(区位)**=陈列同类商品的一个或多个BAY(货架/端架/展台/堆头/挂网/冷柜)。区位与系统商品结构类别**只能近似**(陈列确认后按实际陈列区域定区位)。建模:商品结构表(小类最细)关联课组+区位 → 区位关联货位(参数表/系统设置) → 商品结构表**加课组/货位/区位维度列**→参与分析(+员工表=绩效)。**至此存货/货位管理理论部分结束。**
- **关键词**:区位=一或多BAY(陈列同类) / 区位↔货位↔课组↔小类关联建模 / 货位三位一体的建模落地 / 理论部分收尾

## 4. 建模(15) 库存系列收官 + 数据分析本质 ⭐⭐
- **核心论点**:盲目分析=费力无收获,需清晰掌握分析体系→落地成"模"。库管系统建模(Power BI):**业务单据(数量度量)+财务单据(金额度量)统一一个数据模型**。
  - **零售数据分析主维度=商品**;**条码(货号)=最细粒度**;时期Date/类别(最细小类·先与商品信息表关联)/供应商/门店=建关系主维度列。
  - **再次小类内ABC**:查每个小类内每个商品在整个商品中的ABC及销售。
  - **结论(数据分析本质·全系列收束)**:**数据分析首要任务=建模**(哪怕一个Excel表也是建模);数据模型=维度+度量;**核心很简单:从什么角度(维度)看指标值(度量),服务业务决策。**
- **关键词**:建模=分析首要任务 / 商品=主维度·条码最细粒度·小类先关联商品信息表 / 业务数量+财务金额统一模型 / **维度×度量=分析本质(再收束)**
