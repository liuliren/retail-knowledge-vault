## 1. 分析建模(22) 奢侈的货位棚格(2)·数量为先深化 ⭐⭐
- **核心论点(数量为先的可操作判据)**:
  - **单品数三类来源**:① 系统获取(信息单品/在销品动销/库位满陈列品/当前库存品)② 手工维护(**在陈品**·棚格单品=配置单品+黄金陈列位单品)③ 额外(主推/会员/促销/新品/可订货/ABC/配送/盘亏/报损等)。
  - **⭐单品数大小排序**:`信息单品数 > 在库单品数 ≥ 库位配置单品数 > 在销单品数 > 在陈单品数 ≥ 货位维护单品数 > 新品/促销等受限单品数`(单品数变化分析的参考标尺)。
  - **⭐数量的意义**:重视数量=质量/内核/夯实基础/精细化/长期/重过程;只重销售毛利=眼前利益/短效/省心/拍脑袋。**基于单品数量变化才是真正研究顾客实际需求**(顾客要的首先是小类及小类的单品数)。
  - **⭐衡量 vs 计算(强判据·可直接当绩效/诊断话术)**:① **行为考核/业务管理→以数量衡量**(例:甲丢100件/100元、乙丢10件/2000元·按金额判"乙差"是大错·**商品管理上甲更差**·每月丢100件相当严重)② **绩效/财务成本→以金额计算**(损耗是结果)。用词区分:**衡量(数量)≠计算(金额)**。
  - 不提数量=零售业永远的痛;日本精细化=掌控数量到极致;棚格管理被搞成"奢侈品"(太能做好而没做)。
- **关键词**:单品数三类来源 / **单品数大小排序(信息>在库>=配置>在销>在陈>=维护>受限)** / 数量=研究真实需求 / **衡量(数量)vs计算(金额)·甲乙丢失例**

## 2. 分析建模(23) 货位棚格图表建模(1)·主推≠促销 ⭐⭐
- **核心论点**:双ABC(去年今年)定义**主推商品**(双A·保证不缺货+调黄金位+增陈列面=动态管理)和**适合促销的商品**。**⭐主推品≠促销品**:主推品一般不促销(**很多超市拿A类商品促销=得不偿失**);促销品=动态即时建议(销售下降→促销修正·非事后分析效果)。
  - **建模顺序(印证数量为先)**:先备各类单品数(总/在销/在库/在陈·"像烹饪先知道有哪些食材")→配置表(没配置表就只能做静态棚格)→**先数量再配置再定位置**(层/位/面/深)→Excel INDEX+MATCH(单元格=行列坐标·层=行/位=列)引到棚格表。"很多超市做不了棚格=思考角度和顺序错+没配置表"。
- **关键词**:双ABC定义主推品+促销品 / **主推品≠促销品(拿A类促销得不偿失)** / 促销品=动态即时建议 / 先数量→配置→位置 / INDEX+MATCH棚格

## 3. 分析建模(24) 货位棚格图表建模(2)·PBI ⭐
- **核心论点**:**两条建模轨道**=① 商品状态变化建模 ② 商品流转过程建模。Power BI 棚格:矩形(层=行/位=列/面+值=单品)→筛货号货位→棚格图;Base64货号图;**组合度量**(采购数/销售数/库存数/**售罄率**组合显示·适应多业务场景)。数据来源=单品配置表。
- **关键词**:两条建模轨道(状态变化/流转过程) / PBI棚格(层行位列) / 组合度量(采购/销售/库存/售罄率) / 数据来源=单品配置表
