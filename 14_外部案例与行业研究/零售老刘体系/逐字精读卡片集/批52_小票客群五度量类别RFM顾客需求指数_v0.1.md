# 逐字精读卡片集 · 批52 小票客群五度量·类别RFM·顾客需求指数

> 📌 **本批 = 分析建模10-12(顾客需求指数系列)**。客群/需求量化——**直击 M5 小票客群法**。**双金矿:① 小票客群五度量(客群分析的度量底座)② 类别RFM(顾客RFM→小类RFM,绕开会员缺失)**。
> ⚠️ 下一批续读 分析建模13-17(类别经营属性/价格带/陈列规划/订货量/需求趋势)等。

## 1. 分析建模(10) 顾客需求指数 ⭐⭐
- **核心论点**:精细化必须把销售粒度**下沉到小类+每天**(课组/大类粒度不够);**一小类=一顾客需求**→这类下沉到小类的指数=**顾客需求指数**。建模:小类×每天销售系数。
  - **周系数三运用**:① 周星期几系数**判商圈**(住宅区7天系数接近·合计越接近7=每天越稳定;商业区周六日系数高)② 小类周系数(订货/选品/绩效·业务必回小类才有意义)③ 每天系数(日指标制定)。
- **关键词**:粒度下沉到小类+每天 / 顾客需求指数(一小类=一需求) / 周系数判商圈(住宅区≈7稳定) / 小类系数=业务回小类

## 2. 分析建模(11) 顾客需求变化指数·小票客群五度量 ⭐⭐⭐
- **核心论点(客群度量底座·M5直接可用)**:**一张购物小票=一个顾客的一次购物行为**→以小票替代客群分析(超市会员管理普遍缺失,小票客群是更可靠方向)。
  - **⭐小票客群五核心度量**:
    - **客流数 = 不重复小票数**(`DISTINCTCOUNT(小票号)`·已购物顾客统计)
    - **客单价 = 销额 ÷ 客流数**(整体定价策略+商圈购买力)
    - **客单量 = 销量 ÷ 客流数**(含称重·注意≠品单数)
    - **品单数 = 销售表行数 ÷ 客流数 = 连带率**(商品关联性)
    - **品单价 = 客单价 ÷ 品单数**(商品结构平均价格带+商圈消费水平)
  - 客流是基础度量;**Power BI易实现、Excel难做即时(这是很多分析少涉客流的原因)**。
  - 分析方法=**对比**(任意两期对比每个小类的客流客单变化→洞悉需求变化)。**客流×客单升降→9种可能·4种主要类型**→加「分类判断」列(IF)。**一个小类变化看不出·每个小类恶化加起来=大失误**。
- **关键词**:**一张小票=一个顾客** / **五度量(客流=不重小票/客单价/客单量/品单数=连带率/品单价)** / 客流Excel难做即时 / 两期对比 / 客流×客单9宫格4主类型

## 3. 分析建模(12) 顾客需求与类别属性·类别RFM ⭐⭐⭐
- **核心论点(绕开会员缺失的客群法)**:顾客盲区(购物路径/停留/放弃哪些/今天vs昨天/不满意多少·统统不知道)。**RFM(R最近消费/F频次/M金额)针对会员,但超市会员缺失→对顾客分析意义不大**。
  - **⭐换思维:顾客与商品关联才最有意义**(顾客买的是商品·研究笑没笑最终回到买没买/客单品单)。一小类=一顾客需求→**把顾客RFM转化为小类RFM(类别RFM)**:**R=小类最后被购买时间距今天数·F=客流(引用)·M=销售额**;RFM按平均值计分(大于平均=1否则0)→**8种小类类别属性**(借用客户RFM8分类)。
  - **运用**:小类客流×RFM分级对比——例:"重要价值类别"理想应对应"良好",却对应"**促销过度类别**"→合理吗?为什么?如何改?(诊断切口)。
- **关键词**:顾客盲区 / RFM在超市=会员·会员缺失则意义小 / **顾客与商品关联才有意义** / **类别RFM(小类RFM·R距今·F=客流·M=销额·8分类)** / 重要价值类别vs促销过度类别(诊断)
