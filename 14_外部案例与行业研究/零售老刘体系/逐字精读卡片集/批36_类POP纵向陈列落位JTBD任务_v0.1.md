# 逐字精读卡片集 · 批36 纵向陈列/落位/JTBD任务

## 211. 品类建模(228) 类品的POP ⭐⭐
- **核心论点**:**局部对≠全局对**(畅销品断货临时补 B 牌→A 品被干死/勤劳搞卫生→坪效低)·**经验=指南针非地图**。**宽类窄品(内容/骨架)+VP/PP/IP(空间/肌肉)+POP(沟通/神经)= 三体系协同**;运用三步走:**先以宽类窄品定结构 → 再以 VP/PP/IP 定布局 → 最后以 POP 定沟通**,共同降决策成本。POP 可**反向验证窄品**(熟鸭蛋生熟=窄品独有属性,不需 POP)。
- **关键词**:局部对≠全局对·经验=指南针非地图 / 宽类窄品+VP/PP/IP+POP三体系 / 三步走(结构→布局→沟通)

## 212. 品类建模(229) 类纵向陈列 ⭐⭐⭐
- **核心论点**:**纵向陈列 = "类优先"+"类要清晰"的物理实现**(格式塔接近律:**垂直靠近=同组/水平远离=不同组**·形成"类柱")。同时符合希克定律(横向"一排几十选择"→纵向"先选类柱再选柱内品")。**理论 vs 实践矛盾**(供应商付费买排面要横向品牌墙/品类定义模糊/门店执行便利)→妥协:黄金视线区(2-3层)强制纵向、其余放宽,合同写入纵向条款,数据倒逼供应商。**纵向陈列本质=效率提升**(降顾客+门店时间成本)。"先进先出"≠陈列原则(=仓库管理张冠李戴)。
- **关键词**:纵向陈列=类优先+类清晰物理实现 / 格式塔接近律·类柱 / 希克定律 / 黄金视线区强制纵向 / 本质=降时间成本

## 213. 品类建模(230) 落位和陈列 ⭐⭐
- **核心论点**:**落位(落的是"类")= TP(时间+位置)·"选座位"·资源分配** vs **陈列 = OS(场景+方式)·"坐得好看"·展示优化**(TPOS 拆 TP+OS)。**铁律:先落位后陈列**·"**落位定生死,陈列定高低**"(落位定可能性·陈列定落地性);落位制约陈列/陈列反馈落位。两者双重保障"类要清晰"。**"少谈需求,多谈任务"**(某时段客单价高=一种任务·而该低峰时段门店资源最少=失效)。简化口诀:先问"该放哪"→再问"该怎么摆"→最后"该怎样 POP"。
- **关键词**:落位TP=资源分配vs陈列OS=展示优化 / 先落位后陈列·落位定生死陈列定高低 / 少谈需求多谈任务

## 214. 品类建模(231) 类做了嫁衣 ⭐⭐⭐
- **核心论点**:**"更快的马 vs 一辆车"**(Henry Ford·顾客表达的是缺失感、不是解决方案)。**门店所有业务=给顾客做嫁衣**——顾客来"办事"不是"逛超市",门店=完成"另一件事"的**跳板**。**业务=显性,任务=隐性**;**JTBD"待办任务"零售版**(顾客雇佣商品完成任务、改善处境·任务往往含情感/社会性、深埋日常琐碎)。把隐性任务**翻译**成显性解决方案:预制菜不叫"预制菜"叫"15分钟一桌菜"(→子类:开胃冷盘/主菜/时蔬快炒…)。门店内耗(熵增)都源于显性业务过度竞争、对隐性任务视而不见。
- **关键词**:更快的马vs汽车 / 业务=给顾客做嫁衣·门店=跳板 / 业务显性·任务隐性 / 把隐性任务翻译成显性解决方案

## 215. 品类建模(232) 任务业务层 ⭐⭐⭐
- **核心论点**:**六层业务/需求层级链(自下而上):属性 → 组合 → 品类 → 需求匹配 → 宽类 → 任务(任务类)**。**品类=管理视角、非顾客视角**;从"品类→宽类"=根本转变(内部管理视角→顾客生活场景视角);**任务=宽类的上层聚合**(一个任务对应多个宽类·如"完成孩子学校午餐"=早餐+便当盒+水果+饮料)。三操作贯穿(类优先=第5层宽类/类要清晰=保宽类互斥/落位陈列=覆盖2-6层)。零售从"卖商品"→"卖任务"。**⚠️"卖任务"≠"卖体验"**(体验=任务完成后的附加结果·必须先完成任务·感冒药案例:先有药治好感冒,体验是附加品)。
- **关键词**:六层链(属性→组合→品类→需求→宽类→任务) / 品类=管理视角·宽类=顾客视角 / 任务=宽类上层聚合 / 卖任务≠卖体验

## 216. 品类建模(233) JTBD与任务 ⭐⭐⭐
- **核心论点**:**JTBD(Jobs to be Done·待办任务)=克里斯坦森**·**顾客"雇佣"商品完成任务**(不是要 1/4 英寸钻头,是要那个洞)。任务=进步契机(摆脱困境/获得愉悦)。**75-85% 新品/业务失败=没适配真实任务**。**顾客任务三重维度:功能性(好不好用·入场券)+ 情感性(满不满意·复购理由)+ 社会性(我是谁·溢价来源)**;三层权重不同(大米功能 90%/奢侈品包社会 60%)。跨品类竞争(同一任务的所有替代品都是对手)。误区(竞品当自家需求/直接问顾客=更快的马/只看功能忽略情感社交)。**"多一层多一层钱"**:星巴克 38 元=功能(提神)+情感(仪式感)+社交(辨识度);胖东来=情感任务做到极致。
- **关键词**:JTBD=顾客雇佣商品完成任务(要洞非钻头) / 75-85%失败=没对准任务 / 任务三层(功能/情感/社会) / 多一层多一层钱
