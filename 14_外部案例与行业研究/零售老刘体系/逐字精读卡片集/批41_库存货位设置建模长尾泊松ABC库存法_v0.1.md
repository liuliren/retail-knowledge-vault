# 逐字精读卡片集 · 批41 货位设置·建模运用·长尾泊松·库存优化能力·ABC库存法

## 1. 系列17 货位设置(解决「位置」问题) ⭐⭐
- **核心论点**:库存管理两要点=**位置 + 数量**;系列17专攻位置。货位三级设置(以思迅商锐9.7为例):**1级 机构库位(店/仓分离·4位码如0101)→ 2级 区域管理位(一排货架/一条堆头)→ 3级 货架货位(BAY / BAY第几层 / BAY第几层第几位)**。三种精度按需选:① 管到每个 BAY(最常用);② 管到 BAY 的层(陈列线);③ 管到棚格位(一个商品一个位·维护成本高·仅烟酒柜精品柜等不常变区)。
  - **满陈列量 = 纵向陈列数 × 横向陈列面数**(卖场不含内仓的最大陈列量)→ 直接作系统**自动补货的「按库存量(满陈列)」再订货依据**。这是「位置」与「数量」的咬合点。
  - **课组设置**=货位与人挂钩(哪些人管哪些商品)→ 绩效 KPI 单元。货位也是商品的营运管理属性。
  - **金句**:"再好看的商品陈列,都需要经过**合适的位置与合理的数量**这2者的检验,我们所缺乏的就是这个**检验标准**,没有标准很多库存管理就缺乏依据。"
- **关键词**:货位三级 / 满陈列=纵×横 / 再订货点 / 棚格位 / 课组=人货挂钩 / 位置与数量的检验标准

## 2. 系列18 货位管理单元 BAY + 建模四运用 ⭐⭐
- **核心论点**:SKU 要做减法——**阿尔迪只 450-700 种**(绝对精算),研究称**普通人生活 270 种已够**;而一般超市 SKU ≥20000,做到 10000 以内已算很好。"超市到底经营多少 SKU 合适"是无解之问,但棚格图表+满陈列+订货流程能有效处理「位置×数量」两大问题。
  - **位置→棚格货位模式;数量→满陈列模式**,二者都靠**商品 SKU(条码)关联**。
  - **建模法**:从后台导出货位/课组/商品表 → Excel/Power BI 建表关系+写度量 → 简单分析模型成型。
  - **建模四运用**:① 新店/旧店商品配置优化(按小类销售表现预估每 BAY 单品数·可带对比店参考);② 选品参考(从 BAY 对应小类选品);③ 棚格图表制作(筛 BAY 自动生成棚格表·打印作陈列标准);④ 相关分析(每 BAY 小类单品数+销售→绩效)。
  - **以货位 BAY 为管理单元**:人员日常管理(谁负责哪组 BAY 的卫生/价签/临期)、随时查坪效、按货位盘点报货。课组三级用「课/组/岗位」**不用人名**(人员会变动)。
- **关键词**:SKU做减法(阿尔迪450-700/普通人270够) / 棚格=位置 满陈列=数量 / BAY为管理单元 / 建模四运用 / 岗位非人名

## 3. 系列19 库存现状+异常库存+长尾泊松分布 ⭐⭐⭐
- **核心论点**:① **库存关键还是人**——再好的系统,缺思想重视/科学订货(OPL)/及时盘点/严控,就进"畅销被滞销干死、滞销大陈列"的恶性循环。② **异常库存=负库存+积压+脱销(畅销缺货)+临期**;临期**重在事前预防而非事后处理**,涉及商品全生命周期(新品建档设销售期望期+保质期·入库登生产日期·系统预警);处理按保质期阶段分级折扣(距期1月8折/2周5折/1周2折)。
  - ③ **长尾需求的库存计划(核心难点)**:一个小类=一种顾客需求;小分类越多越满足"一次性购足",但供应链末端面对的注定是**长尾**(几万 SKU 大多每年只周转几次甚至不周转)。矛盾:长尾触觉越来越多,但超市又必须精简商品(20/80)。
  - ④ **长尾不符合正态分布 → 符合泊松分布**。这是**对批39/40 正态 Z 值安全库存模型的关键边界修正**:"我们熟悉的库存计划公式基本都基于正态分布,**并不适用于长尾商品**"。泊松三条件:平均值已知 / 随机(不可测具体时点)/ 独立。**只需一个参数=平均值**即可算未来某时段需求发生 N 次的概率。
  - **公式落地**:`POISSON.DIST(X, 平均值, True)` → X=再订货点,平均值=补货周期内平均需求,返回总需求落在[0,X]的累积概率=有货率。例:进口啤酒周均2箱,**存货4→95%不缺货(均19周缺1次),存货5→98%(均49周缺1次)**。门店据此定库存水位。
- **关键词**:库存关键是人 / 异常库存四型 / 临期重预防非处理 / 长尾≠正态 / **泊松分布只需均值** / POISSON.DIST算再订货点+有货率

## 4. 系列20 库存优化能力论+驱动力+谁做预测+ABC库存法 ⭐⭐⭐
- **核心论点**:① **库存优化的本质=能力问题**——"优化库存最后只能依据**优化能力**来实现",而能力=**组织+系统+流程**的有机结合。两条路:(a) 把库存计划集中到少数有能力的计划员(核心是数据/模型技术);(b) 改变管理模式(重点是人和流程·管好需求与变化)。**最大误区**:"总是期望用**短期的、非连续性的改变结果**,来改变长期的、连续性的、需要以能力改变的结果"(狠盘一次/清库一次→过不久又乱·治标不治本)。
  - ② **驱动力=流程+责任体系**:营采两班子矛盾(库存出问题谁都是受害者)→靠改善流程+明确责任归属破。**例:每天报损单注明原因(报货多/陈列差/采购质量/配送多)→一段时间后原因最多的部门主动担责改进**。采购的"十八般武艺/精通补货都不是关键,关键是**需求预测和采购计划用什么来驱动**"(一将无能累死千军)。
  - ③ **谁制定需求预测**:供应链分工基本准则=**谁离需求最近谁做**(零售门店离需求近预测更准·VMI 由此)。日常报货应由**销售员起 → 销售主管 → 店长 → 采购/供应商**,中间两级预测评估不可跳过。
  - ④ **ABC 库存分类管理法**:按**数量×价值**三分——**A:数量10-20%/价值70-80%;B:数量30%/价值15-25%;C:数量50%/价值5%**(价值可用销量/毛利/客流)。管控**A严B适C松**:A类低安全库存+高频小批量订货+高频盘点+紧盯需求偏差。**循环盘点**:A类全年盘12次/B类4次/C类2次(用 ABC 法的企业最适合循环盘点)。位置与数量两主题都可按 ABC 侧重(A类优先配货位棚格/优先配置/订货系数高于BC)。
- **关键词**:库存优化=能力(组织×系统×流程) / 短期非连续≠长期连续(治标误区) / 报损单注明原因倒逼担责 / 需求预测=离需求最近者做 / **ABC库存法(数量×价值)+循环盘点A12/B4/C2**
