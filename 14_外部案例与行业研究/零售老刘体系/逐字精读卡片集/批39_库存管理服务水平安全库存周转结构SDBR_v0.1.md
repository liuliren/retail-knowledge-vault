## 229. 库存管理系列(05) ⭐⭐⭐ 〔服务水平/安全库存计算·全系列最硬核〕
- **核心论点**：**①服务水平=有货率**（需求产生时立即有合适库存满足的概率）。按销定货（卖100订100）在正态分布下有货率只有 **50%**（高于/低于均值各50%）。要更高有货率必设安全库存。**Z值表（核心工具）**：放1个标准差σ→84.1%，2σ→97.7%，3σ→99.9%；Excel `NORMSINV` 把有货率转 Z：**95%→Z=1.64，99%→Z=2.33**。服务水平本质=一个系数（越高系数越大）。**②安全库存投资回报递减**：95%→99% 只升0.04，但 Z 几乎翻倍→为最后那几个百分点要囤大量库存不划算。出路=**差异化服务水平**（一般订单95%常规周期内/促销97%三天内/团购100%紧急1-2天）。**③铁律：安全库存绝不可分摊进需求预测**——拔高预测（把安全库存摊到未来10周每周+10%）会致存货率低下、长期成本上升；且系统把"多余预测"当过期需求自动取消→实际安全库存远低于应设值→**越畅销越缺货**。需求预测对付"平均需求"，安全库存对付"不确定性"，两个不同概念必须分开处理。促销订货统一乘1.65系数=错（A类畅销该大系数、C类该小系数，须按ABC分）。**④安全库存公式**：把需求不确定性σ与补货不确定性σ整合成综合标准差（根号下前半=需求不确定性/后半=供应不确定性，单位统一为天），再×服务水平系数Z。常简化（假定订货周期固定）。拍脑袋设统一天数/一刀切系数→不该高的高、不该低的低，短缺与过剩共存。
- **关键词**：服务水平=有货率·无安全库存=50% / Z值表(95%→1.64,99%→2.33,投资回报递减) / 差异化服务水平 / 安全库存≠需求预测(分摊=越畅销越缺货) / 促销系数须按ABC / 综合标准差×Z

## 230. 库存管理系列(06) ⭐⭐ 〔库存系统的交易逻辑·数据/货位/状态〕
- **核心论点**：库存系统**数据交互模块**：采购系统/WMS仓库管理系统/财务系统/PDA手持/调度中心(配送中心)。**库存流水**=专门记录每笔库存变动的数据表，须含：单据号及来源类型/变动方向(增加·扣减·锁定·释放)/变动数量/流水时间/变动信息(商品ID·SKU·货主·仓库·库位)；建议优先记流水，多模块须实时同步（否则超卖缺货难发现=好卖没货不好卖一堆）。**货位三层结构**：仓库(库房·最上层)→库区(分区·按门店/按大类)→库位(货位·具体位置编号如SP-12031)。**陈列的准确定义=解决商品在卖场里的"数量和位置"**。**库存六状态**：可销售库存(正常·库存≥0)；不可销售库存=残次/调配(仓间调拨占用)/在途(补货途中)/锁定(折扣促销买赠占用)/虚拟(无实物·负库存)/报损(盘损领用丢失)。库存状态管理=库存管理中**最难维护的部分**（动态性·非恒值），可用参数表+度量(ABC/可销天数)动态计算。**点睛：商品货位架构的基础=商品组织结构表**——经营不善的超市后台商品信息总乱（一个不足500㎡小标超用家乐福2000-3000小类架构用好几年）→**印证 Data Adapter「商品结构表=数字化地基」**。
- **关键词**：库存系统模块(采购/WMS/财务/PDA/配送) / 库存流水五要素 / 货位三层(仓库-库区-库位) / 陈列=解决数量+位置 / 库存六状态(可销/残次/调配/在途/锁定/虚拟/报损) / 货位架构基础=商品组织结构表

## 231. 库存管理系列(07) ⭐⭐ 〔复式库存/实际vs虚拟库存/库存价值两张皮〕
- **核心论点**：系统采**复式库存**：每笔变动同时记减(-)和增(+)，**所有库位库存加起来恒为零**（类似会计借贷）；出入库明细只记"变动"不记库位绝对数，某库位某品数量=目标库位数−源库位数。**实际库存(REAL STOCK)=库位现有实有；虚拟库存(VIRTUAL/FUTURE STOCK)=未来某点数量=现有实际+未来一段入库−未来一段出库（与时间相关，30天后≠5天后）**——**下订单要参考虚拟库存（到货时点的预估），不是现有库存**。OE系统：每仓分三库位(存货Stock/进货Input/出货Output)；商品三类型(可库存品Stockable/消耗品Consumable/服务Service)；库位可虚拟(盘点库位/供应商库位)、可串联成**库位链**（实现质检→入库、调拨二次确认等流程）。**库存价值"两张皮"**（财务与库管算的差很多·销售理解的入库被财务理解成出库）→OE每笔库存移动自动生成会计记录、强制一致→统一成一张皮（出库统一标-/贷方，入库+/借方，db_no列=记账方向）。**收束：零售超市库存管理一定程度上≈供应链管理**，目标=利润水平最大化，**利润水平=利润/(成本+费用)**。作者自承本系列缺系统连贯性，下节转"库存管理的核心"。
- **关键词**：复式库存(库位总和恒为零) / 实际库存vs虚拟库存(下单参考虚拟库存) / 库位链(质检/调拨二次确认) / 库存价值两张皮→自动会计统一 / 零售库存管理≈供应链管理(利润水平=利润/(成本+费用))

## 232. 库存管理系列(08) ⭐⭐⭐ 〔库存管理的核心/本质·哲学根〕
- **核心论点**：**库存管理的核心=库存控制**（站企业角度只一个问题），甚至决定企业生死存亡——作者20多年观察，**80%以上破产倒闭超市都与失败的库存管理有莫大关系，且多数企业还没认识到**。相同销售毛利下，大库存成本的甲企业不如小库存成本的乙企业（库存占资金限制周转→从"物"视角转"财务"视角）。库存认识演进：物品视角(越多越好)→连接润滑作用→"零库存"狂热→"合适的库存"。**库存控制两要求**：①认识观念转变（全员重视，非只财务考核+采购执行；靠供应链大流程的输入输出：预测/订单处理/销售计划/物料计划采购/配送）②合理控制策略（库位/满陈列量/采购周期/订货点/订货量）须配**流程决定组织/业务流程再造**（以顾客需求为中心、打破职能边界）。**直接需求=需求点直接消耗(满陈列量·可不设仓存=前流程) vs 间接需求=基于合适库存(设仓库=后流程)**。狭义库存控制=实物库存控制(盘点/保管/先进先出)；**广义库存控制=为达财务运营目标(现金流)，优化整个需求与供应链流程、合理设ERP策略，在保证销售前提下尽可能实现合理库存、降经营风险**。**🔥库存的本质（两个）**：①企业本身的**无能、无知加无奈**；②企业和相关个人为自己的无能而做出的**各种拍脑袋决策带来的后果**。零库存=皇帝的新装（从没真做到却天天奉为标准）。**库存关键核心矛盾=存货与"顾客动态需求"之间的天然矛盾**→好卖缺货/不好卖积压。"很多中国超市都有超过沃尔玛的决心，但从来没有了解顾客需求的耐心。"计划层级须逐年下沉：大类→中类→小类→动销单品。"一点点的好加起来是大大的好；一点点错误加起来是大大的错误。"促销批判：成功标志=促销前期与后期同期的销售/动销数差不多（不只看促销那几天），别把A类畅销品搞死、让顾客只等促销才买。
- **关键词**：库存控制=库存管理核心(80%倒闭超市与失败库存有关) / 库存本质两个(无能无知无奈+拍脑袋后果)·零库存=皇帝新装 / 存货vs顾客动态需求=天然矛盾(决心有余耐心不足) / 直接需求(满陈列量·前流程)vs间接需求(后流程) / 广义库存控制=优化整个需求供应链流程降现金流风险 / 计划颗粒度逐年下沉

## 233. 库存管理系列(09) ⭐⭐⭐ 〔库存三目标+三大评价指标·定量核心框架〕
- **核心论点**：**库存管理3目标**：合适的数量、尽量低的成本、优质的服务水平（三者相互制约：库存少→降成本但降服务；高库存→保服务但增成本资金压力→须平衡=合理库存）。成本看财务报表，服务水平看有货率(超市常用动销率)，数量最受关注。**三大评价指标**：**①存货周转天数**（首要指标，宜小不宜大=经营周期/资金周转效率；周期越长成本越高风险越大）。参考标准：生产型30-90天(放宽60-120，>120有问题)；**零售超市依商品类别——生鲜1-15天/食品15-60天**，须建自己超市的周转效率评价体系(以小类为单位)。**②存货的量**（不宜过大）：合理库存=某时期存货/同时期销量×%；越小效率越高，**恰当10%-30%、最好10%左右、不超50%**(>50%=过剩库存)；也不宜过低(<3%易缺断货降服务)。**③存货结构（可能是核心所在）**：量与周转天数看着合理而实际不合理(好卖没货/不好卖一堆)=没从结构分析。三维度：**维度一 整体结构**(商品存货/总存货)：优秀超市<50%、较好20-30%、不低于10%；理想订货系数Z=1.46→合理库存=销量1.46倍(满足95%)，促销节庆可突破50%(属概率非高库存理由)。**维度二 商品结构**(更高粒度·体现内部经营优秀程度)：库存是动态量、库存变化=销售变化的反应；**周转天数=当前库存数量/当前日均销量(移动平均)**；须分小类/中类/大类/部门(财务只需整体周转天数、品类采购经理需每小类每单品)；**库存三类型=过剩(大库存)/缺断货(畅销缺货)/滞销(要死不活)**；差异化经营反映在差异化库存(自采/进口/OEM比例)；基础=合理的商品结构表。**维度三 成本结构**(现金流财务管理需要)：采购资金分配/每月每大类采购金额计划/审签制度；毛利率逐级分解(公司20%→大类总和20%→中类/小类/单品)=成本结构；存货=现金，所有财务报表都是成本结构表。
- **关键词**：库存3目标(数量/成本/服务·相互制约=须平衡) / ①周转天数宜小(生鲜1-15天·食品15-60天·以小类建体系) / ②存货量=存货/销量(10-30%·最好10%·不超50%·不低于3%) / ③存货结构=核心(整体<50%/商品结构周转天数=库存÷移动平均日均销·三类型过剩缺货滞销/成本结构=毛利率逐级分解)

## 234. 库存管理系列(10) ⭐⭐ 〔采购计划=系统模型核心·S-DBR与MRP〕
- **核心论点**：采购计划=任何库存控制系统模型的核心，困难重重(算法/文化/责任/人性/换经理重做)→"谁都能做又谁都做不好"；**最简洁的才最实用**。采购计划核心目标=解决库存两大关键（**最优库存量+最大限度解决需求变动量/安全库存**），体现在前流程(入库)与后流程(出库调拨)；三步骤：满足稳定销量→逐步变化订货量→确保补货后存货不断优化。**S-DBR模型**(简化版DBR)：认为**系统瓶颈在外部=市场**(供过于求时代符合多数企业)；只有"出货"一种缓冲应对市场前所有"墨菲效应"；**基于时间管理**(无时间线的计划=鬼话)，但精确时间点(采购周期)因需求不确定实际无意义→**相对时间才可操作可量化**(按采购地/供应商类别固定统一订货周期，或用"顺序"替代：商品分主次/供应商排名定采购顺序)。**每个SKU可看做一个订单赋值(时间或顺序)**，最小颗粒度=计划体系最小单位(天/SKU条码)。D=节拍(顾客需求节拍非库管流程节拍)/B=缓冲(加工品/现有库存/订单库/在途/临期/领用·库存计划更多是订单库排序)/R=各种库提前期或关键点(订货点)。操作要点：控制少数关键流程+几个关键量化维度(动态订货点/销售量/预测量/订单周期)；别一开始追极致精确"先干起来再逐步改善"；**初心=保障准确交付时间(第一原则·放第二位的系统都耍流氓)**。**采购定量公式：采购定量 = 最优库存量×交货时间 + 需求存货量(安全库存)×需求时间**。S-DBR用环境两条件：①任意顺序订单处理不明显影响库存规模②最长作业链采购时间占采购提前期比应很小(实施前<10%、之后<20%)。**MRP(Material Requirement Planning物料需求计划)**：以生产计划为中心联结产供销，零售应用=按到货时期倒排、按提前期下达计划先后；基本逻辑①独立需求定主采购计划MPS(卖什么/多少/何时要货)②要货数据(门店要货清单/新品/品牌增减)③现有库存信息(库位维护/盘点)→算出"我需要什么/已有什么/还缺什么/还应做什么(何时下单)"。订货模型列：要货数量(需要/还缺)+当前库存(已有)+**DMS日均销量(动态需求量)**+建议定量/实际确认定量(优化修订)。
- **关键词**：采购计划=系统核心(最简洁=最实用) / S-DBR(市场=瓶颈·相对时间>精确时间·SKU=订单赋值·D节拍/B缓冲/R提前期) / 采购定量=最优库存量×交货时间+安全库存×需求时间 / 初心=准确交付时间(第一原则) / MRP(我需要/已有/还缺/还应做·DMS日均销)

- **⭐⭐⭐ 安全库存定量底座**：服务水平=有货率·Z值表(无安全库存50%·95%→1.64·99%→2.33·投资回报递减)+差异化服务水平+**安全库存绝不可分摊进需求预测(分摊→越畅销越缺货)**+促销系数须按ABC+综合标准差×Z → [[数量管理]]（新建/深化「安全库存与服务水平」定量块·承批38订货公式 Q=d·LT+Zσ−I）。**这是数量管理从定性走向定量的关键一跃。**
- **⭐⭐⭐ 库存三大评价指标框架**：周转天数(宜小·生鲜1-15天/食品15-60天·以小类建体系)+存货量(10-30%最好10%不超50%)+**存货结构三维度(整体<50% / 商品结构=周转天数÷移动平均日均销+三类型过剩缺货滞销 / 成本结构=毛利率逐级分解)** → [[数量管理]]核心定量框架（可作 report-export 诊断卡的库存健康度量）。
- **⭐⭐⭐ 库存哲学根（接失真库哲学根）**：库存控制=库存管理核心(80%倒闭超市与失败库存有关)+**库存本质两个(无能无知无奈/拍脑袋后果)**+存货与顾客动态需求的天然矛盾(决心有余耐心不足)+零库存=皇帝新装 → [[数量管理]]总则/与 [[商品力失真库]] 哲学根呼应（"好卖缺货不好卖积压"=数量维失真的本质）。
- **⭐⭐ 库存系统逻辑底座**：复式库存(库位总和恒为零)+实际vs虚拟库存(下单参考虚拟库存)+库存六状态(可销/残次/调配/在途/锁定/虚拟/报损)+**商品货位架构基础=商品组织结构表** → [[数据与系统能力底座]]（印证 Data Adapter「商品结构表=数字化地基」·与批23商品信息表呼应）。
- **⭐⭐ 满陈列量=直接需求/前流程 vs 合适库存=间接需求/后流程 + 广义库存控制=优化整个需求供应链流程** → [[数量管理]]/[[分层到陈列一致性模型_v1.0]]（呼应批38「内仓零库存配置法」·满陈列量=货架补货的可得性抓手）。
- **⭐⭐ 采购计划工具**：S-DBR(市场=瓶颈·相对时间·SKU=订单赋值·**采购定量=最优库存量×交货时间+安全库存×需求时间**)+MRP(我需要/已有/还缺/还应做·DMS日均销) → [[数量管理]]（采购计划定量工具·与§6.5 CPFR协同补货呼应）。
