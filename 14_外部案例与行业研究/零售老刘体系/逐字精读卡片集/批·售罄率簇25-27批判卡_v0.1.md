# 逐字精读卡片集 · 售罄率簇 25/26/27 批判卡

> 📌 **本批 = 《分析建模》25-27**（2022-11/12·售罄率专题·细读）。老刘 3 篇把售罄率从"服装批次思维"讲到"生鲜当日节奏"，中间夹带三线库存 + EOQ。**批判层焦点**：这些公式的数据门槛与社区生鲜的适用边界——哪几个能直接搬、哪几个是"知道就好、别硬建模"。

## 1. 聊聊售罄率（25）★核心公式payload
- **核心论点**：售罄率本是服装/鞋/精品（单品少、季节强）的指标，超市少提；但**生鲜日配高频品其实最该用**。关键=必须带时间维度，无时间的售罄率没意义。
  - **4 公式**：① 进货成本/销售额；② **（时段内）销量/（同时段）订货量×100% ← ★生鲜最可落地**；③ **售罄率=毛利率/(1-费用率)=盈亏平衡售罄率**（例 35%/(1-15%)=**46.7%**=成本平衡点，还需减折扣/损耗/退货率）；④ 动销单品数/总进货单品数。
  - **⭐反直觉铁点（牛肉例）**：18kg 牛肉早市须卖 70%、晚市 30%；早市完不成 70%→晚市促销；早市提前到 95%→**订货量不够、缺失销售机会**。**当日完成率 100% ≠ 达标，很可能订货不足**，照此订货会系统性丢销售。
  - **畅销品判据**：统一设售罄率=50%，相同周期销量越多/越早达 50% 越畅销（早10点卖一半 vs 下午6点卖一半，含义相反）。
- **关键词**：售罄率4公式 / **公式2生鲜落地·公式3盈亏平衡服装批次** / 当日100%≠好=订货不足 / 售罄率必带时间维度 / 动销率=某意义的单品售罄率
- **🔴批判·失效条件**：公式3（盈亏平衡 46.7%）是**批次/长周转品语境，套到当日清零的生鲜=严重误判**（生鲜要求当日 95%+）。→ 张力 T2。社区店无 POS/称重手记 → 用"进货重量-剩货"倒推，精度低但可用。

## 2. 售罄率建模运用01（26）三线库存
- **核心论点**：库存不是浪费（跳楼老板掉库存堆上没死的段子）——库存是应对供给/需求不确定性的缓冲，合理库存才对。任何库存问题都涉及售罄率（到哪个售罄率比例该有多少定量/销量/剩余）。
  - **三种库存量**：安全库存量 / 管道库存量 / 最佳采购量。**★管道库存=提前期×日均需求**（日均100×5天=500）=极简可搬。
  - **三条库存线**：安全库存线 / 再订购点线 / 库存上下限。**最大库存=再订购点+最佳定量**。
- **关键词**：库存=不确定性缓冲(7项价值) / **管道库存=提前期×日均** / 三线库存 / 最大库存=再订购点+最佳定量 / 库存问题1+1>2(倍增或倍降)
- **🔴批判**：管道库存公式对社区店可直接用（估到货前留量）；但安全库存需概率统计，社区店退化=经验值≈当日周转量。生鲜当日清零→"再订购点/最大库存"意义弱（呼应 KB-KPI 命题6 超短保安全库存趋近0）。

## 3. 售罄率建模运用02（27）EOQ + 需求管理
- **核心论点**：三线库存做库存优化的触发依据——低于安全库存→紧急处理；低于再订购点→做采购订单；高于最大库存→清理库存（多数店没设三线→紧急处理退化成普通订单、清库存形同虚设）。**需求管理取代需求预测**（预测总是错的=静态假设，要即时动态修正偏差）。
  - **EOQ 经济订货量**：总成本=S+V×Q+D×i×T/2 → **EOQ=√(2DS/v·i)**。例 D=3650/S=5/v=10/i=20%→EOQ=135（取50倍数≈150）。批量处理=准备部分+运行部分；批量↑摊薄单位成本 vs 抬高库存持有成本，找平衡点。
- **关键词**：三线触发(紧急/订单/清库) / **需求管理>需求预测** / EOQ=√(2DS/vi) / 批量平衡点 / 安全库存是订货系统最技术的一环
- **🔴批判·老刘自评**：**"EOQ 很少用于建模，只是理论/粗估工具"**——社区店批量单品不多，用 Excel 经验估算即可，**全套 EOQ 建模 ROI 为负**。→ 命题 M7-2 / 遗漏 O4（EOQ 仅存参考储备）。
