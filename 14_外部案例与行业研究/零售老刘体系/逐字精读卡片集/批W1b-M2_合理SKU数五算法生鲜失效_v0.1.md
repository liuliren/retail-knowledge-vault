# 二遍批判卡 · W1b-M2 合理SKU数五算法的社区生鲜失效

> 与 C1 SKU数约束专题(2026-06-19·active·high)差分:C1 §10/§11 已把五算法公式**全部落表**;本卡不重列公式,只补 C1 完全没有的**失效条件层**——C1 把大卖场公式当"可校准经验值"直接给了花厅坊,这是假冻结(吸收态非评判态)的典型坑。

## 五算法(176/199)与生鲜适用性裁决
老刘合理SKU数五算法:物理空间法 / 经济承载法 / 需求覆盖法 / 边际品效法 / 综合优化法(ABC+Solver)。**共同预设**:单 SKU 有固定陈列面宽、固定条码、可归集销售/成本。此预设在生鲜散装区**从根不成立**。

| 算法 | 公式(见C1) | 生鲜裁决 |
|---|---|---|
| 物理空间法 | (陈列米数×层数×陈列系数)/单品面宽 | ❌ 生鲜叶菜/称重肉禽无固定面宽/条码,"一堆青菜=几个SKU?"无解→分母立不住。仅包装标品用 |
| 理论最大SKU | 货位数×30-50单品/货位 | ❌ 锚点 300货位×50=**15000 SKU**是大卖场量级;社区500-2000㎡总SKU 2000-8000,差一个数量级,不可作容量锚 |
| 经济承载法 | (月销×周转天)/(30×单品成本) | ⚠️ 生鲜日进货/成本日波动/周转1-3天→公式极敏感失真。仅干货标品用 |
| 需求覆盖法 | 总SKU目标×(权重/Σ权重);权重=销售占比×深度系数 | ✅ 可用(含生鲜);深度系数(目标0.8-1.5/便利0.3-0.8)是大卖场值,社区窄客群取更低+实测 |
| 边际品效法 | 品效=总销售/SKU数;边际品效<平均→精简 | ⚠️ 生鲜须改**净品效**(下条);散装难算到单SKU级 |

## 核心批判1:生鲜品效必须扣损耗成"净品效"
- 老刘品效/边际品效拐点是判超配的核心判据,但公式是**毛销售口径**。生鲜损耗常 8-20%,一个"毛品效达标"的生鲜 SKU 扣损耗后可能已亏。
- **裁决**:生鲜 SKU 判据改 **净品效 =(销售额−损耗成本)/SKU数**;边际净品效<平均净品效才判超配可砍。→ 可补进 MD-001 CRITIC补充命题1 的淘汰触发信号(现有仅动销率/价格带重叠/损耗3信号)。
- **失效条件**:低损耗标品(干货/包装)毛品效≈净品效,老刘原式直接可用。

## 核心批判2:花厅坊 SKU 测算必须"分区适用"
- 包装休食/标品干货区 → 全五法可用(接 C1 判断链 §12)。
- **生鲜区 → 弃物理/经济承载/纯毛品效三法**,只用①需求覆盖(角色×深度系数)+②净品效。别对生鲜套物理上限公式硬算 SKU 上限。
- **最轻量替代**(免 Solver):五法计算值离群对比——只算能算的 2-3 法,哪个明显偏高即超配(原文176例:物理法320远高其他4法→该小类SKU偏高)。

## 元判断(失效边界的方向性)
散装非标这道失效边界随生鲜**标准化预包装/净菜化**推进会收窄——老刘五法本质是"预包装化程度的函数";花厅坊生鲜净菜化越深,老刘模型适用面越大。
