# 库存管理

> 库存是产供销系统性失衡的结果，不是采购或销售的单方问题；精准管理的核心是用需求标准差（非历史销量）驱动安全库存，用库存系数（IR）而非周转率驱动订货决策，并在每个流程环节追踪净需求缺口——无法实现精细化时，15.87% 黄金分割点提供容错边界。

---

## 六哥当前可用场景

> 场景：花厅坊库存优化——生鲜品周转天数目标是多少？安全库存怎么设？损耗控制抓哪些杠杆？

**具体用法（3条可执行动作）**：

1. **设定生鲜品周转目标**：用 `IR = 12×(1-毛利率)/年度周转率` 先算出当前库存系数基准；生鲜毛利率约25-35%，对应IR约在0.5-0.8区间。若花厅坊某品类IR持续>1，直接判定为高库存预警，优先缩短采购周期或削减SKU深度。

2. **诊断缺货根因**：不要只看最终库存数字，逐环节追问"漏斗"：门店员工报货了多少→品类经理实际下单多少→收货多少→入库多少。每个环节的差值即净需求缺口，找到最大漏点集中整改，而非全链条同时优化。

3. **设安全库存容错线**：若暂时无法搭建完整MRP系统，先用15%作为运营红线——缺货率≤15%、库存误差≤15%、费用率≤15%；超线即触发复盘。同时，有条件时逐步收集真实需求数据（注意区分销量与需求，缺货期间的实际需求要单独估算补全）。

## 核心命题（来自零售老刘实践体系）

### 1. 库存系数（IR）优于库存周转率作为订货管理的决策基准

库存系数 `IR = 库存/销售额`，直接驱动订货规划（`库存目标 = 销售额 × IR`）；年化库存周转率是财务复盘指标，不适合作为日常订货参考。两者通过公式族关联：`IR = 12×(1-毛利率)/年度周转率`，可双向换算。

关键原则：**IR的稳定性比绝对值更重要**。若销售额增长20%，库存增长应同步但不超比例；若库存增速高于销售增速，说明积压正在形成，需立即介入。

在社区生鲜中，生鲜类IR应显著低于常温品（周期短、损耗敏感），差异化设定各品类IR目标是基础动作，不能用全店平均数掩盖结构问题。

### 2. 销量数据被缺货和限购污染，需求管理优先于预算管理

历史销量≠真实需求。缺货、限购导致记录的销量被压低，隐藏了真实客户需求。例如某单品缺货时销100件，若有货实际需150件——用100件计算出的安全库存公式 `SS = Z×σ` 结果将严重偏低，畅销品由此陷入"越缺货→历史数据越低→安全库存设得越低→越缺货"的恶性循环。

根本性转变：从"按历史销量预算备货"转向"先估算真实需求、再倒推备货量"。实操中须在缺货期间单独记录"本可多卖的需求量"，才能还原真实σ值。

### 3. 采购提前期（VP）与需求波动的复合效应是精准安全库存的关键

安全库存有两层计算逻辑：

- **第一层（纯需求波动）**：`SS = Z × σ`，Z值由目标满足率查正态分布表确定（95%满足率对应Z=1.645）。
- **第二层（加入采购周期）**：`SS = Z × σ × √(VP)`，VP为采购提前期天数。

核心洞察：**越畅销的品类，下一采购周期内需求变化越剧烈，缺货风险越高**。统一采购周期无法适配不同需求波动特征，应按品类差异化设定VP，而非所有品类统一处理。

在花厅坊生鲜场景：高频畅销品（如蔬菜、牛奶）应设极短VP并高频补货；长尾品可适当延长VP降低物流频次，但需相应提高安全库存缓冲。

### 4. 订货流程是"漏斗"结构，净需求追踪是发现失效点的核心工具

文章29揭示的漏斗模型：

```
顾客需求 20
 → 员工报货 15（欠缺 5）
 → 品类经理下单 10（欠缺 5）
 → 收货 5（欠缺 5）
 → 入库 0（欠缺 5）
= 累计缺失 20，等于全部需求未被满足
```

仅追踪最终库存是无效的——库存显示为0，根因可能在报货环节就已失真。MRP核心逻辑是将净需求分为两部分单独管理：`净需求 = 常规周期销量 + 安全库存`，两者分开计算、分开监控，才能精准定位缺口来源。

实操建议：从最容易量化的"收货欠缺"开始记录（验收数 vs 订单数的差值），积累3-4周数据后再向上游延伸到报货、下单环节。

### 5. 高库存是产供销系统性失效，不是采购或销售的单方问题

库存有三层含义，诊断前必须明确当前过高的是哪一层：

1. **结构性存货**：供应链布局决定（如多仓多配），降低须调整网络设计。
2. **运作所需存货**：节点间周期压缩（如从7天补货改为3天），须投资物流能力。
3. **应对波动存货**：需求/供应/质量风险缓冲，降低须提升预测精度或供应商稳定性。

**长鞭效应的本质**：产供销各自为政，局部最优（生产追求批量效率、销售只看销售目标、采购追求采购成本）叠加导致全局库存高企。根本解法是信息共享与端对端流程协同——在花厅坊等门店层面，对应的是"销售数据实时对采购可见"而非事后对账。

### 6. 黄金分割点15.87%为无法精细化计算的场景提供容错边界

借鉴学习心理学最佳新旧信息比例（15.87%），零售老刘将其引入库存管理：

- 缺货率 ≤ 15%：可接受
- 费用率 ≤ 15%：良性
- 库存误差 ≤ 15%：合理

这不是精确管理的替代品，而是**承认现实约束的容错框架**——大多数门店无法搭建完整MRP系统时，以15%作为被允许的模糊范围，超线即预警，优先整改超线环节。适用于初期推行库存标准化的门店（如花厅坊起步阶段）。

## 关键框架与模型

### 框架一：库存系数公式族（两组度量体系）

```
体系A（适合日常订货管理）：
  IR（库存系数）= 库存 / 销售额
  库存目标      = 销售额 × IR
  周转天数      = 365 / 年化周转次数

体系B（适合财务复盘）：
  年化周转率    = 12×(1-毛利率) / IR
  周转天数      = IR × 30（月度）

换算公式：IR = 12×(1-毛利率) / 年度周转率
```

不同分析场景选用不同体系；日常订货用IR，年度财务用年化周转率，不混用。

### 框架二：安全库存双层计算

```
第一层（仅需求波动）：
  SS = Z × σ（需求标准差，非历史销量标准差）
  Z值：满足率90%→1.282；95%→1.645；99%→2.326

第二层（加入采购提前期）：
  SS = Z × σ × √(VP)
  VP：采购提前期天数

注意：σ必须来自真实需求数据，历史销量被缺货污染时需修正
```

### 框架三：MRP净需求分解（漏斗模型）

```
净需求 = 常规量（采购周期内正常销售） + 安全库存（波动缓冲）

诊断漏斗（从下游往上游追）：
  入库量 → 收货量 → 采购订单量 → 报货量 → 真实需求
  每相邻两层的差值 = 该环节净需求欠缺量
```

### 框架四：库存三层分类（高库存诊断用）

| 层级 | 成因 | 降库存解法 |
|---|---|---|
| 结构性存货 | 供应链网络布局 | 调整仓网设计 |
| 运作所需存货 | 节点间周期长 | 缩短补货周期/提升物流频次 |
| 应对波动存货 | 需求/供应/质量风险 | 提升预测精度或供应商稳定性 |

### 框架五：黄金分割点15.87%容错框架

| 指标 | 红线 | 触发动作 |
|---|---|---|
| 缺货率 | ≤15% | 超线→排查漏斗哪环欠缺最大 |
| 费用率 | ≤15% | 超线→审查损耗、报废来源 |
| 库存误差 | ≤15% | 超线→核查数据采集准确性 |

## 实践要点（零售老刘反复强调的操作细节）

- **避免点库存陷阱**：盘点某一时刻的库存量容易造假且无法反映需求动态；应用动态库存系数（一段时期内的平均值）代替点库存快照。
- **需求污染识别**：缺货期间的历史销量必须标记并单独修正，否则所有后续建模的基础均失真。
- **EOQ与安全库存的张力**：经济订货量（EOQ）追求采购成本最小化，安全库存追求服务满足率最大化——两者目标往往冲突。实操中先满足服务水平（Z值）设定安全库存下限，再在此约束下优化订货批量。
- **σ的计算来源**：安全库存公式中的σ必须是需求标准差（包含缺货修正后的真实需求），而非直接用POS销量标准差，两者差异在畅销品上可能超过30%。
- **采购周期差异化**：高频需求品类用短VP+高安全库存；低频长尾品用长VP+低安全库存；不要全店统一采购节奏。
- **长鞭效应预防**：在条件有限的门店，至少做到销售日数据对采购可见（即便只是一张每日销售简报），而非按周对账——信息滞后是长鞭效应的主要燃料。

## 与其他模块的关联

- **→ KB-RETAIL-PRODUCT-001（商品力）**：库存管理的前提是清晰的商品结构——高动销品（A类）设低IR高补货频次，低动销品（C类）设高IR低频次；商品力决定IR目标值的差异化区间。
- **→ 品类管理（KB-CAT-ARCH-001）**：全品类架构中各品类的角色定位（主销/利润/形象/长尾）直接影响安全库存策略——形象品宁可高库存保陈列，利润品追求IR精准。
- **→ 损耗管理**：生鲜品的库存管理与损耗强耦合；IR设得过高等价于允许更高损耗，安全库存的"缓冲"在生鲜品里部分会转化为报废成本，须在服务水平与损耗之间找均衡点。
- **→ 供应商管理**：采购提前期VP是安全库存的核心变量；供应商准时交付率越高，VP方差越小，安全库存可设得更低。供应商稳定性是降库存的重要杠杆。
- **→ 数据系统（ERP/WMS）**：漏斗模型的每个环节追踪依赖数据系统的完整性；在系统不完善时，先手工建立"报货欠缺记录本"（最简版漏斗追踪），积累基础数据后再系统化。

## 矛盾与开放问题（待深入研究）

1. **精确计算 vs 容错模糊**：零售老刘既强调精确IR建模（DAX/Excel），又提出15%容错框架。两者如何衔接？推测路径是：初期用15%红线建立感知→积累数据后逐步引入IR精确计算→成熟阶段安全库存精确建模。但过渡节点的判断标准尚未明确。

2. **历史数据污染 vs 数据驱动**：文章28-30承认历史销量被缺货污染不可信，但文章19的建模完全基于出入库历史明细。如何从被污染的数据中提取可信需求信号？缺货期间真实需求的补全方法（插值/替代品销量推算/顾客调研）在原文中未展开，是该体系的未解问题。

3. **理想MRP流程 vs 组织现实**：文章29的MRP算法假设四环（报货、订单、收货、入库）数据闭合，但文章"不成立三角"揭示现实中产供销数据隔离。在组织能力有限的小型连锁门店中，哪个环节是最小可行的数据收集起点？

4. **零库存理想 vs 安全库存必要**："内仓零库存"理想态（即时配送、无冗余存货）与安全库存理论直接矛盾。在社区生鲜场景，是否存在某类商品可以真正实现零库存（如高频配送的蔬菜），而另一类商品必须维持缓冲库存（如节假日备货品）？区分标准尚需实践验证。
