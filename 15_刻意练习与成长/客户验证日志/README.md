# 客户验证日志

## 一、目录定位

本子目录是 [[CLAUDE.md]] §21「客户验证 24h 硬约束」的**落地容器**。每一次真实客户接触（销售对话 / 首访 / 复盘会 / 老板访谈 / 阶段门评审），都必须在 24 小时内产出一份本目录格式的客户验证日志。

它服务于：

- **反"自洽于自己"**：vault 防止变成自嗨知识库的核心防火墙——无客户验证日志的方法论 = 未验证
- **方法论实战登记**：每次客户接触验证 / 推翻 / 修正了哪些方法论 / SOP / 模板，反向回写到对应文件的"实战登记"段
- **[[Vault健康仪表]] 维度 #2「客户验证度」的数据来源**——目标 ≥40%
- **战役级证据链**：与 `16_客户与战役档案/` 互补——16_ 记"战役怎么走"，本目录记"我们的方法论是不是真的有用"

> 设计源文档：[[2026-05-04_知识库重构方案_v0.2_完整作战方案]] §6.1 / §6.2 / §4.2「实战登记」段。
>
> Schema 锚点：[[CLAUDE.md]] §21 客户验证 24h 硬约束（Phase 0 D1 落地）。

## 二、24h 硬约束时间线（CLAUDE.md §21）

| 时点      | 必做动作                                                        | 落地位置 |
|-----------|-----------------------------------------------------------------|----------|
| **T+0**   | 客户接触发生（销售对话 / 首访 / 复盘会 / 阶段门 / 老板访谈）         | 现场     |
| **T+1h**  | 现场记一句备忘（语音转文字 / 微信 / 便签都可）                   | 临时存 90_ 或本地，T+24h 整合到日志 |
| **T+24h** | **必须**完成验证日志（6 段固定结构）                             | **本目录 `<月份>/<文件>.md`** |
| **T+48h** | **必须**完成对应方法论文件的"实战登记"段反向回写                  | 对应方法论文件（01_ 等）|
| **T+1w**  | 汇入 [[战役指挥板]] 更新 + [[Vault健康仪表]] 客户验证度更新       | 仪表层   |

> **超 T+24h 未完成日志**：标记本次接触为"未验证沉淀"，触发 [[Vault健康仪表]] 维度 #2 黄灯；连续 2 次 = 红灯。
>
> **超 T+48h 未回写**：方法论失去"实战登记"机会，触发"再写也不算验证"的反幻觉规则——参 [[CLAUDE.md]] §13.10。

## 四、6 段固定结构（参 v0.2 §6.2）

每份日志**必须**含且仅含以下 6 段（顺序不可调）：

1. **客户场景**（去标识化：地区+业态+规模+岗位）
2. **用了哪些方法论 / SOP / 模板**（双链 [[]] 列出）
3. **哪些被验证**（客户认可 / 数字支持 / 现场看到效果）
4. **哪些被推翻**（客户不认 / 数字不支持 / 现场推翻）
5. **哪些需要修正**（部分认 / 需补充 / 上下文依赖）
6. **后续动作**（要修哪个文件、改哪条规则）

> 详见 [[15_刻意练习与成长/客户验证日志/_模板/客户验证日志模板|客户验证日志模板]]。

| `01_科学零售方法论/`            | **上下游** | 01_ 是方法论主定义；本目录是 01_ 的实战检验来源（每条 v0.x → v1.0 必须有本目录证据）|
| `16_/<战役>/03_接触日志/`       | **互补**   | 16_/03_接触日志 是战役内部的接触记录（"今天客户说了什么"）；本目录是跨战役的方法论验证（"哪条规则被推翻"）|

## 七、反假忙碌纪律（参 [[三回归仪表]] §六）

1. **不可空填**：6 段中"哪些被验证"必须有具体证据（"客户说了"+"数字 X% 支持"+"现场看到 Y"），不允许"客户表示认可"这种空话
2. **不可代填**：LLM 不能替你填日志——LLM 没去现场。LLM 可帮你结构化整理、检查双链、找方法论引用，但**6 段内容必须真人填写**
3. **不抹历史**：被推翻的方法论留痕——同一规则被多次推翻就触发**强制升 v0.2**
4. **不包装**：客户没说的话不要写。客户说"再考虑考虑"≠ 客户认可
5. **不混签约**：销售对话 → 写本目录是为了验证销售包方法论；签约 / 范围 → 进 `16_客户与战役档案/<客户代号>/`

## 八、连续 2 周客户接触 0 = 升级假忙碌（红线）

> 来源：[[CLAUDE.md]] §20 三回归 + [[三回归仪表]] §四假忙碌扫描。

- 连续 1 周客户接触 0：[[Vault健康仪表]] 维度 #3 红灯，仪式必谈
- 连续 2 周客户接触 0：升级**假忙碌**，**必须停 vault 推进**，直到产生客户接触
- 红线动作：当周仪式自动触发"为什么 2 周没动客户？哪个流程卡住了？"复盘
