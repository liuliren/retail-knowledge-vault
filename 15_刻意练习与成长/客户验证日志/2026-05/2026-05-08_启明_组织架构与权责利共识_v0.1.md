## 3. 哪些被验证 ✅

### 3.1 三力骨架的"组织力"维度可实战

之前 vault 里 [[组织力总纲]] 只有方法论框架，没有真实战役实证。本次 5/8 会议把 4 件事（商品组织 / 人员调配 / 员工培训 / 制度流程）用于花厅坊真实组织架构 → 启明高度认同 + 主动 take ownership。

→ 实战首次验证三力骨架的"组织力"端可以独立产出价值，不必等"商品力做完才动组织"。

### 3.2 "专人专岗 / 多职位捆绑"是中小连锁普遍痛点

启明描述的现状：佳妮（财务+休食采购）/ 文员（文员+饮料低温采购+营运）/ 高彬（副店长+散称采购）/ 启明本人（老板+水果采购）—— **没有任何核心岗位是"专职单一"的**。

→ 验证 vault 里"专人专岗"前提的普遍性：凡是单店 / 早期连锁，必然存在多职位捆绑 + 优先级混乱 + 督导真空。这是"组织力诊断"的入口指标。

### 3.3 "向上沟通+articulate" 不是"夺权博弈"

PDF 里启明谈到要"向老板娘（娜姐）沟通权力分配 + 向老板（陈海鹏）争取赏罚权 / 利益分配权" → 用户 5/9 明确订正：**这是 articulate + 同步**，不是 confrontation。

→ 验证"协同信任默认假设"在中国家族企业咨询中的基础地位（已沉淀 [[feedback_客户家族关系协同信任]]）。

### 3.4 启明的三大金句（可入金句库）

1. **"我们要以终为始，一线更重要，他们才是真正接触客户和商品的人。"** — 一线导向 / Apple "用户中心"中国版
2. **"先学会收上来，再放下去，明确各个权限，让权力在合理的范围内运行。"** — IBM 流程治理"先收权再放权"原则的本土实践
3. **"任何一个生意里边，最关键就是关键人，把关键人抓到，这个组织就稳定。"** — McKinsey "Key Person" 原则的店长视角表达

## 4. 哪些被推翻 ❌

### 4.1 ❌ 推翻：5/8 会议是"启明个人战略会"

我（vault）连续两次（5/9 上午）把会议性质偏读成"启明在家族里把权拿回来"的个人战略会 → **用户两次现场纠偏**：会议核心动机是**标杆项目落地** + **整个门店家族发展**，启明个人能力升级是关联收益不是动机。

**沉淀**：[[feedback_客户家族关系协同信任]]（vault 反幻觉框架性硬约束 / 立即生效）。

### 4.2 ❌ 推翻：之前推断"启明 = 店长股东"

之前推断（基于用户最早一句模糊表达）：启明 = 店长 + 股东。
订正：启明 = **老板 + 小股东 + 水果采购**。门店另有"执行店长"岗（非家族 / 老经验 / 待补姓名）。

### 4.3 ❌ 推翻：之前推断"高斌 / 张丽 / 文妍 / 文元"是真实人名

订正：**OCR 识别错误**：
- 高斌 / 高冰 → **高彬**（副店长 + 散称采购）
- 张丽 → **张力**（水果+蔬菜组主管）
- 文妍 / 文元 → **文员**（同一岗位名 / 单人 / 兼饮料低温采购+营运）

### 4.4 ❌ 推翻：沃尔玛类比中的"夺权戏码"

我（vault）类比沃尔玛早期家族阶段时，**错误地保留了 Sam → Rob 接班的"夺权张力"** → 套到启明身上 → "二代崛起" → 用户纠偏后：保留结构同构（家族联合+多店分管+二代参股+集团整合阶段），删除戏剧化夺权情节。
