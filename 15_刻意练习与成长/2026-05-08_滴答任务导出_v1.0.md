# 📋 5 个月跑道任务导出（滴答兼容）

> **本文件**：vault → 滴答（TickTick）任务导入辅助
> **2 种格式**：Markdown 复制粘贴（A）+ CSV 批量导入（B）
> **任务数**：~50 条 / 4 主线 + 周仪式 + 健康

---

## 🎯 滴答导入操作（3 选 1）

### 方案 A：复制粘贴 Markdown（最快 / 5 分钟）

1. 打开滴答清单 PC 版 / 或 web.dida365.com
2. 创建清单"5 个月跑道（5/8-10/8）"
3. 复制下方 §1-§5 的 markdown 文本
4. 粘贴进滴答的"添加任务"窗口（滴答会自动识别 `- [ ]` 为任务）

### 方案 B：CSV 批量导入（最规范）

1. 滴答清单 → 设置 → 导入 → 从 CSV
2. 复制 §6 CSV 内容 → 保存为 `tasks.csv`（UTF-8 BOM 编码）
3. 上传 → 字段映射（标题 / 优先级 / 截止 / 标签）→ 导入

### 方案 C：URL Scheme 单条添加（命令行）

```bash
# Mac 用户可用 URL scheme 单条添加（不批量）
open "ticktick://add?title=任务名&content=备注&priority=5&due=2026-05-14"
```

## §9. recurring 任务设置（滴答 重复规则）

- `recurring-daily`：每日（睡眠）
- `recurring-weekly`：每周（仪式 / 运动 / 风险扫）
- `recurring-monthly`：每月（月度战略会）

滴答内手动设置：
- 任务 → 编辑 → 重复 → 选择"每日 / 每周 / 每月"
