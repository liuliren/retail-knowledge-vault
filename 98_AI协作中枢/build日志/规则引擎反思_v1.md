## 核心教训（结论先行）

**v3 加了更多关键词，反而比 v2 更差**——命中 SKU ↓21%，待复核 ↑126%。根因：长关键词优先逻辑在排除规则加入具体品牌词后产生反噬，导致大量原本可命中的 SKU 因「同时命中目标+排除」而进入待复核桶。

**教训**：规则引擎不是「关键词越多越好」，每次扩充都要做量化对比测试；精准率与覆盖率之间有 trade-off。

## 根因分析

**表面原因**：v3 新增 1281 个关键词（大量为排除规则加具体品牌词，如「海底捞酸菜鱼」「益力多」）

**深层原因**：匹配引擎使用「长关键词优先」逻辑：
```
对每条SKU扫描所有规则：
  ├─ 仅命中目标 → 挂码（命中）
  ├─ 仅命中排除 → 范围外
  ├─ 同时命中目标+排除 → 待复核
  └─ 都未命中 → 待复核
```
排除规则加入长品牌词后，原本只命中目标规则的 SKU 现在同时命中了排除规则 → 大量 SKU 变成「待复核」。
