> ⚠ **口径澄清（T-02 不是毛利率模型）**：T-02 的 **甲/乙/丙 按毛利额累计贡献分档**，**不等同于毛利率高/中/低**。「甲」= 毛利贡献高（毛利额贡献排前），不代表单品毛利率高。若需判断**毛利率异常**，应通过**毛利率字段**或**负毛利/清仓异常字段**另行识别，不可用甲/乙/丙代替。

#### 3.1.1 C 行毛利率复核闸 · 利润金矿识别（方案 B·六哥签字 2026-06-23）

> 来源：full dry-run 实测（`花厅坊90天full_dryrun审阅_CODEX-Full-DryRun-Execute-001_v0.1`(已删·见SignoffLedger)）暴露——毛利额贡献维下 C 行全丙、利润品/长尾待裁决=0。评审 [[ABC九宫格利润金矿口径方法论评审_P0-ABC-Goldmine-Method-Review-001_v0.1]] 采纳**方案 B**。
> **本补丁=方法规则;代码同步另开 CODEX-ABC-Goldmine-Rule-Fix-001;同步重跑 dry-run 前不进真实写回。**

**① 毛利额贡献 ≠ 毛利率（关键澄清）**：甲/乙/丙 = **毛利额贡献**分档（谁贡献绝对毛利额），**不表示毛利率高低**。毛利额 ≈ 销售额×毛利率，与销额强相关 → **低销 SKU 几乎不可能进毛利额贡献甲/乙** → 「低销高毛利率小众金矿」**毛利额贡献维识别不出**。

**② 利润金矿识别补充**：「利润金矿」**不再仅由 C+甲/C+乙 识别**（该路径在毛利额贡献维下结构性≈0）。C 行商品须再过**毛利率复核闸**。

**③ C 行毛利率复核闸（新增规则）**：
> 若 SKU **销售额贡献=C**，且 **毛利率达毛利率阈值**，同时**排除 缺货 / 新品保护 / 促销异常 / 数据异常**，则标记 **「低销高毛利率·金矿候选」**。
- **毛利率阈值**：优先 **小类 P75**；**小类有效样本 < 20 时降级 全店 P75**。

**④ 金矿候选 = 复核字段，非最终裁决**：`goldmine_candidate` 只表示「值得复核的低销高毛利率商品」，**不直接等同保留/扩面/重点经营**。最终动作仍结合库存/库龄/缺货/新品/陈列/促销异常 + 门店判断，进误判防线后定「保留/缩面/调陈列/淘汰」。

**⑤ 新增字段定义（供 CODEX-ABC-Goldmine-Rule-Fix-001 落地）**：
| 字段 | 含义 | 取值 | 说明 |
|---|---|---|---|
| `gross_margin_rate_tier` | 毛利率分位层级 | high/mid/low/unavailable | 优先小类分位，样本<20 用全店分位 |
| `goldmine_candidate` | 是否金矿候选 | true/false | 仅 C 行参与判断 |
| `goldmine_reason` | 金矿候选原因 | 文本 | 说明触发阈值 + 排除项状态 |

口径：`gross_margin_rate_tier=high` ⇔ 毛利率≥小类 P75（小类样本<20 则≥全店 P75）；`goldmine_candidate=true` ⇔ 销额贡献=C 且 tier=high 且 非缺货 且 非新品保护 且 非促销异常 且 非数据异常。

**⑥ 一线解释**：有些商品卖得不多，但每卖一件都比较赚钱——不能只因销售额低就直接淘汰；但也不是自动保留，要先查是不是缺货/新品/陈列差/促销异常/数据异常，所以先标「金矿候选」进人工复核。

**⑦ 不改项**：原九宫格主裁决（销售额贡献×毛利额贡献）与 9 格标签**不变**；本补丁只在 C 行**叠加**复核闸，不替代九宫格。

#### 3.1.2 金矿候选二级闸：成本可信 + 近90天动销（六哥签字 2026-06-24）

> 来源：剔生鲜复跑暴露——数据异常 35%→11%(生鲜因当前客户成本口径异常暂排,见 §3.1.3,非通用永久规则)，但**死货升为最大污染(34%)**，且仍有非生鲜成本缺失(尚美日化 203 行 100% 缺等)。评审 [[金矿候选二级闸与成本异常排除评审_P0-ABC-Goldmine-Rule-Tighten-001_v0.1]] 采纳。
> **本补丁=方法规则;代码同步另开 CODEX-Goldmine-Rule-Tighten-Fix-001;同步重跑前不进 Execute-Approval。** 原九宫格 9 格 + §3.1.1 不变,本节只**再收窄金矿候选**。

**① 成本可信闸**：金矿候选依赖毛利率。**成本缺失/毛利率不可靠 → 不得 `goldmine_candidate=true`**,进 `cost_missing_review_pool`(不删、不淘汰、不按品类整剔,待人工查结算口径/联营/成本问题)。
- **成本不可信** = 销售成本缺失 / 销售成本≤0 / 毛利率缺失 / 毛利率无法计算 / **毛利率>0.95** / 其它明显成本毛利异常。
- **尚美日化/联营**：暂不单列联营池,成本缺失统一进 `cost_missing_review_pool`;后续若联营口径稳定再另建 `consignment_review_pool`。

**② 近90天动销闸**：金矿候选须有近 90 天**有效动销**。**`goldmine_candidate=true` 须同时:近90天销量≥4 且 库龄≤90 天**。不满足(销量<4 或 库龄>90)→ `goldmine_candidate=false`,进 `dead_stock_review_pool`(高毛利死货,先判清库存/缩面/淘汰,非金矿)。

**③ 暂不上 P85**：当前**保留小类 P75**(样本<20 用全店 P75),**不升 P85**。因剔生鲜后主污染已转成本缺失+死货,优先用成本闸+动销闸清主污染;P85 留 v0.3 备选,待重跑 review 后再定(避免过早误杀低销高毛利率小众好货)。

**④ 完整金矿候选规则(收窄后)**：`goldmine_candidate=true` 当且仅当全部满足——
1. 销售额贡献=C;2. `gross_margin_rate_tier=high`(小类 P75);3. 成本/毛利率可信;4. 近90天销量≥4;5. 库龄≤90;6. 非缺货;7. 非新品保护;8. 非促销异常;9. 非数据异常。

**⑤ 新增字段定义(供 CODEX-Goldmine-Rule-Tighten-Fix-001 落地)**：
| 字段 | 含义 | 取值 |
|---|---|---|
| `cost_reliable` | 成本/毛利率是否可信 | true/false |
| `recently_sold` | 近90天是否有效动销 | true/false |
| `exclusion_pool` | 排除池 | none/cost_missing/dead_stock/fresh_excluded/data_abnormal |
| `cost_missing_review_pool` | 是否进成本缺失复核池 | true/false |
| `dead_stock_review_pool` | 是否进死货复核池 | true/false |

口径:`cost_reliable=false` ⇔ 销售成本≤0/缺 或 毛利率缺/不可算/>0.95;`recently_sold=true` ⇔ 近90天销量≥4 且 库龄≤90;`exclusion_pool=cost_missing`(成本不可信)/`dead_stock`(销量<4 或 库龄>90)。

**⑥ 一线解释**：有些商品账面毛利率高,其实是没成本/成本没录准/联营特殊结算——这不是金矿,是成本异常,先查账;有些毛利率高但很久不动销、库龄老——这是死货风险,先判清库存/缩面/淘汰。真金矿=卖得不多但仍有人买 + 毛利率高 + 成本可信 + 库存不老 + 非缺货/新品/促销/数据假象。

**⑦ 后续代码同步**：本节仅确认方法规则;代码同步另开 CODEX-Goldmine-Rule-Tighten-Fix-001;同步并重跑 full dry-run 前不进 Execute-Approval。

#### 3.1.3 数据质量范围筛选机制（六哥修正 2026-06-24·防过度泛化）

> **关键口径修正**(P0-ABC-Goldmine-Scope-Clarification-001):此前「生鲜剔除」**不得**上升为通用方法论默认假设。**生鲜不是天然/永久排除品类**——是否参与毛利率型金矿判断,**由数据质量决定,不由品类名称决定**。

**① 通用方法论口径(不可外推特例)**：
- 生鲜**成本口径可靠** → **eligible**,可参与毛利率型分析;
- 生鲜/任何品类**成本不可靠**(内购无成本/联营/成本缺失/特殊结算)→ 进数据质量异常,不参与;
- **判定依据=数据质量,不是品类标签**。**禁止把花厅坊/乐购生鲜特例写成「所有超市生鲜永久排除」**。

**② 上游数据质量范围筛选层 `data_quality_scope_filter`**（先于金矿判断）：
| 字段 | 含义 | 取值 |
|---|---|---|
| `data_quality_scope_status` | 是否具备参与毛利率型判断的数据资格 | eligible / client_specific_excluded / cost_unreliable / requires_manual_scope_review |
| `data_quality_scope_reason` | 纳入/排除原因 | 文本 |
| `client_specific_exclusion` | 是否当前客户特例排除 | true / false |

> 仅 `data_quality_scope_status=eligible` 才进入 §3.1/§3.1.1/§3.1.2 金矿判断;其余进对应复核池。

**③ 花厅坊/乐购当前特例(客户级,非通用)**：
- **生鲜 = `client_specific_excluded`**——原因:当前客户生鲜成本口径异常/内购无成本/成本缺失频发(`client_specific_exclusion=true`);**后续该客户数据具备可靠成本口径后可重新纳入**;
- 尚美日化等疑似联营/成本缺失 = `cost_unreliable` 或 `requires_manual_scope_review`;
- 包装食品中成本可信部分 = `eligible`。

**④ 对体系化超市(数据规范)**：生鲜若成本口径可靠 = `eligible`,正常参与毛利率型分析。

**⑤ 对代码的要求(下一轮 CODEX-Goldmine-Rule-Tighten-Fix-001 须遵守)**：剔生鲜须实现为**客户级/门店级数据质量开关**,字段用 `client_specific_excluded` / `data_quality_scope_status`,**不得**用 `fresh_excluded=永久排除` 的通用品类硬编码;生鲜排除挂当前客户配置,不写死进通用逻辑。

> 📌 §3.1.2 ①「生鲜已除」与评审中「生鲜永久排除」表述,据本节**降级**为:**「花厅坊/乐购当前数据中生鲜因成本口径异常暂排除(client_specific_excluded),非通用永久规则」**。

### 3.2 IR 周转系数（T-07）
```
IR = 12 × (1 - 毛利率) / ITO
  毛利率：小数（如 0.25）
  ITO：年库存周转次数
```

### 3.3 安全库存 / A 类水位（T-09）+ 库龄分级（T-10）
```
A类最低水位 = 日均销量 × 到货天数 × 1.5
  → 货架库存低于水位即补货，永不等空（1.5 为通用起点，按业态/补货周期微调）

库龄分级(T-10)：0-30 正常 / 30-60 预警 / 60-90 滞销(进5闸) / >90 重滞(优先清)
售罄闸(T-08)：一个补货周期内售罄率>90% 或经常断货 = 卖太快 → 加订货量，毛利薄可试探提价
```

### 3.4 PSD 单品效率（T-15）
```
PSD = 该区/该档销售额 ÷ 该区/该档 SKU 数
  双信号：PSD 定"该不该补"（高=值得补），毛利率定"补什么款"
  背离时（PSD 高但毛利低）→ 该补，但补品用毛利率二次筛，挑高毛利款
```
