### Codex 处理（新增 `tools/` 三个计算模块，纯函数 + 可单测）

- **D1 `abc_classifier.py`（T-01/T-02）**
  - 单维：按 `销售额` 降序累计占比 → ≤60%=A / 60-90%=B / >90%=C
  - 双维：再按 `毛利额` 降序累计 → ≤60%=甲 / 60-90%=乙 / >90%=丙
  - 交叉裁决出 `身份` 列：销额A+甲/乙=核心引擎 / 销额A+丙=流量品 / 销额C+甲=利润品 / 销额C+丙=双低
  - 输出列：`销额ABC`、`毛利ABC`、`身份`
  - ⚠ 若无 `毛利额` 列，用 `销售额×毛利率` 推算并在 dry-run 标注

- **D2 `ir_calculator.py`（T-07）**
  - 公式：`IR = 12 × (1 - 毛利率) / ITO`（毛利率取小数；ITO=年库存周转次数）
  - 输出列：`IR`
  - ⚠ **ITO 列大概率缺失**：dry-run 须先报 ITO 可得性；若无，本工具标 `blocked_缺ITO`，不强算（不得用占位值伪造），等周转数据补齐再跑

- **D3 `safety_stock.py`（T-09 + T-10）**
  - A类水位：`A类最低水位 = 日均销量 × 到货天数 × 1.5`；货架库存<水位→`补货`信号
  - 库龄分级(T-10)：0-30 正常 / 30-60 预警 / 60-90 滞销 / >90 重滞
  - 售罄闸(T-08，顺带)：补货周期内售罄率>90% 或经常断货 → `卖太快` 标记
  - 输出列：`A类水位`、`补货信号`、`库龄级`、`卖太快`
  - ⚠ **`到货天数` 不可依赖花厅坊数据**（采购无规律，见能力库§五代理法）：作为**可配置参数**传入（全局默认值 + 品类级覆盖表），**不从原表取**；dry-run 打印所用默认值

### 纪律（沿用 CODEX-2026-06-20-01 边界）
- 原始/清洗文件全程只读；输入输出 SHA256 校验；
- **先 dry-run**（写 `_dryrun_preview/`）：报每工具的 **依赖列可得性 / 计算覆盖率 / 异常率 / blocked项**；
