# 作战层生命周期规范 v1.0

> **结论先行**:把**运行时项目文档(作战层)**与**长期资产(知识层)**分离归档,治"作战节奏污染知识库"。打捞 [[2026-06-25_归档打捞backlog_v0.1]] #9(源 v0.2 未采纳素材)。
> 这是 [[佐罗Zero宪章_ZERO-CHARTER-001_v0.2]] §4「操作层 vs 内容层」+ §6 反教堂在**文档治理**上的落地——本会话反复经历的"HOLD/收口"之痛,根因就是作战件混进知识库。

## 1. 两层定义
| | **作战层**(运行时) | **知识层**(长期资产) |
|---|---|---|
| 内容性质 | "现在做什么" | "怎么做" |
| 时效 | 短期(月/季) | 长期(年/多年) |
| 修改频率 | 高(每周) | 低(季度迭代) |
| 适用范围 | 当前项目 | 跨项目复用 |
| 失效方式 | 自然过期 | 不失效,只迭代 |
| 归档 | 月/季/年归档 | 不归档,持续迭代 |

## 2. 分层判据（6 个月测试）
- 6 个月后**还有用** → 知识层。
- 6 个月后**过期** → 作战层。
- 6 个月后**部分有用** → **拆分**:有用部分进知识层,过期部分进作战层归档。

> 典型作战件:主作战图、日记/复盘模板、Backlog、逐日跟踪表、数据看板。典型知识件:方法论、SOP、M-DEC、概念页。

## 3. 归档闭环（核心机制）
### 3.1 月度（如每月底）
1. 当月主作战图/数据表/日记 → 归入 `历史归档/YYYY-MM/`;
2. **从归档提炼"案例文档"** → 归入知识层(如调改案例 → 案例集);
3. 更新"当前作战层文件清单"为下月。

### 3.2 季度（6/30·9/30·12/31）
- 回顾本季作战文档 → 识别可提炼的知识节点 → **反向流入知识层** → 该季文档转归档。

### 3.3 年度（12/31）
- 整年作战归档进 `历史归档/YYYY/` → 形成年度复盘 → 与刻意练习层年度自评对接。

## 4. 与本库的接口
- 本库作战件主要在 `98_AI协作中枢`(任务队列/执行日志/输出区)与 `09_门店案例`(项目战役件)。
- **纪律**:战役结束 → 提炼方法论进 `01_/04_/05_`(知识层)→ 战役过程件归 `99_归档`(冻结)。**不让过期作战件常驻知识主体。**
- 与 [[SENSITIVE-GOV-001_客户数据脱敏与Git保密治理规范_v0.1]] 互补:作战件若含客户数据,先脱敏/进 `_client_private`,再谈分层。
