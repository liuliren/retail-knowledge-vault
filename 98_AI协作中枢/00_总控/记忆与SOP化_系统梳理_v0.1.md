## 二、SOP / Skill 化:分类归纳

> 判据:**确定性越高、复用越频 → 越该 skill 化;判断越重 → 越该留人(方法页)**。

### 类1 工程可封装(确定性·高频)→ 直接 skill 化
| 板块 | 现状 | 动作 |
|------|------|------|
| **脱敏测试链路** | 脚本成型(sanitize+analyze) | **封 SKILL.md**(最现成·P0) |
| **诊断卡导出** | report-export 方法+make-pdf | 封"5段诊断卡"skill |
| **POS 清洗** | 花厅坊POS清洗脚本v0.1 | 去客户化→通用 skill |
| **lint 健康检查** | lint-kb 已是 skill | ✅已封 |
| **周报生成** | 未建 | 新 skill(数据→周报) |

### 类2 半结构判定(规则+少量判断)→ skill 辅助 + 人确认
| 板块 | 可 skill 部分 | 留人部分 |
|------|--------------|----------|
| **复杂度判级 + 失真型预判** | analyze.py 已算 HHI/价格带/CR5 | 判级边界微调 |
| **价格带/IP-Chart 分析** | 分位/分档计算 | 是否拆档决策 |
| **角色闸防误杀** | 五角色匹配/GBA锁定标记 | 最终豁免签字 |
| **ABCZ 评分** | 9维加权算分 | 阈值与汰换定夺 |

### 类3 纯判断(不可 skill·护城河10%)→ 留方法页 + 人
- Z 签字 / 调改取舍 / 客户沟通 / 概念页升 active / 对外发布。**这是六哥不可替代的部分,skill 化反而稀释护城河。**

### 优先级(skill 化下一步)
1. **P0 脱敏链路封 skill**(最现成,1步)。
2. **P0 端到端「单品类诊断」skill**:串 脱敏→分层判级→失真型→诊断卡 = **商品化 90% 的核心引擎**。
3. **P1 周报 skill** + **诊断卡 skill 独立**。
