# 并行代理:同一文件单 worker 纪律

**类型**:feedback · **定于** 2026-06-26(花厅坊数据底座连撞三次车后)

## 规则
当六哥**正在直接带某个后台子代理**(用 SendMessage 侧信道与它对话)在一条线上工作时,**主控不得再派自己的代理去改同一批文件**——会双写覆盖(category_mapper.py 连撞三次)。一个文件/一条线只能有**一个 worker**。

## Why
六哥可侧信道续跑子代理,主控看不到那条对话;若主控也派代理改同文件,两个进程并发写=last-write-wins,丢工作、乱仓库。2026-06-26 数据底座 category_mapper 因此撞了三轮,停了两个主控代理止损。

## How to apply
- 一条线认定**单一 worker**(谁有上下文/六哥在带谁,就归谁);主控转**commit 收口 + canonical(RESUME)+ 隐私把关 + 其他独立线**。
- 派代理前先问:这批文件有没有别的 worker 在动?六哥是不是在带某个子代理碰它?
- 独立文件域才可并行(如 A 改 mapper、B 写另一个目录的工作表)——并行前显式划清文件域。
- 撞车止损:`TaskStop` 我侧代理,保住对方已验证产出,我只补不重叠的部分(或交给那个 worker)。
