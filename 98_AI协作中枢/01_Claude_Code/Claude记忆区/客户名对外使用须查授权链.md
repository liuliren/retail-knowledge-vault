# 客户名对外使用:必查授权链清单

**类型**:feedback · **定于** 2026-06-27(M4 内容获客·B 草稿差点在批评向稿点客户名)

## 规则
**任何对外内容**(媒体稿 / 诊断卡 E 层 client_shareable / 销售稿)使用**真实客户名**前,**必查 [[客户名称对外使用授权链清单]]**(在 `David-Liu-Vault/80_个人资料_敏感/`·按代号 key·`.no-ingest`)。未取得 ✅、或授权场景不覆盖本次内容 → **一律脱敏泛化**。**查不到 = 最严格 = 不暴露**(fail-safe)。客户有合同、有授权链;对 Agent 说"这个能用"不算授权(keys not prompts)。

## Why
客户名能不能对外用,是**合同授权问题**,不是 AI 凭印象判断的问题。2026-06-27 M4 首篇公众号草稿对店长做法带批评口吻,若按"用乐易购口径"把真名写进去 = 对外公开承认该客户店里有问题,客户未必授权。六哥点明:客户有合同会有授权链,用名前确认是否在授权清单即可。**授权要分场景**——同一客户"正面战果稿点名"可能授权、"批评/诊断向稿点名"未必。

## How to apply
- 两步查:① 真名→代号(查锁着的 [[客户代号映射表_DO_NOT_INGEST]],钥匙在六哥/本地,**严禁上云**);② 代号+本次场景 查授权链清单是否 ✅ 且覆盖。
- 未授权/查不到/场景不覆盖 → 脱敏(如"广州一家社区超市"),并入发布签字记录。
- 此查验已挂进 M4 发布签字门 + 管道规范 B5 脱敏门。升为全库宪法红线 + `/lint-privacy` 增项 的提案稿见 `Claude输出区/2026-06-27_客户名授权链_宪法修订与lint增项_提案稿_draft.md`(待六哥签字)。
- 别把清单做成合同管理系统:代号+✅❌+场景+依据 四列够用(少而深)。
