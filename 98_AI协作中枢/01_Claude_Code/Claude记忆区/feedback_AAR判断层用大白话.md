# 复盘/判断层内容誊正文前先大白话转述（六哥 2026-07-11 口述反馈沉淀）

**背景**：花厅坊战役#1 AAR 判断层回填时，先按候选稿(`2026-07-08_战役1_AAR判断层候选稿_draft_v0.1.md`)的书面语列出候选（如"角色判据分层""执行容量测算""防误杀闸门"等术语堆砌），六哥确认内容无误后主动要求"用简洁通俗的大白话转述下"，转述后明确反馈"这么表达就很直接简洁了，语言风格要优化"——即认可口语化版本，要求以此誊入正文，而非候选稿原措辞。

**Why**：候选稿的书面语/术语堆砌是为方法论层内部引用准备的，不等于适合直接进复盘正文——正文是给六哥自己和未来回看的判断记录，不是给系统看的方法论卡，读起来应该像人话，不是黑话。与 [[feedback_培训材料三原则_具象黄金圈实操]] 同一条纪律的另一个应用场景（那条针对一线培训材料，本条针对复盘/判断层文档）。

**How to apply**：往 AAR / M-DEC 判断层 / 复盘类文档誊写"做对了什么""做错了什么""三回归"等判断性结论时，默认先给一版大白话转述供六哥确认，而不是直接照抄候选稿的书面语。候选稿本身可保留术语（供方法论层引用/S4映射等用途），但誊入正式复盘正文的版本优先口语化、去术语堆砌。
