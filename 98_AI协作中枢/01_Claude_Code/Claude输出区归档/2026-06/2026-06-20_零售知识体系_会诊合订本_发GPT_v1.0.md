## A6. RetailOS（最新主力迭代）

把 L2骨架+L3武器封装成"店员可照做、可打印"的门店决策 SOP，纵贯 L2-L4，是"可执行/可售卖"出口。**不是新层，是把现有层压成动作句的操作系统外壳**。
（⚠ 与 98_AI协作中枢 三脑管道是两回事，勿混淆。）

架构 A-G 六层 + 减法/加法：
```
v1.0 减法（汰换）：A库存→B SKU结构→C价格带→D陈列→E ABC裁决→F误判→G门店SOP
v1.1 三项补深：E毛利交叉ABC / A库龄+售罄+水位 / B商品角色
v2.0 加法（选品/引进）：找缺口→门槛4闸→合理SKU数→放哪定价→观察期
```
**v2.0 选品层 4 闸**（本会话头号成果）：①真需求 ②有位置（上新必腾位，置换式一对一挂清退SKU）③算得过来（PSD单品日均销+毛利双信号）④供得上（**可得性代理法**：在购状态+近90天进货活跃+采购规模+是否本地，**不用采购周期**，因客户管理水平不高过程字段不可靠）。
**三轮验证**：T03饼干单小类闭环 → G4-A麦片货架全门槛回测（独立重现专家"压18补7置换式"）→ 沙埔大道店休食1218SKU跨店泛化（非花厅坊过拟合）；供应商能力库v0.2（63供应商/100%覆盖/采购规模分级）。
**商业交付版已出**。**这是体系当前最成熟、最有现金价值的资产**。
