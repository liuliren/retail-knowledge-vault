# 知识节点 CM-GBA-01：社区生活超市品类角色特化

> **节点类型**：补充节点（在品类管理通用品类角色框架基础上的社区超市特化版本）
> **适用业态**：广州/佛山社区生活超市（500㎡~2000㎡，全品类）
> **版本**：v0.1 | 作者：DavidLiu（实战经验）× Claude（结构化整理）

---

## 一、节点定义

**社区超市品类角色特化**是指：
在品类管理通用五大品类角色（Destination/Preferred/Routine/Occasional/Convenience）的框架下，
结合社区生活超市的**业态特征、商圈半径、客群结构**，
对每个品类角色的定义标准、SKU数配比、陈列资源分配、价格策略进行具体化规定，
使其可以直接指导采购决策和陈列规划。

---

## 二、通用品类角色框架（渥美/品类管理原版）

| 角色 | 英文 | 核心功能 | 顾客感知 |
|------|------|---------|---------|
| 目标品 | Destination | 吸引顾客专程来买；建立门店特色 | "这家有我要的" |
| 偏好品 | Preferred | 提供超出期待的品质或选择；提升满意度 | "这家比别家好" |
| 日常品 | Routine | 满足高频日常需求；保障基本客流 | "这家能买到" |
| 季节品 | Occasional/Seasonal | 配合节庆和季节性需求；制造惊喜感 | "正好需要，这里有" |
| 便利品 | Convenience | 满足顺手购买需求；提升客单价 | "顺便也买了" |

---

## 三、社区超市业态特征（角色定义的前提）

社区生活超市与大卖场/精品超市的根本差异，决定了品类角色的分配逻辑：

```
社区超市的本质：
  商圈半径：步行10~15分钟（约1.5km）
  核心客群：周边居民的日常刚需采购
  竞争对手：①附近菜市场（生鲜竞争）
             ②便利店（即时性竞争）
             ③大卖场（价格竞争）
             ④线上生鲜平台（品类竞争）

→ 社区超市的生存逻辑：
  不是靠一两个爆款引流（那是专卖店的逻辑）
  而是靠"一站式满足社区居民的80%日常购物需求"
  → 品类完备性 > 品类专精度
  → 高频刚需的稳定性 > 网红商品的新鲜感
```

## 四、社区超市品类角色特化标准

### 4.1 目标品（Destination）——社区超市版定义

**通用定义**：顾客专程为此而来的品类。
**社区超市特化**：顾客**每周必买、且在此购买是习惯而非偶发**的品类。

```
社区超市的 Destination 品类特征：
  ① 必须是高频（至少每周一次）
  ② 必须是本地居民的饮食刚需（不可被替代）
  ③ 本门店在此品类的表现必须明显优于周边竞争对手

大湾区社区超市 Destination 品类（通常为）：
  - 新鲜蔬菜（特别是叶菜类）
  - 散装猪肉/热鲜肉
  - 豆腐及豆制品
  - 活鲜水产（若有活鲜区）
  - 主食（米/面条/米粉）
```

**Destination 品类的配置要求**：

| 维度 | 要求 | 说明 |
|------|------|------|
| SKU数占比 | 品类内 SKU 数完备，不留明显空缺 | 缺货=流失顾客；顾客不会等你补货 |
| 陈列资源 | 占据最佳陈列位置（入口附近/主动线）| 顾客进门第一眼应看到最好的生鲜 |
| 价格策略 | P域（大众价格带）为绝对主力；价格必须有竞争力 | 与菜市场比价；不能贵太多 |
| 品质标准 | 零容忍劣质/滞销品在陈列区留存 | 生鲜区有一个烂菜，顾客对全店品质产生怀疑 |
| 损耗管理 | 允许合理损耗率；不能为控损耗而牺牲新鲜度 | 生鲜的核心竞争力就是鲜；控损过度=失去目标品地位 |

---

### 4.2 偏好品（Preferred）——社区超市版定义

**通用定义**：品质或选择性优于顾客期待，产生满意度溢出的品类。
**社区超市特化**：**满足顾客"偶尔想吃好一点/特别一点"**的需求；是门店的品质形象塑造工具。

```
大湾区社区超市 Preferred 品类（通常为）：
  - 优质大米（清远丝苗/五常大米/有机米）
  - 进口水果（车厘子/蓝莓/草莓等）
  - 优质海鲜（本地鲜虾/基围虾/新鲜蟹类）
  - 特色调味料（陈村粉/沙湾鱼露/正宗蚝油）
  - 本地特色熟食/烧腊
```

**Preferred 品类的配置要求**：

| 维度 | 要求 | 说明 |
|------|------|------|
| SKU数占比 | 精选；每个细分需求有1~3个代表性SKU即可 | 不需要大而全；要有明显品质区隔 |
| 陈列资源 | 视觉化陈列；强调品质感；可做专题陈列 | 进口水果要有"精品感"的陈列方式 |
| 价格策略 | M域（中档价格带）为主；价格透明，让顾客感到"值" | Preferred≠贵；而是"贵得有理由" |
| 替换频率 | 跟随季节/趋势更新；保持新鲜感 | 进口水果要跟上时令；不能全年卖同一批 |

---

### 4.3 日常品（Routine）——社区超市版定义

**通用定义**：满足日常高频需求；是门店现金流的基石。
**社区超市特化**：**工业食品/日用品/标准化商品中的大众主力品**；顾客不会为这类商品专程来，但如果没有会不方便。

```
大湾区社区超市 Routine 品类（通常为）：
  - 工业食品：酱油/食用油/盐/糖/醋/味精
  - 饮料：矿泉水/茶饮/碳酸饮料
  - 粮食：面条/粉丝/面粉
  - 日用品：卫生纸/洗手液/洗洁精
  - 零食：饼干/糖果/膨化食品（主流价格带）
```

**Routine 品类的配置要求**：

| 维度 | 要求 | 说明 |
|------|------|------|
| SKU数占比 | 覆盖主流需求即可；避免过度重复（减少同质 SKU）| 每个细分保留3~5个价格梯度清晰的 SKU |
| 陈列资源 | 中等陈列资源；位置合理即可，不需要黄金位 | 顾客会主动找，不需要"推" |
| 价格策略 | P域为主；部分促销品可在 Bad Item Price 段引流 | 促销要有计划性；不能全年打折（损毛利）|
| 库存管理 | 重点做补货及时性；断货比价格高更影响满意度 | Routine品类断货=顾客立刻转去竞争对手 |

---

### 4.4 季节品（Occasional/Seasonal）——社区超市版定义

**通用定义**：配合特定时节/节庆的商品；制造门店活力与顾客惊喜。
**社区超市特化**：大湾区节庆节点密集（详见 MC-GBA-01），季节品管理需要**精准的时间节拍**。

```
大湾区社区超市 Seasonal 品类节拍（全年）：

月份    节点        重点 Seasonal 品类
1月     春节前      腊味/年货/礼盒/糯米食材
2月     元宵        汤圆/糯米粉/元宵特供品
3~4月   清明        艾叶/糯米/烧猪（祭祀）
5月     端午        粽叶/糯米/咸蛋黄/绿豆/糖
6~9月   夏季        凉茶原料/冷饮/消暑食材（冬瓜/苦瓜）
9~11月  秋季煲汤季   汤料/滋补干货/秋季时令蔬果
10~12月 腊味季      广式腊肠/腊肉/腊鸭
12月    冬至        糯米/腊味/烧鸭/节庆食材
```

**Seasonal 品类的配置要求**：

| 维度 | 要求 | 说明 |
|------|------|------|
| 上架时机 | 节点前2~3周上架；节后快速清仓 | 太早=占库存；太晚=错过高峰 |
| SKU策略 | 基础款（大众需求）+ 礼品款（送礼需求）两个层级 | 不要只有礼盒（太贵）或只有散装（没有仪式感）|
| 陈列资源 | 节庆期间临时占用促销区/端架；主题化陈列 | 视觉冲击力=节庆采购的触发器 |
| 库存控制 | 节后库存归零为目标；宁可节前缺货也不要节后积压 | 腊味节后积压=重大损耗 |

---

### 4.5 便利品（Convenience）——社区超市版定义

**通用定义**：顾客顺手购买、提升客单价的品类。
**社区超市特化**：**收银台/主动线旁的冲动购买品**；同时也是年轻客群的"尝鲜入口"。

```
大湾区社区超市 Convenience 品类（通常为）：
  - 收银台前：口香糖/小零食/充值卡/湿纸巾
  - 主动线旁：饮料冷柜/冰淇淋
  - 堆头/端架：时令特价品/买一送一促销品
  - 新品试销区：厂商联合推广的新商品
```

**Convenience 品类的配置要求**：

| 维度 | 要求 | 说明 |
|------|------|------|
| SKU数 | 精简；单价低、拿取方便 | 太复杂=失去冲动购买的即时性 |
| 陈列位置 | 收银台前/主动线旁/端架 | 位置就是价值；不需要顾客主动找 |
| 价格 | P域低端或促销价 | 冲动购买=价格不是主要顾虑 |
| 周转 | 快进快出；定期更换 | 同一款商品摆太久=不再产生冲动 |

## 五、社区超市品类角色分配参考比例

```
社区超市（500~2000㎡，大湾区）品类角色 SKU 数参考分配：

  Destination（目标品）：25~35%
  → 生鲜类是核心；这里的 SKU 数一定要足
  
  Routine（日常品）：35~45%
  → 工业食品/日用品的主力；保障现金流
  
  Preferred（偏好品）：10~15%
  → 品质形象；毛利率主要来源
  
  Seasonal（季节品）：5~10%（动态变化）
  → 节庆前临时扩大比例；平时维持基础
  
  Convenience（便利品）：5~8%
  → 客单价提升工具；不能占太多货架资源

注意：以上是 SKU 数比例，不是销售额比例。
实际销售额构成通常为：
  Destination → 40~50%（生鲜主导）
  Routine → 30~40%
  Preferred → 10~15%
  Seasonal → 视节点波动
  Convenience → 3~5%
```

## 六、品类角色判断标准（现场诊断用）

以下问题可用于快速判断一个品类在特定门店中的实际角色定位：

**判断是否为 Destination（目标品）**：
- [ ] 顾客会专程为这个品类来这家店吗？
- [ ] 这个品类的缺货会直接导致顾客当天转去其他店吗？
- [ ] 门店在这个品类的表现，是否明显优于半径1km内的竞争对手？

**判断是否为 Routine（日常品）**：
- [ ] 顾客购买这个品类时，是习惯性顺带购买，而非专程来买？
- [ ] 即使换一个品牌，顾客也不会特别在意？
- [ ] 这个品类的 SKU 数是否存在严重重复（5个 SKU 里有4个实质上是一样的）？

**判断是否为 Seasonal（季节品）**：
- [ ] 这个品类的销售额在一年中有明显的高峰（峰值>均值的2倍）？
- [ ] 高峰期之外，这个品类的销售额是否几乎为零？

## 七、常见品类角色错配诊断

| 错配类型 | 症状 | 根因 | 纠正方向 |
|--------|------|------|---------|
| Destination品类被当作Routine管理 | 生鲜经常断货；品质不稳定；顾客在流失但不知道原因 | 采购认为"反正每天都会卖"；没有把生鲜当成核心竞争力来管 | 提升生鲜采购频率；设定严格的品质标准；赋予更多陈列资源 |
| Routine品类占据过多资源 | 工业食品占了太多货架；SKU高度重复；生鲜区反而太小 | 工业食品进货方便、损耗低；采购路径依赖 | 压缩Routine品类SKU数；释放货架资源给Destination品类 |
| Preferred品类定价过高 | 进口水果/精品食材基本不动销；顾客看看就走 | 错误判断社区顾客的消费力；P域定价=贵 | 重新评估目标顾客画像；调整选品方向到更接近M域入口的产品 |
| Seasonal品类时机错误 | 节庆结束后大量积压；或节庆高峰期货架空了 | 没有年度节奏表；凭感觉上货 | 建立年度节庆时间表；提前预订和测试 |

## 八、关联节点

- **MC-GBA-01**（大湾区TPOS本地化）：品类角色判断的输入；TPOS决定哪些品类是Destination
- **MC-02**（CDT）：每个品类角色内部的商品结构设计工具
- **MC-06**（IPC图）：Destination品类应集中在P域；Preferred品类应在M域
- **RM-06**（库存指标）：不同品类角色对应不同的库存管理标准
- **IM-04**（ABC分析）：在品类角色框架内进行ABC分析；Destination品类中的Z类问题最严重
