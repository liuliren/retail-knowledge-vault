# 晟果新零售_品类系统V4.1_大湾区版 深度审计报告

## 1. 总体结论

本次上传的《晟果新零售_品类系统V4_1_大湾区版.xlsx》已经从 V4.0 的“品类表”升级为较完整的“品类管理操作系统母版”：结构完整度、层级编码、标签体系、宽度诊断和大湾区本地化内容均有明显提升。

但它还不建议直接作为“最终定稿”或“系统导入底表”。当前版本更适合定义为：

> V4.1 候选版 / 结构增强版，可进入 V4.1.1 质量修复与系统字段补齐阶段。

最关键的待修复问题有三个：
1. 公式兼容性风险：部分读取/计算环境中显示 `#NAME?`，涉及 ①/⑤/⑧ 三张公式表。
2. 系统字段不完整：大湾区、角色、季节、场景等字段仍偏展示化，未完全拆成系统标准字段。
3. 商品角色与季节属性仍有混用：`季节款`仍作为商品角色存在，未完全独立为 seasonality_type / festival_tag。

## 4. 公式兼容性风险

当前 workbook XML 中有公式缓存值，业务数字可以被读取；但在当前检查器中，①架构总览等公式单元格出现 `#NAME?` 现象。核心风险不是业务统计错，而是公式跨环境兼容性不足。

主要风险来源：
- 公式大量引用 `②主数据品类框架!`，但未使用单引号包裹中文/特殊字符 Sheet 名称。
- 765 个公式包含未加引号的 `②主数据品类框架!` 引用。
- 公式涉及 ①架构总览、⑤角色×价格矩阵、⑧SKU宽度基准与诊断。

建议修复：
- 将 `②主数据品类框架!$D$4:$D$311` 改为 `'②主数据品类框架'!$D$4:$D$311`。
- 或将统计表改为“脚本生成硬值 + 公式说明字段”，避免客户打开环境导致公式失效。
- 若继续保留公式，必须在 Excel 原生环境中完成打开、重算、保存并再次校验。

构成一致性统计：

主角色缺位的 L3：
- 水饮料 / 碳酸饮料 / 健康碳酸：战略主角色=形象，实际构成=利润1
- 水饮料 / 茶饮料 / 含糖即饮茶：战略主角色=引流，实际构成=利润3/补充1
- 水饮料 / 茶饮料 / 无糖即饮茶：战略主角色=利润，实际构成=形象3

判断：

- 但战略角色、实际角色、季节性属性仍分散在不同 Sheet，未统一沉淀到主数据结构中。
- 5 个主角色缺位的 L3 应进入 V4.1.1 优先校准。

   - is_gba
   - gba_reason
   - gba_confidence
   - primary_role
   - secondary_role
   - seasonality_type
   - festival_tag
   - primary_scene_tag
   - secondary_scene_tags
4. 将“季节款”从主角色中弱化为季节属性，至少在系统字段中独立。

5. 将 `path_code` 映射状态拆为 `path_code` + `path_mapping_status`。
