## 3. 匹配算法

### 3.1 算法说明

**关键词 + N-gram 滑窗匹配** / 按 L4 → L3 → 品牌 → L4 定义关键词四层加分：

| 命中类型 | 加分 |
|---|:---:|
| L4 名整体在商品名（如 L4="棒棒糖" / 商品名含"棒棒糖"）| **+12** |
| L4 名 N-gram 子串（2-4 字滑窗 / 长度越长加分越多）| +5 起步 + 长度奖励 |
| L3 名整体在商品名 | +8 |
| L3 名 N-gram 子串 | +3 起步 + 长度奖励 |
| 品牌精确匹配（V4.0 品牌示例 ⊂ 商品品牌 或 反之）| +5 |
| L4 定义关键词（3 字以上中文）在商品名 | +3 |

### 3.2 置信度门槛

| 置信度 | 分数门槛 | fact_layer |

| **high** | ≥ 14 | observed |

### 3.3 反幻觉边界

## 5. 已知错误模式

匹配算法已知 4 类错误模式 / 需 review 时重点排查（**仅类型描述 / 不含客户 SKU 明细 / 明细见 mapped.csv**）：

| # | 错误模式类型 | 抽象描述 | 应人工核纠正 |
|:---:|---|---|---|
| 1 | **品牌跨类误导** | 某品牌在 V4.0 多个 L4 的"主要品牌示例"中重复出现 / 算法因品牌命中加分高 / 错配到该品牌**非主营 L4** | ✅ |
| 2 | **关键词跨类二义** | 商品名含某关键词（如二字风味 / 原料词）/ 该词同时是另一 L3 名 / N-gram 命中跨类 L3 / 错配 | ✅ |
| 3 | **风味词误导** | 商品名含风味词（如 X 味）/ 与饮料 / 调味区某 L4 共享词 / 错配到非零食类目 | ✅ |
| 4 | **子串过宽 / 无对应 L4** | 商品类型在 V4.0 308 L4 内**无现成对应**（如新兴细分品类）/ 算法返回低分或未匹配 / 需新增 V4.0 候选 L4 | ✅ |

**具体 SKU 案例**：保留在 gitignore 排除的 mapped.csv "匹配依据"列 / W22 review 时可按 fact_layer=pending + 置信度筛出。
