# 品类分析逻辑

> 鲜选决策 v1.0 · 核心方法论文档  
> 适用门店：1000–2000㎡ 社区全品类超市

---

## 一、品类分析的层级与目的

品类分析不是一个单一的操作，而是**分层次、分视角**的体系。在不同层级，分析的问题性质完全不同：

| 层级 | 分析目的 | 核心问题 | 决策输出 |
|------|----------|----------|----------|
| **L1 大类层** | 战略资源分配 | 哪个大类该加大投入？哪个该收缩？ | 陈列面积分配、采购预算分配 |
| **L2 中类层** | 品类组合结构 | 各中类的五角色比例是否合理？ | SKU数量上限、角色调整 |
| **L3 小类层** | SKU执行管理 | 具体哪个SKU保留/调位/下架？ | ABCZ执行、陈列面数、补货周期 |

## 二、品类核心指标体系

### 2.1 基础指标

| 指标 | 定义 | 计算公式 | 健康参考值（社区超市） |
|------|------|----------|----------------------|
| **月销售额** | 过去28天总销售额 | ΣSKUi × 售价i | — |
| **综合毛利率** | 月毛利额 / 月销售额 | Σ毛利额 / Σ销售额 | 引流品类 10-20%，利润品类 28-40% |
| **SKU数** | 在售SKU总数 | — | L3小类通常 3-12个 |
| **活跃SKU数** | 近8周有销售记录的SKU | sold_weeks_8w > 0 | 应 ≥ 总SKU数的85% |
| **零销SKU数** | 连续8周无销售 | sold_weeks_8w = 0 | 应 ≤ 总SKU数的10% |
| **平均周转天数** | 该品类内SKU的加权平均周转 | Σ(周转天数 × 销售额) / Σ销售额 | ≤ 15天为健康 |

### 2.2 综合效率评分（0-100分）

品类效率评分由以下维度加权计算：

```
效率分 = 活跃率 × 35  +  A类占比 × 40  +  毛利率 × 15  +  周转健康度 × 10
```

**参数说明：**
- 活跃率 = 活跃SKU / 总SKU（贡献上限 35分）
- A类占比 = A类SKU / 总SKU（贡献上限 40分）
- 毛利率折算 = 毛利率 × 100（贡献上限 15分）
- 周转健康度 = max(0, 1 - (周转天数-10)/40)（贡献上限 10分）

| 效率分 | 状态 |
|--------|------|
| ≥ 80 | 优秀品类，持续投入 |
| 65-79 | 良好，定向优化 |
| 50-64 | 一般，需要结构调整 |
| < 50 | 低效，需根本性重组 |

## 三、L1 大类层分析：CTM矩阵

### 3.1 CTM（Category Target Matrix）四象限

以**所有品类的中位数**为分割线，将各大类分为四个象限：

```
毛利率
  ↑
  │  现金奶牛           明星品类
  │  高毛利 + 慢周转     高毛利 + 快周转
  │
中位─────────────────────────────→ 周转速度
  │
  │  淘汰候选           潜力品类
  │  低毛利 + 慢周转     低毛利 + 快周转
```

| 象限 | 名称 | 特征 | 战略建议 |
|------|------|------|----------|
| 右上 | **明星品类** | 高毛利 + 快周转（销量好且赚钱） | 持续投入，扩大陈列，引入更多主销款 |
| 左上 | **现金奶牛** | 高毛利 + 慢周转（赚钱但滞销） | 优化SKU动销，精简低效补充款 |
| 右下 | **潜力品类** | 低毛利 + 快周转（卖得好但不赚钱） | 调整价格结构，引入利润款 |
| 左下 | **淘汰候选** | 低毛利 + 慢周转（既不赚钱又卖不动） | 根本性重组，聚焦少数高效单品 |

### 3.2 CTM 计算方法

```python
# 象限判断
median_gm   = 所有品类毛利率的中位数
median_turn = 所有品类周转天数的中位数

def ctm_quadrant(gm, turn):
    high_gm   = gm >= median_gm
    fast_turn = turn <= median_turn   # 注意：天数越少 = 周转越快
    
    if high_gm and fast_turn:  return "明星品类"
    if high_gm:                return "现金奶牛"
    if fast_turn:              return "潜力品类"
    return                            "淘汰候选"
```

**当前样本数据中位数（乐易购佛山禅城店）：**
- 毛利率中位数：约 27%
- 周转天数中位数：约 7.5天

## 四、L2 中类层分析：五角色结构

### 4.1 五角色 SKU 配置模型

在每个中类内，按照 SKU 的功能作用分为五种角色：

| 角色 | 功能 | 占比建议 | 价格带 | 毛利率 |
|------|------|----------|--------|--------|
| **引流款** | 建立价格形象，锚定顾客认知 | 10-20% | P域低价段 | 10-20% |
| **主销款** | 主要销量和收入来源 | 35-45% | P域主流段 | 20-30% |
| **利润款** | 拉高整体毛利结构 | 15-25% | P域高价段或M域 | 30-45% |
| **补充款** | 完善选择面，防止客户流失 | 10-15% | 全价格带 | 不限 |
| **测试款** | 新品孵化，持续迭代 | 3-8% | M域为主 | 不限 |

### 4.2 角色失衡诊断

| 失衡现象 | 问题描述 | 建议动作 |
|----------|----------|----------|
| 引流款 > 25% | 引流过度，毛利结构差 | 将部分引流款替换为主销款 |
| 主销款 > 55% | 缺乏价格弹性 | 引入利润款，测试差异化产品 |
| 利润款 > 30% | 顾客负担重，转化率低 | 补充P域主流价格款 |
| 补充款 > 20% | SKU冗余，占用陈列资源 | 识别并清理重叠功能补充款 |
| 无测试款 | 品类无法持续迭代更新 | 保留1-2个测试位 |

### 4.3 IP-Chart 价格带检验

详见《零售数学模型》文档。在中类层的应用重点是：

1. **检验双波峰是否存在**：P域是否有主波峰，M域是否有次波峰
2. **检验P域占比**：是否达到80%（大众需求覆盖度）
3. **检验M域占比**：是否有10-20%的中高价利润空间

## 五、L3 小类层分析：SKU执行管理

### 5.1 动销分析

| 指标 | 定义 | 预警阈值 |
|------|------|----------|
| **近8周动销周数** | 8周中有实际销售的周数 | ≤ 2周 → C/D类候选 |
| **零销识别** | sold_weeks_8w = 0 | 直接归入D类 |
| **间歇性零销** | sold_weeks_8w ≤ 3 | 纳入C类观察 |
| **稳定销售** | sold_weeks_8w ≥ 6 | 正常品，继续ABCZ分级 |

### 5.2 SKU 结构健康标准

对于每个L3小类，健康的SKU结构应满足：

```
最优结构：
│ A类 ────────────────── 1-2个（核心款）
│ B类 ─────────────────────── 2-4个（主销款组）
│ C类 ─── 0-2个（观察中）
│ D类 ── 0个（立即清理）
│ 合计：通常 3-8个 SKU 为宜
```

**高风险信号：**
- D类 ≥ 2个：需品类整改任务
- 活跃率 < 70%：品类选品方向偏差
- 单一SKU销售额占本品类 > 60%：过度依赖单款，有断货风险

### 5.3 小类间横向对比

系统提供品类热图（Category Heatmap），在以下维度对所有L3小类横向对比：

| 对比维度 | 意义 | 颜色深浅 |
|----------|------|----------|
| 月销售额 | 该品类在全店的规模地位 | 深蓝=大，浅蓝=小 |
| 综合毛利率 | 该品类的盈利能力 | 深蓝=高，浅蓝=低 |
| A类占比 | SKU结构健康度 | 深蓝=健康，浅蓝/红=问题 |
| 平均TF值 | 资金综合效率 | 深蓝=高效，浅蓝=低效 |
| D类数量 | 风险程度 | 红色=有风险，灰色=无风险 |

## 六、品类分析诊断流程

```
Step 1  导入数据
    ↓
Step 2  L1 大类 CTM 矩阵分析
        → 识别明星/淘汰候选大类
        → 确定资源分配优先级
    ↓
Step 3  L2 中类 五角色结构检查
        → 识别角色失衡的中类
        → 检验 IP-Chart 双波峰
    ↓
Step 4  L3 小类 SKU执行分析
        → ABCZ 分层
        → 零销识别，生成下架任务
        → 陈列面数重新分配
    ↓
Step 5  输出动作清单（按优先级排序）
        → 高优：D类下架、品类整改
        → 中优：A类调位、增面
        → 低优：补充款清理、新品引入
    ↓
Step 6  执行看板追踪
```

## 七、品类分析常见问题

### Q1：一个SKU应该归入哪个L3小类？

按照该SKU的**主要功能场景**归类，而非品牌或形态。例如：
- 养乐多 → 乳酸菌饮料（功能：早餐/益生菌）
- 老干妈辣椒油 → 酱料/蚝油（功能：烹饪调味）
- 可口可乐 → 碳酸饮料（功能：佐餐饮料）

### Q2：CTM象限的中位数如何确定？

系统自动计算所有品类毛利率和周转天数的中位数作为分割线，每次数据导入后自动更新。社区超市的经验参考值：毛利率中位数约25-30%，周转天数中位数约8-12天。

### Q3：品类角色应该由谁来设定？

建议由顾问或门店负责人在数据导入后进行人工标注，系统支持在品类分析页直接编辑。角色设定会影响诊断结论和选品建议的方向。

### Q4：L3小类最多配置几个SKU合适？

| L3小类类型 | SKU数建议 |
|-----------|-----------|
| 高频刚需（如液态奶、矿泉水）| 4-8个 |
| 常规主销（如薯片、饼干）| 3-6个 |
| 利润品类（如调味品、卤味）| 2-5个 |
| 补充形象（如家居用品）| 2-4个 |
