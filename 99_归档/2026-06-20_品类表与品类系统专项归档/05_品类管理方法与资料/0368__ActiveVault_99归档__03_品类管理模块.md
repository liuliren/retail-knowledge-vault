## CM-01 品类管理的本质

### 定义
品类管理是解决零售业**四大基本问题**的系统方法：
1. **Products（选品）**：卖什么？
2. **Price（定价）**：卖多少钱？
3. **Presentation（陈列）**：怎么摆？
4. **Promotion（促销）**：怎么推？

任何零售决策，都是这4P中的一个或多个。

### 完整流程
```
①品类定义（Category Definition）
  ↓ 输出：CDT
②品类角色（Category Role）
  ↓ 输出：每个品类的战略地位
③品类战略（Category Strategy）
  ↓ 输出：5种战略方向之一
④品类战术（Category Tactics = 4P）
  ↓ 输出：具体的选品/定价/陈列/促销方案
⑤计划实施（Plan Implementation）
  ↓ 输出：执行
⑥业绩评估（Category Review）
  ↓ 输出：迭代优化
```

### 品类管理 vs. 单纯采购的区别
```
单纯采购：
  关注"我要进什么货？"
  视角：商品→采购员→货架
  
品类管理：
  关注"顾客在这个品类上有什么需求？我如何系统满足？"
  视角：顾客→品类→商品→陈列→促销
```

## CM-02 MECE 原则

### 定义
**MECE = Mutually Exclusive, Collectively Exhaustive**
（彼此独立，且无遗漏）

商品配置的逻辑底座：分类时，类别之间不能重叠（独立），所有商品必须能被分进某一类（无遗漏）。

### 四种状态对比
```
甲：A、B独立但有遗漏 ❌（部分商品没分进任何类）
   ┌─────────────┐
   │  ○ A     ○ B │
   │              │
   └─────────────┘

乙：A、B无遗漏但有重叠 ❌（同一商品被分进两类）
   ┌─────────────┐
   │   ⊕         │  
   │  A  B       │
   └─────────────┘

丙：有遗漏（虽然A、B覆盖了大部分） ❌
   ┌─────────────┐
   │ A   ＿  B   │
   │             │
   └─────────────┘

丁：MECE化 ✅（独立且无遗漏）
   ┌─────────────┐
   │  A  │  B    │
   │     │       │
   └─────────────┘
```

### 商品配置中的MECE检验
```
检验1：每个SKU都能找到唯一的归属类别？
  → 如否：分类标准有重叠

检验2：所有顾客的需求点都对应到某个类别？
  → 如否：分类有遗漏

检验3：相邻类别之间的边界清晰？
  → 如否：陈列时会混乱

实战经验：
  零售商最常犯的MECE错误：
  ① 同一品类用多种分类标准混搭
  ② 边缘商品的归属不明确
  ③ 新品出现时找不到放在哪
```

## CM-03 品类定义四步法

### 定义
将一个抽象的品类（如"宠物用品"）转化为可操作的CDT的标准流程。

### 四个步骤
```
Step 1：确定需求层次（需求三角）
  → 输出：经营范围
  
Step 2：筛选商品群（选品漏斗）
  漏斗的5层筛网：
    ① 满足顾客需求
    ② 互相的关联性
    ③ 互相的替代性
    ④ 零售商的value-chain
    ⑤ 管理可行性
    ⑥ 测定可行性
  → 输出：候选SKU池

Step 3：确定分类层级顺序（消费思维阶梯）
  按顾客思考顺序排列：
  因素①→因素②→因素③→因素④
  → 输出：分类逻辑
  
Step 4：画出CDT
  → 输出：完整的消费者决策树
```

### 宠物用品案例
```
Step 1：需求确定
  食品/排泄/清洁/小服/健康/小说/宠物玩具/装饰/广告
  ↓ 划经营线
  确定经营范围：食品+健康+装饰+训练/奖励

Step 2：漏斗筛选
  从全市场宠物用品中筛出符合条件的SKU

Step 3：消费思维阶梯
  最终用户 → 产品使用用途 → 产品样式
  对应Category分类：
  Sub Category → Segment → Sub Segment

Step 4：画出CDT
  宠物用品
   ├ 猫用品
   ├ 禽类用品
   ├ 狗用品 → 食品 → 湿狗粮/干燥/微湿
   │       → 健康
   │       → 装饰
   │       → 训练/奖励
   ├ 爬行动物用品
   └ 水生动物用品
```

## CM-04 五大品类角色

### 定义
品类管理的核心概念：每个品类在门店中扮演不同角色，对应不同的资源分配和经营策略。

### 五大角色详解

```
1. Destination（目标品/目的品）
   功能：吸引顾客专程来；建立门店特色
   感知："这家有我专门要的"
   占比：5~7%品类
   
2. Preferred（偏好品）
   功能：超出期待的品质或选择；提升满意度
   感知："这家比别家好"
   占比：10~15%品类
   
3. Routine（日常品）
   功能：满足高频日常需求；保障客流
   感知："这家能买到"
   占比：50~60%品类
   
4. Occasional/Seasonal（季节品/偶尔品）
   功能：配合时节制造惊喜
   感知："正好需要，这里有"
   占比：10~15%品类
   
5. Convenience（便利品）
   功能：满足顺手购买；提升客单价
   感知："顺便也买了"
   占比：10~15%品类
```

### 大湾区社区超市角色分配示例
```
品类角色 SKU数比例参考：
  Destination：25~35%（生鲜主导）
  Routine：35~45%（工业食品主力）
  Preferred：10~15%（品质形象）
  Seasonal：5~10%（动态变化）
  Convenience：5~8%（客单价工具）

实际销售额构成通常为：
  Destination：40~50%（生鲜主导）
  Routine：30~40%
  Preferred：10~15%
  Seasonal：视节点波动
  Convenience：3~5%
```

## CM-05 品类战略

### 定义
基于品类角色，制定的5大战略方向：

```
1. 创造客流（Traffic Building）
   → 用于Destination品类
   → 通过低价/独特品种吸引顾客上门
   
2. 创造交易（Transaction Building）
   → 用于Routine品类
   → 通过完善品类满足日常需求
   
3. 创造现金（Cash Generating）
   → 用于Routine品类
   → 维持高周转，保证现金流
   
4. 创造利润（Profit Generating）
   → 用于Preferred品类
   → 通过差异化获取毛利率
   
5. 创造形象（Image Enhancing）
   → 用于Destination/Preferred品类
   → 塑造门店专业形象
```

### 战略选择矩阵
```
品类角色       常用战略          辅助战略
─────────────────────────────────────────────
Destination    创造客流          创造形象
Preferred      创造利润          创造形象
Routine        创造交易          创造现金
Seasonal       创造交易          创造客流（节点期）
Convenience    创造交易          创造现金
```

## CM-06 品类战术（4P）

### 定义
基于品类战略，设计的具体执行方案：

```
Products（选品）：
  Destination：宽度全 + 深度合理
  Preferred：精选 + 深度精
  Routine：覆盖主流 + 减少重复
  Seasonal：节点导向
  Convenience：单价低、易拿取

Price（定价）：
  Destination：P域为主，价格有竞争力
  Preferred：M域为主，"贵得有理由"
  Routine：P域为主，部分促销引流
  Seasonal：基础款+礼品款
  Convenience：P域低端+促销价

Presentation（陈列）：
  Destination：黄金位、入口、主动线
  Preferred：视觉化、强调品质感
  Routine：中等资源、合理位置
  Seasonal：临时端架/促销区主题化
  Convenience：收银台、动线旁

Promotion（促销）：
  Destination：限定+独家活动
  Preferred：捆绑/试用
  Routine：常规促销+会员价
  Seasonal：节点主题促销
  Convenience：随手价/价格冲击
```

## CM-07 竞品商品结构分析

### 定义
通过对竞争对手的商品明细进行系统化分析，反推其商品配置思路，发现差距并制定应对策略。

### 数据获取方法
```
线上途径（推荐）：
  ① 对手在美团/小象/盒马等平台开店
  ② 截图全部商品页面
  ③ 用图文识别软件（OCR）提取信息
  ④ 整理为结构化Excel

线下途径：
  ① 实地市调，拍照记录
  ② 注意：商品明细不属于商业秘密
  ③ 这是公开可见信息，不涉及违规

  注意事项：
    × 不要泄露具体的SKU清单
    × 不要散播友商品名条码等信息
    ✓ 学习的是配置思路，不是抄品
```

### 分析步骤
```
Step 1：导出明细
  品名/规格/产地/价格/销量

Step 2：建立HS矩阵分析框架
  基于自己对该品类的理解，定义H/S轴

Step 3：填入对手SKU到矩阵
  观察对手的配置重点

Step 4：与自店做对比
  ① 哪些格子对手有，自己没有？（可能是机会）
  ② 哪些格子自己有，对手没有？（可能是优势）
  ③ 哪些格子双方都集中？（红海）

Step 5：得出策略
  → 不是照抄，而是判断差异化方向
```

### 美团小象水果案例
```
分析结果（HS矩阵化后）：
        省不省事 →
        多汁-溢手  少汁-润口
脆 甜    13         66
   均    8          13
   酸    4          1
面 甜    20         35
   均    7          58
   酸    14         15

观察：
  小象集中配置：脆甜+少汁润口（66个）
  即：嘴馋+省事 = 写字楼社畜午茶

启示（不是抄）：
  社区超市的客群不同（家庭主妇为主）
  应该集中配置：面甜+多汁（适合家庭分享）
  以及面均+少汁（耐储存的家庭装）
```

## CM-08 CDT → Layout 转化

### 定义
将逻辑层的CDT转化为物理层的Layout（陈列图）的方法。

### 转化原则
```
CDT决定陈列逻辑：
  CDT的分支顺序 ≈ 陈列的分区顺序
  
  例（消暑CDT）：
    根需求：食冷
    分支1：外出解暑（吃前/吃中/吃后）
    分支2：居家解暑（冷食/冷饮）
  
  对应Layout：
    主陈列（C位）：核心商品（冰淇淋）
    关联陈列（环绕）：按使用场景分区
```

### 陈列基准选择
```
按"用途/使用场景"陈列（推荐）：
  优势：符合顾客购买思维
  例：消暑卖场、火锅卖场、烘焙卖场

按"品类"陈列（传统）：
  优势：管理简单
  劣势：割裂关联购买
  例：饮料区、调味料区

何时用哪种：
  Destination/Seasonal品类 → 按场景陈列
  Routine品类 → 按品类陈列
  Preferred品类 → 按品质感陈列
```

### 大湾区蔬菜区Layout示例
```
按场景重组（vs. 传统按品种）：

入口区：当日新鲜（对应"早市新鲜"动机）
  ├ 当日到货叶菜（菜心/通菜/芥蓝）
  └ 活鲜豆制品（豆腐/腐竹）

中区：煲汤场景（对应大湾区高频需求）
  ├ 汤料蔬菜（淮山/玉米/胡萝卜/莲藕）
  ├ 配套汤料（无花果/南北杏）
  └ 关联肉类（猪骨/猪展）

侧区：炒菜场景
  ├ 主菜（青菜叶菜）
  ├ 配菜（葱姜蒜）
  └ 关联调料（蚝油/生抽）

里区：储存型蔬菜
  ├ 根茎类（土豆/萝卜/红薯）
  └ 瓜类（冬瓜/南瓜/老黄瓜）
```

## CM-09 胖东来商品配置分析框架

### 定义
基于胖东来案例总结的"超市商品结构三维评价框架"：

```
评价维度1：IPC形状
  好：右下倾斜（创造现金流能力）
  坏：右上倾斜（顾客感觉贵）

评价维度2：IPC分布
  好：P域占60~80%, M域占20~40%
  坏：分布失衡

评价维度3：理论周转天数（TR）
  好：TR-sku < 60天（包含全品类）
  极好：TR-sku < 45天
  坏：TR-sku > 90天
```

### 综合评分体系
```
得分体系（每项0~3分）：
  IPC形状：右下完美=3分, 右下波动=2分, 右上=0分
  IPC分布：P域80%=3分, P域60%=2分, P域<50%=0分
  TR-sku：<45天=3分, 45~60天=2分, >90天=0分

综合判断：
  9分：教科书级别（如胖东来）
  6~8分：良好
  3~5分：一般，需大幅调整
  0~2分：危险，需重做商品配置
```

### 胖东来分析结论
```
评分：
  IPC形状：3分（右下倾斜完美）
  IPC分布：3分（P域84.2%, M域15.8%）
  TR-sku：3分（40.5天，含服饰和家居）
  综合：9/9 = 教科书级别

亮点：
  ✓ 商品结构完美匹配P/M域分配理论
  ✓ 包含服饰和家居仍能TR<45天，超强造血力
  ✓ 整体分布精确匹配理论曲线

不足：
  ⚠️ TR-stk = 198天，M域价格略偏高
  ⚠️ 高单价段缺重点，配置略散
```
