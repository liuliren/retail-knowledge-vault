## 一、第三轮回灌定位

第一轮回灌已经把 `02` 从“品类讨论稿”推进成了：

> **品类结构总纲底稿**

第二轮回灌又把它推进成了：

> **品类结构总纲 + 承接映射底稿**

第三轮不再继续扩原则，而是进一步做“冻结化处理”。  
也就是把它从“总纲文件”继续推进成：

> **可直接作为系统上游引用的品类体系冻结底稿**

本轮只做 3 件事：

1. **强化 `02 ↔ 07` 的一一映射**
2. **把关键板块分类模板标准化**
3. **把品类体系对页面 / API / 表 / 规则的承接统一索引化**

所以第三轮的定位是：

> **让 `02` 不只是能解释，还能被直接引用。**

---

## 二、第三轮回灌目标

本轮只收口以下 3 个方向：

### A. 一一映射强化
把 `02` 中的原则，明确对应到 `07` 中的结构落点。

### B. 模板标准化
把板块分类模板从“说明型模板”推进成“可复用判断模板”。

### C. 系统索引化
把品类体系在系统中的承接，从“解释关系”推进成“统一索引关系”。

本轮不做：
- 不新增类目
- 不调整 L1/L2/L3/L4 目录
- 不重开某类是否成立的讨论
- 不写字段级表结构

## 四、板块分类模板标准化

第二轮已经抽出了 5 类模板。  
第三轮开始，把这些模板进一步标准化为“判断模板”。

也就是以后新增板块时，不再从零讨论，而是先判断它更接近哪种模板。

---

## 4.1 模板A：快消常温板块模板

### 适用对象
- 饮料
- 休闲零食
- 坚果炒货
- 饼干糕点
- 方便速食
- 冲调早餐/冲饮
- 糖巧果冻
- 蜜饯果干/凉果

### 标准判断模板
```text
第一优先：顾客消费任务是否差异明显
第二优先：价格带 / 品牌带差异是否明显
第三优先：规格带 / 人群带是否差异明显
第四优先：是否需要启用 L4
```

### 何时优先启用 L4
- 同一 L3 下品牌和价格带分化明显
- 同一 L3 下儿童型 / 家庭型 / 功能型明显不同
- 同一 L3 下动作建议容易误伤

### 系统承接特征
- 更容易进入 L4
- 更容易进入模板增强
- 更容易产生价格带与陈列类规则

---

## 4.2 模板B：生鲜食材板块模板

### 适用对象
- 蔬菜
- 水果
- 肉禽蛋
- 水产
- 冷冻水产
- 豆制品
- 净菜/生鲜半成品

### 标准判断模板
```text
第一优先：天然食材属性
第二优先：保鲜 / 鲜度 / 冷冻状态
第三优先：烹饪场景差异
第四优先：是否需要启用 L4
```

### 何时优先启用 L4
- 同一 L3 下鲜活 / 冷冻 / 净切决策路径明显不同
- 同一 L3 下家庭主菜 / 快手菜 / 礼赠路径不同
- 同一 L3 下任务、补货、损耗逻辑明显不同

### 系统承接特征
- 更依赖经营区和损耗逻辑
- 更依赖鲜度、库存、周转类规则
- 页面上更强调日常经营健康度

---

## 4.3 模板C：乳品日配板块模板

### 适用对象
- 乳品
- 酸奶/乳酸菌
- 早餐日配
- 冷藏甜品

### 标准判断模板
```text
第一优先：冷链属性
第二优先：消费时点（早餐 / 日常 / 即饮 / 家庭）
第三优先：规格与人群差异
第四优先：是否需要启用 L4
```

### 何时优先启用 L4
- 同一 L3 下儿童型 / 功能型 / 家庭装差异明显
- 同一 L3 下常温 / 低温路径不同
- 同一 L3 下短保管理动作明显不同

### 系统承接特征
- 更依赖冷链能力模板
- 更容易关联短保、日清、周转类规则
- 页面上更适合看消费时点和人群差异

---

## 4.4 模板D：家庭基础原料板块模板

### 适用对象
- 米面粮油
- 调味品
- 南北干货
- 汤料滋补干货
- 烘焙原料

### 标准判断模板
```text
第一优先：烹饪任务
第二优先：基础原料 / 复合调味 / 汤料滋补
第三优先：区域差异是否明显
第四优先：是否需要启用 L4
```

### 何时优先启用 L4
- 同一 L3 下基础型 / 升级型 / 区域型差异明显
- 同一 L3 下烹饪场景完全不同
- 同一 L3 下规则容易在佐餐与主烹饪调味之间混伤

### 系统承接特征
- 更依赖家庭烹饪任务
- 更容易与区域增强模板结合
- 页面上更适合用“烹饪任务”解释结构

---

## 4.5 模板E：功能型非食板块模板

### 适用对象
- 个护
- 家清
- 纸品
- 女性护理
- 居家环境
- 一次性用品
- 居家纺品/袜拖
- 小家电（部分）

### 标准判断模板
```text
第一优先：功能需求
第二优先：使用场景
第三优先：人群差异
第四优先：规格 / 价格带差异
第五优先：是否需要启用 L4
```

### 何时优先启用 L4
- 同一 L3 下功能路径差异明显
- 同一 L3 下使用场景完全不同
- 同一 L3 下人群差异明显（儿童、女性、敏感型等）

### 系统承接特征
- 更容易与功能型规则结合
- 更容易在页面中使用“功效型解释”
- 更适合按功能和场景做决策组

## 五、品类体系 → 分析逻辑 的统一索引

第三轮开始，把 `02` 对 `03` 的承接，统一收口成索引关系。

---

## 5.1 分析对象索引

| 品类层 | 分析关注点 | 典型问题 | 结果输出方向 |
|---|---|---|---|
| L1 | 板块健康 | 哪个板块拖累门店 | 板块预警 |
| L2 | 组合合理性 | 哪个中类结构失衡 | 中类组合预警 |
| L3 | 结构效率 | 哪个小类应扩/稳/调/压 | L3结构建议 |
| L4 | 决策差异 | 同一 L3 内是否需要区别决策 | 决策组增强建议 |
| SKU | 执行动作 | 对哪个商品具体做动作 | 任务与反馈 |

### 统一结论
- `03` 的分析起点由 `02` 决定
- `03` 的分析深度由 `02` 的层级结构决定

---

## 5.2 规则作用层索引

| 品类层 | 规则作用方式 | 典型规则类型 |
|---|---|---|
| L1 | 板块总览预警 | 板块健康度预警 |
| L2 | 结构预警 | 中类角色失衡、组合异常 |
| L3 | 核心分析预警 | 结构效率、动销、库存、价格带 |
| L4 | 高复杂度类增强预警 | 决策差异相关规则 |
| SKU | 最终动作规则 | expand / clear / relayout / replace |

### 统一结论
品类体系不是规则本身，但它决定规则落在哪一层最有效。
