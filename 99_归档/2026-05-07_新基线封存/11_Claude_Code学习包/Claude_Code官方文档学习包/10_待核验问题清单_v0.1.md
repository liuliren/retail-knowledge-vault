# 待核验问题清单

## 1. 文件定位

集中记录本学习包中所有未亲自验证的事实，每条带"影响"与"怎么核验"。**这是本包的反幻觉硬约束**——任何被引用的"待核验"项必须能在此清单中找到，并随核验进度更新。

## 2. 使用规则

1. 任何待核验项，在被引用的文件中保留「待核验」标记，**不得作为事实在其它文件中引用**
2. 核验通过后：
   - 在本清单标"已核验 / YYYY-MM-DD / 结论"
   - 同步去掉所有引用文件中的"待核验"标记
   - 对应文件版本号 +0.1
3. 每 90 天从头复核一次（Claude Code 文档持续迭代）
4. 核验路径优先级：官方文档原页 > 实测 > 社区 / 第三方资料
