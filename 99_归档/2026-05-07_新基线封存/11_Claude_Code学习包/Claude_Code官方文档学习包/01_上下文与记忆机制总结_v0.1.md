# 上下文与记忆机制总结

## 1. 文件定位

把 Claude Code 启动加载顺序、auto memory 机制、`/compact` 后行为、`/clear` 与 `/resume` 边界一次说清，作为本知识库治理 CLAUDE.md / `.claude/rules/` / Skills 的事实依据。

## 2. 适用场景

- 决定要不要把某条规则写进 CLAUDE.md vs `.claude/rules/` vs skill
- 长会话中判断何时 `/compact` 何时 `/clear`
- 排查"指令没生效"
- 评估 auto memory 的隐私与脱敏风险

## 3. 核心内容

### 3.1 启动时进上下文的内容（按官方 context-window 时间线）

1. System prompt
2. Auto memory `MEMORY.md`（前 200 行 / 25KB 取小）
3. 环境信息（cwd、平台、shell、OS、git 状态）
4. MCP 工具名（schema 默认延迟，仅工具搜索时按需加载，可由 `ENABLE_TOOL_SEARCH=auto/false` 调整）
5. Skill 名 + 单行描述（含 `disable-model-invocation: true` 的不列）
6. CLAUDE.md 全量（项目 / 用户 / 本地 / 管理策略，按目录树由根到 cwd 拼接，再拼 `CLAUDE.local.md`）
7. `.claude/rules/*.md` 中无 `paths:` 的规则（与 `.claude/CLAUDE.md` 同优先级）

### 3.2 CLAUDE.md 加载位置（四级，由高到低）

| 级别 | 路径 | 用途 |
|---|---|---|
| 管理策略 | `/Library/Application Support/ClaudeCode/CLAUDE.md`（macOS）等 | 组织级强制规则 |
| 项目 | `./CLAUDE.md` 或 `./.claude/CLAUDE.md` | 团队共享 |
| 用户 | `~/.claude/CLAUDE.md` | 个人偏好 |
| 本地 | `./CLAUDE.local.md` | 个人项目级（应 .gitignore） |

按目录树由根到 cwd 全部拼接，互相不覆盖；`CLAUDE.local.md` 在每层 CLAUDE.md 之后追加。

### 3.3 Auto memory 机制

- 位置：`~/.claude/projects/<repo>/memory/MEMORY.md` + 主题文件
- 加载：`MEMORY.md` 前 200 行 / 25KB 进上下文，主题文件按需读
- 默认开启；`/memory` 可开关；环境变量 `CLAUDE_CODE_DISABLE_AUTO_MEMORY=1` 可禁
- 机器本地，跨 worktree 共享同一目录

### 3.4 `/compact` 之后

| 机制 | 行为 |
|---|---|
| 项目根 CLAUDE.md / `.claude/CLAUDE.md` | 重新从磁盘注入 |
| Auto memory `MEMORY.md` | 重新注入 |
| 环境信息 / MCP 工具列表 | 重新注入 |
| Skill 列表 | 不重新注入；仅保留**已被调用过**的 skill 本体（且超额会被截断） |
| 嵌套子目录 CLAUDE.md / `paths:` 规则 | 不重新注入，下次读对应文件时再触发 |
| 会话历史 | 替换为结构化总结 |

### 3.5 三个清理命令的边界

- `/clear`（别名 `/reset` `/new`）：开新会话、清空上下文，**旧会话仍可 `/resume` 找回**
- `/compact [指令]`：保留同一会话，把历史压缩为摘要；可附"重点保留代码示例"等指令；也可在 CLAUDE.md 里固化压缩偏好
- `/resume`：恢复旧会话（建议清前先 `/rename` 命名）

### 3.6 `/context` 可视化

显示当前上下文占用色块图与优化建议；怀疑爆窗时优先用。

## 4. 已确认事实

以上 3.1–3.6 均直接对应官方 memory.md / context-window.md / commands.md。

## 5. 操作规则（本 vault）

1. 当前 CLAUDE.md 567 行，**官方建议 < 200 行**；后续应拆 `.claude/rules/`（带 `paths:`）。
2. 重型 SOP（DisplayMap 检查、52 周 MD 模板、商品诊断流程）→ skill 而非 CLAUDE.md。
3. 主题切换 `/clear`；同主题深入 `/compact`；恢复历史 `/resume`。
4. 在 CLAUDE.md 中固化压缩偏好（"压缩时重点保留 X / Y"）。

## 6. 待确认事项

- 嵌套 CLAUDE.md 在 `/compact` 后到底是"完全不重载"还是"下次读触发文件时重载"——官方表述偏后者，但与"survives compaction"列表的措辞略有摩擦。**待核验**：用 `/memory` 在 `/compact` 前后各看一次加载列表，确认行为。
- Auto memory 在跨 worktree / 非 git 仓库时的派生规则——见 [[10_待核验问题清单_v0.1]] #4。
