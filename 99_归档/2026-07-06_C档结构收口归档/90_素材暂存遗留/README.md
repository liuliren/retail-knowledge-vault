# 90_素材暂存与待整理

## 1. 目录定位

本目录是 vault 的**素材缓冲区**，不是正式知识资产区。

定位口径：
- 所有未经过整理、未判断价值、未归类的原始素材**先落本目录**；经过整理后再迁入相应正式目录。
- 本目录里的任何文件**不视为知识库正式结论**，不得在客户交付 / 方法论 / 报告中直接引用。
- 本目录是 vault **反幻觉机制的物理边界之一**：素材没被整理过 → 就还在 90_ → 就不算"知识"。

## 2. 现有结构

```
90_素材暂存与待整理/
├── 01_晟果新零售商品数据库/   ← 商品数据原始素材池
└── 02_晟果新零售系统工具/     ← 系统/工具原始素材池
```

> 备注：[[CLAUDE.md]] §18.1 提到的 `99_待Ingest/`（Phase 0 D5）当前在本目录下尚未发现实体子目录，是否落入别处或仅在 SOP 描述中，待 [[G01_Ingest]] 后续审计核对。本轮不创建该子目录（属于结构变更，超出最小校准边界）。

## 3. 适合放什么

- 客户现场照片、地图截图、竞品照片
- 客户 POS / 销售 / 库存 原始 Excel
- 微信聊天记录、客户访谈录音转写、电话备忘
- 行业研究 PDF、外部资料原文
- 临时灵感、未归类资料、待 Ingest 的 v0.x 增量包

## 4. 不适合放什么

- ❌ 已整理 / 已写结论的方法论文件 → 走 `01_科学零售方法论/`
- ❌ 已整理 / 已写结论的项目报告 → 走 `09_<门店>/` 或 `16_客户与战役档案/<战役名>/`
- ❌ 可复用的交付模板 → 走 `10_咨询交付模板/`
- ❌ 数据脚本、清洗算法、工具代码 → 走 `13_数据分析与工具脚本/`
- ❌ 旧版本归档（请走 `99_归档/`，不要堆在 90_）

## 5. 素材准入与出库规则

### 5.1 进入 90_（Ingest 入口）

每份素材入库必须标注：
1. **采集日期**（YYYY-MM-DD）
2. **来源**（客户名 / 门店名 / 网站 / 访谈对象）
3. **采集人**（六哥 / 同事 / 客户提供）
4. **初步分类**（客户素材 / 行业研究 / 临时灵感 / v0.x 卷宗）

参 [[G01_Ingest]]（治理 SOP）。

### 5.2 出库到正式目录

素材进入正式目录前**必须**完成：
1. 分类判断（参 [[CLAUDE.md]] §11 知识沉淀分类规则）
2. 命名规范化（参 [[文件命名规范]]）
3. 来源标注（采集日期 / 出处 / 采集人）
4. 是否可用判断（高 / 中 / 低 / 待确认）
5. 反幻觉边界（原始素材不修改原文；二次创作落 wiki 层；冲突必须标注不强行合并，参 [[CLAUDE.md]] §13.11）

### 5.3 不可直接出库到以下目录

未整理素材**禁止**直接落入：
- `01_科学零售方法论/`
- `09_门店案例与项目复盘/`（除原始素材子目录 `99_原始素材/` 外）
- `10_咨询交付模板/`
- `04_/中层决策库/`
- `16_客户与战役档案/<战役名>/` 主干（_素材区除外）

### 5.4 滞留治理

[[G03_Lint]] 周巡检会扫描 90_ 下超 30 天未整理素材，超期项进入 [[Vault健康仪表]] 红灯。
