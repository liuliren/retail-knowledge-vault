# SKU 分层逻辑

> 鲜选决策 v1.0 · 核心方法论文档  
> 适用门店：1000–2000㎡ 社区全品类超市

---

## 一、分层体系概述

SKU 分层是商品管理的核心执行工具。系统采用**9维度评分 + ABCZ分层**的双轨机制：

- **9维度评分**：对每个SKU进行综合能力评分（0-100分），客观量化单品价值
- **ABCZ分层**：基于销售额累计贡献和零销识别，将SKU分为四个行动层级

分层结果直接驱动**陈列面数分配**、**库存策略**和**下架/调位任务**的自动生成。

---

## 二、9维度评分体系

### 2.1 评分维度定义

| 维度 | 权重等级 | 数据来源 | 评分逻辑 |
|------|----------|----------|----------|
| **① 销售额贡献** | 高 | 近28天销售额 | 越高越好，与品类均值比较 |
| **② 毛利额贡献** | 高 | 近28天毛利额 | 越高越好，综合衡量盈利价值 |
| **③ 动销率** | 中 | 近8周动销周数/8 | 动销8/8=满分，0/8=零分 |
| **④ 库存周转速度** | 中 | 周转天数 | 越短越好，目标≤15天 |
| **⑤ 价格带匹配** | 中 | 售价 vs P域/M域 | 落在P域或M域内得分，超出范围扣分 |
| **⑥ 规格带合理性** | 低 | 规格带标签 | 是否与目标客群消费习惯匹配 |
| **⑦ 关联销售贡献** | 低 | 连带购买数据 | 带动其他品类销售的能力 |
| **⑧ 供应稳定性** | 低 | 近8周断货次数 | 断货次数越少得分越高 |
| **⑨ 门店适配度** | 低 | 人工标注 / 门店定位匹配度 | 是否符合门店定位和核心客群 |

### 2.2 评分权重参考

```
综合评分 = 
    销售额贡献分 × 0.25 +
    毛利额贡献分 × 0.25 +
    动销率得分   × 0.15 +
    周转速度分   × 0.15 +
    价格带匹配分 × 0.10 +
    规格带得分   × 0.03 +
    关联销售分   × 0.03 +
    供应稳定分   × 0.02 +
    门店适配分   × 0.02
```

### 2.3 评分区间与含义

| 评分 | 含义 |
|------|------|
| 85-100 | 核心高分款，全面优秀 |
| 70-84 | A类主力，结构健康 |
| 50-69 | B类维护，局部优化 |
| 30-49 | C类观察，需改善 |
| 0-29 | D类淘汰候选 |

## 三、ABCZ 分层规则

### 3.1 分层逻辑

ABCZ 分层基于**销售额的帕累托累计分布**，叠加**零销识别**规则：

```
Step 1: 将所有SKU按月销售额降序排列
Step 2: 计算累计销售额占比
Step 3: 零销SKU（sold_weeks_8w = 0）直接归入D类
Step 4: 非零销SKU按累计占比分层：
    累计 0%-70%    → A类
    累计 70%-90%   → B类
    累计 90%-99%   → C类
    末尾 1% 或评分 < 30 → D类
```

### 3.2 四层分级标准

| 层级 | 销售额累计 | 评分区间 | 典型占比（SKU数）|
|------|-----------|----------|-----------------|
| **A 类** | 前 70% | ≥ 70分 | 约 15-25% |
| **B 类** | 70%-90% | 50-69分 | 约 30-40% |
| **C 类** | 90%-99% | 30-49分 | 约 20-30% |
| **D 类** | 末 1% 或零销 | < 30分 | 应 ≤ 10% |

**样本数据（乐易购佛山禅城店 116个SKU）：**
- A类：27个（23.3%）
- B类：74个（63.8%）
- C类：9个（7.8%）
- D类：6个（5.2%）

### 3.3 D 类（零销）触发条件

以下情况自动归入 D 类并触发下架任务：

| 条件 | 说明 |
|------|------|
| `sold_weeks_8w = 0` | 近8周完全无销售 |
| `sales_amt < 极小值 且 sold_weeks ≤ 1` | 销售额极低，接近零销 |
| `综合评分 < 30分` | 九维度综合评估极差 |

## 四、ABCZ 驱动的陈列面数管理

### 4.1 陈列面数分配规则

陈列面数（Facing）决定了一个SKU在货架上占据多少个正面展示位：

| ABCZ层级 | 建议面数 | 陈列位类型 | 黄金层优先 | 库存天数 |
|----------|----------|-----------|------------|---------|
| **A 类** | 3面 | 黄金层（视线层+伸手层）| ✓ 优先 | 7天 |
| **B 类** | 2面 | 常规位（中间区域）| — | 5天 |
| **C 类** | 1面 | 边角位（底层/顶层）| — | 3天 |
| **D 类** | 0面 | 撤出，清库处理 | — | 清库 |

### 4.2 面数调整触发场景

| 场景 | 触发条件 | 动作类型 |
|------|----------|----------|
| A类SKU未在黄金层 | A类但陈列位置偏差 | 调位任务 |
| A类SKU面数 < 3 | A类但面数不足 | 增面任务 |
| C类SKU占用2面以上 | 资源错配 | 减面任务 |
| D类SKU仍在架上 | 零销未清理 | 立即下架任务 |
| B类变为C类 | 连续2期评分下滑 | 面数缩减任务 |

### 4.3 陈列位类型与优先级

```
高价值陈列位（A类SKU竞争）：
├── 端架（入口端、主通道端）  →  引流款、核心款
├── 黄金层（正视线±10cm）   →  A类销量最高款
├── 入口堆头                 →  引流款、季节爆品
└── 收银台旁                 →  小包装、冲动消费品

普通陈列位（B类SKU）：
├── 中间货架常规层            →  B类主销款
└── 同类关联区域              →  搭配品、补充款

低价值陈列位（C类/测试）：
├── 顶层货架                 →  C类观察品
├── 底层货架                 →  C类小众品
└── 测试位                   →  新品测试款（D类候选前置区）
```

## 五、PDCA 循环：进销存闭环

基于单品管理理论，每个 SKU 的管理遵循以下循环：

```
Plan（计划）
    ↓  确定本期销量目标（基于历史数据和季节因子）
Do（执行）
    ↓  按陈列量目标摆货 → 按订货量目标补货
Check（检查）
    ↓  核查实际销量 vs 目标，统计剩余库存
Act（调整）
    ↓  更新ABCZ分层 → 调整面数 → 修订下期订货量
    ↓
（下一个 PDCA 周期）
```

**四个关键目标值：**

| 指标 | 定义 | 计算依据 |
|------|------|----------|
| **销量目标** | 本期计划销售数量 | 历史销量 × 季节系数 |
| **陈列量目标** | 货架上应摆放的数量 | 日销量 × 补货周期 × 1.3安全系数 |
| **订货量目标** | 向供应商下单的数量 | 陈列量目标 + 安全库存 - 现有库存 |
| **剩余量目标** | 期末库存健康范围 | 日销量 × 安全天数（A类7天，B类5天，C类3天） |

## 六、TF 效率函数（辅助分层）

TF 函数是 ABCZ 分层的重要补充工具，详见《零售数学模型》文档。在 SKU 分层中的应用：

### 6.1 ABCZ + TF 联合判断

| ABCZ | TF评级 | 综合判断 | 建议动作 |
|------|--------|----------|----------|
| A类 | 明星 | 🌟 最优SKU | 全力保障，3面黄金位，提高订货频次 |
| A类 | 良好 | ✓ 健康核心款 | 正常保障，关注是否可提升毛利 |
| A类 | 一般 | ⚠ 销量好但效率差 | 检查毛利率是否偏低，考虑调价 |
| B类 | 明星 | ↑ 有上升空间 | 考虑增面或调位，观察能否晋级A类 |
| B类 | 低效 | ↓ 需关注 | 重点观察，若下期仍低效则降至C类 |
| C类 | 任何 | 淘汰候选 | 1面观察，下期未改善则归入D类 |
| D类 | 任何 | 淘汰 | 立即下架，不受TF评级影响 |

### 6.2 TF 异常信号

| 现象 | 意义 | 建议 |
|------|------|------|
| A类SKU但TF < 0.8 | 销量不错但毛利极低或周转极慢 | 检查定价策略，考虑提价或精简规格 |
| B类SKU但TF ≥ 3 | 效率高但销量尚未爆发 | 优先关注，增加陈列面数支持 |
| 同品类均TF < 0.5 | 整个品类资金效率差 | 考虑品类角色重新定位 |

## 七、分层结果的系统输出

### 7.1 自动生成任务

系统根据分层结果，自动生成以下类型的执行任务（存入任务看板）：

```python
# 任务生成规则（伪代码）
for sku in scored_skus:
    if sku.tier == 'D' or sku.sold_weeks == 0:
        create_task(type='立即下架', priority='high', due_days=3)
    
    elif sku.tier == 'C' and sku.turnover_days > 25:
        create_task(type='减面', priority='medium', due_days=7)
    
    elif sku.tier == 'A' and sku.score >= 80:
        create_task(type='调位/增面', priority='medium', due_days=5)
```

### 7.2 品类级整改任务

当某个L2中类的D类数量 ≥ 2个时，系统生成品类整改任务，建议结合五角色模型对该中类进行整体重组。

### 7.3 健康度报警阈值

| 指标 | 预警线 | 严重线 |
|------|--------|--------|
| D类占比 | > 8% | > 15% |
| 零销SKU | > 5个 | > 10个 |
| A类占比 | < 15% | < 10% |
| 平均周转天数 | > 20天 | > 30天 |
