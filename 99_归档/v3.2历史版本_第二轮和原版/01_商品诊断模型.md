# 商品诊断模型

> 鲜选决策 v1.0 · 核心方法论文档  
> 适用门店：1000–2000㎡ 社区全品类超市

---

## 一、模型概述

商品诊断模型以**五层选品模型**为主干，从战略定位到陈列执行，形成逐层递进的分析框架。每一层的诊断结论直接驱动下一层的决策输入，确保选品决策有据可依、可执行、可追溯。

```
第1层  门店定位  ──→  确定目标客群、消费场景、价格形象
   ↓
第2层  品类角色  ──→  三级品类体系，五大品类角色分配
   ↓
第3层  单品筛选  ──→  ABCZ四层分级，TF效率评估
   ↓
第4层  商品组合  ──→  IP-Chart价格带，五角色结构平衡
   ↓
第5层  陈列映射  ──→  执行任务看板，陈列面数落地
```

## 二、第一层：门店定位

### 2.1 核心原则

> **先选人，再选品。** 不能对着商品去猜目标顾客，而是先锁定目标顾客，再围绕他们的生活场景和消费能力选品。

门店定位决定选品方向，定位不清是选品混乱的根本原因。

### 2.2 定位三要素

| 要素 | 定义 | 社区超市典型值 |
|------|------|----------------|
| **目标客群** | 谁是最核心的 80% 买家 | 家庭客群（35-55岁）、老年客群（60+）、宝妈客群 |
| **核心场景** | 他们来这里解决什么问题 | 日常补货、早餐配置、生鲜购买 |
| **价格形象** | 他们对这家店的价格认知 | 中等偏实惠、信任感强 |

### 2.3 选品策略四型

| 类型 | 核心逻辑 | 适用业态 |
|------|----------|----------|
| **安全导向型** | 以大众普遍品为主，现金流优先 | 社区小超市、便利店 |
| **差异竞争型** | 核心品 + 搭配品，提高毛利空间 | 精选超市、精品社区店 |
| **一站式覆盖型** | 多核选品，做加法，覆盖所有需求 | 大型综合超市 |
| **精选效率型** | 少而精，单核聚焦，做减法 | Costco 式会员店 |

### 2.4 诊断维度

- **画像完整度**：目标客群、核心场景、选品策略、竞争分析是否填写完整
- **定位与选品匹配度**：当前在售商品结构是否与定位一致（用IP-Chart验证）
- **竞争差异化程度**：对比周边竞争对手，差异化优势是否清晰可执行

### 2.5 诊断输出

```
门店定位诊断结论（示例）
─────────────────────────────────────────
✓ 目标客群明确：家庭、老年、宝妈三类主力
✗ 价格形象偏模糊：P域商品占比74%（低于目标80%），大众价格带不够强
⚠ 竞争差异化不足：与周边美宜佳、盒马同质化品类占比过高
建议：补充P域主流价格段商品，强化生鲜/熟食差异化优势
```

## 三、第二层：品类角色

### 3.1 五大品类角色

| 角色 | 英文 | 核心功能 | 典型品类 | SKU配置建议 |
|------|------|----------|----------|-------------|
| **引流品类** | Destination | 吸引客流，建立价格形象 | 蔬菜、矿泉水、食用油 | 引流款为主，薄利多销 |
| **主销品类** | Routine | 支撑销量主盘，稳定经营 | 液态奶、薯片、方便面 | 引流款+主销款均衡配置 |
| **利润品类** | Preferred | 毛利来源，差异化竞争 | 酸奶、调味品、卤味 | 利润款 + 主销款为主 |
| **形象品类** | Occasional | 门店形象，品类完整度 | 日用百货、纸品 | 覆盖齐全，不宜过度投入 |
| **机会品类** | Convenience | 季节/趋势机会，灵活配置 | 冷饮、节庆礼品 | 灵活调整，不宜常规化 |

### 3.2 诊断核心问题

针对每个品类，需判断：

1. **角色分配是否准确**：这个品类在本店的功能是引流、主销还是利润？
2. **角色与配置是否一致**：引流品类的SKU是否足够大众化？利润品类的毛利率是否达标？
3. **五角色比例是否合理**：通常建议引流:主销:利润约为 3:4:3

### 3.3 诊断信号

| 信号 | 说明 | 建议动作 |
|------|------|----------|
| 引流品类毛利率 > 30% | 定价偏高，失去引流功能 | 补充大众价格段商品 |
| 利润品类周转天数 > 20天 | SKU动销慢，资金占用高 | 精简低效补充款 |
| 无明显引流品类 | 缺乏客流锚点 | 建立价格形象旗舰款 |
| 形象品类占SKU总数 > 30% | 资源分配失衡 | 收缩形象品类数量 |

## 四、第三层：单品筛选

### 4.1 九维度评分体系

系统对每个 SKU 进行 100 分制综合评分，涵盖以下九个维度：

| 维度 | 权重 | 评分逻辑 |
|------|------|----------|
| 销售额贡献 | 高 | 近8周销售额绝对值及趋势 |
| 毛利额贡献 | 高 | 毛利额绝对值 |
| 动销率 | 中 | 8周中有销售的周数（0-8周） |
| 库存周转天数 | 中 | 越低越好，目标 ≤ 15天 |
| 价格带匹配 | 中 | 是否落在 P域 或 M域 |
| 规格带合理性 | 低 | 规格是否与目标客群匹配 |
| 关联销售贡献 | 低 | 与其他商品的连带购买率 |
| 供应稳定性 | 低 | 是否存在供货断货记录 |
| 门店适配度 | 低 | 是否符合门店定位和客群特征 |

### 4.2 ABCZ 分层规则

| 层级 | 分数区间 | 累计销售额占比 | 陈列策略 | 库存策略 |
|------|----------|----------------|----------|----------|
| **A类** | ≥ 70分 | 前 70% | 3面黄金位 | 7天安全库存 |
| **B类** | 50-69分 | 70%-90% | 2面常规位 | 5天库存 |
| **C类** | 30-49分 | 90%-99% | 1面边角位 | 3天库存 |
| **D类** | < 30分 / 零销 | 末 1% 或零销 | 0面，立即下架 | 清库处理 |

> **Z类（零销）**：连续8周销售额为0，自动归入D类，触发下架任务。

### 4.3 健康 SKU 结构参考

| 指标 | 健康水平 | 预警阈值 |
|------|----------|----------|
| A类占比 | ≥ 20% | < 15% 需重新选品 |
| D类占比 | ≤ 8% | > 10% 需立即清理 |
| 零销SKU数 | ≤ 5个 | > 10个需专项行动 |
| B+A类综合 | ≥ 80% | < 70% 结构失衡 |

## 五、第四层：商品组合

### 5.1 核心工具：IP-Chart 价格带分析

参见《零售数学模型》文档中的 IP-Chart 章节。

商品组合层的核心诊断问题：
- P域（大众价格带）SKU是否达到80%？
- M域（中高价格带）SKU是否有10-20%的利润空间？
- 是否形成"双波峰"价格分布（一个大众主波峰 + 一个中高价次波峰）？

### 5.2 五角色 SKU 结构

在每个中类（L2）内，五种功能角色的 SKU 应保持平衡：

| 角色 | 占比建议 | 功能 |
|------|----------|------|
| 引流款 | 10-20% | 价格锚点，吸引进店 |
| 主销款 | 35-45% | 主要销量来源 |
| 利润款 | 15-25% | 拉高整体毛利 |
| 补充款 | 10-15% | 完善选择面 |
| 测试款 | 3-8% | 新品孵化，持续迭代 |

### 5.3 诊断信号

| 现象 | 问题 | 建议 |
|------|------|------|
| 主销款 > 60% | 品类缺乏毛利支撑 | 引入2-3款利润款 |
| 补充款 > 20% | SKU冗余，资源浪费 | 清理重叠功能的补充款 |
| 无测试款 | 品类无法持续迭代 | 设置1-2个新品测试位 |

## 六、第五层：陈列映射

### 6.1 从分析到执行

五层模型的最终落点是可执行的陈列和补货动作：

| ABCZ层级 | 陈列位 | 面数 | 补货周期 | 库存天数 |
|----------|--------|------|----------|----------|
| A类 | 黄金层（视线层+伸手层）| 3面 | 每日或隔日 | 7天 |
| B类 | 常规位（中间层） | 2面 | 3-4天 | 5天 |
| C类 | 边角位（底层/顶层）| 1面 | 按需 | 3天 |
| D类 | 撤出 | 0面 | 清库或退货 | 0 |

### 6.2 陈列位类型

| 位置类型 | 特征 | 适合商品 |
|----------|------|----------|
| **端架** | 客流量最高，视觉焦点 | A类主销品、促销品 |
| **堆头** | 大量陈列，冲击力强 | 引流款、季节款 |
| **黄金层（视线层）**| 眼睛平视直达区域 | A类核心款 |
| **常规位** | 伸手可及的中间区域 | B类日常品 |
| **测试位** | 低流量角落区域 | 新品测试款、D类候选 |
| **收银台** | 高频接触，冲动消费 | 零食、口香糖、小商品 |

### 6.3 执行任务类型

| 任务类型 | 触发条件 | 优先级 |
|----------|----------|--------|
| **立即下架** | D类零销SKU | 高 |
| **调位** | A类SKU不在黄金位 | 中 |
| **增面** | A类SKU面数不足 | 中 |
| **减面** | C类SKU占用过多面数 | 低 |
| **品类整改** | 某中类D类数量 ≥ 2个 | 高 |
| **关联陈列** | 场景化组合机会 | 低 |

## 七、综合健康度评分

系统综合五层诊断结果，输出 0-100 的门店商品健康度：

| 维度 | 权重 | 评估内容 |
|------|------|----------|
| ABCZ 结构健康 | 30% | A+B类占比、零销率 |
| 毛利结构 | 25% | 综合毛利率、利润款占比 |
| 周转效率 | 20% | 平均周转天数达标率 |
| IP-Chart 平衡 | 15% | P域/M域结构符合度 |
| 执行任务完成率 | 10% | 高优任务执行进度 |

| 健康度 | 评级 | 说明 |
|--------|------|------|
| 80-100 | 优秀 | 结构健康，持续优化 |
| 65-79 | 良好 | 局部问题，定向改善 |
| 50-64 | 一般 | 需要系统性调整 |
| 35-49 | 偏差 | 需专项整改计划 |
| 0-34 | 较差 | 需根本性结构重建 |

## 八、诊断流程

```
数据导入（Excel/CSV）
    ↓
SKU 评分引擎（9维度）
    ↓
ABCZ 分层 + TF 计算
    ↓ ↗ ↗
IP-Chart 分析   CTM 矩阵分析
    ↓
五层诊断报告
    ↓
执行任务生成（自动 + 手动）
    ↓
看板跟踪执行
    ↓
下期数据导入（PDCA 循环）
```
