## C3. 进入 execute 的最低门槛
1. **行数**：≥ 全量门店 SKU（休食类至少数百行；不接受抽样冒充全量）。
2. **9 格全覆盖**：必须（否则裁决逻辑未全验）。
3. **异常分支全覆盖**：必须（缺货/新品/促销/负毛利/无库存等）。
4. **人工复核字段**：必须存在（哪怕默认空）。
5. **先跑 full dry-run**：必须（报每工具覆盖率/blocked/异常率）。
6. **用户签字**：必须（execute=写回客户数据，§3 协作契约⑤）。
> 六项全绿 + §3.1 九宫格确认（**已 active**）→ 才进 CODEX-Execute-Approval-001。

## C4. 数据来源方案（Batch-012 补·区分四类来源）
| 来源类型 | 可用于 | 不可用于 | 纪律 |
|---|---|---|---|
| **① 现场导出真实全量** | **真实 execute（唯一合法来源）** | — | 必须现场导出 + 用户确认；真实条码必须脱敏；**不得把真实 xls/csv 入库** |
| **② Codex 构造合成测试样本** | full dry-run 逻辑验证 / 测试 | 真实 execute | 仅造覆盖 9 格 + 异常分支的假数据，标注「合成·非真实」 |
| **③ 脱敏样本** | dry-run 演示 / 教学 | 真实 execute（脱敏后口径可能失真）| `{{EAN13_已脱敏}}`；不得反推真实值 |
| **④ 不允许的来源** | —— | 一切 | 抽样冒充全量 / 手改真实数据 / 未脱敏条码入库 / 直接 xls·csv 入 git |

> **铁律**：合成样本只能验逻辑；真实 execute 必须用「①现场导出 + 用户签字」。dry-run 用 ②/③，execute 用 ①。
