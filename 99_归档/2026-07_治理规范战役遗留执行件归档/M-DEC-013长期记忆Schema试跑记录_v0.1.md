# M-DEC-013 长期记忆 Schema 试跑记录 v0.1（Phase 0B 预演）

> **预演性质**：本文件用 M-DEC-013 走一遍 Schema v0.1.2 的 working→episodic→回填→晋升 机制，**只读 M-DEC-013、不改其正文、不改其 status、不升 active**。对应迭代方案 v3.0 §6「唯一动作」与 Schema §13 Phase 0B。
> **脱敏**：客户项目代号化（花厅坊=客户样板店），结构比例用抽象区间，不出 SKU 明细 / 条码 / 进价（CLAUDE §6 C 层）。

## 1. 四级记忆层定位（轴一）
| 层 | 命中? | 说明 |
|---|:--:|---|
| 现场观察层 working | （源头）| 单小类诊断中「尾部太厚」的定性现象 |
| **单店决策层 episodic** | ✅ 当前 | 一次完整判断：C+D≈40% SKU 仅产≈10% 销额 → 启动尾部审视 + 六桶防误杀 |
| 跨店规律层 semantic | 🔄 候选 | 同店三小类复现，但**未跨店**，暂不达标 |
| 标准动作层 procedural | ❌ | 未到（须先升 semantic）|

## 2. provenance（脱敏来源·轴一证据链）
- `来源::` 客户样板店 T03饼干 / T01膨化 / T05果干 三小类 L3→Item→SKU 脱敏实跑（C+D 占比约 40–48% 区间，产约 10% 销额）；
- `来源::` B 簇原典（商品 ABC / 双 ABC / 长尾消化）+ 六哥 IP（尾部临界化、跨小类背书）。
- **provenance 合规**：挂 3 实跑 + 原典，无打分（符合 §11 provenance 规则）。

## 3. 晋升门槛检验（§7·排除共因）— **关键发现**
| 门槛项 | 现状 | 判定 |
|---|---|---|
| ≥3 场景 | 3 小类（饼干/膨化/果干）| ⚠ 数量够 |
| **排除共因** | 3 小类**同店/同期/同口径** | ❌ **共因未排除**（同供应商生态/同客群/同季节窗口）|
| 结果同向 | 三小类均≈40-48% 产≈10% | ✅ 方向一致 |
| signoff | 未签 | ❌ |

> **试跑暴露的核心**：M-DEC-013 的 3 场景是**同店三小类**，按 Schema v0.1.1「排除共因」规则 **不算 3 个独立场景**（同店=共因）→ **不可升 semantic/active**。这正验证了「≥3 独立场景 + 排除共因」门槛的有效性——若按 v0.1 旧机械「≥3 场景」会误判达标。**需第二客户/第二门店独立复现方可晋升。**

## 4. 回填点与 resulting 路径（§8 四象限·v0.1.2）
- **回填点（2–4 周）**：瘦身比例临界值实测（砍多少/留多少），依赖库存 / 去年同期 / 订货数据补齐（当前 pending）。
- **resulting 四象限预案**（回填时定状态，本预演不落定）：
  | | 结果好 | 结果差 |
  |---|---|---|
  | 决策对 | 升 active 正例候选 | `果差但决策稳`：记外因（如供应商断供致缺货假滞销），不削弱判据 |
  | 决策错 | `侥幸`：不作正例 | `failed`：判据削弱/推翻（如「40% 阈值」实测系统性误杀刚需）|
- **supersession 可能路径**：若第二客户实测临界值显著不同 → 不删 v0.1，新建 v0.2 `被取代::` v0.1，记取代原因（客户结构差异）。

## 5. 0C 三问预答（供试跑后正式回答）
1. **字段够不够**？——M-DEC-013 已有现象/决策/推理/来源/pending；Schema 模板字段基本覆盖；**缺**「记忆层」「decision_status」两轴字段（M-DEC-013 原文未显式标，但可由本预演推断）。
2. **捕获顺不顺**？——M-DEC-013 是已成型决策，非现场 5 秒捕获场景；真正的捕获路径验证需等花厅坊**新**现场决策。
3. **门槛清不清**？——✅ 非常清楚：本试跑直接暴露「同店三小类=共因，不算独立场景」，门槛可操作。

## 6. 试跑结论
- ✅ Schema v0.1.2 机制**走得通**：层级定位、provenance、晋升门槛（排除共因）、resulting 四象限、supersession 路径均能套用到真实 M-DEC。
- ✅ **门槛规则有效**：成功拦下「同店三小类」伪跨店晋升——这是 v3.0「排除共因」微调的实证价值。
- ⚠ **捕获路径未充分验证**（M-DEC-013 是成型决策非现场速记）→ 需用花厅坊**新现场决策**补验。
- ❌ M-DEC-013 **不升 active**（共因未排除 + 缺第二客户 + 未 signoff）；本预演**未改其原文/status**。
