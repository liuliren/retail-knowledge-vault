# 99_作战层 · README

> **版本**:v0.2 · 2026-05-03
> **目的**:把"运行时项目文档"与"长期资产知识库"分离归档
> **设计哲学**:作战层会随项目周期更替,知识层是稳定资产 — 两者必须分层

---

## 一、为什么把作战层从知识库主体里剥离

```
v0.1 时期的问题:
  5 月作战 v3.0 / 数据看板 / 日记模板 / Backlog
  这些"运行时文档"和知识库主体混在一起

混合的后果:
  ① 知识库被作战节奏污染(每月迭代作战图 ≠ 知识库迭代)
  ② 项目结束后这些文档"过期"还混在知识体系里
  ③ 新人查看知识库,会被作战类文档干扰

v0.2 的解决方案:
  把作战层独立到 99_ 前缀目录
  说明:99_ 表示"末位",视觉上明确这一层是辅助
  日常打开知识库时优先看 0~5_,作战层只在需要执行时打开
```

---

## 二、作战层的定位

```
作战层 = 项目运行时文档
        随项目周期(月/季/年)更替
        过期后归档,不删除(留作历史复盘材料)

知识库主体 = 长期资产
            跨项目可复用
            稳定迭代,不随项目周期变化
```

## 四、作战层的生命周期管理

### 4.1 月度归档机制

```
每月底(如 5/31):
  Step 1:把当月主作战图 / 数据表 / 日记 归入"月度归档"子目录
          例:99_作战层/历史归档/2026-05/
  
  Step 2:从历史归档中提炼"案例文档"
          (如休食区调改案例 → 归入知识库 4_刻意练习层/案例集)
  
  Step 3:更新 99_README.md 的"当前作战层文件清单"为下个月
```

### 4.2 季度复盘机制

```
每季度(如 6/30 / 9/30 / 12/31):
  □ 回顾本季度的作战层文档
  □ 识别哪些经验值得提炼为知识节点
  □ 把可复用的内容反向流入知识层
  □ 该季度文档转入历史归档
```

### 4.3 年度盘点

```
每年 12/31:
  □ 整年的作战层归档进入"年度文件夹"
          例:99_作战层/历史归档/2026/
  □ 形成年度复盘报告
  □ 与刻意练习层的"年度自评"对接(参考 DP-00)
```

## 五、作战层 vs 知识层的边界

| 对比维度 | 作战层(99_) | 知识层(0~5_) |
|---|---|---|
| 时效性 | 短期(月/季) | 长期(年/多年) |
| 修改频率 | 高(每周更新) | 低(季度迭代) |
| 适用范围 | 当前项目 | 跨项目复用 |
| 内容性质 | "现在做什么" | "怎么做" |
| 失效方式 | 自然过期 | 不会失效,只会迭代 |
| 归档方式 | 月度/年度归档 | 不归档,持续迭代 |

**判断一份文档属于哪一层**:
- 如果 6 个月后这份文档对你还有用 → 知识层
- 如果 6 个月后这份文档过期 → 作战层
- 如果 6 个月后部分有用、部分过期 → 拆分:有用部分进知识层,过期部分进作战层归档

## 六、与知识层的反馈机制

```
作战层不是"用完就扔"
作战层是"知识层的真实战场"

反馈路径:

作战层(实战)
   ↓ 实战中发现的问题
   ↓ 实战中验证的方法
知识层(沉淀)
   ↓ 修正知识节点
   ↓ 新增知识节点
作战层(下次实战)
   ↓ 应用更新后的知识
   ↓ 形成新的反馈

→ 作战层与知识层是双向流动的"循环系统"
```

## 八、操作规范

### 8.1 文件命名约定

```
命名格式:[编号]_[文件名]_[版本号].[扩展名]

编号规则:
  00_:主控类(如 00_5月作战v3.0主作战图)
  C_ :数据类(C 表示 Charts/Data)
  F1~F3:复盘模板类(F 表示 Feedback)
  G_ :边界类(G 表示 Guard)
  v3:主作战版本(沿用 v3 命名)
```

### 8.2 修改规范

```
作战层文档修改:
  □ 大改:升版本号(v3.0 → v3.1)
  □ 小改:加日期标注(v3.0_20260510)
  □ 全新:升主版本号(v3 → v4)

修改后必须做的事:
  □ 在文档顶部更新版本号和修改日期
  □ 在文档底部加"修改记录"(简述改了什么)
  □ 旧版本不直接删除,移入"历史归档"
```

### 8.3 跨设备同步

```
建议:
  □ 主存储:Obsidian 本地仓库 + 云盘备份
  □ 跨设备:主作战图 + 数据表 同步到手机/平板
  □ 备份:每周 1 次完整备份到外置存储
```
