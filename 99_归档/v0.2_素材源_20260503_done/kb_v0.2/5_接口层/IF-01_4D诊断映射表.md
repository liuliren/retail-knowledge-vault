## 一、为什么是 4D 而不是其他?

```
设计原则:映射表必须服务"已经稳定的脑回路",
        而不是"一个还没成型的诊断系统"

DavidLiu 现场诊断的真实顺序(2026-05):
  D1 商品结构 → D2 价格 → D3 陈列 → D4 促销活动
  
  ↑ 这是"经典 4P"(Products / Price / Presentation / Promotion)的本土化

  - 7D 系统当前状态我看不到全貌,锚定它会做出"映射到空气"
```

### 3.2 4D 诊断的标准流程

```
Step 1:数据准备(诊断前 1~3 天)
  □ 全店 SKU 明细 + 90 天销售/库存数据
  □ 商圈半径 1.5km 内的居民结构
  □ 主要竞争对手清单

Step 2:D1 商品结构诊断(2~4 小时)
  □ 调用 MC-03 → 输出 TPOS 顾客画像
  □ 调用 MC-01 → 输出主要品类 HS 矩阵
  □ 调用 IM-04 → 输出 ABC-Z 清单

Step 3:D2 价格诊断(2~3 小时)
  □ 调用 MC-06 + MC-GBA-03 → 输出 IPC 图
  □ 调用 MC-09 + RM-06 → 输出 TR 红线表
  □ 调用 RM-04 → 输出毛利分布表

Step 4:D3 陈列诊断(1~2 小时,现场观察)
  □ 调用 IM-05 → 输出 ABC × 陈列错配清单
  □ 调用 CM-08 → 输出 Layout 评估

Step 5:D4 促销活动诊断(1~2 小时)
  □ 调用 MC-GBA-05 → 输出节庆响应度评估
  □ 调用 RM-01/02/03 → 输出销售根因报告

Step 6:综合诊断报告输出
  □ 4D 评分汇总
  □ 3 个最优先行动
  □ 实施时间表
```

## 六、使用注意事项

### 6.1 不要把 4D 当成"打分游戏"

```
4D 的本质是"判断框架",不是"评分量表"
评分只是"沟通工具"
不要为了凑分数而忽略真实问题

正确用法:
  ✅ "这家店 D1=2 分,主要问题是 Destination 品类不清晰"
  ❌ "这家店 4D=11 分,所以是好店"
```

### 6.2 4D 不能完全替代深度诊断

```
4D 覆盖:商品配置层 90% 问题
4D 不覆盖:
  □ 财务健康(资金链/账期)
  □ 团队管理(店长能力/员工执行力)
  □ 供应链(供应商质量/采购效率)
  □ 数字化(POS 系统/会员体系)

→ 大型项目(年度顾问)需要 4D + 财务 + 团队 三轨并行
```

### 6.3 大湾区版的 4D 与外地版的差异

```
本表是大湾区版,核心差异:
  D1:Destination 默认包括"生鲜+本地特色"
  D2:P 域上限按大湾区收入水平校准(MC-GBA-03)
  D3:陈列习惯(散装为主、动线短、本地场景)
  D4:节庆节奏按大湾区版(MC-GBA-05)

去外地诊断时:
  □ 必须重新校准 D1 的 Destination 候选品类
  □ 必须重新校准 D2 的 P/M 分界点
  □ 节庆节奏完全不同(参考全国通用 52 周 MD)
```
