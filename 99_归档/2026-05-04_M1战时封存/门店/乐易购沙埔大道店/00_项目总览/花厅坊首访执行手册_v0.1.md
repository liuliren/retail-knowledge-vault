# 花厅坊首访执行手册 v0.1

## 1. 文件定位

本文件是乐易购花厅坊店首访调研的**现场执行 SOP**，用于指导首访全程的任务分工、现场采集动作、拍摄标准、数据导出方式和资料提交规范。

本文件解决的问题是**怎么做**，而不是**收集什么**。

| 文件 | 职责 |
|------|------|
| [[乐易购花厅坊店_资料采集清单]] | 收集什么、优先级、状态追踪 |
| 本文件 | 怎么做、拍摄标准、命名规则、提交方式、分析路径 |
| [[乐易购花厅坊店_门店基础档案]] | 门店固定属性档案，首访后持续填充 |
| [[乐易购花厅坊店_项目总览]] | 项目背景与诊断方向 |

**重要限制：**

- 本文件仅定义采集动作和提交规范，不包含诊断结论。
- 所有不确定信息统一标注"待确认"，不得预设结论。
- 门店实际情况以首访现场为准，预设假设不得写入正式分析。

---

## 2. 首访目标

首访不是全面诊断，而是**建立事实基础**。

首访结束时需完成以下目标：

| 目标类型 | 具体内容 |
|---------|---------|
| 事实采集 | 完成 [[乐易购花厅坊店_资料采集清单]] 中 P1 级资料的现场采集或数据导出准备 |
| 现场观察 | 完成门店全区域走访，形成初步动线与陈列印象记录 |
| 关系建立 | 与老板、店长完成首次访谈，对齐项目目标与配合意愿 |
| 系统确认 | 确认门店使用的系统名称、数据可导出字段和导出权限 |
| 商圈走访 | 完成门店周边步行圈竞品走访，记录名称、位置、业态 |

首访**不需要**完成：

- 诊断结论
- 整改方案
- 品类规划
- 年度商品计划
- 任何形式的正式交付物

## 3. 首访前准备

### 3.1 时间安排（建议）

| 事项 | 建议时长 | 说明 |
|------|---------|------|
| 门店内部走访 + 拍摄 | 2～3 小时 | 含各品类货架、端架堆头、冷柜、仓储区 |
| 老板访谈 | 30～60 分钟 | 经营目标、痛点、历史背景 |
| 店长访谈 | 20～30 分钟 | 日常运营、执行痛点、人员情况 |
| 系统数据确认 + 导出准备 | 30～60 分钟 | 确认系统名称、导出字段、安排导出时间 |
| 商圈走访 + 竞品拍摄 | 1～2 小时 | 步行覆盖主要竞品，记录位置和业态 |
| **合计建议时长** | **5～7 小时** | 视门店规模和配合情况调整 |

> 建议首访选在门店客流高峰时段，便于观察客群结构。**高峰时段待确认**（首访前向门店确认）。

### 3.2 携带物品清单

| 物品 | 说明 |
|------|------|
| 手机（主拍） | 用于现场拍摄，出发前清理存储空间，确保至少 10GB 可用 |
| 备用设备 | 防止主设备存储不足或电量耗尽 |
| 充电宝 | 全天拍摄耗电量大，务必携带 |
| 笔记本 / 纸 | 现场手绘平面布局草图、记录访谈要点 |
| U盘 / 移动硬盘 | 数据导出用（可用格式待门店确认） |
| 名片 | 建立关系用 |

### 3.3 出发前确认事项

- [ ] 已与门店确认首访时间与接待人
- [ ] 已向老板/店长说明来访目的（资料采集 + 诊断准备，非检查）
- [ ] 已确认门店是否提前准备好平面图/布局图（如有）
- [ ] 已确认门店系统负责人是否首访当天在岗（数据导出需要）
- [ ] 手机电量充足、存储空间充足

## 4. 现场采集执行要点

> 本节只写执行要点，不复写完整采集清单。完整采集项目及优先级见 [[乐易购花厅坊店_资料采集清单]]。

### 4.1 商圈地图截图

**执行要点：**

- 使用高德地图或百度地图，截取以门店为中心的 500 米圈截图（含主干道、居民区名称）
- 截图时关闭"美食/购物"等商业推荐层，保留道路、小区、地名
- 对无法在地图自动识别的竞品，首访后手动补充标注
- 在现场用地图打点确认门店精确坐标并截图存档

> 门店精确坐标**待确认**（首访现场核实）。

### 4.2 门店平面图 / 陈列图

**执行要点：**

- 优先获取门店已有的平面布局图或施工图（草图亦可）
- 如门店无图，现场手绘草图，最低要求：标注入口位置、主通道走向、扶梯/直梯位置、收银台位置、主要品类区域名称、冷柜区、生鲜区
- 手绘草图现场拍照存档，标注"草图-待核实"，不作为正式平面图使用
- 特别关注：顾客从商场入口到超市入口的进入路径（动线前置）

> 门店是否为地下一层（B1）、进出口数量和位置**待确认**。

### 4.3 竞品照片采集

**已知竞品名单（全部待确认，以首访现场实际情况为准）：**

| 竞品名称 | 业态 | 距离 | 拍摄要点 |
|---------|------|------|---------|
| 百欢（待确认） | 待确认 | 待确认 | 门头全景、主通道、主力品类价签 |
| 乐优选（待确认） | 待确认 | 待确认 | 门头全景、主通道、主力品类价签 |
| 钱大妈（待确认） | 生鲜（待确认） | 待确认 | 入口外观、陈列方式、主力品类价格 |
| 赵一鸣（待确认） | 零食折扣（待确认） | 待确认 | 入口外观、陈列方式、客流观察 |
| 罗森（待确认） | 便利店（待确认） | 待确认 | 门店外观、即食区 |
| 美宜佳（待确认） | 便利店（待确认） | 待确认 | 门店外观、门头 |
| 水果店（待确认） | 水果专营（待确认） | 待确认 | 规模、价格区间、客流观察 |
| 其他现场发现竞品（待确认） | 待确认 | 待确认 | 现场判断后补充 |

**拍摄行为规范：**

- 每个竞品至少拍摄：门头全景 + 入口内部 + 1～2 个主力品类货架或价签
- 以正常顾客行为完成，不得在竞品店内摆出明显拍摄姿势
- 每张照片拍摄后立即记录对应竞品名称和拍摄时间

### 4.4 门店现场照片

**P1 必拍（首访必须完成）：**

| 拍摄对象 | 执行说明 |
|---------|---------|
| 门店外观（含招牌） | 正面、侧面各一张 |
| 入口区域 | 进入门店后第一视线全景 |
| 主通道全景 | 沿主通道拍摄连续全景（建议连拍 3～5 张接片） |
| 各品类货架（含品类标签） | 每个品类区域正面全景，需拍到货架顶端和底端 |
| 端架（全部） | 每个端架正面全景，含价签 |
| 堆头（全部） | 每个堆头全景，含商品和价签 |
| 收银台前区 | 含冲动购买区商品陈列 |

**P2 选拍（首访尽量完成）：**

| 拍摄对象 | 执行说明 |
|---------|---------|
| 黄金货架层位（腰部层位） | 选取 3～5 个品类的黄金层位特写 |
| 典型品类价签 | 3～5 个品类的价格区间抽样 |
| 冷柜陈列 | 冷柜全景及分区状态 |
| 动线死角 | 顾客不易到达的区域（如柱子后方、角落区域） |

**P3 可拍（需门店授权）：**

| 拍摄对象 | 执行说明 |
|---------|---------|
| 库存区/仓储 | 须提前获得门店明确授权，不强制 |
| 员工操作区 | 须提前获得门店明确授权，不强制 |

### 4.5 经营数据采集

**首访当天的核心任务是确认，而不是当场完成导出。**

首访当天需确认并记录：

- [ ] 门店使用的系统名称和版本（**待确认**）
- [ ] 销售明细可导出的字段列表（**待确认**）
- [ ] 毛利数据是否在系统内、是否可导出（**待确认**）
- [ ] 库存数据是否实时、是否可导出（**待确认**）
- [ ] 历史销售数据可追溯年限（**待确认**）
- [ ] 数据导出权限负责人及联系方式（**待确认**）
- [ ] 数据导出格式（优先 .xlsx 或 .csv，不接受截图）

如当场可导出，按 [[乐易购花厅坊店_资料采集清单]] P1 级优先执行。

### 4.6 客流补充信息

**现场观察要点（非访谈，靠眼睛记录）：**

- 顾客年龄构成大致比例（老年 / 中年 / 年轻 / 亲子）
- 顾客平均购物篮商品件数（抽样观察 10～20 名顾客）
- 是否出现大包购买行为（大件搬运）或只买 1～2 件的随机补充购买
- 高峰时段客流密度状态

**访谈获取（向店长询问）：**

- 日均客流量估算（**待确认**）
- 高峰时段（**待确认**）
- 主要顾客来源小区（**待确认**）
- 是否有线上订单（饿了么/美团/小程序）及大致占比（**待确认**）

### 4.7 其他补充资料

| 资料 | 采集方式 | 说明 |
|------|---------|------|
| 老板访谈记录 | 面谈，现场速记或录音（征得同意后） | 重点：经营目标、最大痛点、近期销售趋势感知 |
| 店长访谈记录 | 面谈，现场速记 | 重点：日常补货流程、陈列执行、一线痛点 |
| 门店营业时间 | 现场确认 | 早开/晚关时间；是否全年无休 |
| 员工数量与岗位构成 | 访谈获取 | 粗略了解，不深入 |

## 5. 拍摄标准

| 项目 | 标准 |
|------|------|
| 分辨率 | 不低于 1080p（手机正常拍摄模式即可） |
| 清晰度 | 货架价签文字需能清晰辨认（特写时焦距对准价签） |
| 角度 | 货架拍摄以正面平视为主，避免俯拍或仰拍造成变形 |
| 光线 | 优先自然光或门店照明，不使用闪光灯（避免货架反光） |
| 数量 | 宁多勿少，同一货架从不同角度各拍一张 |
| 完整性 | 每个品类货架确保拍到顶层和底层，不遗漏层位 |
| 实时标注 | 拍摄后立即在备忘录中记录照片对应的品类区域和位置描述 |

**禁止行为：**

- 禁止用截图代替实拍
- 禁止删除"拍坏"的照片（模糊、角度错的照片保留，归入待核实）
- 禁止将多张照片拼图合并（每张独立保存）

## 6. 数据导出标准

| 项目 | 标准 |
|------|------|
| 文件格式 | 优先 Excel (.xlsx)，其次 CSV (.csv），不接受 PDF 或截图 |
| 最低字段要求 | 条码 / 商品名称 / 规格 / 销售数量 / 销售金额 / 日期 |
| 时间范围 | P1：近 90 天销售明细；P2：近 12 个月月度汇总 |
| 来源标注 | 每份数据文件注明：门店名称、系统名称、导出时间、导出人 |
| 原始数据保护 | 导出后不得修改原始数据，清洗版本另存副本 |
| 存储位置 | 原始数据归入 `99_原始素材/经营数据/`，清洗版本归入 `13_数据分析与工具脚本` |

> 可导出字段列表和导出权限**待确认**（首访当天向系统负责人确认）。

## 9. 首访后分析主线

首访资料采集完成后，按以下主线推进初步诊断：

```
商圈判断
   └─ 门店位置是否有商圈优势？周边客群密度如何？竞品截流风险在哪里？
      ↓
竞品截流
   └─ 哪些竞品在哪些品类上对花厅坊构成直接截流？截流路径是什么？
      ↓
客群购买任务
   └─ 主力顾客群是谁？他们来这里解决什么生活任务？
      ↓
动线问题
   └─ 门店动线是否顺畅？死角在哪里？黄金区域有没有被有效利用？
      ↓
品类角色
   └─ 各品类在门店中承担什么角色（引流/利润/季节/常规）？
         当前品类角色分配是否合理？
      ↓
商品结构
   └─ 各品类的 SKU 数量、价格带、品种覆盖是否与客群购买任务匹配？
```

> 以上分析主线为首访后的分析路径框架，不是首访当天的诊断结论。
> 每个环节必须基于首访采集的真实数据和现场观察，不得预设结论。
