# 品类体系综合版 v1.2 · 晟果咨询知识结晶

> **来源** — 综合 v3.2 第三轮主版四份核心文档（02 主纲 / 02A 3+1架构 / 02B 复杂类决策 / 07 落地总表）+ 五份 vault 实证品类表（D04 / D05 / 700 SKU 表 / 完整品类表 L4 拆分版 / 晟果咨询完整品类表 L4 全展开）+ 花厅坊休食区 5 个面战役实证 + GPT 04 陈列与空间效率母版。
>
> **定位** — 晟果咨询品类体系的"知识结晶"。**所有方法论 / 实战 / 系统的根基**。本文件是实战调用首选；v3.2 4 份原文是详细论述。
>
> **核心命题** — 品类体系不是分类游戏，是顾客购买路径的物理映射。3+1 架构的本质是：**用最少的层级，承接最多的购买决策**。

---

## 0. Executive Summary（一页就懂）

```
品类体系 = 主树 × 类目属性 × 纳入状态 × 经营维度外挂

主树（3+1 架构 / 不混经营维度）：
  L1 战略板块 → L2 中类组合 → L3 核心分析 → [L4 决策组（仅高复杂类）]
                                                  │
                                                  └─ 4 启用模板 + 5 复杂类模板

类目属性（4 类 / 决定是否启用）：
  标准类 / 可选类 / 区域增强类 / 预留类

纳入状态（3 态 / 决定怎么用）：
  纳入 / 条件纳入 / 暂不纳入

经营维度外挂（不进主树 / 用 tag）：
  节庆 / 散称 / 礼盒 / 经营区 / 区域模板 / 时令窗口

板块模板（5 大 / 决定整体风格）：
  A 快消常温 / B 生鲜食材 / C 乳品日配 / D 家庭基础原料 / E 功能型非食
```

**三个核心命题**：
1. **主树稳定 / 经营维度外挂** — 节庆 / 礼盒不进 L2
2. **进主树 ≠ 进分析** — 三态共存让"保留 vs 启用"分离
3. **L4 仅高复杂类启用** — 避免 L4 = L3 克隆体

## 一、3+1 架构 · 6 层定义

### 1.1 层级语义（v3.2 02 §3.4）

| 层级       | 战略定义        | 落地作用   | 系统作用          | 冲调饮品例              |
| -------- | ----------- | ------ | ------------- | ------------------ |
| **L1**   | 战略资源分配层     | 顶层板块   | 首页 / L1 分析    | 食品饮料               |
| **L2**   | 品类组合管理层     | 中类结构层  | 中类组合分析        | 冲调早餐/冲饮            |
| **L3**   | SKU 执行与分析层  | 核心分析层  | L3 分析 / 规则主入口 | 麦片谷物 / 咖啡冲饮 / 奶粉冲调 |
| **L4**   | 决策组层（仅高复杂类） | 增强层    | 决策差异分析增强      | 即食麦片 / 三合一咖啡       |
| **Item** | 顾客理解层       | 需求表达层  | 中间承接层         | 早餐快泡燕麦 / 学生奶粉      |
| **SKU**  | 执行最小单位      | 任务最终落点 | 评分 / 任务 / 反馈  | 桂格即食燕麦片 1000g      |

### 1.2 层级承接逻辑

```
L1 解决战略问题（板块预警）
   ↓
L2 解决组合问题（中类失衡）
   ↓
L3 解决结构问题（扩 / 稳 / 调 / 压）
   ↓ [L4]
L4 解决决策差异问题（仅复杂类）
   ↓
Item / SKU 解决执行动作（任务 + 反馈）
```

**关键判据**（v3.2 02 §五）：
- L1-L2 = 战略层 → 给老板看
- L3 = 业务层 → 给采购 / 品类经理看
- L4-SKU = 执行层 → 给店长 / 理货员看

## 二、类目 4 属性 × 纳入 3 态（决策矩阵）

### 2.1 类目 4 属性

| 属性 | 含义 | 默认行为 | 落地举例 |
|---|---|---|---|
| **标准类** | 默认启用 + 默认进分析 | 自动纳入 | 食品饮料 / 生鲜 / 乳品日配 / 粮油调味 / 冷冻食品 / 日用百货 |
| **可选类** | 进体系 / 按模板或深度启用 | 条件启用 | 酒类 / 净菜半成品 / 冷藏甜品 / 小家电 / 冷冻烘焙 |
| **区域增强类** | 仅区域模板命中后成立 | 模板触发 | 汤料滋补干货（广东模板） |
| **预留类** | 当前保留 / 不进标准分析 | 不纳入 | 现制即食 |

### 2.2 纳入 3 态

| 状态 | 含义 | 系统后果 |
|---|---|---|
| **纳入** | 进入默认分析 + 规则 + 页面 | 全链路启用 |
| **条件纳入** | 模板 / 启用后才进 | 半启用（结构在 / 分析待） |
| **暂不纳入** | 保留结构 / 不进标准评分 | 仅占位（数据隔离） |

### 2.3 4 属性 × 3 态决策矩阵

| | 纳入 | 条件纳入 | 暂不纳入 |
|---|---|---|---|
| **标准类** | ⭐ 默认 | （罕见 / 暂时禁用）| （罕见 / 临时停）|
| **可选类** | （模板触发后）| ⭐ 默认 | （未启用模板）|
| **区域增强类** | （模板触发后 / 区域成立）| ⭐ 默认 | （未触发）|
| **预留类** | ❌ 不应该 | （罕见）| ⭐ 默认 |

## 三、5 大板块模板（v3.2 02 §四 · 决定整体风格）

> 所有 L1 板块按以下 5 大模板归类。新板块讨论时，**先判断属于哪个模板**，不要从零讨论。

### 3.1 模板 A · 快消常温板块

**适用对象**：饮料 / 休闲零食 / 坚果炒货 / 饼干糕点 / 方便速食 / **冲调早餐/冲饮** / 糖巧果冻 / 蜜饯果干凉果

**标准判断顺序**：
1. 顾客消费任务是否差异明显
2. 价格带 / 品牌带差异是否明显
3. 规格带 / 人群带是否差异明显
4. 是否需要启用 L4

**何时优先启用 L4**：
- 同一 L3 下品牌和价格带分化明显
- 同一 L3 下儿童 / 家庭 / 功能型明显不同
- 同一 L3 下动作建议容易误伤

**系统承接特征**：更容易进 L4 / 模板增强 / 价格带与陈列类规则。

### 3.2 模板 B · 生鲜食材板块

**适用对象**：蔬菜 / 水果 / 肉禽蛋 / 水产 / 冷冻水产 / 豆制品 / 净菜半成品

**标准判断顺序**：
1. 天然食材属性
2. 保鲜 / 鲜度 / 冷冻状态
3. 烹饪场景差异
4. 是否需要启用 L4

**何时优先启用 L4**：
- 同一 L3 下鲜活 / 冷冻 / 净切决策路径明显不同
- 同一 L3 下家庭主菜 / 快手菜 / 礼赠路径不同
- 同一 L3 下任务 / 补货 / 损耗逻辑明显不同

**系统承接特征**：更依赖经营区与损耗逻辑 / 鲜度规则 / 强调日常经营健康度。

### 3.3 模板 C · 乳品日配板块

**适用对象**：乳品 / 酸奶乳酸菌 / 早餐日配 / 冷藏甜品

**标准判断顺序**：
1. 冷链属性
2. 消费时点（早餐 / 日常 / 即饮 / 家庭）
3. 规格与人群差异
4. 是否需要启用 L4

**何时优先启用 L4**：
- 同一 L3 下儿童 / 功能型 / 家庭装差异明显
- 同一 L3 下常温 / 低温路径不同
- 同一 L3 下短保管理动作明显不同

**系统承接特征**：更依赖冷链能力模板 / 短保 / 日清 / 周转规则。

### 3.4 模板 D · 家庭基础原料板块

**适用对象**：米面粮油 / 调味品 / 南北干货 / 汤料滋补干货 / 烘焙原料

**标准判断顺序**：
1. 烹饪任务
2. 基础原料 / 复合调味 / 汤料滋补
3. 区域差异是否明显
4. 是否需要启用 L4

**何时优先启用 L4**：
- 同一 L3 下基础型 / 升级型 / 区域型差异明显
- 同一 L3 下烹饪场景完全不同
- 同一 L3 下规则容易在佐餐与主烹饪调味之间混伤

**系统承接特征**：更依赖家庭烹饪任务 / 与区域增强模板结合 / 用"烹饪任务"解释结构。

### 3.5 模板 E · 功能型非食板块

**适用对象**：个护 / 家清 / 纸品 / 女性护理 / 居家环境 / 一次性用品 / 居家纺品袜拖 / 部分小家电

**标准判断顺序**：
1. 功能需求
2. 使用场景
3. 人群差异
4. 规格 / 价格带差异
5. 是否需要启用 L4

**何时优先启用 L4**：
- 同一 L3 下功能路径差异明显
- 同一 L3 下使用场景完全不同
- 同一 L3 下人群差异明显（儿童 / 女性 / 敏感型等）

**系统承接特征**：更容易与功能型规则结合 / 用"功效型解释" / 适合按功能 + 场景做决策组。

## 四、L4 启用判据（4 模板 · 决定何时拆 L4）

> 来源：v3.2 02A 第三轮 §三。**所有 L4 启用判断必须按这 4 模板统一引用**，不要每次自然语言重述。

### 4.1 L4 启用 4 模板

| 模板编号 | 名称 | 核心问题 | 启用判据 |
|---|---|---|---|
| **L4-PATH-01** | 决策路径分叉型 | 顾客购买路径是否已明显分叉 | **是**：不同人群走不同路径（学生 vs 中老年）|
| **L4-BAND-01** | 价格带/规格带分叉型 | 价格带 / 规格带是否对应不同购买任务 | **是**：普惠 ≤10 vs 主销 10-30 vs 礼盒 50+ |
| **L4-SCENE-01** | 场景/经营表达分叉型 | 使用场景 / 经营区 / 陈列逻辑是否显著不同 | **是**：早餐冷食 vs 热泡 vs 燕窝混冲 |
| **L4-RULE-01** | 规则误伤风险型 | 不拆 L4 是否会让规则解释失真 | **是**：散称 vs 包装的规则不同 / 不拆会误伤 |

### 4.2 L4 启用 4 步决策流程

```
对一个 L3，按顺序问 4 个问题：

1. PATH 问：顾客买这个时，是否会按"对象 / 用途 / 人群" 走不同路径？
   是 → L4-PATH-01 启用
   否 → 进下一问

2. BAND 问：价格带 / 规格带的差异 / 是否对应不同任务？
   是 → L4-BAND-01 启用
   否 → 进下一问

3. SCENE 问：不同 SKU 是否在场景 / 陈列逻辑上显著不同？
   是 → L4-SCENE-01 启用
   否 → 进下一问

4. RULE 问：不分 L4 是否会让规则误伤其他子集？
   是 → L4-RULE-01 启用
   否 → 不启用 L4
```

### 4.3 L4 不启用情形

- L3 已足够细 / 顾客购买路径不分叉
- SKU 数量少 / 不足以支撑分组
- 启用后 L4 ≈ L3（克隆体 / 反而模糊主树）
- 启用 L4 仅为了"显得专业" — **不允许**

## 五、重点复杂品类决策（5 模板 · 决定怎么拆 L4）

> 来源：v3.2 02B 第三轮 §三。**所有重点复杂类讨论必须按这 5 模板统一引用**。
>
> 与 §四 4 模板的关系：§四回答**是否启用 L4**；本节回答**启用后怎么拆**。

### 5.1 复杂类 5 模板

| 模板编号 | 名称 | 核心回答的问题 | 典型适用类 |
|---|---|---|---|
| **CC-PATH-01** | 路径分叉型复杂类 | 顾客是否先按不同购买路径筛选，再在路径内比较商品 | 饮料 / 零食 / 个护 |
| **CC-BAND-01** | 价格带/规格带分叉型复杂类 | 价格带 / 规格带是否已表达不同购买任务 | 饮料 / 调味 / 乳品部分 |
| **CC-SCENE-01** | 场景表达型复杂类 | 顾客是否先按使用场景或经营场景做第一层判断 | 调味 / 家清 / 零食部分 |
| **CC-FUNC-01** | 人群/功能分叉型复杂类 | 顾客是否先按功能或人群做第一层判断 | 个护 / 饮料部分 |
| **CC-GIFT-01** | 礼赠/时段强化型复杂类 | 是否存在礼赠路径或时段强化路径，与日常路径明显不同 | 零食 / 饮料 / 节庆强化结构 |

### 5.2 复杂类 5 模板输入输出

| 模板 | 主要输入 | 主要输出 | 后续承接 |
|---|---|---|---|
| **CC-PATH-01** | 顾客任务 / 决策路径差异 / 路径冲突风险 | 路径分叉说明 | 02A L4 启用 / 03 L4 分析 / 页面路径解释 |
| **CC-BAND-01** | 价格带 / 规格带 / 任务预算差异 | 预算/规格分叉说明 | 价格带分析 / 结构缺口解释 |
| **CC-SCENE-01** | 使用场景差异 / 经营区差异 / 陈列逻辑差异 | 场景分叉说明 | 场景型分析 / 经营表达区 |
| **CC-FUNC-01** | 功能诉求 / 人群差异 / 品牌与安全感差异 | 功能/人群分叉说明 | 功能型解释 / 人群型规则增强 |
| **CC-GIFT-01** | 礼赠路径 / 时段强化 / 展示逻辑差异 | 礼赠/时段分叉说明 | 礼赠分析 / 节庆与展示解释 |

### 5.3 02A 4 模板 vs 02B 5 模板的关系

```
02A 4 模板（L4 启用判据 / 决定"是否拆"）
   ↓ 启用后
02B 5 模板（怎么拆 / 决定"按什么维度拆"）

L4-PATH-01 启用 → CC-PATH-01 给具体拆法
L4-BAND-01 启用 → CC-BAND-01 给具体拆法
L4-SCENE-01 启用 → CC-SCENE-01 / CC-GIFT-01 给具体拆法
L4-RULE-01 启用 → CC-FUNC-01 / 其他给具体拆法
```

## 六、主树 5 条铁律（v3.2 02 + 07 综合）

| # | 铁律 | 反例（错误做法）| 正例 |
|---|---|---|---|
| 1 | **主树稳定 / 经营维度外挂** | ❌ 把"节庆糖巧" 作为 L2 | ✅ 节庆作为 tag 外挂 / "糖巧果冻" 是 L2 |
| 2 | **进主树 ≠ 进分析** | ❌ 把所有 L3 都打"纳入" | ✅ 三态共存 / "现制即食"暂不纳入 |
| 3 | **L4 仅高复杂类启用** | ❌ 给所有 L3 都加 L4 | ✅ 按 4 模板判断 / 不启用就不启用 |
| 4 | **现制即食单独保留** | ❌ 强行纳入标准分析主链 | ✅ 锁定前 3 个 L2 / 不进标准评分 |
| 5 | **区域增强类需模板触发** | ❌ 让所有店都启用汤料滋补干货 | ✅ 仅广东模板命中后成立 |

## 七、经营维度外挂（不进主树 / 必须 tag）

主树**不直接承接**以下经营维度（v3.2 02 §2.2）：

| 维度 | 为什么外挂 | 用什么承接 |
|---|---|---|
| **节庆** | 跨多 L2 / 季节性 / 不是稳定结构 | tag: `节庆` + 经营维度字典 08 |
| **散称** | 经营表达层 / 不是品类本身 | tag: `散称` + 规则引擎 11 |
| **礼盒** | 包装形式 / 跨多 L3 | tag: `礼盒` + 陈列规则 |
| **经营区** | 物理布局 / 不是品类逻辑 | 经营区字典 |
| **区域模板** | 地域增强 / 模板触发 | tag: `region:guangzhou_*` |
| **时令窗口** | 季节性 / 跨多类 | tag: `season:spring_late_humid_shift` |

**反幻觉硬约束**：把任何上述维度作为 L2 / L3 = 主树污染（v3.2 02 §2.2 红线）。

## 八、跨模块承接关系（品类体系是上游）

```
1.2 品类体系（本模块 / 上游根基）
   ↓
   ├→ 1.1 总入口诊断（v3.2 01）：用主树 L1-L4 入口诊断 / DX-STORE / DX-L1 / DX-L3 / DX-SKU 模板
   ├→ 1.4 品类分析逻辑（v3.2 03）：在 L3 / L4 上做分层分析
   ├→ 1.5 SKU 分层（v3.2 04）：用主树定位 SKU 角色（SL-V/M/LC/R/T 五模板）
```
