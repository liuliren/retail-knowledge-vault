# 2026-05-07 休食 SKU 去留表 v0.1（模板 / 待填实）

> **本表是 [[M-DEC-007]] §3 S5 调整动作决策的输出模板**。每个 SKU 必须经过 S2 六维交叉 + S3 ABC-Z 分级 + S4 归因决策树 后填入本表。
>
> **铁律**（[[CLAUDE.md]] §13.6 / §13.13）：
> - 不可仅凭销量决定去留
> - 每个 SKU 必须有归因标签（来自 S4 决策树）
> - 每个动作必须有依据链（指向 S2 六维数据 + S4 归因）

---

## §1. 表格定位

**输入**：休食区全 SKU 六维交叉表（[[M-DEC-007]] §3 S2 出口）+ ABC-Z 分级（S3）+ 归因标签（S4）

**输出**：本表 → 老板/店长 sign off → 实施清单（[[2026-05-07_休食专项作战手册_v0.1]] §F）

**Sign off 入口**：[[16_客户与战役档案/花厅坊样板/05_决策记录]]

---

## §2. 字段定义

| 字段 | 含义 | 来源 |
|---|---|---|
| SKU 条码 | 13 位条码 | 商品档案 |
| SKU 名称 | 完整名称 | 商品档案 |
| 当前层位 | 哪组哪层 | 现场摸排 |
| 当前排面数 | 排面数量 | 现场摸排 |
| ABC-Z 等级 | A/B/C/Z | M-DEC-007 §3 S3 |
| 90 天销量 | 件 | POS |
| 90 天销售额 | 元 | POS |
| 毛利率 | % | 商品档案 / POS |
| 当前库存 | 件 | ERP |
| 库存周转天数 | 天 | 计算 |
| 缺货率 | % | 现场摸排 + ERP |
| 活动期标记 | 自然 / 促销 / 混合 | 老板口述 + ERP |
| 客群标签 | TPOS / CDT | 现场观察 + 老板访谈 |
| **归因标签** | 陈列差 / 活动期外 / 价格虚高 / 缺货 / 客群不匹配 / 真不需要 / 健康 | M-DEC-007 §3 S4 决策树 |
| **决策动作** | 见 §3 取值表 | 综合决策 |
| 优先级 | P0 / P1 / P2 | 库存压力 + 毛利损失 |
| 实施时点 | 5/10 / 5/11 / 5/12 / 5/13 | 老板 sign off 后定 |
| 责任人 | 六哥 / 店长 / 店员 | 同上 |
| 备注 | 特殊说明 | 自由 |

## §3. 决策动作取值表

| 动作 | 触发条件（S4 归因 + S3 等级）| 注意 |
|---|---|---|
| **保留+维持** | A 级 / 健康 | 不轻动 |
| **保留+增排** | A 级 / 缺货率高 / 顾客需求未满 | 占用其他 SKU 排面 |
| **保留+调位** | C 级 / 陈列位差 | 测试 1 周后再决（不淘汰）|
| **保留+调价** | C 级 / 价格虚高 | 测试 1 周后再决 |
| **保留+配活动** | C 级 / 活动期外低 | 进 [[S06_营销活动配套]] |
| **减排** | B 级 / 重复 SKU | 释放排面给增排 |
| **淘汰** | Z 级 / 归因明确 / 老板认可 | 配套 [[单品管理]] 滞销淘汰 |
| **补新 SKU** | 排面释放后 → [[2026-05-07_新商品引进表_v0.1]] | 进 [[单品管理]] 新品引进 |

## §5. 决策依据链（必填）

每个 SKU 决策必须可追溯到：
- **S2 六维数据**：哪一维触发决策（销售低 / 毛利低 / 库存压 / 缺货 / 陈列差 / 活动外 / 客群不匹配）
- **S3 等级**：A / B / C / Z
- **S4 归因**：决策树落点
- **S6 sign off**：老板 / 店长 ack 截图 / 时间
