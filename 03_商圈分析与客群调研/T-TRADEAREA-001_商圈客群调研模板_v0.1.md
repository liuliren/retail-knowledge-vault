## 用途

社区生鲜超市**开店前选址评估**或**在营店客源诊断**时，快速摸清商圈盘子、竞品格局、客群画像的最小可用调研模板。目标：半天现场 + 半天整理，产出「这个商圈能养多大的店、客是谁、抢谁的生意」的结构化判断。

> 客户真名、精确门店地址属可识别信息，**留本地，对外脱敏**。

---

## 一、商圈半径划分（先画盘子）

社区店客源高度依赖步行/短距，按三圈分层：

| 圈层 | 半径 | 定位 | 采集重点 |
|---|---|---|---|
| **核心圈** | 步行 5 分钟（约 300–500m）| 生死盘，贡献主力客流 | 常住户数、社区类型、人口结构 |
| **次级圈** | 步行 10–15 分钟（约 800m–1km）| 补充客流，受竞品拦截 | 竞品位置、过路动线 |
| **辐射圈** | 骑行/短驾（约 1.5–3km）| 弱关联，仅特色品/周末 | 差异化锚点吸引力评估 |

**采集清单**：核心圈常住人口/户数、小区数量与档次、有无学校/菜场/综合体等人流发生器、拆迁/新盘/产业搬迁等盘子变化信号（对应 [[KB-DX-TRAFFIC-001_客流人效诊断方法论_v0.1]] R5 位置/客源根因）。

## 二、竞品清单（谁在抢）

半径 1km 内逐一登记：

| 字段 | 说明 |
|---|---|
| 竞品名/代号 | 对外脱敏 |
| 业态 | 生鲜超市/菜场/社区团购点/便利店/大卖场 |
| 面积估计 | ㎡ |
| 距本店距离 | 步行分钟数 |
| 生鲜强弱 | 强/中/弱（核心竞争维度）|
| 价格带 | 低/中/高 |
| 客流高峰 | 观察其早/晚峰人气 |
| 差异化点 | 有无本店没有的杀手品类 |

**判断**：竞品生鲜是否强于本店？有无价格/品类差异化空间？对手最近动作（新开/大促）是否在分流本店客流。

## 三、客群画像（客是谁 · 买什么任务）

现场观察 + 拦访 15–30 人：

- **年龄结构**：老人 / 中年家庭主理人 / 年轻上班族 / 学生家长 各占几成
- **职业/收入**：本地退休 / 打工家庭 / 白领 / 学生家庭
- **购买任务**（按时段）：早市散买菜（老人）/ 午间即食 / 晚高峰顺路补货 / 周末计划性大采购
- **购物半径与频次**：每周来几次？还去哪买菜？
- **未被满足的需求**：想买买不到的品类？对现有店最不满的点？

> 客群画像直接决定品类底座与差异化锚点选择——如「小学 + 老社区」组合 → 儿童零食 + 中老年常备双锚点。

## 四、动线（客怎么走 · 客怎么来）

- **过店动线**：主要人流从哪个方向来（地铁口/小区门/菜场/学校）？门口过店人流量分时段目测。
- **到店方式**：步行/电动车/开车占比，有无停车/停车位。
- **入口可见性**：招牌/门头在主动线上是否显眼，有无遮挡。
- **竞品截流点**：客流在到达本店前会先经过哪些竞品。

## 五、数据采集规范（保证可复用可对比）

1. **分时段采集**：过店客流至少测早峰、晚峰、平峰各一段（15 分钟外推），标明采集日期/天气/是否节假日。
2. **同口径连续测**：绝对值不必精确，**趋势与量级可信即可**；同一时段、同一方法多次测比单次精测更有价值（[[KB-DX-TRAFFIC-001_客流人效诊断方法论_v0.1]] §五）。
3. **拦访样本**：客群拦访 ≥15 人，覆盖不同时段，避免只在单一时段采集导致画像偏斜。
4. **剔除异常日**：促销日/节令日客流结构异常，须单独标注或剔除，不套用平日系数。
5. **脱敏归档**：竞品与客户信息代号化后归档；原始照片/定位不进云层。

---

## 产出物

- 商圈三圈盘子判断（能养多大店 / 盘子在扩还是缩）
- 竞品格局图 + 差异化空间结论
- 客群画像（年龄/职业/任务/时段）→ 品类底座与锚点建议
- 过店客流分时段基线（供后续客流诊断做趋势对比）

## 张力 / 局限

- 本模板为**最小可用版**，不含精确人口 GIS 数据与消费力测算——那些需政府/第三方数据支撑。
- 现场目测客流误差大，仅用于**趋势与相对判断**，不作绝对承诺。
- 单次调研是快照，商圈盘子变化（新盘入住/拆迁）须持续跟踪。
