# DisplayMap 手工绘制工具包 v0.1（draft·待六哥碰）

> **这是什么**：不用任何软件，店长拿一支笔、几张 A4（或一个 Excel 空表）就能把门店货架"建成图"的傻瓜流程。产出的登记表是错配诊断和 DisplayMap 调整表的输入件。
> **它不是**：陈列方法论（方法口径全部回指 01_ SSOT，见文末装配声明）；也不是缺口总账 #16 的"DisplayMap 无代码工具"（那是 9 月后事件驱动件，本件只做纸面版）。
> **场景锚定**：500–2000㎡ 社区生鲜超市；执行者是店长/理货员，每步都设计成"照抄就能填"。

---

## 全流程一张图（4 步）

```
第①步 给货架编号（半天，一次性）
   ↓
第②步 逐架登记（每架 3-5 分钟，全店 1-2 天）
   ↓
第③步 标错配（顾问带店长做，半天）
   ↓
第④步 出调改清单（顾问出，店长照单干）
```

---

## 第①步｜货架编号规则（指向，不重定义）

编号口径**唯一权威**是 [[2026-05-19_晟果新零售陈列资源命名治理规范_v1.0]]。本节只给店长版速记，不新造规则。

**店内简化格式**（门店内上下文明确时）：

```
区域-G排号-面-子货架
例：XS-G1-A-01 = 休食区 第1排货架 A面 左起第1组
```

店长三句话记住：
1. **区域用两个字母**：休食=XS、生鲜=SX、酒水=JS、调味=TW、米面粮油=LY、日化=RH（全表见命名规范 §6，门店没有的区域不启用）。
2. **货架从入口方向数**：第 1 排=G1，往里递增；每排分 A/B 面；每面子货架从左到右 01、02……（强制两位）。
3. **端架=T、堆头=D、冷柜=COLD**（见命名规范 §4），编完贴一张手写号码贴在货架侧柱上，理货员抬头能看见。

⚠️ 编号一旦贴上就**不再改**（编号稳定优先于口语名称——命名规范总原则 3）。门店原有叫法（如"零食那排"）写进门店适配表当别名，不进编号。

## 第②步｜逐架登记表（核心模板，照抄能填）

每个**子货架**填一张（或 Excel 一行一层）。层号**自下而上数**：第 1 层=最底层。层位含义（形象层/黄金层/补充层）不用店长判断，判断口径在 [[分层到陈列一致性模型_v1.0]] §④，由顾问后续套用。

### 登记表模板

```
门店：＿＿＿＿  登记人：＿＿＿＿  日期：＿＿＿＿

架号：＿＿-G＿-＿-＿＿    货架总层数：＿＿    本架大品类：＿＿＿＿
```

| 层号 | 这层放的品类（小类） | 主力单品（卖最好的1-2个，写品名不写条码） | 面陈数（这层横着数共几个"脸"） | 缺货/异常（√并写一句） |
|---|---|---|---|---|
| 5（顶层） | | | | |
| 4 | | | | |
| 3 | | | | |
| 2 | | | | |
| 1（底层） | | | | |

**面陈数怎么数**（理货员版口径）：站在货架正前方，这一层从左到右能看到几个"商品的脸"就是几；同一单品并排 3 个=3 个面。数不清的深堆/散堆写"堆"字，不硬数。

**三条傻瓜纪律**：
1. **只登记看到的，不登记应该的**——货架上是什么就写什么，哪怕明知摆错了。
2. **缺货也要登记**：空排面写"空"，别跳过（缺货致低销≠商品差，"假A类"口径见 [[M4_陈列货位簇]]）。
3. **每架拍一张全景照**，照片按命名规范命名（同角度，整改后还要拍对比照，见 [[S05_陈列整改与执行]] DoD-2）。

### 填写示例（示例·虚构）

| 层号 | 品类 | 主力单品 | 面陈数 | 缺货/异常 |
|---|---|---|---|---|
| 4 | 魔芋制品 | 某魔芋爽 | 8 | |
| 3 | 卤味零食 | 某鸭脖 | 6 | |
| 2 | 卤味零食 | 某鸡爪 | 6 | 右侧2个面空 |
| 1 | 大袋分享装 | 某混合坚果 | 4 | |

## 第③步｜错配标记法（指向，不重定义）

**什么算错配**，口径唯一权威是 [[KB-CAT-DX-MISMATCH-001_销售陈列错配诊断_v0.1]]（stable·六哥签字）：

> 销售-陈列比 = 该小类销售占比 ÷ 该小类陈列空间占比
> **比值 > 1.5 → 增格候选；比值 < 0.5 → 缩格候选**；0.7–1.5 之间视为匹配。

本工具包只规定**怎么在纸上标**（三色笔法，店长/理货员能懂）：

| 标记 | 含义 | 对应口径 |
|---|---|---|
| 🔴 圈红 | 卖得好但地方小（比值>1.5） | 增格候选 |
| 🔵 圈蓝 | 卖得差但占地大（比值<0.5） | 缩格候选 |
| ⚠️ 三角 | 好卖的不在好位置 / 差卖的霸着好位置 | 层位错配（错配四型之④，见 [[M4_陈列货位簇]] §6） |
| "空"字 | 该有货的排面空着 | 先按缺货处理，不判商品差 |

**操作顺序**（顾问带店长做）：
1. 顾问从 POS 拉各小类 30 天销售占比（数据侧走 /diagnose 或三件套，本件不管数据加工）。
2. 陈列空间占比从第②步登记表算：该小类占的子货架数 ÷ 本面位子货架总数（简化算法即错配诊断 §二第二步）。
3. 算比值 → 在登记表对应行打红/蓝圈。
4. **打完圈不等于就要动**：还要过品类角色校验（目的性品类比值低是补强不是缩——错配诊断 §二第五步），这一步是顾问判断，店长不做。
5. 新品不满 30 天、节令旺季前夕**不标缩格**（失效条件见错配诊断文末）。

### 标记示例（示例·虚构）

> XS-G2-A-03 登记表上：魔芋制品占 1 格，30 天销售占比 4%，空间占比 16.7%，比值 0.2 → 🔵 圈蓝（缩格候选）；卤味零食占 1 格，销售占比 26%，比值 1.6 → 🔴 圈红（增格候选）。

## 第④步｜从登记表到调改清单（判读步骤）

调改动作的**词表唯一权威**是 [[DisplayMap调整表_模板定义_v0.1]] §7 的 12 类陈列动作（换位/上移/下移/放大排面/缩小排面/恢复陈列/清退腾位/新品上架/端架堆头尝试/维持不动/待陈列数据/现场复核），本节不新增动作类型。落位方法回指 [[SOP-02_陈列规划_v1.0]] 规划 5 步。

**判读五步**（顾问执行）：

1. **汇总**：把全店打了圈的行抄成一张"错配点清单"（架号+层号+品类+圈色）。
2. **排序**：按"比值偏离 1 的程度"从大到小排，**只取前 5–10 个动手**（先动错配最严重的几处——错配诊断核心原则；每日 3–5 撬动性任务纪律同源）。
3. **配动作**：每个错配点从 12 类动作里选一个——🔴 红圈→放大排面/换位/上移；🔵 蓝圈→缩小排面/清退腾位/下移；⚠️ 三角→上移或下移；"空"→恢复陈列（先补货再判）。
4. **缺数据不硬给**：没层位/排面/照片信息的点，动作一律填"待陈列数据"+补采清单，**不给精确货位**（铁律见 [[DisplayMap调整表_模板定义_v0.1]] §9）。
5. **落表**：清单逐行填进 [[DisplayMap调整表_Excel工具版_使用说明_v0.1]] 的主表 D/E 组，责任人+完成日期到人到天，交店长执行（执行流程走 [[S05_陈列整改与执行]]）。

### 调改清单示例（示例·虚构）

| 序 | 架号 | 层 | 品类 | 圈 | 动作（12类内） | 责任人 | 期限 |
|---|---|---|---|---|---|---|---|
| 1 | XS-G2-A-03 | 3 | 卤味零食 | 🔴 | 放大排面（+1格，吃魔芋让出的位） | 店长A | T+3天 |
| 2 | XS-G2-A-03 | 4 | 魔芋制品 | 🔵 | 缩小排面（1格→半格） | 理货员B | T+3天 |
| 3 | XS-G1-A-01 | 2 | 某小类 | ⚠️ | 现场复核（疑似层位错配，缺照片） | 顾问 | T+2天 |

## 装配声明

本件是**装配层工具**：货架怎么编号、什么算错配、有哪些动作、怎么落位，口径全部回指 01_/10_ SSOT；本件只贡献"纸笔可跑的流程编排 + 登记表模板 + 三色标记法"。SSOT 更新时本件跟随，不反向定义。
