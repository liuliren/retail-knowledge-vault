# 大湾区社区生鲜节令营销日历 v0.1（draft）

> **这是什么**：一张按月排的营销档期工作表——什么节、当令什么货、广府顾客在过什么日子、适合打什么活动、提前多久备货。用途：策划活动时定档期（配合 [[门店营销活动策划模板_v0.1]] 第 3 栏）。
> **与 SSOT 的分工**：节令对品类销量的**影响系数与诊断修正**是 [[KB-SEASON-MATRIX-001_大湾区节令品类影响矩阵_v0.1]] 的地盘（三相系数/K因子/超季节性SKU规则），本页不重复、不另造系数。本页只是**排期视图**：日期、货、场景、动作类型、提前期。
> **反幻觉声明**：本页只写公认节令事实与岭南时令物产常识（fact_layer: inferred，民俗与物产为公知，具体到店销量影响须查节令矩阵并以本店 POS 校准）；不含任何销售数据或销量基准。**备货提前期为经验建议值〔pending·待实测校准〕**。农历节日公历日期逐年浮动，使用前查当年日历。

## 每月「社区生鲜可打的点」（500–2000㎡ 门店视角）

- **1月**：① 打边炉组合（肥牛/丸料/萝卜/茼蒿一站买齐）承接冷空气；② 年货干货区提前开档，抢在大卖场促销前锁定社区客。
- **2月**：① 意头菜专区（生菜/蒜苗/发菜蚝豉）+ 广府意头标牌，成本极低的氛围抓手；② 春节后段主动去化年货库存，防节后积压。
- **3月**：① 祛湿汤料关联堆头（回南天是广府全民话题，土茯苓/薏米/鸡骨草）；② 清明烧腊预订开单，用预订锁量降损耗。
- **4月**：① 清明套装（烧肉+荞菜+甘蔗）按预订交付，短窗品荞菜不留库存；② 春笋/春菜"尝鲜"POP 教育型试吃。
- **5月**：① 粽子预售+包粽原料专区（自己包的家庭是社区店独有客群）；② 早荔上市抢"第一口鲜"心智，宁少量高频不压货。
- **6月**：① 荔枝节做形象档（岭南人对荔枝品种有鉴别力，桂味/糯米糍分级标示=专业感）；② 端午后立刻切换消暑主题，不恋战。
- **7月**：① 消暑瓜类堆头（冬瓜切件卖=小家庭友好）；② 暑假亲子时段试吃（下午客流谷期激活，接 KB-INSTORE-MKT-001 T3 纪律）。
- **8月**：① 润燥梨/龙眼水果档承接"秋老虎"；② 月饼预售开单——社区店拼不过礼盒渠道，主打散装/现售/街坊口味。
- **9月**：① 中秋家宴生鲜（鸡/海鲜/田螺）是社区店相对礼盒渠道的主场，月饼只做配角；② 柚子/芋头/菱角拜月光组合堆头。
- **10月**：① 煲汤季开锣=大湾区社区生鲜全年最强关联陈列窗口（猪骨+陈皮/淮山/眉豆，广府用语标牌）；② 重阳敬老主题（老龄客是社区店基本盘，做情感不做折扣）。
- **11月**：① 腊味开档仪式感（"秋风起"POP+试吃），腊味是高毛利+强节令双属性品；② 打边炉一站式副展台常设至次年2月。
- **12月**：① 冬至档按"第二个年"投入资源（做冬鸡/烧腊/汤圆），广府冬大过年；② 年货预售启动，用社区群预订对冲春节备货风险。

## 使用规则（装配接口）

1. **定档期** → 查本表选节点，倒推备货提前期，填 [[门店营销活动策划模板_v0.1]] 第 3 栏。
2. **估影响/修诊断阈值** → 一律去 [[KB-SEASON-MATRIX-001_大湾区节令品类影响矩阵_v0.1]] 查三相系数与 K 因子，本页不承载系数。
3. **月饼/粽子/汤圆等超季节性 SKU** → 走节令矩阵 §5 独立进度门控，不按常规备货逻辑。
4. **客群校验**：本表民俗以广府客群为默认；外来务工客群主导的门店，节令拉动系统性弱化（见节令矩阵 §2 K因子），档期投入相应降档。粤西籍客群聚集社区可关注正月–二月「年例」（茂名/湛江习俗）加菜需求。
