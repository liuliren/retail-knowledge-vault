# 中层决策库 — M-DEC 编号体系

## 1. 定位

L4 中层方法论是 vault 三层方法论(L-顶 哲学 / L-中 决策框架 / L-底 工具)中**当前最稀缺的一层**。

中层判据:**可教给伙伴的"在 X 场景下做 Y 决策的判断链"**。粒度比工具粗、比哲学细。

**反例**(不是中层):
- "Z2 砍了 6 个" = 结果(战役级实例)
- "不动销该砍" = 哲学(L-顶)
- "TR-sku 公式" = 底层工具(L-底)

**正例**(是中层):
- "L4 区域调改净减幅度规则:单区域净减 15%-25% 之间,<15% 顾客无感,>25% 投诉风险陡增"

## 2. M-DEC 编号体系

- **格式**:M-DEC-XXX,顺序编号
- **文件名**:`M-DEC-XXX_<决策名>_v0.X.md`
- **跨战役复用**:同一 M-DEC 被多场战役引用 = 中层方法论被验证
- **沉淀路径**:战役级草稿(`16_/<战役>/09_中层方法论沉淀.md`)→ 战后 G06 复盘 → 提炼到本目录(剥离客户敏感)

## 3. 每条 M-DEC 必含 5 字段

| 字段 | 说明 |
|---|---|
| 决策名 | 简短可记 |
| 适用场景 | 在什么情境下用这条决策 |
| 推演链 | 从场景到决策的完整推理 |
| 临界值 | 数字化阈值(< X 不做 / X-Y 做 / > Y 投诉) |
| 反事实 | 推翻条件(在什么情况下这条规则失效) |

## 5. 创建新 M-DEC 的流程

1. 战役期间在 `16_客户与战役档案/<战役>/09_中层方法论沉淀.md` 内累积草稿
2. 战役 G06 战后复盘时,验证草稿(被验证 / 被推翻 / 需修正)
3. 验证通过的草稿剥离客户敏感后,作为 `M-DEC-XXX_<决策名>_v0.1.md` 入库本目录
4. 后续战役引用时自动累积"被验证次数",更新本 README §4 索引
