# 中层决策库 promote 机制 · 制度草案 v0.1

> **法理声明**：draft·审议轮次1·**未启用**。不修改本目录 README（README=编号体系权威，改它须签字）。操作细则见 [[16_客户与战役档案/花厅坊样板/2026-07-08_MDEC_promote机制SOP_draft_v0.1|promote SOP draft]]，本文只定制度骨架，两文件分工=制度/操作。

## 1. 什么卡可以进中层库
同时满足：M-DEC 编号体系内 + status active + 六哥签字 + 有实跑证据文件 + 通过就绪度 4/4（见 §2）。**候选卡（_候选后缀）、draft 卡一律不可进**——中层库是"已验证判断链"的仓库，不是暂存区。

## 2. 就绪度评分
四项检查（状态签字/证据/蒸馏去向/命名合规），4/4 出预审单，<4 出差距清单。评分只判"可否进预审"，**不替代六哥终审**。

## 3. SSOT 归属
入库版=唯一权威版；原战役卡冻结为历史件。消歧三件套：`promoted_from` 字段 + `promote_date` 字段 + 原卡指针块。

## 4. 原卡处理
默认 copy 保留（战役档案完整性优先）；原卡文末追加指针块（签字前候选措辞/签字后完成措辞）；原卡此后不再更新，更新只发生在库内版。

## 5. 签字要求
每例迁入=1 个签字点（终审包一次呈批四处 diff）；SignoffLedger 每例一行；**任何会话不得代签**。

## 6. 方法论登记
入库同时向对应方法论页 §实战登记表追加一行（五列格式）；不改理论正文；卡内临界值不得抽象为通用规则。

## 7. lint 检查项（建议进 G03 v3）
① promoted_from 反向存在性；② 同名双卡缺消歧字段→报双权威；③ 历史件复活检测（原卡 promote 后再被修改）；④ 库内卡 status 必须 active（出现 draft/candidate 即违规入库）。

## 8. 回滚机制
签字前：restore + 删 untracked 副本，零残留。签字后：不删库内版，errata 旁注 + 台账补记；若判断链本身被推翻→库内版标 `deprecated` 走状态机，不物理删除。

## 9. 禁止事项
不自动迁入 / 不批量迁入（每例单独走）/ 不迁候选卡 / 不顺手改卡正文 / 不改 README / 不把库内卡当"已过标准审议的铁律"引用（卡是判断链，不是标准）。

## 10. 首例迁入检查表（通用化自 M-DEC-010 实测）
- [ ] 就绪度 4/4
- [ ] 预审单（裁决点1：迁入/指针/蒸馏去向）
- [ ] dry-run 报告（裁决点2：SSOT/措辞/新建/登记行）
- [ ] 工作区 4 处 diff（04_新建/原卡指针/方法论登记行/台账行）
- [ ] 终审包呈批
- [ ] 六哥签字 → 单主题 commit → 台账终审行

## 11. 示例：M-DEC-010（正例）
active v1.0 已签 + 三小类实跑复现 + 蒸馏去向（品类管理§实战登记）明确 + 命名合规 = 4/4 → 首例迁入流程 8 步走通（终审签字待落）。

## 12. 反例：M-DEC-007（当前不可 promote）
draft v1.0 + signoff_readiness: conditional——§9 四门未全开（跨店/signoff/三维数据/AAR），部分验证≠可入库。**它的存在证明本机制的筛子有效：部分验证的卡被拦在库外，防止"事实在用"污染"法理已验"。**
