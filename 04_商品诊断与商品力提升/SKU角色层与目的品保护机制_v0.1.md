# SKU 角色层与目的品保护机制 v0.1

> **结论先行**:现有体系（ABC / 动销 / 长尾治理）**只判断"表现"，不判断"存在意义"**——会误杀引流品（低销带客流）、形象品（低销撑品质感知）、结构补位品。本方法补上 **SKU"存在理由层"**：**任何"汰换/压缩/优化"决策，必须先过角色校验层，而非仅依赖销量/毛利指标。**
> 来源:打捞 [[2026-06-25_归档打捞backlog_v0.1]] #4（经营型 Taxonomy 任务书 §5.4 五角色词表 + GBA 防误杀双保险）。是 [[品类管理]] §3.3B 与 [[单品管理]] §7.3 的 SSOT。

## 0. 理论根:商品综合效用 PSI（没有绝对好坏·按作用动态评估）

> **结论先行**:本机制全部规则的理论根 = **没有绝对的好商品/坏商品,只有"有没有发挥预期作用"的商品**。SKU 角色层之所以能凌驾于销量/毛利排名之上做防误杀,正源于此:**角色 = 商品被赋予的"预期效用"**;评一个商品,先问它该发挥什么作用、有没有发挥,而不是先看它销量高不高。
> 来源:零售老刘《数据分析架构系列》12「商品综合效用 PSI · 商品重要度」(转述)。（逐字批43回填·零售老刘《数据分析架构系列》12·六哥2026-06-26「先回填」签）

1. **单一指标不足以判好坏**:销量好但毛利低?毛利高但周转慢占库存?周转高但占排面大?看似该砍的"坏商品"淘汰后,老客户流失、关联商品也跟着不好卖。**任何单一维度(销量/销售额/毛利率)都不构成"好/坏商品"的判据**——这正是 §2.1「不能单独作为淘汰依据的指标」的理论来源,也呼应 [[单品管理]]§16 误区1「只看销量判好坏」。
2. **商品综合效用(PSI·商品重要度)= 按作用做差异化考核**:商品在卖场承担不同作用,就该用不同指标侧重评估——
   - 取毛利的商品(→ 利润角色):考核 **销售额 / 毛利额 / 毛利率**;
   - 促销/引流的商品(→ 引流角色):考核 **人气度 / 销量 / 周转 / 关联度**。
   这就是 §1 五角色「处理策略」列与 §2.1「按角色用不同指标判」的底层逻辑:**先定角色(预期效用),再选对应指标考核**,而非一把尺量所有 SKU(对接 §3 起的 9 维评分也按角色侧重读)。
3. **判据金句(老刘原话)**:"只要某商品发挥了预期的作用(价值),就是好商品;没发挥作用,就不是。""如何让有效用的好商品越来越多 = 超市永恒课题。"
4. **指标是动态的**:同一 SKU 的考核指标随 **季节 / 营销重点 / 商圈特点** 变(旺季的季节品按当季效用评、平季按常态评;主推期的引流品看连带、非主推看自身)。呼应 §1 季节角色「按季节日历进退」与 §2.1 防误杀——**效用是情境化的,所以判据也必须动态切换**,不存在一张永恒的好/坏清单。
5. **与目的品保护打通**:**目的品保护 = 保护"承担引流/形象效用"的角色不被销量排名误杀**。引流品的预期效用是带客流(不是自身赚钱)、形象品的预期效用是撑品质感知(不是自身高销)——只要它们在发挥这层效用,就是"好商品",不进汰换池。§2 整个保护层,本质是 PSI「按效用判好坏」在淘汰决策上的落地闸;与 §1「生态位 ≠ 自身利润」(逐字批26)同根。

> **一句话锚点**:角色层不是"给 SKU 贴标签",而是"先认它被赋予的预期效用、再用对应指标判它有没有发挥"——**PSI 是根,五角色(§1)是这棵根长出的分类,目的品保护(§2)是据此设的防误杀闸**。

## 1. SKU 角色分类模型（锚于晟果五角色）

> **主角色五选一，辅角色可空选一**（晟果五角色词表唯一来源,禁另立角色字段）。ChatGPT 式"利润/引流/形象结构"三分,是下表的**保护视角**简化映射(见 §2)。

| 商品角色 | 定义 | 判定逻辑（可操作） | 允许低销? | 处理策略 |
|---------|------|------------------|:---:|---------|
| **引流** | 带客流入店的价格敏感/高关注款 | 高关注度词 + 价格敏感 + 进店带动（连带/客流时段贡献） | ✅ 允许 | 🔒 防误杀:保排面、控价、不凭销量砍 |
| **形象** | 建品质/信任感知，未必高销 | 品类完整性必需 + 品质标杆 + 顾客感知强 | ✅ 允许 | 🔒 防误杀:保留撑感知，按形象价值判 |
| **利润** | 贡献毛利额的盈利款 | 毛利额（非毛利率单看）排名靠前 | ⚠️ 看毛利额 | 看毛利额贡献,非单看销量 |
| **补充** | 满足长尾/一站式需求的常规款 | 无引流/形象/季节/锁定标记的默认角色 | ❌ 不豁免 | 低销 + 无角色 → **优先汰换候选** |
| **季节** | 节令/节庆带动款 | 挂季节日历（24节气/52周表） | ✅ 当季允许 | 按季节日历进退,非看常态销量 |

**GBA 锁定品类（大湾区差异化锚点·一等公民）**:凉果/陈皮/煲汤料/中式糕点/烧腊等带**锁定标记**，独立于五角色，进入**所有淘汰判断的防误杀通道**。

> **引流品防误杀的深层逻辑:生态位 ≠ 自身利润(零售老刘逐字批26回填·六哥 2026-06-25 签)**:几乎所有店的鸡蛋(白皮)都是**微利引流品**——其价值**不在自身利润,而在它在关系网络中的"生态位"**:① 维系与价格敏感型顾客的"初级价格形象关系"(鸡蛋价偏高→第一印象坏→关系难建);② 最大限度激活与店内高毛利品的"关联关系"。**所以引流品的考核单位不是单品利润,而是它带动的购物篮/客流/关联销售**——这正是 §2.1"不能仅凭销量/销售额排名砍引流品"的根。诊断推论:评一个引流品别看它自己赚多少,看"砍掉它,关联品类客流跌多少"。形象品同理(承载高端客群"身份认同关系"+撑门店"高品味信任")。

### 1.1 角色黄金比例基准（零售老刘逐字批24回填·六哥 2026-06-25 签）

> 来源:零售老刘《品类建模》159「核心的逻辑」四大角色(转述)。**给五角色配上结构占比基准**——此前角色闸只判"该不该砍单品",缺"整个品类的角色配比健不健康"的对照尺。

| 角色（老刘三分→晟果五角色映射） | 黄金占比基准 | 典型标签 |
|---|:---:|---|
| **引流品**（→引流） | **约 10–25%** | 高频/低价/知名品牌·敏感商品 |
| **贡献品**（→利润，含部分补充） | **约 60–80%** | 高复购/高毛利/高性价比·自有品牌/主力品类 |
| **形象品**（→形象 + GBA锁定 + 当季季节） | **约 5–10%** | 独特/高端/有故事/高颜值/稀缺 |
| **补充品**（→补充） | **占比最低** | 低频但不可或缺 |

- **用法**:作 **品类级结构异常的对照基准**,非硬阈值。实测显著偏离(如引流品>30%=头部过宽/利润被引流挤压;形象品=0=无品质感知支撑;贡献品<50%=利润盘太薄)→ 结构信号,挂动作。与 [[单品管理]]§7.3「标准ABC基准(品种A10/B30/C60)」并列为两把对照尺:**ABC 看销售额结构,角色比例看存在意义结构**。
- **落地**:Item 表新增「商品角色」标签列,判据=各角色占比。先标角色 → 再看占比偏离 → 才进单品级角色闸(§2.2 五步)。**先看品类角色配比,再砍单品**(避免"逐个砍补充品却没发现整品类缺引流")。

## 2. 目的品保护机制（强制保护层）

> ChatGPT 三类映射:**利润品=利润；引流品=引流；形象/结构品=形象 + GBA锁定 + 季节(当季)**。"目的品"= 顾客专程来买的款,落在引流/形象/锁定里。

### 2.1 保护规则
| 规则 | 内容 |
|------|------|
| **禁止直接汰换** | 引流 / 形象 / GBA锁定 / 当季季节品 —— 默认**保留 + 浅库存**，**不得仅凭销量/动销排名直接进汰换池** |
| **必须人工复核** | 上述角色若被算法标为汰换候选 → **强制人工复核 + 举证**（淘汰举证要求 = 是） |
| **不能单独作为淘汰依据的指标** | ❌ 销量 ❌ 动销率 ❌ 销售额排名 —— 任一单独都**不构成**汰换理由；必须叠加"角色 = 补充 且 无锚点 且 ABCZ=D" |

### 2.2 汰换决策 5 步流程（标准顺序）
```
Step 1  SKU 角色识别      → 标主角色（五选一）+ 是否 GBA 锁定（未标角色不得汰换）
Step 2  动销/毛利/ABCZ 分析 → 跑 9 维评分 + ABCZ 分层（见[[单品管理]]§7.3）
Step 3  角色约束校验（闸） → 引流/形象/锁定/当季 → 直接豁免，不进汰换池
Step 3.5 可得性防误杀（闸） → 低销先查现货率;缺货导致的伪滞销(可得性低)→ 补货而非汰换(零售老刘吸收回填)
Step 4  是否进入汰换池      → 仅"补充角色 + 无锚点 + ABCZ=D（零销/评分<30/末1%）+ 现货率正常"才入池
Step 5  是否需人工复核      → 入池件若曾被标保护角色 / 处边界 → 举证复核后才执行
```
> **第二道防误杀(可得性·六哥 2026-06-25 签)**:角色闸防"低销但有存在意义"的误杀;可得性闸防"低销但其实是缺货"的误杀。`实际可用=理论×现货率×履约率`,现货率低的低销=缺货不是该砍。来源零售老刘,见 [[2026-06-25_零售老刘吸收_方法论回填清单_v0.1]]。

### 2.3 小类内 ABC（非全场 ABC）= SKU 防误杀的硬判据（逐字批45回填·零售老刘《商品库存管理分析建模》4·7·六哥2026-06-26「先回填」签）

> **结论先行**:做商品 ABC 分类(尤其用于选品/淘汰)时,**必须在每个小类(=一个完整顾客需求)内部算 ABC,绝不能只看全场销售占比**。否则一个"在自身需求里不可或缺"的单品,会因全场占比小被误判为 C 而误杀——这是 §2 防误杀闸要拦的又一类典型误杀。
> 来源:零售老刘《商品库存管理分析建模》4·7(转述)。

1. **核心命题:ABC 口径 = 小类内算,不是全场算**。两条理由——① **只有小类才代表一个顾客需求**,所有商品业务判断都要回到小类这一层(全场是无数个需求的总和,对单品没有"需求归属"意义);② **只有单品在它自身小类内的占比,才真实反映其 ABC 状态**(=它在"满足这个需求"上到底是不是佼佼者)。全场销售占比衡量的是"它对总盘子贡献多少",跟"它在自己那个需求里该不该留"是两回事。
2. **关键反例(防误杀靶子):全场 C、小类内 A,不能据全场 ABC 淘汰**。某商品全场销售占比小(全场算是 C 类),但在它所属小类内部却排到 A 类——它是这个顾客需求的结构必需款 / 佼佼者。**此时若按全场 ABC 把它丢进汰换池,就是标准的误杀**:砍掉它,这个小类的需求就缺了一角(顾客找不到 → 流失或转店)。这正是 SKU 防误杀闸要拦的典型对象。
3. **打通既有防误杀条款(同源·硬判据)**:本条与 §2.1「❌销售额排名不能单独作为淘汰依据」、与「目的品/结构品不能按全场销量排名淘汰」是**同一道防误杀闸的不同硬判据**——既有条款防"低销但有存在意义"的误杀,本条防"全场占比小但在需求内不可或缺"的误杀,根都在 §0 PSI「按效用判好坏」。**操作落点**:§2.2 Step 2 的「ABCZ 分层」与 Step 4 的「ABCZ=D」口径,一律按**小类内**算而非全场;§3 与 ABC/ABCZ 对接表里的"表现排名",同样默认小类内口径。
4. **一句话判据**:**淘汰看 ABC,先问"这是哪个口径的 ABC"——小类内 A,永不因全场 C 被砍。**

## 3. 与现有体系的对接

| 现有能力 | 关系 | 说明 |
|---------|------|------|
| **ABC / ABCZ 分层**（[[单品管理]]§7） | **过滤层（前置于汰换，不替代 ABC）** | ABCZ 算"表现排名"；角色层在其后做"存在意义"校验。先 ABCZ 排名 → 再角色豁免 |
| **长尾治理**（[[M-DEC-013_单小类长尾结构治理_v0.1_候选]]） | **保护闸** | 长尾**不可直接砍**:C+D 长尾里凡引流/形象/锁定/当季 → 必须保护；只有"补充+无锚点+D"可砍 |
| **M-DEC 决策记忆** | **写入** | 每次角色判定 + 保护豁免/汰换决策 → 作为 M-DEC 现象→决策→推理→预期 入库;结果已知回填 actual/lessons |

## 3B. 购物篮三角色（零售老刘逐字批05 回填·六哥 2026-06-25 签）

> 来源:零售老刘《品类建模》27 三商品群(转述)。**三套角色体系·各管一层(逐字批06补全)**:
> ① **品类角色 4 大**(目标性/常规性/时令性/便利性)= **品类层**(Nielsen经典,定品类经营定位);
> ② **购物篮三角色**(主力/附属/关联)= **关联层**(定单品在购物篮的连带定位);
> ③ **晟果五角色**(引流/利润/形象/补充/季节)= **SKU 存在理由层**(本机制·定单品该不该保)。
> 三层叠用:品类层定方向、关联层防连带误杀、SKU层定保护。
> **核心商品线 G(逐字批13回填)**:高频刚需/高渗透的**基石型商品线**(占SKU约20%、贡献65%+、渗透>90%、复购>50%、周转>10次/年)——是"目的品/引流"在**品类线级**的对应,**绝对保护、绝不缺货**(缺货=客流流失)。入选:连续3-6月排名前5-10%、复购≥60%、缺货≤5%。

| 购物篮三角色 | 是什么 | 对接 |
|------|------|------|
| **主力商品** | 核心品类·印象/引流·高PI·**IOI 目标商品** | = 目的品保护对象(引流/形象) |
| **附属商品** | 同品类补充·连带(增主力深广度) | = 补充角色,但**连带价值高不可单砍** |
| **关联商品** | 跨品类·随机冲动 | = 连带触发,撑购物篮质量 |

- **比例参考**:主力销 60-70%/品种 20-30%;附属销 15-20%/品种 35-40%;关联销 10-15%/品种 30-40%。
- **「红花需绿叶衬」= 本防误杀机制的外部印证**:缺附属/关联,主力也卖不动 → **附属/关联即便单看低销,也不能因排名直接砍**(它们撑着主力的连带)。这正是角色闸要防的第三类误杀:**连带价值误杀**。
