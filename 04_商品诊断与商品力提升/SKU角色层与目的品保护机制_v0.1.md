# SKU 角色层与目的品保护机制 v0.1

> **结论先行**:现有体系（ABC / 动销 / 长尾治理）**只判断"表现"，不判断"存在意义"**——会误杀引流品（低销带客流）、形象品（低销撑品质感知）、结构补位品。本方法补上 **SKU"存在理由层"**：**任何"汰换/压缩/优化"决策，必须先过角色校验层，而非仅依赖销量/毛利指标。**
> 来源:打捞 [[2026-06-25_归档打捞backlog_v0.1]] #4（经营型 Taxonomy 任务书 §5.4 五角色词表 + GBA 防误杀双保险）。是 [[品类管理]] §3.3B 与 [[单品管理]] §7.3 的 SSOT。

## 1. SKU 角色分类模型（锚于晟果五角色）

> **主角色五选一，辅角色可空选一**（晟果五角色词表唯一来源,禁另立角色字段）。ChatGPT 式"利润/引流/形象结构"三分,是下表的**保护视角**简化映射(见 §2)。

| 商品角色 | 定义 | 判定逻辑（可操作） | 允许低销? | 处理策略 |
|---------|------|------------------|:---:|---------|
| **引流** | 带客流入店的价格敏感/高关注款 | 高关注度词 + 价格敏感 + 进店带动（连带/客流时段贡献） | ✅ 允许 | 🔒 防误杀:保排面、控价、不凭销量砍 |
| **形象** | 建品质/信任感知，未必高销 | 品类完整性必需 + 品质标杆 + 顾客感知强 | ✅ 允许 | 🔒 防误杀:保留撑感知，按形象价值判 |
| **利润** | 贡献毛利额的盈利款 | 毛利额（非毛利率单看）排名靠前 | ⚠️ 看毛利额 | 看毛利额贡献,非单看销量 |
| **补充** | 满足长尾/一站式需求的常规款 | 无引流/形象/季节/锁定标记的默认角色 | ❌ 不豁免 | 低销 + 无角色 → **优先汰换候选** |
| **季节** | 节令/节庆带动款 | 挂季节日历（24节气/52周表） | ✅ 当季允许 | 按季节日历进退,非看常态销量 |

**GBA 锁定品类（大湾区差异化锚点·一等公民）**:凉果/陈皮/煲汤料/中式糕点/烧腊等带**锁定标记**，独立于五角色，进入**所有淘汰判断的防误杀通道**。

## 2. 目的品保护机制（强制保护层）

> ChatGPT 三类映射:**利润品=利润；引流品=引流；形象/结构品=形象 + GBA锁定 + 季节(当季)**。"目的品"= 顾客专程来买的款,落在引流/形象/锁定里。

### 2.1 保护规则
| 规则 | 内容 |
|------|------|
| **禁止直接汰换** | 引流 / 形象 / GBA锁定 / 当季季节品 —— 默认**保留 + 浅库存**，**不得仅凭销量/动销排名直接进汰换池** |
| **必须人工复核** | 上述角色若被算法标为汰换候选 → **强制人工复核 + 举证**（淘汰举证要求 = 是） |
| **不能单独作为淘汰依据的指标** | ❌ 销量 ❌ 动销率 ❌ 销售额排名 —— 任一单独都**不构成**汰换理由；必须叠加"角色 = 补充 且 无锚点 且 ABCZ=D" |

### 2.2 汰换决策 5 步流程（标准顺序）
```
Step 1  SKU 角色识别      → 标主角色（五选一）+ 是否 GBA 锁定（未标角色不得汰换）
Step 2  动销/毛利/ABCZ 分析 → 跑 9 维评分 + ABCZ 分层（见[[单品管理]]§7.3）
Step 3  角色约束校验（闸） → 引流/形象/锁定/当季 → 直接豁免，不进汰换池
Step 4  是否进入汰换池      → 仅"补充角色 + 无锚点 + ABCZ=D（零销/评分<30/末1%）"才入池
Step 5  是否需人工复核      → 入池件若曾被标保护角色 / 处边界 → 举证复核后才执行
```

## 3. 与现有体系的对接

| 现有能力 | 关系 | 说明 |
|---------|------|------|
| **ABC / ABCZ 分层**（[[单品管理]]§7） | **过滤层（前置于汰换，不替代 ABC）** | ABCZ 算"表现排名"；角色层在其后做"存在意义"校验。先 ABCZ 排名 → 再角色豁免 |
| **长尾治理**（[[M-DEC-013_单小类长尾结构治理_v0.1_候选]]） | **保护闸** | 长尾**不可直接砍**:C+D 长尾里凡引流/形象/锁定/当季 → 必须保护；只有"补充+无锚点+D"可砍 |
| **M-DEC 决策记忆** | **写入** | 每次角色判定 + 保护豁免/汰换决策 → 作为 M-DEC 现象→决策→推理→预期 入库;结果已知回填 actual/lessons |

## 4. 应用场景

- 休食全品类长尾治理
- 多门店对比诊断
- RetailOS 决策保护层
