# 滞销与库存挤压判定标准建议 v0.1

> **本文件**：5/9 用户要求我给"滞销 SKU"+"库存挤压"判定标准的建议 / 待启明侧后续 confirm。
> **目的**：vault 内有清晰可复用的判定逻辑 / 不依赖客户口头标准 / 后续算法 / 报告 / 调改决策都可基于此。

---

## §A. 核心区别 — 滞销 vs 库存挤压

| 维度 | 滞销 | 库存挤压 |
|---|---|---|
| **核心问题** | 卖不出去（销售端）| 库存量太多（采购端）|
| **判定指标** | 销售端：销量/销额/天数 | 库存端：库存金额/周转天数/库存月数 |
| **典型表现** | 长期 0 销量 / 库存放着不动 | 销售正常但进货过多 / 资金占用大 |
| **重叠** | 长期严重滞销 → 必然挤压 | 短期挤压（爆款进货过多）≠ 滞销 |
| **解决方向** | 淘汰下架 / 不再补货 | 控制进货节奏 / 促销清库 |

→ **关键区别**：滞销看销售 / 挤压看库存 + 资金。**两个 SKU 池可能重叠但不等价**。

---

## §B. 滞销 SKU 判定标准建议

### B.1 主标准（连续 N 天无销售 / 按品类差异化）⭐

| 品类 L1 | 滞销标准（连续无销售天数）| 理由 |
|---|---|---|
| **生鲜**（蔬菜/水果/猪肉/牛肉/活鲜）| **3-5 天** | 短保产品 / 周转极快 / 5 天无销售即异常 |
| **专柜**（烘焙/熟食 / 自营专区）| **5-7 天** | 短保 / 现烤现卖 |
| **冷冻食品**（雪糕/水饺/汤圆）| **30 天** | 季节性 + 中保 |
| **休闲食品**（零食/果冻/糕点）| **30-45 天** | 中保 / 跟风快 |
| **调味品 / 粮油** | **60-90 天** | 长保 / 必需品 / 慢周转 |
| **酒水**（啤酒/白酒/红酒/果酒）| **60-90 天** | 长保 / 节令性强 / 春节/端午波动 |
| **百货 / 日化 / 用品** | **90-120 天** | 耐用品 / 不易变质 / 季节性 |
| **烟草** | **30-60 天** | 必需品 / 按品种细分 |
| **新品**（上架 < 30 天）| **不判滞销** | 给观察期 / 30 天后正常判 |
| **节令商品**（年货/月饼/礼盒）| **跨季节判** | 至少完整一个销售季后再判 |

### B.2 辅助标准（与主标准联用）

- **库存周转天数 > 主标准 1.5 倍**（如休闲品类：90 天还有库存的 SKU = 强滞销）
- **30/60/90 天累计销量 = 0** + **库存 > 0** = 强滞销标记
- **售出率 < 10%**（库存 / (库存 + 30 天销量) > 90% = 滞销候选）

### B.3 排除项（不判滞销）

- 新品（上架 < 30 天）
- 节令商品（春节 / 端午 / 中秋 / 年货 / 月饼 / 等）
- 季节性 SKU（夏季冷饮 / 冬季火锅 / 等）/ 跨季后判
- 战略 SKU（品牌形象款 / 老板认可款 / 即使低销也保留）

### B.4 4 级滞销分级（建议）

```
🔴 P0 立即淘汰：连续 N 天 0 销售 + 库存 > 0 + 库存金额 > 阈值
🟡 P1 30 天观察：连续 N×0.7 天 0 销售（接近但未到）
🟢 P2 季节性休眠：销售季外 / 不淘汰 / 等季节回来
⚪ P3 战略保留：老板/启明 sign off 后保留
```

## §C. 库存挤压判定标准建议

### C.1 三维联合判定（金额 + 时间 + 周转）

```
库存挤压 = ① 库存金额 > 阈值
        AND ② 库存周转天数 > 阈值
        AND ③ 库存月数（库存 / 月销量）> 阈值
```

### C.2 各品类双阈值（金额 + 时间）

| 品类 L1 | 金额阈值（库存成本）| 库存周转天数 | 库存月数 |
|---|---|---|---|
| **生鲜** | > 1,000 元 | > 7 天 | > 0.5 月 |
| **休闲食品** | > 3,000 元 | > 60 天 | > 3 月 |
| **冷冻食品** | > 5,000 元 | > 60 天 | > 3 月 |
| **调味 / 粮油** | > 5,000 元 | > 90 天 | > 4 月 |
| **酒水** | > 10,000 元 | > 90 天 | > 4 月 |
| **百货 / 日化** | > 5,000 元 | > 120 天 | > 6 月 |
| **烟草** | > 10,000 元 | > 30 天 | > 1 月 |
| **新品** | 不判（30 天观察期）| — | — |

### C.3 严重程度分级

```
🔴 严重挤压：金额 > 阈值 × 2 + 周转 > 阈值 × 2（必处置）
🟡 中度挤压：金额 > 阈值 + 周转 > 阈值（W19+ 处置）
🟢 轻度挤压：单一维度超阈值（监控）
⚪ 健康：所有维度内（不处置）
```

### C.4 处置动作（按严重程度）

```
🔴 严重挤压：① 暂停进货 ② 促销清库（-30% 价格）③ 损耗核销 ④ 进表 D 淘汰
🟡 中度挤压：① 进货节奏减半 ② 促销清库（-15% 价格）③ 监控
🟢 轻度挤压：① 进货节奏 -10% ② 监控周转
```

## §D. 4 个核心建议给启明（5/9 反馈用）

### D.1 建议建立"双指标体系"
- 不混淆"滞销" vs "库存挤压"
- 滞销关注销售端 / 挤压关注库存端
- 双指标可能重叠但不等价

### D.2 建议按品类差异化阈值（不一刀切）
- 生鲜 5 天 vs 调味 90 天 vs 百货 120 天 = 完全不同的标准
- 一刀切（如全店 60 天）会**误杀慢周转必需品**（米/油等）+ **漏掉短保品**（生鲜）

### D.3 建议金额+时间双阈值（不只看天数）
- 单看时间：50 元小货物滞销 90 天 ≈ 0.5 元/天损失（可忽略）
- 单看金额：5000 元货物 / 30 天 ≈ 167 元/天损失（中等）
- 双阈值同时触发 → 才算真正需要处置的"严重挤压"

### D.4 建议建立"战略保留"通道
- 不是所有 D 类都要砍
- 老板/启明 sign off 的战略 SKU（品牌形象 / 客户感受 / 节令储备）应保留
- 战略保留池建议占 D 类 30-50%（v2.0 算法已有"战略 D"概念）
