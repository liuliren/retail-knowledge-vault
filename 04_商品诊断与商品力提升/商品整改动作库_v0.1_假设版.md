# 商品整改动作库 v0.1 · 假设版

> **性质**：假设先行→实战验证→回填修正。`validated` 仅 1 行（价格带过密）；`observed` 行有花厅坊/好家源实跑但未跨店复验；`hypothesis` 行等过品实战升级。**本库不是制度**（审议轮次1），是采购过品时翻的工具表。每轮过品复盘后按第 13 步回填验证状态。
>
> **总红线**：任何淘汰动作前必过防误杀闸门；库存账评级 B/C 的店**先修账再砍货**；动作清单须责任人 sign off 后执行。

## 统一动作标签（15 种）
保留 / 淘汰 / 观察 / 补充 / 替换 / 压缩 / 扩充 / 调陈列 / 调价格带 / 谈供价 / 换供应商 / 做组合 / 做促销 / 建样板 / 暂缓判断

## 使用规则（三条）
1. **过品时逐 SKU 对表**：找到问题类型行→核"必备证据"列有没有→没证据就走"暂缓判断"行，不许拍脑袋
2. **每轮复盘回填最后一列**：hypothesis 行被实战证实→observed；跨店复现→validated；被推翻→rejected（rejected 行不删，划线留痕）
3. 假设值（15%上限/60天/≤3支等）过品两轮后统一校准一次，校准=六哥裁决
