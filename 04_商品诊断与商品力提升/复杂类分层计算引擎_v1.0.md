# 复杂类分层计算引擎 v1.0

> **核心问题**:不同复杂度品类,应在哪一层(L3/L4/SKU)做决策,才能保证结构不失真?
> **一句话**:**让不同复杂度品类在同一套规则下不失真分析。** 是 [[数据与系统能力底座]] §8B「分层计算边界」+ [[品类管理]] §3.1B「L4 启用」的可执行引擎(打捞 #3 升级)。
> **关键洞察**:**复杂度 ≠ SKU 数**,而是"决策逻辑分叉度"(价格带分裂 + 决策路径 + 功能重叠)。

## ① 品类复杂度 3 级模型
判定走 5 维,但**权重在价格带分裂度 + 决策路径数**(不是 SKU 数):

| 维度 | 简单 | 中复杂 | 高复杂 |
|------|------|--------|--------|
| 决策路径(顾客买它的逻辑) | 单一 | 单一/弱分叉 | **明显多条** |
| 价格带分裂(高价带 SKU 是否成群) | 集中单档 | 窄(高价≤少数同质) | **多档都成群** |
| 功能重叠 | 无 | 低 | **高** |
| 品牌数 | 少 | 多 | 多 |
| 动销分散(C+D 产销比) | 低 | 中 | **高(长尾厚)** |
| SKU 数 | 参考,非决定 | 参考 | 参考 |

> **判级铁律**:任一"价格带多档成群 / 决策路径明显多条 / 功能重叠高"触发 → **高复杂**。SKU 再多但同质(单档+单逻辑)仍是中复杂。

## ② 分层决策树（L3 / L4 / SKU）
```
品类进来
  ├─ 决策路径单一 + 价格带集中?  → 【L3 算】(简单/中复杂)
  │     └─ 但 SKU 多/有 Item 群?  → L3 主算 + Item 级看结构(不必 L4)
  ├─ 价格带多档成群 OR 决策路径明显多条? → 【必须下沉 L4】按决策组分别算
  │     └─ L4 决策组内仍有高低混杂掩盖? → 【拆到 SKU】看明细
  └─ 输出:复杂类顺序 = L4 计算 → L3 汇总 → L2 判断
```
- **L3 适用**:决策路径单一 + 价格带集中(简单/中复杂)。
- **L4 适用**:价格带多档成群 或 决策路径多条(高复杂)。
- **SKU 必拆**:L4 组内仍有高低混杂、中位价掩盖两端结构时。

## ③ IP-Chart 防失真机制
**何时失真**:用 **L3 汇总的中位价/单一价格带图**时,若**价格带分裂(高低两端都成群)** → 中位价掩盖真实结构,IP-Chart 形状被拉歪。
**修复规则**:
| 情况 | 规则 |
|------|------|
| 价格带多档成群(高复杂) | **禁止 L3 汇总画一张 IP-Chart**;按 L4 决策组/价格带分档**分别画** |
| L4 组内仍高低混杂 | **拆到 SKU** 看分布,不用聚合值 |
| 价格带窄(高价仅少数同质) | **允许 L3 聚合**(失真风险低) |

## ④ 三品类压力验证（真实数据 / 93 天 / 脱敏）
| 品类 | SKU | 价格带(低/中/高) | 长尾(C+D产销) | 复杂度 | 推荐分析层级 | L3汇总是否失真 | 原因 |
|------|----:|------------------|------|--------|------------|:--:|------|
| **冲调 G4-A** | 74 | 5L3子类·中价过密·跨度1-50+ | 后40%→16% | **高复杂** | **L4**(按麦片/咖啡/奶粉决策组) | **会** | 多决策路径(早餐/冲饮/代餐),L3一锅算拉歪主流价位 |
| **饼干 T03** | 185 | 价0.5-22.8·**高价仅2SKU** | 低销32 | **中复杂** | **L3 + Item级** | **不会** | SKU最多但价格带窄、高价同质,L3汇总不失真 |
| **果干 T05** | 136 | 低21/中109/高6·**跨度1-44.9最大** | C+D=48%产10% | **高复杂** | **L4/SKU**(按价格带分档) | **会** | 价格带分裂最大,中位9.9掩盖1-44.9,必须分档算 |

> **验证结论印证核心洞察**:T03(185SKU)是中复杂、T05(136SKU)是高复杂 —— **SKU 多 ≠ 复杂**;真正决定失真的是**价格带分裂 + 决策路径**。

## ⑤ 系统结论:跨品类稳定性 = **YES**
- **依据**:同一套规则(决策树 + IP-Chart 防失真)在 3 个差异极大的品类上一致适用,且每个品类的失真风险都被"价格带分裂度/决策路径"规则**正确预测并给出修复层级**。
- 规则不靠品类特例,靠"决策逻辑分叉"这一普适判据 → 具备跨品类稳定性。
- **边界**:库存列缺失时(本批 T03/T05 无库存)→ 周转/有货滞销判据 pending,不影响分层结论。
