# 变更控制日志模板

> **使用方法**：复制到 `16_客户与战役档案/<客户代号>/07_变更控制日志.md`。
>
> **本文件性质**：战役范围 / 时间表 / 交付物 / 价格 / 关键人员的**所有变更**留痕。客户提出"再加一点"或我方提出"少做一点"都必须先登记此处，再走流程。
>
> **追加纪律**：只追加 / 不删除。被驳回的变更标 ❌ + 驳回理由——这是反"范围爬升"的核心防线。

## 3. 变更评估流程

```
变更提出（客户 or 我方）
   ↓
登记到本表（先记，再评）
   ↓
影响评估（参 [[变更影响检查清单]] v1.1）
   ↓
小变更（不动里程碑 / 不动价格） → 决策人即时拍板 → 回写
中变更（动里程碑 or 交付物） → 紧急 G05 阶段门评审 → 拍板 → 回写
大变更（动战役命题 or 价格） → 必须重新主持 G04 战役章程 → 章程版本升级
   ↓
所有变更必须回写：00_战役章程 / 01_里程碑 / 06_交付物清单 / 05_决策记录
```

## 5. 范围爬升红线

以下情况**默认驳回**（除非走完整 G04 重新章程流程）：

- 客户口头加项但未签变更确认
- 工作量增加 >20% 但价格不变
- 新增门店或 SKU 范围 >30%
- 老板临时改命题（"我们其实想做的是 ???"）

> 反"再加一点"机制是 [[战役指挥板]] §三本月四不做的延伸。本表 ❌ 驳回栏要直接引用驳回理由。
