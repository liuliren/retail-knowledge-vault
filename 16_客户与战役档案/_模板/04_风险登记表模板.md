# 风险登记表模板

> **使用方法**：复制到 `16_客户与战役档案/<客户代号>/04_风险登记表.md`。
>
> **追加纪律**：只追加 / 不删除。已闭合的风险标 ✅ + 闭合日期，**不允许从表中删除**——AAR 时这是依据。

---

## 1. 风险登记表

| #  | 风险描述 | 类型     | 概率 | 影响 | 优先级 | 应对措施 | 责任人 | 触发日期    | 闭合日期    | 状态           |
|----|----------|----------|------|------|--------|----------|--------|-------------|-------------|----------------|
| R1 | ?        | 数据 / 现场 / 客户 / 内部 / 合规 | 高 / 中 / 低 | 高 / 中 / 低 | P0 / P1 / P2 | ?        | ?      | YYYY-MM-DD  | YYYY-MM-DD  | 🟢监控 / 🟡缓解中 / 🔴触发 / ✅闭合 |
| R2 | ?        | ?        | ?    | ?    | ?      | ?        | ?      | ?           | ?           | ?              |

> **优先级定义**：P0 = 触发即战役停摆 / P1 = 触发即偏离 ≥7 天 / P2 = 可监控、有缓解空间。
>
> **状态字典**：
> - 🟢 监控（已识别、未发生、有缓解预案）
> - 🟡 缓解中（已发生、缓解措施在执行）
> - 🔴 触发（已发生且未受控，须升级 G05 / 决策记录）
> - ✅ 闭合（风险消除或战役结束）

---

## 2. 风险类型说明

| 类型     | 典型场景                                                            |
|----------|---------------------------------------------------------------------|
| 数据     | POS 字段不全 / 销售数据失真 / 库存数据失真 / 数据延迟               |
| 现场     | 包装尺寸与货架不匹配 / 动线问题 / 缺货 / 损耗                       |
| 客户     | 老板拍板拖延 / 店长抗拒整改 / 付款延迟 / 范围爬升                   |
| 内部     | WIP 超限 / 关键路径偏离 / 知识库依据不足 / 团队带宽                  |
| 合规     | NDA 边界 / 客户案例对外讲述风险 / 数据脱敏 / 商业敏感               |

---

## 3. 升级机制

| 风险等级 | 触发动作                                                           |
|----------|--------------------------------------------------------------------|
| P0 触发  | 立即停战役推进 + 紧急 G05 阶段门评审（24h 内）+ 决策记录            |
| P1 触发  | 当周仪式必谈 + 升级 [[战役指挥板]] 健康灯（🟡 → 🔴）                  |
| P2 触发  | 下周仪式扫描 + 视情况升级 P1                                       |

---

## 4. 风险闭合纪律

闭合一条风险时**必须**填：

- 闭合日期
- 闭合原因（消除 / 转化为决策 / 战役结束自然消失）
- 是否产生新风险（如有，新建一条）
- 是否需要回写经验到 [[反幻觉规则总库]] 或对应方法论文件
