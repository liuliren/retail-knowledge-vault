# M-DEC-009/011/012 升级差距清单 · draft

> **性质**：只登记差距，不升级、不改任何 M-DEC 原文。证据基线 = 6/19 T03 饼干实跑证据报告（active·六哥件）。注：该报告当时对 010 的评级是"中·保持candidate"，010 之后已另行升 active v1.0——说明"证据增强→补齐→签字"这条路走得通，009 是下一个最近的。

## M-DEC-009 老品汰换时间维度防误判 —— 差距最小（证据强度：中高）

**已有**：防误杀闸门被真实案例验证（14 个>60天无销 SKU 中 10 个库存=0，按天数一刀切会误杀；4 个有货候选六桶细判后 0 个坐实真滞销——防误杀纵深也验证了）。
**升级还差**：
1. 当期库存表（6/19 时点用的是 May 近似口径，需一次当期数据复验）
2. 订货量/售罄率字段（POS 回填需求清单第②项）
3. ≥1 个第二小类或第二门店的复验（破单小类样本；富城/沙埔数据都可用）
4. 走 §11.3 checklist（0断链/summary/fact_layer 已基本齐）+ 六哥签字

**判断**：数据到位后 1 个会话可完成升级准备。**建议列为 promote 桥第二单**（010 之后）。

## M-DEC-011 黄金位×ABC错配 —— 卡在陈列实测（证据强度：弱/pending）

**已有**：方法论框架完整；T03 想验但表E陈列位置总表是空模板（无 current_barcode，measurement_status=待实测）。
**升级还差**：
1. **表E 实测装表或现场拍照**——唯一硬前提，无陈列坐标一切免谈
2. 实测后跑一次黄金位×ABC 交叉，产出 ≥1 个错配真实例
3. checklist + 签字

**判断**：差距不在文档在现场。**建议挂到下一次到店动作**（花厅坊 D2 后续到店 或 富城 07-13 首访任选，谁先有陈列实测谁先验）。

## M-DEC-012 小类增长率分群 —— 卡在同比数据（证据强度：弱/pending）

**已有**：框架完整；T03 仅单期数据无法算增长率。
**升级还差**：
1. **去年同期或多期连续小类销售数据**——花厅坊 POS 时间范围 2025-01~2026-04（资料采集清单 §4.6），理论上同比数据已在库但**尚未按小类分期清洗**
2. 一次多期清洗+分群实算（可挂 D3 L3 粒度拉数之后）
3. checklist + 签字

**判断**：三卡中唯一"数据可能已在手上、只差加工"的——D3 拉数完成后优先试算。

## 汇总：三卡升级路径依赖图

```
009 ← 当期库存+售罄率字段（数据请求已列清单）+ 第二样本
011 ← 陈列实测（下次到店硬前提）

```
