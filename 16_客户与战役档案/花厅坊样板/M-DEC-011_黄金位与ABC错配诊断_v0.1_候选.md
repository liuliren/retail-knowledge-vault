# M-DEC-011 黄金位×ABC陈列错配诊断(候选)

## 0. 定位说明
- **增强哪个文件**:[[2026-05-07_花厅坊陈列位置总表_v1.0]] × [[2026-05-07_花厅坊休食商品动销分析及跟进表_v1.0]](两表 join 运用)
- **补哪个盲区**:位置总表有"位置等级/坪效"、动销表有"ABCD",但**两者没被 join 起来判错配**;[[M-DEC-006_陈列层数确认与区域落位SOP_v0.1_候选]] 只管"怎么落位"(新建/调整流程),不管"现状错没错"
- **不取代什么**:不新建表、不重定义落位 SOP;只加一层"黄金位×ABC"现状诊断规则

## 1. 决策名
黄金位 × 商品 ABC 的错配诊断 —— 揪出"黄金位喂了 C/D 类、A 类窝在角落"。

## 2. 适用场景
陈列改造前现状诊断 / 周期巡检 / 端架堆头复盘:判断陈列稀缺资源(黄金位)是否给对了商品,产错配换位清单。

## 3. 现有方法盲区
- 位置总表与动销表**各自孤立**:位置总表知道"哪个位是黄金位",动销表知道"哪个商品是 A 类",但**没人把"这个黄金位上放的是什么 ABC"算出来**;
- M-DEC-006 是"落位 SOP"(怎么放新货/调整),**不做存量错配体检**;
- 结果:黄金位长期错配无人量化发现,选品/订货再好也被陈列压住释放不出。

## 4. 新增判断链
```
陈列位置总表 × 动销表 按货位 join → 每货位:位置等级 + 该位商品 abcd_grade + 坪效
├── 黄金位喂C/D:位置等级=顶级 且 该位 abcd∈{C,D} → 浪费稀缺资源 → 换A类
├── A类窝角落:abcd=A 且 位置等级=末级 → 压制畅销 → 上移黄金位
└── 坪效背离:黄金位坪效 < 全店均坪效 → 该位选品有问题
   换位前 → 过 M-DEC-009 防误判(黄金位却低销,先分清"真不行"还是"缺货/价格假A类")
```

## 5. 临界值 [推断,待两表join实测+启明sign off校准]

| 错配类型 | 判据 | 临界值(待校准)|
|---|---|---|
| 🔴 黄金位喂C/D | 位置等级=1级(顶级) 且 该位 abcd∈{C,D} | 顶级位中 C/D 占比 **> X%** → 换 A 类 |
| 🟡 A类窝角落 | abcd=A 且 位置等级=末级 | A 类中处末级位 **> Y%** → 上移 |
| 🟢 坪效背离 | 黄金位坪效 < 全店均坪效 | 黄金位坪效 **< 均值** → 选品复查 |
