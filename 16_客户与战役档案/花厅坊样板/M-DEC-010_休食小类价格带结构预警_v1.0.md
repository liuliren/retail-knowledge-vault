# M-DEC-010 休食小类价格带结构预警 ✅ ACTIVE

> **状态:active v1.0**(2026-06-19 六哥sign off)。**vault 首条 active M-DEC**,双真实门店案例背书(T03饼干+T01薯片)。
> 文件已改名(去"_候选",14处双链已同步)。

## 0. 定位说明
- **增强哪个文件**:[[2026-05-07_花厅坊休食商品品类表_v1.0]](价格带维)
- **补哪个盲区**:品类表存"价格带目标"和"漏档"**文字诊断**(structure_diagnosis 字段),但没有"**多密算过密、多少算断档/漏档**"的量化触发规则 → 靠顾问主观判断
- **不取代什么**:不取代品类表的价格带目标设定、不重定义 [[品类管理]] 价格带管理;只加一层可计算的预警闸门

## 1. 决策名
休食各小类(L3)价格带结构的"过密 / 断档 / 漏档"量化预警。

## 2. 适用场景
季度品类复盘 / 价格带调整 / 端架补货选品时,按小类逐一判:价格带结构哪里堆了、哪里空了,优先补哪段。

## 3. 现有方法盲区
品类表 v1.0 的 `structure_diagnosis` 是**一句话人工判断**("价格带漏档")——
- 不可计算、不可复核、不可跨人复用;
- "漏档"靠经验拍,没有"某价格段 SKU 占比 > X% = 过密"或"相邻段销额跳空 > Y% = 断档"的客观线;
- 结果:T01/T03/T07 的"补中高端/补≥20元"是方向对但**没有量化触发点**,无法判断"补到什么程度算够"。

## 4. 新增判断链
```
某小类 L3 价格带实测分布(各 price_band 的 SKU数% + 销额%)
├── 过密:单一价格段 SKU 占比畸高(如 9.9 段堆 SKU)→ 顾客感知单调 + 内部同质内耗 → 疏
├── 断档:相邻价格段销额占比跳空(中间段几乎无货)→ 需求被挤到两端 / 客单价天花板 → 补中间
├── 漏档:某价格段 SKU 数=0 但该段是核心需求段 → 整段缺失 → 补该段
└── 优先级×毛利:断档/漏档若落在高毛利段(如休闲21.3%、巧克力中高端)→ 优先补(毛利作排序权重,非主判据)
```

## 5. 临界值（过密线已校准 / 断档·漏档待实测）

| 预警 | 判据 | 临界值 | 状态 |
|---|---|---|---|
| 🔴 **中价过密** | **中价段销额占比** | **> 70%** → 重构(砍中价冗余/补两端) | ✅**已校准**(R-SKU-13双峰60-70% + T03 82%/T01 72%双案例) |
| 🔴 单段SKU密 | 单一 price_band 的 SKU 占该小类比 | > X%(如9.9段>35%)→ 疏密/汰同质 | 🟡 待单段实测 |
| 🟡 **断档** | 相邻 price_band 销额占比跳空 | 相邻段销额占比差 > Y 个点 且中间段 < Z% → 补中间 | 🟡 待实测 |
| 🟡 **漏档** | 核心需求段 SKU 数 | 该段 SKU = 0 且属核心段 → 补整段 | 🟡 待实测 |
> **超低价/超高价配套(零售老刘R-SKU-12)**:每小类必有 ≥1 超低价走量品(必要条件);超高价是充分条件(可无,非硬缺口)。

## 5.1 跨案例证据（active 依据）
| 维度 | T03饼干(185) | T01膨化/薯片(145) | T05果干蜜饯(136) | 跨案例 |
|---|---|---|---|---|
| 中价段销额 | 82% | 72% | 88% | ✅ 三案例均>70%过密 |

> 三小类(饼干+膨化+果干)中价过密同时成立 → 远超"跨2小类重复"硬门槛 → active 更稳。

## 5.2 候选增补（待签字 v1.1）：价格带预警按品类天花板分型
> 三小类实测暴露：**「补高端/补中高端」不是通用动作**，须按品类价格天花板分型，否则会给错方向。
- **天花板低型**（如膨化：高价带=0、max仅13.9）→ 本是低单价冲动品类，**「补高端」未必符合品类属性**，慎补；优先做中价段疏密与尾部瘦身。
- **跨度大型**（如果干蜜饯：高6、max44.9）→ 有高端空间，断档/漏档判据照常用。
- **微高端型**（如饼干：高2、max22.8）→ 介于两者，按需求点确认。
> **判据**：触发"补高端"前，先看该 L3 实测价格天花板与品类属性；天花板天然低的品类，预警转向"中价疏密+尾部治理"，不机械补高端。**本节为 candidate，待六哥签字后并入 §4 判断链升 v1.1。**
