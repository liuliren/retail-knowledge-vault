# M-DEC-009 休食老品汰换的时间维度防误判(候选)

## 0. 定位(读前必看)
> 本 M-DEC **不是**新的滞销标准 —— 现有 [[2026-05-09_滞销与库存挤压判定标准_v1.0]](active)已定义 days_no_sales + 库存三维 + 4级。
> 本 M-DEC = **给它打 3 个"时间维度防误判"补丁**,堵现有标准的盲区,避免**误杀畅销品**。是增强,不是取代。

---

## 1. 决策名
休食老品汰换前,用时间维度三查防止"误杀"——把"假滞销"挡在淘汰表门外。

## 2. 适用场景
对 7,139 滞销 SKU(尤其休食)执行 P0 淘汰前的**终判闸门**:days_no_sales 触发滞销后,**不直接进淘汰表**,先过本 M-DEC 三查。

## 3. 推演链(为什么需要)
现有标准只看"连续无销售天数"。但**一个 SKU 无销售,可能是 4 个完全不同的原因**:
```
无销售(days_no_sales 高)
├── 真滞销:有货、顾客不要 → 该淘汰 ✅
├── 假滞销①缺货:断货了根本没货卖(M1假需求/M4假A类)→ 不该淘汰,该补货 ❌误杀
├── 假滞销②季节:季外正常(年货/冷饮/火锅)→ P2休眠,不淘汰
└── 假滞销③衰退中但仍有底:去年A今年C → 观察非立即砍
```
**现有标准对"缺货导致的假滞销"无显式防护 → 会把畅销断货品误杀进淘汰表**。这是零售老刘"畅销的才是缺货的"补的核心盲区。

## 4. 临界值(三查闸门)[推断,待POS回填校准]

| 查 | 武器(来源)| 判据 | 临界值(花厅坊休食 待校准)|
|---|---|---|---|
| **查1 防缺货误杀** ⭐ | M1假需求/M4假A类 | days_no_sales 高 **但** 期间 inventory=0 占比高 → 假滞销(缺货)| 无销售期内 **库存=0 天数 > 50%** → 标 `假滞销_缺货`,**退出淘汰、转补货** |
| **查2 双ABC衰退** | M2双ABC | 去年ABC vs 今年ABC | **去年A/B 今年C 或消失** → 标 `衰退`,进 P1观察(非P0)|
| **查3 售罄率速度** | M1售罄率 | 有货期内售罄达50%的天数 | 达50%售罄 **> 行业基准天数**(休食待定)→ 真滞销;**远低于** → 真畅销(别砍)|

> **过闸结果**:三查全过(真滞销)→ 进淘汰表 D;任一命中假滞销 → 转对应通道(补货/观察/休眠)。

## 5. 反事实(不做的代价)
- 不做查1:花厅坊曾"越畅销越缺货"——把断货的畅销品当滞销砍掉 → 双倍损失(砍掉真需求 + 留下真滞销)
- 不做查2:等 days_no_sales 触发才动,已晚;双ABC 能提前 1 个销售周期预警衰退
- 数据反例:7,139 滞销里 [推断] 有相当比例是 P2季节(年货/冷饮)+ 缺货假滞销,一刀切按天数砍 = 误杀

## 6. 落地动作(花厅坊)
1. 动销跟进表加 3 列:`fake_sluggish_oos`(查1)/`abc_decline`(查2去年ABC)/`sellthrough_days`(查3)
2. POS 回填:last_sales_date + inventory 历史 + 去年ABC + 订货量(算售罄率)
3. 对 7,139 跑三查 → 拆出 真滞销 / 假滞销缺货 / 衰退观察 / 季节休眠 四桶
4. 仅"真滞销"桶进 [[M-DEC-007_商品动销综合诊断与调整SOP_v0.1_候选]] 的淘汰 SOP
