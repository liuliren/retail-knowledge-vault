# M-DEC-012 小类增长率分群与资源再分配(候选)

## 0. 定位说明
- **增强哪个文件**:[[2026-05-07_花厅坊休食商品品类表_v1.0]](动态维)
- **补哪个盲区**:品类表存"**占比目标 vs 实测**"(静态偏差,看的是"达没达标"),但没有"**这个小类在涨还是在跌**"的动态增长率分群,更没有"涨的加资源、跌的缩资源"的再分配规则
- **不取代什么**:不改占比目标、不重判品类角色(角色取自品类表 §3-4)、不重定义 [[品类管理]];只在角色之上加一层"动态增长率→资源调整"的决策

## 1. 决策名
休食小类(L3)按增长率动态分群(增长/稳定/下降),驱动排面·订货·端架资源的再分配。

## 2. 适用场景
季度/月度品类复盘:在"占比达标判断"之外,识别哪些小类在长、哪些在衰,把资源从衰退类挪向增长类(动态调仓)。

## 3. 现有方法盲区
- 品类表是**静态快照**:占比实测 vs 目标,只回答"够不够",不回答"**趋势往哪走**";
- 一个小类可能"占比达标但正在快速下滑",静态视角看不出 → 等占比掉下来才反应,已晚;
- 没有把"增长率"转成"资源动作"的规则 → 排面/订货长期按老占比分,涨的喂不饱、跌的占着茅坑。

## 4. 新增判断链
```
各小类 L3 增长率(同比或环比销额)→ 分群
├── 增长(>+20%):需求在扩 → 加排面 / 提订货量(联动 M1 订货公式)/ 上端架
├── 稳定(±10%):维持 → 守现有资源
└── 下降(<−20%,连续 N 期):需求在退 → 缩排面 / 降订货 / 让位
   ├── 结合角色(品类表):流量类下降→慎缩(可能是引流不赚钱但带客);利润类下降→优先救/换品
   └── 下降类里的具体滞销 SKU → 交 M-DEC-009 判汰换;价格带结构问题 → 交 M-DEC-010
```

## 5. 临界值 [推断,待POS增长率+启明sign off校准]

| 分群 | 判据(增长率)| 临界值(零售老刘通论起点 / 花厅坊待校准)|
|---|---|---|
| 🟢 增长 | 同比/环比销额增长率 | **> +20%** → 加资源 |
| ⚪ 稳定 | 增长率 | **−10% ~ +10%** → 维持 |
| 🔴 下降 | 增长率 + 连续期数 | **< −20% 且连续 N 期(N待定)** → 缩资源/让位 |

> 阈值 ±10/−20/+20 来自零售老刘通论(13篇),**非花厅坊实证**;花厅坊真实阈值待 POS 增长率分布 + 启明 sign off。
