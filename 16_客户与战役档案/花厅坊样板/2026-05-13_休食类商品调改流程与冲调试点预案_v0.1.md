## 3. 核心原则：商品先于陈列

> **商品判断先于陈列判断。陈列不是直接凭经验调整，而是由以下链路反推：**
>
> **TPOS / CDT / 子品类角色 / 单品动作 / 新品候选池 → 陈列表达**

### 3.1 反推顺序（5 步 / 不可颠倒）

1. **先判断顾客购买任务与消费场景**（TPOS）
2. **再判断子品类角色**（流量 / 形象 / 毛利 / 补充 / 节令 / 待核验）
3. **再判断单品动作**：保留 / 观察 / 淘汰 / 补充 / 候选 / 待核验
4. **再形成新品候选池与补充方向**
5. **最后反推陈列层级 / 排面 / 组合 / 动线位置 / 展示资源**

### 3.2 严禁

- ❌ **不得写成"先调陈列，再补商品"**
- ❌ **不得把 DisplayMap 当成起点**
- ✅ DisplayMap 只能是**商品结构判断后的表达工具**
