# M-DEC-005 错品类货架重建 SOP v0.1(候选)

> **触发**:2026-05-06 花厅坊二访 G4-A 冲调货架 06 组发现严重品类断层(干果 / 凉果 / 奶酪 / 即食混杂在"冲调"主轴中),传统"调位置"调改无效,必须先**品类清理**再**货架重建**。
>
> **本 SOP 解决**:当一组货架的实际投放品类与设计主轴严重不符(L 轴品类外溢),如何系统化诊断 + 6 步重建。
>
> **战役验证**:5/6 G4-A 战役 + 5/7-5/13 8 面战役持续验证 → 5/14 复盘 → 5/31 提炼到 `04_商品诊断与商品力提升/中层决策库/M-DEC-005_错品类货架重建SOP_v1.0.md`(全局可复用)。

---

## §1. 触发条件 / 适用场景

### §1.1 必要条件(同时满足才适用)

| # | 条件 | 判断标准 |
|---|---|---|
| 1 | 货架有"主轴品类"设定(如 G4-A = 冲调) | DisplayMap / 品类规划文件可查 |
| 2 | 实际投放品类与主轴存在严重外溢(≥ 30% SKU 不属主轴 L 类) | 现场 L 轴标注 + 抽样 |
| 3 | 货架物理结构无重大变化需求(只是品类乱) | 不需更换货架 / 不动其他货架 |
| 4 | 有调改作业窗口(夜班 / 闭店 / 客流低谷) | 单组重建需 1.5-3 小时 |

### §1.2 不适用场景(走其他 SOP)

| 场景 | 走哪个 SOP |
|---|---|
| 整面货架品类结构重新规划 | 走"品类规划 SOP"(05_)|
| 货架物理结构调整(加/减层 / 加端架)| 走"货架结构调整 SOP"(待立项) |
| 单 SKU 替换 / 价格调整 | 走"商品诊断 SOP"(04_)|
| 整店动线 / 区域调整 | 走"门店动线诊断 SOP"(02_)|

## §2. ⭐ 实战发现(G4-A 06 组案例)

### §2.1 现场观察(2026-05-06 二访)

| 组 | 设计主轴(冲调)| 实际 2 层主品类 | 是否符合 |
|---|---|---|---|
| 01 | 燕麦片 | L-冲调(袋装中低端 5-25 元)| ✅ |
| 02 | 燕麦片 | L-冲调(中端 + 核桃 / 红枣 / 花生牛奶) | ✅ |
| 03 | 麦片 | L-冲调 | ✅ |
| 04 | 奶粉 / 罐装麦片 | L-冲调 + 边缘(70-80 元高单价) | ⚠️ 边缘 |
| 05 | 茶 + 奶茶 + 杯装麦片 | L-茶饮料类 + L-冲调 边缘 | ⚠️ 边缘 |
| **06** | (设计未定 / 应为大包装冲调) | **L-干果蜜饯 + L-奶酪即食 + L-茶饮料类** | ❌ **错品类** |

### §2.2 根因分析(为何 06 组成杂物区)

| # | 根因 | 证据 |
|---|---|---|
| 1 | **6 组物理层数特殊(7 层)→ 多 2 层空间无主品类规划** | 现场观察 / DisplayMap 缺失 |
| 2 | **店员把"放不下的杂品"自然堆积到这里** | 老板 / 店员访谈待补 |
| 3 | **L 干果蜜饯本应在 SC 散称区,但 SC 容量不足或缺品类设计** | SC 现场待复核 |
| 4 | **L 奶酪即食本应在 SX 生鲜墙冷藏柜,但常温 SKU 无家** | SX 冷藏柜布局待复核 |
| 5 | **缺乏"L 轴品类越界"的日常巡检机制** | 店长 / 店员品类意识弱 |

### §2.3 战役意义

⭐ **5/6 G4-A 调改的核心动作**:不是"调位置",而是 **6 组品类清理 + 重定位为大包装冲调专区**。

## §3. ⭐ 错品类货架重建 6 步法

### §3.1 6 步流程总览

```
Step 1 → Step 2 → Step 3 → Step 4 → Step 5 → Step 6
诊断    清单    迁移    重建    验证    沉淀
```

| 步 | 任务 | 工具 | 输出 |
|---|---|---|---|
| 1 | **L 轴诊断** | 三轴命名 + L 标注 | 异常 SKU 清单 |
| 2 | **迁移目的地清单** | vault 标准品类表 + 门店区域图 | 迁出/迁入对照表 |
| 3 | **物理迁移** | 夜班调改 | SKU 实物全部迁出 |
| 4 | **空架重建** | DisplayMap + 品类规划 | 新主轴投放完成 |
| 5 | **24h 验证** | 同机位 3 时段对比 + 销售数据 | 验收报告 |
| 6 | **方法论沉淀** | 战役档案 + 中层决策库 | M-DEC 候选 / 升 v1.0 |

### §3.2 各步详细 SOP

#### Step 1 — L 轴诊断(2 小时,白天)

| 动作 | 工具 | 标准 |
|---|---|---|
| 1.1 拍照 L1 整面_全景 + L2 整组图 | v0.3 命名规范 | 6 组各 1 张全景 |
| 1.2 拍照 L3 单层 + 价签(每组每层 1 张)| v0.3 命名规范 | 5/6/7 层全覆盖 |
| 1.3 现场标注每个 SKU 的 L 主品类 | 手机笔记 / Excel | 1 SKU 1 L 代号 |
| 1.4 计算每组 L 偏离率(非主轴 SKU 数 / 总 SKU 数) | Excel 自动 | ≥ 30% = 错品类组 |
| 1.5 出 L 轴诊断报告 | 模板 | 异常组清单 + 异常 SKU 清单 |

#### Step 2 — 迁移目的地清单(1 小时,白天)

| 动作 | 工具 | 标准 |
|---|---|---|
| 2.1 对每个异常 SKU 查 vault 标准品类表 | `05_/标准品类表` | 1 SKU 1 L 代号唯一 |
| 2.2 查 L 代号对应的主 P 区 | v0.3 §4 L 轴映射表 | 1 L 1 主 P 区 |
| 2.3 现场核实主 P 区是否有容量 | DisplayMap + 现场走查 | 容量 / 空位 / 拥挤度 |
| 2.4 出迁出/迁入对照表 | Excel | SKU / 当前位 / 目标位 / 容量预警 |
| 2.5 容量不足 → 同步规划目标区扩容 | 串联 §4 主 P 区调整 SOP | 必要时延期主战役 |

#### Step 3 — 物理迁移(夜班 22:00-24:00,2 小时)

| 动作 | 工具 | 标准 |
|---|---|---|
| 3.1 异常 SKU 全部下架(整箱 / 排面) | 推车 / 周转箱 | 1 SKU 1 周转箱 + 标签 |
| 3.2 价签同步下架 | 价签夹 | 1 SKU 1 价签夹 |
| 3.3 异常 SKU 迁入目标 P 区(按 §3.2 对照表) | 推车 + 现场走查 | 1 SKU 1 目标位 |
| 3.4 目标位价签同步上架 + 核对 | 价签机 / 手工 | 1 SKU 1 价签 |
| 3.5 拍照 24h 复拍(目标位 + 原位空架) | v0.3 命名规范 | _调改后_ + _夜灯后_ + 时段 |

#### Step 4 — 空架重建(夜班 24:00-02:00,2 小时)

| 动作 | 工具 | 标准 |
|---|---|---|
| 4.1 6 组物理空间评估(7 层 → 5/6/7 层是否合理) | 卷尺 + 现场 | 决定是否回归标准层数 |
| 4.2 6 组重定位主轴(如"大包装冲调专区") | 品类规划文件 | 1 组 1 主轴 |
| 4.3 主轴 SKU 选品(按品类规划)| `04_/商品诊断` + `05_/品类规划` | 5/6/7 层填满 |
| 4.4 SKU 上架 + 价签同步 | 现场作业 | 1 SKU 1 价签 / 排面整齐 |
| 4.5 拍照 6 组重建后 L1 + L2 + L3 | v0.3 命名规范 | _调改后_ + _夜灯后_ |

#### Step 5 — 24h 验证(次晨 09:00-10:00,1 小时)

| 动作 | 工具 | 标准 |
|---|---|---|
| 5.1 同机位 3 时段对比拍照(夜灯前 / 夜灯后 / 次晨白光) | v0.3 命名规范 | 3 时段全拍 |
| 5.2 SKU 错位 / 价签错检查 | 人工核对 | 错位率 < 5% |
| 5.3 排面整齐度 / 朝向 / 无缺货 | 人工核对 | 整齐度 ≥ 90% |
| 5.4 客流时段(次晨 09:00-12:00 高峰)旁观 30 min | 现场旁观 + 笔记 | 顾客拿取顺畅度 |
| 5.5 1 周内每天追销售数据 | POS 导出 | 6 组 SKU 销售环比 |
| 5.6 出验收报告 | 模板 | 通过 / 不通过 / 修正项 |

#### Step 6 — 方法论沉淀(5/13 战役收口后)

| 动作 | 工具 | 标准 |
|---|---|---|
| 6.1 战役档案 09_中层方法论沉淀更新 | 16_/花厅坊样板/09 | M-DEC-005 验证状态 |
| 6.2 销售数据 + 顾客行为 + 异常拦截记录 | Excel | 通过 / 修正 / 推翻 |
| 6.3 5/31 前提炼到全局中层决策库 | `04_/中层决策库/` | M-DEC-005 v1.0 |
| 6.4 销售话术沉淀 | 话术库 | "我们能让你的杂物组变销冠"|
| 6.5 伙伴 onboarding 教学 | 培训手册 | 6 步 SOP 标准化 |

## §4. ⭐ 三轴 P+L+G 在本 SOP 中的应用

### §4.1 L 轴是核心诊断工具

| 没有 L 轴 | 有 L 轴 |
|---|---|
| "6 组卖得不好" | "6 组 L-干果蜜饯偏离率 60% / L-奶酪即食偏离率 25% / L-冲调主轴覆盖仅 15%"|
| "调位置就行" | "必须先迁出 L-干果蜜饯到 SC / L-奶酪即食到 SX,再重建 L-冲调主轴"|
| 凭经验 / 主观 | 数据化 / 可证明 / 可复用 |

### §4.2 P 轴是迁移路径

```
异常 SKU 在:HS-G4-A-06-2-01-L干果蜜饯
迁移到:    SC-散1-A-01-2-01-L干果蜜饯
路径:      HS → SC(休食区 → 散称区)
```

### §4.3 G 轴是物理操作坐标

```
迁出位:HS-G4-A-06(整组下架)
迁入位:SC-散1-A-01-1-01 / 02 / 03 ...(按目标 P 区可用容量分配)
```

### §4.4 三轴交叉表(战役第二天 G4-A 06 组实战)

| 异常 SKU | 当前 G 轴 | 当前 L 轴 | 应在 L 轴 | 目标 P 区 | 目标 G 轴(预估)| 容量预警 |
|---|---|---|---|---|---|---|
| 干果(10 个 SKU)| HS-G4-A-06-2/3/4 | 误标 / 无 | L-干果蜜饯 | SC | SC-散1-A-01-1/2/3 | 待复核 |
| 凉果(8 个 SKU)| HS-G4-A-06-2/3 | 误标 / 无 | L-干果蜜饯 | SC | SC-散1-A-02-1/2 | 待复核 |
| 干奶酪(5 个 SKU)| HS-G4-A-06-3/4 | 误标 / 无 | L-奶酪即食 | SX | SX-G3-A-冷藏 | ⚠️ 需冷藏空间 |
| 杯装即食(6 个 SKU)| HS-G4-A-06-4 | 误标 / 无 | L-奶酪即食 | SX | SX-G3-A-冷藏 | ⚠️ 需冷藏空间 |
| 立顿茶 / 罐装茶(4 个 SKU)| HS-G4-A-05 | L-茶饮料类 | L-茶饮料类 | HS | HS-G4-B-或新规划 | 待复核 |

> **战役第二天必做**:迁移前先复核 SC / SX 容量,如不足 → §3.2 Step 2.5 串联主 P 区扩容(可能延期 G4-A 战役 1-2 晚)。

## §5. ⭐ 6 组重建后新主轴定位(候选 3 方案)

| 方案 | 6 组新主轴 | 说明 | 优劣 |
|---|---|---|---|
| **A** | 大包装冲调专区(L-冲调 大包装)| 1.5kg+ 燕麦袋装 / 罐装麦片 / 罐装奶粉 | ⭐ 推荐 / 与冲调主轴一致 / 利用 7 层物理优势 |
| B | 高端冲调进口区(L-冲调 进口)| 进口燕麦 / 高端奶粉 | 销量风险 / 高单价低周转 |
| C | 即食冲调早餐区(L-冲调 杯装 + L-奶酪即食)| 早餐场景 / 杯装麦片 + 牛奶 + 奶酪 | 跨 L 类 / 不利 L 轴清晰 |

> **6 组 7 层标准布局(方案 A)**:
> - 第 1 层(顶):顶层囤货(整箱大包装燕麦)
> - 第 2 层:形象层(进口高端大包装麦片)
> - 第 3 层:黄金视线(主销大包装燕麦 1.5kg)
> - 第 4 层:黄金手拿(主销罐装麦片 / 罐装奶粉)
> - 第 5 层:补充(二线大包装)
> - 第 6 层:大包装(超大包装 3kg+)
> - 第 7 层(底):围货(整箱整摞)

## §6. 销售话术沉淀(候选)

### §6.1 客户痛点表达

> "你们店是不是经常发现:某一组货架明明是冲调的,但顾客总买到干果?
> 这不是顾客错,是你们的 L 轴(逻辑品类)和 G 轴(物理货架)错位了。
> 长期错位 → 主轴销量低 + 杂品周转低 + 店员补货慌 + 顾客找不到 + 老板算不清账。"

### §6.2 价值主张

> "我们用三轴体系(P 区 + L 类 + G 架)+ 6 步重建 SOP,
> 能在 1 个夜班(2 小时迁移 + 2 小时重建 + 1 小时验证)把杂物组变销冠组。
> 不动货架结构 / 不增 SKU / 不加员工 / 不影响客流。
> 24h 验证 + 1 周追数 + 1 月稳定。"

### §6.3 ROI 估算(待 5/13 战役验证)

> 假设单组 6 月销售环比 +30% / 单店若有 5-10 个错品类组 → 年销售 +5-15% / 投入产出比 1:8 以上。
