# M-DEC-002 大包装区瘦身比例 v0.1（候选）

> **本 SOP 解决**：单一货架面的大包装区 SKU 数应瘦身多少？比例如何确定？瘦身依据是什么？
>
> **核心修正（5/7 反向回写）**：M-DEC-002 v0.1 假设"老板能直接判断哪些是死库存"= 错。**实际必须先有数据底盘**（表 D 异常清单 / 表 B 60 天滞销标记）→ 数据驱动 → 再决策。

## §1. 决策定位

**决策名**：大包装区瘦身比例（数据驱动 / 不凭老板印象）
**类型**：单品管理 / 货架结构 / 中层方法论 L4

## §2. 适用场景

- 单一货架面有"袋装囤量"角色组（如 Z6 / 16+ SKU）
- 老板 / 顾问怀疑大包装区"动销不好 / 库存压"但**未量化**
- 调改前需瘦身决策

## §3. 推演链

### 3.1 瘦身决策的前置条件（5/7 修正后铁律 ⭐）

```
❌ 错误路径（v0.1 原假设 / 5/6 老板访谈推翻）：
   "老板凭印象判断 → 拍板瘦身比例 X%"
   
✅ 正确路径（5/7 修正）：
   1. 表 A 商品基础表 v1.0.1 数据装载（含 sku_status / first_listed_date / cost_price）
   2. 表 B 动销分析跟进表 v1.0.1 跑分析（abcd_grade / z_flag / sales_qty_90d / stock_turnover_days / is_60d_slow_moving）
   3. 表 D 异常清单 sheet 自动筛（负库存 / 零库存 / 超 60 天）
   4. M-DEC-007 §S4 归因决策树（陈列差 / 活动期外 / 价格虚高 / 缺货 / 客群不匹配 / 真不需要）
   5. 数据驱动决策 → 瘦身比例 = f(D 类 SKU 占比, 60 天滞销 SKU 占比, 库存成本占比)
   6. 老板 sign off → 实施 → 24h-1 周复测
```

### 3.2 瘦身比例计算公式（v0.1 起步）

```
瘦身比例 = 0.4 × (D 类 SKU 占比) + 0.3 × (60 天滞销占比) + 0.3 × (库存成本占比 > 阈值)

示例（G4-A Z6 大包装 16 SKU）：
- D 类（abcd_grade='D'）= 7 SKU / 16 = 44%
- 60 天滞销 = 8 SKU / 16 = 50%
- 库存成本占该面 > 30% = TRUE
  
瘦身比例 = 0.4 × 44% + 0.3 × 50% + 0.3 × 100% = 17.6% + 15% + 30% = 62.6%
推荐净减：16 × 62.6% / 2 ≈ 5 SKU（保守估计 / 留缓冲）
最终 16 → 11（瘦身 31%）/ 或 16 → 9（瘦身 44%）按老板 risk 偏好
```

> **5/7 修正版**：原 G4-A 候选 16 → 9（瘦身 44%）= 公式上限值 / 风险较高 / 应分阶段（先 16 → 12 / 1 周后再 → 9）。

## §4. 临界值

| 维度 | 临界值 | 触发动作 |
|---|---|---|
| 瘦身比例下限 | < 15% | 数据不支持瘦身 / 不动 |
| 瘦身比例上限 | > 50% | **分阶段**（先一半 / 1 周后看数据 / 再继续）|
| 单次瘦身绝对数上限 | > 8 SKU | 必须分阶段（避免顾客投诉）|
| 库存成本释放下限 | < 1000 元 | 瘦身价值低 / 不优先 |
| D 类 + 60 天滞销 合计占比 | < 30% | 数据不支持 / 不动 |
| D 类 + 60 天滞销 合计占比 | > 70% | 该角色组**整体重构**（M-DEC-001 重启动）|

## §5. 反事实（推翻条件）

本规则在以下情况**不适用**：
- 表 A / 表 B 数据未装载完整（必须 D 类 4 阶确认走完 / 表 B 字段 32）
- 客户未 sign off（CLAUDE.md §13.13）
- 季节性高峰前 1 个月（如年货季前 / 大包装爆发期 / 不瘦身）
- 该 SKU 是供应商战略合作品（合同条款限制）
- 老板"特殊情况"通道（II 内核灵活象限 / 单次例外允许 / 必须复盘登记）
