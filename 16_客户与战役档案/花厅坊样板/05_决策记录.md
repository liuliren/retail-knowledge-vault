### D004 — 调改节奏调整:每天 1 面 × 8 天夜班(22:00-凌晨)

- **决策日期**:2026-05-06
- **决策人**:六哥
- **状态**:✅ 生效
- **决策类型**:战役范围 / 节奏

#### 一、决策背景

5/6 早晨六哥在战场反馈:
1. 整体调改计划升级 — 不只 G4-A 一面,要每天调 1 面,共 8 天连续战
2. 5/6 当天加班到很晚,**调整货架要等到晚上 10:00 之后**(夜班 22:00-凌晨)

原章程 §3.1 IN 范围只含 G4-A 一面,§B 里程碑按白天作业设计,与实战节奏不符,需登记。

#### 二、可选方案

| 方案 | 描述 | 优势 | 劣势 |
|---|---|---|---|
| A | 维持原节奏(G4-A 一面 5/6 完,W19 后启动其他 5 面)| 战役更轻 | 客户接触红线再亮 / W18 三回归红灯 |
| B | **每天 1 面 × 8 天连续夜班**(G4-A→G4-H,5/6-5/13)| 一周内完成所有面 + M2 5/13 即可数据复盘 | 体力 P0 风险 / 拍摄约定需调整 / 单兵带宽紧 |
| C | 8 天分散到 2 周(每隔 1 天 1 面)| 体力可控 | 节奏拖到 5/20+ / M3 阶段门延期 |

#### 三、依据链

- 5/6 早晨六哥战场实时反馈
- W18 三回归三维红灯(客户接触 0 / 沉淀验证比 28:1 / 客户验证度 21%)
- 6 机位拍摄原约定基于白天日光,夜间 22:00 后店内日光灯色温不同
- M-DEC-003 原临界值"到货率 < 50% 强制延期"在夜班 22:00 前到货率检查窗口下需修订

#### 四、最终决策

**方案 B**:每天 1 面 × 8 天连续夜班(22:00-凌晨)。

调改面顺序(待六哥回字符确认 5/7 起调哪一面):
- 5/6 G4-A 冲调正面(已锁定)
- 5/7 待定:A G4-B 同货架反面水饮 / B 邻排 G3-A 或 G5-A / C 其他

配套调整:
1. **里程碑** M1 拆分为 M1.0-M1.7(8 个夜班)+ M1.8(5/13 一周复测)
2. **风险登记** R5 单兵带宽 P2 → P0;新增 R9 8 天连续夜班体力 / R10 夜间拍摄约定调整
3. **M-DEC-003 临界值修订**:夜班 22:00 前到货率检查 → < 80% 当面延期 1 天 / < 50% 当周不补
4. **6 机位拍摄约定修订**:白天时段切片(M1 全幅 / M2-4 段特写 / M5-6 黄金层)+ 夜间补拍(调改前 22:00 + 调改后 02:00)+ 次日早晨 09:00 同机位补拍(白天日光对比)
5. **24h 同机位复拍节奏**:5/7 22:00 复拍 5/6 凌晨调改前 → 真 24h 对比

#### 五、影响范围(双向同构 5 问)

1. ✅ 涉及方法论 / SOP:S04 / S05 / S07 — Claude Code 镜像层是骨架(空指针,M1-M3 不修)
2. — 镜像层暂不同步
3. ✅ 5/13 后实战回写 S05 v0.1 → v0.5(夜班作业模式 / 6 机位时段切片夜间版)
4. ✅ 引入新"待确认":5/7 调哪一面 / 体力恢复节奏(5/9 / 5/11 中场休息?)/ 夜间日光灯色温对照度
5. ✅ client_safety = internal_only

#### 六、回写动作

- [x] 回写到 `00_战役章程.md` §3.1 IN(8 面)+ §B 里程碑 M1 拆分 + §E 决策点新增 + §G RACI

### D002 — L4 中层方法论沉淀机制建立(5/8 前 M-DEC-001/002/003 必产)

- **决策日期**:2026-05-04
- **决策人**:六哥
- **状态**:✅ 生效
- **决策类型**:战役推进 / 战略升级

#### 一、决策背景

memory user_六哥.md 自评:"真瓶颈是 L 中层方法论缺失"。每次战役 = 一次中层方法论上量机会,不是消耗。本战役升级为"生产 L4 中层方法论的工厂",L4 是当前最稀缺的产出。

#### 二、可选方案

| 方案 | 描述 | 优势 | 劣势 |
|---|---|---|---|
| A | 不产 L4,只做 L1/L2/L3 三层 | 战役更轻 | 真瓶颈不解 |
| B | **产 3 条 L4(M-DEC-001/002/003)** | 直接命中真瓶颈 | 5/8 前压力大 |
| C | 产 5+ 条 L4 | 上量大 | PRD 完美主义陷阱 |

#### 三、依据链

- memory user_六哥.md / project_花厅坊战役.md / feedback_战时模式.md
- 三层方法论结构(L-顶 / L-中 / L-底)— 中层判据"可教给伙伴的判断链"
- 5/6 调改 6 组 SKU 角色重构 = M-DEC-001 实证起点
- Z6 16→9 = M-DEC-002 实证起点
- 5/6 当日到货率 = M-DEC-003 实证起点

#### 四、最终决策

**方案 B**:5/8 周三前必产 M-DEC-001/002/003 三条草稿(适用场景 / 推演链 / 临界值 / 反事实 / 依据链 5 字段齐全)。

沉淀机制(2026-05-05 已建):
- 草稿池:`16_客户与战役档案/花厅坊样板/09_中层方法论沉淀.md`
- 战役级模板:`16_客户与战役档案/_模板/09_中层方法论沉淀模板.md`
- 提炼库:`04_商品诊断与商品力提升/中层决策库/`

#### 五、影响范围(双向同构 5 问)

1. ✅ 涉及方法论 / SOP:S04 / S07 — Claude Code 镜像层是骨架(空指针,M1-M3 不修)
2. ✅ 中层决策库目录新增 → CLAUDE.md §11 应增加 L4 类(待 5/31 战后回写)
