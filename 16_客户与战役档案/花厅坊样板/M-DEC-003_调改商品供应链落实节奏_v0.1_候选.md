# M-DEC-003 调改商品供应链落实节奏 v0.1（候选）

> **本 SOP 解决**：调改新品引进时供应商落实节奏问题 — 当日到货 vs 账期 vs 现金采购？多少天前下单？到货率不足时怎么处理？
>
> **核心修正（5/7 反向回写）**：M-DEC-003 v0.1 原假设"7 个新品需账期"被 5/6 老板访谈推翻 — 实际花厅坊**惯用现金采购**（休食等很多品类）→ 反而简化供应链节奏。

## §1. 决策定位

**决策名**：调改商品供应链落实节奏（现金 vs 账期 双轨）
**类型**：单品管理 / 供应链 / 中层方法论 L4

## §2. 适用场景

- 调改时引进新 SKU（如 G4-A Z4+2 + Z5+5 = 7 新品）
- 供应商资质 / 配送能力 / 账期条件混杂
- 需要快速上架（如 1-3 天内）

## §3. 推演链

### 3.1 第一性问句（M-DEC-003 v0.1 修正后铁律 ⭐）

```
❌ 错误路径（v0.1 原假设 / 5/6 推翻）：
   "默认账期 → 7 新品需协商 X 天账期 → 到货率管理"
   
✅ 正确路径（5/7 修正）：
   1. 先问采购方式：现金 vs 账期 (老板拍板 / 30 秒决定)
   2. 现金采购 → 简化路径（无账期协商 / 当日到货可行 / 资金压力但灵活）
   3. 账期采购 → 完整路径（账期 30/60/90 天 / 信用评估 / 配送频次）
   4. 混合采购 → 按品类（休食常现金 / 主食常账期）
```

### 3.2 现金采购路径（5/7 验证 ✅）

```
Step 1: 老板 / 财务批准现金额度（如 5000 元 / 7 SKU 试销）
Step 2: 选供应商（优先 7 家既有 / 非新供应商 / 减少风险）
Step 3: 当日下午下单 → 当晚或次日到货
Step 4: 上架 + 试销 30-60 天
Step 5: 试销期满 → 走 [[M-DEC-007]] 7 步评估 → 长存 / 淘汰
```

### 3.3 账期采购路径（备用 / 跨客户复用）

```
Step 1: 评估供应商资质（信用 / 配送 / 销售记录）
Step 2: 协商账期（30 / 60 / 90 天 / 季结 / 月结）
Step 3: 首批小试（10-30 件 / 不囤货 / 看动销）
Step 4: 30 天动销验证 → 续单或停单
Step 5: 账期落实记录在表 A 字段 22-24（supplier_name / supplier_id / delivery_frequency）
```

## §4. 临界值

| 维度 | 临界值 | 触发动作 |
|---|---|---|
| 调改前到货率 | < 80% | 当面延期 1 天 / 不强上 |
| 调改前到货率 | < 50% | 当周不补 / 走 R2 应急 |
| 现金采购单次额度 | > 10000 元 | 老板必 sign off / 不允许顾问代决 |
| 账期协商超期 | > 7 天未谈定 | 该 SKU 暂不引进 / 走 G-3 候选商品池 |
| 试销期 | < 30 天 | 数据不足 / 不可决策淘汰 |
| 试销期 | > 90 天 | 强制评估 / 走 [[M-DEC-007]] 7 步 |
| 7 家既有供应商利用率 | < 60% | 优先用既有 / 不开新供应商 |

## §5. 反事实（推翻条件）

本规则在以下情况**不适用**：
- 现金流极度紧张（5/6 老板访谈 §C #3 "门店从开业到 2026-1 月才打平" / 现金优先供主食 / 不为休食占用）
- 供应商垄断（如某地方特产仅 1 家可供 / 接受其条件）
- 季节性极强商品（年货 / 中秋 / 必须提前 60 天 / 时间线倒推）
- 战略合作品（与品牌方有联合营销协议 / 条件特殊）

## §6. 实战起源 + 5/7 修正

- **5/4 G4-A 备战**：原假设"7 新品需协商账期"
- **5/6 老板访谈 §D #3 验证 ✅**：老板原话"**可以现金采买。目前休食等很多品类都是现金采购**" → 推翻"默认账期"假设 + 验证"现金路径可行"
- **5/7 修正**：v0.1 重写 = 现金 vs 账期**双轨** / 第一问句"先问采购方式"
