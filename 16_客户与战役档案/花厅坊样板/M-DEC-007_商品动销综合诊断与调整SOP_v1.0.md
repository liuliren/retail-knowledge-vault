# M-DEC-007 商品动销综合诊断与调整 SOP v1.0

> **立项触发**：2026-05-07 用户在 G4-A 首战调改前指出：vault 有 POS 清洗 / 单品管理骨架 / 战役军师简报等单点资产，但**端到端商品动销 SOP 串不起来**。原 [[CLAUDE.md]] §13 反幻觉红线（已于 CLAUDE.md v2.x 治理重组，当前版本无 §13 章节）要求"商品诊断必须同时检查活动、陈列、缺货、价格、库存、客群影响"，但只立了禁令、没立做法——规则已内化入本 SOP §4（见 §4 表头注记）。
>
> **本 SOP 解决**：从 POS 数据 → 多维交叉 → 动销分级 → 归因决策 → 调整动作 → 任务清单 → 实施闭环 7 步端到端方法论。
>
> **v1.0 状态说明**：框架从 v0.1 候选完整继承并充实为可执行版本。`status: draft` — 升 `active` 须六哥完成 §9 四项签字门（含战役 #2 跨店验证数据）。原候选文件保留于同目录，不删除。

## §1. SOP 定位

**适用场景**：
- 战役内商品调改决策（如 G4-A 至 G4-H 8 面 SKU 选择）
- 品类宽深度调整（如 G4-B 牛奶瘦身 38→28 SKU 候选）
- 单品淘汰 / 保留 / 增排面 / 换位决策
- 新品引进 / 候选商品池筛选

**核心定位**：本 SOP 是 [[M-DEC-005_错品类货架重建SOP_v0.1_候选|M-DEC-005]]（错品类）+ [[M-DEC-006_陈列层数确认与区域落位SOP_v0.1_候选|M-DEC-006]]（陈列层数）的**前置 / 平级**（决定"放什么 SKU"），三者构成**"陈列调改三件套"**：

```
M-DEC-007 选 SKU(放什么) → M-DEC-006 定层数(放哪层) → M-DEC-005 清错品类(必要时)
                                       ↓
                                   货架最终方案
```

**禁止用本 SOP 替代**：
- 整面货架陈列结构调整 → [[M-DEC-006_陈列层数确认与区域落位SOP_v0.1_候选|M-DEC-006]]
- 错品类货架重建 → [[M-DEC-005_错品类货架重建SOP_v0.1_候选|M-DEC-005]]
- 整体品类规划 → [[品类管理]] 主流程

## §3. 各步详细 SOP

### S1 数据准备（2-4 小时 / 首访整理后启动）

清洗细节由 `/posclean` skill + `13_数据分析与工具脚本/POS清洗库_v0.1/` 全套涵盖（原为 [[POS数据清洗与商品标准化规范_v0.1]]，已由脚本库取代，2026-07-07）。本 SOP 不重复，仅说明**入口数据要求**：

| 项 | 必须 | 出处 |
|---|---|---|
| 1.1 POS 销售数据（SKU 级 / 近 90 天 / 含毛利字段）| ✅ | [[资料采集清单]] §四 4.1-4.5 |
| 1.2 全量商品档案（条码 / 名称 / 规格 / 进价 / 售价 / 品类 L1-L4） | ✅ | §五 5.1 |
| 1.3 当前 SKU 级库存数 | ✅ | §四 4.3 |
| 1.4 现场摸排（SKU × 排面 × 层位 × 包装 × 缺货）| ✅ | [[M-DEC-006_陈列层数确认与区域落位SOP_v0.1_候选|M-DEC-006]] §3 S1 |
| **1.5 活动期标记**（近 90 天活动 SKU + 期间）| ⚠️ **须主动采集**| §四 建议加项（见 §7.1 验证记录）|
| **1.6 价格变动历史** | ⚠️ **须主动采集**| 同上 |
| **1.7 竞品/区域市场价格参考**（≥3 个同类 SKU 的周边竞店价或线上市价，用于 S4 价格虚高判断）| ⚠️ **须主动采集** | 现场踩点 / 电商比价（同规格 SKU）/ 建议加入 [[资料采集清单]] §六（见 §7.1 验证记录）|

> **操作说明**：1.5 / 1.6 / 1.7 是本 SOP §4 反幻觉红线的数据基础维度（规则源自原 CLAUDE.md §13，已内化入本 SOP，见 §4 表头注记）。采集时：① 向门店店长或系统管理员索取近 90 天活动记录与调价记录；② 1.7 竞品价格通过走访周边同类门店或线上平台（拼多多 / 美团买菜）比对，记录采集日期与来源。若系统不支持导出，则通过访谈还原并记录不确定性，标注 `fact_layer: inferred`，进 S2 时降权处理。

### S2 七维数据交叉（1-2 天 / Excel 跑数据）

**核心铁律**：本 SOP §4 反幻觉红线要求 **7 维必须同时检查**，不可单维结论（原自 CLAUDE.md §13，已重组内化入本 SOP §4）。

| 维度 | 数据 | 看什么 | 红线 |
|---|---|---|---|
| 1. 销售 | 件数 / 金额 / 客单 | 排序 + 排序波动 | 不可仅凭销量判商品力（见 §4 红线②）|
| 2. 毛利 | 毛利额 / 毛利率 | 与销售交叉看真实贡献 | 高销低毛 vs 低销高毛 |
| 3. 库存 | 当前 / 周转 / 缺货率 | 缺货是低销还是真低需求? | 见 §4 红线④：不无依据扩写 |
| 4. 陈列位 | 黄金 / 边缘 / 顶 / 底 | 低销是位置差还是品错? | 见 §4 红线⑦：须综合考虑 |
| 5. 活动期 | 当前在活动? 近 90 天活动期外销量? | 自然动销 vs 促销拉动 | 见 §4 红线③：必须区分 |
| 6. 客群 | TPOS 用户群 / CDT 决策路径 | 卖给谁? 我们门店有这群人吗? | [[品类管理]] / [[顾客决策树CDT]] |
| **7. 价格** | **S1 第1.7项采集的竞品/市场价对比** | **售价 vs 区域均价偏离度；内部调价历史** | **不得凭印象认定价格虚高，须有 S1.7 采集数据支撑（见 §4 红线⑧）** |

**S2 出口模板**（SKU × 七维交叉表）：

```
SKU  | 销 | 毛 | 库 | 缺 | 陈列位   | 活动期    | 价格对比 | 客群  | 综合判断
A01  | 高 | 高 | 健 | 0  | 黄金视线 | 自然      | 市价平   | 主力  | 维持 / 增排
A02  | 中 | 低 | 滞 | 0  | 黄金手拿 | 促销拉动  | 市价平   | 边缘  | 调价 / 候补淘汰
A03  | 低 | 高 | 健 | 0  | 顶层     | 自然      | 高于市价  | 高端  | 调到黄金视线 / 考虑调价 / 观察
...
```

**数据缺失处理规则**（v1.0 新增）：
- 活动期数据缺失 → 标注 `活动=未知`，不可默认"无活动"；S4 归因时该分支悬留待补充
- 毛利字段空缺 → 向店长确认进价后手工录入，不可用类目均值代替
- 陈列位数据缺失 → S4 第一问"陈列位差？"标注"待现场实测"，不得推定
- **价格对比数据缺失（1.7）→ S2 第7维标注"价格=未采集"；S4"价格虚高？"分支必须悬留，不得凭主观印象走任何分支**

#### S2 + 第 7 维：客户视角验收（调改后 / 进入 S7）

> **客户原话**（花厅坊启明老板 §D#5）："**商品充足，销量有提升改善，核心是顾客匹配度。**"

原 6 维（销售+毛利+库存+陈列位+活动期+客群）= **调改前诊断维度**。
新增第 7 维「客户视角验收」= **调改后验证维度** / 不进 S2 表 / 进 S7 复测。

| 维度 7 | 数据 | 看什么 | 红线 |
|---|---|---|---|
| 客户视角验收（事后）| 销量提升 + 顾客匹配度 + 老板"自己说出来"的价值 | 调改后客户原话能否清晰复述价值？/ 不能复述 = 没产生价值 | 见本 SOP §4 红线⑧（不只看单点）/ S7 +24h/+1周/+14天三段验证 / [[CLAUDE.md]] 三回归"回归到价值" |

**触发节点**：S7 实施 + 复测段 / +24h / +1 周 / +14 天三段必走。

### S3 动销分级（ABC-Z 模型）

**ABC-Z 临界值定义**（**本店本品类**校准 / 不可跨品类直接套）：

| 等级 | 销售排序占比 | 累计销售贡献 | 调整方向 |
|---|---|---|---|
| **A** 高动 | 前 20% | 70-80% | 黄金位 / 增排 / 不轻动 |
| **B** 中动 | 21-50% | 15-20% | 黄金辅助位 / 维持 |
| **C** 低动 | 51-80% | 5-10% | 进 S4 归因 / 待决 |
| **Z** 异常 | 后 20% | <5% | **强制进 S4 归因** / 多数候选淘汰 |

**临界值校准说明**（花厅坊战役实测待填，见 §7.2）：
- 20/50/80 是帕累托经典起点，实际按品类调整
- 高周转生鲜类（如牛奶）A 级门槛可提高至前 15%；休食长尾类可适度放宽
- 各品类校准值回填至 §7.2 后，本节更新为实测值

**多维降级规则**（v1.0 新增 · 消除销售排序单维矛盾）：

> 上表为**销售排序初分**。初分后须叠加毛利降级检验，否则"高销但 0 毛利"SKU 无法操作。

| 触发条件 | 降级规则 | 目标等级 | 后续动作 |
|---|---|---|---|
| 初分 A 级，但毛利率 < [X]% 或毛利额为负 | 降至 B 末位 | B（末位）| 强制进 S4 归因（调价？减少促销成本？） |
| 初分 B 级，且毛利率 < [Y]% 且销售排序后 30% | 直接降至 C 级 | C | 进 S4 归因 |

> [X] / [Y] 为回填槽位 — 默认不填写，由六哥在 §7.2 花厅坊实测后确定。在槽位填入前，遇高销低毛 SKU 时执行以下替代规则：**将该 SKU 单独标注 `毛利异常=待裁定`，不自动降级，须六哥或老板现场裁决后再分级**（严守 §4 红线③：不得把推测写成事实）。

### S4 低动销归因决策树（C / Z 必经）

```
低动销 SKU
  ├── 陈列位差? (顶 / 底 / 边缘)
  │     ├── Yes → S5 调位测试(不淘汰) / 观察期 1 周
  │     └── No → 下一问
  ├── 活动期外的销量? (剔除促销影响后)
  │     ├── 自然销量也低 → 下一问
  │     └── 仅活动期外低 → S5 配活动 + 观察
  ├── 价格虚高? (vs 市价 / 竞品)
  │     ├── Yes → S5 调价测试 / 观察期 1 周
  │     └── No → 下一问
  ├── 缺货导致低销?
  │     ├── Yes → 进缺货管理流程（[[单品管理]] 缺货补货规则 / 不归因为低动销）
  │     └── No → 下一问
  ├── 客群不匹配? (TPOS / CDT 不符)
  │     ├── Yes → S5 候选淘汰
  │     └── No → 下一问
  └── 真不需要?
        ├── Yes → S5 淘汰（须老板确认）
        └── No → 留观察(下周复看)
```

**铁律**：每个低动销 SKU 必须有归因标签，**不允许"销量低 → 直接淘汰"**。

**各分支防误判提示**（v1.0 补充）：
- "陈列位差"需有实测数据支撑（层位 + 排面位置），不可凭印象认定
- "客群不匹配"须参照 CDT 或门店 TPOS 画像，非主观判断
- "真不需要"是最后一道门，须老板/店长 ack 才可执行淘汰

### S5 调整动作决策

调整动作清单（可组合）：

| 动作 | 触发条件 | 注意 |
|---|---|---|
| 保留+维持 | A 级 / 健康 | 不轻动 |
| 保留+增排 | A 级 / 缺货率高 / 顾客需求未满 | 增排面占用其他 SKU 资源 |
| 保留+调位 | C 级 / 陈列位差 | 测试 1 周后再决 |
