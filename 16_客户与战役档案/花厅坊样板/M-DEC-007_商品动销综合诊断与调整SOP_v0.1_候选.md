# M-DEC-007 商品动销综合诊断与调整 SOP v0.1（候选）

> **立项触发**：2026-05-07 用户在 G4-A 首战调改前指出：vault 有 POS 清洗 / 单品管理骨架 / 战役军师简报等单点资产，但**端到端商品动销 SOP 串不起来**。[[CLAUDE.md]] §13.5 反幻觉红线（"商品诊断必须同时检查活动、陈列、缺货、价格、库存、客群影响"）只立了禁令、没立做法。
>
> **本 SOP 解决**：从 POS 数据 → 多维交叉 → 动销分级 → 归因决策 → 调整动作 → 任务清单 → 实施闭环 7 步端到端方法论。
>
> **战役验证状态**：v0.1 候选立项（2026-05-07）→ 5/13-5/20 战役 #1 G4-A 至 G4-H 8 面 8 次实战 → 5/27 G05 阶段门 → 5/27-6/30 跨战役 #2 验证 → 6/30 升 v1.0 落 `04_商品诊断与商品力提升/中层决策库/M-DEC-007_..._v1.0.md`（与 [[M-DEC-005]] / [[M-DEC-006]] staging 一致）。
>
> **本文件位置约定**：`16_客户与战役档案/花厅坊样板/`（战役档案候选层）→ **禁止以 v0.1 状态进入 [[04_商品诊断与商品力提升]] 通用方法论层**（[[CLAUDE.md]] §16.7 + [[战役指挥板]] §三月四不做 #4）。

## §1. SOP 定位

**适用场景**：
- 战役内商品调改决策（如 G4-A 至 G4-H 8 面 SKU 选择）
- 品类宽深度调整（如 G4-B 牛奶瘦身 38→28 SKU 候选）
- 单品淘汰 / 保留 / 增排面 / 换位决策
- 新品引进 / 候选商品池筛选

**核心定位**：本 SOP 是 [[M-DEC-005]]（错品类）+ [[M-DEC-006]]（陈列层数）的**前置 / 平级**（决定"放什么 SKU"），三者构成**"陈列调改三件套"**：

```
M-DEC-007 选 SKU(放什么) → M-DEC-006 定层数(放哪层) → M-DEC-005 清错品类(必要时)
                                       ↓
                                   货架最终方案
```

**禁止用本 SOP 替代**：
- 整面货架陈列结构调整 → [[M-DEC-006]]
- 错品类货架重建 → [[M-DEC-005]]
- 整体品类规划 → [[品类管理]] 主流程

## §3. 各步详细 SOP（骨架版 / 5/13 战役收口后充实细则）

### S1 数据准备（2-4 小时 / 首访整理后启动）

[[POS数据清洗与商品标准化规范]] 全套已涵盖清洗细节。本 SOP 不重复，仅说明**入口数据要求**：

| 项 | 必须 | 出处 |
|---|---|---|
| 1.1 POS 销售数据（SKU 级 / 近 90 天 / 含毛利字段）| ✅ | [[资料采集清单]] §四 4.1-4.5 |
| 1.2 全量商品档案（条码 / 名称 / 规格 / 进价 / 售价 / 品类 L1-L4） | ✅ | §五 5.1 |
| 1.3 当前 SKU 级库存数 | ✅ | §四 4.3 |
| 1.4 现场摸排（SKU × 排面 × 层位 × 包装 × 缺货）| ✅ | [[M-DEC-006]] §3 S1 |
| **1.5 活动期标记**（近 90 天活动 SKU + 期间）| ⚠️ **vault 缺**| §四 暂无 → **建议加项** |
| **1.6 价格变动历史** | ⚠️ **vault 缺**| 同上 |

> **Gap**：1.5 / 1.6 是 §13.5 红线维度但 [[资料采集清单]] §四未覆盖 — backlog 已记入 [[project_knowledge_base_improvement_backlog]] §六 / 5/8 复盘日触发资料采集清单升 v0.5。

### S2 六维数据交叉（1-2 天 / Excel 跑数据）

**核心铁律**：§13.5 反幻觉红线 **6 维必须同时检查**，不可单维结论。

| 维度 | 数据 | 看什么 | 红线 |
|---|---|---|---|
| 1. 销售 | 件数 / 金额 / 客单 | 排序 + 排序波动 | 不可仅凭销量判商品力（§13.6）|
| 2. 毛利 | 毛利额 / 毛利率 | 与销售交叉看真实贡献 | 高销低毛 vs 低销高毛 |
| 3. 库存 | 当前 / 周转 / 缺货率 | 缺货是低销还是真低需求? | §13.10 不无依据扩写 |
| 4. 陈列位 | 黄金 / 边缘 / 顶 / 底 | 低销是位置差还是品错? | §13.13 综合考虑 |
| 5. 活动期 | 当前在活动? 近 90 天活动期外销量? | 自然动销 vs 促销拉动 | §13.7 必须区分 |
| 6. 客群 | TPOS 用户群 / CDT 决策路径 | 卖给谁? 我们门店有这群人吗? | [[品类管理]] / [[顾客决策树CDT]] |

**S2 出口模板**（SKU × 六维交叉表）：

```
SKU  | 销 | 毛 | 库 | 缺 | 陈列位   | 活动期    | 客群  | 综合判断
A01  | 高 | 高 | 健 | 0  | 黄金视线 | 自然      | 主力  | 维持 / 增排
A02  | 中 | 低 | 滞 | 0  | 黄金手拿 | 促销拉动  | 边缘  | 调价 / 候补淘汰
A03  | 低 | 高 | 健 | 0  | 顶层     | 自然      | 高端  | 调到黄金视线 / 观察
...
```

#### S2 + 第 7 维：客户视角验收（5/8 §21 反向回写补强 / 老板访谈金句 #5 + §D#5）

> **客户原话**（§D#5）："**商品充足，销量有提升改善，核心是顾客匹配度。**"
> **客户原话**（金句 #5）："门店所有人员的整体素质不高 / **需要简单的语言和方式 / 尽快拿到实实在在的结果**"

原 6 维（销售+毛利+库存+陈列位+活动期+客群）= **调改前诊断维度**。
新增第 7 维「客户视角验收」= **调改后验证维度** / 不进 S2 表 / 进 S7 复测。

| 维度 7 | 数据 | 看什么 | 红线 |
|---|---|---|---|
| 客户视角验收（事后）| 销量提升 + 顾客匹配度 + 老板"自己说出来"的价值 | 调改后客户原话能否清晰复述价值？/ 不能复述 = 没产生价值 | §13.14 不只看单点 / §21 24h 验证 / [[CLAUDE.md]] §20 三回归"回归到价值" |

**触发节点**：S7 实施 + 复测段 / +24h / +1 周 / +14 天 三段必走。

### S3 动销分级（ABC-Z 模型）

**ABC-Z 临界值定义**（**本店本品类**校准 / 不可跨品类直接套）：

| 等级 | 销售排序占比 | 累计销售贡献 | 调整方向 |
|---|---|---|---|
| **A** 高动 | 前 20% | 70-80% | 黄金位 / 增排 / 不轻动 |
| **B** 中动 | 21-50% | 15-20% | 黄金辅助位 / 维持 |
| **C** 低动 | 51-80% | 5-10% | 进 S4 归因 / 待决 |
| **Z** 异常 | 后 20% | <5% | **强制进 S4 归因** / 多数候选淘汰 |

**注意**：
- 临界值 20/50/80 是经典帕累托起点，**本店实测后按品类调整**
- 单维度排序（销售）不够 — 必须 S2 六维交叉后再分级（高销但 0 毛利的 SKU 不能算 A 级）

### S4 低动销归因决策树（C / Z 必经）

```
低动销 SKU
  ├── 陈列位差? (顶 / 底 / 边缘)
  │     ├── Yes → S5 调位测试(不淘汰)
  │     └── No → 下一问
  ├── 活动期外的销量? (剔除促销影响后)
  │     ├── 自然销量也低 → 下一问
  │     └── 仅活动期外低 → S5 配活动 + 观察
  ├── 价格虚高? (vs 市价 / 竞品)
  │     ├── Yes → S5 调价测试
  │     └── No → 下一问
  ├── 缺货导致低销?
  │     ├── Yes → 进 §13.10 缺货管理(不归因低销)
  │     └── No → 下一问
  ├── 客群不匹配? (TPOS / CDT 不符)
  │     ├── Yes → S5 候选淘汰
  │     └── No → 下一问
  └── 真不需要?
        ├── Yes → S5 淘汰
        └── No → 留观察(下周复看)
```

**铁律**：每个低动销 SKU 必须有归因标签，**不允许"销量低 → 直接淘汰"**。

### S5 调整动作决策

调整动作清单（可组合）：

| 动作 | 触发条件 | 注意 |
|---|---|---|
| 保留+维持 | A 级 / 健康 | 不轻动 |
| 保留+增排 | A 级 / 缺货率高 / 顾客需求未满 | 增排面占用其他 SKU 资源 |
| 保留+调位 | C 级 / 陈列位差 | 测试 1 周后再决 |
| 保留+调价 | C 级 / 价格虚高 | 测试 1 周后再决 |
| 保留+配活动 | C 级 / 活动期外低 | 进 [[S06_营销活动配套]] |
| 减排 | B 级 / 重复 SKU | 释放排面给增排 |
| 淘汰 | Z 级 / 归因明确 / 老板认可 | 配套 [[单品管理]] 滞销淘汰规则 |
| 补新 SKU | 排面释放后 / 候选商品池有 | 进 [[单品管理]] 新品引进 |

## §4. 反幻觉红线（[[CLAUDE.md]] §13）

| 红线 | 违反例 | 正确做法 |
|---|---|---|
| §13.4 | "活动期销量好就保留 SKU" | S2 必须区分自然 vs 促销拉动 |

| §13.7 | "把推测写成事实" | 调整动作必须有 S2-S4 依据链 |

| §13.13 | "SKU 决策不综合" | 销量+毛利+库存+陈列+活动+客群 |
| §13.14 | "活动效果只看单点" | S7 必须前 / 中 / 后三段 |
