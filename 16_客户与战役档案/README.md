# 16_客户与战役档案

## 一、目录定位

本目录是**战役视角**的客户与项目档案中心 —— 一个客户/一个交付项目对应一场"战役"，每场战役一个独立目录，沉淀战役全周期治理材料：章程 / 里程碑 / 进展 / 风险 / 决策 / 交付物 / 变更 / 复盘。

它服务于：
- 让 vault 不仅有"知识"还有"战役全过程治理痕迹"
- 让 PM 控制层（[[战役指挥板]]）有真实的档案抓手，而不是只在仪表上画格子
- 让战役完成后的 AAR（G06 战后复盘）有完整的过程依据可查
- 让伙伴 Onboarding（Phase 1+）能直接读懂任何一场历史战役

> 架构定位：7 层架构第 3 层「PM 控制层」配套档案目录（参 [[2026-05-04_知识库重构方案_v0.2_完整作战方案]] §3.1 / §4.3.2）。

## 二、新增一级目录的原因（按 [[CLAUDE.md]] §15.1）

### 2.1 新增原因

vault 原有 09_ 是**业务模块视角**的门店案例目录（按"商品诊断 / 品类规划 / 陈列整改 / 营销活动 / 经营复盘"等业务条线组织）。但战役治理需要的"章程 / 里程碑 / 决策 / 风险 / 变更 / AAR"是**横跨业务条线的战役元数据**，强行塞进 09_ 会破坏 09_ 的业务模块组织逻辑。

v0.2 重构方案确定：**业务知识沉淀 vs 战役过程治理**是两套正交视角，应分目录承载。

### 2.2 边界

本目录**只存战役全周期治理材料**：

- 战役章程、里程碑、进展、风险、决策、交付物清单、变更控制日志、AAR
- 战役层接触日志（销售对话 / 老板访谈 / 阶段门评审记录）
- 战役级 RACI 与 §G 段（参 v0.2 §4.6.2 / 5.1）

本目录**不存**：

- 商品诊断、品类规划、陈列方案、营销活动方案 → `09_门店案例与项目复盘/<门店>/`
- 客户交付报告成品 → `10_咨询交付模板/03_客户报告模板/` 或门店案例的"08_交付报告"
- 商圈调研、客群画像 → `03_商圈分析与客群调研` + `09_/<门店>/01_商圈与客群`
- 已发布的对外内容 → `12_自媒体内容与表达转化`
- 客户验证日志（与本目录互补）→ `15_刻意练习与成长/客户验证日志/`

### 2.3 影响范围

- **新增**：vault 一级目录从 16 个（含 90_/99_）扩为 17 个
- **配套修改**：[[CLAUDE.md]] §4 一级目录结构 + §5 各目录用途说明 + [[知识库总索引]] 新增 16_ 段
- **不动**：所有既有 09_ 门店项目文件、所有既有 10_ 模板文件、所有 01_ 方法论文件
- **协同**：[[战役指挥板]]（00_/）改为本目录的"全战役视角入口"，本目录内 [[16_客户与战役档案/00_战役指挥板]] 是"档案目录视角入口"，二者互链

### 2.4 与既有目录的辨析

| 目录             | 视角         | 核心问题             | 与 16_ 的关系                                     |
|------------------|--------------|----------------------|---------------------------------------------------|
| `09_门店案例与项目复盘` | 业务模块视角 | 这家门店在每个业务条线发生了什么？ | **正交关系**：09_ 按业务条线（商品 / 品类 / 陈列 / 营销 / 经营），16_ 按战役元数据（章程 / 里程碑 / 决策 / 风险 / AAR）。同一战役在两边都有目录，互引但不重复。|
| `10_咨询交付模板` | 模板视角     | 给客户交付什么标准物？  | **上游**：16_ 内的"06_交付物清单"指向 10_ 的具体模板版本。10_ 是模板库，16_ 是某场战役实际产出哪些交付物的清单。|
| `15_刻意练习与成长/客户验证日志` | 个人成长视角 | 这次客户接触验证 / 推翻了哪些方法论？ | **互补关系**：16_ 记"战役怎么走"，15_ 记"我们的方法论是不是真的有用"。同一接触可能两边都有记录。|
| `00_/战役指挥板` | PM 仪表视角   | 全部战役总览状态 | **互链**：00_ 是仪表（一行一战役），16_ 是档案（一目录一战役）。|

## 四、新战役开案 SOP（与 G04 战役章程仪式联动）

> 详细治理 SOP 见 `10_咨询交付模板/00_战役SOP/_治理SOP/G04_战役章程.md`（Phase 0 D4 落地）。本节是档案侧操作指引。

1. 在本目录下新建 `<客户代号>/`（命名按 [[文件命名规范]]，去标识化）
2. 从 `_模板/` 复制 8 份模板进入新目录
3. 同步在 `09_门店案例与项目复盘/<客户代号>/` 建业务条线目录（如战役涉及实地交付）
4. 在 `00_战役指挥板.md`（本目录）+ `00_/战役指挥板.md`（vault 总）双向登记
5. 主持 G04 战役章程会议（1 小时）填 00_战役章程.md
6. 标注 frontmatter `quadrant`：未签约期 III（外延复制），签约后视客户敏感度可调 IV
7. 标注 `client_safety`：默认 `internal_only`，需对外演示派生 `client_ready` 副本
