# 麦肯锡 + 苹果 简报/视觉设计规范深挖 v0.1

> **结论先行**：做"一页看懂"的店长诊断卡，应取**麦肯锡的逻辑骨架**（行动式标题 + 金字塔结论先行 + 一图一信息 + 灰底高亮一处）和**苹果的视觉皮肤**（巨大字号 hero、克制配色黑白灰+一抹蓝、大留白、负字距）。两者交集就是：**一句结论当标题 + 一个核心数字/红绿灯当主角 + 其余信息退为灰色背景**。
>
> **可信度说明**：本报告标注每条来源与交叉验证情况。⚠️ 标记 = 单源/不确定，需落地前再核。所有 hex 色值均为第三方品牌库重建值（非官方 token），客户正式交付前建议直接查 apple.com / 麦肯锡官方实色。

## ① 麦肯锡可落地规范

### 1.1 Action Title（行动式标题）—— 骨架中的骨架

| # | 规则 | 可落地做法 | 来源/验证 |
|---|---|---|---|
| AT-1 | 标题是**结论（so-what）不是主题** | "Market Overview" ✗ → "德国市场年增12%，是美国的3倍" ✓ | 多源：deckary, analystacademy, slidescience, slideworks |
| AT-2 | 必须是**完整句子**（有动词，无冒号/省略号/碎片） | 写成主谓宾的陈述句 | 多源：deckary, analystacademy, slideworks |
| AT-3 | **最多 2 行，绝不超过** | CSS 限高 + 超出即拆页 | 强多源（两站逐字一致） |
| AT-4 | **≤15 词**（中文≈≤30字），一行最佳 | — | 多源：slideworks, deckary |
| AT-5 | **一标题一信息**；出现"和/且"就拆成两页 | 标题里若要写"and"，说明该拆 | 多源：deckary |
| AT-6 | **横向连读成故事**：只读全卡标题应能读懂整个论证（横向逻辑） | 自检："盖住正文只读标题，故事完整吗？" | 多源：deckary, slideworks |
| AT-7 | **主动语态 + 尽量量化** | "营收增长14%"而非"营收被增长" | 多源：deckary, slidescience |
| AT-8 | 位置：**页面最顶部**；全卡所有标题**字号完全一致**，不因内容多寡而缩放 | 同类元素永远同号 | 多源：analystacademy, deckary |

**好坏标题对照（逐字摘自来源）**
- 弱："供应链流程可优化" → 强："优化供应链流程，降本20%"（slideworks）
- 流水账："我们访谈了专家和内部干系人以识别降本杠杆" → 结论："识别出8个高影响降本杠杆"（slideworks）
- 被动："项目结构与时间表由指导委员会决定" → 主动："指导委员会决定项目结构与时间表"（slideworks）

### 1.2 金字塔原理（Minto）+ SCQA

| # | 规则 | 落法 | 来源 |
|---|---|---|---|
| PY-1 | **结论先行/自上而下**：先抛主结论，再分组支撑；表达顺序与推理顺序相反 | 卡首=答案，卡身=论据 | 多源：modelthinkers, wikipedia(Minto) |
| PY-2 | **纵向逻辑**：每层是其下层的总结（标题=该页正文的"金字塔节点"） | action title 即金字塔节点 | 多源 |
| PY-3 | **横向逻辑**：同组论点靠演绎或归纳相连 | — | 多源 |
| PY-4 | **分组必须 MECE**（互斥且穷尽，Minto originator） | — | 强多源 |
| PY-5 | ⚠️ "三点法则"：每个结论约配3个支撑点 | 启发式，非硬规则 | 单源 modelthinkers |
| SCQA-1 | **SCQA = 情境→冲突→疑问→答案**，构成开场；Answer 即金字塔顶的统领思想 | 诊断卡开头：现状→问题→所以怎么办 | 多源：modelthinkers, thinkinsights |

### 1.3 图表规范（Zelazny《Say It With Charts》+ Tufte）

**Zelazny 核心：先定"比较类型"，图随之而定（不是看数据选图，是看你要说的话选图）。**

| 比较类型 | 说什么 | 用什么图 | 信号词 |
|---|---|---|---|
| Component 构成 | 占总体的百分比 | **饼图** | "份额""占比""近一半" |
| Item 排序 | 项目排名 | **条形图（横）** | "排名""谁最高""大于" |
| Time series 时序 | 随时间变化 | **柱图**（点少）/**折线**（点多） | "上升""趋势""增长" |
| Frequency 分布 | 落在各区间 | **直方图** | "范围""分布""集中" |
| Correlation 相关 | 两变量关系 | **散点图** | "随…增加""与…相关" |

来源：强多源（informit-Zelazny章节 + think-cell + strategyu，映射逐字一致）。注：横条=排名 / 竖柱=时序，是 Zelazny 的关键区分。

**图表去 chartjunk / 提高数据墨水比（Tufte）的具体动作**
- ION-1 **一图一信息**；图标题=结论句（action title），不是主题。自检："盖住数据只读标题，还懂吗？"（多源）
- ION-2 **删**：网格线、边框、图例、3D、阴影、背景图案、装饰色、多余刻度。（多源：caylent, holistics, nngroup）
- ION-3 **直接标注代替图例**——标签贴在数据旁，别让读者来回找图例。（多源：umbrex, analystacademy, caylent）
- ION-4 **只高亮一条**：主角用强调色，其余全置灰。"全都响=没重点"。（强多源：umbrex, analystacademy, storytellingwithdata）
- ION-5 **条形按数值排序**（非字母/随机），让叙事顺眼；时序/有序类别保持自然序。（多源）
- ION-6 **饼图切片≤5**；只在某片>50%且想突出时才用饼，否则用柱图。（多源：strategyu, EU指南）
- ION-7 **工作马三件套 = 条/柱/线**，覆盖约80-90%需求；饼/散点/直方按 Zelazny 表按需取用。（多源，editorial stance）
- ION-8 最重要的那个数字**做到最大 + 强调色**，其余退场。（多源：strategyu, analystacademy）

### 1.4 版式 / 配色 / 字体

| 维度 | 规范 | 来源/可信度 |
|---|---|---|
| 单页结构 | **标题(顶)→单一主图/正文→来源脚注+页码(底)**；所有元素跨页网格对齐，翻页不跳位 | 高，多源 deckary |
| 视线流 | 一页一信息，左对齐正文，仅短标题居中；大留白（"别填满每寸"，四边留呼吸） | 高，deckary |
| 留白 | 慷慨留白；⚠️"标题下加横线""章节导航条""Z型流"为常见但**本轮来源未明确证实**，按风格选用 | 标注不确定 |
| 来源脚注 | 底部小号灰字，每个数据须标源，以"Source:"起；脚注在源行上方 | 高，deckary+analystacademy |
| 页码 | 始终有（底角）；可选日期/保密标记 | 高，deckary |
| Take-away 框 | 单独 call-out 框（常在底部）放核心影响句，用唯一强调色（真实麦肯锡卡用橙色框） | 中，analystacademy 单卡观察 |
| **配色** | **深蓝主**：McKinsey "Black Pearl" `#051C2C`（深navy基色）+ "Blue Ribbon" `#2251FF`（亮蓝强调）+ 白 `#FFFFFF`；灰作次级 | 高（encycolorpedia+brandfetch一致），但⚠️均为品牌库重建值非官方 |
| 调色纪律 | **全卡≤3-4色**：1深基色 + 灰(次级) + 1强调色(仅强调，不装饰)；同色永远代表同类数据(如绿=正) | 高，deckary+analystacademy |
| 同行差异 | McKinsey 蓝 / BCG 绿(≈`#147B58`⚠️单源) / Bain 红(`#ee3224`) | 中，hex 单源不确定 |
| **字体** | 品牌字体：**Bower**(衬线,展示) + **McKinsey Sans**(无衬线,正文)；实际 PPT 用安全字替：**Georgia**(标题) / **Arial**(正文)。BCG/Bain 默认 Arial/Helvetica | 高，slideworks+deckary |
| 字号 | ⚠️ 各源给值不一(非官方)：标题~24-28pt，正文~16-18pt(底10-12)，图标签~12-14，源行8-10。**稳的是相对规则**：标题最大且全卡同号，绝不缩正文塞内容 | 字号绝对值标注不确定 |

**可落地起步色板（CSS）**：
```css
--mck-base:   #051C2C;  /* 深navy 基色/标题底 */
--mck-accent: #2251FF;  /* 亮蓝 强调（只用于强调）*/
--mck-bg:     #FFFFFF;
--mck-grey:   #6E6E6E;  /* 次级文本/置灰，灰为惯例确认，具体值自定 */
/* 字体 */ font-family: Georgia, serif;            /* action title */
/* 正文 */ font-family: Arial, Helvetica, sans-serif;
```

## ② 苹果可落地规范

### 2.1 字体与字号（SF）

| # | 规则 | 落法 | 来源/可信度 |
|---|---|---|---|
| SF-1 | **SF 光学尺寸切换点=20pt**：≥20pt 用 SF Pro **Display**，≤19pt 用 SF Pro **Text** | 大标题用 Display，正文用 Text | 高，多源(wikipedia-SF, iOS HIG镜像) |
| SF-2 | OS 自动切换，**Web/设计工具不会**自动切——`-apple-system` 在 Safari 会自动解析光学尺寸 | 自托管时手动分 Display/Text | 单源强 wikipedia-SF |
| SF-3 | Web 字体栈：`-apple-system, BlinkMacSystemFont, "Helvetica Neue", Arial, sans-serif`；⚠️ 别把 `-apple-system` 放在 `font` 简写首位(被当厂商前缀跳过)，用 `font-family` | 直接用 | 高，多源(css-tricks) |
| SF-4 | 历史：乔布斯时代 keynote 用 **Myriad Pro**(Semibold/Light)；2016起换 SF。做"复古乔氏"风可用 Myriad Pro Semibold | — | 单源强 wikipedia-typography |

**iOS HIG 字号阶梯（pt，含字重）—— 多源证实尺寸，⚠️标题字重历代漂移，现行用 Regular**

| 层级 | 字号 | 字重 |
|---|---|---|
| Large Title | 34 | Regular |
| Title 1 | 28 | Regular（旧版曾 Light）|
| Title 2 | 22 | Regular |
| Title 3 | 20 | Regular（=Display 切换点）|
| **Headline** | 17 | **Semibold** |
| Body | 17 | Regular |
| Callout | 16 | Regular |
| Subhead | 15 | Regular |
| Footnote | 13 | Regular |
| Caption 1 | 12 | Regular |
| Caption 2 | 11 | Regular（11pt 是可读下限）|

来源：codershigh(iOS HIG镜像)+eonist gist 两独立表证实尺寸；developer.apple.com 现行表 JS 渲染无法逐字抓取。**字重唯一确定差异：Headline=Semibold，其余正文/标题=Regular。**

### 2.2 字重 / 大字行为 / 字距

- AP-W1 **大 hero 字/数字用 light/regular 字重** + 大行间距；颜色纯白/灰白/近黑。（单源 VoltAgent，与观察一致）
- AP-W2 **负字距(tracking)**：大字收紧、小字放松，苹果按尺寸自动调。落地参考：34px+ 标题 `letter-spacing: -0.02em ~ -0.03em`；17px 正文 ≈ `-0.025em`(-0.43px)；15px 中性；11-13px 略正(+0.01em)。（多源：eonist+VoltAgent；符号与量级可靠，小数为近似⚠️）
- 通则：**字越大越要收紧字距**（同字面在 16px 合适，72px 会显"松散"）。

### 2.3 一屏一念 / 留白 / 高对比

- AP-1 **一屏一念（one idea per slide）**：Garr Reynolds《Presentation Zen》核心，明确师承乔布斯 keynote；用单念页序列替代项目符号列表。五支柱：简单/清晰/克制/和谐(留白)/连接。（高，presentationzen）
- AP-2 **3 秒一瞥测试**（Duarte）：slide 是"glance media"像广告牌，应 3 秒内看懂。（多源 hbr+duarte）
- AP-3 **留白是 slide 的氧气**；任何为塞内容牺牲留白的页都太挤。（Duarte）
- AP-4 **高对比 + 单一大 hero**：一个大数字 或 一张居中产品图，巨量负空间，hero 用轻字重。（综合 Reynolds/Duarte/乔氏 keynote，约定非逐字）

### 2.4 设计哲学（Markkula 备忘录）→ 版式

1977 Markkula《The Apple Marketing Philosophy》三原则（高可信，多源逐字）：
- **Empathy 共情**："比任何公司都更懂客户的需求。"→ 卡面向店长说话，用他们的语言（不堆术语）。
- **Focus 专注**："为做好决定做的事，必须砍掉所有不重要的机会。"→ 一页只讲一件事，删尽非核心。
- **Impute 灌输/印象**："人确实以貌取书。我们就算有最好的产品，若呈现得潦草，就会被认为潦草；呈现得专业有创意，就能灌输出我们想要的品质。"→ **诊断卡的精致度本身就是质量信号**：粗糙的卡=粗糙的诊断，干净专业的卡把"靠谱、严谨"印到内容上。
- 配套座右铭："Simplicity is the ultimate sophistication"——深度简单(懂本质才能去掉非本质)，非表面整洁。（高）

## ③ 给"店长诊断卡"的合成建议（红绿灯卡）

> 把**麦肯锡的脑**装进**苹果的脸**。店长素养不高 → Empathy(说人话) + Focus(一页一事) + Impute(精致即靠谱) 三条苹果哲学正好治"看不懂的报告"。

### 3.1 标题怎么写（取 AT-1/2/4/7 + Empathy）
- 用**行动式标题=一句结论**，主动语态、量化、≤2行、店长能听懂的大白话。
- ✗"水产品类分析" → ✓"**水产毛利偏低，3个单品拖了后腿，换掉能多赚X元/月**"
- 全卡若多页，**标题连读成故事**（AT-6）：现状结论→问题结论→怎么改结论。
- 卡内不出现术语/条码（符合 client_shareable 0术语铁律）。

### 3.2 字号层级（融合苹果 SF 阶梯 + 麦肯锡"标题最大且同号"）
建议三档，对比拉满（一瞥测试 AP-2）：
- **Hero 主数字/结论**：48-64px，light/regular，负字距 `-0.02em`（苹果 hero 法）——一眼抓住"赚还是亏"。
- **Action 标题**：28-34px（≈苹果 Title 1/Large Title），全卡同号不缩（麦肯锡 AT-8）。
- **正文/红绿灯表/SKU**：16-17px Regular；**脚注/来源** 11-13px 灰字。
- 字体：可直接用苹果 web 栈 `-apple-system,...`（中文配苹方/思源黑体），干净现代、Impute 拉满。

### 3.3 留白与版式（苹果留白 + 麦肯锡单页结构）
- **单页锚定结构**：顶=一句结论标题；中=红绿灯主表(主角)；底=调改三步 + 来源/日期小灰字。
- 大留白（AP-3），四边留呼吸，元素网格对齐。
- 视线流左上→右下：左上放最痛的红灯，右下放"怎么办"。

### 3.4 配色（苹果克制底 + 麦肯锡强调纪律 + 红绿灯语义）
- **底色用苹果**：白底 `#FFFFFF` / 浅灰分区 `#F5F5F7`，正文近黑 `#1D1D1F`——专业、不刺眼。
- **品牌强调用麦肯锡深蓝**：标题/表头 `#051C2C`，强调蓝 `#2251FF`（或苹果 `#0071E3`，二选一，**只用一个蓝**当主强调）。
- **红绿灯是唯一允许的多色**，且语义固定（ION-4 同色同义）：
  ```css
  --light-red:    #E5484D;  /* 差/汰换 */
  --light-amber:  #F5A623;  /* 警示/观察 */
  --light-green:  #2F9E44;  /* 优/保留 */
  ```
  红绿灯之外一律黑白灰，**只让红绿灯"响"**——全卡最多 深蓝+灰 + 红黄绿三盏灯，不超 ION-2/4。

### 3.5 SKU 清单怎么排（取 ION-3/4/5/8 + 直方思维）
- **按数值排序**（ION-5）：默认按毛利/动销从差到好（或反），让差的扎堆在顶，店长一眼看到该砍谁。
- **直接标注**（ION-3）：每行 SKU 名旁直接标数字+灯色，不要图例。
- **只高亮要动手的行**（ION-4）：建议汰换的行底色淡红，保留的灰，别整张表花。
- **最关键数字做最大**（ION-8）：如"该砍N个"用 hero 字号。
- 一张表只回答一个问题（ION-1）；问题多就拆卡（AT-5）。
- 底部 take-away 框（麦肯锡）放一句"**所以：本周先做这3件**"，用强调蓝框。

### 3.6 落地自检清单
- [ ] 盖住表格只读标题，店长懂结论吗？(AT-1, ION-1)
- [ ] 3 秒一瞥能看懂红灯在哪、要干啥吗？(AP-2)
- [ ] 全卡是否≤深蓝+灰+红黄绿？多余色删掉。(ION-2/4)
- [ ] 有无术语/条码漏出？(client_shareable 铁律)
- [ ] 标题全卡同号、最大文本？(AT-8)
- [ ] 来源/日期小灰字在底部？(麦肯锡脚注)
