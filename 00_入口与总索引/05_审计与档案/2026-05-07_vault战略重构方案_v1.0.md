# 晟果新零售 vault 战略重构方案 v1.0

> **触发**：2026-05-07 用户要求 vault 全盘规划 + 世界级咨询公司（类麦肯锡）标准结构 + Karpathy LLM wiki 方法论结合 + 晟果独有方法论显式化。
>
> **设计原则**：
> 1. **McKinsey 信息架构** — 5 层 hierarchy（战略 / 方法论 / 治理 / 作战 / 审计）/ MECE 分类
> 2. **Karpathy LLM wiki** — Main entry + Topical pages + LLM-friendly frontmatter + 双链密集
> 3. **晟果独有方法论** — 三力骨架 + 经营控制链 + 战役+M-DEC 双轨 + 24h 客户验证
> 4. **MVP-first** — Phased / 立刻可用 / 实战驱动 / 不追求一次完美
>
> **不变量**：
> - CLAUDE.md §4 vault 19 一级目录骨架 — **不动**
> - 6 v1.0 表 + 引进表 + 表 E — **不动**
> - 16_/花厅坊样板 战役档案 — **不动**
> - 99_归档/ 已封存 — **不动**

## §1. 当前状态 vs 目标状态

### 1.1 当前痛点（McKinsey 审计 5/7 已识别）

| # | 痛点 | 数据 |
|---|---|---|
| 1 | **00_入口 21 文件 flat 无层级** | 各类目混杂（战略 / 方法论 / 治理 / 作战 / 审计 同级）|
| 2 | **缺 Karpathy 风格 main entry page** | LLM 入口分散 / 无统一导航 hub |
| 3 | **晟果独有方法论分散** | 三力骨架 / 经营控制链 / 四象限 散落多文件 / 缺主索引 |
| 4 | **frontmatter 不一致** | quadrant 缺 35%（51 文件）/ 治理基线漏 |
| 5 | **客户验证 → 主方法论 反向回写机制缺** | 0 真实日志 / Karpathy "实战驱动 wiki" 未跑起来 |
| 6 | **wiki 双链 lint 未自动化** | 手动维护负担大 / LLM 未承担 maintainer 角色 |

### 1.2 目标状态（v2.0 锚定）

```
┌── 用户体验目标 ──────────────────────────────────┐
│ 1. session 启动 ≤ 30 秒定位"今天到哪里了"        │

│ 4. LLM (Claude) 承担 90% maintenance 负担         │

└──────────────────────────────────────────────────┘
```

## §2. McKinsey + Karpathy + 晟果三力 整合架构

### 2.1 5 层 hierarchy（McKinsey 风格）

```
┌── Layer 0: 战略与治理（Strategy & Governance）█ 内核稳固
│   00_入口与总索引/   ← 重构（详见 §3）
│   16_客户与战役档案/  ← 战役治理（已建）
│   99_归档/           ← 历史封存（已建）
│
├── Layer 1: 方法论核心（晟果独有 / Karpathy main domain）█ 内核灵活
│   01_科学零售方法论/  ← 三力骨架 + 经营控制链 + 13 主文件
│   04_商品诊断与商品力提升/
│   05_品类管理与商品规划/
│   06_陈列规划与DisplayMap/
│   08_经营分析与数据看板/
│
├── Layer 2: 业务作战层（Operations）█ 外延权变
│   02_零售企业生意诊断/
│   03_商圈分析与客群调研/
│   07_营销活动策划与实施/
│   09_门店案例与项目复盘/  ← 实战核心
│
├── Layer 3: 工具与交付（Tools & Delivery）█ 外延复制
│   10_咨询交付模板/  ← SOP + 销售包 + 报告模板
│   11_系统产品与PRD/  ← 待 reposition（Claude Code 学习包已封存）
│   12_自媒体内容与表达转化/
│   13_数据分析与工具脚本/  ← POS 清洗 + SKU 算法（已实战）
│
└── Layer 4: 学习与积累（Learning & Knowledge）█ 外延权变
    14_外部案例与行业研究/
    15_刻意练习与成长/  ← 客户验证日志 / 24h 闭环
    90_素材暂存与待整理/  ← Ingest 缓冲（现已空）
```

> **vault 19 一级目录骨架不动** / 仅按 5 层逻辑分组（mental model）+ 00_/ 内部重构。

### 2.2 晟果独有 4 大支柱（显式标识）

```
1. 三力骨架（商品力 + 组织力 + 运营力）
   主居 01_ / 散居 02-08_
   ↓
2. 经营控制链（金额 + 数量 + 品质）
   主居 01_ / 散居 02-08_ + 13_数据
   ↓
3. 战役治理双轨（战役档案 + 中层方法论 M-DEC）
   主居 16_ + 09_ + 04_中层决策库
   ↓
4. 客户验证 24h 闭环（§21）
   主居 15_客户验证日志 / 反向回写主方法论
```

### 2.3 Karpathy LLM wiki 4 个核心机制

```
1. Main entry page（晟果科学零售 wiki 主入口）
   00_入口/_wiki入口/科学零售wiki入口.md ← NEW
   - LLM 主索引 / 链入全 vault 关键节点
   - Karpathy 风格"个人知识图谱主页"

2. Topical deep dives（主方法论 13 文件）
   01_/各文件 + §"实战登记"段
   - Karpathy "personal note" 风格
   - 实战产物反向回写机制

3. Index pages（跨主题导航）
   00_入口/02_方法论索引/
   - 知识库总索引 + 核心概念主定义 + 标签体系

4. LLM-friendly format
   - frontmatter 12 字段标准（CLAUDE.md §10）
   - quadrant 强制（§19 / 当前缺 35%）
   - 双链密集（每文件 ≥ 5 `wiki-link`）
   - 反幻觉规则一致
```
