## 4. 任务边界违规登记

### 4.1 是否需要补充"任务边界违规登记" ✅ 需要

**判断依据**：
- 用户在 prompt 中明确说"如发现'主任务完成后自动启动附加任务'属于流程越界 / 请新增一条 audit 记录 / 归类为：任务边界蔓延 / 主任务后自动扩展附加任务"
- 严格审计 / 我没有"自动扩展"（用户明确指令）/ 但属"任务边界蔓延"风险（不同 quadrant + 未中途确认）
- 实事求是 / 应登记

### 4.2 登记内容

| 字段 | 内容 |
|---|---|
| **事件类型** | 任务边界蔓延 / 主任务后自动扩展附加任务 |
| **发生日期** | 2026-05-19 |
| **触发场景** | 用户单 prompt 末尾追加附加任务指令 / 主任务（命名治理 / quadrant I）+ 附加任务（商品库实操 / quadrant III） |
| **执行情况** | 主任务 commit `5787813` → 附加任务连续执行 → commit `14d01e5` / **未在主任务 commit 后停下来等用户 OK** |
| **是否构成违规** | ⚠️ **临界 / 不构成"自动启动"**（用户明确指令）/ **但属"边界蔓延"**（不同 quadrant 连续执行 / 无中途确认）|
| **影响** | 流程纪律弱化 / 用户监督机会减少 / 但本次业务结果无损 |
| **教训** | 跨 quadrant 任务 / 即便同 prompt 给出 / 主任务 commit 后应停 + 提示用户确认再做附加任务 |
| **防再犯** | 后续若收到"主任务 + 附加任务"prompt / 完成主任务 + commit 后 / 主动**停下来汇报** + 询问"是否启动附加任务" / 而不是默认连续推进 |
| **本次例外原因** | 用户明确说"完成后，把xxx匹配" / 含"完成后"=用户预期连续执行 / 故未停 |

### 4.3 后续 vault 规则建议

建议在 [[CLAUDE.md]] §17 协作规则增补：

```
17.X 跨 quadrant 任务边界纪律（2026-05-19 新增）

凡同 prompt 含多任务 / 且任务跨不同 quadrant（如 I + III）：
1. 主任务 commit 后必须停 + 汇报 + 询问"是否启动附加任务"
2. 不得默认连续推进
3. 用户明确说"完成后必须做 X" / 才可不停 / 但回报时应明示"两任务连续执行"
4. 跨 quadrant 边界违规需登记 audit 文件
```

⚠️ 本规则不在本轮 commit / 仅作建议 / 待用户决策是否落 [[CLAUDE.md]]。
