> "部分文件名、frontmatter、内容中的日期，是由于时间基准错误生成 / 不能作为真实业务日期。"

- ❌ "提前生成"

## 6. 后续时间门禁规则（强制）

### 6.1 触发词

凡任务中出现以下词语 / Claude **必须先执行时间基准检查**：

| 类别 | 触发词 |
|---|---|
| 相对时间 | 今天 / 明天 / 后天 / 昨天 / 最近 / 当前 |
| 周内时间 | 本周 / 下周 / 周一 / 周二 / 周三 / 周四 / 周五 / 周六 / 周日 |
| 月内时间 | 月初 / 月中 / 月底 / X 月 X 日前 |
| vault 编号 | Wxx-Dayx / vault 内部 phase 编号 |
| 模糊时间 | 之前 / 之后 / 不久 / 即将 |

### 6.2 时间基准检查 5 步

| 步 | 动作 |
|:---:|---|
| 1 | **确认真实当前日期**（`date` 命令 + CLAUDE.md `currentDate` 双源核对）|
| 2 | **将所有相对时间转换为绝对日期**（不带星期 / 带星期 / 都列）|
| 3 | **区分真实业务日期 vs vault 文件日期**（错时文件不替代真实日期）|
| 4 | **输出时间映射表**（用户原话 → 真实日期 → 业务事项）|
| 5 | **用户没有要求写文件时不得直接写计划** / **未完成时间映射不得 commit** |

### 6.3 硬约束

- ❌ 不根据 vault 文件名推算今天
- ❌ 不根据 W21-Day 编号推算今天
- ❌ 不把文件创建日期当业务执行日期
- ❌ 不把未来文件名当已发生
- ❌ 不在未核对真实日期前生成任何时间表
- ❌ 不在未核对真实日期前生成验收文件
- ❌ 不按对话上下文推断今天 / 必须显式核对

## 10. v0.2 小订正：下周一 / 下周二日期映射修正（2026-05-19 EOD 追加）

### 10.1 错误说明

上一轮（v0.1）已修正"今天 = 2026-05-19"主要错误 / 但最终一句仍把"下周一 / 下周二"错按 **+2 / +3 天顺延** 推算：

| 错（v0.1 最终一句）| 对（v0.2 按真实日历）|
|---|---|
| 下周一 = 5/21 周四 | **下周一 = 5/25 周一** |
| 下周二 = 5/22 周五 | **下周二 = 5/26 周二** |

### 10.2 错误根因

- 5/19 周二 + 2 天 = 5/21 周四 ❌（不是下周一 / 是后天）
- 5/19 周二 + 6 天 = 5/25 周一 ✅（这才是下周一）
