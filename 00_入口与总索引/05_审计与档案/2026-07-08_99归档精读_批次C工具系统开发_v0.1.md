## ③ ⭐回填候选（现在还没做、有价值的设计思路）

1. **⭐⭐⭐ 空间陈列系统三模块（棚割/planogram 引擎设计）** — `3.3.2/3_空间陈列系统/`
   - 模块 2 SQL DDL：商品物理尺寸主数据（宽/高/深 mm、包装形态、陈列限制、堆叠上限）+ 道具母体表（货架类型/内径/孔距）+ 组件表（层板/挂钩/筐）；模块 3：引擎配置字典 JSON（物理吸附/排面规则）；模块 4 PRD：拖拽入架、排面拉伸复制、**三种导出物——实操作战图 PNG / BOM 级订货清单 Excel（货架-层板-条码-面宽-纵深-堆叠-理论容量）/ A4 混合指导书 PDF（上半图供执行、下半表供打价签核对）**。
   - **活跃库无等效物**：`read-floorplan` skill 只做平面图读取与空间建图（门店级），`KB-SPACE-ARCH-001` 是坪效架构（draft），均不到**货架/层板/排面级**。这是全库唯一一套货架级陈列引擎设计。花厅坊陈列调改交付物（尤其"上图下表 A4 指导书"格式）可直接借用其输出物定义，不必开发系统。
2. **⭐⭐ 归类引擎"三层资产体系 + 冲突词表 + 学习闭环"** — `全品类归类引擎_资料准备与落地总控方案_v1.md`
   - 标准层/判断层/学习层三层资产划分，及 8 份资料模板的字段级口径：品牌词典、别名词典、规格单位词典、**冲突词表（口味词"蜂蜜味/牛奶味"、活动词"赠品/促销装"、包装词"家庭装/补充装"的误判排除）**、末级品类样例商品表、未识别词回收表、错配修正记录表。
   - 现行品类匹配算法包有历史映射层（M01），但**冲突词表和"人工修正→回灌样本→提炼词条→调规则"的显式学习闭环资产**尚未见等效物。posclean/品类匹配下一次迭代时值得参照（候选，不建议单独立项）。
3. **⭐⭐ SKU 评分 9 维中的补充维度** — `3.3.5/SKU评分规则表.md`
   - 现行 ABCZ/7D 之外，其"价格带价值(10)/规格带价值(10)/连带潜力(10)/供应稳定性(5)/门店适配度(5)"五个结构性维度，以及"引流款放宽毛利但销售必须过关、季节品按窗口单看"的修正规则，可作 7D 打分维度校准的参照材料。已挂 MVP 历史索引待裁决项下，不另立动作。
4. **⭐（史料）Skill 封装方法学早熟证据** — 3.3.1 的封装方案在 2026-04 已提出"输入契约/固定输出 sheet/guardrails/低置信强制复核"完整 skill 规范，与后来 Claude Code skill 铁律高度同构。可作 skill 体系演化史的一条 source 引注，无需回填正文。
