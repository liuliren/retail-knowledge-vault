## § 3 · 图谱逻辑说明（为什么这样连接）

### 3.1 核心图谱结构

```
KB-METHOD-CHAIN-001（总公式·枢纽）
    ↓ 诊断顺序
KB-DX-FRAMEWORK-001（评分框架）
    ↓ 五模块
    ├── KB-FRESH-001（M1生鲜·30分）
    ├── KB-CAT-ROLE-5WAY-001（M2品类·25分）
    │       ↓
    │   KB-CAT-ARCH-001（品类架构权威）
    │   KB-CAT-ROLE-GBA-001（大湾区特化）
    ├── KB-CAT-ZONE-SOP-001（M3陈列·20分）
    │       ↓
    │   KB-CAT-CDT-001（子格落位）
    │   KB-CAT-DX-MISMATCH-001（错配诊断）
    │   KB-CAT-TIMEEFF-001（时间效率评价）
    ├── KB-DX-MATH-001（M4数据·15分）
    └── KB-DX-QUESTION-001（78题题库）

选品工具链：
KB-SHOPPER-TASK-001（顾客需求）→ KB-SELECTION-TOOLS-001（工具）→ KB-DX-WIDTH-001（宽度诊断）→ KB-SKU-ROLE-001（SKU角色）

大湾区本地化：
KB-GBA-TPOS-001 + KB-CAT-ROLE-GBA-001 → 修正所有诊断工具的本地参数
```

### 3.2 图谱密度规律

- **stable 页** = 必然是图谱枢纽，多条入链；是调用起点
- **candidate 页** = 次级工具，通常被 1–2 个 stable 页引用
- **KB-RETAIL-\* 页** = 外部研究叶子节点，连向 stable 核心页（单向）
- **孤立节点** = 尚未与图谱连接的页，lint 时会被标记

### 3.3 如何用 Obsidian 图谱辅助学习

推荐过滤设置（在 Graph view 过滤框输入）：
```
-path:"99_归档" -path:"98_AI协作中枢" -path:"99_原始素材" -path:"零售老刘体系"
```
剩下的就是活跃知识网络。`stable` 节点颜色最深，是图谱的中心。
