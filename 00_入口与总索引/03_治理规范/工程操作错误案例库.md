## 案例 #001：cd /tmp 切走 Bash cwd / git 致命错误（2026-05-09）⭐

### A. 触发场景

5/9 W19 数据审计期间 / 需要用 Python 读 /tmp/01_门店数据材料_xlsx/ 真实数据 / 跑 openpyxl 解析。

### B. 错误做法

```bash
# Bash 第 N 次调用
cd /tmp && python3 << 'EOF'
import openpyxl
wb = openpyxl.load_workbook("/tmp/01_门店数据材料_xlsx/滞销商品.xlsx")
...
EOF

# Bash 第 N+1 次调用（仍在 /tmp / 因为 cwd 持久化）
cd /tmp && python3 << 'EOF'
...
EOF

# Bash 第 N+2 次调用（执行 git commit）
git add -A && git commit -m "..."
# ⚠️ "致命错误：不是 Git 仓库（或者任何父目录）：.git"
```

### C. 错误根因

**Bash 工具的工作目录在命令之间持久化**。一次 `cd /tmp` 影响所有后续 Bash 命令 / 直到下一次 `cd` 切回。

「细节差异叠加 = 系统性错误」：
- cd 是无意识的"切目录"动作
- /tmp 不是 git 仓库
- git add -A 在非 git 仓库中报致命错误
- 但**只是 cwd 漂移** / 并非真实 git 仓库损坏

### D. 修复路径

```bash
# 立即修复
cd /Users/davidliu/KnowledgeBase/retail-knowledge-vault && git add -A && git commit -m "..."

# 长期修复（写入 CLAUDE.md §17.11）
正例 1：Python 脚本内 os.chdir()
正例 2：直接用绝对路径
正例 3：git -C <repo>
```

### E. 教训 + vault 沉淀

| 教训 | vault 沉淀 |
|---|---|
| **Bash cwd 持久化是默认行为**（不是 bug）/ 必须当作硬约束防御 | [[CLAUDE.md]] §17.11 / [[工程操作纪律手册]] §3 |
| **可疑命令前必须 `pwd` 验证**（git / Read / Write）| [[pre-flight_check_checklist_v1.0]] §1.1 |
| **cd 是有副作用的命令** / 必须显式管理或避免 | [[工程操作纪律手册]] §3.1 5 条硬约束 |
| **"致命错误"不等于"严重事故"**：本次只是 cwd 漂移 / git 仓库本身完好 | feedback memory 沉淀避免恐慌 |
| **同类错误绝不再犯**（Tesla First Principles）| 案例 #001 锁定 + W22 月度审计验证 |
