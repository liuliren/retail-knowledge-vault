# 文件命名规范

> 📌 **当前主规则声明（P1-GOV-Naming 2026-06-22 六哥批准）**：自 2026-06-22 起，本文件作为晟果新零售科学零售知识库的**文件命名主规则（Naming SSOT）**。
> [[2026-05-13_命名规范总纲_v0.1|命名规范总纲]]、[[2026-05-13_命名规范索引_v0.1|命名规范索引]]、[[2026-05-13_命名规范冲突清单_v0.1|命名规范冲突清单]] 中的命名说明，仅作**历史依据 / 导航辅助 / 冲突案例材料**使用；**如与本文件冲突，以本文件为准**。
> 后续如需合并旧命名规则，须另开专项（P1-GOV-Naming-Integrate）：先盘点、再预览、再确认、再最小写入；**合并前不改名、不删旧文件**。

本文件定义知识库的文件命名规则，确保文件名稳定、清晰、可检索。

---

## 一、基本原则

1. **以中文为主**，避免英文缩写（除行业通用词如 SKU、PRD、SOP、DisplayMap）。
2. **文件名要能自说明**，不看内容就能知道它是什么。
3. **文件名要稳定**，一旦建立尽量不改（改名会破坏 Obsidian 双链）。
4. **禁止使用特殊字符**：`/ \ : * ? " < > |` 不可用于文件名。
5. **避免过长**：文件名建议不超过20个汉字（约40个字符）。

---

## 二、版本号规则

版本号加在文件名末尾，用下划线分隔：

| 版本号 | 含义 | 示例 |
|--------|------|------|
| 无版本号 | 当前唯一活跃版本（推荐） | `商品诊断标准流程.md` |
| `_v1.0` | 正式版存档时使用 | `商品诊断标准流程_v1.0.md` |
| `_v0.1` | 草稿或可讨论版存档 | `陈列规划框架_v0.1.md` |

**注意：** 活跃使用中的文件不需要在文件名中加版本号，版本信息写在 frontmatter 的 `version` 字段中。归档时才在文件名加版本号。

### 2.1 版本号语义（v1.2 新增 / 三级）

| 类型 | 格式 | 含义 | 示例 |
|---|---|---|---|
| **major** | v0.x → v0.y / v0 → v1 | 结构性 / 重大内容增加 | 读图 v0.1 → v0.2（5/6 增 §6 堆头 86 张诊断）|
| **minor** | v0.x → v0.x.5 | 中等修订（罕用）| — |
| **point** | v0.x → v0.x.y | 小补丁 / 增补段 | G4-A 首战手册 v0.1 → v0.1.1（增 §A.3.9 时间效率基线）|

### 2.2 升版与文件名同步约定（v1.2 新增）

带版本后缀的文件（如 `_v0.1.md`），version 字段升版后**是否同步重命名文件**取决于升版类型：

| 升版类型 | 文件名处理 | 实战做法 |
|---|---|---|
| **major bumps**（v0.1 → v0.2）| **改文件名** | 创建新文件 `_v0.2.md`：① 保留旧 `_v0.1.md` 作历史版（如 [[2026-05-06_门店陈列照片命名规范_v0.2]] / `_v0.3` 共存）；或 ② 就地 mv（如 5/6 读图 v0.1.md → v0.2.md，无需保留旧版的活文档场景）— 视是否需保留旧版而定 |
| **point bumps**（v0.1 → v0.1.1）| **就地编辑保留 filename** | frontmatter version 升 v0.1 → v0.1.1 + §版本记录 加 v0.1.1 行 + 文件名 `_v0.1.md` 不动（如 G4-A 首战作战手册）|

**反幻觉一致性原则**：
- frontmatter `version` 字段 = 正文 §版本记录最后一行的版本号（必须同步）
- 文件名后缀 = 按上表约定（major 改名 / point 不改）
- 三者**任何不一致 = §14 逻辑一致性硬冲突 / G03 必扫**（参 [[文档工程化标准]] v1.2 §一 version 字段升版语义）

## 三、不同类型文件的命名约定

### 方法论文件
- 格式：`[方法论名称].md`
- 示例：`科学零售总纲.md`、`品类管理.md`、`52周MD.md`

### 模板文件
- 格式：`[内容描述]模板.md` 或 `[内容描述]SOP.md`
- 示例：`门店诊断报告模板.md`、`堆头活动执行SOP.md`

### 规则文件
- 格式：`[内容描述]规则.md` 或 `[内容描述]清单.md`
- 示例：`滞销品淘汰规则.md`、`反幻觉检查清单.md`

### 案例/项目文件
- 格式：`[门店名]_[阶段].md`
- 示例：`乐易购花厅坊店_商圈调研报告.md`

### 门店项目总览文件
- 格式：`[门店名]_项目总览.md`
- 示例：`乐易购花厅坊店_项目总览.md`
- **说明：** 禁止使用通用名 `项目总览.md`。多个门店项目目录下若都存在同名文件，Obsidian 双链会产生解析歧义（无法确定指向哪个）。门店名前缀是必须的。

### 分析报告
- 格式：`[门店名]_[类型]_[年月].md`（可选年月）
- 示例：`乐易购花厅坊店_经营月报_202605.md`

### PRD文件
- 格式：`[系统名]PRD_v[版本].md`
- 示例：`商品诊断系统PRD_v0.1.md`

### DisplayMap
- 格式：`[门店名]_DisplayMap_[区域]_v[版本]`
- 示例：`花厅坊店_DisplayMap_冲调饮品区_v1.0`

## 四、目录命名规则

- 一级目录：使用已定义的 `NN_中文名称` 格式，不得随意新增。
- 门店项目目录：使用门店名称（简短可识别）。
- 项目子目录：使用 `NN_阶段名称` 格式，如 `05_陈列整改`。

---

## 五、禁止事项

- 禁止用日期开头命名文件，**除非满足以下"按时间纵向追踪"场景之一**（v1.2 新增例外列表 / 治理债清理）：
  - 工作日志 / 日记类（如 `2026-05-06_六哥日记.md`）
  - 客户验证日志（参 [[15_刻意练习与成长/客户验证日志]]）
  - 现场记录 / 二访读图 / 复盘类（如 `2026-05-06_二访_读图分析_v0.2.md`）
  - 战役治理工作档案（如 `2026-05-11_G03首扫_预置数据_v0.1.md`）
  - 战役阶段输出 / 大型作战方案（如 `2026-05-04_知识库重构方案_v0.2_完整作战方案.md`）
- 禁止用"临时""草稿""test""new"等词命名正式文件。
- 禁止一个文件名对应多个不同内容（每个文件只做一件事）。
- 禁止在文件名中加入"_最终版""_终稿"——这类文件大概率会再修改。
