# 花厅坊90天 金矿5桶分层抽样说明 v0.1

> 据 Review-003 五桶(六哥临时认可·**待样本验证·非方法论规则**)分层抽样,供人工逐桶判。**明细表 gitignored 不入 git;本说明无逐 SKU/真实条码/进价/供应商裸名。** 不出 B/D 正式明细、不进 Execute-Approval、不写回、不改 §3.1.x。

## 1. 抽样目的
验证 Review-003 的 5 桶临时口径是否贴合现场:**每桶抽样让六哥逐桶判**「这桶的处置动作(保留/控补货/缩面/清库/复核)对不对」,而非直接对 B/D 出正式裁决。

|---|---|--:|--:|

## 3. 抽样字段(明细表·gitignored)
处置桶 / 成本复核横切 / **sku_display_id(脱敏 hash)** / 品名 / 类别名称 / 身份 / 毛利率 / gross_margin_rate_tier / 销量 / 销售额 / 库存数量 / 库存成本金额 / 库龄天数 / old_inventory_risk / goldmine_reason / **人工判断(空) / 建议动作(空) / 人工备注(空)**。
> 货号/进价/真实条码**不在**明细(仅 hash)。

## 4. 人工复核方法（逐桶判·每桶看几条即可估口径对不对）
| 桶 | 看什么 | 判口径对不对 |
|---|---|---|
| A 保留 | 新货是否真值得保留/升核心 | 8 条 |
| B 控补货 | 老但卖得动(销量≥12)是否真该「小批勤补」而非清 | 15 条(重点)|
| C 缩面 | 高库存金额是否真该缩排面降水位 | 12 条 |
| D 清库 | 一年以上非快销是否真该促销清库 | 16 条(重点)|
| E 人工复核 | 难归类的现场怎么定 | 7 条 |
| 成本复核横切 | 毛利率>0.85 是否成本仍不准 | 4 条 |

**人工判断选项**:口径对(桶分准) / 口径偏(该挪桶,注明应入哪桶) / 成本异常 / 需现场看 / 待定。
**建议动作选项**:保留观察 / 控补货(小批勤补)/ 缩面降水位 / 促销清库 / 成本复查 / 现场复核。

## 5. 复核后如何用
- 若各桶口径**大体准**(如 B/D 抽样 80% 判「口径对」)→ 桶口径可作为**该客户当前临时分层**,再议是否对 B/D 出完整明细(另轮,需授权);
- 若**口径偏**(大量该挪桶)→ 回 Review/调桶阈值,不急出明细;
- 成本复核 4 条若确认成本不准 → 该批回 cost_unreliable,不算金矿。
> **本轮只抽样验口径,不固化为方法论规则,不出正式 SKU 裁决,不写回。**
