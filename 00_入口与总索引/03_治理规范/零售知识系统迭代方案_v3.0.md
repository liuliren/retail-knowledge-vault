> v3.0 取代 v2.0 为当前策略层;规则细节仍见《晟果新零售M-DEC长期记忆系统Schema v0.1.1》。

## 0. v2.0 → v3.0 做了什么

不是再调设计,而是**给设计找证据、据证据微调**:
1. 用三条独立证据柱(AI 工程 / 决策科学 / 笔记纪律)交叉验证我们的核心选择;
2. 据证据做 2 处微调:① `failed` 的「resulting」再分类(重要);② 前瞻把握度字段(可选,留待 v0.2);
3. 把"最简"从口号变成制度(技术债哨兵)。

## 1. 反复论证:三条证据柱

### 柱一 · AI 工程(mem0 / Zep / Letta,2026)
- **时序是公认最难的问题**:2026 记忆领域三大未解问题是跨会话身份、时序抽象、记忆陈旧;Mem0 新算法最大增益来自时序(+29.6)与多跳(+23.1)。→ **"事实随时间改变"是全领域最难点,我们把 supersession 放中心,押对了。**
- **最佳时序方案是显式取代,不是衰减**:Zep/Graphiti 用双时序知识图谱,LongMemEval 63.8% vs Mem0 49.0%。→ 验证 supersession-not-decay。
- **厂商自评注水**:Mem0 自报 94.4%,独立评测仅 49.0%。→ **别信 benchmark,要在自己数据上验** = 0B 的理由。
- **陈旧仍是未解开放问题**。→ 人工闸门 + supersession 是合理答案,不是落后。

### 柱二 · 决策科学(Annie Duke《Thinking in Bets》)
- **决策日志对抗后见之明偏差**:结果揭晓前写下推理,是把"好决策"与"好运气"分开的唯一办法;无日志则反馈被系统性扭曲。→ **M-DEC 的「推理」必须在拍板当下写,不能事后补。**
- **resulting**:别把决策质量=结果质量。好决策可能坏结果(运气),坏决策可能好结果(侥幸)。→ 逼出微调一(§3)。
- **校准需要足够具体的反馈**才能更新信念。→ 回填闭环的意义。
- 前瞻概率("我 62% 有把握")让人更易在事后承认判断错、抑制后见之明。→ 微调二(§3,可选)。

### 柱三 · 笔记纪律(Zettelkasten / 常青笔记)
- **收藏家谬误**:藏 ≠ 懂;收集令人自我感觉良好但不处理就不增知;短循环胜长循环。→ 重申:**杀掉条数指标、只求处理、周级短循环**。
- **反向收藏家谬误**:让"现在的你"替"未来的你"决定丢什么,本身是谬误;想法被取代时不删除,而建带双链的后续笔记看思考演变。→ **从笔记学独立验证 supersession + failed 保留**。
- **过度工程的最硬警告**:常青笔记提出者 Matuschak 因技术债太重而弃用该实践,退回轻量项目大纲。→ **发明人都退回轻量,我们守"最简正确版"是对的。**

## 2. 三柱合流:我们押对了什么 / 该改什么

| 设计选择 | 柱一·AI 工程 | 柱二·决策科学 | 柱三·笔记纪律 | 结论 |
|---|---|---|---|---|
| supersession 取代 decay | Zep 双时序=显式取代 | resulting:旧判断当时可能对 | 反向收藏家谬误:不删 | ✅ 三柱齐验 |
| provenance 不打分 | 厂商分数注水 | 推理链 > 数字 | 写给未来自己、可核 | ✅ 验证 |
| 人工闸门 / 拒自动富集 | 陈旧仍未解,自动写会污染 | 决策质量靠过程不靠结果 | 处理而非收集 | ✅ 验证 |
| 杀硬指标、只求不断流 | — | 校准要真反馈非凑数 | 收藏家谬误 | ✅ 验证 |
| 最简、机器后置 | 复杂度=未解风险 | — | Matuschak 退回轻量 | ✅ 强验证 |
| failed 一刀切 | — | **resulting:需再分类** | — | ⚠️ 微调(§3) |

## 3. 据证据的两处微调

### 微调一(重要):`failed` 的「resulting」再分类 → 同步 Schema §7

回填结果时,**把决策质量与结果质量拆成两轴**,落四象限:

| | 结果好 | 结果差 |
|---|---|---|
| **决策对** | `active` 保留,可作**晋升正例** | `果差但决策稳`:保留,**不削弱规则**,记外因(供应商/季节/竞争) |
| **决策错** | `侥幸`:保留,**严禁作晋升正例**(危险假阳性) | `failed`:削弱/推翻规则,**最值钱教训** |

铁律:**只有「决策错」才动规则;外因导致的坏结果不动规则。** 这防止你把被外部冲击拖垮的好判断,错当坏规则丢掉(Duke 的 resulting)。

状态字段相应扩展:`active / superseded / failed / 果差但决策稳 / 侥幸 / archived`,全部保留不删。

### 微调二(可选,留待 v0.2,0B 先不加):前瞻把握度字段

拍板当下记一个 `把握度: 高|中|低`(或 %),回填时对照,长期积累成你的**校准曲线**(哪类判断你常过度自信)。
- 明确:这是**前瞻自我校准**,不是给事实打 confidence score(后者仍禁止)。两者方向相反:一个是决策前对自己,一个是事后对知识。
- 0B 先不加——先证明基础模板跑得通,再考虑;不然违反最简。

## 4. 把"最简"制度化:技术债哨兵

Matuschak 的教训不能只当口号。设一条月度自检(并入 G03_Lint 或周复盘):
- 知识系统是否开始变重(字段/规则/流程在膨胀)?
- 它是否在偷花厅坊主线的注意力?
- 维护成本是否开始 > 它产生的复用价值?

**任一为是 → 退回更轻的形态**(砍字段、降频率、回到纯 inbox+周回填)。
连常青笔记发明人都退回轻量大纲——我们没有理由比他更重。

## 5. 路线与现状(承 v2.0,压缩)

- **原则**:最简正确版;provenance 不打分 / supersession 不衰减 / signoff 人工闸门 / 命名边零 LLM / markdown+git 真相源。
- **路线**:Phase 0(立规则+试填)→ Phase 1(挂靠花厅坊跑生命周期,软目标 3–5 条/≥1 回填/不强求晋升)→ Phase 2(触发条件:第二客户+跨店复用)→ Phase 3(页数逼近 100–200 才上检索/自动化)。GBrain = Phase 3 天花板参照,非现做。
- **现状**:Phase 0A 完成(Schema v0.1.1 落盘),现处 **0B**。
