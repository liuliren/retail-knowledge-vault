# M-DEC-013 现场补验清单（Phase 0B）+ trial 审阅 v0.1

> 审阅 [[M-DEC-013长期记忆Schema试跑记录_v0.1]] 是否符合 Schema v0.1.2，并把下一次花厅坊现场要抓什么写成一线可执行清单。**只读 M-DEC-013、不改其正文/status、不升 active、不改 Schema active。**

## B. 为什么 M-DEC-013 不能升 active
| 阻塞项 | 当前证据 | 为什么阻塞 | 解除条件 |
|---|---|---|---|
| 同店三小类存在共因 | 饼干/膨化/果干均花厅坊同店、同期、同口径 | 同店=同供应商生态/同客群/同季节窗口,非独立 | 第二门店/第二客户独立复现 |
| 未达 3 个独立场景 | 实为 1 店 × 3 小类 | §7 门槛=「≥3 独立场景 + 排除共因」 | 跨店各 1 即可累计独立场景 |
| 缺第二客户验证 | 仅花厅坊单客户 | 单客户=单一经营风格,规律未必普适 | 第二客户实跑 |
| 捕获路径未验证 | M-DEC-013 是成型决策非速记 | 捕获=现场 5 秒→30 秒落 inbox,未走过 | 见 C 现场补验 |
| 未形成 2-4 周回填结果 | 瘦身临界 pending | 无回填=只是记录不是记忆(§8)| 现场动作后 2-4 周回填 resulting |
| 未有 active signoff | 无 | active 须六哥签字(§3②)| 上述全绿后六哥签 |

> **铁律重申**：不为推进而强行 active。failed/pending/blocked 都是资产——M-DEC-013 现在的 candidate+共因暴露，本身就是有价值的留痕，不是缺陷。

## C. 现场补验清单（下一次花厅坊现场抓什么）
### C1. 现场要捕获的内容
| 捕获项 | 现场问题 | 记录方式 | 必填 |
|---|---|---|:--:|
| 现场观察 | 这个小类/货架我看到什么异常? | 一句话或拍照 | ✅ |
| 触发问题 | 它触发哪个 IssueTag(长尾过宽/价格带过密/陈列错位/缺货误判/负毛利清仓)? | 选 1 个 code | ✅ |
| 初始判断 | 我当下怎么判? | 一句话(拍板当下写,不事后补)| ✅ |
| 决策动作 | 我准备做什么(瘦身/替换/不动/观察)? | 一句话 | ✅ |
| 反证因素 | 我担心的误判是什么(缺货/陈列/低频刚需/节令)? | 一句话 | ✅ |
| 店员/店长反馈 | 一线怎么说? | 引述一句 | ◑ |
| 后续观察点 | 2-4 周后看什么指标判对错? | 指标名 | ✅ |
| 回填时间 | 哪天回填? | 日期 | ✅ |
| 是否新 M-DEC 候选 | 这是新规律还是已有决策的新证据? | 新候选/并入 M-DEC-013 | ✅ |
| 是否只是 working | 还不够成决策? | working/episodic | ✅ |

### C2. 30 秒口述模板（现场照着说）
```text
我现在看到的是：____。
它触发的问题是：____（长尾过宽/价格带过密/陈列错位/缺货误判/负毛利清仓）。
我的初始判断是：____。
我准备采取的动作是：____。
我担心的误判是：____。
需要 2-4 周后回填的是：____。
这条先记为 working / episodic，不直接升规则。
```

### C3. 捕获纪律
- **现场 5 秒**口述/拍照 → **30 秒**落 inbox(当天,不求格式)→ **周末 triage**三出口(升决策/并入 M-DEC-013/丢弃记原因);
- 拍板当下写「初始判断+担心的误判」——这是 Duke 决策日志的核心,**事后补会被后见之明污染**;
- 先 working,**不当场升规则**。

## D. M-DEC-013 后续验证路径
```
现状: episodic, candidate, 同店3小类(共因)
  │
  ├─ 路径1(捕获补验): 花厅坊新现场决策走 C 清单 → 验捕获路径顺不顺 → 回答 0C 三问
  │
  ├─ 路径2(独立场景): 第二客户/第二门店复现「C+D≈40%产≈10%」
  │     └─ 跨店各1 + 排除共因 → 达「≥3 独立场景」→ 可升 semantic
  │
  ├─ 路径3(回填): 现场动作后 2-4 周回填 resulting 四象限
  │     ├─ 决策对+果好 → active 正例候选
  │     ├─ 决策对+果差 → 果差但决策稳(记外因,不削规则)
  │     ├─ 决策错+果好 → 侥幸(不作正例)
  │     └─ 决策错+果差 → failed(削弱/推翻判据)
  │
  └─ 路径4(签字): 上述全绿 → 六哥 signoff → 升 active 并入 [[M2_选品配置簇]]
```
> **晋升方式**：升 semantic/active 时**上层新建规律页 `晋升为::` 回指 M-DEC-013，不改写 M-DEC-013 原文**。

## E. 对 Schema 的反哺建议（只建议·不改 active）
1. **建议**：Schema §9 模板可显式加 `memory_level` 与 `decision_status` 两轴字段示例（M-DEC-013 原文未显式标，本 trial 靠推断）——留待 Schema v0.2，本轮不改。
2. **建议**：捕获路径需「现场速记」类样本验证，回溯型决策（如 M-DEC-013）只能验晋升门槛，不能验捕获——0C「捕获顺不顺」必须用新现场决策回答。
3. **不改 Schema active 状态**（仍 candidate）。
