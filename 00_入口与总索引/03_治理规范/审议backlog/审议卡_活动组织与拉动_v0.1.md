# 审议卡 · 活动组织与拉动（KB-EVENT-ACTIVATION-001）

- **提案一句话**：营销活动四类分类 + 7 要素 + ROI 三分流方向。
- **载体**：[[KB-EVENT-ACTIVATION-001_活动组织与拉动_v0.1]]

- **花厅坊相关度**：**低** — 方法论层
- **触发词**：标准 · 原则

## 三轮
- **第 1 轮 · 六哥（用户视角）**：☐ 待 — 三问：四类分类穷尽吗？7 要素够用吗？ROI 三分流方向对不对？
- **第 2 轮 · 批判角色（魔鬼代言人）**：☐ 待
  - 预置批判焦点（原）：① 四类会不会漏掉社区店高频的"到店引流/社群裂变"类？② 7 要素是清单还是可执行模板？③ ROI 三分流的口径能否在花厅坊现有数据下算出？
  - **【深化·三条最锋利反例】**
    - **反例①（分类不穷尽·套不上实战）·漏掉社区店命脉的私域裂变类**：若四类沿用传统促销框架（直接降价/满减/赠品/联合促销），会**漏掉社区生鲜实战最高频的"社群秒杀 / 团长裂变 / 9.9 引流菜到店"**——这类私域玩法恰是社区店的命脉。分类不穷尽，店长遇到实际最常做的活动反而无处归类，框架空转。**失效根因**：四类若从大卖场档期促销抽象而来，天然缺私域这一维。须补第五类"私域裂变/社群"或把四类重构为"引流/清货/利润/私域"。
    - **反例②（成本·规则被架空）·7 要素对高频小活动是过重模板**：7 要素（目标/预算/时间/商品/机制/渠道/复盘）若作"每次活动必填"，社区店**每周特价这类高频小活动根本不会填**——填一次的成本 > 活动本身价值，规则会被店长默默架空，沦为走过场文档。批判四连之成本问：遵守它的成本是否压垮高频场景？须分档——大活动填全 7 要素，高频小活动只填"最小必填 3 项"。
    - **反例③（数据前提不成立·五年后）·ROI 三分流在花厅坊现有数据下算不出**：社区生鲜活动 ROI **难归因**——引流带来的连带购买、次日复购，现有 POS 数据分不清"这单是不是活动拉来的"。三分流方向（引流/清货/利润）对，但缺增量归因数据，落地会退化成"拍脑袋填 ROI 归类"。**五年后**：私域会员数据打通后增量归因才可能成立；本轮宜标"ROI 为粗估、不作硬考核"。
- **第 3 轮 · 专业角色（行业 steelman）**：☐ 待
  - 预置对标（原）：成熟活动管理框架（4P promotion / 活动 ROI 归因）是否有更成熟替代。
  - **【深化·行业成熟对标】**
    - **对标①·成熟锚点=trade promotion 分类 + 增量 lift 归因**：促销管理有成熟分类（EDLP vs Hi-Lo、引流品 loss leader、清货 markdown、利润品），ROI 有成熟的**增量销售法（incremental lift over baseline）**。建议活动**分类锚定成熟的"loss-leader / markdown / margin"三意图 + 私域裂变一类**，而非自造无理论根的四类；ROI 用 baseline 增量法，但标注社区店数据下为粗估。
    - **对标②·最稳形态=意图标签 + 最小必填 + 粗估 ROI**：**进库最稳形态**：① 分类降为"活动意图标签"四类（引流/清货/利润/私域裂变），穷尽且可套实战；② 7 要素分档——大活动填全、高频小活动只填"意图/商品/机制"3 项；③ ROI 用 baseline 增量法但明确"社区店数据下为粗估、不作硬考核"。三条把过重框架降到可落地。
