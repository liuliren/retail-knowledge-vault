# 花厅坊90天金矿候选人工复核抽样说明 v0.1

> 据 review 结论 B 生成金矿候选抽样包,供六哥人工判「真金矿 / 伪金矿」,决定是否上 P85+动销二级闸。
> **明细表在 gitignored 目录,不入 git。本说明只写策略/字段/方法,零逐 SKU 明细、零真实条码/进价/供应商裸名。**

## 1. 抽样目的
判断 P75 单闸筛出的 1,686 金矿候选**是否真有业务价值**,还是仅「每类毛利率 top quartile 的数学影子」→ 为「是否 P85+近 90 天动销二级闸」提供现场证据。

## 2. 抽样来源
- 来源:`processed/花厅坊_90天_dryrun结果_*.xlsx`(gitignored,goldmine_candidate=True 共 1,686);
- 脚本:`tools/goldmine_sample_pack.py`(只读 dry-run 结果,脱敏 SKU 标识 hash)。

## 3. 抽样结果
| 抽样层 | 数量 | 目的 |
|---|--:|---|
| Top 高毛利率 | 20 | 是否真高利润小众品 |
| 高库存金额 | 20 | 高毛利但积压? |

## 5. 人工复核方法
逐条看(或先看 Top 高毛利率 + 重滞两层),按下表填「人工判断」+「建议动作」。重点回答:**这条是真低销高毛利率好货,还是死货/积压/促销假象?**

### 人工判断选项
| 判断 | 含义 |
|---|---|
| 真金矿 | 低销但高毛利率,有保留/优化价值 |
| 伪金矿-死货 | 毛利率高但长期不动销/高库龄 |
| 伪金矿-高库存风险 | 毛利率高但库存金额过高 |
| 伪金矿-促销污染 | 可能受促销/临时价格影响 |
| 伪金矿-数据异常 | 毛利率/成本/销售数据异常 |
| 需现场看陈列 | 数据无法判断,要到货架前看 |
| 待定 | 信息不足 |

### 建议动作选项
保留观察 / 调陈列 / 缩面保留 / 清库存 / 淘汰候选 / 现场复核 / 数据复查。

## 6. 复核后如何决定二级闸
- 若「真金矿」占比高(如 ≥60%)→ P75 单闸尚可,**可不上 P85**(避免误杀小众好货,正是你的顾虑);
- 若「伪金矿-死货/积压/促销」占比高 → 上 **P85 + 近 90 天有动销** 二级闸收窄;
- 若分层差异大(如重滞层多伪金矿、Top 高毛利率层多真金矿)→ 二级闸用**动销/库龄**而非单纯抬 P85。
> 决策走 P0-ABC-Goldmine-Rule-Tighten-001(评审)→ 若改则 CODEX 同步重跑。
