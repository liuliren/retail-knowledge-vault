# 世界级外部基准规范 BENCH-001 v0.1

> **一句话**：本库四维（逻辑/稳固/实用/系统）评分与"世界级"判定，一律锚定以下四个外部基准的成熟度阶梯，**禁止无锚印象分**；任何评分必须引用本文并给出判据证据。评分操作层见 [[2026-07-06_世界级四维评分框架_SCORE-FRAME-001_draft_v0.1]]（5判据×20分）。

## 锚一 · 工程稳固性 ← DORA / CI 成熟度（软件工程）

| 阶 | 外部定义 | 本库映射判据 |
|---|---|---|
| L1 手工 | 无测试无自动化，改动靠人肉验证 | 脚本零测试、lessons 只记台账 |
| L2 可重复 | 有回归测试，改动必跑；无副本漂移 | `13_/tests/` 必过；口径常量单处定义 |
| L3 持续检 | lint/CI 自动执法，红线机械拦截 | G03_Lint 月跑 + pre-commit hook（W31 落地） |
| L4 精英 | 变更前置时间短、失败率低、可回滚 | commit 粒度小+可 revert；仪表盘趋势可查 |

**当前位置：L2（2026-07-06 起·回归测试已建）→ 目标 L3（hook 落地日）**

## 锚二 · 逻辑一致性 ← 法规体系一致性审计（Legal Consistency Audit）

| 阶 | 外部定义 | 本库映射判据 |
|---|---|---|
| L1 | 条款冲突靠事后发现 | 靠审计撞出（GOV-001 vs §6 曾如此） |
| L2 | 有单一权威源与优先级裁决序 | 双宪法裁决序 + SSOT 声明区（已有） |
| L3 | 新规入库前过冲突扫描；签字件出厂过校对 | 标准审议铁律轮2 + **签字≠校对补丁：签字前对照被引文件逐行核**（DEC-03 教训） |
| L4 | 一致性自动化（引用图谱+词表校验） | lint 词表查 + 悬空引用查（部分已有） |

**当前位置：L2.5 → 目标 L3（签字前核对纪律入 checklist）**

## 复评纪律（3 条）

1. **评分必引锚**：任何"打分/评估/世界级"表述，必须引用本文对应锚的阶位 + 证据，否则视为印象分退回。
2. **升阶必有事件**：阶位变化只认可验证事件（如"hook 上线"→稳固 L3），不认可"感觉变好了"。
3. **季度复锚**：每季度问一次——外部基准本身是否更新（DORA 年报/MDM 实践演进），锚过时则修本文（记版本行）。
