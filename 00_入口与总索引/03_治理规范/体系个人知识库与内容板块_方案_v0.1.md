## 四、库间关系与治理（§9 双库正交 → 多库正交）
- **一处沉淀**:方法论只在其归属库写一份,余库 wikilink 指过来,**绝不拷贝**。
- **内容只在主库进料一次**,编译后路由,子库消费蒸馏(不重复爬)。
- **隐私分层贯通**:客户机密→retail `_client_private`;个人 T2-T3→主库 `80_敏感`;都本地+`.no-ingest`+gitignore。
- **全局宪法管红线**,各库 CLAUDE.md 管细则,冲突按更严格执行(已有 [[Claude双宪法边界声明...]])。

## 五、token / 调用纪律（跨库统一）
- 跨库检索仍 **G02 编译层优先**:读各库 MOC+summary,不扫 raw。
- 主库建一张**跨库总索引**(各子库 MOC 的 MOC)→ 我从一个入口找到任意库的编译层。
- 内容板块的本质 = **把洪流挡在默认上下文外,只喂蒸馏**(与 Data Adapter 同构)。

## 六、落地步骤（分阶段·少而深)
1. **P0 收口拓扑**:主库建「跨库总索引 + 体系拓扑图」(本方案毕业到主库 00_)。
2. **P0 内容中枢选址**:主库定 raw 进料目录 + 台账 + `.no-ingest`(零售 Clippings 经验直接复用)。
3. **P1 方法固化**:写**跨库 `/ingest` skill**(进料→摘要→编译→路由→台账),各库通用。
4. **P1 多模态编译**:音视频转录 / 图 caption 的进料时编译流程(工具待定,先文章/网页跑通)。
5. **P2 回流闭环**:已发布内容(公众号/视频)回流为新 source(§8 公开构建)。

## 七、方法固化（你说的"形成一套方法")
**`/ingest` skill(跨库通用)** = 输入问题的最终方法:
```
输入(任意渠道/模态) → 摘要+标签 → 落 raw(冻结) → 编译成概念页(带summary)
  → 路由到对应库 → 台账登记 → (可选)媒体选题
```
一处定义,retail/growth/build/家庭库共用。**这就是"输入问题解决后形成的那套方法"。**
