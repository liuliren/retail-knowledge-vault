# 标签命名空间规范 v0.1

> **问题**(审计 v0.3):全库 1756 个唯一标签,被同时当状态/主题/来源/项目/类型/临时/自动摘录标记用 → 标签不再是稳定分类,而是混合噪声。
> **目标**:收成**7 个受控命名空间**;新标签必须进命名空间;旧裸标签渐进迁移(不一次性 churn)。
> **配套**:[[笔记命名与链接消歧规范_v0.1]](文件名/链接) · 本规范(标签)。

## 一、7 个命名空间(写死)
| 前缀 | 用途 | 例 |
|---|---|---|
| `#status/` | 状态机 | `#status/seed` `/draft` `/active` `/deprecated` |
| `#domain/` | 领域 | `#domain/retail` `/ai` `/growth` |
| `#project/` | 项目/客户(脱敏代号) | `#project/花厅坊` `/乐易购` |
| `#type/` | 内容类型 | `#type/moc` `/concept` `/case` `/template` `/tool` `/spec` `/dec` |
| `#source/` | 来源 | `#source/clippings` `/零售老刘` `/audit` |
| `#workflow/` | 回路 | `#workflow/ingest` `/lint` `/review` |
| `#risk/` | 敏感 | `#risk/client_confidential` `/raw_sensitive` |

## 二、铁律
1. **新标签必须落某命名空间**,不再新增裸标签(如裸 `clippings`/`零售`/`方法论`)。
2. **clippings 标签 = raw 元数据,不进正式知识判断**:进料中枢已 `.no-ingest` + 排除出 Obsidian 图谱/搜索,故 raw 的 `clippings` 裸标签不污染正式标签系统;**正式库内若需标来源,用 `#source/clippings`**(编译后概念页用)。
3. **状态只用 `#status/*` 或 frontmatter `status:`**,二选一不重复(本库以 frontmatter `status:` 为主,标签为辅)。
4. **不一次性重打全库标签**(churn):新页按规范打;旧页在被编辑/lint 时顺手迁移。

## 三、clippings 标签专项处置(M6)
- **现状**:545 老刘 raw + 69 clip raw 带 `tags: [clippings]`,全在进料中枢(已排除图谱)→ **不污染正式标签计数**(审计 1756 含被排除的 raw 标签,实际正式标签远少)。
- **动作**:① 不改 raw 标签(源头冻结);② 正式库新概念页统一用 `#source/clippings`(已在 build 6 页落地);③ lint 增"裸标签检查"——发现正式库裸 `clippings` 标签提示迁移。

## 四、迁移节律
- 渐进:lint 每轮报"未进命名空间的高频裸标签 top20",编辑时顺手迁;不设大规模重打任务。
- 核心 MOC/方法论页优先规范化(它们被引用最多)。
