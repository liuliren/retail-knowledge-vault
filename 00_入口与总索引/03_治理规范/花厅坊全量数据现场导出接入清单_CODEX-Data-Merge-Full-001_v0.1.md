# 花厅坊全量数据现场导出接入清单 v0.1

> 给六哥一份**可执行的现场导出清单**，为 CODEX-Data-Merge-Full-001 备料。**本轮不接真实数据、不处理 xls/csv。** 字段口径对齐 [[全量合并版execute前置方案_CODEX-Data-Merge-Full-001_v0.1]]（26 字段）。

## 1. 必导出文件
| 文件/报表 | 目的 | 必需字段 | 时间范围 | 注意事项 |
|---|---|---|---|---|
| 商品档案 | 主键/角色/品类 | 商品编码、商品名称、条码、品类路径(大/中/小类)、供应商、进价、售价、是否新品 | 当前快照 | 条码导出后脱敏 |
| 销售明细/汇总 | 销额ABC/动销/日均 | 商品编码、销售额、销量、最近销售日期 | **近 90 天**(与现样本一致) | 按 SKU 汇总即可 |
| 库存表 | 安全库存/有货滞销 | 商品编码、库存数量、库存金额、入库日期 | 当前快照 | —— |
| 供应商信息 | 供得上闸 | 商品编码、主供应商、供货状态 | 当前 | 供应商名按需脱敏 |
| 进价/售价 | 毛利校验/价格带 | 商品编码、进价、售价、毛利额、毛利率 | 当前 | 进价=裸值红线,不入 git |
| 促销记录 | 异常高销识别 | 商品编码、是否促销、促销期 | 近 90 天 | 系统有则导 |
| 销售流水/最近销售日期 | 滞销天数 | 商品编码、最近销售日期 | 近 90 天 | 可并入销售明细 |
| 库龄/入库日期 | 库龄分级(T-10) | 商品编码、库龄天数 或 入库日期 | 当前 | **缺则 T-10 blocked** |
| 缺货/断货记录 | 缺货假滞销 | 商品编码、是否缺货 | 近 90 天 | 系统有则导 |
| 清仓/特价/负毛利 | 异常分支 | 商品编码、是否清仓、负毛利标记 | 当前 | 系统有则导 |

> ITO/库存周转(IR 公式必需)：若系统有周转报表一并导；**缺则 T-07 IR blocked**，不硬算。

## 2. 脱敏要求（红线）
1. **真实 EAN-13 不入 git**；条码展示只允许 `{{EAN13_已脱敏}}` 或前 3 后 2（如 `690****12`）；
2. 导出文件**先本地留存**（99_原始素材 本地区），**不进 git**（raw_sensitive，§6 D 层）；
3. **进价裸值 = 红线**，不入正文、不入 git，只做结构化汇总；
4. 客户名 / 供应商敏感字段按需脱敏（代号化）。

## 3. 最低可跑条件
- **最低必需**：商品编码、商品名称、销售额、销量、毛利额或毛利率、库存、供应商 → 缺一不可跑基础裁决；
- 缺库龄/ITO → T-10/T-07 模块 **blocked**，不硬算（脚本已支持 blocked 分支）；
- 缺进价 → 供应商/毛利判断**降级**（毛利转推断）；
- 缺最近销售日期 → 滞销判断**降级**。

## 4. 现场执行说明（一线指令）
> **「到系统里把上面这些表导出来，不要改名、不要手动整理、不要删行，原样发给数据处理流程。导出的真实文件先放本地，不要传 git。条码和进价我们会先脱敏再用。缺哪张表就说一声，对应模块会自动标 blocked，不影响其它部分先跑。」**

## 5. 与 execute 闸门的关系
- 本清单到位 = 满足 [[Codex执行前置状态登记表_v0.1]] #6（全量真实数据）的**准备前置**；
- 数据导出后仍需：full dry-run（#8）→ 审阅 → **用户 execute signoff（#9）** → 才允许 execute。
- **本清单本身不触发 execute。**
