# 花厅坊现场决策捕获包 v0.1

> **站货架前直接用的轻量工具。** 目标:抓 1 条真实现场决策,记为 working/episodic,验证捕获路径。**不求完美、不当场升规则、不改 M-DEC-013。**

## 1. 什么时候用
站在货架前,发现一个**商品/陈列/价格/库存/动线**问题,准备做个小决策,但不确定能否成规则 → 掏手机,照下面记。**关键:记下「当下判断」,别等回来事后合理化。**

## 2. 5 秒速记模板（手机备忘录,照抄）
```text
位置：
问题：
判断：
动作：
担心误判：
回填日期：
```
> 5 秒记不全没关系,先记「问题+判断+担心误判」三行,其余回来补。

## 3. 30 秒口述模板（语音/对店长说,照念）
```text
我现在看到的是：____。
它触发的问题是：____（长尾过宽/价格带过密/陈列错位/缺货误判/负毛利清仓 选一）。
我的初始判断是：____。
我准备采取的动作是：____。
我担心的误判是：____。
需要 2-4 周后回填的是：____。
这条先记为 working / episodic，不直接升规则。
```

## 4. 拍照 / 观察 / 访谈记录规则
- **拍照**:问题货架拍 1-2 张(整体+特写);**条码/价签出现就别拍清晰,或回来打码**;
- **观察**:只记「看到什么」,不先下结论(结论进「判断」行);
- **访谈**:问店长/店员一句「这个为什么这样摆/这个卖得怎样」,引述原话一句;
- **不整理**:现场只抓原始料,别现场排版。

## 5. working / episodic / candidate 分流（现场判一下）
| 记成 | 什么情况 |
|---|---|
| **working** | 只是个现象/疑问,还没成判断 → 先存着 |
| **episodic** | 我做了一次完整判断(现象+判断+动作+担心)→ 一条决策 |
| **candidate** | ❌ 现场**不要**直接定 candidate;回来过 triage 再说 |
> 现场只在 working / episodic 之间选;**升 candidate/规则一律回来再判**。

## 6. 回来后贴给 Claude 的整理 Prompt（复制即用）
```text
这是我今天在花厅坊现场捕获的 1 条决策（Phase 0B 捕获包）。请按 M-DEC 长期记忆 Schema v0.1.2 帮我整理成一条 episodic 记录草稿，要求：
1. 套用 30 秒模板字段（现象/触发IssueTag/初始判断/动作/担心误判/回填点）；
2. 标 memory_level=单店决策层、decision_status=draft、result_status=pending；
3. 挂 provenance（来源::），不打分；
4. 判断它是「新 M-DEC 候选」还是「并入 M-DEC-013 的新证据」；
5. 明确：先 draft，不升 active，不改 M-DEC-013 原文；
6. 给我 2-4 周回填提醒。
原始捕获如下：
【把 5 秒/30 秒记录 + 照片说明 + 店长原话 贴这里】
```

## 7. 2-4 周回填提醒模板（到期照填）
```text
回填日期：____（捕获日 +2~4 周）
当时判断：____
实际结果：____
resulting 四象限：
  □ 决策对+果好 → active 正例候选
  □ 决策对+果差 → 果差但决策稳（记外因，不削规则）
  □ 决策错+果好 → 侥幸（不作正例）
  □ 决策错+果差 → failed（削弱/推翻判据）
是否够格升 semantic：____（需跨店/第二客户独立场景，单店不算）
```

## 8. 三条铁律（贴手机壳上）
1. **当下记判断**——防后见之明(Duke 决策日志);
2. **先 working/episodic,不当场升规则**;
3. **失败也记**——failed/侥幸/外因坏果都是资产,不删。
