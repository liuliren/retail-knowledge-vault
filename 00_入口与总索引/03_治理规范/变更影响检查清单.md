# 变更影响检查清单

在修改**核心规则、方法论定义、数据字段、目录结构**等重要内容之前，必须先使用本清单评估变更的影响范围。

---

## 第一步：变更描述（填写后再继续）

> **我要修改的是：** ___________
>
> **修改原因：** ___________
>
> **修改内容（简述）：** ___________

---

## 第二步：影响范围判断

### A. 概念定义变更

若修改的是某个核心概念的定义（如修改"金额管理"的定义）：

- [ ] 是否已查 [[核心概念主定义索引]]，确认该概念的主定义文件？
- [ ] 引用该概念的文件有哪些？（在 Obsidian 中查看反向链接）
- [ ] 哪些文件的表述会因为定义变更而需要同步修改？
- [ ] 是否有已交付给客户的文件使用了旧定义？

### B. 方法论规则变更

若修改的是某个业务规则（如修改 ABCD 分类的维度）：

- [ ] 依赖这个规则的下游文件有哪些？（交付模板、分析模板、系统 PRD、案例文件）
- [ ] 哪些真实项目正在使用这个规则？已交付的报告是否需要说明？
- [ ] 修改后的规则是否与其他规则产生矛盾？

### C. 数据字段变更

若修改的是数据字段定义（如字段字典中的字段名或计算方式）：

- [ ] 哪些 Excel 模板使用了这个字段？
- [ ] 哪些 Python 脚本引用了这个字段？
- [ ] 系统 PRD 中是否有对应字段需要同步修改？
- [ ] 历史数据是否受影响？

### D. 目录结构变更

若修改的是目录结构（如重命名目录）：

- [ ] 有哪些文件通过双链引用了该目录下的文件？
- [ ] README.md 是否需要更新？
- [ ] [[知识库总索引]] 是否需要更新？
- [ ] Obsidian 中的双链是否需要批量更新？

---

## 第三步：变更执行规范

- [ ] 修改前是否已说明修改计划？
- [ ] 是否需要先生成旧版本的备份？（归入 `99_归档`）
- [ ] 修改后是否更新了文件的 `updated` 和 `version` 字段？
- [ ] 修改后是否在 [[版本更新记录]] 中追加了记录？
- [ ] 是否通知了需要同步修改的相关文件？

---

## 第四步：双向同构 5 问（vault ↔ Claude Code）

任何 vault 重要变更，必须答这 5 问。如有"是"，登记到 `[[.claude/_镜像图]]`（Phase 0 D5 落地）。

5 问的本质是：**vault 不再是单向工具，它和 Claude Code 互为镜像**——一边动了另一边必须知道，否则会出现"方法论在 vault、行为基线在 Claude Code、两边各跑各的"。

- [ ] **1. 镜像存在性**：这次变更涉及的方法论 / SOP / 规则，在 Claude Code 侧（`.claude/rules` / `.claude/skills` / `.claude/agents`）有镜像吗？
- [ ] **2. 镜像新建判断**：如果还没有镜像，是否值得现在同步建？（默认否——Phase 0/1 不建，Phase 2 W4-W5 统一建）
- [ ] **3. 镜像同步更新**：如果有镜像，需要同步更新吗？哪一个？谁负责同步？
- [ ] **4. 待确认登记**：这次变更是否引入了新的"待确认"事项？登记到哪份待确认清单（[[版本更新记录]] 本条记录的"待确认事项"段 / 本文件的具体勾选项 / [[当前重点项目看板]]）？
- [ ] **5. client_safety 派生**：这次变更的 [[文档工程化标准]] `client_safety` 等级是什么？是否需要派生客户演示版（`with_disclaimer` / `client_ready`）？

**触发场景示例：**
- 改 [[反幻觉检查清单]] → 必须问 5 问（未来 `.claude/rules/反幻觉.md` 是镜像）
- 改销售包模板 → 必须问 5 问（未来 `.claude/skills/销售对话` 是镜像）
- 改 SOP 套件（10_/00_战役SOP/）→ 必须问 5 问（多个 skill 镜像可能受影响）
- 改 [[文档工程化标准]] frontmatter 字段 → 必须问 5 问（影响所有未来文件）
- 仅修正错字 / 排版 → 不必问 5 问

## 变更级别判断参考

| 变更类型 | 级别 | 处理方式 |
|----------|------|----------|
| 修正错字、补充说明 | 低 | 直接修改，更新 updated 字段 |
| 增加新内容（不影响已有内容）| 低-中 | 直接修改，更新版本号 |
| 修改规则判断标准 | 中-高 | 先检查本清单，再修改，记录版本更新 |
| 修改核心概念定义 | 高 | 必须检查本清单，通知所有下游文件 |
| 删除或重命名文件/目录 | 高 | 必须检查本清单，确认无活跃引用后操作 |
