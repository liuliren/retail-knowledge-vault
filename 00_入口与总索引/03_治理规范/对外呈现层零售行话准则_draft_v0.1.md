# 对外呈现层零售行话准则 · draft v0.1

> **来源**：六哥 2026-07-08 口述核心裁决（原话要义）："系统做分析出来的东西，表达方式太 IT 化、技术流。要用零售行业通用说法、行话去表达，大家听得懂看得明白。这是未来一定要迭代的点，是重要的对外呈现层准则——分析报告、各种情况都适用，除非是内部搞系统开发的场景。"
> **地位**：核心任务，持续推进；含"准则"字样，按标准审议铁律走三轮后升 stable，当前 draft 即开始执行（六哥口述即启用，审议补法理）。

## 一、适用范围
**一切对外呈现层**：客户报告（988/深度/诊断卡）、培训材料、PPT、清单附件、沟通话术、公众号内容、复盘呈现件。
**唯一例外**：内部系统开发场景（脚本/schema/lint/工具文档可保留技术语言）。
**灰区处理**：内部方法论页（01_/04_ 等）正文可保留系统语言，但其"对外表达段/交付模板引用段"须行话化。

## 二、行话对照表 v0.2（15对起步+11对盘点新增·2026-07-08）

| ❌ IT/系统流 | ✅ 零售行话 |
|---|---|
| SKU | 单品 / 支（"这支货"） |
| 动销率 / 零动销 | 动销 / 不动销（"卖得动、卖不动"） |
| ABCZ 分级 | 畅销、平销、滞销分类（ABC 分类可保留——行业本有） |
| 价格带过密 / 断档 | 价格带扎堆 / 缺档（"都挤在十来块钱那一段"） |
| 长尾 SKU | 尾货 / 拖后腿的品 |
| 数据清洗 / 字段 | 数据整理 / 栏目、项 |
| 主数据 / SSOT | 商品档案总表 |
| 负库存 / 账实差 | 账错了 / 账和货对不上 |
| 防误杀闸门 | 先查清楚再砍（"断货不是卖不动"） |
| 复盘回填 / 迭代 | 复盘 / 改进 |
| 证据链 / fact_layer | 依据 / 看到的、听到的、推断的 |
| 假设版 / hypothesis | 初版打法 / 待实战检验 |
| 诊断 | 体检 / 把把脉（对老板）；看货（对采购） |
| M-DEC / 判断链 | 决策依据 / 为什么这么定 |
| 触发条件 / 阈值 | 什么情况下动手 / 警戒线 |
| observed / inferred / pending（英文标签直显） | 看到的数 / 推算的数 / 等您补资料的数 |
| P25 / 中位 / P75 | 便宜那头 / 中间价 / 贵那头 |
| Z类 | 三个月一支没卖动的货 |
| trio / 三件套（对客） | 商品三张表（谁畅销/动不动销/价位分布） |
| 防呆警报 / 匹配率<95%线 | 账和单对不上超一成就先别下结论 |
| ERP / POS(vX) | 进销存系统 / 收银系统 |
| D1-D7 / M1-M8 / 7D / T码 / G4-A 等内部编码 | 对外只用四模块名与品类名直呼（如"饼干类"） |
| 根公式 | 销售=客流×客单价 这条公式 |
| 五分法 | 价位分五档（低/中低/中/中高/高） |
| 证据级 B/C 代号 | （现场看到的）/（数据推算的·待核） |
| 口径（对老板） | 怎么算的 / 按什么算 |

## 三、执行机制
1. **产出前**：对外件起草时对照本表自查；训练/交付材料沿用《培训材料三原则》（具象/黄金圈/实操）
2. **产出后**：/lint-privacy 之外增加"行话检查"步骤（grep 禁词表）——待纳入 /publish、/draft、/report-export 各 skill 的检查清单（skill 修改另行授权）
3. **对照表扩充**：每次被六哥指出"太 IT"的表述，立即加一行对照，本表持续生长
4. **既有交付模板**（T-*-DX 三模板/诊断卡等）：下次修订时逐一过行话化（列入 P2-1 审议时同批）
