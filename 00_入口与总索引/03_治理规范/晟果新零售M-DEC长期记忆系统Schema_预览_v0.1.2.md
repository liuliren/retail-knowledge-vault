## 3. 核心设计原则

### 3.1 provenance 取代 confidence score
- **不打** 0.85 这种假精确置信度；所有结论必须**挂来源**；可信度来自**可见证据链**。

### 3.2 supersession 取代 decay
- 决策**不过期，只被取代**；旧判断**保留**（审计价值）；新判断用 `被取代::` 显式连接 + **取代原因（自由文本，不强制枚举）**。

### 3.3 signoff 作为人工写入闸门
- AI 可生成 draft；进入 active / 跨店规律 / 标准动作**必须 signoff**；无 signoff 不得作正式规则或客户交付依据。
- **拒 dream cycle**：不抄自动富集 / 事件驱动自动写入——与人工闸门冲突，会悄悄污染判断（判断是 10% 护城河）。

### 3.4 两轴分清：层级 ⊥ 状态
> v0.1 把「层级」和「状态」混进单字段 `live`，v0.1.1 拆为**两个正交轴**。

**轴一·四级记忆层（生来固定，不随时间变）**：
| 记忆层 | 对应 | 说明 |
|---|---|---|
| **现场观察层** | working | 当天看到的问题 / 数据 / 现象 |
| **单店决策层** | episodic | 一次完整 M-DEC 判断 |
| **跨店规律层** | semantic | 多店验证后的经营规律 |
| **标准动作层** | procedural | SOP / 模板 / RetailOS 动作 |

**轴二·生命周期状态（会流转·v0.1.2 扩 resulting 四象限）**：
```
draft → active → superseded（被取代）/ failed（决策错+果差）
                 / 果差但决策稳（决策对+果差·外因）/ 侥幸（决策错+果好）/ archived（封存）
```
- 字段：`decision_status`（draft/active/superseded/failed/果差但决策稳/侥幸/archived）。
- **全部保留不删**；回填时按 §8 resulting 四象限定状态。
- **层级靠晋升上移（§7），状态靠生命周期流转——两者互不替代。**

### 3.5 failed 记录是资产
- 决策被证伪 → 标 `decision_status: failed` **永久保留**，进 lint 的「**教训视图**」；
- **严禁删除 / 淡化 failed 记录**——失败也是知识，重犯错误的代价远高于留痕。

### 3.6 schema 承载主纪律，机器后置
- 先用规则 / 模板 / 字段 / signoff 跑通；手动跑过几十轮再自动化；未到规模线**不上向量库 / 图数据库**（连 10 万页的 GBrain 都以 markdown+git 为真相源）。

## 4. 捕获路径（schema 再好，料喂不进就空转）
> **捕获优先**：v0.1.1 补的关键真瓶颈。

1. **现场 5 秒**：看到问题/现象，一句话口述或拍照；
2. **30 秒落 inbox**：当天落 `20_inbox`/现场记录（最小，不求格式）；
3. **周末 triage 三出口**（绑花厅坊周时间块，不另起战场）：
   - **升为决策** → 按 §6 模板补成单店决策层 M-DEC；
   - **并入已有** → 作为已有决策的新证据（provenance 累积，不算重复）；
   - **丢弃记原因** → 不要的也记一句「为什么丢」，避免反复捡起。

| Case 案例 | 项目证据 | project / store / date / output |
| Method KB 方法论 | 跨案例沉淀 | 来源 / 适用条件 / 限制条件 / signoff |

> **IssueTag 先只 5 个**（不做大标签工程）：`长尾过宽 / 价格带过密 / 陈列错位 / 缺货误判 / 负毛利清仓`。

## 6. 命名边 + 命名约定（Obsidian 可用）
**命名边**：
```
来源:: [[案例/门店/数据摘要]]
诊断出:: [[IssueTag]]
触发:: [[决策]]
导致:: [[结果]]
被取代:: [[新决策]]
晋升为:: [[方法论/规则/SOP]]
适用于:: [[业态/品类/场景]]
限制于:: [[适用边界]]
```
**命名约定**（稳定可检索）：
- 决策页：`M-DEC-<编号>_<一句话标题>_<状态/版本>`（编号递增不复用）；
- IssueTag：用上述 5 个固定 code，不自造同义标签；
- 门店/品类：以术语表与主定义索引的规范名为准，不用别名建边。

**去重 / 冲突规则**：
1. 同一现象在不同门店重复出现 = **新证据，不是重复**；
2. 去重按 **provenance 身份**，不按文字相似度；
3. 真冲突进入**待审**，不自动合并；
4. 不要求每条都有全部命名边，但 `来源::` **必须优先**。

## 7. M-DEC 晋升路径（靠证据，不靠时间）
1. 现场观察层 → 单店决策层；
2. 单店决策层 → **回填结果**；
3. 多个独立场景验证 + **排除共因** → 跨店规律层；
4. 跨店规律层 → 标准动作层；
5. 进入 active / 标准动作层**必须 signoff**。

**默认门槛**：**≥3 个独立场景 + 排除共因**——**同供应商 / 同季节 / 同区域 / 同口径不算独立**（防 3 店共因伪规律）。
**例外通道**：强证据 + 六哥 signoff 可提前晋升，例外理由**须溯源**。
**晋升方式**：**上层新建页回指（`晋升为::`），不改写原记录**——原决策保持原样，规律页引用它。
**禁止**：① 仅凭 AI 总结晋升；② 仅凭一次门店判断晋升 active；③ 为凑数制造伪 M-DEC。

## 8. 回填机制（含 resulting 四象限·v3.0 微调一）
1. 没有结果回填的 active 决策 = **孤儿**，lint 每周揪出；
2. 回填周期建议 **2–4 周**；周复盘检查 `result_status: pending`；
3. **回填时按 resulting 四象限定状态**（决策质量 × 结果质量两轴）：

| | 结果好 | 结果差 |
|---|---|---|
| **决策对** | `active` 保留，可作**晋升正例** | `果差但决策稳`：保留，**不削弱规则**，记外因（供应商/季节/竞争）|
| **决策错** | `侥幸`：保留，**严禁作晋升正例**（危险假阳性）| `failed`：削弱/推翻规则，**最值钱教训** |

> **铁律：只有「决策错」才动规则；外因导致的坏结果不动规则。** 防止把被外部冲击拖垮的好判断错当坏规则丢掉（Duke 的 resulting）。
4. **全部保留不删**——失败/侥幸/外因坏果都是知识，进 lint 教训视图。

## 9. M-DEC 最小记录模板（30 秒可口述）
```markdown
## [日期] 决策｜[门店]｜[一句话标题]
- 现象：
- 决策：
- 推理：
- 来源:: [[门店]] / [[IssueTag]] / [[现场记录]]
- 结果：
- 记忆层：单店决策层
- 状态：draft
- 回填日期：
- signoff：
```
配套 frontmatter：
```yaml
title:
version: v0.1
status: draft

```
> **只定义模板，不改现有 M-DEC 文件。**

## 10. supersession 规则
```yaml
decision_status: superseded
superseded_by: "[[新决策文件]]"
superseded_reason: "自由文本（如：价格带变化 / 客群变化 / 数据口径修正 / 竞争变化 / 签字修正）"
superseded_at: 2026-06-22
```
要求：① 不删旧决策；② 不用 decay；③ 不用「过期」模糊处理；④ 旧判断保留审计价值。

## 11. provenance 规则
每条正式结论**至少一个来源**。来源类型：① 门店案例；② 现场观察；③ 脱敏数据摘要；④ 周复盘；⑤ 客户反馈；⑥ 外部研究；⑦ 已签 M-DEC。
禁止：① 无来源 AI 判断写成 active；② 用 confidence score 替代来源；③ 引用真实条码 / 真实敏感数据。

## 12. query（诚实标注未知）+ lint 检查
**query·gap-analysis**（新店诊断前先查库）：返回结果须**诚实标注未知**——最近一次记录何时 / 有无矛盾 / 有无空洞，不假装全知（借 GBrain gap-analysis）。

**lint 检查建议（G03_Lint v2 周度）**：
| 检查项 | 规则 |
|---|---|
| 缺 provenance | active / candidate M-DEC 必须有来源 |
| 缺 signoff | active 必须有 signoff |
| 缺回填 | result_status=pending 超 4 周提醒 |
| 孤儿决策 | 无入链 / 无索引 / 无 MOC |
| superseded 缺新链接 | 标 superseded 但无 superseded_by |
| failed/侥幸 教训视图 | 汇总 failed + 侥幸 + 果差但决策稳 记录供复盘（不删不淡化）|
| 逼近晋升门槛 | 接近 ≥3 独立场景的规律提醒人工裁决 |
| 条码红线 | git tracked 文件真实 EAN-13 = 0 |
| **技术债哨兵（月度）** | ①字段/规则/流程是否在膨胀？②是否在偷花厅坊主线注意力？③维护成本是否 > 复用价值？**任一为是 → 退回更轻形态**（砍字段/降频/回纯 inbox+周回填）|

## 13. 候选池 grandfather + 试运行
**grandfather**：M-DEC 候选池**存量不回填改造**；**新候选起**按本 Schema 走。

**试运行（软目标，不设硬指标·防 Goodhart 造假）**：
- Phase 0B（当前）：用 M-DEC-013 真跑一遍，验字段 / 验捕获路径 / 验 ≥3 独立场景门槛好不好判；
- Phase 1（5–8 月挂花厅坊）：4 周跑通 **3–5 条高质量 M-DEC + ≥1 回填**；**不强求晋升、不设硬条数**；只求**不断流**。

## 14. 禁止事项（不做什么）
1. ❌ schema 不写进 CLAUDE.md（只引用）；2. ❌ 不打 confidence score；3. ❌ 不上 decay/遗忘；4. ❌ 不上向量库/图数据库/自动写入/dream cycle；5. ❌ 不晋升时改写原记录；6. ❌ 不删/不淡化 failed；7. ❌ 不做 IssueTag 大标签工程（先只 5 个）；8. ❌ 不设硬条数指标；9. ❌ 不让知识系统抢花厅坊主线注意力；10. ❌ 不用真实条码入 git。

### 试填样例：M-DEC-013 单小类长尾结构治理候选
- **记忆层**：单店决策层 → 跨店规律层候选
- **现象**：单一小类内 SKU 数偏多但贡献集中度低，长尾 SKU（销额 C 段）占架面比例过高，挤占核心品排面（脱敏：占比为抽象区间）。
- **决策**：对该小类执行长尾结构治理——按双维 ABC + 商品角色闸筛出「双低且非目的品」候选汰换，腾排面给核心引擎与利润品。
- **推理**：长尾 SKU 拉低小类周转与坪效；先过角色闸保护目的品，再过误判 4 问，避免误杀利润品（销额 C+毛利甲）。
- `来源::` [[T03饼干脱敏实跑]] / [[T01膨化脱敏实跑]] / [[T05果干脱敏实跑]]
- **结果**：待后续复盘回填（result_status: pending）
- **记忆层**：单店决策层　**状态**：candidate
- **是否可晋升**：暂不可——3 条均为**花厅坊同店**，属**共因（同店/同期/同口径）**，不算 3 个独立场景；需第二客户/第二门店独立验证。
- `signoff`：未签 active，仅作候选样例。

> 该样例命中多条规则：`来源::` 优先、**晋升门槛未达（共因未排除）**、回填待办、未 signoff 不可作正式规则。
