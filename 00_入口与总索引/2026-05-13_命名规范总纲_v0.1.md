# 2026-05-13_命名规范总纲_v0.1

## 1. 文件定位

本文件是 vault 全库命名规范的**总纲**：定义跨场景命名总原则 + 分隔符总规则 + 日期版本规则 + 各类命名维度的引用入口。

**本文件不替代专项规范**：详细规则查阅各专项规范（参 §12 引用优先级 + [[2026-05-13_命名规范索引_v0.1]]）。

**触发**：W20-Day1 vault v3.0 治理周期 / GOV-NAMING-001 任务 / 命名规则散落多文件 / 收口需要。

## 2. 命名治理总原则

| # | 原则 | 含义 | 反例 |
|---|---|---|---|
| 1 | **统一优先于习惯** | 当个人习惯与统一规则冲突时 / 服从统一规则 / 保留习惯通过 alias 处理 | "我一直用 G3_A" → 不再允许 / 改 G3-A |
| 2 | **可读性与可解析性并重** | 命名既给人看 / 也给 Python / Excel / Claude 解析 / 不能为了好看牺牲机器解析 | `_` 既分隔大字段又用于内部 → 切分不准 |
| 3 | **字段职责分离** | `-` 与 `_` / 大字段与子字段 / 物理编号与逻辑角色 / 各司其职不混用 | 货架编号 G4-A 与拍摄机位编号 M1-M6 混用 |
| 4 | **主定义优先** | 同概念只能有一份主定义 / 其他文件双链引用 / 不复制粘贴 | 6 机位定义同时埋在二访 checklist + 口袋卡 |
| 5 | **不确定字段标记待确认** | 现场或实操中遇到不在锁定值范围内的 / 标 `[待确认]` / 不自创 | "黄金层" / 应是 "黄金视线" 或 "黄金手拿" |
| 6 | **不自创锁定字段** | L 主品类 / 层位语义 / 内容字段 / 时段字段 全部锁定 / 现场不得自创 | "AM" / 应是 "夜灯前 / 次晨白光 / 常规 / 夜灯后" |
| 7 | **项目文件和知识库文件分层命名** | 09_/16_ 客户级文件含客户名 / 01-08_/10_ 跨客户文件不含客户信息 | SOP 主定义住 09_/<门店>/ 子目录 |
| 8 | **文件名版本与内容版本可短期不一致** | in-place 升级时 / 文件名保留 v0.1 / frontmatter 升 v0.5 / 但**必须在正文顶部说明** | W20-Day4 v0.2 / vault v3.0 设计稿 v0.5 |
| 9 | **未确认字段标记 [待确认]** | 不写"模糊词" / 而是显式 [待确认] + 期望确认日 | "约 1.5m 黄金视线" → §9 待确认表 |
| 10 | **临时文件加 _temp_ 前缀** | 临时文件、测试、过渡 / 一律 _temp_ 前缀 / 7 天内整理或删除 | `临时.md` / `test.md` / 应改 `_temp_<主题>_<日期>.md` |
| 11 | **归档文件落 _历史/ 或 99_归档/** | 旧版本不删 / 移至同模块 _历史/ 或全库 99_归档/ / 命名带版本号 | v1.0 / v2.0 / v2.1 / v3.0 同存平铺 |

## 3. 分隔符总规则

| 场景 | 分隔符规则 | 主依据 |
|---|---|---|
| **Markdown 文件名** | 主分隔用 `_` / 子片段可用 `-` / 不否定长期习惯 | [[文件命名规范]] v1.2 §三 |
| **Excel / PDF / 报告文件名** | 同 Markdown | 同上 |
| **门店现场照片命名** | **字段内部用 `-` / 字段之间用 `_`**（4 大字段：位置编码 / 内容 / 日期 / 时段）| [[2026-05-06_门店陈列照片命名规范_v0.3]] §7.2 v0.4 强制 |
| **决策编号（M-DEC / SOP / G）** | 内部优先用 `-`（M-DEC-001 / SOP_6机位拍摄法 / G05）| 散落 / 待 GOV-NAMING-002 主定义 |
| **货架 G 编号** | `G<排号>-<面 A/B>-<NN组>-<M层>-<NN位>` 全部 `-` 内连 | [[2026-05-06_门店陈列照片命名规范_v0.3]] §5.1 |
| **品类 L 字段** | `L<品类名>` / 不带 `-` 或 `_` / 单段 | [[2026-05-06_门店陈列照片命名规范_v0.3]] §4.1 |
| **frontmatter 字段名** | snake_case（`business_module` / `source_type` / `client_safety`）/ 全英文小写 | [[2026-05-12_vault_v3.0顶层结构设计稿_v0.1]] §8 |
| **空格使用** | **关键分隔位禁止用空格** / 仅在表格列文字内允许 | 通用 |

**核心硬约束**（照片命名 / 已固化）：

```text
✅ HS-G3-A-04-3-L冲调-黄金视线_商品价签_20260513_常规.jpg
❌ HS-G3_A-04-3_茶叶_黄金层_冲调茶叶柜眼平_20260513_AM.jpg
```

## 4. 日期与版本规则

### 4.1 日期格式

| 场景 | 格式 | 例 |
|---|---|---|
| Markdown 文件名前缀 | `YYYY-MM-DD_` | `2026-05-13_命名规范总纲_v0.1.md` |
| 照片命名日期字段 | `YYYYMMDD`（无连字符）| `20260513` |
| frontmatter `created` / `updated` / `date` | `YYYY-MM-DD` | `2026-05-13` |
| 报告内文表格 / 章节 | `YYYY-MM-DD` | `2026-05-13` |
| 周编号 | `W<NN>` | `W18 / W19 / W20` |
| Day 编号 | `W<NN>-Day<N>` | `W20-Day4` |

### 4.2 版本号格式

参 [[文件命名规范]] v1.2 §二 + §2.1 三级语义：

| 类型 | 格式 | 含义 |
|---|---|---|
| **major** | v0.x → v0.y / v0 → v1 | 结构性 / 重大内容增加 / **改文件名** |
| **point** | v0.x → v0.x.y | 小补丁 / 增补段 / **就地编辑保留 filename** |
| **minor**（罕用）| v0.x → v0.x.5 | 中等修订 |

### 4.3 v0.x 阶段语义

| 版本 | 阶段 | 用法 |
|---|---|---|
| v0.1 | 草稿 | 初稿 / 仅个人 / 内部讨论 |
| v0.5 | 验证 / 审阅 | 已 review / 待跨场景验证 / 候选稳定 |
| v1.0 | 正式稳定 | 跨场景 / 跨客户 / 跨战役验证后 / 落主库 |
| v1.x | 维护增量 | major / point bumps |

### 4.4 文件名版本与内容版本不一致处理

**强制规则**（已多处实战）：
1. in-place 升级时 / 文件名保留旧版本号（路径稳定 / 双链不破）/ frontmatter version 升新版本号
2. 正文顶部 callout 必须明示：「**当前内容版本：v0.5** / 文件名保留 v0.1 是为了保持路径稳定」
3. §版本记录加新行 / 含变更摘要

**反例**（违规 / 应避免）：
- frontmatter v0.5 / 文件名 _v0.1 / 但正文不写"当前内容版本"→ 用户不知是哪版

**正例**：
- [[2026-05-13_W20-Day4_现场执行口袋卡_v0.1]]（v0.3 内容 / 文件名 v0.1 / 正文顶部明示）
- [[2026-05-12_vault_v3.0顶层结构设计稿_v0.1]]（v0.5 内容 / 文件名 v0.1 / 正文顶部明示）

## 5. 客户 / 门店 / 项目 / 子组命名规则

| 字段 | 规则 | 例 | frontmatter 字段 |
|---|---|---|---|
| 客户名 | 中文全称 / 简称 1:1 不混用 | `乐易购超市` | `client` |
| 门店名 | 中文 / 不带客户前缀 | `花厅坊店` | `store` |
| 战役名 | 中文 / 含战役定位词 | `花厅坊样板`（不写"花厅坊样板战役"）| `project` |
| 子组名 | `子组_<X>` / X 是中文业务模块 | `子组_方便速食` / `子组_冲调` | `business_module` |
| business_module | 中文锁定值（已用：方便速食 / 冲调 / 糕点 / 调味 / 大包装区）/ 不自创 / 新建需 GOV-NAMING-002 沉淀 | `方便速食` | `business_module` |
| 责任人 | 中文姓名 / 单字段 | `佳妮` / `高彬` / `张力` | `responsible_staff` |

**中文使用边界**：
- ✅ 文件名 / 目录名 / frontmatter value 字段 / Markdown 正文 — 中文为主
- ❌ frontmatter 字段名 / 编号字段（M-DEC-005 / W20-Day4 / G3-A）/ 文件扩展名

## 7. 照片命名总规则

详细规则见 [[2026-05-06_门店陈列照片命名规范_v0.3]]（含 v0.4 §7.2 分隔符规则）。本节仅提示核心：

**主公式（8 段）**：

```text
<P>-<G>-<NN组>-<M层>[-<NN位>][-<L主品类>][-<层位语义>]_<内容>_<日期>_<时段>.jpg
```

**标准示例**：

```text
HS-G3-A-04-3-L冲调-黄金视线_商品价签_20260513_常规.jpg
```

**4 大字段拆解**：
| 字段 | 内容 | 内分隔 | 与下一段 |
|---|---|---|---|
| 位置编码 | `HS-G3-A-04-3-L冲调-黄金视线` | `-` | `_` |
| 内容 | `商品价签`（§9 锁定值）| 单段 | `_` |
| 日期 | `20260513` | 单段 | `_` |
| 时段 | `常规`（§10 锁定值）| 单段 | `.jpg` |

**4 类锁定值**：
- P 物理区域 10 代号（RK / HS / SX / JG / SC / TL / SY / TD / SQ / JP）
- L 主品类（L冲调 / L水饮 / L速食 / 等 15+ / 参规范 §4.1）
- 层位语义（黄金视线 / 黄金手拿 / 形象层 / 顶层囤货 / 围货 / 等 / 参规范 §6.3）
- 内容字段 16 锁定值 + 时段字段 4 锁定值（参规范 §9 + §10）

## 10. frontmatter 命名规则


```yaml
status: draft|review|active|deprecated|archived

related: ["[[关联文件]]"]

aliases: [...]
```

- frontmatter 字段名一律 **snake_case 全英文小写**（`business_module` 不是 `BusinessModule`）
- 字段值中文用法见 §5
- `null` 与空字符串区分使用：跨客户 / 跨场景显式写 `null`（非空字符串）

## 11. 双链与 alias 命名规则

| 场景 | 写法 | 何时允许 |
|---|---|---|
| **完整文件名双链** | `[[2026-05-13_命名规范总纲_v0.1]]` | **永远允许** / 默认推荐 |
| **短码双链** | `[[M-DEC-005]]` | **仅当目标文件 frontmatter 含 aliases 字段时允许** |

| **跨目录引用** | 用 basename / 不用相对路径 | Obsidian wiki-link 自动 reroute |
| **显示名变体** | `[[完整名|显示文字]]` | 允许 / 但首选 alias |
