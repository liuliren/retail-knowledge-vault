# 品牌资产手册 v0.1

## 写作前置说明

本手册是 **I 象限**（内核稳固 + 可复制）的品牌总纲，定义"对外讲什么 / 不讲什么 / 怎么讲"的统一口径。

- **架构定位**：7 层架构第 7 层「品牌资产层」（参 [[2026-05-04_知识库重构方案_v0.2_完整作战方案]] §3.1 / §4.7）
- **变更门槛**：最高（I 象限），任何变更必须经核心团队议事 + 走 [[变更影响检查清单]] 顶级条目
- **覆盖范围**：销售包 / 客户报告 / 视频号 / 长文 / 直播 / 学徒招募 / 客户演示 — 一切对外内容均受本手册口径约束
- **本期状态**：v0.1 = **骨架版**。多数内容标"**待六哥定**"，由六哥在 W18-W21 逐步充实。每条决策入库后版本号小升（v0.1.1, v0.1.2, ...），段落全充实后升 v1.0
- **充实节奏**：参 v0.2 §10 优先级判据——先"销售签约 / 客户报告"两段（落地最急），再"视频号 / 学徒招募"两段，再视觉资产
- **本手册不替六哥决策**：LLM 可帮整理结构、检查一致性、收集素材，但**§1.1 / §1.2 / §1.3 品牌内核三段必须由六哥本人填写**

## 4. 内容审核三道关

> **每篇对外内容**（视频号 / 长文 / 直播稿 / 客户报告 / 销售包）发布前**必须**全过下列三道关。任何一关不过，**不准对外发布**。
>
> 这是 v0.2 §4.7 的核心机制——它把"反幻觉 / 客户安全 / 品牌口径"三个独立约束整合成一条审核流水线。

### 第一关：反幻觉关

- **审核位置**：[[反幻觉检查清单]]
- **核心问题**：内容中是否区分了事实 / 推断 / 建议 / 待确认？
- **15 条规则**：参 [[CLAUDE.md]] §13.1-§13.15
- **未过判定**：内容中存在**未标待确认**的虚构 / 推测 / 编造数据 / 未本土化的外部案例

### 第二关：client_safety 关

- **审核位置**：本手册 §4.2 + frontmatter `client_safety` 字段
- **核心问题**：内容是否标 `client_ready`？真实客户名 / 真实数据是否已脱敏？
- **三种 `client_safety` 等级**：
  - `internal_only`：vault 内部使用，不允许对外
  - `client_ready`：可对外发布版本，已脱敏
  - `public`：完全公开（如行业研究 / 公开案例）
- **未过判定**：标 `internal_only` 的内容混进对外发布；或 `client_ready` 但未脱敏

### 第三关：品牌口径关

- **审核位置**：本手册 §2 对外口径统一表
- **核心问题**：本次发布场景对应的"必说 / 必不说"清单，是否命中？
- **检查方式**：发布前用本手册 §2 对应行逐条对照
- **未过判定**：内容中说了"必不说"清单中的事 / 没说"必说"清单中的事

### 4.x 三道关执行 SOP

```
内容定稿
   ↓
第一关：反幻觉（用 [[反幻觉检查清单]] 自查）
   ├─ 未过 → 改 → 回到反幻觉关
   └─ 过 ↓
第二关：client_safety（确认 frontmatter + 脱敏）
   ├─ 未过 → 脱敏 / 派生客户演示版 → 回到 client_safety 关
   └─ 过 ↓
第三关：品牌口径（用本手册 §2 对应行对照）
   ├─ 未过 → 改 → 回到品牌口径关
   └─ 过 ↓
对外发布（视频号 / 客户报告 / 销售包等）
   ↓
回写：在对应场景的 §1.2 接触日志 / 战役档案 / 客户验证日志登记发布事件
```
