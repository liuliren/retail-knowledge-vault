# 标签体系说明

本文件定义知识库的标签分类体系，规范标签的使用方式。在 Obsidian 中，标签是检索和过滤文件的重要工具。

---

## 一、标签使用规则

1. 标签写在 frontmatter 的 `tags` 字段中。
2. 标签使用中文，不加空格。
3. 标签不超过5个/文件（避免过度标注）。
4. 优先使用本文件已定义的标签，避免随意新增。
5. 新增标签时，先在本文件登记。

---

## 二、标签分类体系

### A 类：内容类型标签（必填，每文件1个）

| 标签 | 含义 |
|------|------|
| `#方法论` | 方法论定义或框架文件 |
| `#案例` | 真实门店案例或项目记录 |
| `#模板` | 可复用的结构化模板 |
| `#工具` | 数据分析工具或脚本 |
| `#研究` | 行业研究或外部案例 |
| `#内容` | 自媒体内容素材 |
| `#规范` | 知识库治理规范 |
| `#索引` | 索引或导航文件 |
| `#PRD` | 产品需求文档 |

### B 类：业务领域标签（按需填写）

| 标签 | 含义 |
|------|------|
| `#科学零售` | 科学零售总体方法论 |
| `#商品力` | 商品力相关 |
| `#组织力` | 组织力相关 |
| `#运营力` | 运营力相关 |
| `#生意诊断` | 门店经营诊断 |
| `#商圈分析` | 商圈与客群相关 |
| `#商品诊断` | SKU 诊断与商品力提升 |
| `#品类管理` | 品类管理与规划 |
| `#陈列` | 陈列规划与 DisplayMap |
| `#营销活动` | 活动策划与执行 |
| `#经营分析` | 经营数据与看板 |
| `#数据分析` | 数据分析工具 |
| `#自媒体` | 内容创作相关 |

### D 类：项目标签

| 标签 | 含义 |
|------|------|
| `#乐易购` | 乐易购超市相关 |
| `#乐易购花厅坊店` | 乐易购花厅坊店项目 |

| `#晟果新零售咨询` | 晟果新零售咨询相关 |

### E 类：状态标签

| 标签 | 含义 |
|------|------|
| `#待整理` | 需要进一步整理 |
| `#待确认` | 内容需要核实 |
| `#归档` | 已归档不活跃内容 |
| `#检查清单` | 可操作的检查表 |
| `#版本记录` | 版本变更记录 |

## 三、frontmatter 中的 source_type 字段说明

| 值 | 含义 |
|----|------|
| `method` | 方法论文件 |
| `case` | 案例文件 |
| `template` | 模板文件 |
| `data` | 数据分析或工具 |
| `prd` | 产品需求文档 |
| `content` | 自媒体内容 |
| `research` | 行业研究 |
| `reference` | 索引或参考文件 |
