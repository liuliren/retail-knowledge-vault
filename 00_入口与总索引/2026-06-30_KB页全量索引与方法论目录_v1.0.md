## 一、什么是 KB 页（概念定义）

**KB 页（Knowledge Base Page，知识库页）** 是 `/ingest` 回路从原始资料编译后形成的**互联知识单元**。

与普通笔记的核心区别：

| 普通笔记 | KB 页 |
|---|---|
| 记录"发生了什么" | 编译"这意味着什么，能做什么" |
| 摘要复述原文 | 提炼可直接调用的判断式洞见 |
| 独立存在 | 必须有 wikilink 互联其他页 |
| 没有状态机制 | draft → candidate → stable 单向流转 |
| AI 读完还要再搜索 | AI 读完能直接给花厅坊下一步行动 |

### KB 页三检验铁律（质量底线）

每条命题须同时通过：

1. **直接调用性**：Agent 读完，能立刻给出「花厅坊→下一个具体行动」，无需再读其他资料
2. **场景具体性**：含具体场景（社区生鲜/500-2000㎡/大湾区/具体品类），非通用零售原则
3. **张力存在**：指明在什么条件下失效；不通过张力检验的不是洞见，是常识

### KB 页状态机

```
seed（概念占位）→ draft（内容完整但未验证）→ candidate（自检通过/准备签字）→ stable（六哥签字·3+案例验证）
```

- 升 **stable** 须：六哥签字 + ≥3 案例或 ≥2 客户验证
- 降级/废弃须签字，不得自行 mv 到归档
- **stable 是不可逆确认**，已签字的论点构成方法论底座

### KB 页标准结构

```markdown
---
id: KB-XXX-001
title: 标题
status: draft/candidate/stable
signoff: "六哥 YYYY-MM-DD"（stable 必填）
summary: ≤40字摘要（供廉价首遍检索）
tags: [标签]
related: [wikilink 列表]
---

## 核心论点（结论先行）
## 花厅坊直接调用（必填·AI诊断时优先读这块）
## 失效条件/张力（必填）
## 使用原则
```

## 六、诊断框架与流程完整梳理

### 6.1 顶层诊断体系

```
晟果科学零售诊断总架构（KB-METHOD-CHAIN-001）
├── 五层经营链顺序（禁止跳步）
│   ① 目标顾客清楚？→ ② 需求场景明确？→ ③ 商品配置失衡？
│   → ④ 品类角色错位？→ ⑤ 单品执行跑偏？→ ⑥ 零售数学验证
│
└── 百分评分框架（KB-DX-FRAMEWORK-001）
    M1 生鲜护城河 (30分) ← KB-FRESH-001
    M2 品类结构 (25分)   ← KB-DX-WIDTH-001 + KB-SELECTION-TOOLS-001
    M3 陈列效率 (20分)   ← KB-CAT-ZONE-SOP-001 + KB-CAT-TIMEEFF-001
    M4 活动动销 (15分)   ← KB-METHOD-4TACTICS-001
    M5 运营基础 (10分)   ← KB-DX-MATH-001
```
