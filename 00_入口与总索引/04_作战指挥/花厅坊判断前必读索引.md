## 使用流程

### 流程 A：判断类任务（GO / NO-GO / 分支 / RGY / 是否到位）

```
1. 读 1️⃣ 2️⃣ 3️⃣（事实表 + 数据表 + sign off / 15-25 分钟）
2. 读 4️⃣ 5️⃣（最新推进计划 + 最近订正）
3. 任务输入 vs 1️⃣ 事实表对照 → 如有冲突 confront 用户
4. 任务输入 vs 2️⃣ 数据表对照 → 如有"数据已有"立即引用
5. 不擅自判 ❌ 0 到位 / 必先扫数据资产覆盖表
6. 标 fact_layer：observed / client_told / inferred / pending
7. 输出判定 + 完整证据链
```

### 流程 B：分析类任务（v0.6 复盘 / G05 评审 / SKU 候选 / 排面分析）

```
1. 读 1️⃣ 2️⃣ 3️⃣ 4️⃣ 5️⃣ 6️⃣（事实 + 数据 + sign off + 战役状态 + 订正 + 客户日志 / 30-45 分钟）
2. 列出本任务依赖的 F-XX 事实 + D-XX 数据资产
3. 列出本任务需补的 pending 项
4. 标严格 / 近似 / pending 严格分
5. 输出分析 + 完整证据链 + 不下结论清单
```

### 流程 C：命名 / 照片 / 视觉证据类任务

```
1. 读 1️⃣ 2️⃣（事实 + 数据）+ 7️⃣ 7️⃣.A 7️⃣.B 8️⃣（命名 公司级+门店级+休食区项目级 + 照片 audit）
2. 引用命名 v1.0 规则（公司级 → 门店级 → 项目级 三层）
3. 不擅自改名 / 不批量升级 / 等用户 sign off
4. 不把花厅坊取值（G1-G4 / T01-T08 / XS-A01-A04）写成公司固定值 / 跨门店任务须用公司级标准
```

触发：货架 / 通道 / 端架 / 堆头 / 照片命名 / rename map / 商品 × 陈列位映射 类任务

1. 读 7️⃣.A 公司级通用命名治理规范（方法 / 字段 / 编码 / 16 步流程）

## 反 4 类常见错误

### 错误 1：只扫新增文件就判"数据 0"

❌ "5/19+ 数据目录 0 新文件 / 启明 8 天 0 回传 / 判 0/3 到位"

### 错误 2：只扫文件名就判"事实未 confirm"

❌ "find 文件名 0 命中 / 判 X 未 confirm"

✅ **先查 1️⃣ 事实表**：佳妮 = 休食采购（F-08 / 5/8 + sign off ⑤）/ 启明 = 老板（F-09 / 5/8）

❌ 用户说"洗护区" → 直接接受
❌ 用户说"店长启明" → 直接接受

✅ **vs 1️⃣ 事实表交叉**：F-02 主战役 = 休食区（不是洗护）/ F-09 启明 = 老板（不是店长）→ confront 用户
