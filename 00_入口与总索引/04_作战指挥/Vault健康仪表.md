## 二、各维度释义与采集口径

### 2.1 沉淀 vs 验证比

**定义**：vault 中所有 v0.1+ 方法论 / SOP / 模板文件总数 ÷ 有客户验证日志关联的文件数。

**采集**：每周一手工清点（D3 客户验证日志机制落地后改自动统计）。

**为什么管它**：vault 反"自洽于自己"的核心指标。比值越大，说明"自嗨知识"越多，离客户越远。

**红线动作**：> 10:1 时，本月起冻结新方法论文件，先补客户验证日志。

### 2.2 客户验证度

**定义**：有 `## §X. 实战登记` 段且至少 1 条真实客户记录的 v0.1+ 文件占比。

**采集**：D3 落地后，从客户验证日志反向统计。

**为什么管它**：见 [[CLAUDE.md]] §21 + v0.2 方案 §6.1。

**红线动作**：< 40% 时，新方法论文件必须配套客户验证计划。

### 2.3 客户接触次数（本周）

**定义**：本周（周一 00:00 – 周日 23:59）发生的真实客户接触次数（销售对话 / 首访 / 复盘会 / 老板访谈，每次单独计数）。

**采集**：从客户验证日志（`15_刻意练习与成长/客户验证日志/`）按日期聚合。

**为什么管它**：三回归"回归到人"的客观计数。0 次 = 本周可能在假忙碌。

**红线动作**：连续 2 周 0 接触，触发 [[三回归仪表]] §四 假忙碌扫描。

### 2.4 孤立页

**定义**：无任何双链入 / 出的 .md 文件数（不含 README.md / 索引类）。

**采集**：D4 G03 Lint SOP 占位后用脚本扫描。

**为什么管它**：孤立页 = 无法被检索调用的文件 = 半死的资产。

**红线动作**：> 5 时，下周 Lint 仪式必须处理（合并 / 删除 / 补链）。

### 2.5 待核验项总数

**定义**：vault 全文 `待确认 / 待核验` 标记的累计数（包含 frontmatter 与正文）。

**采集**：grep 全库 `待确认\|待核验` 计数。

**为什么管它**：待确认项是知识债，长期累积会导致"反幻觉"机制失效。

**红线动作**：> 30 时，本月起每周清理 ≥ 5 项；新增不能超过清理。

### 2.6 关键路径偏离度

**定义**：当前活跃战役关键里程碑的实际进度 vs 计划进度的最大偏离天数。

**采集**：从 [[战役指挥板]] §一对照 v0.2 方案 §7 时间表。

**为什么管它**：偏离 > 7 天意味着回款节奏被打破。

**红线动作**：> 7 天时，触发 G05 阶段门评审。

### 2.7 阶段门通过率

**定义**：30 天滚动窗口内，[[G05_阶段门评审]] 「通过 / （通过 + 部分 + 退回）」比率。

**采集**：从战役档案 §05 决策记录（基于 [[16_客户与战役档案/_模板/05_决策记录模板]]）+ G05 SOP 评审记录统计。

**起算日**：2026-05-04（[[G05_阶段门评审]] D4 占位日）；30 天滚动窗口首次成立日 = 2026-06-03。

**为什么管它**：通过率过高 = 阶段门形同虚设；过低 = 战役推进失控。

**红线动作**：< 80% 时，复盘 SOP 是否过严或战役选择失误。

### 2.8 高优风险数

**定义**：当前所有战役登记的红 / 橙级风险（未消解）数。

**采集**：从战役档案 §04 风险登记表（D3 落地后）+ [[战役指挥板]] §一汇总。

**为什么管它**：> 3 时，注意力会被分散。

**红线动作**：> 3 时，必须停下推进、先消解风险或降级。
